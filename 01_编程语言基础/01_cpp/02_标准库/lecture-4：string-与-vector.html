String

基本概念

概念：string 类型

  • 标准库类型 string 表示可变长的字符序列，使用 string 类型必须首先包含 string 头文件；
  • #include <string>

概念：主要操作

    1. 判空：
    • s.empty()
    1. 字符个数：
    • s.size()
    1. 获得第 n 个字符：
    • 注意 n 从 0 开始；
    • s[n]
    1. 连接字符串：
    • s1+s2
    1. 判断相等：
    • s1==s2

概念：getline 操作

  • 代码：
int main()
{
	string line;
	while(getline(std::cin,line))
	{
		std::cout<<line<<std::endl;
	}
	return 0;
}

字符串比较与相加

比较

概念：字符串比较

  • 概念：
    • string 类定义了几种用于比较字符串的运算符，这些比较运算符逐一比较 string 对象中的字符，并且对大小写敏感；
  • 概念：相等
    • 相等性运算符 (== 和 != )分别检验两个 string 对象相等或不相等，string 对象相等意味着它们的长度相同而且所包含的字符也全都相同；
  • 概念：关系比较
    • 关系运算符 <、<=、>、>= 分别检验一个 string 对象是否小于、小于等于、大于、大于等于另外一个 string 对象；
    • 上述这些运算符都依照 (大小写敏感的) 字典顺序；

相加

概念：字符串相加

  • 概念：
    • 当把 string 对象和字符字面值及字符串字面值混在一条语句中使用时，必须确保每个加法运算符 (+) 的两侧的运算对象至少有一个是 string；
  • 举例：
    • string s4 = s1+","; // 正确
    • string s5 = "Hello," + ","; // 错误
  • 分析：使用 左侧对象 + 右侧对象 时，其中必须有一个是 string 对象；
    • string s6 = s1 + "," + "world"; // 正确
    • 之所以这个正确，是因为以上代码可以转化为以下形式：
    • string s6 = (s1 + ",") + "world";
    • 而其中的 (s1 + ",") 是一个 string 对象，因此可以和 "world" 相加；
  • 补充：字符串字面值 != string 对象
    • 比如 "world" 是字符串字面值；
    • 比如 string s1 = "w1"; 中的 s1 是 string 对象；

字符串处理

常用处理方法

概念：string 常用处理方法

    1. tolower(c)：如果字符串 c 是大写字母，则输出对应的小写字母；
    1. toupper(c)：如果字符串 c 是小写字母，则输出对应的大写字母；

范围处理：使用迭代 for

概念：使用 for 处理

  • 概念：
    • 如果想对 string 对象中的每个字符做操作，最好的办法是使用 C++11 新标准提供的一种语句：范围 for 语句；
    • 这种语句遍历给定序列中的每个元素，并对序列中的每个值执行某种操作；
  • 语法：
    • for(declaration : expersion)
      • statement
  • 解释：
    • expression 部分是一个对象，用于表示一个序列；
    • declaration 部分负责定义一个变量，该变量将被用于访问序列中的基础元素；
    • 每次迭代，declaration 部分的变量会被初始化为 expression 部分的下一个元素值；
  • 使用：
    • 需要注意的是，如果使用 for 逐个字符的改变 string 对象的值，要使用对对象的引用；
  • 代码：
string s = "Hello,World!";
for(auto &c : s)
{
	c = topper(c)
}
cout << s << endl;

单个处理：使用下标

概念：单个处理 - 使用下标

  • 概念：
    • 要想访问 string 对象中的单个字符有两种方式；
    • 一种是使用下标，另外一种是使用迭代器；
  • 概念：下标
    • 下标运算符 ([]) 接收的输入参数是 string:: size_type 类型的值，这个参数表示要访问的字符的位置，返回值是该位置上字符的引用；
    • 下表必须大于 0 而且小于 s.size()
    • 标的值称作“下标”或“索引”，任何表达式只要它的值是一个整型值就能作为索引。不过，如果某个索引是带符号类型的值，将自动转换成由 string::sizetype 表达的无符号类型；
  • 注意：使用下表访问之前，先判空
    • 在访问指定字符之前，首先检查 s 是否为空；
    • 其实不管什么时候，只要对 string 对象使用了下标，都要确认在那个位置上确实有值；
    • 如果 s 为空，则 s[0] 的结果将是未定义的；
    • 使用 if(!s.empty())

Vector

基础概念

概念：什么是 Vector

  • 概念：什么是 Vector
    • 标准库类型 vector 表示对象的集合，其中所有对象的类型都相同；
    • 集合中的每个对象都有一个与之对应的索引，索引用于访问对象；
    • 因为 vector 容纳着其他对象，所以它也常被称作容器 (container)；
  • 概念：vector 中的类型
    • vector 能容纳绝大多数类型的对象作为其元素；
    • 但是因为引用不是对象，因此不存在包含引用的 vector；
    • 除此之外，其他大多数 (非引用) 内置类型和类类型都可以构成 vector 对象，甚至组成 vector 的元素也可以是 vector；
  • 使用：
    • #include <vector>
    • using namespace std::vector;
  • 概念：vector 与模板
    • 代码：vector<int> ivec;
    • 代码：vector<vector<string> file;
      • 该容器中存储的元素是 vector 对象，这个 vector 对象中存储的元素是 string 类型；
  • 注意：vector 是模板而非类型
    • vector 是模板而非类型，由 vector 生成的类型必须包含 vector 中元素的类型，例如 vector<int>；

初始化

概念：vector 的初始化

  • 概念：使用列表初始化
    • vector<string> v1 = {"1","2","3"};
    • 如果用的是花括号，可以表述成我们想列表初始化 (list initialize)该 vector 对象；
    • 也就是说，初始化过程会尽可能地把花括号内的值当成是元素初始值的列表来处理，只有在无法执行列表初始化时才会考虑其他初始化方式；
      • 比如：vector<string> v1{10};
      • 在这里面是表示有 10 个默认初始化的元素，而不是表示元素 10，因为这里的类型是 string；
  • 概念：创建指定数量的元素
    • 方式一：指定初始值
      • vector<int> ivec(10,-1);
      • 这个代码表示创建 10 个 int 类型的元素，并且每个都初始化为 -1；
    • 方式二：仅指定数量
      • vector<int> ivec(10);
  • 注意：注意区分不同括号的含义
      1. 圆括号：vector<int> v1(10,1);
      • 表示有 10 个元素，每个初始值都是 1；
      1. 花括号：vector<int> v1{10,1};
      • 表示有 2 个元素，分别是 10 和 1；

添加元素

概念：添加元素

  • 概念：push_back 方法
    • push_back 负责把一个值当成 vector 对象的尾元素压到 (push) vector 对象的尾端 (back)；
  • 代码：
void Lecture1::vectorRun()
{
    string word;
    vector<string> text;
    while (cin >> word)
    {
        text.push_back(word);
    }
}
  • 注意：
      1. 范围 for 语句体内不应改变其所遍历序列的大小；
      1. 不能用下标形添加元素，只能对明确知道已存在的元素执行下表操作；
      • 试图用下标的形式去访问一个不存在的元素将引发错误，不过这种错误不会被编译器发现，而是在运行时产生一个不可预知的值；
      • 不幸的是，这种通过下标访问不存在的元素的行为非常常见，而且会产生很严重的后果；
      • 所谓的缓冲区溢出 (buffer overflow) 指的就是这类错误，这也是导致 PC 及其他设备上应用程序出现安全问题的一个重要原因；
      1. 确保下标合法的一种有效手段，就是尽可能使用范围迭代 for 语句；
