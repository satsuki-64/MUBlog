Lecture 1：变量与基本类型

const 关键词

基础概念

概念：const 关键词

  • 概念：
    • const 必须初始化；
  • 概念：默认状态下，const 对象仅在文件内有效
    • 编译器将在编译过程中把用到该变量的地方都替换成对应的值；
    • 也就是说，编译器会找到代码中所有用到 const 的地方，然后用其值替换；
    • 默认情况下，const 对象被设定为仅在文件内有效。当多个文件中出现了同名的 const 变量时, 其实等同于在不同文件中分别定义了独立的变量；
  • 补充：const 和 define 的区别
    • const 行为和 #define 宏替换有点像，但 const 是类型安全的（其有数据类型检查），而宏是纯文本替换（无类型检查）；

概念：const 的作用域

  • 概念：
    • C++ 中，未加 extern 修饰的 const 变量（默认状态），其作用域被编译器限定为「当前编译单元」（即单个 .cpp 文件，也叫 “翻译单元”），而非整个程序。编译器会默认给这类 const 变量加上 static 特性（注意：是编译器隐式加的，不是你手动写的 static），使其成为 “文件局部常量”，相当于这个 const 变量只属于当前 .cpp 文件，其他文件看不到；
    • 哪怕多个 .cpp 文件里定义了同名、同值的 const 变量（比如都写 const int MAX = 100;），编译器会为每个文件分别创建独立的内存对象（地址不同）。但因为编译器会把所有引用处替换成字面量值，所以表面上用起来 “好像是同一个变量”，且不会触发 “多重定义” 的链接错误；
  • 概念：让 const 在其他文件有效
    • 解决的办法是，对于 const 变量不管是声明还是定义都添加 extern 关键字，这样只需定义一次就可以了；
    • file 1.h 头文件中的声明也由 extern 做了限定，其作用是指明 bufsize 并非本文件所独有，它的定义将在别处出现；

const 与引用

概念：const 的引用

  • 概念：
    • const int& ref = variable; // ref是variable的常量引用
  • 分析：
    • 可以把引用绑定到 const 对象上，就像绑定到其他对象上一样，我们称之为对常量的引用；
    • 与普通引用不同的是，对常量的引用不能被用作修改它所绑定的对象；
      • const int ci = 1024;
      • const int &r1 = ci; // 正确
      • r1 = 42 // 错误
      • int &r2 = ci; // 错误，试图让一个非常量引用指向常量对象
      • 因为不允许直接为 ci 赋值，当然也就不能通过引用去改变 ci；
      • 因此，对 r2 的初始化是错误的，假设该初始化合法，则可以通过 r2 来改变它引用对象的值，这显然是不正确的；
  • 特点：
      1. 不可修改性：通过 const 引用只能读取数据，不能修改原始对象；
      1. 延长临时对象生命周期：const 引用可以绑定到临时对象（如表达式结果或函数返回值），并延长其生命周期至引用作用域结束；
      • const int& ref = 5 + 3; // 合法：临时对象生命周期延长
  • 应用：
      1. 函数参数：避免拷贝，同时防止函数内部修改传入的值；
      • void print(const std::string& str)
      1. 返回临时对象：返回临时对象的引用时，通常用 const 引用接收
      • const std::string& getString()
      • {return "Hello"; // 返回临时对象的const引用}
  • 总结：

const 与指针

概念：const 与指针

  • 概念：
    • 指针是对象而引用不是，因此就像其他对象类型一样，允许把指针本身定为常量；
    • 常量指针 (const pointer) 必须初始化，而且一旦初始化完成，则它的值 (也就是存放在指针中的那个地址)就不能再改变了；
  • 概念：情况一 - *const
    • 把 * 放在 const 关键字之前用以说明指针是一个常量，这样的书写形式隐含着一层意味，即不变的是指针本身的值，而非指向的那个值；
      • int errNumb = 0;
      • int *const curErr = &errNumb; // 指针curErr一直指向errNumb
    • 因为只是限定指针本身指向的地址，而没有限定此地址的值，因此可以修改其内容：
      • *curErr = 0; // 正确
  • 概念：情况二 - const*
    • 指针指向的对象是常量，​不能通过该指针修改对象的值，但指针本身可以指向其他对象；
      • const int* ptr; // ptr是一个指向常量int的指针
    • 可以修改指针的指向（ptr = &new_value），但不能通过指针修改指向的对象（*ptr = 10 会报错）；
  • 概念：情况三 - 双重常量
    • 指针本身和指向的对象都是常量，既不能修改指针的指向，也不能通过指针修改对象的值；
    • const int* const ptr = &a; // 必须初始化
  • 总结：

常量表达式：constexpr

概念：常量表达式 - constexpr

  • 引入：
    • 在一个复杂系统中，很难分辨一个初始值到底是不是常量表达式，当然可以定义一个 const 变量并把它的初始值设为我们认为的某个常量表达式，但在实际使用时，尽管要求如此却常常发现初始值并非常量表达式的情况；
    • 可以这么说，在此种情况下，对象的定义和使用根本就是两回事儿；
  • 概念：
    • C++11 新标准规定，允许将变量声明为 constexpr 类型以便由编译器来验证变量的值是否是一个常量表达式；
    • 声明为 constexpr 的变量一定是一个常量，而且必须用常量表达式初始化；
    • constexpr int mf = 20; // 20是常量表达式
    • constexpr int limit = mf + 1; // mf+1是常量表达式
    • constexpr int sz = size(); //当size是一个constexpr函数时才是正确的声明

类型处理

类型别名

概念：类型别名

  • 概念：
    • 类型别名 (typealias) 是一个名字，它是某种类型的同义词；
    • 使用类型别名有很多好处，它让复杂的类型名字变得简单明了、易于理解和使用，还有助于程序员清楚地知道使用该类型的真实目的；
  • 方法：使用 typedef
    • typedef double wages;
    • typedef wages base,*p;
    • 其中含有 typedef 的声明语句定义的不再是变量，而是类型别名；
  • 举例：
typedef struct ListNode 
{
    // 数据域：存储结点数据（示例用int，可替换为string/自定义类型）
    int data;
    
    // 指针域：指向“下一个结点”的指针（类型为struct ListNode*，别名后可写Node*）
    struct ListNode* next; 

    // C++结构体构造函数（初始化结点，简化创建）
    ListNode(int val) : data(val), next(nullptr) {}
} Node; // 别名Node：此后Node等价于struct ListNode
  • 方法：使用 using
    • 使用别名声明 (alias declaration) 来定义类型的别名：
    • using SI = Sales_item
    • SI item; // 等价于 Sales_tiem item

auto 关键字

概念：auto 关键字

  • 概念：
    • auto 关键字让编译器替我们去分析表达式所属的类型；
    • 和原来那些只对应一种特定类型的说明符 (比如 double) 不同，auto 让编译器通过初始值来推算变量的类型，因此 auto 定义的变量必须有初始值；
  • 使用：
    • auto item = val1 + val2; // item初始化为 val1和 val2 相加的结果

概念：auto 和引用类型

  • 概念：使用引用类型
    • 使用引用其实是使用引用的对象，特别是当引用被用作初始值时, 真正参与初始化的其实是引用对象的值；
    • 因此此时编译器以引用对象的类型作为 auto 的类型；
    • 比如：int i =0,&r = i;
    • 然后：auto a = r; // a 是一个整型
  • 概念：声明引用类型的 auto
    • 情况一：auto &g=ci; // g 是一个整型常量引用，绑定到 ci
    • 情况二：auto &h=42; // 错误: 不能为非常量引用绑定字面值
    • 情况三：const auto &j= 42; // 正确: 可以为常量引用绑定字面值

概念：auto 和 const

    1. auto 一般会忽略掉顶层 const，同时底层 const 则会保留下来；
    1. 如果希望推断出的 auto 类型是一个顶层 const，需要明确指出 const auto f = ci;
    1. 要在一条语句中定义多个变量，切记，符号 & 和 * 只从属于某个声明符，而非基本数据类型的一部分，因此初始值必须是同一种类型；

总结

decltype

概念：decltype 关键词

  • 概念：
    • decltype 是 C++11 引入的关键字，用于推导表达式的类型，但不会计算表达式的值；
    • 它主要用于类型推断，特别是在模板编程和复杂类型声明中非常有用；
  • 举例：
    • decltype(f()) sum = x;
    • 在这行代码当中使用 f() 的结果的类型来定义 sum 变量，但是并不实际的调用并执行 f()；
  • 概念：decltype 与引用
    • decltype 的结果可以是引用类型：
    • 比如假设 int i = 42,&r = i;
    • 则当执行 decltype(*p) c 时将会报错，因为 c 时引用类型 int&，因此必须初始化；
  • 总结：区别
  • 注意：
    • decltype((variable)) (注意是双层括号) 的结果永远是引用，而 decltype(variable) 结果只有当 variable 本身就是一个引用时才是引用；

基本类型与指针

int* 与 int

概念：int* 与 int

  • 概念：
    • 32 位系统：int（4 字节）、int*（4 字节，指针大小等于系统寻址位数）
    • 64 位系统：int（仍为 4 字节，C++ 标准仅规定 int 至少 16 位，主流平台保持 4 字节）、int*（8 字节，对应 64 位寻址）
    • int* 的价值都不在于 “占用多少字节”，而在于它的指针语义，即指向内存地址、间接操作数据的能力；
    • 并且 C++ 中用 new /C 语言用 malloc 分配动态内存时，返回的必然是指针（如 int*），因为动态内存的地址需要被记录和管理，int 无法存储内存地址；
  • 代码：
#include <iostream>
using namespace std;

int main() {
    // 动态分配一个int，返回其地址（必须用int*接收）
    int* dynamicInt = new int(20);
    cout << *dynamicInt << endl; // 输出20（解引用访问动态内存）
    
    // 动态分配int数组（只能用int*管理）
    int* arr = new int[5]{1,2,3,4,5};
    for (int i=0; i<5; i++) {
        cout << arr[i] << " "; // arr[i] 等价于 *(arr+i)，通过指针访问数组
    } // 输出1 2 3 4 5
    
    // 释放动态内存（必须用指针）
    delete dynamicInt;
    delete[] arr;
    return 0;
}

int* a 与 int a[] 的区别

概念：int* a 与 int a[] 的区别

  • 概念：
    • 这两个数组声明的核心差异，本质是静态栈数组和动态堆数组的区别：指针符号的有无，正是由它们的内存分配方式、类型本质决定的；
  • 区别：
维度 静态数组 int a[] = {1,2,3,4,5} 动态数组 int* arr = new int[]{1,2,3,4,5}
内存分配位置 栈（stack） 堆（heap）
生命周期 随作用域结束自动释放（如函数执行完） 手动调用 delete[] arr 才释放，否则内存泄漏
大小信息 编译器可知（sizeof(a)=20 字节） 编译器不可知（sizeof(arr)=4/8 字节，仅指针大小）
类型本质 数组类型 int[5]（常量指针） 指针类型 int*（普通指针）
可修改指向 数组名a是常量指针，不能重指向 指针arr可重指向其他内存（如arr = new int[10]）
初始化限制 编译期必须确定大小（字面量初始化） 运行期可动态确定大小（如 new int[n]，n 是变量）
  • 分析：
    • 动态数组返回的是 int* 指针，编译器只知道 “这是一个指向 int 的指针”，不知道指向的内存有多少个元素，所以 sizeof(arr) 只能得到指针本身的大小；
    • int a[] 告诉编译器 “我要在栈上分配一块连续的内存，存储 5 个 int，变量名是 a”，数组名 a 的类型是 int[5]（数组类型），但编译器会隐式将 a 当作 const int*（常量指针） 使用，比如 a[0] 等价于 *(a+0)；

new Test 和 new Test() 区别

初始化方式 new Test（默认初始化） new Test()（值初始化）
核心逻辑 编译器按 “类型特性” 做最少初始化 编译器强制做 “零 / 合法初始化”
对有自定义默认构造函数的类 调用自定义默认构造函数（和值初始化效果一致） 调用自定义默认构造函数（和默认初始化效果一致）
对无自定义默认构造函数的类 内置类型成员（如 int）未初始化（随机值） 内置类型成员强制零初始化（值为 0）
对内置类型（如 int） new int → 内存未初始化（随机值） new int() → 内存初始化为 0
