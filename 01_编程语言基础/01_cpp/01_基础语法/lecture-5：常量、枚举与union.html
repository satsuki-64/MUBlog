Lecture 5：常量、枚举与union

常量与枚举

const 和 enum 基础概念

概念：const 和 enum 基础概念

  • 概念：const
    • 核心
      • 在编译器的视角下，define 不被视为 C++ 语言的一部分，因此可能存在在编译器的观测角度，代码的行为与编译器行为不一致的情况，此时可以改用 const 来替代宏；
    • 概念：两类特殊使用情况
      • 常量指针
        • 在编写常量指针时，有必要编写两个 const：const char* const name = "Mike"
      • class 专属常量
        • 创建专属于某一个 class 的常量：static const int Number = 5;
  • 概念：enum
    • 核心
      • 一个属于枚举类型的数值可以权充 ints 使用（enum hack）
    • 举例
      • enum { NumTurns = 5};
      • int scores[NumTurns];
    • 分析
        1. enum 的行为更类似于 define 而不是 const，如果这是你想要的；
        1. 对 enum 取地址是不合法的，而对 const 取地址是合法的。而取一个 define 的地址也不合法；
        1. enum 和 define 一样，不会非必要的内存分配；

const 的使用

分析：尽可能使用 const

  • 概念：
      1. 将某些东西声明为 const 可帮助编译器侦测出错误用法，const 可被施加于任何作用域内的对象、函数参数、函数返回类型、成员函数本体；
      1. 编译器强制实施 bitwise constness ，但你在编写程序时应该使用 conceptual consrness (概念上的常量性)；
      1. 当 const 和 non-const 成员函数有着实质上等价的实现时，令 non-const 版本调用 const 可以减少代码重复；
  • 使用：
    • const 指针
      • 被指示物常量
        • MyClass const* myClass;
        • 如果 const 出现在星号左边，表示被指示物是常量；
      • 指针常量
        • MyClass *const myClass;
        • 如果 const 出现在星号右边，表示指针自身为常量；
      • 总结：
        • 使用 const 时，* 在后，不能改值，* 在前，不能改指针；
    • const 函数
      • bitwise constness
        • 成员函数只有在不更改对象的任何成员变量（static 除外）时才可以说是 const 的；
        • 即不更改对象内的任何一个 bit；
        • 问题：可以通过在外部重新声明指针指向此值，修改此内容；
      • logical contness
        • 一个 const 成员函数可以修改它所处理的对象内的某些 bits，但只有在客户端无法侦测出的情况下；
    • const 转型
      • 就一般的守则而言，转型是一个糟糕的想法；
      • 但是允许 non-const 版本调用 const 可以有效减少代码重复；
      • 相反的，不应该使用 non-const 版本调用 const 内容；

union

union 基础概念

概念：union

  • 概念：
    • union 是一个用户定义类型，其中所有成员都共享同一个内存位置；
    • 此定义意味着在任何给定时间，union 都不能包含来自其成员列表的多个对象。这还意味着，无论 union 具有多少成员，它始终仅使用足以存储最大成员的内存；
    • 如果具有大量对象但内存有限，union 可用于节省内存；
    • 但是，需要格外小心才能正确使用 union。你负责确保始终访问分配的同一成员，如果任何成员类型具有不常用构造函数，则必须编写代码来显式构造和销毁该成员。使用 union 之前，应考虑是否可以使用基 class 和派生的 class 类型来更好地表示尝试解决的问题；
  • 语法：
    • union tag opt{member-list};
  • 示例：
// declaring_a_union.cpp
union RecordType  
{
    char   ch;
    int    i;
    long   l;
    float  f;
    double d;
    int *int_ptr;
};

int main()
{
    RecordType t;
    t.i = 5; // t holds an int
    t.f = 7.25; // t now holds a float
}

union 的使用

概念：使用枚举区分 union 中类型

  • 概念：
    • 任何访问 union 的代码都需要了解保存数据的成员。解决此问题的最常见解决方案被称为 “可区分 union”；
    • 它将 union 括在 struct 中，还加入一个 enum 成员，用于指示当前存储在 union 中的成员类型；
  • 代码：
#include <queue>

using namespace std;

enum class WeatherDataType
{
    Temperature, Wind
};

struct TempData
{
    int StationId;
    time_t time;
    double current;
    double max;
    double min;
};

struct WindData
{
    int StationId;
    time_t time;
    int speed;
    short direction;
};

// 使用一个枚举来区分当前Union存储的类型
struct Input
{
    WeatherDataType type;
    union
    {
        TempData temp;
        WindData wind;
    };
};

// 给定参数类型的函数
void Process_Temp(TempData t) {}
void Process_Wind(WindData w) {}

void Initialize(std::queue<Input>& inputs)
{
    Input first;
    first.type = WeatherDataType::Temperature;
    first.temp = { 101, 1418855664, 91.8, 108.5, 67.2 };
    inputs.push(first);

    Input second;
    second.type = WeatherDataType::Wind;
    second.wind = { 204, 1418859354, 14, 27 };
    inputs.push(second);
}

// 使用演示
int main(int argc, char* argv[])
{
    // Container for all the data records
    queue<Input> inputs;
    Initialize(inputs);
    
    while (!inputs.empty())
    {
        Input const i = inputs.front();
        
        // 利用enum来得到当前Union的类型，传入switch、进行不同的处理
        // 对存储在Union的不同类型结构体进行不同处理
        switch (i.type)
        {
        case WeatherDataType::Temperature:
            Process_Temp(i.temp);
            break;
        case WeatherDataType::Wind:
            Process_Wind(i.wind);
            break;
        default:
            break;
        }
        inputs.pop();

    }
    return 0;
}
