Lecture 7：智能指针

为什么需要智能指针

概念：为什么需要智能指针

  • 智能指针就是帮我们 C++ 程序员管理动态分配的内存的，它会帮助我们自动释放 new 出来的内存，从而避免内存泄漏；
  • 比如 new 了一个字符串指针，但是没有 delete 就已经 return 结束函数了，导致内存没有被释放，内存泄露；
  • 使用指针，我们没有释放，就会造成内存泄露。但是我们使用普通的对象却不会；
  • 因此，如果我们分配的动态内存都交由有生命周期的对象来处理，那么在对象过期时，让它的析构函数删除指向的内存就可以避免上述的内存泄露；

auto_ptr

auto_ptr 基础概念

概念：什么是 auto_ptr

  • 概念：
    • auto_ptr 是 c++ 定义的智能指针模板，其定义了管理指针的对象，可以将 new 获得（直接或间接）的地址赋给这种对象；
    • 当对象过期时，其析构函数将使用 delete 来释放内存；
    • 智能指针重载了 * 和 -> 运算符， * 返回普通对象，而 -> 返回指针对象；
  • 用法：
    • 引用头文件：#include <memory>
    • 使用：auto_ptr<类型> 变量名(new 类型)
  • 注意：
    • C++11 之后已经废弃了 auto_ptr，转而使用 unique_ptr 代替；
  • 示例：
class Test {
public:
	Test() { cout << "Test的构造函数..." << endl; }
	~Test() { cout << "Test的析构函数..." << endl; }

	int getDebug() { return this->debug; }

private:
	int debug = 20;
};

int main(void) 
{
	// 定义智能指针
	auto_ptr<Test> test(new Test);

	// 使用智能指针，和普通的指针一样
	cout << "test->debug：" << test->getDebug() << endl;
	cout << "(*test).debug：" << (*test).getDebug() << endl;

	return 0;
}

auto_ptr 常用方法

方法：get()

  • 作用：
    • 获取智能指针托管的指针地址；
    • 但我们一般不会这样使用，因为都可以直接使用智能指针去操作；
  • 示例：
// 定义智能指针
auto_ptr<Test> test(new Test);

Test *tmp = test.get();		// 获取指针返回
cout << "tmp->debug：" << tmp->getDebug() << endl;

方法：release()

  • 作用：
    • 取消智能指针对动态内存的托管；
  • 示例：
// 定义智能指针
auto_ptr<Test> test(new Test);

Test *tmp2 = test.release();	// 取消智能指针对动态内存的托管
delete tmp2;	// 之前分配的内存需要自己手动释放

方法：reset()

  • 作用：
    • 重置智能指针托管的内存地址，如果地址不一致，原来的会被析构掉；
  • 示例：
// 定义智能指针
auto_ptr<Test> test(new Test);

test.reset();			// 释放掉智能指针托管的指针内存，并将其置NULL

test.reset(new Test());	// 释放掉智能指针托管的指针内存，并将参数指针取代之

auto_ptr 的问题

概念：auto_ptr 指针复制或者赋值都会改变资源的所有权

// auto_ptr 被C++11抛弃的主要原因
auto_ptr<string> p1(new string("I'm Li Ming!"));
auto_ptr<string> p2(new string("I'm age 22."));

cout << "p1：" << p1.get() << endl;
cout << "p2：" << p2.get() << endl;

// p2赋值给p1后，首先p1会先将自己原先托管的指针释放掉，然后接收托管p2所托管的指针，
// 然后p2所托管的指针制NULL，也就是p1托管了p2托管的指针，而p2放弃了托管。
p1 = p2;	
cout << "p1 = p2 赋值后：" << endl;
cout << "p1：" << p1.get() << endl;
cout << "p2：" << p2.get() << endl;

unique_ptr

unique_ptr 基础概念

概念：unique_ptr 基础概念

  • 引入：
    • auto_ptr 是用于 C++11 之前的智能指针，由于 auto_ptr 基于排他所有权模式，两个指针不能指向同一个资源，复制或赋值都会改变资源的所有权，auto_ptr 主要有三大问题：
    • （1）复制和赋值会改变资源的所有权，不符合人的直觉；
    • （2）在 STL 容器中使用 auto_ptr 存在重大风险，因为容器内的元素必需支持可复制（copy constructable）和可赋值（assignable）；
    • （3）不支持对象数组的操作；
    • 因此需要使用 unique_ptr 来解决上述问题；
  • 概念：
    • C++11 用更严谨的 unique_ptr 取代了 auto_ptr；
  • 特点：
      1. 基于排他所有权模式：两个指针不能指向同一个资源；
      1. 无法进行左值 unique_ptr 赋值构造，也无法进行左值复制赋值操作，但允许临时右值赋值构造和赋值；
      1. 保存指向某个对象的指针，当它本身离开作用域时会自动释放它指向的对象；
      1. 在容器中保存指针是安全的；
  • 示例：
unique_ptr<string> p1(new string("I'm Li Ming!"));
unique_ptr<string> p2(new string("I'm age 22."));
	
cout << "p1：" << p1.get() << endl;
cout << "p2：" << p2.get() << endl;

// 错误：p1 = p2;					// 禁止左值赋值
// 错误：unique_ptr<string> p3(p2);	// 禁止左值赋值构造

unique_ptr<string> p3(std::move(p1)); // 使用p1生成其右值，传入p3的构造函数中
p1 = std::move(p2);	// 使用move把左值转成右值就可以赋值了，效果和auto_ptr赋值一样

cout << "p1 = p2 赋值后：错误" << endl;
cout << "p1：" << p1.get() << endl;
cout << "p2：" << p2.get() << endl;

// 会自动调用 delete[] 函数去释放内存
unique_ptr<int[]> array(new int[5]);	// 支持这样定义，即允许临时右值赋值构造

unique_ptr 的问题

概念：auto_ptr 与 unique_ptr 智能指针的内存管理陷阱

  • 分析：以下问题是由于 auto_ptr 与 unique_ptr 的排他性所导致的；
auto_ptr<string> p1;
string *str = new string("智能指针的内存管理陷阱");
p1.reset(str);	// p1托管str指针
{
	auto_ptr<string> p2;
	p2.reset(str);	// p2接管str指针时，会先取消p1的托管，然后再对str的托管
}

// 此时p1已经没有托管内容指针了，为NULL，在使用它就会内存报错！
cout << "str：" << *p1 << endl;
  • 分析：独占所有权导致无法拷贝，仅支持移动
    • unique_ptr 的核心设计是 “独占对象所有权”，因此它禁用了拷贝构造和拷贝赋值运算符，只能通过 std::move 转移所有权。这是设计特性，但也导致它无法用于需要 “拷贝” 的场景，新手易因误拷贝触发编译错误；
    • 比如：下面代码当中的 std::move(p1) 只是 “标记” p1 的所有权可转移，转移后 p1 变为 “空指针”，若误操作 p1 会触发未定义行为；
    • 因此，无法将 unique_ptr 直接存入需要拷贝的容器 / 函数参数，必须显式移动，增加了使用成本；
  • 示例：
#include <iostream>
#include <memory>
using namespace std;

class Test {
public:
    Test() { cout << "Test构造" << endl; }
    ~Test() { cout << "Test析构" << endl; }
};

int main() {
    unique_ptr<Test> p1(new Test);
    
    // 错误1：直接拷贝unique_ptr（编译失败）
    unique_ptr<Test> p2 = p1; // 编译报错：拷贝构造函数被删除
    
    // 错误2：函数参数传递时的隐式拷贝
    auto func = [](unique_ptr<Test> p) {};
    func(p1); // 编译报错：无法拷贝unique_ptr
    
    // 正确：必须用std::move转移所有权
    unique_ptr<Test> p3 = move(p1); 
    func(move(p3)); // 转移后p3失效
    
    // 注意：p1已失效，解引用会导致未定义行为
    // cout << p1.get() << endl; // 输出nullptr（多数编译器），但操作p1指向的对象会崩溃
    return 0;
}
  • 分析：指向数组的语法陷阱
    • unique_ptr 对 “单个对象” 和 “数组” 的处理是分离的：
    • （1）unique_ptr<T>：析构时调用 delete（针对单个对象）
    • （2）unique_ptr<T[]>：析构时调用 delete[]（针对数组）
    • ，用 unique_ptr<T> 指向数组，导致析构时仅释放第一个元素，引发内存泄漏；
  • 示例：
#include <iostream>
#include <memory>
using namespace std;

int main() {
    // 错误写法：用unique_ptr<int>指向数组（析构调用delete，而非delete[]）
    unique_ptr<int> arr_err(new int[5]{1,2,3,4,5}); 
    // 析构时仅释放第一个int，剩余4个int内存泄漏（未定义行为）
    
    // 正确写法：用unique_ptr<int[]>指向数组
    unique_ptr<int[]> arr_ok(new int[5]{1,2,3,4,5});
    // 析构时调用delete[]，正确释放整个数组
    
    return 0;
}

shared_ptr

shared_ptr 基础概念

概念：shared_ptr 基础概念

  • 原理：
    • 记录引用特定内存对象的智能指针数量，当复制或拷贝时，引用计数加 1，当智能指针析构时，引用计数减 1，如果计数为零，代表已经没有指针指向这块内存，那么我们就释放它。这就是 shared_ptr 采用的策略；
  • 实现：
    • 每个 shared_ptr<T> 对象本身只包含两个指针，64 位系统下共 16 字节，32 位下 8 字节；
    • （1）T* ptr：指向被管理的实际对象（裸指针）
    • （2）ControlBlock* cb：指向控制块（核心，存储引用计数、删除器等）
    • shared_ptr 本身不存储引用计数，所有共享同一对象的 shared_ptr，其 cb 指针都指向同一个控制块，这是实现 “共享计数” 的核心；
  • 概念：控制块
    • 控制块是 shared_ptr 底层最核心的结构，由编译器自动创建，存储所有管理对象所需的元数据；
    • （1）std::size_t use_count 强引用计数：记录当前有多少 shared_ptr 指向该对象（独占所有权的计数）
    • （2）std::size_t weak_count 弱引用计数：记录当前有多少 weak_ptr 指向该控制块（仅观察，不持有所有权）
    • （3）T* object_ptr：指向被管理对象的裸指针（与 shared_ptr 的 ptr 一致，或为 nullptr）
    • （4）Deleter deleter 自定义删除器：默认是 delete/delete[]，支持用户自定义销毁逻辑；
    • （5）Allocator allocator 内存分配器：用于释放控制块自身内存；
    • 控制块的类型是 “类型擦除” 的，无论删除器 / 分配器是什么类型，控制块都能统一管理，因此 shared_ptr<void> 也能正确释放任意类型的对象（删除器知道具体类型）
  • 示例： 
class Person {
public:
	Person(int v) {
		this->no = v;
		cout << "构造函数 \t no = " << this->no << endl;
	}

	~Person() {
		cout << "析构函数 \t no = " << this->no << endl;
	}

private:
	int no;
};

// 仿函数，内存删除
class DestructPerson {
public:
	void operator() (Person *pt) {
		cout << "DestructPerson..." << endl;
		delete pt;
	}
};
  • 注意：
    • 绝对不要用同一个裸指针创建多个独立的 shared_ptr，会创建多个控制块，导致重复释放；
    • Test* raw = new Test();
shared_ptr<Test> p1(raw); // 创建控制块1，use_count=1
shared_ptr<Test> p2(raw); // 创建控制块2，use_count=1 → 析构时p1、p2各自释放raw，触发double free崩溃

shared_ptr 的使用

概念：引用计数

  • 概念：
    • 调用 use_count() 函数可以获得当前托管指针的引用计数；
  • 构造：
  • 注意：
    • （1）使用 shared_ptr 时，一定要注意是否有循环引用。使用 shared_ptr 智能指针时，要注意避免对象交叉使用智能指针的情况，否则会导致内存泄露；
    • （2）可以使用 weak_ptr 来避免上述问题；
  • 示例：
shared_ptr<Person> sp1;

shared_ptr<Person> sp2(new Person(2));

// 获取智能指针管控的共享指针的数量	use_count()：引用计数
cout << "sp1	use_count() = " << sp1.use_count() << endl;
cout << "sp2	use_count() = " << sp2.use_count() << endl << endl;

// 共享
sp1 = sp2;

cout << "sp1	use_count() = " << sp1.use_count() << endl; // 2
cout << "sp2	use_count() = " << sp2.use_count() << endl << endl; // 2

shared_ptr<Person> sp3(sp1);
cout << "sp1	use_count() = " << sp1.use_count() << endl; // 3
cout << "sp2	use_count() = " << sp2.use_count() << endl; // 3
cout << "sp2	use_count() = " << sp3.use_count() << endl << endl;  // 3
  • 分析：
    • sp1 = sp2; ->sp1 和 sp2 共同托管同一个指针，所以他们的引用计数为 2；
    • shared_ptr< Person > sp3(sp1); -> sp1 和 sp2 和 sp3 共同托管同一个指针，所以他们的引用计数为 3；

weak_ptr

weak_ptr 基础概念

概念：weak_ptr 基础概念

  • 概念：
    • std::weak_ptr 是 C++11 引入的弱引用智能指针，它并非独立的所有权管理者，而是 std::shared_ptr 的 “辅助工具”，核心作用是观察 shared_ptr 管理的对象，却不持有对象的所有权，因此不会影响对象的生命周期；
    • 其设计初衷是解决 shared_ptr 最典型的问题：循环引用导致的内存泄漏；
  • 区别：
特性 shared_ptr weak_ptr
所有权 持有对象所有权（强引用） 无所有权（弱引用）
影响对象生命周期 递增 / 递减强引用计数，计数为 0 时释放对象 递增 / 递减弱引用计数，不影响对象释放
直接解引用 支持（*p/p->） 不支持（必须先转换为shared_ptr）
核心目标 安全共享对象所有权 观察对象，解决循环引用
  • 分析：
    • weak_ptr 完全依赖 shared_ptr 的控制块（Control Block） 实现功能，自身无独立的内存管理；
    • 每个 weak_ptr<T> 对象仅包含一个指针：
    • weak_ptr<T> { ControlBlock* cb; // 指向 shared_ptr 的控制块（与共享同一对象的 shared_ptr 指向同一个控制块） };
    • 所有观察同一对象的 weak_ptr，其 cb 指针都指向同一个控制块；
    • 因此，weak_ptr 仅操作控制块的 weak_count，不碰 use_count；
  • 补充：
    • 控制块的释放条件：仅当 use_count == 0（对象已释放）且 weak_count == 0（无 weak_ptr 观察）时，才释放控制块内存；

weak_ptr 的使用

概念：weak_ptr 的使用

  • 概念：
    • weak_ptr 提供的成员函数不多，核心围绕 “检查对象存活” 和 “安全获取强引用” 展开；
  • 方法：lock()
    • 尝试获取一个指向对象的 shared_ptr（强引用），是唯一能安全操作对象的方式；
  • 语法：
std::shared_ptr<T> sp = wp.lock();
if (sp) { // 先判断是否非空
    sp->func(); // 安全操作对象
}
  • 方法：expired() 检查对象是否已销毁
if (wp.expired()) {
    cout << "对象已销毁" << endl;
} else {
    auto sp = wp.lock(); // 再获取强引用
}

概念：解决 shared_ptr 循环引用

  • 分析：
    • shared_ptr 双向引用时，强引用计数会 “互锁”，导致计数永远无法减到 0，对象无法释放（内存泄漏）
    • weak_ptr 可以替换一侧的 shared_ptr，仅保留弱引用，打破循环；
  • 示例：循环引用的问题
#include <iostream>
#include <memory>
using namespace std;

struct Node {
    shared_ptr<Node> next; // 强引用指向另一个节点
    ~Node() { cout << "Node析构" << endl; } // 永远不会执行！
};

int main() 
{
    shared_ptr<Node> n1(new Node);
    shared_ptr<Node> n2(new Node);
    
    n1->next = n2; // n2的use_count=2
    n2->next = n1; // n1的use_count=2
    
    // 函数结束时，n1/n2析构：n1的use_count=1，n2的use_count=1
    // 强引用计数未到0，对象无法释放 → 内存泄漏
    return 0;
}
  • 示例：
#include <iostream>
#include <memory>
using namespace std;

struct Node {
    weak_ptr<Node> next; // 弱引用，不增加强引用计数
    ~Node() { cout << "Node析构" << endl; } // 正常执行
};

int main() {
    shared_ptr<Node> n1(new Node);
    shared_ptr<Node> n2(new Node);
    
    n1->next = n2; // 仅递增n2的weak_count，n2的use_count仍=1
    n2->next = n1; // 仅递增n1的weak_count，n1的use_count仍=1
    
    // 函数结束时，n1/n2析构：
    // n1的use_count=0 → 释放n1对象；n2的use_count=0 → 释放n2对象
    // 最后weak_count=0，释放控制块
    return 0;
}
