Lecture 3：线程创建

Thread 类

基础概念

概念：Thread 类

  • 概念：
    • System.Threading.Thread 是 .NET 中用于创建和控制线程的基础类。它提供了对线程最直接的控制；
    • 线程是操作系统能够进行运算调度的最小单位，它被包含在进程之中，是进程中的实际运作单位。使用 Thread 类可以创建前台或后台线程，并可以设置线程的优先级等属性；
  • 方法：
      1. 创建线程：
      • 通过实例化 Thread 类，并传入一个 ThreadStart 或 ParameterizedThreadStart 委托；
      • ParameterizedThreadStart 委托用于需要传递参数的情况；
      1. 启动线程：
      • 调用 Start() 方法；
      • 对于 ParameterizedThreadStart，可以通过 Start(object parameter) 传递参数；
      1. 等待线程结束：
      • 可使用 Join() 方法阻塞当前线程，直到目标线程执行完毕；
      1. 强制终止线程：
      • 可通过 Abort() 方法强制终止，但这可能导致资源未正确释放，应尽量避免；
  • 概念：Join 方法
    • Join 方法会阻塞调用它的线程，比如下列示例当中就是主线程。直到被调用的线程（如 thread1, thread2, thread3）执行完；
  • 特点：
    • 优点：提供对线程的精细控制​（如优先级、前后台状态）
    • 缺点：频繁创建和销毁线程会影响性能，并且需要开发者手动管理线程生命周期、同步等问题；

代码示例

代码

using System;
using System.Threading;

class Program
{
    static void Main()
    {
        // 创建无参线程
        Thread thread1 = new Thread(new ThreadStart(DoWork));
        thread1.Start();

        // 创建带参数的线程
        Thread thread2 = new Thread(new ParameterizedThreadStart(DoWorkWithParam));
        thread2.Start("Hello Parameter");

        // 使用Lambda表达式简化
        Thread thread3 = new Thread(() => DoWorkWithParam("Lambda Parameter"));
        thread3.Start();

        // 等待线程结束
        thread1.Join();
        thread2.Join();
        thread3.Join();

        Console.WriteLine("All threads completed.");
    }

    static void DoWork()
    {
        Console.WriteLine($"无参线程执行，线程ID: {Thread.CurrentThread.ManagedThreadId}");
    }

    static void DoWorkWithParam(object param)
    {
        Console.WriteLine($"带参数线程执行，参数: {param}，线程ID: {Thread.CurrentThread.ManagedThreadId}");
    }
}
  • 分析：
注意：使用 Join 的注意事项
    1. 要特别注意线程间相互等待对方释放资源而导致的死锁问题；
    • 例如，如果线程 A 等待线程 B 结束，而线程 B 又在等待线程 A 释放某个锁，两个线程就会永远等待下去；
    1. 不要对当前线程调用 Join​：
    • ​绝对不要在你的线程内部调用 CurrentThread.Join()，这会导致线程无限期地等待自己结束，造成程序死锁；

Task 类

基础概念

概念：Task 类

  • 概念：
    • System.Threading.Tasks.Task 出现在 .NET Framework 4.0 之后，它基于线程池，提供了更高级的抽象，用于表示一个异步操作；
    • Task 代表一个异步操作，它基于线程池工作，比直接使用 Thread 更高效。它提供了丰富的功能，如返回值、异常传播、延续任务、取消操作等；
  • 方法：创建和启动任务
      1. Task.Run(Action action)
      • 最常用的方式，将工作排在线程池队列并立即启动；
      • 当你调用 Task.Run(() => { /* 你的代码 */ }) 时，它内部会使用一组优化过的以下默认参数：
      • （1）默认调度器：使用 TaskScheduler.Default，即 .NET 线程池调度器。这确保了工作通常在线程池线程上执行；
      • （2）创建选项：默认包含 TaskCreationOptions.DenyChildAttach
        • 这个选项阻止任何在任务内部创建的子任务通过 AttachedToParent 选项附加到父任务；
        • 这有助于避免来自第三方库的意外子任务行为导致父任务等待或异常传播的问题；
      • Task.Run 非常适合大多数不需要对任务执行进行精细控制的场景；
      1. Task.Factory.StartNew(Action action)
      • 提供了更多选项，如任务创建选项；
      • 默认的创建选项（TaskCreationOptions）通常不包含 DenyChildAttach，这意味着子任务可以附加到父任务上；
  • 代码：Task 代码示例
using System;
using System.Threading.Tasks;
using System.Threading; // 需要引入 Threading 命名空间以使用 Thread.Sleep 和 Thread.ManagedThreadId

class Program
{
    // 将 Main 方法标记为 async，并指定返回类型为 Task，使其成为异步入口点（C# 7.1及以上支持）
    static async Task Main()
    {
        // 使用 Task.Run 方法将指定的工作（一个Lambda表达式）排入线程池队列等待执行
        // 该方法会立即返回一个代表该异步操作的 Task 对象
        Task backgroundTask = Task.Run(() =>
        {
            // 输出当前执行此代码块的线程ID，这通常是一个线程池线程的ID
            Console.WriteLine($"后台任务在线程 {Thread.CurrentThread.ManagedThreadId} 上运行");
            
            // 使用 Thread.Sleep 模拟一个耗时1秒的阻塞性工作（在实际应用中，应优先使用非阻塞的异步操作，如 Task.Delay）
            Thread.Sleep(1000);
            
            // 模拟工作完成后打印信息
            Console.WriteLine("后台任务完成");
        }); 

        // 主线程（通常是调用线程，这里很可能是主应用程序线程）在启动后台任务后立即继续执行，不会被 Task.Run 阻塞
        Console.WriteLine($"主线程 {Thread.CurrentThread.ManagedThreadId} 继续处理...");

        // 使用 await 关键字异步等待 backgroundTask 任务完成
        // 此时，控制权会返回给 Main 方法的调用者（如果是控制台程序，则可能涉及更复杂的同步上下文）
        // 当 backgroundTask 完成后，剩余代码（此例中没有）会在线程池线程或原始上下文（如有）上恢复执行
        await backgroundTask; // 等待后台任务完成

        // 在此可以继续执行其他需要等待 backgroundTask 完成后才能进行的工作
        Console.WriteLine("所有工作已完成！");
    }
}

任务状态与设置回调

概念：任务状态

  • 概念：Task.Status
    • Task.Status 属性返回一个 TaskStatus 枚举值，精确反映了任务从创建到结束的各个状态；
    • 常见状态：
      • Created: 任务已初始化，但尚未被计划；
      • WaitingToRun: 任务已被计划，等待执行；
      • Running: 任务正在运行；
      • RanToCompletion: 任务已成功完成；
      • Faulted: 任务因未处理异常而完成；
      • Canceled: 任务已通过取消令牌被取消；
    • 可以直接调用 Task 实例的状态，比如 Console.WriteLine($"任务当前状态: {task.Status}");
  • 示例：在主线程检查新 Task 的状态
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main()
    {
        // 使用 Task.Run 创建一个简单的任务
        Task task = Task.Run(() =>
        {
            Console.WriteLine("任务开始执行...");
            Task.Delay(2000).Wait(); // 模拟 2 秒耗时操作
            Console.WriteLine("任务执行完毕。");
        });

        Console.WriteLine($"任务初始状态: {task.Status}");

        // 在任务完成前，循环检查其状态
        while (!task.IsCompleted) // IsCompleted 可判断任务是否进入最终状态（完成/故障/取消）
        {
            Console.WriteLine($"任务当前状态: {task.Status}");
            await Task.Delay(500); // 每500毫秒检查一次，避免过度占用CPU
        }

        // 任务完成后的最终状态
        Console.WriteLine($"任务最终状态: {task.Status}");

        if (task.Status == TaskStatus.RanToCompletion)
        {
            Console.WriteLine("任务成功完成！");
        }
        else if (task.Status == TaskStatus.Faulted)
        {
            Console.WriteLine("任务执行过程中出现异常。");
            // 可通过 task.Exception 查看异常详情
        }
        else if (task.Status == TaskStatus.Canceled)
        {
            Console.WriteLine("任务已被取消。");
        }
    }
}

概念：设置回调

  • 概念：ContinueWith 方法
    • ContinueWith 方法允许你指定一个“延续任务”，它会在原任务完成后自动触发，无需在主线程中主动等待或轮询；
    • 可以在延续任务中检查原任务的状态、结果或异常；
  • 代码：
using System;
using System.Threading.Tasks;

class Program
{
    static void Main()
    {
        Task task = Task.Run(() =>
        {
            Console.WriteLine("任务开始执行...");
            Task.Delay(2000).Wait(); // 模拟工作
			Console.WriteLine("任务执行完毕。");
        });

        // 注册延续任务，处理完成后的逻辑
        // antecedentTask 即前面的 task 对象
        task.ContinueWith(antecedentTask => 
        {
            // 根据前述任务的状态进行处理
            if (antecedentTask.IsFaulted) // 是否有未处理异常
            {
                Console.WriteLine($"任务失败: {antecedentTask.Exception?.InnerException?.Message}");
            }
            else if (antecedentTask.IsCanceled) // 是否被取消
            {
                Console.WriteLine("任务被取消。");
            }
            else // 成功完成
            {
                Console.WriteLine("任务成功完成！");
                Console.WriteLine($"最终状态: {antecedentTask.Status}");
            }
        });

        Console.WriteLine("主线程在任务运行时继续执行其他工作...");
        Console.ReadLine(); // 防止主线程提前退出
    }
}
