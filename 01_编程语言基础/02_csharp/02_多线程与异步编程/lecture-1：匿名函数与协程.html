Lecture 1：匿名函数与协程

匿名函数

什么是匿名函数

概念：匿名函数

  • 概念：
    • 匿名函数（Anonymous Function）是指没有显式名称的函数，可以直接作为委托参数或赋值给委托变量使用；
    • 匿名函数的本质是编译器生成的临时方法，它们会被编译为普通的静态方法或实例方法，只是开发者无需手动定义这些方法；
  • 代码：匿名函数和委托的关系
// C# 1.0：显式定义委托和方法
delegate void TestDelegate(string s);
void NamedMethod(string s) { Console.WriteLine(s); }
TestDelegate del = new TestDelegate(NamedMethod);

// C# 2.0：匿名方法
TestDelegate del = delegate(string s) { Console.WriteLine(s); };

// C# 3.0：Lambda表达式
TestDelegate del = s => Console.WriteLine(s);

IEnumerator 和 IEnumerable

基本概念

概念：IEnumerator 和 IEnumerable

  • 概念：IEnumerable
    • IEnumerable 接口是 C# 中表示可枚举集合的核心接口。它定义了一个标准方式，表明实现该接口的对象包含一组元素，可以被逐个遍历；
    • IEnumerable 接口位于 System.Collections 命名空间，其只包含一个方法：GetEnumerator()，这个方法返回一个实现了 IEnumerator 接口的对象，它的泛型版本 IEnumerable<T> 也定义了一个 GetEnumerator() 方法，但返回的是泛型的 IEnumerator<T>；
    • 其主要作用是支持 foreach 迭代​：任何实现了 IEnumerable 或 IEnumerable<T> 的集合都可以使用 foreach 循环进行遍历。foreach 语句在编译时会自动处理获取枚举器、调用 MoveNext() 和访问 Current 的复杂性，因此推荐使用 foreach 而不是直接操作枚举器；
  • 概念：IEnumerator 接口
    • IEnumerator 接口是实际执行迭代操作的接口，它定义了遍历集合所需的基本协议，用于逐个访问集合中的元素；
    • IEnumerator 接口定义了三个成员：
      • （1）bool MoveNext()：将枚举器推进到集合的下一个元素。如果成功推进到下一个元素，返回 true；如果已越过集合的末尾，则返回 false；
      • （2）object Current { get; }：获取集合中位于枚举器当前位置的元素。这是一个只读属性；
      • （3）void Reset()：将枚举器设置到其初始位置，该位置位于集合中第一个元素之前；
    • 它的泛型版本 IEnumerator<T> 除了继承自 IEnumerator 的成员外，还定义了一个泛型的 Current 属性（返回 T 类型），并且实现了 IDisposable 接口；

使用 MoveNext() 进行迭代

概念：使用 MoveNext() 进行迭代

  • 概念：迭代过程
      1. 初始：
      • 枚举器最初定位在集合中第一个元素之前​，在首次调用 MoveNext() 之前，读取 Current 是未定义的，必须先调用 MoveNext() 将枚举器前进到集合的第一个元素，然后才能读取 Current；
      1. 调用：
      • 只要 MoveNext() 返回 true，就可以通过 Current 获取当前元素。Current 会返回相同的对象，直到再次调用 MoveNext()；
      • 一旦 MoveNext() 越过集合末尾（返回 false），枚举器就会定位在集合中最后一个元素之后，此时再次读取 Current 是未定义的；
      1. 重新遍历：
      • 如果需要再次从头遍历集合，不能简单地调用 Reset()，许多实现并不真正支持 Reset，可能会抛出 NotSupportedException；
      • 更常见的做法是获取一个新的枚举器实例，即通过再次调用 GetEnumerator()；
  • 总结：IEnumerable 和 IEnumerator 的关系是​“可枚举集合”与“枚举器”​的关系
    • IEnumerable 是可枚举集合，其职责是通过 GetEnumerator() 方法提供一个枚举器；
    • IEnumerator 是枚举器，其职责是通过 MoveNext()、Current 执行实际的迭代过程；
    • 每次调用 GetEnumerator() 都会返回一个新的枚举器实例，这使得可以同时进行多个独立的遍历操作，而它们的迭代状态不会相互干扰；
总结：foreach 的本质
  • foreach 循环本质上就是基于这两个接口工作的；
  • 编译器会将 foreach 循环转换为调用 GetEnumerator()、然后在 while 循环中调用 MoveNext() 和 Current 的代码；

代码：完整的代码示例

using System;
using System.Collections;
using System.Collections.Generic;

/// <summary>
/// 一个简单的自定义集合类，表示一篮子水果，实现了IEnumerable接口。
/// 这使得该集合可以使用foreach循环进行遍历。
/// </summary>
public class FruitBasket : IEnumerable
{
	// 私有数组，用于内部存储水果名称
	private string[] _fruits;

	/// <summary>
	/// 构造函数，初始化水果篮
	/// </summary>
	/// <param name="fruitsArray">用于初始化水果篮的字符串数组</param>
	public FruitBasket(string[] fruitsArray)
	{
		// 将传入的数组赋值给内部数组
		_fruits = fruitsArray;
	}

	/// <summary>
	/// 实现IEnumerable接口的GetEnumerator方法。
	/// 此方法返回一个枚举器，该枚举器可以遍历集合中的元素。
	/// </summary>
	/// <returns>一个用于遍历集合的IEnumerator对象</returns>
	public IEnumerator GetEnumerator()
	{
		// 返回我们自定义的枚举器实例，并将内部数组传递给它
		return new FruitEnumerator(_fruits);
	}
}

/// <summary>
/// 一个自定义的枚举器类，实现了IEnumerator接口
/// 它负责实际遍历FruitBasket集合中的元素
/// </summary>
public class FruitEnumerator : IEnumerator
{
	private string[] _fruits; // 要遍历的数组的引用
	private int _position = -1; // 当前枚举器在集合中的位置（索引），初始为-1（第一个元素之前）

	/// <summary>
	/// 构造函数
	/// </summary>
	/// <param name="fruitList">要遍历的水果数组</param>
	public FruitEnumerator(string[] fruitList)
	{
		// 存储对要遍历的数组的引用
		_fruits = fruitList;
	}

	/// <summary>
	/// 获取集合中位于枚举器当前位置的元素
	/// 这是 IEnumerator 接口的 Current 属性
	/// </summary>
	public object Current
	{
		get
		{
			// 尝试返回当前位置的元素
			try
			{
				return _fruits[_position];
			}
			// 如果位置无效（例如在第一次调用MoveNext之前或遍历结束后），捕获异常
			catch (IndexOutOfRangeException)
			{
				// 抛出一个更明确的异常，指示操作无效
				throw new InvalidOperationException("枚举器位于第一个元素之前或最后一个元素之后，无法获取Current");
			}
		}
	}

	/// <summary>
	/// 将枚举器推进到集合的下一个元素。
	/// 这是IEnumerator接口的MoveNext方法。
	/// </summary>
	/// <returns>如果枚举器成功推进到下一个元素，则为true；如果枚举器已越过集合的末尾，则为false。</returns>
	public bool MoveNext()
	{
		// 将位置索引加一
		_position++;
		// 检查是否仍在数组边界内
		return (_position < _fruits.Length);
	}

	/// <summary>
	/// 将枚举器设置到其初始位置，该位置位于集合中第一个元素之前。
	/// 这是IEnumerator接口的Reset方法。
	/// 注意：许多现代实现中不常用，甚至可能抛出NotSupportedException。
	/// </summary>
	public void Reset()
	{
		// 将位置重置为-1（第一个元素之前）
		_position = -1;
	}
}

class Program
{
	static void Main(string[] args)
	{
		// 初始化一个字符串数组作为水果篮的数据
		string[] myFruits = { "Apple", "Banana", "Orange", "Grape" };

		// 创建我们的自定义FruitBasket集合实例
		FruitBasket myBasket = new FruitBasket(myFruits);

		// 使用foreach循环遍历FruitBasket中的每个元素
		// 这背后会调用myBasket.GetEnumerator()，然后使用返回的枚举器的MoveNext和Current
		Console.WriteLine("Fruits in the basket:");
		foreach (string fruit in myBasket)
		{
			Console.WriteLine($" - {fruit}");
		}

		Console.WriteLine("\n手动使用枚举器遍历:");
		// 手动使用枚举器的方式（foreach的底层实现）
		IEnumerator enumerator = myBasket.GetEnumerator();
		while (enumerator.MoveNext()) // 只要还有下一个元素
		{
			string fruit = (string)enumerator.Current; // 获取当前元素
			Console.WriteLine($" - {fruit}");
		}

		// 对于实现了IDisposable的枚举器（如泛型IEnumerator<T>），应调用Dispose。
		// 此例中我们的简单枚举器没有非托管资源，但养成良好的习惯是值得的。
		// 如果枚举器是IDisposable，应使用using语句或手动调用Dispose。
		// 注意：我们的FruitEnumerator未实现IDisposable，因此无需调用。

		Console.ReadKey(); // 等待用户按键退出
	}
}

Unity 中的协程

基本概念

概念：协程

  • 概念：
    • 协同程序，简称协程；
    • 协程是“假”的多线程，它并不是多线程。它的主要作用将代码分时执行，不卡主线程；
    • 简单理解，是把可能会让主线程卡顿的耗时的逻辑，分时分步的执行；
    • 主要使用场景：异步加载文件、异步下载文件、场景异步加载、批量创建时防止卡顿；
  • 区别：
    • 新开一个线程是独立的一个管道，和主线程并行执行；
    • 新开一个协程是在原线程之上开启，进行逻辑的分时分步执行；
    • 并且在协程中可以创建死循环，并且不会阻塞主循环；
  • 举例：使用了几个 yield return，就表示将当前函数分成了几个部分，每个部分分时执行；
IEnumerator MyCoroutine(int i,string str)
{
	print(i)
	// 注意：协程函数当中，必须使用yield return 进行返回
	yield return new WaitrForSeconds(5f);
	print(str);
	yield return new WaitrForSeconds(1f);
	print("2");
	yield return new WaitrForSeconds(1f);
	print("3");
}

yield return 内容

概念：yield return 内容

  • 总结：
      1. 下一帧执行：
      • yield return 数字;
      • yield return null;
      • 在 Update 和 LateUpdate 之间执行；
      1. 等待指定秒之后执行：
      • yield return new WaitForSeconds(秒);
      1. 等待下一个固定物理帧更新时执行：
      • yield return new WaitForFixedUpdate();
      1. 等待摄像机和 GUI 渲染完成后执行：
      • yield return new WaitForEndOfFrame();
      1. 一些特殊类型的对象，比如异步加载相关函数返回的对象
      • 这些一般在 Update 和 LateUpdate 之间执行；
  • 概念：
    • 协程开启后组件和物体销毁，协程不执行；
    • 物体失活，协程不执行，组件失活协程执行

WaitUntil 方法

概念：WaitUntil 方法

  • 概念：
    • WaitUntil 方法能让你暂停协程的执行，直到某个特定条件被满足。这非常适合处理那些需要等待某些游戏事件发生后才继续执行的逻辑；
    • 它的核心作用是让协程挂起（暂停执行），直到你提供的委托或函数返回 true​；
  • 定义：
    • yield return new WaitUntil(Func<bool> predicate);
  • 解释：
    • 这里的 Func<bool> predicate 是一个返回布尔值的方法委托（可以理解为一个条件判断函数）；
    • Unity 在每一帧都会检查这个条件，一旦条件为 true，协程就会从 yield return 之后的地方继续执行；
  • 区别：WaitUntil 方法和其他类似方法的区别

协程的原理

基础概念

概念：什么是协程

  • 概念：
    • 协程可以分成两部分：
        1. 协程函数本体；
        1. 协程调度器；
    • 协程本体就是一个能够中间暂停返回的函数；
    • 协程调度器是 Unity 内部实现的，会在对应的时机帮助我们继续执行协程函数；
    • Unity 只实现了协程调度部分协程的本体，协程本质上就是一个 C# 的迭代器方法；
  • 分析：
      1. 协程函数本体
      • 如果我们不通过开启协程方法来执行协程，Unity 的协程调度器是不会帮助我们管理协程函数的；
      • 但是我们可以自己执行迭代器函数内容；
      1. 协程调度器
      • 继承 MonoBehavior 后开启协程；
      • 相当于是把一个协程函数 (迭代器)放入 Unity 的协程调度器中，帮助我们管理进行执行；
      • 具体的 yield return 后面的规则也是 Unity 定义的一些规则；
  • 总结
    • 你可以简化理解，迭代器函数 C# 看到迭代器函数和 yield return 语法糖，就会把原本是一个的函数变成"几部分"，我们可以通过迭代器从上到下遍历这“几部分"进行执行，就达到了将一个函数中的逻辑分时执行的目的；

协程原理分析

概念：协程原理分析

  • 概念：协程的原理 - 状态机 + 迭代器
    • 当编译器检测到包含 yield return 的方法时，会将其自动转换为一个状态机类，该状态机内部通过 switch 语句管理协程的执行进度（每个 yield 对应一个状态分支）
    • 协程通过迭代器的 MoveNext() 方法逐步推进状态。每次调用 MoveNext() 时，状态机根据当前 _state 执行对应代码块，直到遇到下一个 yield 或者结束；
  • 概念：协程函数本体
    • 协程函数本体是一个返回类型为 IEnumerator 的方法，内部使用 yield return 定义暂停点；
    • 协程运行在主线程，通过分帧避免阻塞，但非真正的多线程，每次暂停时，局部变量和指令位置被保存在状态机中，恢复时还原上下文；
    • 常用暂停指令：
  • 概念：协程调度器
    • 调度器负责追踪所有运行的协程，根据 yield return 返回的指令决定何时唤醒协程（如帧结束、物理更新后），通过 StartCoroutine() 启动协程，StopCoroutine() 强制终止，或自动销毁关联 MonoBehaviour 时清理协程；
    • 工作流程：
        1. 注册协程：调用 StartCoroutine() 将协程迭代器加入调度队列；
        1. 帧循环检测：每帧遍历所有协程，检查其暂停条件是否满足
        • 若条件满足（如等待时间结束），调用 MoveNext() 推进状态机；
        • 若条件未满足，保留至下一帧检测；
        1. 终止处理：
        • 协程自然结束（MoveNext() 返回 false）时移出队列，手动调用 StopCoroutine() 或销毁 GameObject 时立即终止；
    • 协助关系：
