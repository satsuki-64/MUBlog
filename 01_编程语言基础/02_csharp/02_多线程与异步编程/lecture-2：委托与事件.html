Lecture 2：委托与事件

委托与事件的基础概念

概念：委托与事件的基础概念

  • 概念：委托
    • 委托是一种类型安全的函数指针，它定义了方法的签名（参数类型和返回类型），从而可以将方法作为参数传递或赋值给变量。委托的本质是一个类，它封装了对方法的引用，使得程序能够动态地调用不同的方法；
  • 概念：事件
    • 事件则是建立在委托之上的一个封装机制，它对外提供了一种发布-订阅模型；
    • 事件允许一个对象（发布者）在特定情况发生时通知其他对象（订阅者），而发布者不需要知道订阅者的任何细节；
    • 事件像是一个受保护的委托实例，外部代码只能通过 += 和 -= 操作符来订阅或取消订阅，而不能直接重置事件或触发它，这大大增强了安全性和封装性；
  • 总结：委托、事件、Func、Action 与 UnityAction
    • 关系：
    • 委托：
      • 可以把委托理解为一个方法的容器或者蓝图，它定义了“什么样的方法可以被装进来”，即方法的参数类型和返回类型，一旦定义了蓝图，任何符合这个格式的方法都可以被放入这个容器中；
      • 在 C# 中，除了使用 delegate 关键字自定义委托，.NET 还提供了内置的泛型委托，如 Action（无返回值）和 Func（有返回值），这大大减少了自定义委托类型的需要；
    • 事件：
      • 事件是封装在类内部的委托，主要目的是为了实现安全的发布-订阅（Publish-Subscribe）模式；
      • 在声明事件的类外部，其他代码只能进行订阅（+=）​​ 和取消订阅（-=）​​ 操作，​不能直接触发事件，也不能用 = 覆盖其他订阅者。这确保了事件只能由事件的发布者（即定义它的类）来触发，提高了封装性和安全性；
    • UnityAction：
      • UnityAction 是 Unity 在 UnityEngine.Events 命名空间下定义的委托类型，本质上与 .NET 的 Action 委托相同，代表无返回值的方法；
    • UnityEvent：
      • UnityEvent 是 Unity 引擎的一个核心类，它继承了 UnityEvent 的机制，并允许通过序列化在 ​ Unity Editor 的 Inspector 面板上直接配置和绑定回调函数；
    • 总结：

委托

使用委托

概念：使用委托

    1. 声明委托类型：使用 delegate 关键字来定义一种新的委托类型，它指定了可以引用的方法必须具有的签名；
    • public delegate int MathOperation(int a, int b);
    1. 实例化委托：创建一个该委托类型的实例，并将其绑定（或指向）一个具体的方法。这个方法必须与委托的签名完全匹配；
    1. 调用委托：
    • 像调用普通方法一样调用委托实例：int result = operation(5, 3);
    • 对于多播委托，使用使用 += 和 -= 来组合多个方法：
      • operation += Multiply;
      • result = operation(5, 3);

多播委托

概念：多播委托

  • 概念：什么是多播委托
    • 多播委托机制允许一个委托实例同时持有并顺序调用多个方法，其使用 += 添加方法，-= 移除方法，方法按添加顺序依次调用；
    • 如果出现多个非 void 返回值时，​只保留最后一个方法的返回值；
  • 概念：如何使用
    • 多播委托通常用于返回 void 的方法。如果委托有非 void 返回值，调用后你只能得到最后一个方法的返回值，前面方法的返回值会被丢弃；
    • 如果确实需要收集每个方法的返回值，必须手动遍历调用列表；
  • 概念：多播委托的底层机制 - 基于调用列表
    • C# 中使用 delegate 关键字声明的任何委托类型，在编译后都会继承自 System.MulticastDelegate 类，每个委托实例内部维护着一个调用列表（Invocation List）​，你可以将其理解为一个方法引用的数组；
    • 当你使用 += 添加方法时，系统会创建一个新的委托实例，并将新方法引用添加到列表末尾；
    • 当调用委托时，其实就是按顺序遍历这个列表并执行其中的每个方法；

代码：基于 GetInvocationList() 方法获得调用列表的多个返回值

using System;

// 1. 声明一个委托类型
public delegate int MathOperation(int a, int b);

class Program
{
    // 符合委托签名的方法
    static int Add(int x, int y) {
        Console.WriteLine($"Adding {x} and {y}");
        return x + y;
    }
    
    static int Multiply(int x, int y) {
        Console.WriteLine($"Multiplying {x} and {y}");
        return x * y;
    }
    
    static int ProblematicMethod(int x, int y) {
        throw new InvalidOperationException("Simulated error");
    }

    static void Main()
    {
        // 2. 创建多播委托并绑定多个方法
        MathOperation multiDelegate = Add;
        multiDelegate += Multiply;
        multiDelegate += ProblematicMethod; // 添加一个会抛出异常的方法

        // 3. 获取调用列表
        Delegate[] invocationList = multiDelegate.GetInvocationList();
        Console.WriteLine($"委托链中共有 {invocationList.Length} 个方法。");

        // 4. 遍历调用列表并单独执行每个委托
        foreach (Delegate del in invocationList)
        {
            // 将委托转换回具体的委托类型
            MathOperation operation = (MathOperation)del;
            try
            {
                // 调用当前委托并获取返回值
                int result = operation(5, 3);
                Console.WriteLine($"方法执行结果: {result}");
            }
            catch (Exception ex)
            {
                // 捕获并处理当前方法的异常，不影响后续方法的执行
                Console.WriteLine($"方法执行出错: {ex.Message}");
            }
        }
    }
}
注意：关于多播委托
    1. 每次使用 += 或 -= 操作时，都会创建一个新的委托实例，而非修改原有实例。在高性能要求的场景下需注意此开销；
    1. 在多线程环境中，对同一个委托实例进行加减操作不是线程安全的，需要自行加锁；
    1. 通过 GetInvocationList 获得的数组是原始委托链的一个快照。对此数组进行修改（如增删委托）​不会影响原始的多播委托。要修改委托链，仍需使用 += 和 -= 运算符；

Action 与 Func

概念：Action 与 Func

  • 概念：Action 与 Func 特性总结
  • 代码：使用 Action
// 1. 无参数的Action
Action greet = () => Console.WriteLine("Hello, World!");
greet(); // 输出: Hello, World!

// 2. 带参数的Action
Action<string> printMessage = (message) => Console.WriteLine(message);
printMessage("Hello, C#!"); // 输出: Hello, C#!

Action<int, int> printSum = (x, y) => Console.WriteLine(x + y);
printSum(5, 3); // 输出: 8

// 3. 将现有方法分配给Action
void DisplayToWindow(string text) { MessageBox.Show(text); }
Action<string> showAction = DisplayToWindow;
showAction("Koani");
  • 代码：使用 Func
// 1. 无参数，有返回值
Func<int> getRandom = () => new Random().Next();
int number = getRandom(); // number 的值为一个随机整数

// 2. 带参数，有返回值
Func<int, int, int> multiply = (a, b) => a * b;
int result = multiply(5, 3); // result 的值为 15

Func<string, string> toUpperCase = (input) => input.ToUpper();
string upperCase = toUpperCase("hello"); // upperCase 的值为 "HELLO"

事件

基础概念

概念：什么是事件

  • 概念：
    • C# 中的 event 关键字提供了一种强大的机制来实现发布-订阅模式，让对象之间能够进行松耦合的通信；
    • 具体而言，事件是一种特殊的委托类型成员，它允许一个对象（称为发布者）在某个特定动作或状态变化发生时，通知其他多个对象（称为订阅者）；
    • 这种机制的核心价值在于实现了发布者与订阅者之间的解耦​：
      • 发布者只知道“某事发生了”，但不知道也不关心有哪些对象会对此做出响应；
      • 订阅者则决定“当某事发生时，我要做什么”，而无需知道事件是如何触发的；
    • 因此，事件本质上是一种对于多播委托的一种封装，带来了更好的安全性和可控性；
  • 概念：事件的实现机制
    • 假设声明了 public event EventHandler MyEvent; 之后，编译器会为这行代码生成以下内容：
        1. 一个私有的多播委托字段​（例如 EventHandler _myEvent），这才是真正存储所有事件处理程序引用的地方；
        1. 两个公共的 add 和 remove 访问器​，分别对应 +=（订阅）和 -=（取消订阅）操作；
    • 这种自动实现的封装意味着：
        1. ​从类的外部，你只能通过 += 和 -= 来添加或移除事件处理程序；
        1. 在类的外部，你不能直接对事件赋值（如 MyEvent = null;），也不能在类的外部直接调用（触发）事件；
    • 因此，触发事件的权力被严格限制在声明该事件的类内部，这有效防止了外部的随意干预，提升了代码的健壮性；

概念：事件和委托的区别

  • 概念：封装性的不同
    • 委托的封装性低，可视为一个指向方法的类型安全的函数指针；
    • 封装性高，是对委托的包装，外部只能进行订阅和取消订阅操作，不能直接调用事件；
  • 概念：使用权限
    • 委托可以在类外部直接调用，并且可以使用 = 进行赋值、使用 +=、-= 来设置多播委托；
    • 事件只能在声明它的类的内部触发，外部只能使用 += 和 -= 来添加或移除处理程序，保证了委托链的安全；

事件的使用

概念：事件的使用

  • 概念：事件组成
      1. 事件参数：EventArgs
      • 用于在事件触发时向订阅者传递数据；
      • 自定义事件参数类应继承自 EventArgs；
      1. 委托类型：
      • 定义了事件处理方法的签名（即参数和返回类型），建议使用内置的 EventHandler 或 EventHandler<TEventArgs> 委托；
      1. 事件声明：
      • 在类内部使用 event 关键字进行声明；
  • 概念：为事件注册函数
    • 当执行类似于 thermostat.TemperatureChanged += HandleTemperatureChange; 的代码来为其注册新事件时，这个待注册的函数 HandleTemperatureChange 需要符合以下规则：
      1. 参数一为事件源，通常为 this (即 sender)，订阅者通过 sender获取触发事件的对象引用，可进行类型转换后操作；
  • 概念：事件的使用规范
      1. 优先使用内置的 EventHandler 或 EventHandler<TEventArgs>；
      • 其中 EventHandler 用于不携带额外数据的事件，EventHandler<TEventArgs> 用于需要传递自定义数据的事件；
      1. 自定义的事件参数类应继承自 EventArgs，并且类名最好以 EventArgs 结尾；
      1. 触发事件的方法：通常定义一个名为 On[事件名]、protected 和 virtual 的方法来封装触发事件的逻辑。这样既方便派生类重写触发行为，也确保了触发前的安全检查；
      1. 待注册的函数，第一个参数定义为 object sender，第二个参数类型为当前事件的参数；
  • 示例：不带参数事件
public class AlarmClock
{
    // 声明事件，使用非泛型的 EventHandler
    public event EventHandler AlarmRing;

    protected virtual void OnAlarmRing()
    {
        AlarmRing?.Invoke(this, EventArgs.Empty); // 使用空的 EventArgs
    }

    public void Start()
    {
        // 模拟闹钟到时
        OnAlarmRing();
    }
}

// 订阅
var clock = new AlarmClock();
clock.AlarmRing += (sender, e) => Console.WriteLine("起床时间到了！");
clock.Start();
  • 示例：带参数事件
using System;

// 1. 定义自定义事件参数，其继承自 EventArgs
public class TemperatureChangedEventArgs : EventArgs
{
    public double NewTemperature { get; }
    public double OldTemperature { get; }

    public TemperatureChangedEventArgs(double oldTemp, double newTemp)
    {
        OldTemperature = oldTemp;
        NewTemperature = newTemp;
    }
}

// 2. 事件发布者（Publisher）
public class Thermostat
{
    private double _currentTemperature;

    // 3. 使用 EventHandler<T> 声明事件
    // 这里的事件的含义，是当温度（_currentTemperature）发生变化时，调用函数进行处理
    public event EventHandler<TemperatureChangedEventArgs> TemperatureChanged;

	// 设置当前温度时会触发事件
    public double CurrentTemperature
    {
        get => _currentTemperature;
        
        set
        {
            if (_currentTemperature != value)
            {
                double oldTemp = _currentTemperature;
                _currentTemperature = value;
                
                // 4. 调用触发事件的方法
                OnTemperatureChanged(oldTemp, value);
            }
        }
    }

    // 5. 定义触发事件的受保护虚方法
    // 命名为 On+事件名，protected virtual 允许派生类重写触发逻辑
    protected virtual void OnTemperatureChanged(double oldTemp, double newTemp)
    {
        // 6. 安全地触发事件
        // 将事件赋值给一个临时变量
        var handlers = TemperatureChanged;
        handlers?.Invoke(this, new TemperatureChangedEventArgs(oldTemp, newTemp));
    }
}

// 7. 事件订阅者（Subscriber）
public class TemperatureDisplay
{
    public TemperatureDisplay(Thermostat thermostat)
    {
        // 8. 使用 += 运算符订阅事件
        // 将自己的对温度的响应函数，注册到事件当中
        thermostat.TemperatureChanged += HandleTemperatureChange;
    }

    // 9. 事件处理方法，签名必须与 EventHandler<TemperatureChangedEventArgs> 匹配
    private void HandleTemperatureChange(object sender, TemperatureChangedEventArgs e)
    {
        Console.WriteLine($"温度发生了变化：从 {e.OldTemperature}°C 变为 {e.NewTemperature}°C");
    }

    // 10. 取消订阅的方法（重要！防止内存泄漏）
    public void Unsubscribe(Thermostat thermostat)
    {
        thermostat.TemperatureChanged -= HandleTemperatureChange;
    }
}

// 使用示例
class Program
{
    static void Main()
    {
        var thermostat = new Thermostat();
        var display = new TemperatureDisplay(thermostat);

        thermostat.CurrentTemperature = 25.0; // 触发事件
        thermostat.CurrentTemperature = 30.5; // 再次触发事件
    }
}
注意：关于取消订阅
  • 如果订阅者（如一个窗体）的生命周期短于发布者（如一个全局的单例服务），并且没有取消订阅，那么发布者将一直持有对订阅者的引用，阻止垃圾回收（GC）回收订阅者对象；
  • 因此，​在不再需要接收事件时，务必使用 -=取消订阅；
