Lecture 6：动态类型与匿名类型

dynamic 与 var

dynamic 与 var 区别对比

分析：dynamic 与 var

  • 分析：dynamic 与 var 的不同点
    • var 是隐式类型，类型确定时间为编译时。因此其类型安全性更高，类型一旦推断便固定，类型错误在编译时发现。一旦初始化，类型便固定；
    • dynamic 是动态类型，类型确定时间为运行时。因此类型可变，操作不当会导致运行时错误，允许改变类型，同一个变量可以先后被赋值为完全不同类型的值；
  • 应用：
    • var 主要应用于局部变量；
    • dynamic 主要反射、与动态语言对象交互；
  • 总结：

var 类型

分析：var 的类型

  • var 是一个语法糖，它让代码更简洁，但变量仍然是静态类型的。编译器在编译时会根据赋值符号右侧的表达式精确推断出变量的类型，并用这个实际类型替换 var。因此，使用 var 并不会降低类型安全或性能；
  • 当变量类型名称很长或显而易见时，简化代码：
    • 如
      • Dictionary<string, List<MyCustomClass>> data = new Dictionary<string, List<MyCustomClass>>();
    • 可以简写为：
      • var data = new Dictionary<string, List<MyCustomClass>>();
注意：var 的使用限制
  • var 只能用于局部变量的声明，并且必须在声明时初始化。它不能用于字段（类成员变量）、方法参数或返回类型；

dynamic 类型

分析：dynamic 的类型

  • dynamic 是一个真正的类型，它告诉编译器：“这个变量的类型在编译时不确定，请不要做任何检查，所有类型相关的问题都留到运行时解决。” ；
  • 这为 C# 带来了动态语言的能力，但也牺牲了编译时的安全性；
  • 当处理反射时使用 dynamic 可以使代码更简洁；

类型与 object 转换

代码

  • object 作为所有类型的基类，可以接收任何值，但使用时通常需要显式类型转换；
object obj = "Hello";
string str = (string)obj; // 需要显式转换

dynamic dyn = "Hello";
string str2 = dyn; // 无需转换，但运行时才检查

var s = "Hello";
string str3 = s; // 无需转换，编译时已知s就是string

var 代码示例

代码：var 代码示例

  • 代码：简化局部变量声明
// 明显可知是 List<string> 类型，使用 var 简化声明
var nameList = new List<string>(); 
nameList.Add("Alice");
nameList.Add("Bob");

// 遍历集合时，编译器能推断出 item 是 string 类型
foreach (var item in nameList)
{
    Console.WriteLine(item.ToUpper()); // 编译时即知有 ToUpper() 方法
}
  • 代码：处理 LINQ 查询
// 假设有一个 Person 类
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
    public string City { get; set; }
}

List<Person> people = new List<Person>
{
    new Person { Name = "John", Age = 30, City = "London" },
    new Person { Name = "Jane", Age = 25, City = "New York" }
};

// LINQ 查询返回的类型是 IEnumerable<anonymoustype>，使用 var 接收非常方便
var youngPeopleInCities = from p in people
                          where p.Age < 35
                          select new { p.Name, p.City }; // 匿名类型

foreach (var personInfo in youngPeopleInCities)
{
    // 由于 personInfo 是匿名类型，必须用 var 声明循环变量
    Console.WriteLine($"Name: {personInfo.Name}, City: {personInfo.City}");
    // 编译时即可获得 IntelliSense 支持，如 personInfo.Name
}
  • 代码：使用匿名类型
// 创建匿名类型对象。该对象具有 Name 和 Score 属性。
var student = new { Name = "Tom", Score = 95 };

// 可以正常访问属性，因为编译器在编译时已知其类型结构
Console.WriteLine($"{student.Name} scored {student.Score}");

// 注意：匿名类型的属性是只读的
// student.Score = 100; // 这行会编译错误

dynamic 代码示例

代码：dynamic 代码示例

  • 代码：处理动态数据
dynamic myDynamicVariable = "Initially I am a string";
Console.WriteLine(myDynamicVariable.ToUpper()); // 运行时调用 string.ToUpper()

myDynamicVariable = 42; // 现在它变成了一个整数
Console.WriteLine(myDynamicVariable * 2); // 运行时进行整数乘法

myDynamicVariable = new List<string> { "a", "b", "c" }; // 现在它是一个列表
foreach (var item in myDynamicVariable) // foreach 循环在运行时解析
{
    Console.WriteLine(item);
}
// 注意：虽然灵活，但过度使用会让代码难以理解和维护
注意：Unity 中对于 dynamic 的使用兼容性
  • Unity 默认使用 .NET Standard 2.1​，其 API 兼容性级别不包含 dynamic 类型所需的完整运行时绑定支持，因此如果要是有 dynamic，你需要将其切换到完整的 .NET Framework；
  • 更改方法：在 Player Settings 的 Configuration 区域下，找到以下两个选项并进行修改
    • 第一步：Scripting Runtime Version (脚本运行时版本)​​：将其从 .NET 4.x Equivalent 更改为 ​ .NET Framework；
    • 第二步：Api Compatibility Level (API 兼容性级别)​​：将其从 .NET Standard 2.1 更改为 ​ .NET Framework​；

匿名类型

概念：什么是匿名类型

  • 概念：
    • 匿名类型是一个由编译器临时创建来存储一组值的简单类。如果需要创建一个匿名类型，则可以使用 new 关键字，后面加上对象初始化器，指定该类型包含的属性和值；
    • 比如：var dude = new { Name = "Bob",Age = 23};
    • 编译器会将其转化为：
  • 概念：匿名类型中的属性名称
    • 匿名类型的属性名称可以从一个本身为标识符（或者以标识符结尾）的表达式推断得到：
    • int Age = 23;
    • var dude = new {Name = "Bob",Age,Age.ToString().Length};
    • 上述代码等价位：var dude = new{ Name="Bob",Age =Age,Length=Age.ToString().Length };
  • 注意：
      1. 匿名类型只能通过 var 关键字来引用，因为它并没有一个名字；
      1. 方法无法有效地返回匿名类型的对象，因为在 Lambda 语句中，将函数返回类型指定为 var 是非法的；

示例：使用匿名类型

  • 代码：创建匿名类型的数组
var dude = new[]
{
	new { Name = "Bob", Age = 32},
	new { Name = "Tom", Age = 31}
};
  • 代码：在 Lambda 语句中使用 dynamic 接收返回值
dynamic Foo() => new { Name = "Bob", Age =30 };
