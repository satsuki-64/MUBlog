Lecture 5：特性

什么是特性

基础概念

概念：什么是特性

  • 概念：
    • 特性是一种允许我们向程序的程序集添加元数据的语言结构，它是用于保存程序结构信息的某种特殊类型的类；
      • 本质是继承自 System.Attribute 的类；
    • 特性提供功能强大的方法，其将声明信息与 C# 代码 (类型、方法、属性等) 相关联，特性与程序实体关联后，即可在运行时使用反射查询特性信息；
    • 特性不直接影响代码逻辑，但可通过反射在运行时读取，用于控制程序行为或提供额外信息；
  • 目的：
    • 特性的目的是告诉编译器把程序结构的某组元数据嵌入程序集中，它可以放置在几乎所有的声明中 (类、变量、函数等等申明)；
    • 我们可以利用特性类为元数据添加额外信息，比如一个类、成员变量、成员方法等等，为他们添加更多的额外信息之后可以通过反射来获取这些额外信息；
  • 总结：特性的使用用途

自定义特性

代码

  • 继承自一个 Attribute 类之后，其就是一个自定义特性；
class MyCustomAttribute : Attribute
{
	// 特性中的成员
	public string info;
	
	public MyCustomAttribute(string info)
	{
		this.info = info;
	}
}

特性的使用

基础概念

概念：特性的使用

  • 语法：
    • [特性名(参数列表)]
    • 写在类、函数、变量上一行，表示他们具有该特性信息；
  • 概念：
    • 特性的使用，本质上就是在调用特性类的构造函数；
  • 示例：
// 注意：这里默认省略了后背的 Attribute
[MyCustom("这是一个自己写的类")]
class MyClass
{
	// ...

	[MyCustom("这是一个自己写的成员变量")]
	public int valueTemp;
	

	[MyCustom("这是一个自己写的方法")]
	public void TestFun()
	{
	
	}
}

代码：特性的场景用途

	}
}

限制自定义特性的使用范围

概念：限制自定义特性的使用范围

  • 概念：
    • 通过为特性类加特性，可以限制其使用范围；
  • 语法：
    • 语法：
      • [AttributeUsage (AttributeTargets.Class | AttributeTargets.Struct, AllowMultiple = true, Inherited = true)]
    • 参数：
      • 其中 AllowMultiple 若为 true，可像 [Author("Alice")][Author("Bob")] 这样叠加使用；若 false 则编译报错；
      • 其中 Inherited 若设为 false，派生类不会自动继承基类的特性标记。需显式重新应用特性；
  • 示例：
// 参数一：表示只能使用在类以及成员变量上
// 参数二：表示可以被多次添加同一个特性
// 参数三：表示不会自动继承基类的特性标记
[AttributeUsage (AttributeTargets.Class | AttributeTargets.Field, AllowMultiple = true, Inherited = false)]
class MyCustomAttribute : Attribute
{
	// 特性中的成员
	public string info;
	
	public MyCustomAttribute(string info)
	{
		this.info = info;
	}
}

系统自带特性

概念：常用特性

  • 概念：Obsolete
    • 示例：
      • [Obsolete("此方法已过时"),false]
    • 参数：
      • 第一个参数为报错的内容；
      • 第二个参数为真时，调用此方法时程序将直接报错。第二个参数为假时，只输出警告信息；
  • 概念：Conditional - 条件编译特性
    • 概念：
      • Conditional 特性需要和预处理指令 #define 一起使用；
      • 需要引用命名空间 using system.Diagnostics，主要可以用在一些调试代码上、有时想执行有时不想执行的代码；
    • 示例：
#define Fun

class Test
{
	// 只有在定义了 Fun 符号之后，这个部分代码才会被编译进去
	[Conditional("Fun")]
	static void Fun()
	{
		Console.Write("111");
	} 

}
  • 概念：外部 DLL 包特性
    • 概念：
      • 用来标记非 .Net(C#) 的函数，一般用来调用 C 或者 C++ 的 DLL 包写好的方法；
      • 此特性用于表明：该函数在一个外部的 DLL 中定义；
    • 注意：
      • 需要引用命名空间 using system.Runtime.Interopservices
    • 举例：
class Test
{
	// 其中括号内的内容，是一个要引用的DLL包的路径
	[DllImport("Test.dll")]
	public static extern Add(int a,int b);

	// ...
}
