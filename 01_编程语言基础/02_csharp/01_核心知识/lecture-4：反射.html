程序集和元数据

概念：程序集和元数据

  • 概念：元数据
    • 元数据 (Metadata)在 C# 中是指 "描述数据的数据"，它提供了关于程序的结构、类型定义、成员信息等关键内容的信息；
    • 元数据是 .NET 平台中不可或缺的组成部分，它使得程序能够自我描述，即程序能够携带关于其自身的信息；
    • 在 C# 中 ，元数据主要包含以下几类信息：
    • 元数据以非特定语言的方式描述在代码中定义的每一类型和成员，这使得不同 .NET 语言编写的程序能够无缝互相操作；
  • 概念：程序集
    • 程序集 (Assembly)是 .NET 应用程序的基本构建块和部署单元，通常表现为 DLL (动态链接库)或 EXE (可执行文件)文件；
    • 其主要包含以下内容：
        1. IL 代码：中间语言 (Intermediate Language)，是 .NET 代码编译后的中间形式，需要 JIT (即时编译)转换为机器码；
        1. 元数据：描述程序集中类型、成员、引用关系等信息的数据；
        1. 资源文件：如图标、图像、字符串等非代码资源；
        1. Manifest 清单：程序集的元数据，包括版本号、依赖项、标识信息等；
    • 元数据与程序集密不可分，程序集是元数据的物理载体，而元数据是程序集的"自描述"机制：
        1. 编译时：代码编译为 PE (可移植可执行)文件时，编译器会生成元数据并嵌入到程序集中；
        1. 执行时：CLR (公共语言运行时) 在执行程序时会加载元数据，用于发现类、成员、继承等信息；
        1. 反射：元数据为反射 API 提供了基础数据，使得程序能够在运行时动态检查和使用类型信息；
  • 概念：程序集中的 Manifest
    • 程序集清单 (Assembly Manifest) 是 .NET 程序集的核心组成部分，它包含了程序集的元数据信息，描述了程序集的结构、依赖关系和版本控制等重要信息；

反射基础概念

概念：什么是反射

  • 概念：
    • 反射允许程序在运行时动态地检查、访问和修改自身的结构和行为，程序正在运行时，可以查看其它程序集或者自身的元数据，一个个运行的程序查看本身或者其它程序的元数据的行为就叫做反射；
    • 因此，通过反射，开发者可以在程序运行时获取程序集、模块、类型及其成员的信息，并能够动态创建类型实例、调用方法、访问属性和字段等；
  • 概念：反射的核心功能
      1. 动态获取类型信息​：通过 typeof 运算符或 GetType() 方法获取 System.Type 对象，该对象包含类型的完整元数据；
      1. 动态创建对象​：使用 Activator.CreateInstance 方法或通过 ConstructorInfo 调用构造函数来实例化对象；
      1. 动态调用方法​：通过 MethodInfo 对象调用方法，包括静态方法和实例方法；
      1. 访问成员信息​：通过 PropertyInfo、FieldInfo、EventInfo 等类访问和操作属性、字段和事件；
      1. 动态加载程序集​：使用 Assembly.Load 或 Assembly.LoadFrom 方法在运行时加载外部程序集；

概念：Type

  • 概念：
    • system.Type 的实例代表了类型的元数据。由于 Type 的应用领域非常广泛，因此它存在于 System 命名空间，而非 System.Reflection 命名空间中；
    • Type 类是 C# 反射机制的核心组件，它提供了运行时访问类型元数据的能力，使开发者能够动态检查和操作类、接口、数组、值类型等类型信息；
  • 作用：

    • 调用任意对象的 GetType 方法或者使用 C# 的 typeof 运算符都可以得到 System.Type 的实例：

实例化类型

概念：实例化类型

  • 从类型中创建对象实例有两种方法：
  • 方法一：调用静态的 Activator.CreateInstance 方法；
  • 方法二：获得类型的构造函数的元数据，调用 ConstructorInfo.Invoke 方法，并使用 Type 的 GetConstructor 方法的返回值作为参数；

使用 Activator 类创建类型

概念：Activator 类

  • 概念：
    • Activator 类是 C# 反射 (Reflection) 机制的重要组成部分，它专门用于在运行时动态创建对象实例，是 .NET 框架中实现动态编程的关键组件之一；
    • Activator 类位于 System 命名空间下，主要提供以下核心功能：
        1. 无需在编译时知道具体类型，即可在运行时创建对象实例；
        1. 支持无参和有参构造函数的动态调用；
        1. 能够从外部程序集 (如 DLL)中加载并创建类型的实例；
        1. 通过 Activator.CreateInstance<T>() 方法支持泛型类型的实例化；
    • 与直接使用 new 关键字相比，Activator 的最大优势在于其运行时动态性，它允许程序根据运行时的条件 (如配置文件、用户输入等)决定要实例化的具体类型；
  • 概念：Activator 类与反射的关系
    • Activator 是反射机制的一部分，二者关系可概括为：
      1. 反射是基础*：反射提供了获取类型信息 (Type)的能力，而 Activator 利用这些信息进行实例化；
      1. 负责不同的内容：
      • 反射负责"探查"，获取程序集、模块、类型及成员信息，Activator 负责"创建"，专注于对象实例的动态创建；

使用构造函数实例化类型

概念：使用构造函数来实例化类型

  • 概念：
    • 当所用参数值无法甄别重载构造器时，就需要调用 ConstructorInfo 的 Invoke 方法；
    • 例如，假设 X 类拥有两个构造器：一个构造器接受 string 参数，另一个接受 StringBuilder 类型的参数；
    • 如果向 Activator.CreateInstance 传递 null 参数，则调用目标就具有二义性, 此时就需要使用 ConstructorInfo；
  • 示例：
// 1. 获得指定参数重载版本的构造函数
// GetConstructor 方法的参数指定构造函数的参数类型
ConstructorInfo ci = typeof(X).GetConstructor(new[] {typeof(string)});

// 2. 使用构造函数创建对象
object foo = ci?.Invoke(new object[]{null});
注意：动态实例化的开销
  • 使用动态实例化构造对象会增加几微秒的时间，相对来说这个时间比较长；
  • 因为 CLR 实例化对象的速度是很快的，使用 new 实例化一个简单的小型类只需要消耗几十纳秒的时间；

class Program
{

	{
		// 参数1：指定委托

			typeof(IntFunc), new Program(), "Cube"
		);
		
		// 调用方法

		Console.WriteLine(f(3));
	}
}

public void Add<TProperty>(string name,BindableProperty<TProperty>.ValueChangedHandler valueChangedHandler )
{
	// 这里的 T 是类的泛型参数，使用 typeof(T) 在运行时获取 T 的真实类型
	// 然后再使用 GetField 得到类型当中变量名为 name 的字段的元数据，此字段为Public修饰符
	var fieldInfo = typeof(T).GetField(name, BindingFlags.Instance | BindingFlags.Public);
	
	if (fieldInfo == null)
	{
		throw new Exception(string.Format("Unable to find bindableproperty field '{0}.{1}'", typeof(T).Name, name));
	}

	_binders.Add(viewmodel =>
	{
		GetBindablePropertyFromViewModel<TProperty>(name, viewmodel, fieldInfo).OnValueChanged += valueChangedHandler;
	});

	_unbinders.Add(viewModel =>
	{
		GetBindablePropertyFromViewModel<TProperty>(name, viewModel, fieldInfo).OnValueChanged -= valueChangedHandler;
	});
}

private BindableProperty<TProperty> GetBindablePropertyFromViewModel<TProperty>(string name, T viewModel,FieldInfo fieldInfo)
{
	// 从 viewModel 的实例中，得到 fieldInfo 字段对应的实例
	var value = fieldInfo.GetValue(viewModel);
	
	BindableProperty<TProperty> bindableProperty = value as BindableProperty<TProperty>;
	if (bindableProperty == null)
	{
		throw new Exception(string.Format("Illegal bindableproperty field '{0}.{1}' ", typeof(T).Name, name));
	}

	return bindableProperty;
}

动态绑定及其实现

概念：什么是动态绑定

  • 概念：
    • 可以通过 GetMember 或者 GetFields 等方法得到类型当中得成员得元数据信息；
    • 比如：假设得到了 MethodInfo、PropertyInfo 或者 FieldInfo 对象，我们就可以动态对其进行调用，或者得到并设置它们的值。这称为“动态绑定”或者“后期绑定”，它会在运行时 (而不是在编译时) 来决定成员的调用；
  • 示例：
// 静态绑定
string s = "Hello";
int length = s.Length;

// 动态绑定
object s = "Hello";
PropertyInfo propInfo = s.GetType().GetProperty("Length");
int length = (int) propInfo.GetValue(s,null);
  • 解析：
    • 第一步：使用实例的 GetType 成员方法，得到类型元数据；
    • 第二步：对类型使用 GetXXX 方法，得到指定种类（比如属性、字段、方法等）的元数据信息，参数当中传入的是变量的名称；
    • 第三步：使用元数据的 GetValue 方法，第一个参数传入实例，第二个参数指定 GetValue 的一些操作；

概念：设置/获取元数据对于实例的值 - Get/SetValue(实例，访问属性)

  • 介绍：
    • 通过反射获取 PropertyInfo 或 FieldInfo 等元数据信息后，GetValue 和 SetValue 方法与这些成员的实际值进行交互的桥梁；
  • 概念：GetValue 方法

    • 定义：

      • object GetValue(object obj)​：获取指定对象 obj 的属性或字段的值；
      • object GetValue(object obj, object[] index)​：主要用于获取索引属性​（C# 中的索引器）的值。index 参数是一个对象数组，用于传递索引值。对于非索引属性，比如属性，此参数应传入 null

    • 返回：

      • 返回类型为 object。如果获取的是值类型（如 int, bool），返回值将是该值类型的装箱对象；

    • 示例：

      • using System;
using System.Reflection;

      public class MyClass
      {
      public string InstanceProperty { get; set; } = "Hello";
      public static int StaticProperty { get; set; } = 42;
      private string _privateField = "Private Info";
      }

      // 获取实例属性值
      MyClass obj = new MyClass();
      PropertyInfo instanceProp = typeof(MyClass).GetProperty("InstanceProperty");
      object value1 = instanceProp.GetValue(obj); // value1 包含 "Hello"

      // 获取静态属性值
      PropertyInfo staticProp = typeof(MyClass).GetProperty("StaticProperty");
      // 注意：静态变量，不需要在GetValue当中传入实例，可以传入null表示当前获取的是静态变量
      object value2 = staticProp.GetValue(null); // obj 为 null，value2 包含 42

      // 获取私有字段值（需要设置绑定标志）
      FieldInfo privateField = typeof(MyClass).GetField("_privateField", BindingFlags.NonPublic | BindingFlags.Instance);
      object value3 = privateField.GetValue(obj); // value3 包含 "Private Info"
      ```

  • 概念：SetValue 方法
    • 定义：
      • void SetValue(object obj, object value)​：将指定对象 obj 的属性或字段设置为 value；
      • void SetValue(object obj, object value, object[] index)​：用于设置索引属性的值。index 参数用于传递索引值；
    • 注意：
      • 参数 obj 含义同 GetValue，实例成员传对象，静态成员传 null；
      • value 表示要设置的新值，类型为 object。方法内部会尝试将其转换为目标属性/字段的实际类型。如果类型不兼容，会抛出 ArgumentException；

元数据的筛选器：BindingFlags

概念：元数据的筛选器 - BindingFlags

  • 概念：
    • 类型上所有检测元数据的方法，例如 GetProperty、GetField 等，都含有使用 BindingFlags 枚举的重载方法，而这个枚举参数就是元数据的筛选器，使用它可以更改默认的筛选标准；
    • 它的常见用法是检索非公有成员 (仅仅适用于桌面应用程序)；
  • 分类：BindingFlags 枚举
  • 示例：常见组合
      1. 获取所有成员：BindingFlags.Public | BindingFlags.NonPublic | BindingFlags.Instance | BindingFlags.Static
      • 这是一个非常宽泛的搜索条件，它会返回指定类型的所有成员，无论其可访问性（公共或非公共）还是类型（实例或静态）
      1. 获取公共实例成员：BindingFlags.Public | BindingFlags.Instance
      • 搜索所有公共的实例成员；
      1. 获取非公共静态成员：BindingFlags.NonPublic | BindingFlags.Static
      1. 访问私有实例方法：BindingFlags.NonPublic | BindingFlags.Instance | BindingFlags.InvokeMethod
      • 此组合允许你调用一个对象的非公共实例方法，可以用于单元测试，用于测试类的私有方法逻辑；
  • 注意：
    • 像 Type.GetMethod(string name) 这样的重载（不接收 BindingFlags 参数），其默认行为等价于只指定了 Public | Instance | Static，也就是说，它默认会搜索公共的实例方法和静态方法，但不会搜索非公共方法；

Assembly 类

概念：Assembly 类

  • 概念：程序集
    • Assembly 是 .NET 应用程序的基本部署单元，通常以 .dll（类库）或 .exe（可执行文件）形式存在。它不仅是编译输出的物理文件，更是一个包含代码、资源和元数据的逻辑单元；
    • .NET 中的 Assembly 主要分为两类：
        1. 私有：仅供单个应用程序使用，通常存放在应用程序目录中；
        1. 公有：可被多个应用程序共享，通常存放在全局程序集缓存 (GAC) 中；
    • 全局程序集缓存(GAC)是 .NET 框架提供的特殊文件夹，用于存储共享 Assembly，支持不同版本的同一 Assembly 共存，避免 DLL Hell 问题；
  • 概念：程序集类；
    • 主要用来加载其它程序集，加载后才能用 Type 来使用其它程序集中的信息；
    • 如果想要使用不是自己程序集中的内容，需要先加载程序集比如 dll 文件 (库文件)；
    • 简单的把库文件看成一种代码仓库，它抓供给使用者一些可以直接拿来用的变量、函数或类；

模块

概念：模块

  • 概念：
    • 调用多模块程序集对象的 GetTypes 方法将会返回所有模块中的所有类型，但是，当模块之间具有关联性时，就应该对元数据的令牌进行处理；
    • 元数据令牌是一个整数，它对于模块范围内的每一个类型、成员、字符串或者资源都是唯一的；
    • IL 会使用这些元数据令牌，因此在解析 IL 的过程中，需要解析这些元数据令牌，相应的方法定义在 Module 类型的 ResolveType、ResolveMember、ResolveString 和 ResolveSignature 中；
  • 方法：
    • GetModules 方法：可以获得程序集中的所有模块列表，此外还可以通过 ManifestModule 属性直接访问程序集的主模块；
  • 代码：
using System;
using System.Reflection;
using System.Linq;

namespace ModuleReflectionDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            // 1. 加载当前程序集（也可以是外部多模块程序集）
            Assembly assembly = Assembly.GetExecutingAssembly();
            Console.WriteLine($"程序集全名: {assembly.FullName}");

            // 2. 获取指定的程序集中的所有模块
            Module[] modules = assembly.GetModules();
            Console.WriteLine($"模块数量: {modules.Length}");

            // 3. 访问当前应用程序的主模块（ManifestModule）
            Module mainModule = assembly.ManifestModule;
            Console.WriteLine($"主模块名称: {mainModule.Name}");

            // 4. 遍历所有模块并分析模块内的类型
            foreach (Module module in modules)
            {
                Console.WriteLine($"\n--- 模块 {module.Name} 分析开始 ---");
                
                // 获取模块中的所有类型
                Type[] types = module.GetTypes();
                Console.WriteLine($"模块中的类型数: {types.Length}");

                // 示例：解析模块中第一个类型的元数据令牌（如果存在）
                if (types.Length > 0)
                {
                    Type sampleType = types[0];
                    int typeToken = sampleType.MetadataToken; // 获取元数据令牌
                    
                    Console.WriteLine($"类型: {sampleType.Name}, 元数据令牌: 0x{typeToken:X}");
                    
                    // 使用元数据令牌解析类型（验证令牌有效性）[5](@ref)
                    Type resolvedType = module.ResolveType(typeToken);
                    Console.WriteLine($"令牌解析结果: {resolvedType?.Name}");
                }

                // 5. 演示多模块场景下的类型关联性处理
                if (module != mainModule)
                {
                    Console.WriteLine("处理辅助模块中的类型关联性...");
                    // 此处可添加针对模块间依赖关系的特殊处理逻辑
                }
            }

            // 6. 综合示例：通过模块解析方法成员
            DemonstrateMemberResolution(assembly);
        }

        static void DemonstrateMemberResolution(Assembly assembly)
        {
            // 获取第一个模块中的第一个类型
            Module module = assembly.Modules.First();
            Type targetType = module.GetTypes().FirstOrDefault(t => t.GetMethods().Length > 0);
            
            if (targetType != null)
            {
                MethodInfo sampleMethod = targetType.GetMethods().First();
                int methodToken = sampleMethod.MetadataToken;
                
                Console.WriteLine($"\n[成员解析示例]");
                Console.WriteLine($"方法: {sampleMethod.Name}, 令牌: 0x{methodToken:X}");
                
                // 通过模块解析方法成员[5](@ref)
                MemberInfo resolvedMember = module.ResolveMember(methodToken);
                Console.WriteLine($"解析后的成员: {resolvedMember?.Name}");
            }
        }
    }

    // 示例类1
    public class SampleClass1
    {
        public void MethodA() => Console.WriteLine("MethodA called");
    }

    // 示例类2
    public class SampleClass2
    {
        public int MethodB(int x) => x * 2;
    }
}

动态生成代码

什么是动态生成代码

概念：什么是动态生成代码

  • 概念：
    • 动态代码生成指的是在程序运行时创建并编译新的类、方法或整个程序集的能力，而非在传统的编译时（compile-time）预先写好；
    • 这与使用反射 Reflection 来“查看”已有元数据不同，它是反向的“创造”过程；
  • 概念：Emit 技术
    • System.Reflection.Emit 命名空间包含了可以在运行时创建元数据和 IL 的类型，System.Reflection.Emit 命名空间下的类来动态生成中间语言（IL）代码的技术
    • 可以通过 Emit 代码向它发出指令（定义程序集、模块、类型、方法），它就在内存中为你实时制造出新的、可用的类型和方法；
  • 概念：多种动态生成代码的方法

技术原理：IL 与评估栈

概念：技术原理 - IL 与评估栈

  • 概念：
    • 编写的 C# 代码首先被编译器编译为 IL 语言​，这是一种与处理器无关的指令集。程序运行时，.NET 的即时编译器（JIT）再将 IL 编译成本地机器码执行；
    • 而 IL 是一种基于栈的语言。大部分 IL 指令的工作方式都是从评估栈上取出操作数，执行操作，再将结果压回栈顶；
    • 例如，执行加法 a + b 的 IL 指令流程是：
      • ldarg.1：将第一个参数 a 压入栈；
      • ldarg.2：将第二个参数 b 压入栈；
      • add：从栈顶弹出 a 和 b，相加，将结果压回栈顶；
      • ret：从栈顶弹出结果并返回；
  • 概念：Emit 与 IL 的关系
    • ILGenerator 类就是用来生成这些 IL 指令的工具，通过调用它的 Emit 方法，传入不同的 OpCodes（操作码），来一步步“编织”出方法的逻辑；

使用 DynamicMethod 生成 IL

概念：使用 DynamicMethod 生成 IL

  • 概念：
    • DynamicMethod 类是位于 System.Reflection.Emit 命名空间的一个轻量级工具类，该类可以在运行时生成方法；
    • 与 Type Builder 不同，它不需要首先创建包含该方法的程序集、模块和类型；
    • 这个特点使其非常适用于完成简单的任务，因此我们将其作为介绍 Reflection.Emit 的起点；
  • 代码：
// 引入必要的命名空间
using System;
using System.Reflection.Emit;

public class DynamicMethodDemo
{
    public static void Main()
    {
        // 1. 创建一个 DynamicMethod 实例
        // 参数含义："Foo"（方法名）, null（返回类型）, null（参数类型）, typeof(Program)（关联的类型）
        var dynamicMethod = new DynamicMethod("SayHello", typeof(void), null, typeof(DynamicMethodDemo));

        // 2. 获取 IL 生成器
        ILGenerator il = dynamicMethod.GetILGenerator();

        // 3. 开始生成 IL 指令
        il.Emit(OpCodes.Ldstr, "Hello, World from DynamicMethod!"); // 将字符串压入评估栈
        il.Emit(OpCodes.Call, typeof(Console).GetMethod("WriteLine", new[] { typeof(string) })); // 调用 Console.WriteLine
        il.Emit(OpCodes.Ret); // 方法返回

        // 4. 调用动态生成的方法
        // 第一个参数为 null 因为这是静态方法
        dynamicMethod.Invoke(null, null); 
        // 输出：Hello, World from DynamicMethod!
    }
}

使用 Reflection.Emit 创建动态类和方法

概念：使用 Reflection.Emit 创建动态类和方法

  • 代码：
using System;
using System.Reflection;
using System.Reflection.Emit;

public class FullEmitDemo
{
    public static void Main()
    {
        // 1. 定义动态生成的程序集和模块
        AssemblyName assemblyName = new AssemblyName("MyDynamicAssembly");
        AssemblyBuilder assemblyBuilder = AssemblyBuilder.DefineDynamicAssembly(assemblyName, AssemblyBuilderAccess.Run);
        ModuleBuilder moduleBuilder = assemblyBuilder.DefineDynamicModule("MainModule");

        // 2. 在模块中定义一个新的公共类 "Calculator"
        // 使用 TypeBuilder 类型来装类型创造的容器，其定义在 moduleBuilder 模块中，类型修饰符为公开
        TypeBuilder typeBuilder = moduleBuilder.DefineType("Calculator", TypeAttributes.Public);

        // 3. 在类中定义一个公共方法 "Add"
        MethodBuilder methodBuilder = typeBuilder.DefineMethod(
            "Add",   // 定义方法名
            MethodAttributes.Public,  // 方法的访问修饰符
            typeof(int), // 方法的返回类型为 int
            new Type[] { typeof(int), typeof(int) } // 方法的两个 int 类型的参数
        );

        // 4. 获取方法的 IL 生成器，并编写方法体
        ILGenerator il = methodBuilder.GetILGenerator();
        il.Emit(OpCodes.Ldarg_1); // 加载第一个参数 (a)
        il.Emit(OpCodes.Ldarg_2); // 加载第二个参数 (b)
        il.Emit(OpCodes.Add);     // 弹出当前IL栈上的两个参数，执行加法操作，并将结果保存在栈上
        il.Emit(OpCodes.Ret);     // 返回结果

        // 5. 创建类型信息，使其可用
        Type calculatorType = typeBuilder.CreateType();

        // 6. 使用动态创建的类
        object calculatorInstance = Activator.CreateInstance(calculatorType);
        MethodInfo addMethod = calculatorType.GetMethod("Add");
        int result = (int)addMethod.Invoke(calculatorInstance, new object[] { 10, 20 });

        Console.WriteLine($"10 + 20 = {result}"); // 输出：10 + 20 = 30
    }
}
