Lecture 2：编队移动转向系统

协同移动理论

概念：协同移动基础概念

  • 引入：
    • 游戏越来越需要角色群体以协同的方式移动，协同移动可分为两个层级：
      • 个体可以做出互补的决策，让自身的移动看起来协调统一；
      • 也可以作为一个整体做出决策，按照预设规则以协同群体的形式移动；
    • 此部分仅讨论在已确定需要共同移动的前提下，让角色群体保持凝聚力的移动方法，这类方法通常被称为编队移动；
  • 概念：
    • 编队移动是指一组角色在移动过程中保持特定群体组织形态的移动方式。最简单的编队移动，是按照固定的几何阵型（如 V 形、横队线形）移动，但编队移动的应用场景不限于此。编队也可以结合环境进行设计，例如，角色小队仅需对编队转向做少量修改，就能以编队形式在各个掩护点之间移动；
    • 编队移动广泛应用于团队体育类游戏、小队制游戏、即时战略游戏，同时也被越来越多的第一人称射击游戏、驾驶游戏和动作冒险游戏采用。它是一种简单灵活的技术，相比协同战术决策，编队移动的代码编写和执行效率更高，还能实现更稳定的行为表现；

三类编队方式

固定编队

概念：固定编队

  • 概念：
    • 最简单的编队移动类型，采用固定几何阵型的编队。一个编队由一组槽位定义，槽位即角色可以站位的位置，图展示了军事题材游戏中几种常见的编队阵型；
    • 编队中会有一个槽位被标记为领队槽位，编队中其他所有槽位，都相对于该领队槽位进行定义。实际上，领队槽位定义了整个编队的位置与朝向的“零点基准”。处于领队位置的角色，和非编队角色一样在游戏世界中移动。它可以由任意转向行为控制，可以跟随固定路径，也可以通过管线式转向系统融合多种移动需求。无论采用哪种移动机制，领队角色都无需考虑自身处于编队中的定位；
    • 整个编队阵型在游戏世界中的位置与朝向，会始终跟随领队：保证领队处于其对应的槽位中，面朝预设的方向。当领队移动时，整个阵型也会在游戏世界中同步移动、旋转，阵型中的每个槽位也会随之同步移动和旋转；
    • 编队中其余的每个槽位，都可以由一个额外的角色填充。每个角色的位置可以直接通过编队的几何结构确定，无需自身的运动学系统或转向系统。通常，槽位中角色的位置和朝向会被直接设置；

    • 实际应用中，受限于对领队移动的这些约束，这类编队仅能满足非常简单的编队需求，例如在玩家可操控上万个单位的策略游戏中，控制小型步兵小队的移动；

可伸缩编队

概念：可伸缩编队

  • 概念：
    • 在很多场景中，编队的具体结构会取决于参与编队的角色数量。例如防御环形编队，20 个防御者的阵型会比 5 个防御者的阵型更宽；如果有 100 个防御者，还可以将编队设计为多层同心环形结构；
    • 可伸缩编队的实现，通常不会显式定义槽位的位置和朝向列表。例如，可以通过一个函数，根据编队中的总角色数量，动态返回对应的槽位位置。
    • 这种隐式的可伸缩编队，在《家园》（Relic Entertainment, 1999）中有着非常典型的应用。当有新的舰船加入编队时，编队会自动适配，调整槽位的分布以容纳新成员。和前文的示例不同，《家园》采用了更复杂的算法来实现编队的整体移动。

涌现式编队

概念：涌现式编队

  • 概念：
    • 涌现式编队为编队的可伸缩性提供了另一种解决方案。每个角色都拥有基于抵达行为的独立转向系统，角色会根据群体中其他角色的位置，选择自己的目标位置；
    • 以创建大型 V 形编队为例：我们可以让每个角色选择前方的另一个目标角色，然后将转向目标设定在该角色的后方侧方位置；如果已有其他角色选择了该目标，就重新选择；同理，如果已有其他角色的目标位置和当前位置非常接近，也会继续寻找新的目标。
    • 目标位置一旦选定，就会在后续所有帧中沿用，并根据目标角色的位置和朝向实时更新。如果目标位置变得无法抵达，例如目标位置进入了墙体内部，则会重新选择新的目标。
    • 最终，这套机制会让整个群体自发形成 V 形编队。如果编队成员数量很多，V 形的间隙中还会填充出更小的 V 形结构。如图所示，无论编队中的角色数量多少，最终都会呈现出非常明显的箭头形整体效果，图中的线条连接了每个角色与其跟随的目标角色；
  • 核心特点：
    • 这种方法无需定义整体的编队几何结构，群体也不一定需要领队，当然，如果群体中有一个成员不需要根据其他成员调整自身位置，会更利于编队的稳定。
    • 编队是从每个角色的个体规则中自发涌现出来的，和前文群集行为从每个群体成员的转向行为中涌现的原理完全一致；
    • 该方法的另一个优势，是允许每个角色独立对障碍物和潜在碰撞做出反应。在考虑转向或墙体躲避时，无需将编队的整体尺寸纳入考量，因为编队中的每个个体都会做出合理的行为，只要其转向系统中包含了对应的躲避行为；
  • 局限性：
    • 尽管这种方法简单有效，但很难通过调整规则精准得到想要的阵型。在前文的 V 形编队示例中，经常会有多个角色在 V 形的中心位置争抢站位；如果每个角色的目标选择出现偏差，同样的规则可能会让编队最终形成一条长长的斜线，完全看不到预期的 V 形特征。
    • 和所有涌现式行为一样，涌现式编队的调试难度很高。其最终的整体效果往往是可控的无序状态，而非精准的编队移动。对于军事类群体而言，这种无序的特性让涌现式编队几乎没有实际应用价值；

双层编队转向

双层编队转向基础概念

概念：双层编队转向

  • 概念：
    • 我们可以通过双层转向系统，将严格的几何编队和涌现式方法的灵活性结合起来。和前文一样，我们采用由固定槽位阵型定义的几何编队，初始阶段我们先假设编队存在领队角色，后续会移除这一限制；
    • 这种方法不会直接将角色放置到对应的槽位中，而是沿用涌现式的思路：将槽位作为抵达行为的目标位置，角色可以拥有自己的碰撞躲避行为，以及其他所需的复合转向逻辑；
    • 这种方法被称为双层转向，因为它包含两个依次执行的转向系统：
      • （1）首先，领队控制编队阵型的整体移动；
      • （2）然后，编队中的每个角色通过转向来保持在阵型的槽位中；
    • 只要领队不以最大速度移动，每个角色在兼顾环境、保持自身处于槽位的过程中，就会拥有一定的调整空间；
  • 图示：
    • 如图展示了多个智能体以 V 形编队穿过树林的场景，图中可以看到清晰的 V 形特征，但每个角色都为了避开树木，相对槽位位置有轻微的偏移；
    • 即使角色要抵达的槽位暂时无法到达，其转向算法也能保证角色做出合理的行为；

双层编队转向系统优化

概念：双层编队转向系统优化

  • 核心优化 1：移除领队
    • 在前文的示例中，如果领队需要横向移动躲避树木，编队中所有的槽位都会随之横向偏移，其他所有角色也会跟着横向移动来跟上槽位。这种效果看起来会很怪异，因为尽管其他角色在很大程度上可以自主处理障碍物，但它们的行为会完全复刻领队的动作；
    • 我们可以将引导编队的职责从领队身上移除，让所有角色都以相同的方式响应自己的槽位。此时，编队的整体移动由一个隐形领队控制：这是一个独立的转向系统，它控制整个编队的移动，但不控制任何单个角色，这就是双层编队的第二层转向逻辑；
    • 由于这个新的领队是隐形的，它无需担心小型障碍物、与其他角色的碰撞，或是小型的地形特征。这个隐形领队在游戏世界中依然有固定的位置，这个位置会被用于排布编队阵型，确定所有角色对应的槽位位置；但阵型中领队槽位的位置，不会对应任何实际的角色。由于它不再起到槽位的作用，我们将这个位置称为阵型的锚点；
    • 为编队设置独立的转向系统，通常会简化代码实现。我们不再需要为不同角色设置不同的职责，也无需担心当某个角色阵亡时，需要让另一个角色接管领队的职责；
    • 同时，锚点的转向逻辑通常可以做简化处理。例如，在户外场景中，我们可能只需要使用一个高层级的抵达行为，或是路径跟随行为；在室内场景中，锚点的转向依然需要考虑大型障碍物，比如墙体。如果编队直接穿墙而过，会导致所有角色被困，无法跟上对应的槽位；
  • 核心优化 2：编队移动的速度调节
    • 问题
      • 到目前为止，信息只有单向的流动：从编队传递到编队内的角色。当我们使用双层转向系统时，这种单向流动会引发问题：编队可能会一直向前移动，完全无视编队内的角色已经难以跟上的情况；
      • 当编队由实际角色领队时，这个问题并不突出，因为编队中其他角色遇到的困难，领队大概率也会遇到。但当我们直接控制锚点移动时，锚点通常可以忽略小型障碍物和其他角色，而编队内的角色需要绕开这些障碍物，其实际移动耗时会远高于预期，最终导致编队和角色的位置严重不同步；
    • 解决：
      • 解决方案 1：降低编队的移动速度。这是一个常用的经验法则，是将编队的最大速度设置为角色最大速度的一半左右。但在较为复杂的环境中，所需的减速幅度是无法预测的，为了少数高速移动会出问题的场景，就让整个游戏的编队移动都保持低速，是非常不合理的；
      • 解决方案 2：根据编队槽位内角色的当前位置，调节编队的移动速度。这实际上就是给锚点加上一个“牵引绳”，如果槽位内的角色难以抵达目标位置，整个编队就应该放慢速度，给角色跟上的机会；
    • 具体实现方式：
      • 每帧都重置锚点的运动学参数，将锚点的位置、朝向、速度、旋转速度，都设置为所有槽位内角色对应属性的平均值；
      • 如果锚点的转向系统先执行，它会向前移动一小段距离，带动所有槽位向前移动，进而迫使角色也向前移动；当所有槽位内的角色完成移动后，锚点会被往回拉，避免它向前移动得太远；
      • 由于每帧都会重置锚点的位置，当角色执行转向时，目标槽位只会在角色前方一小段距离。使用抵达行为时，角色对于移动这么短的距离会非常轻松，槽位内角色的移动速度也会随之降低；这反过来又会降低编队的整体速度，因为编队速度是槽位内角色移动速度的平均值。在下一帧，编队的速度会进一步降低，几帧之后，编队就会减速到完全停止；

补充：锚点、执行与寻路路径

  • 分析：
    • 双层编队转向系统中，锚点等于质心与编队沿路径移动并不矛盾。关键在于：锚点的“目标位置”由路径点决定，而锚点的“实际位置”则受编队质心约束并通过速度调节机制动态调整；
    • 锚点的目标位置（Target）：由高层寻路系统（如 A*、路径点序列）计算得出，代表编队整体想要到达的位置；
    • 锚点的实际位置（Actual）：每帧根据公式 ​计算得出，其中 pc​是编队质心，vc​是质心速度。这个实际位置是编队当前状态的反映，用于确定槽位坐标；
  • 特点：
    • 路径跟随与编队稳定性的平衡：锚点的高层转向系统确保编队整体沿路径移动；而锚点实际位置与质心的绑定（通过速度调节）确保了编队不会因角色掉队而分散；
    • 自然避障：角色在向槽位移动时，各自的避障行为可以处理局部障碍，而编队整体路径只需考虑大型障碍（由高层寻路处理）；

协同移动实现

两级编队系统实现

概念：两级编队系统的核心实现

  • 概念：
    • 我们可实现一套完整的两级编队系统，该系统以编队管理器为核心，负责处理编队模式规则，为占据对应槽位的角色生成目标位姿；
    • 编队管理器的核心能力覆盖角色与槽位的全生命周期管理、编队漂移补偿、槽位目标位姿的实时计算与下发；
  • 核心实现逻辑：
    • （1）槽位分配更新：
      • 采用极简的顺序分配算法，为列表中的每个分配项依次分配连续的槽位编号，同步计算并更新当前已填充槽位对应的编队漂移偏移量；
    • （2）角色添加逻辑：
      • 先校验编队模式是否支持新增槽位，支持则为新角色创建槽位分配项，更新全量槽位分配后返回成功；不支持则直接返回添加失败；
    • （3）角色移除逻辑：
      • 查找目标角色对应的槽位，校验槽位有效性后移除该分配项，再更新剩余角色的槽位分配；
    • （4）槽位目标更新逻辑：
      1. 获取编队的锚点位姿，将锚点的朝向转换为变换矩阵；
      2. 遍历所有槽位分配项，获取对应槽位相对于锚点的本地位姿；
      3. 通过锚点的变换矩阵，将槽位本地位姿转换为世界坐标系下的绝对位姿；
      4. 叠加漂移偏移补偿，消除编队整体的位置与朝向偏差；
      5. 将最终计算出的目标位姿下发给对应角色，作为其转向行为的目标；

    • 编队锚点获取：getAnchorPoint

            self.slotNumber = 0

    def __init__(self):
        # 槽位分配列表
        self.slotAssignments = []
        # 存储当前已填充槽位的漂移偏移量
        self.driftOffset = Static()
        # 编队模式实例
        self.pattern = None

    # 更新角色与槽位的分配关系
    def updateSlotAssignments(self):
        # 极简分配算法：为列表中的每个分配项分配连续的槽位编号
        for i in range(len(self.slotAssignments)):
            self.slotAssignments[i].slotNumber = i
        
        # 更新漂移偏移量
        self.driftOffset = self.pattern.getDriftOffset(self.slotAssignments)

    # 向第一个可用槽位添加新角色，无可用槽位时返回false
    def addCharacter(self, character):
        # 获取当前已占用的槽位数量
        occupiedSlots = len(self.slotAssignments)
        
        # 检查编队模式是否支持新增槽位
        if self.pattern.supportsSlots(occupiedSlots + 1):
            # 创建新的槽位分配项
            slotAssignment = self.SlotAssignment()
            slotAssignment.character = character
            self.slotAssignments.append(slotAssignment)
            
            # 更新槽位分配并返回成功
            self.updateSlotAssignments()
            return True
        
        # 无法添加角色，返回失败
        return False

    # 从槽位中移除指定角色
    def removeCharacter(self, character):
        # 查找角色对应的槽位
        slot = None
        for idx, assignment in enumerate(self.slotAssignments):
            if assignment.character == character:
                slot = idx
                break
        
        # 校验槽位有效性
        if slot is not None and 0 <= slot < len(self.slotAssignments):
            # 移除该槽位分配项
            self.slotAssignments.pop(slot)
            # 更新槽位分配
            self.updateSlotAssignments()

    # 获取编队锚点位姿（可根据需求重写）
    def getAnchorPoint(self):
        # 默认实现：返回编队质心
        anchor = Static()
        if not self.slotAssignments:
            return anchor
        
        # 根据平均位置和平均方向计算锚点的位姿
        for assignment in self.slotAssignments:
            anchor.position += assignment.character.position
            anchor.orientation += assignment.character.orientation
        
        count = len(self.slotAssignments)
        anchor.position /= count
        anchor.orientation /= count
        return anchor

    # 为每个角色更新槽位目标位置
    def updateSlots(self):
        # 获取编队锚点位姿
        anchor = self.getAnchorPoint()
        # 将锚点朝向转换为变换矩阵
        orientationMatrix = anchor.orientation.asMatrix()
        
        # 遍历所有角色
        for i in range(len(self.slotAssignments)):
            # 获取槽位相对于锚点的本地位姿
            relativeLoc = self.pattern.getSlotLocation(self.slotAssignments[i].slotNumber)
            
            # 通过锚点变换，将本地位姿转换为世界位姿
            location = Static()
            
            # 矩阵变换计算位置
            loc_x = relativeLoc.position.x * orientationMatrix[0][0] + relativeLoc.position.z * orientationMatrix[0][2]
            loc_z = relativeLoc.position.x * orientationMatrix[2][0] + relativeLoc.position.z * orientationMatrix[2][2]
            location.position = Vector3(loc_x, 0, loc_z) + anchor.position
            
            # 叠加朝向
            location.orientation = anchor.orientation + relativeLoc.orientation
            
            # 叠加漂移偏移补偿
            location.position -= self.driftOffset.position
            location.orientation -= self.driftOffset.orientation
            
            # 将目标位姿写入角色
            self.slotAssignments[i].character.setTarget(location)

概念：示例编队模式——防御环形编队

  • 概念：
    • 防御环形编队会将角色均匀布置在圆周上，使角色背部朝向圆心，可支持任意数量的角色，是最常用的防御类编队模式；
    • 该编队模式的核心逻辑包含槽位数量计算、漂移偏移量计算、槽位位姿计算、槽位数量支持判断四个模块。
  • 核心实现逻辑：
    • 槽位数量计算：
      • 通过槽位分配数据，获取当前已占用的最高槽位编号，加 1 后得到所需的总槽位数量；若使用前文的顺序分配算法，可直接简化为返回槽位分配项的总数；
    • 漂移偏移量计算：
      • 遍历所有槽位分配项，累加各槽位的位置与朝向，求平均值后得到编队的质心，即需要补偿的漂移偏移量；
    • 槽位位姿计算：
      1. 根据槽位编号与总槽位数量，计算该槽位在圆周上的对应角度；
      2. 根据角色半径与总槽位数量，计算环形编队的半径，确保角色之间无间隙贴合；
      3. 基于圆周角度与编队半径，计算槽位的相对位置；
      4. 设置槽位朝向为圆周向外的方向，确保角色背对圆心；
    • 槽位支持判断：
      • 该编队模式无槽位数量上限，始终返回支持；

# 防御环形编队模式类
class DefensiveCirclePattern(FormationPattern):
    def __init__(self):
        super().__init__()
        # 单个角色的半径，用于计算环形编队的最小半径
        self.characterRadius = 1.0

    # 非接口方法：根据槽位分配数据计算编队所需的槽位数量
    def calculateNumberOfSlots(self, assignments):
        # 找到已占用的最高槽位编号
        filledSlots = 0
        for assignment in assignments:
            if assignment.slotNumber > filledSlots:
                filledSlots = assignment.slotNumber
        
        # 最高槽位编号+1为总槽位数量
        self.numberOfSlots = filledSlots + 1
        return self.numberOfSlots

    # 计算编队的漂移偏移量
    def getDriftOffset(self, assignments):
        # 先更新槽位数量
        self.calculateNumberOfSlots(assignments)
        
        # 初始化质心
        center = Static()
        center.position = Vector3.zero
        center.orientation = 0.0

        # 遍历所有槽位分配项，累加位置与朝向
        for assignment in assignments:
            location = self.getSlotLocation(assignment.slotNumber)
            center.position += location.position
            center.orientation += location.orientation
        
        # 求平均值得到质心（漂移偏移量）
        numberOfAssignments = len(assignments)
        center.position /= numberOfAssignments
        center.orientation /= numberOfAssignments
        return center

    # 按照编队的要求，计算指定slotNumber的槽位的本地位姿
    def getSlotLocation(self, slotNumber):
        # 槽位数量为0时直接返回原点
        if self.numberOfSlots <= 0:
            return Static()
        
        # 根据槽位编号计算圆周角度
        angleAroundCircle = (slotNumber / self.numberOfSlots) * math.pi * 2
        
        # 计算环形编队半径，确保角色之间无间隙
        radius = self.characterRadius / math.sin(math.pi / self.numberOfSlots)
        
        # 初始化槽位位姿
        location = Static()
        # 计算圆周位置
        location.position.x = radius * math.cos(angleAroundCircle)
        location.position.z = radius * math.sin(angleAroundCircle)
        # 朝向设置为圆周向外
        location.orientation = angleAroundCircle
        
        return location

    # 判断是否支持指定数量的槽位，该编队支持任意数量槽位
    def supportsSlots(self, slotCount):
        return True

概念：槽位角色与优化分配方案

  • 概念：槽位角色分配基础
    • 此前的编队系统默认任意角色均可占据所有槽位，但实际游戏中，大量编队会为不同槽位设定专属职责，例如军事模拟游戏的步枪小队，会将步枪手、掷弹兵、机枪手、小队长安排在固定位置；即时战略游戏中，也通常会将重型火炮部署在防御编队的中心，将机动步兵部署在先锋位置；
    • 为编队槽位设定角色规则后，仅符合角色要求的单位可占据对应槽位；无论是否使用槽位角色，都需要一套合理的分配机制，避免角色进入编队时出现混乱穿插的情况；无角色限制的槽位分配难度较低，加入角色限制后，分配会成为一个复杂问题，游戏开发中通常采用简化方案即可获得足够好的效果；
  • 概念：硬角色与软角色规则
    • 硬角色规则：
      • 为编队槽位设定固定的角色准入要求，角色仅能占据自身可胜任角色对应的槽位；
      • 示例：RPG 游戏的地牢探索编队中，设定“近战”“远程”“法术”三类槽位，战士仅能进入近战槽位，法师仅能进入法术槽位，弓箭手仅能进入远程槽位；
      • 局限性：当队伍角色构成与编队槽位角色不匹配时，会出现槽位无法被填充的情况，需要为不同的队伍配置设计大量专属编队，开发成本高；
    • 软角色规则：
      • 为工程化折中方案，打破硬角色的绝对限制；
        • 不为角色设定可胜任的角色列表，而是为每个角色设定对应每类槽位的槽位成本，成本越高代表该角色越不适合占据此槽位；
      • 核心规则：
        1. 理想适配的槽位，成本设为 0；完全无法适配的槽位，成本设为极大值（不使用数据类型上限，避免累加时出现数值溢出）；
        2. 分配算法的核心目标是，将角色分配到槽位后，总槽位成本最小化；
        3. 当没有理想适配的槽位时，角色仍可被分配到非最优槽位，避免出现角色无槽位可去的情况；
      • 优势：
        • 当队伍配置与编队槽位角色匹配时，效果与硬角色完全一致；
        • 当配置不匹配时，也能完成有效分配，不会出现系统失效的情况，大幅降低编队设计的开发成本；

槽位分配算法

概念：槽位分配问题基础

  • 槽位分配在游戏中触发频率很低，通常仅在无序角色组被分配到编队、或是战术移动中角色需要主动切换槽位时执行，绝大多数时间角色仅需跟随自身槽位的目标移动；
  • 槽位分配属于 NP 完全问题：将 k 个角色分配到 n 个槽位，可能的分配方案数量为排列数
  • 因此 20 个角色分配到 20 个槽位时，方案数接近 2500 万亿，无法通过穷举所有方案找到全局最优解；
  • 游戏工程中采用启发式算法，无法保证得到全局最优解，但能在极短时间内得到足够优质的分配结果，完全满足游戏运行需求。

概念：启发式分配核心算法

  • 基础逻辑：
    • 按顺序遍历每个角色，为其分配当前剩余的、成本最低的槽位；
    • 该方案存在缺陷，若将灵活度高的角色优先分配，可能会导致灵活度低的角色最后无合适槽位可去；
  • 优化方案：
    • 优先分配灵活度最低、最难找到合适槽位的角色，最后分配灵活度高的角色；

    4. 若角色无可用槽位，执行异常处理（如向玩家报错）；
    5. 所有角色分配完成后，更新编队的槽位分配数据。

概念：通用槽位成本扩展

  • 概念：

    • 可纳入成本计算的常见因素：
      1. 距离成本：角色当前位置到槽位目标位置的距离，距离越远成本越高，可让角色优先选择就近的槽位，减少编队集结时的穿插混乱；
      2. 掩体质量成本：防御编队中，槽位对应的掩体防护效果越差，成本越高，确保优质掩体槽位被优先分配；
      3. 机动能力成本：机动能力弱的单位，远距离移动的成本更高，适配不同机动属性的单位；

        def __init__(self):
            self.character = None
            self.assignmentEase = 0.0
            self.costAndSlots = []

    def updateSlotAssignments(self):
        # 角色数据列表
        characterData = []
        # 成本阈值，超过该值的槽位被视为不可用
        COST_LIMIT = 10000.0

        # 编译每个角色的分配数据
        for assignment in self.slotAssignments:
            # 创建角色数据项
            datum = self.CharacterAndSlots()
            datum.character = assignment.character

            # 遍历所有槽位，筛选可用槽位
            for slot in range(self.pattern.numberOfSlots):
                # 获取角色占据该槽位的成本
                cost = self.pattern.getSlotCost(assignment.character, slot)
                # 过滤不可用槽位
                if cost >= COST_LIMIT:
                    continue

                # 存储槽位信息
                slotDatum = self.CostAndSlot()
                slotDatum.slot = slot
                slotDatum.cost = cost
                datum.costAndSlots.append(slotDatum)

                # 累加角色的分配易度值
                datum.assignmentEase += 1 / (1 + cost)

            # 将角色数据加入列表
            characterData.append(datum)

        # 初始化槽位占用标记数组
        filledSlots = [False] * self.pattern.numberOfSlots
        # 初始化新的分配结果列表
        newAssignments = []

        # 按分配易度值升序排序，最难分配的角色排在最前
        characterData.sort(key=lambda x: x.assignmentEase)

            characterDatum.costAndSlots.sort(key=lambda x: x.cost)

                if not filledSlots[slot]:

                    assignment = self.SlotAssignment()
                    assignment.character = characterDatum.character
                    assignment.slotNumber = slot
                    newAssignments.append(assignment)

                    # 标记槽位为已占用
                    filledSlots[slot] = True
                    assigned = True
                    break

            # 若角色无可用槽位，抛出异常
            if not assigned:
                raise Exception(f"角色{characterDatum.character}无可用槽位，分配失败")

        # 更新槽位分配数据
        self.slotAssignments = newAssignments
        # 更新漂移偏移量
        self.driftOffset = self.pattern.getDriftOffset(self.slotAssignments)
