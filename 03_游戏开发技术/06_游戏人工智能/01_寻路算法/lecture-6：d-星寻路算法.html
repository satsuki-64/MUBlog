Lecture 6：D 星寻路算法

D 星算法基础概念

概念：D 星算法基础概念

  • 概念：
    • D*（Dynamic A*）是一种专为动态、未知环境设计的路径规划算法，尤其适用于机器人在移动过程中遇到障碍物时的高效重规划场景；
  • 特点：
    • 反向搜索：D* 与 A* 最根本的区别在于搜索方向，A* 从起点向终点正向探索，而 D* 是从终点出发，反向计算到所有节点的最优路径代价；
    • 动态更新：当机器人在移动中发现新的障碍物或环境变化时，算法不需要重新规划整条路径，只需对受影响的局部节点进行代价更新，就能快速生成新的绕障路径；
  • 概念：关键参数
    • （1）g(s)：
      • 从目标点反向传播到节点 s 的实际最优代价，代表当前已知的从 s 到目标点的最小路径成本；
    • （2）rhs(s)：
      • 基于后继节点计算出的节点 s 的启发式代价估计，公式为：
      • 其中 s' 是 s 的若干后继节点，比如 s 节点周围的四个可行走节点；
    • （3）h(s)：
      • 在 D* Lite 中，它是节点 s 到机器人当前位置的距离（而非到终点），用于引导反向搜索向机器人靠近；
    • （4）km：
      • 动态补偿量，用于修正机器人移动后启发值的变化，避免因位置改变而重新计算所有节点的启发值；
      • 当机器人从 robot_old 移动到 robot_new 时，所有节点的 h(s) 都会发生变化，如果直接重新计算所有节点的 h(s)，会导致算法效率大幅下降；
      • 因此，km 的作用就是动态补偿这种变化，避免全局重新计算；
    • （5）key：
      • key(s) 是每个节点的优先级排序值，它的作用是维护一个优先队列，让算法能够高效地选择下一个需要处理的节点，从而保证路径规划的动态更新效率；
      • 在优先队列中，key 值越小的节点会被优先处理，当环境变化或机器人移动时，算法会通过 key 值的变化，快速定位需要重新计算代价的节点；

概念：状态不一致

  • 概念：
    • D* Lite 中节点的 “一致状态” 和 “不一致状态” 有严格定义：
    • 一致状态：|g(s) - rhs(s)| < Epsilon（g 和 rhs 值几乎相等）→ 节点的 “已知最优代价” 和 “预估最优代价” 收敛，无需处理；
    • 不一致状态：|g(s) - rhs(s)| ≥ Epsilon → 又分两种
      • 过一致：g(s) > rhs(s)（预估代价更优，需要修正 g）
      • 欠一致：g(s) < rhs(s)（预估代价失效，需要修正 rhs）
    • 只有不一致状态的节点，才需要被加入优先队列；一致状态的节点，算法会确保它不在队列中（或被移出）
  • 分析：不一致节点与优先队列
    • 优先队列在 D* Lite 中的核心角色是「待处理节点池」，算法的目标是通过处理这些节点，让所有相关节点的 g/rhs 收敛到一致状态，最终让起点达到一致（得到最优路径）
    • 具体原因为，不一致意味着节点的代价（g/rhs）还未收敛，基于这个代价计算的路径要么非最优、要么错误；必须把这些节点加入队列，等待处理来修正 g/rhs，让代价可靠；
    • 不一致节点入队后，算法会按 Key 优先级处理它，进而修正其前驱节点的代价：如果不一致节点不入队，代价修正就会中断，无法从终点扩散到起点；
    • 并且由于 D* Lite 是支持环境变化的动态寻路算法，当网格代价改变时，会触发一批节点从 “一致” 变 “不一致”；把这些节点入队，算法才能主动处理环境变化，重新规划路径

核心参数计算逻辑

概念：核心参数计算逻辑

  • 概念：key 的计算
    • 公式：
    • 解释：
      • key 是两个部分构成的“二元组”，计算结构不是一个数；
      • 其规则是：先比 k1，k1 小的优先级更高。然后如果 k1 相同，再比 k2，k2 小的优先级更高；
    • 解释：第一部分 min(g(s), rhs(s)) + h(s, sstart) + km
      • min(g(s), rhs(s))：代表节点 s 的 “有效代价”，取实际代价 g(s) 和启发式代价 rhs(s) 中的较小值；
      • h(s, sstart)：是节点 s 到机器人当前位置 sstart 的启发距离（如曼哈顿距离），用于引导反向搜索向机器人靠近；
      • km：代表累计补偿量，用于动态修正机器人移动后启发值的整体偏移，避免全局重新计算；
    • 解释：第二部分 min(g(s), rhs(s))
      • 这是节点 s 的 “基础代价”，作为优先级比较的次要依据；
      • 当两个节点的第一部分 key 值相同时，基础代价更小的节点会被优先处理；
      • 这个设计确保了算法在处理节点时，会优先关注代价更小、更可能成为最优路径一部分的节点；
  • 分析：为什么要 min(g(s), rhs(s))
    • 单来说，取 min(g(s), rhs(s)) 是为了处理节点的 “不一致” 状态，确保优先队列（Priority Queue）总能选出最需要被修正的节点；
      • g(s)：从节点 s 到目标点的当前记录代价（旧值、缓存值）
      • rhs(s)：根据邻居节点计算出的、从 s 到目标点的一步前瞻代价（新值、理论最优值）
      • rhs(s) = min( g(s') + cost(s, s') ) （s' 是 s 的邻居）
    • 理论状态时，g(s) == rhs(s)。此时节点代价是准确的，无需处理；
    • 但是当环境变化（障碍物出现 / 消失）或机器人移动后，g(s) 和 rhs(s) 不再相等。此时分为两种情况：
        1. 过一致 (Over consistent): g (s) > rhs (s)
        • 之前记录的代价 g(s) 太大了，现在发现了一条更短的路（代价 rhs(s)）
        • 因此需要把 g(s) 降低到 rhs(s) 的水平（更新代价）
        1. 欠一致（Under consistent）：g(s) < rhs(s)
        • 之前计算的路径被堵了或代价变高了，现在实际应该付出的代价 rhs(s) 比记录的 g(s) 更大；
        • 此时等待被重新计算，或者等待邻居更新后再修正自己；
        • 可以理解为：节点 s 的 “预估代价” 远大于 “已知代价”，说明预估代价（rhs）已经失效 / 错误，不再能反映真实的最优代价，同时前驱节点 u（依赖 s 作为最优后继）的 rhs 值也会失效；
        • 因此欠一致时，重置失效的节点 s 的代价，修正所有依赖 s 的前驱节点的预估代价，进而让失效的代价网格重新收敛；
    • 因此取 min(g(s), rhs(s)) 的目的，就是在计算 key 值（即优先级）时，我们取两者的最小值，公平且正确地给这两种 “问题节点” 排序，过一致的时候采用 rhs 的值，欠一致的时候先继续采用 g 的值
  • 特点：
    • 取 min(g(s), rhs(s)) 作为 key 的基础，主要有以下作用：
      1. 优先处理 “变好” 的路径：
      • 当发现捷径（rhs 变小）时，min 会立刻选用这个更小的值，让该节点的 key 变小，从而插队进入优先队列顶部，被优先处理，确保最优路径能被快速更新；
      1. 延迟处理 “变坏” 的路径：
      • 当路径被堵（rhs 变大）时，min 会选用旧的 g 值；
      • 这使得该节点的 key 值不会突然暴增，避免了优先级队列的剧烈震荡。算法会在合适的时候（处理到它时）再去修正它，保证了效率；
  • 概念：km 的定义与更新
    • km 是一个累计值，每次机器人移动时都会更新：
    • 这里的 h(robot_old, robot_new) 通常是机器人新旧位置之间的曼哈顿距离或欧几里得距离；
    • 它类似于 A* 中的 g + h 启发式估计，用于评估节点的整体优先级；

完整算法流程

概念：完整算法流程

  • 概念：
      1. 初始化：
      • km=0，所有的 rhs=∞，g=∞
      • rhs(goal)=0 ，将终点插入优先队列 U。优先队列使用 key 值进行排序。key 值计算公式如上文；
      1. 主函数：
      • 计算初始路径 path = compute_shortest_path()；
      • 令 robot=start, robot_1=robot；
      • 判断机器人位置是否等于终点：
          1. 等于表示到达终点结束算法；
          1. 不等于表示还未到达终点：
          • 执行移动，机器人移动到到它的后继节点 robot=back(robot)
          • 检测环境变化
            • 令计算累计调整量 km=km+h (robot_1, robot)，同时更新 robot_1 = robot
            • 新增障碍物 v：此时，g(v)<rhs(v)，对节点 v 的前驱节点重新计算 rhs；
            • 新减障碍物 v：此时，g(v)>rhs(v)，更新 v 点的前驱节点以及 v 点的 rhs；
            • 重新规划最优路径；
      1. 最优路径计算：compute_shortest_path
      • while 循环；
      • 退出条件：优先队列 U 中没有值，或者起点的键值 key 小于优先队列里面的最小值或者起点的 rhs 大于 g
        • 取出优先队列的堆顶，s_key 以及 s
        • 令 k_old=s_key; k_new = calculate_key(s)
        • 如果 k_old<k_new 表明当前的键值已经过时，将新键值 k_new、s 放入优先队列；
        • 如果 g[s]>rhs[s] 为过一致状态，令 g[s]=rhs[s],收敛节点 s；
        • 如果 g[s]<rhs[s] 为欠一致状态，重置 g[s]=∞，更新以 s 为反向节点的 rhs；

import numpy as np
import heapq

def plan(self):
    """主循环：机器人运动控制+动态环境检测+路径重规划（D* Lite核心主流程）"""
    # 1. 初始化：记录机器人初始位置（robot_old），对应笔记中km更新的旧机器人位置
    robot_1 = self.robot  # robot_1始终表示「上一次更新km时的机器人位置」
    path = [self.start]   # 初始化路径列表，保存机器人走过的所有节点
    
    # 2. 计算初始最短路径：从终点反向搜索到起点的初始最优路径，对应笔记「完整算法流程-初始化+首次路径计算」
    initial_path = self.compute_shortest_path()  
    
    # 3. 主循环：机器人未到达终点则持续移动，退出条件self.robot == self.goal对应笔记「到达终点结束算法」
    while self.robot != self.goal:  
        # 机器人向终点移动一步：self.back[self.robot]是当前节点到终点的最优后继节点（反向搜索的核心）
        # 对应笔记「主函数-执行移动，机器人移动到它的后继节点」
        self.robot = self.back[self.robot]  
        path.append(self.robot)    # 记录当前步的节点，更新路径列表
        
        # 4. 安全检查：判断当前机器人位置是否有可行路径到终点
        # 若 g(robot)=无穷，说明无任何路径可达终点，对应笔记「主函数-无可行路径退出」
        if self.g[self.robot] == np.inf:
            print("无可行路径")
            return None  # 无路径则返回None，终止规划
        
        # 5. 环境变化检测：模拟/实际感知地图中边的代价变化（障碍增/删），返回变化的边字典{节点v: 新状态值}
        # new_val=1表示v变为障碍，new_val≠1表示v的障碍消失（可行走）
        changed_edges = self.detect_changes()
        
        # 6. 若检测到环境变化，执行动态更新+路径重规划，对应笔记「主函数-检测环境变化」
        if changed_edges:
            # 6.1 更新累计补偿量 km，严格对应笔记中 km 的更新公式：km_new = km_old + h(robot_old, robot_new)
            # self.gavl(robot_1, self.robot) 就是启发函数 h(旧机器人位置, 新机器人位置)
            self.km = self.km + self.gavl(robot_1, self.robot)  
            robot_1 = self.robot  # 更新robot_1为当前机器人位置，为下一次km更新做准备
            
            # 6.2 遍历所有发生变化的节点v，分「障碍新增」和「障碍消失」两种情况处理
            for v, new_val in changed_edges.items():
                if new_val == 1:
                    # 情况1：节点v变成障碍物（代价升高），对应笔记「主函数-新增障碍物v」
                    # 遍历 v 的所有前驱节点u（反向搜索中，u 能直接走到v）
                    for u in self.get_pred_neighbors(v):
                        # 若 u 的最优后继节点是 v（back[u]==v），说明 u 的原路径依赖 v，现在 v 成障碍需重新计算 u 的 rhs
                        if u != self.goal and self.back[u] == v:
                            self.update_rhs(u)  # 重新计算u的rhs：rhs(u) = min(g(s')+cost(u,s'))
                        # 对u执行节点更新：重新计算key并维护优先队列，对应笔记中的UpdateVertex函数
                        self._update_vertex(u)
                else:
                    # 情况2：节点v的障碍物消失（代价降低），对应笔记「主函数-新减障碍物v」
                    for u in self.get_pred_neighbors(v):
                        if u != self.goal:
                            # 重新计算u的rhs，取「原有rhs」和「u→v→终点的代价」的最小值
                            # 对应笔记中rhs的计算公式：rhs(u) = min(g(s') + cost(u,s'))
                            self.rhs[u] = min(self.rhs[u], self.count_cost(u, v) + self.g[v])
                        # 对u执行节点更新，重新加入优先队列等待处理
                        self._update_vertex(u)           
            
            # 6.3 基于更新后的节点代价，重新规划最优路径，对应笔记「主函数-重新规划最优路径」
            self.compute_shortest_path()       
        
        # 7. 路径可视化：实时绘制机器人当前走过的路径（工程实现的可视化功能）
        self.visualize(path)             
    
    # 8. 机器人到达终点，返回完整路径
    return path

    • plan 是包括了人物移动+寻路计算的一个完整流程函数，如果假设路径没有发生变化，则当前 plan 函数只会执行 if changed_edges: 当中的内容，然后每一次判断下是否发生了改变。如果检测到路径发生变化，则首先要重新计算 KM 的值，然后单独遍历所有发生了变化的节点 v，并且按照变化的类型，分别进行处理：
        1. 从可行走 -> 障碍物：此时需要遍历当前变化了的节点 v 的所有连接着的可行走的节点 u，分别对其更新 RHS 的值，然后再更新 u 的数据、并重新计算 key 的值；
        1. 从障碍物 -> 可行走：此时同意遍历 v 节点的所有连接着的非终点节点，计算器更新后的 RHS，然后更新 u 的数据并重新计算 key 的值；
    • 完成上面的计算后，再次执行一次 compute_shortest_path 函数，然后继续循环、移动；
  • 分析：compute_shortest_path 函数
    • 负责处理优先队列中取出的节点的三种情况：键值过时、过一致状态、欠一致状态；
    • k_old：从优先队列中取出的节点 s 的旧 key 值（入队时的 key）
    • k_new：节点 s 的新 key 值（当前时刻重新计算的 key，calculate_key(s) 按照的 key 公式）

# 情况2：过一致状态（g[s] > rhs[s]），对应笔记「核心参数-min(g,rhs)的作用-过一致」
# 含义：发现了从s到终点的更优路径，rhs[s]是新的最优代价，需要将g[s]收敛到rhs[s]
if self.g[s] > self.rhs[s]: 
   self.g[s] = self.rhs[s]  # 核心操作：将g[s]下调到rhs[s]，让节点s恢复一致状态（g==rhs）
   self.state[s] = 2  # 工程扩展：标记s为已处理的一致节点，无算法核心作用
   # 遍历s的所有前驱节点u，更新u的代价（因为s的代价变优，u可能通过s获得更优路径）
   for u in self.get_pred_neighbors(s):
       # 若u不是终点，且「u原有rhs」>「u→s→终点的代价」，说明u能通过s获得更优路径
       if u != self.goal and self.rhs[u] > self.count_cost(u, s) + self.g[s]:
          self.back[u] = s  # 更新u的最优后继为s（反向路径表）
          # 重新计算u的rhs为最优代价，对应笔记中rhs的计算公式
          self.rhs[u] = self.g[s] + self.count_cost(u, s)
       # 对u执行节点更新：重新计算key并维护优先队列，让u进入队列等待处理
       self._update_vertex(u)

# 情况3：欠一致状态（g[s] < rhs[s]），对应笔记「核心参数-min(g,rhs)的作用-欠一致」
# 含义：s的原路径失效（如障碍新增），实际代价rhs[s]升高，原有g[s]是乐观值，需要重置并重新计算
else:
     self.g[s] = np.inf  # 核心操作：重置g[s]为无穷大，标记原代价失效
     # 处理对象：s自己 + s的所有前驱节点u（因为这些节点的代价可能依赖s的旧值）
     for u in self.get_pred_neighbors(s) + [s]:

         if u != self.goal and self.back[u] == s:

         # 对u执行节点更新：重新计算key并维护优先队列，让u进入队列等待处理

public static bool WorldToGrid(Vector3 worldPos,out Vector2Int gridPos)  
{  
    int gridX;  
    int gridY;  
    Vector3 local = worldPos - worldOrigin;  
    gridX = Mathf.FloorToInt(local.x / cellSize);  
    gridY = Mathf.FloorToInt(local.z / cellSize);  
  
    bool valid = gridX >= 0 && gridX < gridSize.x && gridY >= 0 && gridY < gridSize.y;  
    if (!valid)  
    {        gridX = -1;   
        gridY = -1;  
    }  
    gridPos = new Vector2Int(gridX, gridY);  
    return valid;  
}

/// <summary>  
/// 获取当前网格节点的8邻域可行走邻居  
/// 直向邻居：仅自身可行走即可  
/// 对角线邻居：自身可行走 + 相邻两个直向邻居都可行走  
/// </summary>  
/// <param name="gridPos">当前网格坐标</param>  
/// <returns>可行走的邻居节点列表（无则返回空列表）</returns>  
public List<Vector2Int> GetNeighbors(Vector2Int gridPos)  
{  
    // 初始化邻居列表（避免返回 null 导致空引用）  
    List<Vector2Int> neighbors = new List<Vector2Int>();  
    int curX = gridPos.x;  
    int curY = gridPos.y;  
  
    // 健壮性校验：当前节点本身无效，直接返回空列表  
    if (!IsWalkable(curX, curY))  
    {        Log.Warning($"GridSystem：当前网格坐标({curX},{curY})无效/不可行走，无法获取邻居", LogModule.AI);  
        return neighbors;  
    }  
    // 定义8个方向偏移量：前4个=上下左右（直向），后4个=左上/右上/左下/右下（对角线）  
    Vector2Int[] dirOffsets = new Vector2Int[]  
    {  
        new Vector2Int(-1, 0), // 左  
        new Vector2Int(1, 0),  // 右  
        new Vector2Int(0, -1), // 下  
        new Vector2Int(0, 1),  // 上  
        new Vector2Int(-1, -1),// 左上（对角线）  
        new Vector2Int(-1, 1), // 右上（对角线）  
        new Vector2Int(1, -1), // 左下（对角线）  
        new Vector2Int(1, 1)   // 右下（对角线）  
    };  
  
    // 遍历所有方向  
    for (int i = 0; i < dirOffsets.Length; i++)  
    {        Vector2Int offset = dirOffsets[i];  
        int neighX = curX + offset.x;  
        int neighY = curY + offset.y;  
  
        // 步骤 1：邻居坐标超出网格范围，直接跳过  
        if (neighX < 0 || neighX >= gridSize.x || neighY < 0 || neighY >= gridSize.y)  
        {            continue;  
        }  
        // 步骤2：区分直向/对角线邻居，按规则校验  
        if (i < 4) // 前4个：直向邻居  
        {  
            // 直向规则：仅邻居自身可行走即可  
            if (IsWalkable(neighX, neighY))  
            {                neighbors.Add(new Vector2Int(neighX, neighY));  
            }        }        else // 后4个：对角线邻居  
        {  
            // 对角线规则：1.邻居自身可行走  2.相邻两个直向节点都可行走  
            // 节点校验：offset=(ox,oy)，直向节点为(curX+ox, curY) 和 (curX, curY+oy)            bool isDir1Walkable = IsWalkable(curX + offset.x, curY);  
            bool isDir2Walkable = IsWalkable(curX, curY + offset.y);  
            if (IsWalkable(neighX, neighY) && isDir1Walkable && isDir2Walkable)  
            {                neighbors.Add(new Vector2Int(neighX, neighY));  
            }        }    }  
    return neighbors;  
}  
private bool IsWalkable(int x, int y)  
{  
    return GetCost(x, y) != -1;  
}
