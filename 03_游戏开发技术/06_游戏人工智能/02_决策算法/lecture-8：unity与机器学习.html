模仿学习

什么是模仿学习

概念：模仿学习

  • 引入：
    • 如果只是使用强化学习，难以应对复杂的环境，尤其是达成目标、获得奖励需要连续完成序列的动作时，强化学习需要很长的时间来慢慢随机、训练出行为；
  • 概念：
    • 相比于让 Agent 完全随机的采取动作、进行学习，不如尝试模仿玩家所做的行为，并以此为基础；
    • 因此模型学习类似于监督学习；

使用模仿学习

概念：使用模仿学习

  • （1）为 Agent 添加记录器
  • （2）填写选项
      1. 讲 Record 按钮点开
      1. 设置记录的步数，如果设置为 0、则表示全部记录
      1. 设置当前记录器的名称和目录
  • （3）将 Agent 的动作类型更改为 Heuristic Only
  • （4）开启游戏运行，此时玩家的动作将被记录
    • 当完成上面三步的设置后，此时进行游戏，玩家的动作将会被记录；
    • 此时、玩家可以操作 Agent 按照正确的动作序列，在完成此次游戏后、可以让 AI 去进行学习；
  • （5）结束游戏，找到训练得到的记录
  • （6）找到当前的 config 文件，为其在 reward_signals 部分添加 gail
      1. 找到 config
      1. 添加 gail 并修改属性（gail 是一种模仿学习，还有另一种成为 bc ）
      • gail 表示生成对抗模仿学习，这表示 AI 将不会尝试击败你所提供的演示；
      • 其算法层面、是通过创建一个鉴别器的第二个学习算法，这个算法的目标是通过确定某个动作是来自 Agent 还是 Demo，当随着训练进行，Agent 会运行的更像 Demo；
      • 在 gail 下，可以通过 strength 设置当前演示的所占影响的强度，如果 strength=1 则表示要求 Agent 和演示完全相同，因此通常 strength 不设置为 1，而是设置为比如 0.5；
      • 另外还要设置 demo_path 为当前训练结果的路径：
  • （7）在 config 文件中添加 bc：
    • behavioral_cloning 表示行为模仿。gail 是分辨器假装行为来自 Demo，而 bc 则是直接复制演示的动作，因此其永远不会比演示的动作效果更好；
    • 为了得到更好的动作，应当把 bc 和 gail 混合使用。bc 是直接复制行为，gail 是尝试去模仿行为，extrinsic 是强化学习，其会继续改进行为；
  • （8）不断地调整参数，进行训练
    • 训练：mlagents-learn config/foodAgent.yaml --run-id=FoodAgent Imitation
    • 前期可以 gail 的强度设置的更高、后期可以将 gail 的强度设置的更多，让模型基于强化学习得到更好的结果；

方式一：Camera Sensor

概念：Camera Sensor

  • 图示：
  • 概念：
    • 为当前场景添加一个 Camera Sensor 组件，以下是其属性：
    • Camera：表示当前传输视觉的相机；
    • Width/Height：表示当前相机的宽和高；
    • Grayscale：如下图
      • 如果是 50*50 的图片，如果是记录灰度值，则大小是 2500，如果是 Color，则需要 7500 大小；
    • Observation Stacks：表示当前利用 AI 做出的决定是基于多少帧
      • 如果 Observation Stacks=1，则表示当前做出的决定是基于一帧；
      • 如果 Observation Stacks=10，则表示当前做出的决定是基于十帧；
      • 因此如果需要从当前的图像当中判断方向，需要至少不少于 1 帧的画面；
      • 但设置的更高将导致更高的代价；
