Lecture 5：PureMVC 架构

PureMVC 是什么

概念：PureMVC 是什么

  • 概念：
    • 基于 MVC 思想和一些基础设计模式建立的一个轻量级的应用框架，是一个免费开源框架；
    • 它最初是执行的 ActionScript 3 语言使用的，现在已经移植到几乎所有主流平台；
  • 链接：
    • http://puremvc.org/
  • 图示：PureMVC 基本结构
  • 概念：PureMVC 的基本结构
    • PureMVC 的基本结构，可以理解成是 MVC 的基本思想，加上若干设计模式（代理模式、中介者模式、外观模式、命令模式、观察者模式、单例模式）；
    • 比如：
    • （1）Model (数据模型)：
      • 关联 Proxy (代理模式) 对象，由代理来作为 Model 的一部分来负责处理数据；
    • （2）View (界面)：
      • 关联 Mediator (中介模式) 对象，由 Mediator 关联具体的 UI 对象，负责处理界面；
    • （3）Controller (业务控制)：
      • 管理 Command (命令模式) 对象，由 Common 负责处理具体的业务逻辑；
    • （4）Facade (外观)：
      • 是 MVC 三者的经纪人，统管全局可以获取代理、中介、命令；
      • 可以在 Facade（外观模式）的单例对象里面，随时获取数据、界面、控制器的 Proxy、Mediator、Common 等内容；
    • （5）Notification：
      • Notification（观察者模式）通知，就是图中的一条条线，负责传递信息；
  • 总结：
    • PureMVC = MVC+Proxy+Mediator+Command+Facade；

使用 Model 和 Proxy

Model 的具体实现

概念：使用 Model 和 Proxy

  • 概念：PureMVC 的 Model 的具体实现
    • （1）单例模式：Model 具有私有静态实例 instance，通过 GetInstance() 工厂方法获取全局唯一实例；
    • （2）Proxy 字典：
      • 使用线程安全的 ConcurrentDictionary<string, IProxy> 作为 proxyMap，以 ProxyName 为键来存储和检索 IProxy 实例；
      • 作为 Proxy 的容器，实现高效的、线程安全的 Proxy 查找、添加和移除操作；
    • （3）生命周期管理：
      • 提供 RegisterProxy, RetrieveProxy, RemoveProxy, HasProxy 等核心方法；
      • 在注册和移除时，会分别自动调用 proxy.OnRegister() 和 proxy.OnRemove() 方法；
      • 为 Proxy 对象提供了完整的生命周期管理钩子，使其能在注册到框架或从框架移除时执行必要的初始化或清理逻辑；
    • （4）模板方法 (Template Method)​：
      • 定义了一个受保护的虚方法 InitializeModel()，允许子类在不重写构造函数的情况下，对 Model 单例进行特定的初始化，增强了框架的灵活性；

创建 Proxy

概念：创建 Proxy

  • 概念：
      1. 继承 Proxy 父类；
      1. 写构造函数；
    • 写构造函数有两个很重要的点：
      1. 代理的名字；
      1. 代理的相关数据；
  • 代码：Proxy 的实现
/// 
/// Proxy 类 - 数据模型代理
/// 在 PureMVC 框架中，Proxy 负责管理应用程序的数据模型部分
/// 继承自 Notifier，因此可以发送通知，但不接收通知
/// 实现 IProxy 接口
/// 
public class Proxy: Notifier, IProxy
{
    ///  
    /// Proxy 的默认名称常量
    /// 
    public const string NAME = "Proxy";

    /// 
    /// 构造函数
    /// Proxy 负责保存数据对象并存取数据，可发送通知
    /// 
    /// <param name="proxyName">Proxy 实例的名称标识</param>
    /// <param name="data">Proxy 管理的初始数据对象（可选）</param>
    public Proxy(string proxyName, object data = null)
    {
        // 如果未提供代理名称，使用默认名称 NAME
        // 否则使用提供的 proxyName
        ProxyName = proxyName ?? NAME;
        
        // 如果提供了初始数据，则设置数据对象
        if (data != null) Data = data;
    }

    /// 
    /// 当 Proxy 在 Model 中注册时由 Model 调用
    /// 子类可以重写此方法执行初始化逻辑
    /// 例如：建立数据连接、加载初始数据等
    /// 
    public virtual void OnRegister()
    { 
        // 默认实现为空，子类按需重写
        // 当 Proxy 注册到 Model 时，会触发此方法
    }

    /// 
    /// 当 Proxy 从 Model 中移除时由 Model 调用
    /// 子类可以重写此方法执行清理逻辑
    /// 例如：关闭数据连接、释放资源等
    /// 
    public virtual void OnRemove()
    {
        // 默认实现为空，子类按需重写
        // 当 Proxy 从 Model 移除时，会触发此方法
    }

    /// 
    /// Proxy 的名称标识
    /// 用于在 Model 中唯一识别此 Proxy 实例
    /// 通过名称可以在 Model 中检索和访问特定的 Proxy
    /// 
    public string ProxyName { get; protected set; }

    /// 
    /// Proxy 管理的数据对象
    /// 用于存储和访问 Proxy 所负责的业务数据
    /// 可以是任意类型的数据：实体对象、集合、值等
    /// 
    public object Data { get; set; }
}
  • 解析：
    • 当需要在外部使用代理中的数据，可以使用 Proxy 的 Data 属性；
    • 而 Data 属性可以在 Proxy 的构造函数 Proxy (string proxyName, object data = null) 的第二个参数传入，在构造函数内部对其进行赋值；
  • 方式：
// 方式一：在构造函数的base当中传入数据

using PureMVC.Interfaces;
using PureMVC.Patterns.Observer;

namespace PureMVC.Patterns.Mediator
{
    /// 
    /// 一个基础的 Mediator 实现。
    /// <para>
    /// - 负责管理特定的视图组件（View Component），例如一个UI面板或控件组
    /// - 监听用户操作（如按钮点击），并转化为PureMVC内部的通知（Notification）进行发送
    /// - 监听并处理其他部分发来的通知，以更新其管理的视图组件
    /// - 继承自 <c>Notifier</c>，因此自身也具备发送通知的能力
    /// </para>
    /// 
    /// <seealso cref="PureMVC.Core.View"/>
    public class Mediator : Notifier, IMediator
    {
        /// 
        /// 默认的Mediator名称常量。
        /// 注意：实际的Mediator子类通常会重写此常量，使用自己独特的名称
        /// 
        public static string NAME = "Mediator";

        /// 
        /// 构造函数。
        /// 
        /// <param name="mediatorName">此中介者的名称。通常建议使用子类的类型名称</param>
        /// <param name="viewComponent">此中介者所管理的视图组件（例如一个Unity的GameObject或UI面板）。可以为空，后续设置</param>
        public Mediator(string mediatorName, object viewComponent = null)
        {
            // 如果未提供 mediatorName，则使用默认名称 "Mediator"
            MediatorName = mediatorName ?? NAME;
            // 关联的视图组件 View
            ViewComponent = viewComponent;
        }

        /// 
        /// 列出此 <c>Mediator</c> 感兴趣并希望被通知的 <c>INotification</c> 名称。
        /// <para>
        /// - 在Mediator注册到View层时，框架会自动为其订阅此处返回的通知[7](@ref)。<br/>
        /// - 当有这些通知被发出时，其 <c>HandleNotification</c> 方法会被调用[1](@ref)。
        /// </para>
        /// 
        /// <example>
        /// 例如，一个管理玩家信息面板的Mediator可能会对"玩家金币更新"和"玩家等级更新"通知感兴趣[6](@ref)。
        /// <code>
        /// public override string[] ListNotificationInterests()
        /// {
        ///     return new string[] { NotificationConstants.PLAYER_COIN_CHANGED, NotificationConstants.PLAYER_LEVEL_UP };
        /// }
        /// </code>
        /// </example>
        /// <returns>感兴趣的通知名称数组。默认返回一个空数组，子类应重写此方法[1](@ref)。如果对任何通知都不感兴趣，可不重写或返回空数组。</returns>
        public virtual string[] ListNotificationInterests()
        {
            // 默认实现，返回一个空字符串数组，表示不关心任何通知。
            // 子类必须重写此方法以声明其关心哪些通知。
            return new string[0];
        }

        /// 
        /// 处理接收到的 <c>INotification</c>。
        /// <para>
        /// 当Mediator感兴趣的通知（由 <c>ListNotificationInterests</c> 声明）被发出时，此方法会被框架自动调用[7](@ref)。
        /// </para>
        /// 
        /// <remarks>
        /// 通常，此方法内部会使用一个switch语句或if-else链，根据不同的 <c>notification.Name</c> 来执行不同的逻辑[1](@ref)。
        /// 通过 <c>notification.Body</c> 可以获取通知所携带的数据[7](@ref)。
        /// </remarks>
        /// <param name="notification">被发出的通知对象，包含名称（Name）和可选的数据体（Body）。</param>
        /// <example>
        /// 处理更新玩家金币的通知。
        /// <code>
        /// public override void HandleNotification(INotification notification)
        /// {
        ///     switch (notification.Name)
        ///     {
        ///         case "PlayerCoinChanged":
        ///             int newCoinCount = (int)notification.Body;
        ///             UpdateCoinDisplay(newCoinCount); // 更新UI显示
        ///             break;
        ///         case "PlayerLevelUp":
        ///             // ... 处理升级逻辑
        ///             break;
        ///     }
        /// }
        /// </code>
        /// </example>
        public virtual void HandleNotification(INotification notification)
        {
            // 默认实现为空方法。子类应根据需要重写此方法，以响应特定的通知。
        }

        /// 
        /// 当Mediator被注册到View层时，由View调用此方法[7](@ref)。
        /// <para>
        /// 这是进行一些初始化操作（例如获取视图组件引用、为按钮添加监听事件）的理想位置[6](@ref)。
        /// </para>
        /// 
        public virtual void OnRegister()
        {
            // 默认实现为空方法。子类可重写以完成Mediator注册后的特定初始化工作。
        }

        /// 
        /// 当Mediator从View层中被移除时，由View调用此方法[7](@ref)。
        /// <para>
        /// 这是进行清理操作（例如移除事件监听器、释放资源）的理想位置，以防止内存泄漏[6](@ref)。
        /// </para>
        /// 
        public virtual void OnRemove()
        {
            // 默认实现为空方法。子类可重写以完成Mediator移除前的清理工作。
        }

        /// 
        /// 此Mediator的名称。
        /// <para>在Facade或View中，通常通过此名称来检索或管理Mediator[7](@ref)。</para>
        /// 
        public string MediatorName { get; protected set; }

        /// 
        /// 此Mediator所管理的视图组件。
        /// <para>
        /// 视图组件可以是任何对象，在Unity中通常是一个GameObject或某个MonoBehaviour[6](@ref)。Mediator通过操作此属性来更新视图[1](@ref)。
        /// </para>
        /// 此属性允许在Mediator生命周期内动态设置或更改[7](@ref)。
        /// 
        public object ViewComponent { get; set; }
    }
}

      1. 继承 Command 相关脚本 SimpleCommand；

      1. 使用 RegisterCommand 注册命令于通知；
      • 参数一：在 PureNotification 中定义的函数名称；

      1. 一定要具有一个启动函数；

分析：完整的 Command 的使用代码

  • 分析：
    • 当前要实现 ShowPanelCommand 类来提供展示面板的功能，在 Execute 类中，根据当前传入的消息体 notification.Body 的内容来判断当前要创建哪一个面板，并调用 Facade.RegisterMediator 来为对应的面板创建 Mediator；
    • 这些 Mediator 包裹着 View 对象；
  • 代码：
public class ShowPanelCommand : SimpleCommand
{
	/// <summary>
	/// 可以在其中写面板创建的逻辑
	/// </summary>
	/// <param name="notification"></param>
	public override void Execute(INotification notification)
	{
		base.Execute(notification);
		
		string panelName = notification.Body.ToString();
		
		switch (panelName)
		{
			case "MainPanel":
				// 如果要是有 Mediator，一定也要在 Facade 中去注册
				// Command、Proxy 一样，要用就需要注册
				// 可以在命令中，直接使用 Facade，代表的就是唯一的 Facade
				
				// 如果没有 MainViewMediator，则为其 New 一个 Mediator
				if (!Facade.HasMediator(MainViewMediator.NAME))
				{
					Facade.RegisterMediator(new MainViewMediator());
				}
				
				// Facade 中得到 Mediator 的方法
				MainViewMediator mm = Facade.RetrieveMediator(MainViewMediator.NAME) as  MainViewMediator;

				// 实例化面板
				if (mm.ViewComponent == null)
				{
					// 有了 Meditaor，下一步就是创建界面、创建预设体
					GameObject res = Resources.Load<GameObject>("UI/MainPanel");
					GameObject obj = GameObject.Instantiate(res);
					obj.transform.SetParent(GameObject.Find("Canvas").transform);
				
					// 得到预设体上的 View 脚本，关联到 Mediator 上
					mm.SetView(obj.GetComponent<MainView>());
				}
				
				break;
			
			case "RolePanel":
				if (!Facade.HasMediator(RoleViewMediator.NAME))
				{
					Facade.RegisterMediator(new RoleViewMediator());
				}
				
				// Facade 中得到 Mediator 的方法
				RoleViewMediator mm2 = Facade.RetrieveMediator(RoleViewMediator.NAME) as RoleViewMediator;

				// 实例化面板
				if (mm2.ViewComponent == null)
				{
					// 有了 Meditaor，下一步就是创建界面、创建预设体
					GameObject res = Resources.Load<GameObject>("UI/RolePanel");
					GameObject obj = GameObject.Instantiate(res);
					obj.transform.SetParent(GameObject.Find("Canvas").transform);
				
					// 得到预设体上的 View 脚本，关联到 Mediator 上
					mm2.ViewComponent = obj.GetComponent<RoleView>();
				}
				break;
		}
	}
}

public class GameFacade : Facade
{
	public static GameFacade Instance
	{
		get
		{
			if (instance == null) 
			{
				instance = new GameFacade();
			}

			return instance as GameFacade;
		}

	}

	/// <summary>
	/// 初始化 Controller 相关内容
	/// </summary>
	protected override void InitializeController()
	{
		base.InitializeController();
		
		RegisterCommand(PureNotification.START_UP, () =>
		{
			return new StartupCommand();
		});
		
		RegisterCommand(PureNotification.SHOW_PANEL, () =>
		{
			return new ShowPanelCommand();
		});
	}

	public void StartUp()
	{
		// 发送一条 开始 的通知，并且调用 StartupCommand 中的 Execute 
		SendNotification(PureNotification.START_UP);
		
		// 
	}
}

补充：Facade 的具体实现

概念：Facade 的具体实现

  • 代码：
using System;
using PureMVC.Interfaces;
using PureMVC.Core;
using PureMVC.Patterns.Observer;

namespace PureMVC.Patterns.Facade
{
    public class Facade : IFacade
    {
        // 静态单例实例
        protected static IFacade instance;

        // Controller 引用
        protected IController controller;
        // Model 引用
        protected IModel model;
        // View 引用
        protected IView view;

        // 单例消息常量
        protected const string SingletonMsg = "Facade Singleton already constructed!";

        // 此 IFacade 实现是单例，因此不应直接调用构造函数，而应调用静态工厂方法：Facade.getInstance(() => new Facade())
        public Facade()
        {
            // 单例模式的核心检查：确保实例唯一
            if (instance != null) throw new Exception(SingletonMsg);
            instance = this; // 将当前实例赋值给静态字段
            InitializeFacade(); // 初始化Facade
        }


        // Facade 单例工厂方法。Func<IFacade> facadeFunc 是创建 IFacade 的委托函数
        public static IFacade GetInstance()
        {
            // 如果实例不存在，则使用提供的委托创建它
            if (instance == null)
            {
                instance = facadeFunc();
            }
            return instance;
        }
 
        // 初始化单例的实例，由构造函数自动调用。您可以在子类中重写此方法以执行特定于子类的初始化
        // 但请确保调用 base.InitializeFacade()
        protected virtual void InitializeFacade()
        {
            // 按顺序初始化 MVC 三大核心组件
            InitializeModel();
            InitializeController();
            InitializeView();
        }

        protected virtual void InitializeModel()
        {
            // 获取 Model 的单例实例
            model = Model.GetInstance(() => new Model());
        }
        protected virtual void InitializeController()
        {
            // 获取 Controller 的单例实例
            controller = Controller.GetInstance(() => new Controller());
        }
        protected virtual void InitializeView()
        {
            // 获取 View 的单例实例
            view = View.GetInstance(() => new View());
        }

        // ================== Command 相关方法 ================== 
        // 这些方法将调用委托给 Controller 实例

        // 根据通知名称向 <c>Controller</c> 注册一个 <c>ICommand</c>。
        // <param name="notificationName">与 <c>ICommand</c> 关联的 <c>INotification</c> 的名称</param>
        // <param name="factory">创建 <c>ICommand</c> 的工厂委托</param>
        public virtual void RegisterCommand(string notificationName, Func<ICommand> factory)
        {
            controller.RegisterCommand(notificationName, factory);
        }

        /// 从 Controller 中移除先前注册的 <c>ICommand</c> 到 <c>INotification</c> 的映射。
        /// <param name="notificationName">要移除 <c>ICommand</c> 映射的 <c>INotification</c> 的名称</param>
        public virtual void RemoveCommand(string notificationName)
        {
            controller.RemoveCommand(notificationName);
        }

        /// 检查是否已为给定通知注册了 Command。
        /// <param name="notificationName">通知名称</param>
        /// <returns>当前是否为给定的 <c>notificationName</c> 注册了 Command</returns>
        public virtual bool HasCommand(string notificationName)
        {
            return controller.HasCommand(notificationName);
        }

        // ================== Proxy 相关方法 ================== 
        // 这些方法将调用委托给 Model 实例

        /// 用一个名称向 <c>Model</c> 注册一个 <c>IProxy</c>。
        /// <param name="proxy">要注册到 <c>Model</c> 的 <c>IProxy</c> 实例</param>
        public virtual void RegisterProxy(IProxy proxy)
        {
            model.RegisterProxy(proxy);
        }

        /// 按名称从 <c>Model</c> 中检索一个 <c>IProxy</c>。
        /// <param name="proxyName">要检索的 proxy 的名称</param>
        /// <returns>先前用给定 <c>proxyName</c> 注册的 <c>IProxy</c> 实例</returns>
        public virtual IProxy RetrieveProxy(string proxyName)
        {
            return model.RetrieveProxy(proxyName);
        }


        /// 按名称从 <c>Model</c> 中移除一个 <c>IProxy</c>。
        /// <param name="proxyName">要从 <c>Model</c> 中移除的 <c>IProxy</c> 的名称</param>
        /// <returns>从 <c>Model</c> 中移除的 <c>IProxy</c></returns>
        public virtual IProxy RemoveProxy(string proxyName)
        {
            return model.RemoveProxy(proxyName);
        }

        /// 检查是否已注册某个 Proxy。
        /// <param name="proxyName">Proxy 名称</param>
        /// <returns>当前是否已使用给定的 <c>proxyName</c> 注册了 Proxy</returns>
        public virtual bool HasProxy(string proxyName)
        {
            return model.HasProxy(proxyName);
        }

        // ================== Mediator 相关方法 ================== 
        // 这些方法将调用委托给 View 实例

        /// 向 <c>View</c> 注册一个 <c>IMediator</c>。
        /// <param name="mediator">对 <c>IMediator</c> 的引用</param>
        public virtual void RegisterMediator(IMediator mediator)
        {
            view.RegisterMediator(mediator);
        }

        /// 从 <c>View</c> 中检索一个 <c>IMediator</c>。
        /// <param name="mediatorName">Mediator 名称</param>
        /// <returns>先前用给定 <c>mediatorName</c> 注册的 <c>IMediator</c></returns>
        public virtual IMediator RetrieveMediator(string mediatorName)
        {
            return view.RetrieveMediator(mediatorName);
        }

        /// 从 <c>View</c> 中移除一个 <c>IMediator</c>。
        /// <param name="mediatorName">要移除的 <c>IMediator</c> 的名称</param>
        /// <returns>从 <c>View</c> 中移除的 <c>IMediator</c></returns>
        public virtual IMediator RemoveMediator(string mediatorName)
        {
            return view.RemoveMediator(mediatorName);
        }

        /// 检查是否注册了某个 Mediator。
        /// <param name="mediatorName">Mediator 名称</param>
        /// <returns>是否已使用给定的 <c>mediatorName</c> 注册了 Mediator</returns>
        public virtual bool HasMediator(string mediatorName)
        {
            return view.HasMediator(mediatorName);
        }

        // ================== 通知 (Notification) 相关方法 ================== 

        /// 
        /// 创建并发送一个 <c>INotification</c>。
        /// 
        /// <remarks>
        ///     <para>
        ///         避免我们在实现代码中构造新的通知实例。
        ///     </para>
        /// </remarks>
        /// <param name="notificationName">要发送的通知的名称</param>
        /// <param name="body">通知的主体（可选）</param>
        /// <param name="type">通知的类型（可选）</param>
        public virtual void SendNotification(string notificationName, object body = null, string type = null)
        {
            // 创建通知实例并通知所有观察者
            NotifyObservers(new Notification(notificationName, body, type));
        }

        /// 通知 <c>Observer</c>s。
        /// <remarks>
        ///     <para>
        ///         此方法公开主要是为了向后兼容，并允许您使用 Facade 发送自定义通知类。
        ///     </para>
        ///     <para>
        ///         通常您应该只调用 SendNotification 并传递参数，而不必自己构造通知。
        ///     </para>
        /// </remarks>
        /// <param name="notification"><c>View</c> 将通知其 <c>Observers</c> 的 <c>INotification</c></param>
        public virtual void NotifyObservers(INotification notification)
        {
            // 将通知派发到 View 层，由 View 负责通知所有注册的观察者
            view.NotifyObservers(notification);
        }
    }
}
