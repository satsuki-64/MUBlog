AB 包基础概念

了解 AB 包

概念：什么是 AB 包

  • 用处：
    • 特定于平台的资产压缩包，有点类似压缩文件；
    • 资产包括: 模型、贴图、预设体、音效、材质球等等；
  • 注意：
      1. 资源可以打包为 AB 包，但是 C# 脚本不行，因为 C# 脚本是编译型语言；
      1. 如果要热更新脚本，需要使用 Lua 语言；
  • 区别：
      1. 相对 Resources 下的资源 AB 包更好管理资源；
      • Resources 和 AB 包
      1. 减少包体大小；
        1. 压缩资源；
        1. 减少初始包大小；
      1. 支持热更新
      • 热更新基本规则：通过资源对比文件，向服务器端进行比对、确定需要更新或下载哪些 AB 包资源；

AB 包内容

概念：AB 包内容

  • 概念：
    • Unity 的 AssetBundle 生成的 AB 包在打包后会生成多种文件，每种文件具有特定的作用和含义；
    • 主要包含：
        1. 各个 AB 包本体；
        1. 主包文件：用于依赖管理；
        1. 所有 .manifest 文件，仅用于查看信息；
    • 加载 AB 包时，需先加载主包、获取 AssetBundleManifest，再按依赖顺序加载其他 AB 包；
  • 图示：
  • 主文件：PC
    • 和目录名一样的包；
    • 这里面记录了资源间的依赖关系；
  • AB 包文件：比如 model、head
    • 没有后缀名的就是资源文件；
  • 配置信息：.manifest
    • 依赖信息：Dependencies

AB 包依赖

什么是 AB 包依赖

概念：AB 包依赖

  • 概念：
    • 关于 AB 包的依赖：一个资源身上用到了别的 AB 包中的资源，这个时候如果只加载自己的 AB 包，会出现资源丢失的情况，创建对象这种时候、需要把依赖包一起加载了才能正常；
    • 依赖包的关键知识点：利用主包获取依赖信息；
  • 举例：
    • 需要把 AB 包本身、以及 AB 包依赖的资源都加载到内存中，才可以正常的显示；

通过主包加载依赖信息

概念：通过主包得到依赖信息

  • 概念：GetAllDependencies 方法
    • Unity 不会自动加载依赖的 AB 包，若未手动加载依赖项，会导致资源丢失（如贴图、材质显示为粉色）。此方法帮助开发者明确需按顺序加载的依赖链；
    • 该方法返回一个字符串数组，包含指定 AssetBundle 所依赖的所有其他 AB 包的名称；
  • 解释：
    • 参数：
      • 需传入目标 AB 包的名称，名称需与打包时定义的完全一致（区分大小写）；
    • 返回值：
      • 字符串数组，按加载顺序排列的依赖包名称。若无依赖则返回空数组；
  • 概念：核心流程
    • 第一步：加载主包
    • 第二步：加载主包中的固定文件
    • 第三步：从固定文件中获得依赖信息
  • 注意：
    • 依赖关系在打包时生成，不同平台（如 Android/iOS）的 AB 包依赖可能不同，需分别进行处理；
  • 代码：
// 1. 加载主包
// 主包名称和目录名称一致
AssetBundle abMain = AssetBundle.LoadFromFile(Application.streamingAssetsPath + "/" + "PC");

// 2. 加载主包中的固定文件
AssetBundleManifest abManifest = abMain.LoadAsset<AssetBundleManifest>("AssetBundleManifest");

// 3. 从固定文件中获得依赖信息
// 传入的参数是要加载的目标文件，获得的依赖信息是关于这个目标文件的依赖信息的字符数组
string[] strs = abManifest.GetAllependencies("model");

AB 包资源加载管理器

同步加载管理器

概念：AB 包资源加载管理器

  • 概念：
    • AB 包管理器目的是让外部更方便的进行资源加载；
    • 并且因为 AB 包不能够重复加载，重复加载会报错，因此可以使用 C# 中的字典，用字典来存储加载过的 AB 包；
  • 实现：
    • 加载步骤：
    • （1）加载 AB 包；
    • （2）获取依赖包相关信息；
    • （3）加载资源来源包；
    • （4）加载资源来源包中的关键配置文件；
    • （5）获取依赖包加载依赖包；
    • （6）加载目标包；
  • 核心：
    • 想加载目标 AB 包里面的一个资源，需要先加载此 AB 包的依赖包；
    • 而为了知道其都有哪些依赖包，因此需要先加载主包，得到主包的配置文件，从中得到都有哪些依赖包；
    • 把依赖包全部加载后、再加载目标 AB 包；

代码：单例模板

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

//C#中 泛型知识点
//设计模式 单例模式的知识点
//继承这种自动创建的 单例模式基类 不需要我们手动去拖 或者 api去加了
//想用他 直接 GetInstance就行了
public class SingletonAutoMono<T> : MonoBehaviour where T : MonoBehaviour
{
    private static T instance;

    public static T GetInstance()
    {
        if( instance == null )
        {
            GameObject obj = new GameObject();
            //设置对象的名字为脚本名
            obj.name = typeof(T).ToString();
            //让这个单例模式对象 过场景 不移除
            //因为 单例模式对象 往往 是存在整个程序生命周期中的
            DontDestroyOnLoad(obj);
            instance = obj.AddComponent<T>();
        }
        return instance;
    }

}

异步加载管理器

概念：异步加载管理器

  • 概念：
    • 从异步加载时，先使用 LoadResAsync() 方法启动协程，并且传入委托的事件；
    • 然后在协程函数中进行加载；
    • 异步加载结束后，通过委托传递给外部来使用；
  • 注意：
    • 以下代码只实现了对于资源加载的异步，没有实现 AB 包加载的异步；

完整代码

using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.Events;

public class ABMgr : SingletonAutoMono<ABMgr>
{
    //AB包管理器 目的是
    //让外部更方便的进行资源加载

    // 主包
    private AssetBundle mainAB = null;
    // 依赖包获取用的配置文件
    private AssetBundleManifest manifest = null;

    // AB包不能够重复加载 重复加载会报错
    // 使用字典来存储 加载过的AB包 
    private Dictionary<string, AssetBundle> abDic = new Dictionary<string, AssetBundle>();

    /// <summary>
    /// 这个AB包存放路径 方便修改
    /// </summary>
    private string PathUrl
    {
        get
        {
            return Application.streamingAssetsPath + "/";
        }
    }

    /// <summary>
    /// 主包名，由于不同的平台可能会打不同的主包，因此在这里使用宏来决定主包名
    /// </summary>
    private string MainABName
    {
        get
        {
#if UNITY_IOS
            return "IOS";
#elif UNITY_ANDROID
            return "Android";
#else
            return "PC";
#endif
        }
    }

    /// <summary>
    /// 加载 AB 包，作为工具方法进行提供
    /// </summary>
    /// <param name="abName"></param>
    public void LoadAB(string abName)
    {
        // 第一步：加载AB包
        if (mainAB == null)
        {
            mainAB = AssetBundle.LoadFromFile(PathUrl + MainABName);
            manifest = mainAB.LoadAsset<AssetBundleManifest>("AssetBundleManifest");
        }
        
        // 第二步：获得依赖包信息
        AssetBundle ab = null;
        // 用一个数组、存储所有依赖包的信息
        string[] strs = manifest.GetAllDependencies(abName);
        for (int i = 0; i < strs.Length; i++)
        {
            // 2.1 由于AB包不能重复加载，所以需要在当前字典中、判断包是否加载过
            if (!abDic.ContainsKey(strs[i]))
            {
	            // 加载依赖包
                ab = AssetBundle.LoadFromFile(PathUrl + strs[i]);
                abDic.Add(strs[i], ab);
            }
        }
        
        // 加载资源来源包
        // 如果没有加载过 再加载
        if (!abDic.ContainsKey(abName))
        {
	        // 加载依赖包
            ab = AssetBundle.LoadFromFile(PathUrl + abName);

			// 将目标包存进字典当中
            abDic.Add(abName, ab);
        }
    }

    //同步加载，不指定类型
    public Object LoadRes(string abName, string resName)
    {
        // 加载AB包，并且存储到字典当中
        LoadAB(abName);
        
        // 为了外面方便 在加载资源时 判断一下 资源是不是GameObject
        // 如果是 直接实例化了 再返回给外部
        Object obj = abDic[abName].LoadAsset(resName);
        
        if (obj is GameObject)
            return Instantiate(obj);
        else
            return obj;
    }

    //同步加载 根据type指定类型
    //由于Lua不支持泛型，因此要使用参数来传递类型
    public Object LoadRes(string abName, string resName, System.Type type)
    {
        //加载AB包
        LoadAB(abName);
        //为了外面方便 在加载资源时 判断一下 资源是不是GameObject
        //如果是 直接实例化了 再返回给外部
        Object obj = abDic[abName].LoadAsset(resName, type);
        if (obj is GameObject)
            return Instantiate(obj);
        else
            return obj;
    }

    //同步加载 根据泛型指定类型
    //这里要加上 where:T 来约束类型，因为LoadAsset<T>(resName)返回值也添加了where:T约束为Object
    public T LoadRes<T>(string abName, string resName) where T:Object
    {
        //加载AB包
        LoadAB(abName);
        //为了外面方便 在加载资源时 判断一下 资源是不是GameObject
        //如果是 直接实例化了 再返回给外部
        T obj = abDic[abName].LoadAsset<T>(resName);
        if (obj is GameObject)
            return Instantiate(obj);
        else
            return obj;
    }
    
    //异步加载的方法
    //这里的异步加载 AB包并没有使用异步加载
    //知识从AB包中 加载资源时 使用异步
    //根据名字异步加载资源
    public void LoadResAsync(string abName, string resName, UnityAction<Object> callBack)
    {
        StartCoroutine(ReallyLoadResAsync(abName, resName, callBack));
    }
    
    private IEnumerator ReallyLoadResAsync(string abName, string resName, UnityAction<Object> callBack)
    {
        //加载AB包
        LoadAB(abName);
        
        //为了外面方便 在加载资源时 判断一下 资源是不是GameObject
        //如果是 直接实例化了 再返回给外部
        //注意：这里使用的是异步加载的方法
        AssetBundleRequest abr = abDic[abName].LoadAssetAsync(resName);
        
        yield return abr;
        
        //异步加载结束后 通过委托 传递给外部 外部来使用
        if (abr.asset is GameObject)
            callBack(Instantiate(abr.asset));
        else
            callBack(abr.asset);
    }
    
	private IEnumerator ReallyLoadResAsync(string abName, string resName, System.Type type, UnityAction<Object> callBack)
    {
        //加载AB包
        LoadAB(abName);
        //为了外面方便 在加载资源时 判断一下 资源是不是GameObject
        //如果是 直接实例化了 再返回给外部
        AssetBundleRequest abr = abDic[abName].LoadAssetAsync(resName, type);
        
        yield return abr;
        
        //异步加载结束后 通过委托 传递给外部 外部来使用
        if (abr.asset is GameObject)
            callBack(Instantiate(abr.asset));
        else
            callBack(abr.asset);
    }
    
    //根据Type异步加载资源
    public void LoadResAsync(string abName, string resName, System.Type type, UnityAction<Object> callBack)
    {
        StartCoroutine(ReallyLoadResAsync(abName, resName, type, callBack));
    }
    
    //根据泛型 异步加载资源
    public void LoadResAsync<T>(string abName, string resName, UnityAction<T> callBack) where T:Object
    {
        StartCoroutine(ReallyLoadResAsync<T>(abName, resName, callBack));
    }
    
    private IEnumerator ReallyLoadResAsync<T>(string abName, string resName, UnityAction<T> callBack) where T : Object
    {
        //加载AB包
        LoadAB(abName);
        //为了外面方便 在加载资源时 判断一下 资源是不是GameObject
        //如果是 直接实例化了 再返回给外部
        AssetBundleRequest abr = abDic[abName].LoadAssetAsync<T>(resName);
        yield return abr;
        //异步加载结束后 通过委托 传递给外部 外部来使用
        if (abr.asset is GameObject)
            callBack(Instantiate(abr.asset) as T);
        else
            callBack(abr.asset as T);
    }

    //单个包卸载
    public void UnLoad(string abName)
    {
        if( abDic.ContainsKey(abName) )
        {
	        // 调用AB包的Unload方法
            abDic[abName].Unload(false);
            // 将AB包从字典移除
            abDic.Remove(abName);
        }
    }

    //所有包的卸载
    public void ClearAB()
    {
        AssetBundle.UnloadAllAssetBundles(false);
        //将字典直接清空
        abDic.Clear();
        mainAB = null;
        manifest = null;
    }
}

构建 AssetBundle 参考策略

总结：构建 AssetBundle 参考策略
  • 根据对象的修改频率差异拆分不同的 AssetBundle；
  • 具有相同生命周期的资源打成一个 AssetBundle，如果一个 AB 中只有不到 50% 的资源经常加载，可以考虑拆到不同的 AssetBundle 中；
  • 不可能同时加载的高低配置资源，也可以拆分到不同的 AssetBundle；
  • 如果多个对象依赖于某个 AssetBundle 中的单个资源，这类单个资源可以单独打包，或者根据生命周期将这类被多个对象依赖的资源打成共享 AssetBundle；
  • 如果一组对象只是一个对象的不同版本，可以考虑使用 AB 变体，注意 Addressable 不支持 AB 变体；
注意：构建 AssetBundle 值得注意的点
  • AB 包体不能因为避免重复引用，拆的过于细碎，移动平台下，2M-5M 的包体是一个合理的参考值。过小的包体，包头占用内存压力会比较大；
  • LZMA 格式压缩的 AB 包较小，但需要完全解压后再加载，加载时间更长，LZ4 格式压缩的 AB 包按块加载，加载时间较快，但包体大小较大，移动平台建议使用 LZ4 压缩格式；
  • 如果确定使用相同版本 Unity 发布打包，后续开发不会升级 Unity 版本，AB 包体不做 Unity 版本兼容时，可以尝试开启打包选项 BuildAssetBundleOption.DisableWriteTypeTree 选项，这样 TypeTree 信息不会被打到 AB 中，可以极大减小包体大小及运行加载时的内存开销；
  • 避免同一个资源被打入多个 AB 中，会增加运行时内存开销，可以使用 UPRAssets Checker 工具来做检查；
  • 不要使用 AB 包文件的 MD5 值作为更新包依据，因为可能包内资源没有变化的情况下，包的 MD5 值也会变换。一般都是选择计算包内原始资源的 MD5 值和路径变化，作为 AB 包变化的依据；
