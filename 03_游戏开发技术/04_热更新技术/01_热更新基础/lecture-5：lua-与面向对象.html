基于表实现 Lua 中的类

概念：Lua 中的类和结构体

  • 概念：
    • Lua 中是默认没有面向对象的，需要我们自己来实现；
    • 在 Lua 中，使用表来类似的表达类的概念；
    • Lua 中类的表现，更像是一个类中有很多静态变量和函数；
  • 总结：
    • （1）Lua 中的代码是从上往下执行的；
    • （2）Lua 的类里面的方法中，访问的是全局变量，不能直接访问到类中的局部变量;
      • 想要在表内部函数中调用表本身的属性或者方法，一定要指定是谁的，使用 表名.属性 或 表名.方法 来得到局部变量；
    • （3）Lua 中，点和冒号的区别：
      • 冒号调用方法：Stduent:Learn()；
      • 冒号调用方法时，会默认把 调用者作为第一个参数 传入方法中；
      • 并且在外部声明的时候，也可以使用冒号；
    • （4）Lua 中有一个默认的关键字 self，用于表示传入的第一个参数；
  • 示例：
Student = 
{
	-- 年龄
	age = 1,

	-- 性别
	set = true,

	-- 第一种声明函数的方式
	-- Lua中的函数是一种特殊的类型，这种类型就是function
	Up = function()
		-- 这样写 这个age 和表中的age没有任何关系 它是一个全局变量
		-- 错误：print(age)
		-- 正确：print(Student.age)
		print("这是一个函数")
		print(Student.age) -- 输出 1
	end

	-- 传入参数
	Learn = function(t)
		print("学习中")
		-- 传入的内容是表，然后可以点出来其内部的局部变量
		print(t.sex)
	end
}

-- 申明表过后，可以在表外去申明表有的变量和方法
-- 类似于其他语言中改变类内静态变量的值
Student.name = "老王"

-- 第二种声明函数的方式
Stduent.Speak = function()
	print("说话")
end

function Student.Speak2()
	print("2")
end

-- 第三种声明函数的方式
function Student:Speak3()
	-- lua中有一个默认的关键字self，用于表示传入的第一个参数
	print(self.name .. "说话三") -- 这里是一个字符串拼接
	print("3")
end

print(Student.age) 
Student.Up()
Stduent.Speak()
Stduent.Speak2()
Student:Speak3() -- 使用冒号调用时，默认讲类自己作为第一个参数传入
Student.Speak3(Student) -- 和上面的效果一致

封装

概念：封装

  • 概念：
    • Lua 中的面向对象的类，全部都是基于 table 实现的；
    • 使用表和元表实现 new 方法，本质上就是返回了一个新的局部空表；
    • 而通过给这个空表设置元表，可以实现通过这个新的表、来访问原本的表的属性；
    • 即：将新表的 __index 设置为是老表；
  • 注意：
    • （1）修改创建出来的对象的属性变量时，为这个空表对象新建了一个成员属性；
    • （2）冒号：表示调用这个函数的对象作为第一个传入的参数，而 self 代表函数调用者；
  • 总结：
    • Lua 中的实例化、以及继承，本质上都是两个操作：1. 新建一张表；2. 设置其 __index；
  • 示例：
    • 当访问一个表中不存在的键时，Lua 会检查该表的元表是否有 __index 字段；
    • 如果是表（如 self.__index = self），Lua 会递归在该表中查找缺失的键；
-- 这个 Object 更像是 C# 中的静态类
Object = {}

-- 这里的 Object.id 类似于 C# 中的静态变量 
Object.id = 1

-- 冒号是会自动将调用这个函数的对象，作为第一个参数传入的写法
function Object:new()
	-- self代表的是默认传入的第一个参数
	-- 对象就是变量，返回一个新的变量
	-- 返回出去的内容，本质上就是对象
	local obj = {}

	-- 元表：当找自己的变量找不到时，就会去找元表当中 __index 指向的内容
	self.__index = self
	-- 把当前声明出的局部变量 obj 的元表设置为是Object
	-- 这样当在Obj中进行访问时，可以通过__index访问到元表中的内容，而元表就是这里self传入的老表Object
	setmetatable(obj,self)
	return obj
end

local myObj = Object:new()
-- 打印结果：1
-- 即返回老表的内容
print(myObj.id)

-- 这个Object更像是C#中的静态类
Object = {}
Object.id = 1

-- self：谁调用，其表示的就是谁
-- 如果是Object调用，则表示的就是Object
-- 如果是myObj调用，则表示的就是myObj
function Object:Test()
	-- 打印当前对象的id属性
	-- 若self是Object，输出静态变量Object.id；若self是实例（如myObj），输出实例属性
	print(self.id)
end

-- 冒号是会自动将调用这个函数的对象，作为第一个参数传入的写法
function Object:new()
	-- self代表的是默认传入的第一个参数
	-- 对象就是变量，返回一个新的变量
	-- 返回出去的内容，本质上就是对象
	local obj = {}

	-- 元表：当找自己的变量找不到时，就会去找元表当中 __index 指向的内容
	self.__index = self
	
	-- 把当前声明出的局部变量 obj 的元表设置为是Object
	-- 通过setmetatable设置元表，启用继承链
	setmetatable(obj,self)
	return obj
end

local myObj = Object:new()

-- myObj 自身无 id 属性，通过元表 __index 找到 Object.id
print(myObj.id)  -- 打印结果：1

-- myObj.Test() 等价于 myObj.Test(myObj)，self 指向 myObj
-- 执行时 self 为 myObj，但 myObj.id 未定义 → 继续回溯到 Object.id
myObj.Test() -- 打印结果：1

-- 尽管为myObj设置了元表，但是对其属性进行赋值时、若此属性在表中未定义，则对其属性进行定义，不会影响到元表
-- 直接对`myObj.id`赋值会创建实例属性，遮蔽元表中的`Object.id`
myObj.id = 666
print(OBject.id) -- 输出结果：1
print(myObj.id) -- 输出结果：666

-- Test() 中 self.id 始终遵循Lua的查找规则（实例优先 → 元表）
myObj:Test() -- 输出结果：666

继承

概念：继承

  • 概念：
    • Lua 中的继承，是写一个用于继承的方法；
  • 步骤：
      1. 定义父类：
      • 创建一个表作为父类，并设置 父类.__index = 父类，确保子类可通过 __index 查找父类方法；
      • 实现父类的构造函数（new 方法），初始化实例属性；
      1. 定义子类：
      • 首先在方法当中，在父类的大 G 表中、注册一个子类名称的全局变量；
      • 然后声明一个新的表，新表的内容等于大 G 表中注册的全局变量；
      • 设置子类表的元表为夫表，子类未定义的方法或属性会通过元表链（__index）回溯到父类查找；
      1. 使用方法：
      • 子类 Person 本身没有 new 方法，但通过元表链调用父类 Object:new()；
  • 示例： 
Object = {}
Object.id = 1

function Object:subClass(className)
	-- _G 是总表，所有声明的全局变量，都以键值对的形式存放在_G其中
	-- 因此传入的参数将作为一个新的表，放在 _G 表中
	_G[className] = {}

	-- 继承规则实现
	local obj = _G[className]

	-- 当前父类调用subClass，传入的第一个参数为父类本身，比如Object，第二个参数为子类，即className
	-- 设置父类的元表
	self.__index = self 
	-- 设置子类的元表为父类，比如设置Person的元表为Object
	setmetatable(obj,self)
end

function Object:new()
	local obj = {}
	self.__index = self
	setmetatable(obj,self)
	return obj
end

Object:subClass("Person")

-- 可以直接使用Person表
print(Person)

-- 可以使用Person类的父类Object的new方法
-- 这里可以形成双重继承链，实例 `p1` 的元表是 `Person`，而 `Person` 的元表是 `Object`
-- 即：p1 -> Person -> Object
local p1 = Person:new()
print(p1.id) -- 可以使用父类的id

多态

概念：多态

  • 概念：
    • Lua 中多态，就是相同方法，不同的执行逻辑；
    • 如果想要保留父类逻辑执行，加入自定义 base 属性，在 subClass 方法中赋值；
  • 注意：
    • 通过 base 调用父类方法时，一定记住不要用冒号，通过 . 然后自己传第一个参数进入父类函数内部；
  • 示例：多态示例
Object = {}
Object.id = 1

function Object:subClass(className)
	-- _G是总表，所有声明的全局变量，都以键值对的形式存放在_G其中
	-- 因此传入的参数将作为一个新的表，放在_G表中
	_G[className] = {}

	-- 继承规则实现
	local obj = _G[className]
	-- 当前父类调用subClass，传入的第一个参数为父类本身，比如Object，第二个参数为子类，即className
	-- 设置父类的元表
	self.__index = self 
	-- 设置子类的元表为父类，比如设置Person的元表为Object
	setmetatable(obj,self)
end

function Object:new()
	local obj = {}
	self.__index = self
	setmetatable(obj,self)
	return obj
end

Object:subClass("GameObject")
GameObject.posX = 0
GameObject.posY = 0

function GameObject:Speak()
	self.posX = self.posX + 1
	self.posY = self.posY + 1
	print(self.posX)
	print(self.posY) 
end

GameObject:subClass("Player")

-- 多态的实现：同一个方法，不同的实现
function Player:Move()
	-- 子类的同名方法，没有逻辑
end

local p1 = Player:new()
p1:Move()
  • 示例：base 的实现，调用父类方法 
Object = {}
Object.id = 1

function Object:subClass(className)
	_G[className] = {}

	local obj = _G[className]
	self.__index = self 
	
	-- 自定义了子类的一个属性，让其指向元表，即其父类
	obj.base = self
	setmetatable(obj,self)
end

function Object:new()
	local obj = {}
	self.__index = self
	setmetatable(obj,self)
	return obj
end

Object:subClass("GameObject")
GameObject.posX = 0
GameObject.posY = 0

function GameObject:Speak()
	self.posX = self.posX + 1
	self.posY = self.posY + 1
	print(self.posX)
	print(self.posY) 
end

GameObject:subClass("Player")

-- 多态的实现：同一个方法，不同的实现
function Player:Move()
	-- 子类的同名方法 
	-- 调用了父类的方法 
	-- 这种方式调用，相当于是把基类表作为第一个参数传入方法当中
	-- 需要避免把基类表传入方法当中，这样想打关于公用一张表的属性了
	self.base:Move()

	-- 如果要执行父类逻辑，不要通过冒号，而是要通过点
	self.base.Move(self)
end

local p1 = Player:new()
p1:Move()
