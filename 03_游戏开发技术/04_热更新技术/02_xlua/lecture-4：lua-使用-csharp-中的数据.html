Lua 中使用 CSharp 脚本

Lua 中使用 Unity 中类

概念：Lua 中使用 CSharp 脚本

  • 原理：
    • C# 是静态编译型语言，运行在 .NET 环境；Lua 是动态解释型脚本语言，通过虚拟机执行。两者无法直接交互，必须通过 C 语言层​（如 Lua C API）桥接；
    • Lua 虚拟机（Lua State）由 C# 通过 P/Invoke 调用 Lua C 库（如 lua51.dll）创建，Lua 脚本的执行完全依赖 C# 主动触发；
  • 概念：Lua 中使用 CSharp 中的类
      1. Lua 中使用 C# 的类的固定方法：
      • CS.命名空间.类名；
      1. Lua 中使用 Unity 中的类：
      • CS.UnityEngine.类名；
      1. 实例化对象：
      • CS.命名空间.类名()；
      • 执行 CS.命名空间.类名() 之后，默认调用的就是类的无参构造函数；
      1. 使用静态方法和对象：
      • 使用类中的静态方法和对象可以直接使用 . 来使用；
      1. 使用对象中的成员方法：
      • 如果使用对象中的成员方法，一定要使用 : 来调用；
  • 概念：存储 C# 中的类 - 别名
    • 为了方便使用，可以将 C# 中的类存储起来，存储为全局变量；
    • 比如：GameObject = CS.UnityEngine.GameObject;
  • 注意：
    • Lua 没有办法直接访问 C#；
    • 一定是先从 C# 调用 Lua 脚本后，才把核心逻辑交给了 Lua 来编写；

Lua 中使用 CSharp 扩展方法

概念：拓展方法

  • 概念：
    • 扩展方法是 C# 语言的一项重要特性，它允许开发者向现有类型"添加"方法，而无需修改原始类型或创建派生类型；
    • 扩展方法本质上是一种特殊的静态方法，但可以像实例方法一样调用，这极大地增强了代码的可读性和可维护性；
  • 概念：使用条件
      1. 必须定义在静态类中：扩展方法所在的类必须是静态类；
      1. 方法本身必须是静态的：扩展方法必须使用 static 关键字声明；
      1. 第一个参数使用 this 修饰符，并且需要指定要扩展的类型，后面跟着参数名称；
  • 示例：
  • 示例：调用方式
    • 扩展方法可以像实例方法一样直接调用；
    • 编译器在编译时会自动将上述调用转换为静态方法调用：StringExtensions.Reverse(text)；

概念：Lua 中使用 CSharp 扩展方法

  • 概念：
    • 在 Lua 中调用 C# 扩展方法与调用普通成员方法一致，需要使用冒号 (:) 语法，尽管扩展方法本质上是静态方法；
  • 概念：[LuaCallCSharp] 特性
    • 为了使 Lua 能够调用 C# 扩展方法，必须添加 [LuaCallCSharp] 特性​，在包含扩展方法的静态类上添加 [LuaCallCSharp] 特性；
  • 补充：使用建议
    • 建议 Lua 中要调用的 C# 类都加上 [LuaCallCSharp] 特性，可以提升执行效率；
    • 如果不加该特性，Lua 会通过反射机制调用 C# 类，效率低；

Lua 中使用 CSharp 的 ref 和 out 参数

概念：ref 和 out

  • 概念：ref
    • 需要传入初始值，且会同时作为输入和输出。调用后，修改后的值会作为返回值的一部分返回；
    • ref 参数在 C# 中既是输入也是输出，因此 Lua 需显式传入初始值。如果不传入初始值，则默认会传入 0 进去，但最好还是写进去占位置；
  • 概念：out
    • 在 Lua 中调用时不需要显式传入，C# 方法内部会为其赋值，最终 out 参数会作为返回值的一部分返回；
    • 和 ref 类似，out 参数会以多返回值的形式返回给 lua，如果函数存在返回值，那么第一个值就是该返回值。之后的返回值就是 out 的结果从左到右一对应；
    • 和 ref 不同的是，out 中不需要传占位置的值；
  • 概念：ref 和 out 综合
    • 混合使用时，综合上面的规则，ref 需占位、out 不需要占位，第一个是函数的返回值，之后从左到右依次对应 ref 或者 out；
  • 代码：
public class Lesson5
{
	public int RefFun(int a,ref int b,ref int c,int d)
	{
		b = a + d;
		c = a - d;
		return 100;
	}

	public int OutFun(int a,out int b,out int c,int d)
	{
		b = a + d;
		c = a - d;
		return 100;
	}
	
	public int RefOutFun(int a,ref int b,out int c)
	{
		return 100;
	}
}
  • 代码：
Lesson5 = CS.Lesson5

local obj = Lesson5()

-- ref参数：会以多返回值的形式，返回给lua
-- 如果函数存在返回值，则第一个值就是返回值
-- 之后的返回值，就是ref的结果、从左到右一一对应
-- ref参数需要传入一个摸认值占位置
-- a：函数返回值； b：第一个ref；c：第二个ref 
local a,b,c = obj:RefFun(1,0,0,1)

Lua 中使用 CSharp 的重载函数

概念：Lua 中使用 CSharp 的重载函数

  • 概念：
    • Lua 通过 XLua 等桥接工具调用 C# 时，会根据参数类型和数量匹配对应的重载函数。XLua 会生成一个封装层（MethodWrap），在调用时遍历所有重载方法，依次检查参数是否匹配（通过 Check 方法）
    • 匹配顺序遵循 C# 中重载方法的定义顺序，若参数类型完全匹配，如 Lua 字符串对应 C# string，则直接调用；若存在模糊性，如 Lua 的 number 对应 C# 的 int / float / double，则按定义顺序选择第一个匹配的重载函数；
  • 分析：Lua 与 C# 的类型映射
      1. Lua 的 number 类型对应 C# 的 int、float、double 等多种数值类型，但 Lua 无法区分这些子类型；
      • 例如，10.2 在 Lua 中是 number，但在 C# 中可能匹配 float 或 double 重载；
      1. 其他类型（如 nil对应 null、table对应 List/Dictionary）也有明确的映射规则；
  • 代码：
public class Lesson6 
{
    // 无参数版本，返回固定值100
    public int Calc() {
        return 100;
    }

    // 两个整数参数版本，返回两数之和
    public int Calc(int a, int b) {
        return a + b;
    }

    // 单整数参数版本，返回参数本身
    public int Calc(int a) {
        return a;
    }

    // 单浮点数参数版本，返回参数本身
    public float Calc(float a) {
        return a;
    }
}
  • 代码：
local obj = CS.Lesson6()

-- 虽然 Lua 本身不支持写重载函数，但是 Lua 中支持调用 C# 的重载函数
print(obj:Calc())
print(obj:Calc(15,1))

-- Lua虽然支持调用C#重载，但是因为Lua中的数值类型只有Number
-- 因此对于多精度的重载函数支持不好、可能出现分不清楚导致的意料之外问题
print(obj:Calc(10))  -- 输出10
-- 可能调用 int Calc(int a)，而非预期的 float Calc(float a)
print(obj:Calc(10.2))  -- 输出0

分析：解决重载函数含糊的问题 - 基于 GetMethod 方法

  • 步骤：
      1. 通过 GetMethod 方法得到 C# 函数；
      1. 使用 xlua.tofunction 将其转化为 Lua 函数使用；
      1. 和 Lua 中函数一样正常使用；
  • 代码：
-- 得到指定函数的参数信息
local m1 = typeof(CS.Lesson6):GetMethod("Calc",{typeof(CS.System.Int32)})
local m2 = typeof(CS.Lesson6):GetMethod("Calc",{typeof(CS.System.Single)})

-- 通过XLua中提供的方法，将它转化为Lua函数使用
local f1 = xlua.tofunction(m1)
local f2 = xlua.tofunction(m2)

-- 成员方法，第一个参数是函数的调用者，第二个参数是传入的值
print(f1(obj,10)) -- 输出10
print(f1(obj,10.2)) -- 输出 10.199，这里有一点浮点误差

Lua 中使用 CSharp 的委托和事件

概念：Lua 中使用 CSharp 的委托和事件

  • 概念：委托
    • C# 委托是类型安全的函数指针，Lua 通过将其映射为可调用的对象来操作；
    • 注意 Lua 没有 += 和 -= 操作符，需用 + 和 - 手动管理委托链。首次赋值需直接使用 =，后续用 + / - 增减。若委托为 nil，需先初始化再操作；
  • 概念：事件
    • 事件在 C# 中受保护，Lua 不能直接赋值，需通过 :+ / :- 语法操作；
    • 除此之外，事件只能在定义它的类内部触发（如 DoEvent() 方法）
    • 语法：
      • 对象:事件名("+/-",函数名)
    • 执行：需要使用 C# 中定义的执行事件的方法来执行事件方法
  • 代码：
public class Lesson7
{
	// 声明委托和事件
	public UnityAction del;
	
	public event UnityAction eventAction;
	
	public void DoEvent()
	{
		if(eventAction!=null)
		{
			eventAction();
		}
	}
	
	public void ClearEvent()
	{
		eventAction = null;
	}
}
  • 代码：Lua 中使用委托
local obj = CS.Lesson7()

-- 委托是用来装函数，使用C#中的委托就是用来装Lua函数的
local fun = function()
	print("Lua函数Fun")
end

-- Lua中没有符复合运算符，不能 “+=”
-- 如果第一次往委托中加函数，因为是nil，所以不能直接+
-- 第一次，要先用 “=” 符号
obj.del = fun
-- 不建议这样写，这样子不好从委托中删除函数
obj.del = obj.del + function
	print("临时声明的函数")
end

obj.del() -- 输出：Lua函数Fun 临时声明的函数

-- 开始减去函数
obj.del = obj.del - fun 
obj.del() -- 输出：临时声明的函数

-- 清空所有的函数
obj.del = nil 
obj.del() -- 输出：
  • 代码：Lua 中使用事件
local fun2 = function()
	print("Lua使用事件")
end

-- 事件加减函数和委托很不一样
-- lua中使用C#事件加函数，有点类似于成员方法
obj:eventAction("+",fun2)
obj:eventAction("+",fun2)
obj:DoEvent()

-- 事件减
obj:eventAction("-",fun2)

-- 事件清除
-- 清除事件不能直接设为空，因为事件在C#当中是不能在外部直接调用或者设置的，因此需要在C#当中写一个ClearEvent方法
obj:ClearEvent()

Lua 中使用 CSharp 的二维数组

概念：Lua 中使用 CSharp 的二维数组

  • 概念：

Lua 中使用 CSharp 系统类型互相访问

概念：Lua 中使用 CSharp 系统类型互相访问

  • 概念：[CSharpCallLua] 特性
    • CSharpCallLua 是 XLua 中用于优化 C# 调用 Lua 代码的核心机制，它通过代码生成技术避免了反射带来的性能开销；
    • CSharpCallLua 的主要作用是为 C# 端提供高效访问 Lua 函数、table 等元素的能力。当我们需要在 C# 中调用 Lua 定义的函数或访问 Lua 中的数据结构时，通过给 C# 端的委托或接口添加 [CSharpCallLua] 特性，XLua 会在代码生成阶段创建适配代码，使得 C# 可以以接近原生调用的性能访问 Lua 内容；
    • 其实现原理是 XLua 在编译前扫描所有标记了 [CSharpCallLua] 的类型，生成对应的桥接代码；
  • 概念：[LuaCallCSharp] 特性
    • 控制哪些 C# 类型和方法可以被 Lua 调用，并优化这些调用的性能；
    • LuaCallCSharp 的主要作用是明确指定哪些 C# 类型和方法可以暴露给 Lua，为标记的 C# 类型生成适配代码，避免反射调用；
    • 或者是在拓展方法时使用此特性，但是建议每个被 Lua 调用的类都添加这个特性，提升性能；
  • 代码：Lua
GameObject = CS.UnityEngine.GameObject
UI = CS.UnityEngine.UI

-- 得到场景中的Slider对象
local slider = GameObject.Find("Slider")
print(slider)

-- 为Slider组件添加监听事件，这里的AddListener实际上就是加一个委托
-- 在没有Lesson10的CS代码前，代码出现的报错：LuaException：c# exception:This type must add to CSharpcallLua
-- 有了Lesson10的CS，可以正常添加监听事件

	print(f)
end)
  • 分析：系统类型加特性
public static class Lesson10
{
	// XLua 在代码生成时会扫描所有标记了 [CSharpCallLua] 的静态字段（比如下面的List<Type>类型），将其中的类型加入白名单
	// 定义一个静态的 List<Type> 字段，用于存储需要与 Lua 交互的系统类型
	[CSharpCallLua]
	public static List<Type> CSharpCallLuaList = new List<Type>(){
		// 将系统类型UnityAction<float>（Unity的泛型委托）添加到列表中
		// 通过typeof获取类型元数据，XLua会根据此信息为UnityAction<float>生成适配代码，使得Lua可以调用该委托或C#可以通过该委托调用Lua函数
		typeof(UnityAction<float>)
	};
	
	[LuaCallCSharp]
	public static List<Type> luaCallCSharpList = new List<Type>{
		typeof(GameObject),
		typeof(RigidBody)
	} 
}

    • 注意一定是在静态类当中，写完之后一定要重新生成代码；

Lua 中使用 CSharp 的协程

概念：Lua 中使用 CSharp 的协程

  • 概念：C# 中的协程
    • C# 协程本质上是一个迭代器 (IEnumerator)​，通过 yield return 语法糖实现非抢占式的任务分片执行；
    • 在 Unity 中，协程由引擎的 MonoBehaviour 生命周期管理，通过 StartCoroutine 启动后，引擎会在每帧检查 Current 状态，满足条件时调用 MoveNext 继续执行；
  • 概念：Lua 中的协程
    • Lua 协程是一个协作式的任务调度机制，通过 coroutine.create 创建、coroutine.resume 启动、coroutine.yield 挂起。与 C# 协程不同：
      1. 完全由 Lua 虚拟机管理，不依赖 Unity 引擎；
      1. 同一时刻只有一个协程运行（非抢占式）；
      1. 需要手动管理协程状态（suspended/running/dead）；
  • 概念：Lua 中使用 CSharp 的协程
    • C# 中的协程启动都是通过继承自 Mono 的类，使用其中的启动函数 StartCoroutine；
    • Lua 中不能直接使用 C# 当中的 yield return，必须使用 Lua 中的协程返回；
    • 并且不能直接将 Lua 函数传入到 C# 的协程当中进行开启，而是要先引用 XLua 提供的一个工具脚本 xlua.util 之后才可以使用；
  • 总结：
      1. 协程开启：
      • （1）引用 xlua.util；
      • （2）调用 util.cs_generator (fun)；
      • （3）使用 C# 中的 StartCoroutine 开启协程；
      1. yield return：
      • C# 的 yield return 相当于 Lua 中的 coroutine.yield(返回值)
  • 代码：
GameObject = CS.UnityEngine.GameObject
WaitForSceonds = CS.UnityEngine.WaitForSceonds

-- 在场景中新建空物体，然后为其挂在一个继承自 Mono 的类
local obj = GameObject("123")

-- 希望能够被传入 C# 的协程当中的 Lua 函数

	local a = 1
	while true do
		-- 注意：Lua 中不能直接使用 C# 当中的 yield return
		-- 必须使用 Lua 中的协程进行返回
		-- 这里的代码的含义，就是每隔一秒执行一次输出，并进行自增
		coroutine.yield(WaitForSceonds(1))
		print(a)
		a = a + 1
		
		-- 关闭协程
		if a > 10 then 
			mono:StopCoroutine(b)
		end	
	end
end

-- 两个方法：一个yield、一个停止协程

Lua 中使用 CSharp 的泛型函数

概念：Lua 中使用 CSharp 的泛型函数

  • 概念：Lua 中使用 CSharp 的泛型函数
    • 默认情况下，Lua 中仅支持有约束、有参数的 C# 中泛型函数，其他情况都不支持；
    • 如果想支持其他情况，需要使用 xlua.get_generic_method；
  • 概念：xlua.get_generic_method(类，"函数名")
    • xlua.get_generic_method 是 XLua 为解决 Lua 调用 C# 泛型方法而设计的核心 API，其工作原理基于运行时泛型特化技术；
    • C# 泛型方法在编译时只是"模板"，需在运行时指定具体类型参数后才会生成实际方法；
    • XLua 通过此 API 在 Lua 端模拟这一过程：
        1. 在 XLua 侧，会扫描目标类的所有方法，匹配名称和参数数量（忽略泛型参数），返回一个可特化的泛型方法引用；
        1. 在 C# 侧，XLua 通过反射获取 MethodInfo 方法信息，然后动态创建特化版本的委托；
        1. Lua 侧调用时，会检查类型参数合法性并缓存已特化的方法；
    • 在 IL2CPP 模式下，受限于 AOT 编译限制，只有 C# 代码中已实例化过的泛型组合才能在 Lua 中调用；
  • 注意：
      1. 如果是使用 Mono 模式打包，则 xlua.get_generic_method 的方式能够支持使用；
      1. 但是如果使用 IL2CPP，只有泛型参数是引用类型时、才可以使用；
      1. 如果使用 IL2CPP，如果泛型参数是值类型，除非 C# 那边已经调用过了，同类型的泛型参数 Lua 中才能够使用；
  • 代码：
