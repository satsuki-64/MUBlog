WWW 类

WWW 类基础概念

概念：WWW 类基础概念

  • 概念：
    • WWW 是 Unity 提供给我们简单的访问网页的类，可以通过该类下载和上传一些数据；
    • 在使用 HTTP 协议时，默认的请求类型是 Get，如果想要 Post 上传，需要配合下节课学习的 WWWFrom 类使用；
    • 它主要支持的协议
        1. http: 和 https: 超文本传输协议
        1. ftp文件传输协议（但仅限于匿名下载）
        1. file：本地文件传输协议，可以使用该协议异步加载本地文件（PC、IOS、Android 都支持）
  • 概念：WWW 类常用方法和变量
      1. assetBundle：如果加载的数据是 AB 包，可以通过该变量直接获取加载结果；
      1. audioclip：如果加载的数据是音效切片文件，可以通过该变量直接获取加载结果；
      1. bytes：以字节数组的形式获取加载到的内容；
      1. bytesDownloaded：过去已下载的字节数；
      1. error：返回一个错误消息，如果下载期间出现错误，可以通过它获取错误信息；
      1. isDone：判断下载是否已经完成；
      1. movie：如果下载的视频，可以获取一个 MovieTexture 类型结果；
      1. progress：下载进度；
      1. text：如果下载的数据是字符串，以字符串的形式返回内容；
      1. texture：如果下载的数据是图片，以 Texture 2D 的形式返回加载结果；
  • 代码：
// 1. 构造函数  
WWW www = new WWW("http://192.168.50.109:8000/HttpServer/实战就业路线.ipg");  
  
// 2. 下载音效切片  
www.GetAudioClip();  
  
// 3. 将下载得到的贴图加载  
Texture2D texture2D = new Texture2D(100, 100);  
www.LoadImageIntoTexture(texture2D);  
  
// 4. 从缓存加载AB包对象  
WWW www2 = WWW.LoadFromCacheOrDownload("http://192.168.50.109:8000/HttpServer/test.assetbundle",1);

}

WWWFrom 类上传文件

概念：WWWFrom 类基础概念

  • 概念：
    • 如果想要使用 www上传数据时 ，就需要配合 WWWFrom 类进行使用；
    • 而 WWWFrom 主要就是用于集成数据的，可以设置上传的参数或者 2 进制数据，当结合 WWWFrom 上传数据时，它主要用到的请求类型是 Post，它使用 Http 协议进行上传处理；
  • 代码：
IEnumerator UpLoadData()
{
	WWWForm data = new WWWForm();
	data.AddField("Name","WWW",Encoding.UTF8);
	data.AddField("Age",99);
	data.AddBinaryData("file", File.ReadAllBytes(Application.streamingAssetsPath + "/test.jpg"),"PNGName.jpg");
	
	// WWW 的一种重载，可以传入 WWWForm 对象来继承对象的上传
	WWW www = new WWW("http://192.168.50.109:8000/HttpServer/",data);
	
	yield return www;

	if (www.error == null)
	{
		Debug.Log("上传成功");
	}
}
总结：
  • WWW 结合 WWWFrom 上传数据，需要配合后端服务器来指定上传规则，也就是说我们上传的数据，后端需要知道收到数据后应该如何处理，通过这种方式我们没办法像 C# 类当中完成文件的上传。但是该方式非常适合用于制作短连接游戏的前端网络层；
  • 我们可以对 WWW 进行二次封装，专门用于上传自定义消息给对应的 Web 服务器；

UnityWebRequest 类

UnityWebRequest 类基础概念

概念：UnityWebRequest 类基础概念

  • 概念:
    • UnityWebRequest 是一个 Unity 提供的一个模块化的系统类，用于构成 HTTP 请求和处理 HTTP 响应；
    • 它主要目标是让 Unity 游戏和 Web 服务端进行交互，它将之前 WWW 的相关功能都集成在了其中；
    • 所以新版本中都建议使用 UnityWebRequest 类来代替 WWW 类；
  • 注意：
      1. UnityWebRequest 和 WWW 一样，需要配合协同程序使用；
      1. UnityWebRequest 和 WWW 一样，支持 http、ftp、file 协议下载或加载资源；
      1. UnityWebRequest 能够上传文件到 HTTP 资源服务器；

	{
		// 得到下载得到的文本信息
		Debug.Log(req.downloadHandler.text);
		
		// 将其转换为字节数组
		byte[] bytes = req.downloadHandler.data;
	}
	else
	{
		Debug.Log("获取失败" + req.error + req.result + req.responseCode);
	}
}

public const string HTTP_SERVER_PATH = "http://192.168.12.13/http_server/";

/// <summary>
/// 上传文件
/// </summary>
/// <param name="fileName">上传文件的文件名</param>
/// <param name="localPath">上传文件的路径</param>
/// <param name="action">完成之后执行的回调，包含存储结构的Result参数</param>
public void UploadFile(string fileName,string localPath,UnityAction<UnityWebRequest.Result> action)
{
	StartCoroutine(UploadFileAsync(fileName, localPath, action));
}

private IEnumerator UploadFileAsync(string fileName, string localPath, UnityAction<UnityWebRequest.Result> action)
{
	// 添加要上传文件的数据
	List<IMultipartFormSection> list = new List<IMultipartFormSection>();
	list.Add(new MultipartFormDataSection(fileName,File.ReadAllBytes(localPath)));
	
	UnityWebRequest req = UnityWebRequest.Post(HTTP_SERVER_PATH, list);
	
	yield return req.SendWebRequest();
	action?.Invoke(req.result);

	// 如果不成功
	if (req.result != UnityWebRequest.Result.Success)
	{
		Debug.LogWarning("上传出现问题：" + req.error);
	}
}

UnityWebRequest 高级操作

概念：UnityWebRequest 高级操作

  • 引入：高级操作指什么？
    • 在常用操作中，我们使用的是 Unity 为我们封装好的一些方法，我们可以方便的进行一些指定类型的数据获取；
    • 比如下载数据时：
        1. 文本和 2 进制
        1. 图片
        1. AB 包；
    • 如果我们想要获取其它类型的数据应该如何处理呢？
  • 概念：
    • 上传数据时：
        1. 可以指定参数和值
        1. 可以上传文件；
    • 如果想要上传一些基于 HTTP 规则的其它数据应该如何处理呢？高级操作就是用来处理常用操作不能完成的需求的；
    • 它的核心思想就是：UnityWebRequest 中可以将数据处理分离开，比如常规操作中我们用到的，DownloadHandlerTexture 和 DownloadHandlerAssetBundle 两个类，就是用来将 2 进制字节数组转换成对应类型进行处理的；
    • 所以高级操作时指让你按照规则来实现更多的数据获取、上传等功能；

DownloadHandlerScript 类

概念：DownloadHandlerScript 类

  • 概念：
    • DownloadHandlenScript 是一个特殊类，就其本身而言，不会执行任何操作。如果你发现内置的下载处理器（如 DownloadHandlerFile）无法满足特定需求，例如需要实现断点续传、实时监控下载速度或自定义缓存逻辑，那么自定义一个 CustomDownloadFileHandler 类就是一个非常有效的解决方案
    • 此类可由用户定义的类继承，此类接收来自 UnityWebRequest 系统的回调，然后可以使用这些回调在数据从网络到达时执行完全自定义的数据处理；
  • 概念：核心方法
  • 代码：
public class CustomDownLoadFileHander : DownloadHandlerScript
{
	// 用于保存本地存储时的路径
	private string savePath;

	// 用于缓存收到的数据的饿容器
	private byte[] cacheBytes;
	
	// 当前数据索引
	private int index = 0;
	
	public CustomDownLoadFileHander() : base()
	{
		
	}

	public CustomDownLoadFileHander(byte[] bytes) : base(bytes)
	{
		
	}

	public CustomDownLoadFileHander(string path) : base()
	{
		savePath = path;
	}
	
	protected override byte[] GetData()
	{
		if (cacheBytes == null)
		{
			return cacheBytes;
		}

		return null;
	}

	/// <summary>
	/// 从网络收到数据后，每帧会调用的方法
	/// </summary>
	/// <param name="data"></param>
	/// <param name="dataLength"></param>
	/// <returns></returns>
	protected override bool ReceiveData(byte[] data, int dataLength)
	{
		Debug.Log("收到数据的长度：" + dataLength);
		
		// 将当前接收到的字节数组拷贝到当前的缓存容器当中，从index开始
		data.CopyTo(cacheBytes, index);
		index += dataLength;
		
		return true;
	}

	/// <summary>
	/// 从服务器收到 content-Length标头时 会调用
	/// </summary>
	/// <param name="contentLength"></param>
	protected override void ReceiveContentLengthHeader(ulong contentLength)
	{
		// 根据说到的标头，决定字节数据容器的大小
		cacheBytes = new byte[contentLength];
		
		base.ReceiveContentLengthHeader(contentLength);
	}

	/// <summary>
	/// 当父类执行完农村之后，自动调用此方法
	/// </summary>
	protected override void CompleteContent()
	{
		Debug.Log("消息收到");
		
		// 将所有的字节保存到本地
		File.WriteAllBytes(savePath,cacheBytes);
	}
}

public class DownLoadHandlerMsg : DownloadHandlerScript
{
	private BaseMsg msg;
	private byte[] cacheBytes;
	private int index = 0;
	
	public DownLoadHandlerMsg() : base()
	{
		
	}

	protected override byte[] GetData()
	{
		return cacheBytes;
	}

	public T GetMsg<T>() where T : BaseMsg
	{
		return msg as T;
	}

	protected override void ReceiveContentLengthHeader(ulong contentLength)
	{
		cacheBytes = new byte[contentLength];
	}

	protected override bool ReceiveData(byte[] data, int dataLength)
	{
		data.CopyTo(cacheBytes, index);
		index += dataLength;

		return true;
	}

	protected override void CompleteContent()
	{
		// 默认服务器下发的是继承BaseMsg的消息,那么我们在完成时解析它
		index = 0;
		
		int msgID = BitConverter.ToInt32(cacheBytes, index);
		index += 4;
		
		int msgLength = BitConverter.ToInt32(cacheBytes, index);
		index += 4;

		switch (msgID)
		{
			case 1001:
				msg = new PlayerMsg();
				msg.Reading(cacheBytes, index);
				break;
		}

		if (msg == null)
		{
			Debug.Log("对于的ID没有处理");
		}
		else
		{
			Debug.Log("处理成功");
		}
	}
}

概念：下载远程数据或加载本地数据

/// <summary>
/// 通过UnityWebRequest去获取数据
/// </summary>
/// <param name="path">远端或者本地的数据路径</param>
/// <param name="action">获取成功后执行的回调函数</param>
/// <param name="localPath">如果是下载到本地，需要第三个参数</param>
/// <param name="type">是否又音频</param>
/// <typeparam name="T">byte[]、Texture、AssetBundle、AudioClip、File(自定义的，如果是file证明要保存到本地)</typeparam>
public void UnityWebRequestLoad<T>
(
	string path,UnityAction<T> action,
	string localPath = "",AudioType type = AudioType.MPEG
) where T : class
{
	StartCoroutine(UnityWebRequsetLoadAsync<T>(path, action, localPath, type));
}

private IEnumerator UnityWebRequsetLoadAsync<T>(string path,UnityAction<T> action,string localPath = "",AudioType type = AudioType.MPEG) where T : class
{
	UnityWebRequest req = new UnityWebRequest(path,UnityWebRequest.kHttpVerbGET);

	if (typeof(T) ==  typeof(byte[]))
	{
		req.downloadHandler = new DownloadHandlerBuffer();
	}
	else if (typeof(T) == typeof(Texture))
	{
		req.downloadHandler = new DownloadHandlerTexture();
	}
	else if (typeof(T) == typeof(AssetBundle))
	{
		req.downloadHandler = new DownloadHandlerAssetBundle(req.url,0);
	}
	else if (typeof(T) == typeof(File))
	{
		req.downloadHandler = new DownloadHandlerFile(localPath);
	}
	else if (typeof(T) == typeof(AudioClip))
	{
		req = UnityWebRequestMultimedia.GetAudioClip(path, type);
	}
	else
	{
		// 如果出现美誉哦的类型，则不需要继续执行
		yield break;
	}
	
	yield return req.SendWebRequest();

	if (req.result == UnityWebRequest.Result.Success)
	{
		Debug.Log("获取成功");
		
		if (typeof(T) ==  typeof(byte[]))
		{
			action?.Invoke(req.downloadHandler.data as T);
		}
		else if (typeof(T) == typeof(Texture))
		{
			action?.Invoke((req.downloadHandler as DownloadHandlerTexture).texture as T);
		}
		else if (typeof(T) == typeof(AssetBundle))
		{
			action?.Invoke((req.downloadHandler as DownloadHandlerAssetBundle).assetBundle as T);
		}
		else if (typeof(T) == typeof(File))
		{
			// 如果 T 是 File，则需要将文件保存到本地，因此需要自己定义规则
			action?.Invoke(null);
		}
		else if (typeof(T) == typeof(AudioClip))
		{
			action?.Invoke(DownloadHandlerAudioClip.GetContent(req) as T);
		}
	}
}
