HTTP 基础知识

HTTP 工作原理

概念：HTTP 工作原理

  • 概念：
    • HTTP 定义了 Web 客户端（一般指浏览器）如何从 Web 服务器请求 Web 页面，以及服务器如何把 Web 页面传送给客户端；
    • HTTP 客户端首先与服务器建立 TCP 连接，然后客户端通过套接字发送 HTTP 请求，并通过套接字接收 HTTP 响应；
    • 由于 HTTP 采用 TCP 传输数据，因此不会丢包、不会乱序；
  • 概念：工作原理
    • HTTP 通过添加标头（header）的方式向服务端提供本次 HTTP 请求的相关信息，即在主要数据前添加一部分额外信息，称为元信息；
    • 元信息里主要包含：传送的对象属于哪种类型，采用的是哪种编码等等；
    • 因此 HTTP 的元信息标头，类似我们讲解 Socket 通信时用于区分消息类型，处理分包黏包时，在消息体前方加的自定义信息；
    • 在 HTTP 协议中，它也定义了类似的规则，在头部包含了一些额外信息；
  • 概念：HTTP 协议的请求类型和响应状态码
    • 请求类型：
      • HTTP/1.0 中：GET、POST、HEAD；
      • HTTP/1.1 中：GET、POST、HEAD、PUT、DELETE、OPTIONS、TRACE、CONNECT
  • 概念：C# 当中提供的 HTTP 相关类
    • WebRequest、HttpWebRequest、HttpWebResponse 等；
  • 概念：Unity 当中提供的类
    • UnityWebRequest、WWW、WWWFrom 等；

搭建 HTTP 服务器

概念：搭建 HTTP 服务器

  • 概念：
      1. 使用别人做好的 HTTP 服务器软件，一般作为资源服务器时使用该方式（学习阶段建议使用）
      1. 自己编写 HTTP 服务器应用程序，一般作为 Web 服务器或者短连接游戏服务器时使用该方式（工作由后端程序员来做）
  • 概念：使用别人做好的 HTTP 服务器软件来搭建 HTTP 资源服务器
    • 第一步：打开 HFS 软件，修改或者复制端口
      • 默认端口是 80，这里就是默认端口；
    • 第二步：新建一个文件夹作为 HTTP 服务器的资源
      • 将资源放在对于文件夹当中，比如的 Http_server 文件夹；
  • 概念：使用其他人做好的 Web 服务器做测试
    • 我们在学习过程中，可以直接在别人做好的 Web 服务器上获取信息和资源，比如我们可以下载任意网站上可被下载的图片；
    • 可以随便打开一个网站，然后可以使用 HTTP 连接来下载对应的资源，比如它们网站上的图片；
  • 总结：
    • 在实际商业项目开发当中，HTTP 资源服务器可以自己写也可以用别人做好的软件；
    • HTTP 网站服务器或游戏服务器需要自己根据需求进行实现，这些工作一般都是由后端或者运维程序员来进行制作；
    • 主要着重学习前端 HTTP 相关的知识点，在游戏开发时，我们更多时候需要的是 HTTP 的资源服务器；
    • 除非你要做短连接游戏，那么后端程序可以以 HTTP 协议为基础来开发服务端应用程序；
    • 我们只需要学习前端用于进行 HTTP 通信的相关知识即可；

判断资源是否可用

概念：Head - 判断资源是否可用

  • 概念：
    • 主要用于获取文件的一些基础信息可以用于确定文件是否存在；
  • 代码：
private void HeadRes()
{
	try
	{
		// 1. 创建 HTTP 通讯用连接对象 HttpWebRequest 对象
		HttpWebRequest req = HttpWebRequest.Create(new Uri("http://192.168.50.109:8000/Http_server/测试.jpg")) as HttpWebRequest;
	
		// 2. 设置请求类型或其他参数
		req.Method = WebRequestMethods.Http.Head;
		req.Timeout = 2000;
	
		// 3. 发送请求。获取响应结果 HttpWebResponse 对象
		HttpWebResponse res = req.GetResponse() as HttpWebResponse;
	
		// 4. 使用状态码，判断是否下载成功
		if (res.StatusCode == HttpStatusCode.OK)
		{
			Debug.Log("文件存在且可用，长度为：" + res.ContentLength);
		}
		else
		{
			Debug.Log("文件不可用");
		}
	}
	catch (WebException e)
	{
		Debug.Log("获取出错：" + e.Message + e.Status);
	}
}

下载资源到本地文件

概念：Get - 下载资源到本地文件

  • 概念：
    • 主要用于传递信息给服务器，用于获取具体信息；
    • 服务器返回的信息，可以通过 Response 中的流来获取
    • 用 Get 请求时，可以在连接中携带一些额外参数（在链接后面加上 ?参数名=参数值&参数名=参数值&参数名=参数值&）
    • 正常的 http 服务器应用程序，都会去解析 Get 请求时连接中的参数进行逻辑处理（后端程序的工作）
    • 我们主要要掌握的知识点：
        1. 额外参数按格式书写
        1. 通过 response 对象中的流来获取返回的数据；
        • 数据的类型多种多样，可以是文件、自定义消局等等，我们按照规则解析即可；
    • 在和 http 服务器通信时，我们经常会使用额外参数的形式传递信息，特别是以后和一些运营平台对接时；
    • 文件下载功能和 Ftp 非常类似，只是其中使用的类、协议、请求类型不同而已；
  • 代码：
private void LoadRes()
{
	try
	{
		// -------------- 下载资源 -------------------
		// 1. 创建 HTTP 通讯用连接对象 HttpWebRequest 对象
		HttpWebRequest req = HttpWebRequest.Create(new Uri("http://192.168.50.109:8000/Http_server/测试.jpg")) as HttpWebRequest;
	
		// 2. 设置请求类型或其他参数
		req.Method = WebRequestMethods.Http.Get;
		req.Timeout = 3000;
	
		// 3. 发送请求。获取响应结果 HttpWebResponse 对象
		HttpWebResponse res = req.GetResponse() as HttpWebResponse;
	
		// 4. 使用状态码，判断是否下载成功
		if (res.StatusCode == HttpStatusCode.OK)
		{
			Debug.Log("文件存在且可用，长度为：" + res.ContentLength);
			
			// 5. 将资源下载到本地的路径
			using (FileStream fileStream = new FileStream(
					   Application.persistentDataPath + "/ArtRes/测试.jpg", FileMode.Create))
			{
				// 获取响应得到的流
				Stream downLoadStream = res.GetResponseStream();
				byte[] bytes = new byte[1024];
				
				// 读取数据，将其存到容器bytes当中
				int contentLength = downLoadStream.Read(bytes, 0, 1024);
				
				// 写入本地
				while (contentLength != 0)
				{
					fileStream.Write(bytes, 0, contentLength);
					contentLength = downLoadStream.Read(bytes, 0, 1024);
				}
				
				fileStream.Close();
				downLoadStream.Close();
				res.Close();
			}
		}
		
		Debug.Log("下载成功");
	}
	catch (WebException e)
	{
		Debug.Log("下载资源出错：" + e.Message + e.Status);
	}
}

HTTP 上传数据

Post 基础概念

概念：Post 基础概念

  • 概念：Get 和 Post 的区别
    • 相同：
      • Get 和 Post 都可以传递口些额外的参数数据给服务端；
    • 不同：
      • （1）在传递参数时，Post 相对 Get 更加的安全，因为 Post 看不到参数
        • Get 传递的参数都包含在连接中（URL 资源定位地址），是暴露式的；
        • Post 传递的参数放在请求数据中，不会出现在 URL 中，是隐藏式的；
      • （2）Get 在传递数据时有大小的限制，因为它主要是在连接是有限制的（最大长度一般为 2048 个字符），Post 在传递数据时没有限制；
      • （3）传输次数可能不同
        • Get：建立连接 -> 请求行、请求头、请求数据、次传输 -> 获取响应 -> 断开连接；
        • Post：建立连接 -> 传输可能分两次 -> 请求行，请求头第一次传输 -> 请求数据第二次传输—>获取响应 -> 断开；
  • 概念：Post 携带额外参数
    • 将 content-Type 设置为 application/x- www-form-urlencoded 键值对类型；
    • 以下是以下常见的类型：
  • 概念：ContentType 中的重要的类型
      1. 通用二进制类型：
      • application/octet-stream
      1. 通用文本类型：
      • text/plain
      1. 键值对类型：
      • application/x-www-form-urlencoded
      1. 复合类型；
      • multipart/form-data
  • 代码：
HttpWebRequest req = HttpWebRequest.Create("http://192.168.50.109:8000/Http_Server/") as HttpWebRequest;
req.Method = WebRequestMethods.Http.Post;
req.Timeout = 3000;

// 设置上传内容的类型
req.ContentType = "application/x-www-form-urlencoded;charset=UTF-8";

// 要上传的数据
string str = "Name=WWW&ID=2";
byte[] bytes = Encoding.UTF8.GetBytes(str);
req.ContentLength = bytes.Length; // 注意在上传之前要设置内容的长度

// 上传数据
Stream stream = req.GetRequestStream();
stream.Write(bytes, 0, bytes.Length);
stream.Close();

// 发送数据，然后得到响应结果
HttpWebResponse res = req.GetResponse() as HttpWebResponse;
Debug.Log(res.StatusCode);

HTTP 上传数据实现

概念：HTTP 上传数据实现

  • 概念：上传文件到 HTTP 资源服务器需要遵守的规则
    • HTTP 上传文件相对比较麻烦，需要按照指定的规则进行内容拼接，达到上传文件的目的；
    • 上传文件时内容的必备规则：
      1. ContentType =“multipart/form-data；boundary=边界字符串"；
      1. 上传的数据必须按照格式写入流中：
      1. 保证服务器允许上传；
      1. 写入流前需要先设置 contentLength 内容长度；
  • 代码：
// 1. 创建对象
HttpWebRequest req = HttpWebRequest.Create("http://192.168.50.109:8000/Http_Server") as HttpWebRequest;

// 2. 相关设置
req.Method = WebRequestMethods.Http.Post;
req.ContentType = "multipart/form-data;boundary=WWW";
req.Timeout = 500000;
req.Credentials = new NetworkCredential("WWW","123123");
req.PreAuthenticate = true; // 先验证身份，再上传数据，因此需要开启这个

// 3. 按格式拼接字符串并且转为字节数组之后用于上传
string head = "--WWW\r\n" +
			  "Content-Disposition:form-data;name=\"file\";filename=\"http上传的图片.jpg\"\r\n" +
			  "Content-Type:application/octet-stream\r\n\r\n";
byte[] headBytes = Encoding.UTF8.GetBytes(head);

// 结束的边界信息，表示边界字符串
byte[] endBytes = Encoding.UTF8.GetBytes("\r\n--WWW--\r\n");

// 4.写入上传流
using (FileStream localFileStream = File.OpenRead(Application.streamingAssetsPath + "/test.png")) 
{
	// 4-1.设置上传长度
	// 总长度是前部分字符串+文件本身多大+后部分边界字符串
	req.ContentLength = headBytes.Length + localFileStream.Length + endBytes.Length;
	// 用于上传的流
	Stream upLoadStream = req.GetRequestStream();
	
	// 4-2.先写入前部分头部信息
	upLoadStream.Write(headBytes, 0, headBytes.Length);
	
	// 4-3.再写入文件数据
	// 文件可能很大，因此一次写一部分
	byte[] bytes = new byte[2048];
	int contentLength = localFileStream.Read(bytes, 0, bytes.Length);
	while (contentLength != 0) 
	{
		upLoadStream.Write(bytes, 0, contentLength);
		contentLength = localFileStream.Read(bytes, 0, bytes.Length);
	}
	
	// 4-4.在写入结束的边界信息
	upLoadStream.Write(endBytes, 0, endBytes.Length);
	upLoadStream.Close();
}

// 5 上传数据，获取响应
HttpWebResponse res = req.GetResponse() as HttpWebResponse;
if (res.StatusCode == HttpStatusCode.OK)
{
	Debug.Log("上传成功");
}
else
{
	Debug.Log("上传失败");
}
