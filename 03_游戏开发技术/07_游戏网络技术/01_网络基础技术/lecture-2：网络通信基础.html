Lecture 2：网络通信基础

游戏类型与常见协议

概念：长连接和短连接游戏

  • 短连接游戏：
    • 概念：
      • 需要传输数据时，建立连接，传输数据，获得响应，断开连接
    • 通信特点：
      • 需要通信时再连接，通信完毕断开连接；
    • 通信方式：
      • HTTP 超文本传输协议、HTTPS 安全的超文本传输协议 (他们本质上是 TCP 协议)
  • 长连接游戏：
    • 概念：
      • 不管是否需要传输数据，客户端与服务器一直处于连接状态，除非一端主动断开，或者出现意外情况，比如客户端关闭或服务端崩溃等；
    • 通信特点：
      • 连接一直建立，可以实时的传输数据；
    • 通信方式：
      • TCP 传输控制协议或 UDP 用户数据报协议；

概念：常见协议

  • 概念：Socket - 网络套接字
    • 是对网络中不同主机上的应用进程之间进行双向通信的端点的抽象，一个套接字就是网络上进程通信的一段，提供了应用层进程利用网络协议交换数据的机制；
    • 主要要学习的就是 Socket 网络套接字当中的各种 API 来进行网络通信；
    • 主要用于制作长连接游戏 (强联网游戏)
  • 概念：Http/Https (安全的) 超文本传输协议
    • 是一个简单的请求-响应协议，它通常运行在 TCP 协议之上，它指定了客户端可能发送给服务端什么样的信息以及得到什么样的响应，主要用于制作短连接游戏 (弱联网游戏)，也可以用来进行资源下载；
  • 概念：FTP 文件传输协议；

IP 地址和端口类

IPAddress 类

概念：IPAddress 类

  • 作用：
    • C# 中提供了对应的 IP 和端口相关的类，来声明对应信息；
  • 概念：IPAddress 类
    • 命名空间：System.Net
  • 方法：初始化 IP 信息的方式
      1. 用 byte 数组进行初始化
      1. 用 long 长整型进行初始化 4 字节对应的长整型
      • 一般不建议大家使用；
      1. 推荐使用的方式：使用字符串转换
      • IPAddress.Parse 方法；
  • 补充：
    • （1）获取可用的 IPv6 地址：IPAddress.IPv6Any
    • （2）127.0.0.1 代表本机地址；
  • 示例：
// 方法一：用 `byte` 数组进行初始化
byte[] ip1 = new byte[]{118,102,111,11}

// 方法二：用字符串转换
IPAddress ip2 = IPAddress.Parse("118.102.111.11");

IPEndPoint 类

概念：IP 和端口的组合类 - IPEndPoint 类

  • 概念：
    • IPEndPoint 是一个 IP 和端口的组合类；
    • IPEndPoint 类将网络端点表示为 IP 地址和端口号，表现为 IP 地址和端口号的组合；
  • 方法： 
// 第一个参数为IP地址，第二个参数为端口号
IPEndPoint ipPoint2 = new IPEndPoint(IPAddress.Parse("118.111.111.11"),8080);

域名解析

域名解析基础概念与实现

概念：域名解析

  • 概念：
    • 域名解析就是从域名到 IP 地址的转换过程；
    • 域名的解析工作由 DNS 服务器完成；
  • 概念：IPHostEntry 类
    • 命名空间：
      • System.Net
    • 类名：
      • IPHostEntry
    • 作用：
      • 存储域名解析后的返回值的变量类型，可以通过该对象获取 IP 地址、主机名等等信息；
    • 概念：
      • 该类不会自己声明，都是作为某些方法的返回值返回信息，我们主要通过该类对象获取返回的信息；
        1. 获取关联 IP 的成员变量：AddressList；
        1. 获取主机别名列表成员变量：Aliases；
        1. 获取 DNS 名称成员变量：HostName；
  • 概念：Dns 类
    • 命名空间：
      • System.Net
    • 类名：
      • Dns
    • 作用：
      • Dns 是一个静态类，提供了很多静态方法，可以使用它来根据域名获取 IP 地址；
  • 补充：
    • 可以使用异步方法来获取 Dns 相关信息；

示例：代码示例

using System.Threading.Tasks;
using UnityEngine;
 
public class Test : MonoBehaviour
{
    void Start()
    {
        //常用方法
        //1.获取本地系统的主机名
        print(Dns.GetHostName());
 
        //2.获取指定域名的IP信息
        //根据域名获取
        //同步获取
        //注意：由于获取远程主机信息是需要进行网路通信，所以可能会阻塞主线程
        //IPHostEntry entry = Dns.GetHostEntry("www.baidu.com");
        //for (int i = 0; i < entry.AddressList.Length; i++)
        //{
        //    print("IP地址：" + entry.AddressList[i]);
        //}
        //for (int i = 0; i < entry.Aliases.Length; i++)
        //{
        //    print("主机别名" + entry.Aliases[i]);
        //}
        //print("DNS服务器名称" + entry.HostName);
 
 
        // 2. 获取指定域名的IP信息
        // 异步获取
        GetHostEntry();
    }
 
    private async void GetHostEntry()
    {
	    // 异步的去执行Dns.GetHostEntryAsync方法
	    // DNS查询属于I/O密集型操作，.NET 使用操作系统的I/O完成端口机制，实现真正的异步I/O
	    // 如果使用了Task.Run等方法来包装一个同步的阻塞操作，那么它可能会在新的线程上执行。这样做的目的是为了将阻塞操作转换为异步操作，以避免阻塞主线程
        Task<IPHostEntry> task = Dns.GetHostEntryAsync("www.baidu.com");
        
        // 使用 await task 等待异步操作完成，此时控制权会返回给调用方
        await task;
	    
	    // 通过 Result.AddressList获得当前得到的IP地址序列
        for (int i = 0; i < task.Result.AddressList.Length; i++)
        {
            print("IP地址：" + task.Result.AddressList[i]);
        }
        
        // 通过 Result.Aliases获得主机别名序列
        for (int i = 0; i < task.Result.Aliases.Length; i++)
        {
            print("主机别名" + task.Result.Aliases[i]);
        }
        
        print("DNS服务器名称" + task.Result.HostName);
    }
  • 分析：Dns.GetHostEntryAsync("www.baidu.com") 的异步效果
    • Dns.GetHostEntryAsync 是 .NET 提供的原生异步 DNS 查询方法，其底层通过操作系统的异步 I/O 机制（如 Windows 的 I/O 完成端口）实现，当调用该方法时：
      1. 请求被提交给操作系统内核，由内核直接处理 DNS 查询；
      1. 在等待 DNS 响应期间，不占用任何线程资源；
      1. 当操作系统完成 DNS 查询后，通过回调通知 .NET 任务系统，恢复异步方法执行；
  • 分析：使用 Task 是否会启动新线程？
    • 使用 GetHostEntryAsync 的是真正的异步 IO，依赖操作系统支持，无需线程等待；
    • 而基于线程池的异步包装（如 Task.Run）将同步方法（如 Dns.GetHostEntry）强制放到线程池线程执行，实际仍会阻塞线程池线程；
    • 因此关键区别是：如果使用 Task.Run(() => Dns.GetHostEntry("www.baidu.com"))，确实会创建线程池线程，但 GetHostEntryAsync 是专门设计的异步 API，直接使用操作系统原生异步能力；
  • 结果：
    • 两个 DNS 服务器的地址，没有主机别名；

异步方法获得 Http 响应

示例：异步方法获得 Http 响应

  • 概念：
    • 在 C# 中，异步编程是一种用于处理长时间运行的操作（如 I/O 操作、网络请求或大量计算）的技术，以避免阻塞主线程，从而提高应用程序的响应性和性能；
    • C# 提供了多种方法来实现异步编程，其中最常见的是使用 async 和 await 关键字；
  • 实现：
    • 使用 HttpClient 类进行网络请求，并使用 await 关键字等待请求完成；
  • 语法：
    • async 关键字：
      • 用于标记一个方法为异步方法；
      • 异步方法允许在方法内部使用 await 关键字；
    • await 关键字：
      • 用于等待异步操作的完成，但不会阻塞调用线程；
      • 它只能在标记为 async 的方法内部使用；
  • 注意：
    • 异步方法中的异常需要在调用方法中使用 try-catch 块捕获；
  • 示例： 
using System;  
using System.Net.Http;  
using System.Threading.Tasks;  
  
class Program  
{  
    static async Task Main(string[] args)  
    {  
        string url = "https://jsonplaceholder.typicode.com/posts/1";  
        string content = await FetchDataAsync(url);  
        Console.WriteLine(content);  
    }  
  
    // 使用async关键字标记为异步方法。
    // 返回一个Task<string>，表示该方法在异步操作完成后返回一个字符串。
    static async Task<string> FetchDataAsync(string url)  
    {  
        // 使用HttpClient类进行网络请求，并使用await关键字等待请求完成。
        using (HttpClient client = new HttpClient())  
        {  
	        // 得到HTTP响应，存储在response中
            HttpResponseMessage response = await client.GetAsync(url);  
            response.EnsureSuccessStatusCode();  
            string responseBody = await response.Content.ReadAsStringAsync();  
            return responseBody;  
        }  
    }  
}

序列化

二进制数据序列化与反序列化

概念：二进制数据序列化与反序列化

  • 概念：
    • 在网络通信中，我们把想要传递的类对象信息序列化为 2 进制数据 (一般为 byte 字节数组)，再将该 2 进制数据通过网络传输给远端设备远端设备，获取到该 2 进制数据后、再将其反序列化为对应的类对象；
  • 概念：和网络通信相关的重要知识点
      1. BitConverter 类：
      • 主要用于处理各类型和字节数组间的相互转换；
      1. Encoding 类：
      • 主要用于处理字符串类型和字节数组间的相互转换；
      1. 加密相关：
      • 了解 2 进制数据加密的常用手段和思路；
  • 概念：本地持久化知识点
      1. File 类：文件操作类，用于操作文件
      1. FileStream 类：文件流类，以流的形式进行文件存储读取操作
      1. MemoryStrem：内存流对象
      1. BinaryFormatter：2 进制格式化对象
  • 注意：
    • 在 2 进制知识点我们讲解了 BinaryFormatter 类，它可以快速的将 C# 类对象转换为字节数组数据；
    • 但是在网络开发时，我们不会使用 BinaryFormatter 类来进行数据的序列化和反序列化，因为客户端和服务端的开发语言大多数情况下是不同的，BinaryFormatter 类序列化的数据无法兼容其它语言；

数据序列化

概念：字符串与非字符串类型转化为字符数组

  • 概念：非字符串类型转字节数组
    • 示例：byte[] bytes = BitConverter.GetBytes(1);
  • 概念：字符串类型转字节数组
    • 示例：byte[] bytes = Encoding.UTF8.GetBytes("测试一下");
  • 总结：
    • 我们对类对象的 2 进制序列化主要用到的知识点是
      1. BitConverter 转换非字符串的类型的变量为字节数组；
      1. EncodingUTF8 转换字符串类型的变量为字节数组；
  • 注意:
    • 为了考虑反序列化，我们在转存 2 进制、序列化字符串之前，先序列化字符串字节数组的长度；

概念：将类对象转化为二进制

  • 概念：
    • 网络通信中我们不能直接使用数据持久化 2 进制知识点中的 BinaryFormatter 2 进制来格式化类；
    • 因为客户端和服务器使用的语言可能不一样，BinaryFormatter 是 C# 的序列化规则，和其它语言之间的兼容性不好。如果使用它，那么其它语言开发的服务器无法对其进行反序列化；
    • 我们需要自己来处理将类对象数据序列化为字节数组；
  • 方法：
    • 第一步：明确字节数组的容量
    • 第二步：声明一个装载信息的字节数组容器
    • 第三步：将对象中的所有信息转为字节数组，并放入该容器中；
  • 示例：
using System;

public class PlayerInfo
{
	public int lev;
	public string name;
	public bool sex;
}

public class Lesson : MonoBehaviour 
{
	void Start()
	{
		// 第一步：明确字节数组的容量
		PlayerInfo info = new PlayerInfo();
		info.lev = 10;
		info.name = "小王";
		info.sex = true;

		// 得到的 这个Info数据 如果转换成 字节数组 那么字节数组容器需要的容量
		int indexNum = sizeof(int) + sizeof(bool) + Encoding.UTF8.GetBytes(info.name).Length;

		// 第二步：申明一个装载信息的字节数组容器
		byte[] playerBytes = new byte[indexNum];

		// 第三步：将对象中的所有信息转为字节数组，并放入该容器中 
		int index = 0; // 从 playerBytes 数组中的第几个位置去存储数据

		// 把lev的四个字节从第0个位置放入到字节数组中
		// 使用CopyTo方法，将lev字节复制到playerBytes中
		BitConverter.GetBytes(info.lev).CopyTo(playerBytes,index);
		index += sizeof(int);

		// 名字
		byte[] strBytes = Encoding.UTF8.GetBytes(info.name);
		int num = strBytes.Length;

		// 存储的是姓名转化为字节数组后，字节数组的长度
		BitConverter.GetBytes(num).CopyTo(playerBytes,index);
		index += sizeof(int);

		// 存储字符串的字节数组
		strBytes.CopyTo(playerBytes,index);
		index += num;

		// 性别
		BitConverter.GetBytes(info.sex).CopyInfo(playerBytes,index);
		index += sizeof(bool);
	}

}
