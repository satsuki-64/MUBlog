协议生成工具

什么是协议生成工具

概念：什么是协议生成工具

  • 概念：
    • 协议生成工具，一般指消息（协议）生成工具，就是专门用于自动化生成消息的程序；
    • 当需要一个新消息时，我们需要手动的按照规则去声明新的类，这部分工作费时又费力，技术含量也不高；
    • 如果前后端是统一的语言，我们按照语法声明一次就行，但是如果前后端语言不统一，比如前端用 C#，后端用 C++ 或者 Java，那么前后端分开去声明，也容易造成沟通不一致，声明不统一的问题；
    • 所以如果靠我们手动的去声明消息类是一件费时、费力、还容易出问题的事情；
    • 因此所以我们在商业游戏开发时，往往就需要使用协程生成工具来帮助我们自动化的声明消息类；
  • 举例：
    • 之前写了一个 BaseMsg 类之后，如果增加了其他的消息子类，比如 PlayerMsg，那就需要写一遍以下相关内容：
    • 如果又有一个其他的继承自 BaseMsg 的类，又需要写一个继承自 BaseMsg 的类，耗时耗力；
  • 示例：实现的功能示例
    • 比如说现在有一个 XML 文件：
    • 可以看到，其中配置了比如说 data 类型、enum 类型、message 类型这三种类型，然后可以定义不同的枚举、数据、消息；
    • 因此，如果需要一些不同的消息类型，则此时就可以在 XML 中写此类型具有的一些属性。然后配置好 XML 后，只要点击按钮，就可以为其生成对应的 C# 代码；
  • 实现：
    • 对于协议生成工具来说，主要需求如下：
        1. 通过配置文件，配置消息或数据类名字、变量等；
        1. 工具根据该配置文件信息动态的生成类文件（脚本文件，代码是自动生成的）；
        1. 我们可以在开发中直接使用生成文件中声明好的消息和数据结构类进行开发；
    • 根据需求，需要实现以下内容：
        1. 确定协议配置方式
        • 可以使用 JSON、XML、自定义格式进行协议配置；
        • 主要目的，是通过配置文件确定：
        • 1：消息或者数据结构类名字
        • 2: 字段名等
        1. 确定生成格式
        • 最终我们是要自动生成类声明文件，所以具体类应该如何生成需要确定格式；
        • 比如：继承关系固定写法、序列化、反序列化固定写法、提取出共同点；

生成枚举类

概念：什么是协议生成

  • 概念：
    • 协议生成主要是使用配置文件中读取出来的信息动态的生成对应语言的代码文件，每次添加消息或者数据结构类时我们不需要再手写代码了；
    • 我们不仅可以生成 C# 脚本文件，还可以根据需求生成别的语言的文件，比如 JAVA、C++；
  • 概念：制作功能前的准备工作
    • 协议生成是不会在发布后使用的功能，主要是在开发时使用，所以我们在 Unity 当中可以把它作为一个编辑器功能来做；
    • 因此我们可以专门新建一个 Editor 文件买（专门放编辑器相关内容，不会发布），在其中放置配置文件、自动生成相关脚本文件；
  • 实现：ProtocolTool.cs - 获取 XML 节点

  • 代码：GenerateCSharp.cs - 生成 C# 枚举脚本
public class GenerateCSharp
{
	public void GenerateEnum(XmlNodeList nodes,string savePath)
	{
		// 生成枚举脚本的逻辑
		string namespaceStr = "";
		string enumNameStr = "";
		string fieldStr = "";

		foreach (XmlNode enumNode in nodes)
		{
			// 获取命名空间配置信息
			namespaceStr = enumNode.Attributes["namespace"].Value;
			
			// 获得枚举名称
			enumNameStr = enumNode.Attributes["name"].Value;

			// 获取所有的字段节点然后进行字符串拼接
			XmlNodeList enumFields = enumNode.SelectNodes("field");
			fieldStr = "";
			foreach (XmlNode enumField in enumFields)
			{
				fieldStr += "\t\t" + enumField.Attributes["name"].Value;
				if (enumField.InnerText != "")
				{
					fieldStr += " =" + enumField.InnerText;
				}
				
				fieldStr += ",\r\n";
			}
			
			// 对所有可变的内容进行拼接
			string enumStr = $"namespace {namespaceStr}\r\n" +  
							 "{\r\n" + 
							 $"\tpublic enum {enumNameStr}\r\n" + 
							 "\t{\r\t" + 
							 $"{fieldStr}" + 
							 "\t}\r\n" + 
							 "}";
			
			// 保存文件
			string path = savePath + namespaceStr + "/Enum";
			Debug.Log(path);
			if (!Directory.Exists(path))
			{
				Directory.CreateDirectory(path);
			}
			
			// 字符串保存，存储为枚举脚本
			File.WriteAllText(path + "/" + enumNameStr + ".cs",enumStr);
		}
	}
}

public class PlayerData : BaseData
{
	// 1
	public string name;
	public int atk;
	public int lev;
	
	// 2 
	public override int GetBytesNum()
	{
		return 4 + 4 + 4 + Encoding.UTF8.GetBytes(name).Length;
	}

	// 3
	public override byte[] Writing()
	{
		int index = 0;
		byte[] bytes = new byte[GetBytesNum()];
		WriteString(bytes, name, ref index);
		WriteInt(bytes, atk, ref index);
		WriteInt(bytes, lev, ref index);
		
		return bytes;
	}

	// 4
	public override int Reading(byte[] bytes, int beginIndex = 0)
	{
		int index = beginIndex;
		name = ReadString(bytes,ref index);
		atk = ReadInt(bytes, ref index);
		lev = ReadInt(bytes, ref index);
		
		return index - beginIndex;
	}
}

public void GenerateData(XmlNodeList nodes,string savePath)
{
	// 生成枚举脚本的逻辑
	string namespaceStr = "";
	string classNameStr = "";
	string fieldStr = "";
	
	foreach (XmlNode node in nodes)
	{
		// 获取命名空间配置信息
		namespaceStr = node.Attributes["namespace"].Value;
		
		// 获得类名
		classNameStr = node.Attributes["name"].Value;

		// 获取所有的字段节点然后进行字符串拼接
		XmlNodeList fields = node.SelectNodes("field");
		fieldStr = GetFieldStr(fields);
		
		string dataStr = "using System.Collections.Generic;\r\n" +  
						 $"namespace {namespaceStr}\r\n" +  
						 "{\r\n" + 
						 $"\tpublic class {classNameStr} : BaseData\r\n" + 
						 "\t{\r\t" + 
						 $"{fieldStr}" +
						 "\t}\r\n" + 
						 "}";
		
		// 保存文件
		string path = savePath + namespaceStr + "/Data";
		Debug.Log(path);
		if (!Directory.Exists(path))
		{
			Directory.CreateDirectory(path);
		}
		
		// 字符串保存，存储为枚举脚本
		File.WriteAllText(path + "/" + classNameStr + ".cs",dataStr);
	}
}

/// <summary>
/// 获取成员变量声明内容
/// </summary>
/// <param name="fields"></param>
/// <returns></returns>
private string GetFieldStr(XmlNodeList fields)
{
	string fieldStr = "";
	
	foreach (XmlNode field in fields)
	{
		// 得到变量类型
		string type = field.Attributes["type"].Value;
		
		// 变量名
		string fieldName = field.Attributes["name"].Value;

		// 变量前面的类型声明
		if (type == "list")
		{
			string T = field.Attributes["T"].Value;
			fieldStr += "\t\tpublic " + $"List<{T}>";
		}
		else if (type == "array")
		{
			string data = field.Attributes["data"].Value;
			fieldStr += "\t\tpublic " + data + "[] ";
		}
		else if (type == "dic")
		{
			string TKey = field.Attributes["Tkey"].Value;
			string TValue = field.Attributes["Tvalue"].Value;
			fieldStr += "\t\tpublic " + $"Dictionary<{TKey}, {TValue}>";
		}
		else
		{
			fieldStr += "\t\tpublic " + type + " ";
		}
		
		// 变量名声明
		fieldStr += fieldName + ";\r\n";
	}

	return fieldStr;
}

概念：Proto 配置规则

  • 概念：
    • 注意后缀必须要是 proto 格式
  • 概念：
// 1. 第一行是版本号
syntax = "proto3"

// 2. 命名空间
package GamePlayerTest

// 3. 消息类
message TestMsg{
	// 4. 成员类型和唯一编号
	// 浮点数：float、double
	// 整数：变长编码-int32，int64，uint32，uint64；固定字节数-fixed32,fixed64，sfixed32,sfixed64
	// 其它类型：bool，string，bytes
	// 唯一编号：配置成员时需要默认给他们一个编号从1开始
	// =1 不代表默认值，而是代表唯一编号，方便我们进行序列化
	
	// 格式： 类型 变量名 = 唯一编号;
	// 浮点数类型
	float testF = 1;
	double testD = 2;
	
	// 变长编码整数
	// 所谓变长就是会根据数字的大小来使用对应的字节数来存储1、2、4，Protobuf 帮助我们优化的部分，可以尽量少的使用字节数来存储
	int32 testInt32 = 3; // C# - int 它不太适用于来表示负数，表现负数请使用sint32
	int64 testInt64 = 4; // C# - 1ong 它不太适用于来表示负数请使用sint64
	
	// 表示负数类型的整数
	sint32 testSInt32 = 5; // C# - int 适用于来表示负数的整数
	sint64 testSInt64 = 6; // C# - 1ong 适用于来表示负数的整数
	
	// 无符号变成编码
	uint32 testUInt = 7; // C# - uint 变长的编码
	uint64 testULong = 8; // C# - ulong 变长的编码
	
	// 固定字节数的类型
	fixed32 testFixed32 = 9; // C# - uint 它通常用来表示大于2的28次方的数，其固定始终是4个字节
	fixed64 testFixed64 = 10; // C# - ulong 它通常用来表示大于2的56次方的数，比uint64更有效始终是8个字节
	sfixed32 testSFixed32 = 11; // C# - int 始终4个字节
    sfixed64 testSFixed64 = 12; // C# - int 始终8个字节
    
    //其它类型
	bool testBool = 13; // C#-bool
	string testStr = 14; // C#-string
	bytes testBytes = 15; // C#-Bytesstring字节字符串
}
