Lecture 1：ScriptableObject

基础概念

概念：什么是 ScriptableObject

  • 概念：
    • ScriptableObject 是 Unity 引擎中一个数据容器类，它允许你创建用于创建不依赖于场景中 GameObject 的独立数据资源（.asset 文件），更优雅地管理游戏配置和数据，提升项目的可维护性和效率；
  • 区别：
  • 应用：
      1. 游戏配置管理​：存储角色属性（血量、速度）、游戏平衡参数（重力、时间缩放）、难度级别配置等；
      1. 资源管理系统​：构建物品数据库（武器、道具、装备）、技能效果配置、音效和视觉特效预设库；
      1. ​编辑器扩展​：为自定义工具配置、批量处理设置和数据验证规则提供支持；

使用 ScriptableObject

使用方法

概念：ScriptableObject 使用方法

  • 概念：定义 ScriptableObject 类
using UnityEngine;

[CreateAssetMenu(fileName = "NewCharacterData", menuName = "Game Data/Character Data", order = 1)]
public class CharacterData : ScriptableObject
{
    public string characterName;
    public int maxHealth;
    public float moveSpeed;
    public Sprite icon;
    
    // 可以添加方法
    public void TakeDamage(int damage) {
        maxHealth -= damage;
    }
}
  • 概念：创建资产 - 使用编辑器
    • 定义 ScriptableObject 类之后，可以为其创建 asset 资源。可以在 Unity 编辑器 Project 窗口 右键 -> ​Create -> Game Data -> Weapon Data，即可创建一个名为 "NewWeaponData.asset" 的资源文件；
    • 你可以在 Inspector 窗口中可视化地编辑它的所有公共字段；
  • 概念：创建资产 - 使用 API
public class ScriptableObjectTool
{
	[MenuItem("ScriptableObject/CreateMyData")]
	public static void CreateMyData()
	{
		CharacterData asset = ScriptableObject.CreateInstance<CharacterData>();
		
		// 通过编辑器API，根据数据创建一个数据资源文件
		AssetDatabase.CreateAsset(asset,"Assets/Resources/MyDataTest.asset");
		
		AssetDatabase.SaveAssets();
		AssetDatabase.Flush();
	}
}
注意：声明成员的注意事项
  • 我们可以声明任意类型的成员变量，但是如果希望能够在 Inspector 中编辑，那么这里的变量声明规则，要和 MonoBehaviour 中 public 变量的规则一样；

注意：
  • 编辑器中的数据持久化，通过代码修改数据对象中内容，会影响数据文件，相当于达到了编辑器中数据持久化的目的，但是该数据持久化只是在编辑模式下的持久，发布运行时并不会保存数据；
  • 因为在 Unity 编辑器中对 ScriptableObject 资源文件（.asset 文件）所做的修改会被保存到磁盘上，即使关闭项目再打开，数据也会保留。这使它非常适合用作配置文件；
  • 如果需要永久保存运行时的更改，需要配合其他持久化方案（如 JSON、XML）；

ScriptableObject 非持久化数据

概念：ScriptableObject 非持久化数据

  • 概念：
    • ScriptableObject 非持久化数据，指的是不管在编辑器模式还是在发布后都不会持久化的数据；
    • 我们可以根据自己的需求，随时创建对应数据对象进行使用，就好像直接 new 一个数据结构类对象；
  • 方法：ScriptableObject.CreateInstance<>()
    • 利用 scriptableobject 中的静态方法 CreateInstance<>()，该方法可以在运行时创建出指定继承 scriptableobject 的对象，该对象只存在于内存当中，可以被 GC，调用一次就创建一次；
  • 代码：
private ChacracterData data;
private void Test()
{
	// 使用此静态方法，在其内部会自动实例化 ChacracterData 对象
	// 通过这种方式创建出来的对象，其默认值不会受脚本中设置的影响，只会使用 null 或者 0 等默认值
	data = ScriptableObject.CreateInstance<ChacracterData>();

	data.PrintInfo();
}
  • 作用：
    • 只是希望在运行时能有一组唯一的数据可以使用，但是这个数据又不太希望保存为数据资源文件浪费硬盘空间，那么 Scriptableobject 的非持久化数据就有了存在的意义；
    • 它的特点是只在运行时使用，在编辑器模式下也不会保存在本地；

ScriptableObject 持久化数据

概念：ScriptableObject 与 JSON

// 将数据对象序列化为JSON字符串  
string str = JsonUtility.ToJson(data);  
Debug.Log("persistentDataPath：" + Application.persistentDataPath);  
// 将序列化数据存储到本地

// 结合JSON读取数据  
string str2 = File.ReadAllText(Application.persistentDataPath + "/ChacracterData.json");  
// 将读取出来的JSON转换后，覆盖到 data 当中  
JsonUtility.FromJsonOverwrite(str2, data);

ScriptableObject 的应用

ScriptableObject 作为配置文件

概念：ScriptableObject 作为配置文件

  • 概念：
    • ScriptableObject 作为配置文件的三个特点
      1. 配置文件的数据在游戏发布之前定规则；
      1. 配置文件的数据在游戏运行时只会读取出来使用，不会改变内容；
      1. 在 Unity 的 Inspector 窗口进行配置更加的方便；
  • 概念：配置方式
    • 之前学习过的配置文件，比如 XML、JSON 等都需要使用第三方的文件，而使用 ScriptableObject 直接在 Unity 当中进行配置；
  • 代码：
using System.Collections.Generic;
using UnityEngine;

namespace Test
{
    [CreateAssetMenu(fileName = "RoleInfo",menuName = "ScriptableObjects/RoleInfo")]
    public class RoleInfo : ScriptableObject
    {
        [System.Serializable]
        public class RoleData
        {
            public int id;
            public string res;
            public int atk;
            public string tips;
            public int lockMoney;
            public int type;
            
            public void PrintInfo()
            {
                Debug.Log(id);
                Debug.Log(res);
                Debug.Log(atk);
                Debug.Log(tips);
                Debug.Log(type);
                Debug.Log(lockMoney);
                Debug.Log(type);
            }
        }

        public List<RoleData> roleList;
    }
}
  • 总结：
    • 我们可以利用 scriptableobject 数据文件来制作编辑器相关功能；
    • 比如：unity 内置的技能编辑器、关卡编辑器等等。我们不需要把编辑器生成的数据生成别的数据文件，而是直接通过 scriptableobject 进行存储，因为内置编辑器只会在编辑模式下运行，编辑模式下 scriptableobject 具备数据持久化的特性；

使用 ScriptableObject 节约内存

概念：使用 ScriptableObject 节约内存

  • 对于不同对象的相同数据，如果使用类来存储，则每一个实例都会占据一份内存。但如果使用 ScriptableObject ，则若干对象的相同数据都使用同一个 ScriptableObject 存储，更改这个 ScriptableObject 的值时，会影响到所有使用这个 ScriptableObject 的物体。因此可以节约内存；

数据带来的多态行为

概念：数据带来的多态行为

  • 概念：
    • 某些行为的变化是因为数据的不同带来的，我们可以利用面向对象的特性和原则，以及设计模式相关知识点，结合 scriptableobject 做出更加方便的功能，比如随机音效，物品拾取，AI 等等等；
  • 示例：基于里氏替换原则和依赖倒转原则实现随机音效，播放音乐时，可能会随机播放多个音效当中的一种；
public abstract class AudioPlayBase : ScriptableObject
{
	public abstract void Play(AudioSource source);
}

[CreateAssetMenu(fileName = "Music",menuName = "ScriptableObject/RandomPlayMusic")]
public class RandomPlayMusic : AudioPlayBase
{
	/// <summary>
	/// 希望随机播放的音乐文件
	/// </summary>
	public List<AudioClip> clips;
	
	public override void Play(AudioSource source)
	{
		if (clips.Count == 0)
		{
			return;
		}
		
		// 设置音效切片并且播放
		source.clip = clips[Random.Range(0, clips.Count)];
		source.Play();
	}
}

[CreateAssetMenu(fileName = "Music",menuName = "ScriptableObject/PlayMusic")]
public class PlayerAudio : AudioPlayBase
{
	public AudioClip clip;
	
	public override void Play(AudioSource source)
	{
		this.clip = source.clip;
		source.Play();
	}
}
