文件流

什么是文件流

概念：什么是文件流

  • 概念：
    • 在 C# 中提供了一个文件流类 Filestream 类；
    • 它主要作用是用于读写文件的细节，之前学过的 File 只能整体读写文件，而 FileStream 可以以读写字节的形式处理文件；
    • 文件流)是 C# 中对文件进行 字节级操作 的类，它继承自抽象基类 Stream，提供了对文件内容的顺序或随机访问能力；
    • 从技术角度看，文件流是连接应用程序与物理文件的"数据管道"，通过这个管道，数据可以以字节序列的形式流动；
    • 比如：我可以先存一个 int (4 个字节)，再存一个 bool (1 个字节)，再存一个 string (n 个字节)；
  • 特点：
      1. 多种打开模式​：通过 FileMode 枚举控制文件创建/打开行为；
      1. 精细访问控制​：通过 FileAccess 指定读、写或读写权限；
      1. 共享管理​：通过 FileShare 控制多进程/线程的文件访问方式；
      1. 位置定位​：支持 Seek() 方法实现随机访问；
  • 分析：文件流的实现
    • 当创建 FileStream 实例时，操作系统会执行以下操作：
        1. 根据路径和模式参数检查或创建文件；
        1. 分配文件句柄和系统资源；
        1. 建立内部缓冲区；
        1. 初始化文件指针；
    • 数据读取时，内部数据流动为：
      • 应用程序内存 ↔ FileStream 缓冲区 ↔ 系统 IO 缓存 ↔ 物理存储设备；

文件流的读取

概念：文件流的读取

  • 方法一：用不同的字节数组按个读取
    • 概念：
      • 文件流当中包含一个指针，一直指向上一次读取结束之后、最后指向的位置，而当前的参数三用于指定从此位置开始、往后读取多少数据；
    • 参数：
      • 参数一：用于存储读取的字节数组的容器；
      • 参数二：容器中开始的位置；
      • 参数三：读取多少个字节到容器；
      • 返回值：当前流索引前进几个位置；
// 读取第一个整型
byte[] bytes2 = new byte[4];
int index = fs2.Read(bytes2,0,4);
int i = BitConverter.ToInt32(bytes2,0);

// 读取第二个字符串
// 先读取出当前字符串的长度
index = fs2.Read(bytes2,0,4);
int length = BitConverter.ToInt32(bytes2,0);
bytes2 = new byte[length];

// 再根据我们存储的字节数组的长度，来声明一个新的字节数组、用于存储新读取的数据
fs2.Read(bytes2,0,length);
print(Encoding.UTF8.GetString(bytes2));
  • 方法二：一次性读取再挨个读取
FileStream fs3 = File.Open(Application.persistentDataPath + "/Lesson3.tang",FileMode.Open,FileAccess.Read);
byte[] bytes3 = new byte[fs3.Length];
fs3.Read(bytes3,0,(int)fs3.Length);
fs3.Dispose();

// 读取整数
// 这里读取整型、因为整型大小是一样的，因此不需要指定读取的长度，只需要指定读取的位置（index=0）
BitConverter.ToInt32(bytes3,0);
// 读取字符串长度
int length2 = BitConverter.ToInt32(bytes3,4);
// 得到字符串
Encoding.UTF8.GetString(bytes3,8,lenght2);

安全的使用文件流对象

概念：安全的使用文件流对象

  • 概念：using 关键字
    • using 关键字是资源管理的核心机制，尤其在与文件流等非托管资源结合时，能有效避免资源泄漏；
  • 作用：
      1. 导入命名空间：using System.IO; 简化类型访问；
      1. 创建别名：解决命名冲突，例如 using Project = MyCompany.Project;
      1. 作为语句：用于管理实现 IDisposable 接口的对象（如文件流、数据库连接），编译器自动将其转换为 try-finally 块，在作用域结束时调用 Dispose() 释放资源，即使发生异常也能确保安全；

文件夹操作

文件夹基础操作

概念：文件夹操作

  • 概念：
    • 文件夹操作主要通过 System.IO 命名空间下的类实现，主要包括 ​静态工具类（Directory）​​ 和 ​实例化对象类（DirectoryInfo）​​ 两类；
  • 概念：Directory 类
    • Directory 类中的方法主要为静态方法，无需实例化，直接通过类名调用，适合一次性操作；
  • 代码：
// 1. 判断文件夹存在
if(Directory.Exists(Application.dataPath + "/数据持久化4"))
{
	print("存在文件夹");
}else
{
	print("不存在文件夹");
}

// 2. 创建文件夹
DirectoryInfo info = Directory.CreateDirectory(Application.dataPath + "/数据持久化4");

// 3. 删除文件夹
// 参数一：路径
// 参数二：是否删除非空目录，如果为true，将删除整个目录，如果是false，仅当该目录为空时才可删除
Directory.Delete(Application.dataPath + "/数据持久化4",true);

// 4. 查找文件夹和文件
// 4.1 得到指定路径下的所有文件夹名
string[] strs = Directory.GetDirectories(Application.dataPath);
for(int i=0;i<strs.Length;i++)
{
	print(strs[i]);
}

// 4.2 得到指定路径下的所有文件名
strs = Directory.GetFiles(Application.dataPath);
for(int i=0;i<strs.Length;i++)
{
	print(strs[i]);
}

// 5. 移动文件夹
// 如果第二个参数所在的路径已经存在了一个文件夹，那么会报错
// 移动会将文件夹中的所有内容、移动到新的路径
Directory.Move(Application.dataPath + "/数据持久化4",Application.dataPath + "/12312345");

Unity 常用文件路径

三类文件路径

概念：Unity 常用文件路径

  • 概念：Application.persistentDataPath
    • 总结：
      • 设备上的可读写目录，可读可写；
    • 概念：
      • Application.persistentDataPath 是一个指向持久化数据存储目录的路径，用于存储需要在游戏运行之间保留的数据，例如用户存档、配置文件或下载的资源；
      • 该路径在不同平台上的具体位置不同，但通常指向一个可读写的目录；
    • 概念：不同平台下的路径
        1. PC 与 Editor：返回 C:/Users/用户名/AppData/LocalLow/公司名/项目名；
        1. 安卓：/data/data/xxx.xxx.xxx/files
        • 一看也就知道了，这是应用程序的沙盒路径，但还不止，安卓端是可以设置其为沙盒或是 SD 卡，在下图所示的位置可被设置；
    • 特点：
        1. 持久化：数据不会因应用更新而被清除（除非用户手动删除）；
        1. 可读写：开发者可以在此目录下创建、修改或删除文件；
  • 概念：Application.dataPath
    • 总结：
      • Android 中 打包在 APK 内部；
    • 概念：
      • 指向游戏数据文件夹的路径，通常是安装包内的资源目录，在编辑器下是项目的 Assets 文件夹，但在移动端指向 APK 或 IPA 文件，不可直接读写；
      • 在 Editor 里就是 Assets，PC 就是 XXX_DATA 文件夹 (XXX 是程序名称)。但是，在安卓中，返回的是 Apk 的安装路径，是无法读取到 Apk 内的文件的。因此，对于安卓程序而言，不建议使用此方法；
  • 概念：Application.streamingAssetsPath
    • 概念：
      • 指向 StreamingAssets 文件夹的路径，用于存放打包时需原样包含的资源（如音频、视频或配置文件）。该路径在移动端是只读的，但在 PC 端可读写，其特点是存储的是静态资源，其文件在打包时原样复制、不压缩；
      • 其用于存储无需修改的二进制文件；
    • 概念：不同平台下的路径
      • Application.streamingAssetPath 返回流数据的缓存目录，返回路径为相对路径；
        1. 在 Editor 中是 StreamingAssets 文件夹；
        1. PC 则是 XXX_DATA 中的 StreamingAssets 文件夹；
        1. 对于安卓来说：
        • （1）可以是：jar:file:///data/app/com.xxx.xxx.apk/!/assets
        • （2）或是：jar:file:///data/app/com.xxx.xxx.apk!/assets
    • 注意：
        1. 如果想要发布的 Apk 文件中资源文件原封不动的打包进去，只需要把 Publishing Setting 中的 “ Use legacy SDK tools ”勾选，则文件会保留在 Apk 的 Assets 中；
        1. SteamingAsset 文件夹是不是写错了，脚本中是 streamingAssetPath，文件夹是 StreamingAsset，注意大小写；
  • 补充：
    • 一种常见的思路是把默认资源放在在 streamingAssetPath 中，然后在其中读取二进制文件后，移到 persistentPath 下进行后续读写，并且玩家的存档等文件也存放在 persistentPath 下；
总结：
    1. StreamingAssets：存放初始只读资源（如默认配置、首次安装的只读资源）。在 Android 上，必须使用 UnityWebRequest；
    1. PersistentDataPath：存放运行时生成或下载的可修改资源（如用户存档、下载的热更资源）。可以使用 System.IO 或者 UnityWebRequest 进行文件操作；
    1. 使用 UnityWebRequest 来加载路径下的文件时，注意是否需要加上 "file://"前缀；

对类数据进行二进制化并保存

using System.IO;
using UnityEngine;

public class Test : MonoBehaviour
{
	private void Start()
	{
		Student s = new Stduent();
		s.age = 18;
		s.name = "你好";
		s.number = 12314;
		s.sex = false;
		s.Save("测试文件");
		
		Student s2 = Stduent.Load("测试文件");
	}

}

public class Student
{
	public int age;
	public string name;
	public int number;
	public bool sex;
	
	public void Save(string fileName)
	{
		Debug.Log(Application.persistenDataPath);
		
		// 如果不存在指定路径,则创建一个文件夹
		if(!Directory.Exist(Application.persistenDataPath + "/Student"))
		{
			Directory.CreateDirectory(Application.persistenDataPath + "/Student");
		}
		
		// 新建一个指定名字的文件，并返回文件流进行字节的存储
		using(FileStream fs = new FileStream(Application.persistenDataPath + "/Student/" + fileName + "./tang", FileMode.OpenOrCreate, FileAccess.Write))
		{
			// 存储age
			byte[] bytes = BitConverter.GetBytes(age);
			fs.Write(bytes,0,bytes.Length);
			
			// 存储name
			bytes = Encoding.UTF8.GetBytes(name);
			// 存储字符串字符数组的长度
			fs.Write(Bitconverter.GetBytes(bytes.Length),0,4);
			fs.Write(bytes,0,bytes.Length);
			
			// 存储number
			bytes = BitConverter.GetBytes(number);
			fs.Write(bytes,0,bytes.Length);
			
			// 存储sex
			bytes = BitConverter.GetBytes(sex);
			fs.Write(bytes,0,bytes.Length);
		}
		
		fs.Flush();
		
		// Dispose当中会自动调用Close
		fs.Dispose();
	}
	
	public static Student Load(string fileName)
	{
		// 判断文件是否存在
		if(!File.Exists(Application.persistentDataPath + "/Student/" + fileName + ".tang"))
		{
			Debug.LogWarning("没有找到对应的文件");
			return null;
		}
		
		Student s = new Student();
		
		// 加载二进制文件进行赋值
		using(FileStream fs = File.Open(Application.persistenDataPath + "/Student/" + fileName + "./tang", FileMode.OpenOrCreate, FileAccess.Read))
		{
			// 把文件中的字节全部读取出来
			byte[] bytes = new byte[fs.Length];
			fs.Read(bytes,0,bytes.Length);
			fs.Close();
			
			int index = 0;
			// 挨个读取其中的内容
			// 由于是调用ToInt32方法，因此自动读取四字节
			s.age = BitConverter.ToInt32(bytes,0);
			index += 4;
			
			// 先读取字符串字节数组的长度
			int lengthTemp = BitConverter.ToInt32(bytes,index);
			index += 4;
			s.name = Encoding.UTF8.GetString(bytes,index,lengthTemp);
			index +=lengthTemp;
			
			// 读取number
			s.number = BitConverter.ToInt32(bytes,index);
			index += 4;
			
			// 读取sex
			s.sex = Bitconverter.ToBoolean(bytes,index);
			index += 1;
		}
		
		return s;
	}
}
