Lecture 1：XML

什么是 XML

概念：什么是 XML

  • 概念：
    • XML（Extensible Markup Language）是一种可扩展标记语言，用于结构化存储和传输数据；
  • 特点：
    • 自定义标签​：标签名称可自由定义，如 <player><name>Kimi</name></player>；
    • 跨平台性​：纯文本格式，兼容所有操作系统和编程语言；
  • 区别：
  • 举例：
<?xml version="1.0" encoding="UTF-8"?>
<student>
  <student1 name="椎名">
    <age>5</age>
    <sex>man</sex>
  </student1>
  <student2 name="mikan">
    <age>10</age>
    <sex>woman</sex>
  </student2>
  <student3 name="みかん">
    <age>15</age>
    <sex>woman</sex>
  </student3>
</student>

Unity 中使用 XML

创建 XML 文件

概念：创建 XML 文件

  • 概念：XmlDocument
    • 概念：
      • XmlDocument 是 .NET Framework 和 .NET Core 中用于表示和操作 XML 文档的类，属于 System.Xml 命名空间；
      • 它将 XML 数据解析为树状结构​（DOM 模型），允许开发者通过节点（Node）进行增删改查操作；
    • 特点：
        1. 内存中树状模型：加载后，整个 XML 文档以节点树的形式存在内存中，便于遍历和修改；
        1. 支持 XPath 查询​：可通过 SelectSingleNode 或 SelectNodes 快速定位节点；
    • 属性：
      • DocumentElement：获取 XML 的根节点（XmlElement 类型）
      • ChildNodes：返回当前节点的所有子节点集合（XmlNodeList）
      • InnerXml：获取或设置节点及其子节点的 XML 标记字符串；
    • 概念：常用方法
      • Load/LoadXml：从文件、流或字符串加载 XML 数据；
      • Save：将 XML 树保存到文件或流中；
      • CreateElement：创建新的元素节点（需配合 AppendChild 添加到树中）
      • ImportNode：从其他 XML 文档导入节点（跨文档操作）；
  • 概念：XmlNode
    • 表示 XML 文档中的单个节点，是 DOM 树的基类（抽象类）；
  • 概念：XmlElement
    • 继承自 XmlNode，表示 XML 元素节点（如 <tag>content</tag>）；
  • 概念：XmlNodeList
    • 表示有序的节点集合，通常由查询或遍历返回；
  • 总结：
  • 代码：加载与读取 XML - 使用 Load 
XmlDocument xmlDoc = new XmlDocument();

// 方式一：从文件加载
xmlDoc.Load("data.xml"); 
// 方式二：直接解析字符串
xmlDoc.LoadXml("<root><item>test</item></root>");

// 获取根节点下的所有子节点，这里是查找所有匹配的节点，即文档中的所有item节点
XmlNodeList nodes = xmlDoc.DocumentElement.SelectNodes("//item");

foreach (XmlNode node in nodes) 
{
    Console.WriteLine(node.InnerText); // 输出节点内容
}

修改与保存 XML

概念：修改与保存 XML

  • 概念：创建与添加节点
    • 使用两个方法：CreateElement 相当于"制造"了一个节点零件，然后使用 AppendChild 把这个零件"安装"到文档这台机器上指定的位置；
    • CreateElement("item")：这个方法由 XmlDocument 对象调用，它的主要作用是在内存中构造一个符合该文档规范的新元素。你可以理解为，这个新元素已经“知道”它未来要归属于哪个文档，但它本身还没有被纳入文档的树形结构中；
    • AppendChild(newItem)：这个方法用于建立节点间的父子关系。直到你调用 AppendChild（或其他插入方法如 InsertBefore）之前，这个新创建的节点只是独立的存在于内存里，文档的其余部分“看不见”它，它也不会被保存到文件中。调用之后将其加到指定父节点的子节点列表的末尾；
  • 概念：插入节点
    • InsertBefore(newNode, referenceNode) ​：将新节点插入到某个已存在的参考节点之前；
    • PrependChild(newNode)​：将新节点作为第一个子节点插入；
  • 代码：修改和保存 XML - 使用 SelectSingleNode
// 修改节点属性
// SelectSingleNode 获取单个 XmlNode对象
// 找到第一个匹配项后即停止搜索，通常更快
XmlNode item = xmlDoc.SelectSingleNode("//item[@id='1']");
item.Attributes["price"].Value = "99.99";

// 添加新节点
XmlElement newItem = xmlDoc.CreateElement("item");
newItem.InnerText = "New Product";
xmlDoc.DocumentElement.AppendChild(newItem);

// 保存修改
xmlDoc.Save("modified_data.xml");

添加与删除子节点

概念：添加与删除子节点

  • 代码：添加子节点 - 使用 AppendChild()
XmlDocument xmlDoc = new XmlDocument();

// 创建根节点
XmlElement root = xmlDoc.CreateElement("Players");
xmlDoc.AppendChild(root);

// 添加子节点（玩家1）
XmlElement player = xmlDoc.CreateElement("Player");
player.SetAttribute("id", "1");
player.AppendChild(xmlDoc.CreateElement("Name")).InnerText = "Kimi";
player.AppendChild(xmlDoc.CreateElement("Level")).InnerText = "10";
root.AppendChild(player);
  • 代码：从零创建 XML 文件及其子节点
using System.Xml;
using UnityEngine;

public class XMLCreator : MonoBehaviour 
{
    void Start() 
    {
	    // 1. 创建一个XML的文档
        XmlDocument xmlDoc = new XmlDocument();
        
        // 2. 为XML文档添加 XML 声明，包含版本和编码格式
        XmlDeclaration declaration = xmlDoc.CreateXmlDeclaration("1.0", "UTF-8", null);
        xmlDoc.AppendChild(declaration);
        
        // 3. 为当前XML文档创建根节点
        XmlElement root = xmlDoc.CreateElement("Players");
        xmlDoc.AppendChild(root);

        // 4. 为XML文档添加子节点，如下是玩家1的数据
        XmlElement player = xmlDoc.CreateElement("Player");
        // 为XML元素<Player>添加一个名为id的属性，并将其值设置为"1"
        player.SetAttribute("id", "1");
        player.AppendChild(xmlDoc.CreateElement("Name")).InnerText = "Kimi";
        player.AppendChild(xmlDoc.CreateElement("Level")).InnerText = "10";
        
        // 5. 将子节点附加到根节点上
        root.AppendChild(player);

        // 6. 保存文件
        string path = Path.Combine(Application.persistentDataPath, "players.xml");
        xmlDoc.Save(path);
        Debug.Log("XML created at: " + path);
    }
}
  • 补充：两种添加内容的方式
      1. 设置属性 SetAttribute：
      • 用于为 XML 元素设置属性（以键值对形式），属性直接嵌入在元素的开始标签内；
      • 属性通常用于描述元素的元数据，不包含嵌套结构；
      1. 添加子节点 AppendChild + CreateElement：
      • 用于创建并添加子元素，形成嵌套的层次结构；
      • 需要显式创建元素节点（CreateElement），再通过 AppendChild 添加到父节点，例如 <Player><Name>Kimi</Name></Player> 中的 Name 是子元素；
      • 子元素可以包含文本内容、其他元素或属性，适合表示复杂的数据关系；

分析：如何在 Unity 中读取和写入 XML 文件？请比较 XmlDocument 和 XmlTextReader 的优缺点。​

  • XmlDocument 适合小型 XML 文件，支持 DOM 操作；XmlTextReader 适合流式读取大型文件，内存占用低；
  • 使用 XmlDocument 读取节点属性值，如 player.Attributes["name"].Value；
补充：XmlNodeList 和 XmlNode
  • XmlNodeList 是对 XmlNode 封装后的索引器；

获得 XML 中元素

概念：获得 XML 中元素 - 基于标签名获得元素

  • 概念：基于标签名获得元素
      1. 声明 XmlDocument 变量；
      1. 从本地加载 Xml 文件；
      1. 使用 GetElementsByTagName 方法，获得所有的目标节点，并将其存储进链表；
      1. 遍历链表中内容，并使用 item.InnerText 得到节点内容；
  • 代码：
XmlDocument xmlDoc = new XmlDocument();
xmlDoc.Load("data.xml");

// 方式一：获得某个节点
// 先获得根节点
XmlNode root = xmlDoc.SelectSingleNode("PlayerInfo");
XmlNode nodeName = root.SelectSingleNode("Name");
XmlNode nodeList = root.SelectSingleNode("Item");

// 方式二：获得节点链表，再遍历所有节点
XmlNodeList items = xmlDoc.GetElementsByTagName("item"); // 获取所有<item>节点
foreach (XmlNode item in items) 
{
    Debug.Log(item.InnerText); // 输出节点内容
}

概念：获得 XML 中元素 - 基于 XPath 定位元素

  • 概念：什么是 XPath
    • XPath（XML Path Language）是一种用于在 XML 和 HTML 文档中定位和选择节点的查询语言，它通过路径表达式（类似文件系统的路径）来导航文档的树状结构，支持对元素、属性、文本等节点的精确或模糊匹配，广泛应用于数据抓取、自动化测试和 XML 文档处理等领域；
  • 举例：
      1. 获得子节点：
      • /bookstore/book：选择根节点下 bookstore 的直接子节点 book；
      1. 提取子节点值：
      • //a/text()：获取所有 <a> 标签的文本内容
  • 总结：XPath 语法
    • 方括号：用于附加条件
      • //div[1]：选择第一个 div
      • //book[last()]：选择最后一本书
      • //input[not(@disabled)]：选择未禁用的输入框
  • 概念：基于 XPath 定位元素
    • 使用 XPath 表达式精准定位节点（支持属性过滤、层级查询），使用 SelectSingleNode / SelectNodes 方法；
  • 代码：
XmlNode player = xmlDoc.SelectSingleNode("/Players/Player[@id='101']"); // 获取id=101的玩家
XmlNodeList enemies = xmlDoc.SelectNodes("//Enemy[Health>50]"); // 获取血量>50的所有敌人

XML 与序列化

IXmlSerializable 接口

概念：IXmlSerializable 接口

  • 引入：为什么要将 XML 序列化
    • 将对象的状态转换为 XML 格式的数据，以便存储或传输，反序列化则是将 XML 数据还原为对象；
    • 需要通过序列化和反序列化，使得 XML 数据能够跨平台数据交换（如 Web 服务）、配置文件持久化、存档系统等；
  • 概念：IXmlSerializable 接口
    • 默认序列化无法序列化私有字段或特殊类型（如 Dictionary 类），因此需要自定义 XML 结构（如分块处理大数据）；
    • 因此可以实现 IXmlSerializable 接口，进而完全控制序列化的相关行为；
  • 方法：
    • WriteXml：自定义如何将对象写入 XML（通过 XmlWriter）；
    • ReadXml：自定义如何从 XML 读取对象（通过 XmlReader）；
    • GetSchema：返回 XML 模式（通常返回 null）；

ReadXml 与 WriteXml 方法

概念：ReadXml 方法

  • 概念：ReadXml
    • ReadXml 是 IXmlSerializable 接口的关键方法，用于从 XML 数据流（XmlReader）中反序列化对象，必须与 WriteXml 方法配对实现，确保序列化和反序列化的逻辑一致；
    • 当 XmlSerializer.Deserialize 执行时，框架会自动调用此方法；
    • 调用时，XmlReader 已定位到对象的起始标签​（如 <Test>），需手动处理整个元素（包括子节点和属性）；
  • 概念：XmlReader
    • 概念：
      • XmlReader 是 .NET 中一个抽象基类，提供非缓存、前向只读的 XML 数据访问方式；
    • 方法：
      • Read()：移动到下一个节点，返回 bool 表示是否成功；
      • NodeType：获取当前节点类型（如 Element、Text、EndElement）；
      • Name / Value：节点名称和值（根据 NodeType 决定是否有效）；
  • 概念：WriteXml 方法
    • 将对象转换为 XML 格式，支持写入文件、流或 XmlWriter；
  • 概念：三种 WriteXml 的方法
      1. 写属性：
      • 为当前 XML 元素添加属性（键值对形式）；
        • writer.WriteAttributeString("属性名", "属性值");
      • 比如：
        • writer.WriteAttributeString ("testInt", testInt.ToString ()); // 添加整型属性
        • writer.WriteAttributeString ("testName", testName); // 添加字符串属性
        • 输出结果为 <Element testInt="123" testName="Kimi" />
      1. 写节点：
      • 直接写入一个完整的元素节点（含标签和文本内容）
        • writer.WriteElementString("节点名", "节点值");
      • 比如：
        • writer.WriteElementString("testInt", testInt.ToString()); // 写入整型节点
        • writer.WriteElementString("testName", "TonyChang"); // 写入字符串节点
        • 输出效果为 <testInt>123</testInt> <testName>TonyChang</testName>
      1. 写包裹的节点：
      • 通过 XmlSerializer 序列化复杂对象，嵌套在自定义标签内；
      • 过程：
          1. 创建序列化器：new XmlSerializer(typeof(类型))
          1. 开始包裹标签：writer.WriteStartElement("标签名")
          1. 序列化写入内容：serializer.Serialize(writer, 对象)
          1. 结束包裹标签：writer.WriteEndElement()
      • 示例：
  • 代码：
using System;
using System.Xml;
using System.Xml.Schema;
using System.Xml.Serialization;

[Serializable]
public class Person : IXmlSerializable
{
    public string Name { get; set; }
    public int Age { get; set; }
    public DateTime BirthDate { get; set; }
    public List<string> Hobbies { get; set; } = new List<string>();

    // IXmlSerializable 接口实现
    public XmlSchema GetSchema() => null; // 默认返回 null

    /// <summary>
    /// 自定义序列化逻辑：将对象写入 XML
    /// </summary>
    public void WriteXml(XmlWriter writer)
    {
        // 1. 写入属性（可选）
        writer.WriteAttributeString("version", "1.0");

        // 2. 写入元素
        writer.WriteElementString("Name", Name);
        writer.WriteElementString("Age", Age.ToString());
        writer.WriteElementString("BirthDate", BirthDate.ToString("yyyy-MM-dd"));

        // 3. 处理集合类型
        writer.WriteStartElement("Hobbies");
        foreach (var hobby in Hobbies)
        {
            writer.WriteElementString("Hobby", hobby);
        }
        writer.WriteEndElement(); // 关闭 Hobbies 标签
    }

    /// <summary>
    /// 自定义反序列化逻辑：从 XML 重建对象
    /// </summary>
    public void ReadXml(XmlReader reader)
    {
        // 1. 读取属性（如果有）
        if (reader.GetAttribute("version") != null)
        {
            // 可在此处理版本兼容性逻辑
        }

        // 2. 移动到第一个元素（Name）
        reader.ReadStartElement(); // 跳过外层开始标签（如 <Person>）

        // 3. 逐个读取元素
        Name = reader.ReadElementContentAsString("Name", "");
        Age = reader.ReadElementContentAsInt("Age", "");
        BirthDate = DateTime.Parse(reader.ReadElementContentAsString("BirthDate", ""));

        // 4. 处理集合类型
        if (reader.IsStartElement("Hobbies"))
        {
            reader.ReadStartElement("Hobbies");
            while (reader.IsStartElement("Hobby"))
            {
                Hobbies.Add(reader.ReadElementContentAsString("Hobby", ""));
            }
            
            reader.ReadEndElement(); // 跳过 Hobbies 的结束标签
        }

        // 5. 确保读取完整（跳过外层结束标签）
        reader.ReadEndElement(); // 如 </Person>
    }
}
