Lecture 2：JSON

JSON 概述

什么是 JSON

概念：JSON

  • 概念：
    • JSON（JavaScript Object Notation）是一种轻量级数据交换格式，采用键值对结构，支持嵌套对象和数组，在 Unity 中主要用于配置存储、网络通信和序列化；
  • 概念：JSON 在 Unity 中常用解析库
      1. JsonUtility：Unity 内置 JSON 解析库；
      • 性能高，只支持公有资源，需要显示存储；
      1. LitJson：
      • 性能较低，支持属性、私有字段、复杂嵌套，可以跨平台进行复杂数据交换；
  • 注意：JsonUtility 不支持直接解析数组
    • 当 Unity 中解析 JSON 数组时遇到 ArgumentException: JSON must represent an object type 错误，说明 JsonUtility 不支持直接解析数组；
    • 因此需将其包装成对象，如 { "data": [...] }；
  • 示例：
// JsonUtility 示例
[Serializable]
public class PlayerData 
{
    public string name;
    public int score;
}

void Start() {
    // 反序列化
    string json = "{\"name\":\"Player1\",\"score\":100}";
    PlayerData data = JsonUtility.FromJson<PlayerData>(json);
    
    // 序列化
    string newJson = JsonUtility.ToJson(data);
    Debug.Log(newJson); // 输出: {"name":"Player1","score":100}
}

JSON 配置规则

概念：JSON 配置规则

  • 概念：
    • 采用键值对格式："key": value，其中键必须双引号；
    • 变量类型：字符串（"text"）、数字、布尔值、数组（[]）、对象（{}）、null；
  • 概念：JSON 语法规则
  • 举例：C# 类举例
using System.Collections.Generic;

// MrTang类包含基础类型、集合类型和嵌套类引用
public class MrTang 
{
    public string name;        // 字符串类型
    public int age;           // 整型
    public bool sex;          // 布尔型
    public List<int> ids;     // 整型列表
    public List<Person> students; // Person对象列表（2个引用中的第1个）
    public Home home;         // Home对象引用（1个引用）
    public Person son;        // Person对象引用（2个引用中的第2个）
}

// Person类被MrTang类通过students和son字段引用（共2处）
public class Person 
{
    public string name;
    public int age;
    public bool sex;
}

// Home类被MrTang类通过home字段引用（共1处）
public class Home 
{
    public string address;
    public string street;
}
  • 举例：对应 JSON 代码
// 大括号包裹的就表示一个对象
{
	// 逗号就是分割成员变量的间隔符
	"name":"小王",
	// JSON中的键一定要用双引号包裹，后面的值是否用双引号包裹取决于类型
	"age":18,
	"sex":true,
	
	// JSON中的浮点数，不需要加 f
	"testF":1.4,
	
	// 中括号代表数组
	"ids":[1,2,3,4],
	
	// 这里的Students是一个存储了Person类的数组
	"students":[
		{"name":"小红","age":10,"sex":false},
		{"name":"小蓝","age":11,"sex":true}],
	
	// 这里的Home是一个类
	"home":{"address":"成都","street":"春熙路"},
	
	// 表示空，里面没数据
	// 最后一个成员变量，后背不需要加逗号
	"son":null
}

C# 读取存储 JSON 文件

Jsonutility 序列化

概念：Jsonutility

  • 概念：
    • Jsonutlity 是 Unity 自带的用于解析 Json 的公共类，它可以
      1. 将内存中对象序列化为 Json 格式的字符串；
      1. 将 Json 字符串反序列化为类对象；
  • 概念：从文件中存读字符串
// 1. 存储字符串到指定路径文件中
// 将"存储的JSON文件名称"这个字符串内容存入文件中
// 注意：这种方式来进行存储，必须要保证当前文件路径存在
// 第一个参数 必须是存在的文件路径"如果没有对应文件夹 会报错
File.WriteAllText(Application.persistentDataPath + "Test.json","存储的JSON文件名称");

// 2. 在指定路径文件中读取字符串
File.ReadAllText(Application.persistentDataPath + "/Test.json");
  • 概念：使用 Jsonutlity 进行序列化
using System.Collections.Generic;

// Student类定义
public class Student
{
    public int age;     // 整型年龄字段
    public string name; // 字符串名字字段
}

// MrTang类定义
public class MrTang
{
    // 基础类型字段
    public string name;     // 字符串
    public int age;        // 整型
    public bool sex;       // 布尔型
    public float testF;    // 单精度浮点
    public double testD;   // 双精度浮点

    // 集合类型字段
    public int[] ids;                     // 整型数组
    public List<int> ids2;                // 整型泛型列表
    public Dictionary<int, string> dic;   // 键值对字典（int→string）
    public Dictionary<string, string> dic2; // 键值对字典（string→string）

    // 嵌套类字段
    public Student s1;           // Student类实例
    public List<Student> s2s;     // Student泛型列表

    // 访问限制字段
    private int privateI;     // 私有整型
    protected int protectedI; // 受保护整型
}

// 使用
MrTang t = new MrTang();
t.name = "小王";
t.age = 18;
t.sex = true;
t.testF = 1.4f;
t.testD = 1.4;

t.ids = new int[]{1,2,3,4};
t.ids2 = new List<int>{1,2,3,4};
t.dic = new Dictionary<int,string>(){{1,"123"},{2,"234"}};
t.dic2 = new Dictionary<string,string>(){{"1","123"},{"2","1231431"}};

t.s1 = new Student(1,"小红");
t.s2s = new List<Student>(){new Stduet(2,"小明"),new Student(3,"小强")};

// ToJson()方法的返回值是一个string类型，它会将对象t序列化为JSON格式的字符串；
// 因此必须用字符串变量（如`jsonStr`）来接收这个结果
// ToJson()会生成类似 {"name":"value","age":25} 的字符串
string jsonStr = JsonUtility.ToJson(t);
File.WriteAllText(Application.persistentDataPath + "/MrTang.json",jsonStr);
  • 注意：
      1. float 序列化时看起来会有一些误差；
      1. 自定义类需要加上序列化特性；
      1. 想要序列化私有变量需要加上特性 [SerializeField]；
      1. JsonUtility 不支持字典；
      1. JsonUtility 存储 null 对象不会是 null，而是默认值的数据；

Jsonutility 反序列化

概念：Jsonutility 反序列化

  • 概念：
jsonStr = File.ReadAllText(Application.persistentDataPath + "/MrTang.json");

// 方法一
MrTang t2 = JsonUtility.FromJson(jsonStr,typeof(MrTang)) as MrTang;

// 方法二：推荐使用的泛型方法
MrTang t3 = JsonUtility.FromJson<MrTang>(jsonStr);
  • 注意：
      1. 如果 JSON 中读取的数据有缺失，读取到内存中类对象时不会报错，而是会用默认值代替；
      1. JSON 无法直接读取数据集合（比如想要读取一个数组，数组的类型是某个自定义的类），此时需要使用一个数据结构，比如 List 将其包裹；
      1. 文本的编码格式必须得是 UTF-8；

LitJson 序列化

概念：LitJson 序列化

  • 概念：LitJson 是什么
    • 它是一个第三方库，用于处理 Json 的序列化和反序列化；
    • LitJson 是 C# 编写的，体积小、速度快、易于使用；
    • 它可以很容易的嵌入到我们的代码中，只需要将 LitJson 代码拷贝到工程中即可；
  • 概念：获取 LitJson
      1. 前往 LitJson 官网；
      1. 通过官网前往 GitHub 获取最新版本代码；
      1. 将代码拷贝到 Unity 工程即可开始使用 LitJson；
  • 图示：
  • 概念：使用 LitJSON 进行序列化

// Student类定义
public class Student

      1. LitJson 中存储的类不需要加上 Unity 中的序列化属性，既可直接转成 JSON 格式；
      1. 私有的部分不会被存储；
      1. 支持字典类型，但是建议字典的 Key 为 String 类型；
      1. 需要引用 LitJson 的命名空间；
      1. LitJson 可以准确的存储 null 类型；

LitJson 反序列化

概念：LitJson 反序列化

  • 概念：
    • LitJson 是 Unity3D 开发中广泛使用的一个轻量级 JSON 处理库，它提供了简单高效的 JSON 序列化和反序列化功能；
  • 概念：JsonMapper.ToObject 方法
    • LitJson 提供了几种主要的反序列化方法，最常用的是 JsonMapper.ToObject 方法；
  • 代码：
    • 当不确定 JSON 结构或需要动态处理数据时，可以使用 JsonMapper.ToObject 不带泛型参数的形式，返回一个 JsonData 对象；
    • JsonData 是 LitJson 中的核心数据类型，它表示 JSON 数据结构中的任意元素；
    • 所有通过 LitJson 解析的 JSON 数据都会被转换为 JsonData 对象，它可以表示 JSON 中的对象、数组以及各种基本数据类型； 
jsonStr = File.ReadAllText(Application.persistentDataPath + "/MrTang.json");

// 重载一：返回值是一个JsonData的对象
JsonData data = JsonMapper.ToObject(jsonStr);
print(data["name"]);
print(data["age"]);

// 重载二：返回值直接指定类型
MrTang t2 = JsonMapper.ToObject<MrTang>(jsonStr);
  • 注意：
      1. 类结构必须需要无参构造函数，否则在反序列化时报错；
      1. 字典虽然支持，但是键在使用为数值时会有问题，需要使用字符串类型；
      1. LitJson 可以直接读取数据集合；
  • 示例：LitJson 可以直接读取数据集合
jsonStr = File.ReadAllText(Application.streamingAssetsPath + "/RoleInfo.json");
RoleInfo[] arr = JsonMapper.ToObject<RoleInfo[]>(jsonStr);
List<RoleInfo> list = JsonMapper.ToObject<List<RoleInfo>>(jsonStr);
  • 总结：
      1. LitJson 提供的序列化、反序列化方法：JsonMapper.ToJson 和 ToObject；
      1. LitJson 无需加特性；
      1. LitJson 不支持私有变量；
      1. LitJson 支持字典序列化反序列化；
      1. LitJson 可以直接将数据反序列化为数据集合；
      1. LitJson 反序列化时自定义类型需要无参构造；
      1. Json 文档编码格式必须是 UTF-8；

JsonUtility 和 LitJson 对比

分析：JsonUtility 和 LitJson 共同点

    1. 他们都是用于 json 的序列化、反序列化；
    1. Json 文档编码格式必须是 UTF-8；
    1. 都是通过静态类进行方法调用；

分析：不同点

    1. Jsonutlity 是 Unity 自带，LitJson 是第三方，需要引用命名空间；
    1. Jsonutlity 使用时自定义类需要加特性，LitJson 不需要；
    1. Jsonutlity 支持私有变量 (加特性)，LitJson 不支持；
    1. Jsonutlity 不支持字典，LitJson 支持 (但是键只能是字符串)
    1. Jsonutlity 不能直接将数据反序列化为数据集合 (数组字典)，LitJson 可以；
    1. JsonUtlity 对自定义类不要求有无参构造，LitJson 需要；
    1. JsonUtlity 存储空对象时会存储默认值而不是 null，LitJson 会存 null；
  • 个人建议：建议使用 LitJson；
  • 原因: LitJson 不用加特性，支持字典，支持直接反序列化为数据集合，存储 null 更准确；
