堆栈跟踪工具

堆栈跟踪工具的作用

概念：堆栈跟踪使用工具的作用

  • 概念：
    • 目前我们使用 Android Logcat 连接真机进行测试；
    • 打印、警告、报错等等信息都会直接在主窗口中打印出来，让我们进行分析；
    • 相当于它帮助我们进行了自动的堆栈跟踪解析；
  • 概念：自动堆栈跟踪解析
    • 什么是自动堆栈跟踪解析呢? 就是我们可以在打印窗口看到具体是哪个脚本的哪行代码出现了打印、警告、报错等等信息；
    • 而 Tools (工具) 中的 stacktrace utility (堆栈跟踪实用工具) 的作用是允许我们复制粘贴自定义日志并解析堆栈跟踪；
  • 作用：
    • 主要用于没有真机连接设备调试时，或者之后正式发布版本后，我们可以获取应用程序日志信息；
    • 将这些日志信息复制到该工具中用于翻译 (一般项目会在崩溃后，在本地存储一份崩溃日志，我们可以获取该日志)；
    • 因为这种情况下的日志信息中，不会明确指明是哪个脚本哪行代码，往往是一堆看不懂的符号：
      • E CRASH : #01 pc 01c65330 /data/app/com.CrashComp.Crash-J2Z_LOXSsSAZPkt9lab2rQ==/lib/arm/lib/university.so (DiagnosticsUtils_Bindings::ForceCrash(DiagnosticsUtils_Bindings::ForcedCrashCategory,ScriptingExceptioNPtr*)+48)
    • 这里面都是一堆特殊符号，并没有明确告诉你哪个脚本哪行代码；
    • 通过翻译后堆栈跟踪实用工具翻译后，我们就够获取到哪个脚本哪行代码打印了输出了该信息；
    • 总而言之。该工具就是用来翻译发布后安卓应用程序的日志信息的；

内存窗口工具

内存窗口工具的作用

概念：内存窗口工具的作用

  • 概念：
    • 在学习之前，首先了解 RAM 对于我们的意义。如果应用程序出现闪退情况，往往都是因为应用程序运行时内存的使用超出了设备允许的最大内存量，而被手机系统强行关闭；
  • 注意：
    • 不同品牌、型号的手机配置都是不同的，手机运行的程序数量也是不同的，所以要根据实际情况进行测试；
    • 获取目标设备的内存极限值可以选取市面上的主流机型来进行测试；
  • 概念：工具的作用
    • 内存窗口工具的作用是用于跟踪应用程序 RAM (运行时内存) 的分配情况的，我们可以使用它来分析应用程序可能存在的内存问题，可以使用它定期自动的或者手动的捕获内存快照。这里的内存快照就是捕获的这一瞬间应用程序的运行时内存使用情况；
    • 根据内存快照我们可以分析内存问题，比如：
      1. 切换场景前和切换场景后的内存快照比较，判断是否存在内存泄露；
      1. 通过分析闪退前的内存快照，来分析设备占用多少内存时造成了闪退，确定内存瓶颈；

内存窗口工具的使用

概念：内存窗口工具的使用

  • 开启内存窗口工具，默认是不启用。其中 Auto Capture 代表自动捕获，Unity 会定期捕获应用程序的内存快照；
  • 注意开启自动捕获，可能会影响性能表现，可能会造成卡顿，如果严重影响测试，建议使用手动捕获；
  • 使用 Manual Capture 表示手动捕获，观看不同帧下的内存占用；

内存占用解析

概念：内存详细信息面板 - 显示每种内存类型分配的内存量

  • Resident Set Size：常驻集大小
    • 应用程序在运行时内存中分配的内存总量；
    • 一般情况下不使用这个选项，因为信息太少；
  • Proportional Set Size：比例集大小
    • 应用程序主动使用的运行时内存总量，信息比较详细；
    • 包含当前时刻的 Native 堆与 Java 堆占的内存，代码、栈、图像等占的内存；
  • Heap Alloc：堆分配
    • 应用程序使用 java 分配器和本机堆分配的内存总量，当检查内存泄露时，通过它可以进行很好的分析；
  • Heap Size：堆大小
    • 应用程序保留的总内存，总是大于堆分配；

概念：内存图表 - 包含一段时间内连接的应用程序分配的内存图表

ADB 调试

ADB 是什么

概念：ADB 是什么

  • 概念：
    • ADB 是 Android Debug Brige (安卓调试桥) 的简称，它是我们调试 Android 设备的一套指令集，它可以让我们通过指令来进行一些操作，来获取日志信息；
    • 比如关机重启、安装启动卸载应用程序、删除移动复制文件、查看日志信息；
    • 比如，对于资源（如模型、纹理、音效）的修改，可以将其打包成 AB 包。在构建 APK 时，将这些 AB 包放在 StreamingAssets 目录下，它们会被直接复制到 APK 中而不增加编译时间。然后在调试时，只需将新的 AB 包用 adb push 命令推送到设备的持久数据路径（如 /sdcard/），游戏运行时加载新资源即可，避免了重复构建完整的 APK；
  • 总结：
    • 我们最常使用的就是查看日志信息的功能，之前学习的 Android Logcat 工具其实就是利用了 ADB 来获取的信息；
    • 由于 Android Logcat 工具只能在 Unity 2019.4 及其以上版本才能使用，所以对于老版本的 Unity 来说，我们一般使用 ADB 来帮助我们进行调试；

如何使用 ADB

概念：如何使用 ADB

  • 概念：
    • 在 Android SDK 的路径中寻找 platform-tools 文件夹，在其中可以找到 adb.exe 文件；
    • 我们需要配合命令调试符窗口 (Windows)使用 adb 指令集；
  • 方法：直接在 Android Logcat 中使用
    • 直接在 Android Logcat 窗囗中使用 Tools 中的 open Terminal (打开终端)，window 系统会自动打开命令调试符窗口，MacOS 系统会自动打开终端窗口；
    • 会直接来到 SDK 文件夹中的 platform-tools (平台工具) 文件夹，在后面直接输入指令即可；
    • 比如输入 adb logcat 指令后会打印日志信息：
  • 总结：
    • Android Logcat 其实就是对 ADB 的封装，利用了 ADB 提供给我们用于调试的指令集做了很多功能；
    • 对于 Unity 2019.4 及其以上版本，我们使用 Android Logcat 即可；
    • 对于低版本，我们需要直接使用 ADB 指令集来进行调试，我们经常会使用到的功能就是日志信息获取，可以让我们获取 Unity 中的打印信息，报错信息等帮助我们排查 bug；

常用 ADB 指令

概念：常用 ADB 指令

# ==================================================
# ADB（Android Debug Bridge）常用指令集
# 适用于 Android 开发和 Unity 调试
# ==================================================

# 常用指令部分

# 1. 显示完整的设备日志信息
# 实时输出 Android 设备的系统日志，包含所有应用的日志信息
adb logcat

# 2. 过滤显示 Unity 相关的日志信息
# -s 参数表示只显示指定标签的日志，这里过滤出 Unity 引擎的日志
adb logcat -s Unity

# ==================================================
# 扩展指令（做了解的指令）
# ==================================================

# 1. 查看 ADB 帮助信息
# 显示所有可用的 ADB 命令和参数说明
adb help

# 2. 查看 ADB 版本信息
# 显示当前安装的 ADB 工具版本号
adb version

# 3. 安装 APK 应用程序
# 将指定的 APK 文件安装到连接的设备上
adb install [apk路径]

# 4. 覆盖安装应用程序
# -r 参数表示重新安装已存在的应用，保留数据
adb install -r [apk路径]

# 5. 卸载应用程序
# 根据包名完全卸载应用，同时删除应用数据
adb uninstall [应用程序包名]

# 6. 保留数据卸载应用程序
# -k 参数表示卸载应用但保留数据和缓存目录
adb uninstall -k [应用程序包名]

# 7. 查看已连接的设备列表
# 显示当前通过 USB 或网络连接到电脑的 Android 设备
adb devices

# 8. 将文件从电脑推送到手机
# 将电脑上的文件或目录复制到手机的指定路径
adb push [电脑上文件路径] [需要复制到的手机路径]

# 9. 将文件从手机拉取到电脑
# 将手机上的文件或目录复制到电脑的指定路径
adb pull [手机上文件路径] [需要复制到的电脑路径]

# 10. 查看设备上安装的所有应用包名
# 通过 shell 执行 pm（package manager）命令列出所有已安装应用
adb shell pm list packages

# ==================================================
# 使用示例（替换括号内的内容为实际值）：
# adb install C:\app\demo.apk
# adb uninstall com.company.game
# adb push ./config.json /sdcard/
# adb pull /sdcard/log.txt ./
# ==================================================
