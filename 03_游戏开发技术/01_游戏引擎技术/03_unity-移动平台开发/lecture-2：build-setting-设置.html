纹理压缩格式

概念：纹理压缩格式

  • 概念：Texture Compression - 纹理压缩
    • 设置发布后的纹理压缩格式。默认使用 Use PlayerSetting 中设置的压缩格式，纹理将遵循在 ​ Edit -> Project Settings -> Player -> Other Settings ​ 中为对应平台（如 Android）设置的默认纹理压缩格式；
    • ETC：Ericsson Texture Compression，ETC1 ​ 是初始版本，一个主要限制是不支持透明通道（Alpha 通道）​。对于不透明的纹理（如漫反射贴图、法线贴图），它是非常安全的选择。主要用于需要最大限度兼容老旧 Android 设备的不透明纹理，对于带透明度的纹理，需要将颜色和 Alpha 通道拆分成两张 ETC1 纹理，这会增加工作量和不便；
    • ETC2：ETC1 的升级版，是 ​ OpenGL ES 3.0 ​ 的标准功能，支持透明通道（Alpha 通道）​，并且压缩质量比 ETC1 更好，​ Android 开发的首选格式，可以统一设置为 ETC2，并为不支持 ETC2 的老设备在 ​​“ETC2 fallback”​​ 选项中设置一个后备格式（如 ETC1），这样就能在保证新设备高质量的同时兼容老设备；
    • ASTC：提供从高压缩率（低质量）到低压缩率（高质量）的多种块大小选项（如 4x4, 6x6, 8x8 等），开发者可以根据纹理内容在质量和大小之间进行精细权衡。支持透明通道和各种颜色格式，质量通常优于 ETC2，需要设备硬件支持（并非所有 Android 设备都支持，但近年来主流中高端芯片均已支持）；
    • DXT：S3 Texture Compression，也称为 S3TC，是 PC 和游戏主机（如 Xbox, PlayStation）上的传统标准格式，​主要用于 PC 和主机平台的游戏开发。在移动平台（Android/iOS）上应避免使用；
    • PVRTC：PowerVR Texture Compression，是苹果设备（iOS, tvOS）芯片（历代 A 系列芯片）的专属原生格式，在 iPhone 和 iPad 上具有最佳的兼容性和性能；
  • 概念：ETC2 fall back
    • 32-bit：32 位
      • 以全分辨率存储 RGBA 纹理，使用 32 位存储每个像素，质量最高内存占用最多；
    • 16-bit：16 位
      • 以全分辨率存储 RGBA 纹理，使用 16 位存储每个像素，质量较低，纹理可能会丢失有价值的颜色信息，内存占用相对较少；
    • 32-bit, half resolution：32 位
      • 半分辨率以半分辨率存储 RGBA 纹理，使用 32 位存储每个像素，由于是半分辨率，纹理会变得模糊，内存占用较小；

  • 概念：
    • 符号链接源；
    • 当勾选 Exprot Project 时，才能勾选此选项。决定是否在 Unity 和导出的 Gradle 项目之间共享 Java 和 Kotlin 源文件；

调试相关

概念：调试相关

  • 概念：Create Symbol.zip
    • Unity 会生成一个包含本机 Unity 库的符号文件的包，它包合一个表，将活动内存地址转换为我们可以使用的信息，比如方法名，我们可以利用符号包来检测调试应用程序的信息；
  • 概念：Run Device
    • 连接的设备，通过直接运行到设备上，进行更方便的调试；
  • 概念：Build to Device
    • 构建到设备，不创建完整的构建，而是将发生修改的单个文件直接部署到设备上，相对来说更加快速、方便调试
  • 概念：Development Build
    • 开发模式构建，是否包含脚本调试符号和性能分析器到你的项目中，如果你想要调试应用程序，请选择该选项；
  • 概念：Autoconnect Profiler
    • 自动连接分析器，是否自动将分析器连接到生成的应用程序；
  • 概念：Deep Profiling
    • 深度剖析；
    • 是否启用深度剖析在分析器中，可以让分析器能够检测应用程序中每个函数调用，从而返回更加详细的分析数据；
    • 注意：该选项会降低脚本的执行速度；
  • 概念：Script Debugging
    • 是否允许调试应用程序的脚本文件；
    • Wait For Managed Debugger：是否等待托管调试器程序在运行脚本代码之前是否等待调试器连接；

其他设置

概念：其他设置

  • 概念：压缩方法
    • 控制 ​ Unity 在构建最终应用程序包（如 APK/IPA）时​ 如何压缩项目内的资源数据；
    • 此设置影响最终用户的下载包体大小和应用程序的运行时内存占用。选择压缩率更高的方法（如 LZ4HC）可以减小包体，但可能会略微增加运行时解压的开销；
  • 概念：最大纹理大小 - Max Texture Size
    • 这是一个开发期设置。​减小此值可以显著加快在编辑器内导入纹理资源以及切换不同目标平台（如从 Android 切换到 iOS）的速度，因为它减少了需要处理的像素总量；
    • 通常在为移动平台开发时，不建议设置为超过 2048；
