Lecture 5：AssetDatabase 类

前置知识

GUID 系统

概念：GUID 系统

  • 概念：
    • GUID (全局唯一标识符) 是 Unity 资源管理的核心机制，每个资源都有唯一的 GUID，存储在 .meta 文件中；
  • 概念：GUID 转路径
    • string path = AssetDatabase.GUIDToAssetPath("a5a31e6d725a1414e8a07b7d256d3a1a");
    • 返回结果如："Assets/Textures/icon.png"
  • 概念：路径转 GUID
    • string guid=AssetDatabase.AssetPathToGUID("Assets/Prefabs/Character.prefab");
  • 注意：
      1. 复制资源时连同 meta 文件一起拷贝可保留 GUID 和导入设置，使用 Ctrl+D 复制资源会生成新 GUID；
      1. 删除 meta 文件会导致 GUID 重新生成，破坏现有引用；
      1. 脚本中尽量使用 AssetDatabase API 而非文件 I/O；

AssetImporter 资源导入设置

概念：AssetImporter 资源导入设置

  • 概念：
    • AssetImporter 是 Unity 中所有资源导入器的基类，当资源被导入到 Unity 项目时，Unity 会根据资源类型自动创建对应的 AssetImporter 实例；
    • 它类似于一个"资源解码器"，能够解析不同类型文件的数据，并提供访问和修改的功能；
  • 概念：Unity 中资源导入的流程
      1. 资源被拷贝到 Assets 目录；
      1. AssetDatabase 检测到变更；
      1. 调用对应类型的 AssetImporter 处理资源；
      1. 进行格式转换与优化；
      1. 生成 .meta 和 Library 缓存文件；
  • 概念：不同类型的 AssetImporter
  • 概念：AssetImporter 核心方法
      1. 通过资源的路径获得 AssetImporter 实例：
      • AssetImporter importer = AssetImporter.GetAtPath("Assets/Textures/character.png");
      1. 子资源重映射：
      • AddRemap/RemoveRemap​
  • 注意：
    • 使用 AssetImporter 修改完资源后，需要调用成员方法 SaveAndReimport() 将资源的修改保存；
  • 示例：批量处理选中图片的压缩格式
[MenuItem("Assets/Batch Set Compression")]
static void BatchSetCompression()
{
    foreach(Object obj in Selection.objects)
    {
	    // 得到 Unity 中的 Object 对象后，可以使用 AssetDatabase 来得到其路径
        string path = AssetDatabase.GetAssetPath(obj);
        
        // 使用后缀来判断当前的文件类型
        if(path.EndsWith(".png"))
        {
	        // 通过路径获得文本导入器
            TextureImporter importer = (TextureImporter)AssetImporter.GetAtPath(path);
            
            // 设置贴图的压缩格式
            importer.textureCompression = TextureImporterCompression.Compressed;
            
            // 保存修改
            importer.SaveAndReimport();
        }
    }
}
  • 分析：
    • Selection -Object对象-> AsesetDatabase -文件Path-> AssetImporter -Importer实例-> 进行导入设置；

AssetDatabase

基础概念

概念：AssetDatabase

  • 概念：
    • AssetDatabase 是 Unity 编辑器环境下最核心的资源管理 API 之一，是 Unity 编辑器内置的资源数据库管理系统，它作为 Unity 资源管道的核心组成部分，负责追踪和管理项目中所有资源文件及其元数据；
    • 与直接使用文件系统操作不同，AssetDatabase 提供了一套更高层次的抽象接口，确保资源操作的安全性和一致性。Unity 会为每个资源维护元数据（存储在 .meta 文件中），使用 AssetDatabase 能确保这些元数据被正确更新；；
  • 注意：
    • AssetDatabase 仅限编辑器脚本使用，运行时需改用 Resources 或 AssetBundle；
  • 分析：使用 AssetDatabase 进行资源操作的优点
      1. 维护资源文件的 GUID（全局唯一标识符）系统，确保资源引用不因路径改变而失效；
      1. 跟踪资源之间的依赖关系；
      1. 管理资源的导入和刷新流程；
      1. 提供事务性操作，确保复杂操作的安全性；
  • 概念：AssetDatabase 调用场合
    • 在 Unity 编辑器环境下，我们通常不需要将使用 AssetDatabase 的代码放在诸如 void Start() 这样的 MonoBehaviour 生命周期方法中，因为这些方法主要是在运行游戏时调用的；
    • 对于使用 AssetDatabase 的代码，更常见的做法是将其放在自定义的 Editor 脚本中的静态方法中，或者在执行特定编辑器操作，比如菜单命令、Inspector 窗口中的按钮点击时调用；
    • 这样可以确保这些代码只在 Unity 编辑器环境下运行，而不会在构建和运行游戏时被执行；
  • 概念：AssetDatabase.Refresh()
    • 强制刷新 Unity 资源数据库，使编辑器立即检测并应用项目文件夹中的变更（如新增、删除、修改资源文件）
    • Unity 通常会自动监测文件系统变化，但在脚本操作后需手动调用此方法以确保同步，其底层是重新扫描 Assets 目录下的文件结构，更新元数据，触发资源重新导入（Reimport），根据文件类型调用对应的导入器；

AssetDatabase 创建或删除文件

概念：AssetDatabase 常用方法

  • 代码：存储 Unity 原生资源
[MenuItem("Assets/Create/New Material")]
static void CreateNewMaterial()
{
    Material material = new Material(Shader.Find("Standard"));
    
    // 根据 Unity 原生资源实例，为其创建本地资源
    AssetDatabase.CreateAsset(material, "Assets/Materials/NewMaterial.mat");
    
    // 保存资源并刷新资源数据库
    AssetDatabase.SaveAssets();
    AssetDatabase.Refresh();
}
  • 代码：创建资源
[MenuItem("Assets/Create/New Asset")]
static void CreateNewAsset()
{
	// 创建一个新的 ScriptableObject 类型资产
    ScriptableObject newAsset = CreateInstance<ScriptableObject>();
    
    // 设置资产的名称
    newAsset.name = "NewAsset";
    
    // 设置新资产的保存路径
    string path = "Asset/MyAssets/NewAsset.asset";
    
    // 使用AssetDatabase方法创建Unity中的新资产
    AssetDatabase.CreateAsset(newAsset,path);
    AssetDatabase.SaveAssets();
    AssetDatabase.Refresh();
}
  • 代码：创建文件夹
[MenuItem("Assets/Create/New Folder")]
static void CreateNewFolder()
{
	// 指定新文件夹的路径
	string folderPath = "Assets/MyFolders/NewFolder";
	
	// 使用AssetDatabase.CreateFolder方法创建新文件夹
	AssetDatabase.CreateFolder("Assets/MyFolders", "NewFolder");
	
	// 刷新资源数据库
	AssetDatabase.SaveAssets();
	AssetDatabase.Refresh();
}
  • 代码：删除指定资产或文件夹
[MenuItem("Assets/Delete Selected Asset")]
static void DeleteSelectedAsset()
{
	// 获取当前选中的资产的路径
	string assetPath = AssetDatabase.GetAssetPath(Selection.activeObject);
	
	// 使用AssetDatabase.DeleteAsset方法删除资产
	// 如果当前路径不为空，并且资产数据库当中包含选中的物体时，才会在资产数据库中进行删除
	if (!string.IsNullOrEmpty(assetPath) && AssetDatabase.Contains(Selection.activeObject))
	{
		AssetDatabase.DeleteAsset(assetPath);
		AssetDatabase.SaveAssets();
		AssetDatabase.Refresh();
	}
}
  • 代码：一次性删除多个资产或文件夹
[MenuItem("Assets/Delete Multiple Assets")]
static void DeleteMultipleAssets()
{
	// 获取所有选中的资产的路径
	string[] assetPaths = Selection.assetGUIDs.Select(AssetDatabase.GUIDToAssetPath).ToArray();
	
	// DeleteAssets方法传入string数组，数组里面的每一个元素都是一个路径，将其原本删除
	AssetDatabase.DeleteAssets(assetPaths);
	AssetDatabase.SaveAssets();
	AssetDatabase.Refresh();
}

AssetDatabase 获取物体

概念：AssetDatabase 获取物体

  • 概念：获得资产的路径
    • 第一步：获得某个资产的路径
      • string assetPath = AssetDatabase.GetAssetPath(someGameObject);
      • 返回的结果是带后缀的完整路径；
    • 第二步：获得该资产所在的文件夹路径
      • string folderPath = Path.GetDirectoryName(assetPath);
  • 概念：按路径加载资源 - LoadAssetAtPath
    • 语法：
      • LoadAssetAtPath<T>(string path)
    • 解释：
      • 可以使用 AssetDatabase.LoadAssetAtPath 方法来获取 Assets 目录下的一个游戏物体（如 GameObject）。这个方法需要一个表示资源路径的字符串作为参数，并返回该路径下指定类型的资源；
      • 直接加载指定路径资源，如纹理、材质等。适用于精确控制资源加载位置；
    • 示例：
      • Texture2D tex = AssetDatabase.LoadAssetAtPath<Texture2D>("Assets/Textures/icon.png");
    • 补充：
      • 也可以使用 LoadAllAssetsAtPath 可加载同一文件中的多个子资源（如 FBX 中的多个网格）；
  • 概念：按路径加载路径下所有资源
    • 示例：
      • object[] objs = AssetDatabase.LoadAllAssetsAtPath("Assets/Models/character.fbx");
    • 解释：
      • 加载路径下的所有资源，适用于处理包含多个子资源的文件（如 FBX 模型）；
  • 概念：文件操作
    • 使用 MoveAsset、DeleteAsset、RenameAsset 等方法替代直接的文件操作，避免破坏 Unity 元数据一致性；
    • string result = AssetDatabase.MoveAsset("Assets/OldPath.mat", "Assets/NewPath.mat");

资源查找

概念：AssetDatabase 与资源查找

  • 概念：FindAssets 方法
    • FindAssets 是 Unity Editor 环境下用于搜索项目资源的静态方法，它基于 Unity 的资源数据库进行查询，返回匹配资源的 GUID 数组；
  • 定义：
    • 参数一为"搜索过滤语法"，参数二为"限定搜索范围"；
  • 概念：搜索过滤语法
      1. 按照类型过滤：以 t: 开头
      • 查找所有预制体：string[] guids = AssetDatabase.FindAssets("t:Prefab");
      • 查找所有材质：string[] matGuids = AssetDatabase.FindAssets("t:Material");
      • 支持的类型标识符包括：Prefab、Scene、Material、Texture、Script、Shader 等；
      1. 按照标签过滤：以 l: 开头
      • 查找所有标记为 UI 的资源：
      • string[] uiGuids = AssetDatabase.FindAssets("l:UI");；
      1. 按名称过滤：
      • string[] heroGuids = AssetDatabase.FindAssets("hero");
      • string[] iconGuids = AssetDatabase.FindAssets("icon*");
      1. 组合条件：
      • 查找名称包含 go 的所有 GameObject 预制体：
      • string[] comboGuids = AssetDatabase.FindAssets("go t:GameObject");
  • 概念：限定搜索范围
    • 在指定的单个文件夹中查找：string[] sceneGuids = AssetDatabase.FindAssets("t:Scene",new[] { "Assets/Scenes/Levels" });
    • 在指定的多个文件夹中查找：string[] prefabGuids = AssetDatabase.FindAssets("t:Prefab",new[] { "Assets/Prefabs/Characters", "Assets/Prefabs/Weapons" });
  • 代码：
// 查找所有名为"hero"的预制体
string[] guids = AssetDatabase.FindAssets("hero t:Prefab");

// 查找所有标签为"Environment"的纹理
string[] envTextures = AssetDatabase.FindAssets("l:Environment t:Texture");

// 在指定文件夹中查找
string[] sceneGuids = AssetDatabase.FindAssets("t:Scene", new[] {"Assets/Scenes/Levels"});

AssetDatabase 进阶知识

资源依赖关系与修改检测

概念：资源依赖关系

  • 概念：资源依赖的类型
    • Unity 中的资源依赖主要分为两种：
      1. 直接依赖​：
      • 资源 A 直接引用资源 B，如材质直接引用纹理；
      • 通过 AssetDatabase.GetDependencies(path, false) 可获取直接依赖列表；
      1. 间接依赖：
      • 资源 A 通过中间资源间接引用资源 C，如预制体引用材质，材质又引用纹理；
      • 使用 AssetDatabase.GetDependencies(path, true) 可获取完整的依赖链；
  • 概念：AssetDatabase.GetDependencies
// 获得单个资源的依赖
string[] deps = AssetDatabase.GetDependencies("Assets/Prefabs/Character.prefab");

// 获得多个资源的共同依赖
string[] paths = {"Assets/Prefabs/A.prefab","Assets/Prefabs/B.prefab"};
string[] commonDeps = AssetDatabase.GetDependencies(paths);

// 递归的查询完整依赖链
// 注意：递归查询(recursive=true)会包含输入资源本身
string[] allDeps = AssetDatabase.GetDependencies("Assets/Scene.unity",true);

概念：利用哈希计算检测资源是否更改

  • 概念：AssetDatabase.GetAssetDependencyHash
    • AssetDatabase.GetAssetDependencyHash 返回一个资源所有依赖项的聚合哈希值 (Hash128)，这个哈希值会随着原资源路径或内容、源文件更改、目标平台更改而发生变化；
    • 当这个哈希值发生变化时，表明该资源的导入结果可能已经改变，相关的 AssetBundle 需要重新构建；
  • 分析：哈希计算原理
    • Unity 采用类似内容寻址的哈希计算方式，具体包含以下要素：
      1. 资源内容​：资源的实际二进制内容；
      1. ​依赖关系​：资源引用的其他资源及其版本；
      1. 导入设置​：在 Import Settings 中配置的参数；
      1. 平台特性​：不同平台下的资源处理差异；
    • 这种计算方式确保了只要资源或其依赖的任何部分发生变化，哈希值就会改变，从而实现精确的变更检测；
  • 定义：
  • 代码：
[MenuItem("TOols/Check Material Dependency Hash")]
static void CheckMaterialHash()
{
	// 1. 使用AssetDatabase从本地加载资源
	Material mat = AssetDatabase.LoadAssetAtPath<Material>("Assets/Materials/MyMat.mat");
	
	// 2. 获得加载资源的Hash值
	Hash128 srcHash = AssetDatabase.GetAssetDependencyHash("Assets/Materials/MyMat.mat");
	
	// 3. 对资源进行修改，并保存
	mat.color  = Color.red;
	EditorUtility.SetDirty(mat);
	AssetDatabase.SaveAssets();
	
	// 4. 读取新的哈希值
	Hash128 newHash = AssetDatabase.GetAssetDependencyHash("Assets/Materials/MyMat.mat");
	
	Debug.Log($"Original Hash: {originalHash}");
	Debug.Log($"New Hash: {newHash}");
	Debug.Log($"Changed: {originalHash != newHash}");
}

资源批量处理

概念：资源批量处理

  • 概念：TextureImporterType - 纹理导入类
    • TextureImporterType 定义了纹理在 Unity 中的使用方式，决定了 Unity 如何优化和处理该纹理资源；
    • 主要分类如下：
      • Default：通用纹理类型，适用于大多数标准贴图（如颜色贴图），关键参数包括 sRGB（颜色空间）和 Mip Maps 生成选项；
      • Normal map ​：专用于法线贴图，必须勾选"Create from grayscale"选项才能正确工作，Unity 会自动将灰度图转换为法线图的蓝色通道；
      • Sprite：用于 2D 精灵和 UI 元素，需要配置 Sprite Mode（Single/Multiple）和 Pixels Per Unit 等参数；
      • Advanced ​：提供完全自定义设置，允许手动配置所有纹理参数，适合特殊需求；
  • 概念：TextureImporterFormat - 纹理压缩格式
    • TextureImporterFormat 定义了纹理在目标平台上的压缩存储格式，主要分为几大类：
  • 代码：
using System.IO;
using UnityEngine;
using UnityEditor;
using System.Collections.Generic;

/// <summary>
/// 图片资源批量处理器
/// 功能：批量修改选中文件夹下所有图片的导入设置
/// </summary>
public class TextureBatchProcessor : EditorWindow
{
    // 存储当前选中的文件夹路径
    private string selectedFolderPath;
    
    // 纹理类型设置
    private TextureImporterType textureType = TextureImporterType.Sprite;
    
    // 纹理压缩设置
    private TextureImporterFormat androidFormat = TextureImporterFormat.ETC2_RGBA8;
    private TextureImporterFormat iosFormat = TextureImporterFormat.ASTC_RGBA_6x6;
    
    // 是否生成MipMap
    private bool generateMipMaps = false;
    
    // 是否可读写
    private bool isReadable = false;
    
    // 滚动视图位置
    private Vector2 scrollPos;
    
    // 处理进度(0-1)
    private float progress = 0;
    
    // 添加菜单项
    [MenuItem("Tools/资源处理/图片批量处理器")]
    public static void ShowWindow()
    {
        // 创建窗口实例
        var window = GetWindow<TextureBatchProcessor>();
        
        // 设置窗口标题
        window.titleContent = new GUIContent("图片批量处理器");
        
        // 展示窗口
        window.Show();
    }
    
    // 绘制自定义窗口的GUI
    private void OnGUI()
    {
	    // 创建可滚动视图区域，用scrollPos记录当前的滚动值
        scrollPos = EditorGUILayout.BeginScrollView(scrollPos);
        
        // 1. 文件夹选择区域
        EditorGUILayout.Space();
        // EditorStyles.boldLabel：将文本加粗渲染
        EditorGUILayout.LabelField("文件夹选择", EditorStyles.boldLabel);
        
        // 显示当前选中的文件夹路径
        EditorGUILayout.LabelField("当前文件夹:", selectedFolderPath ?? "未选择");
        
        // 选择文件夹按钮
        if (GUILayout.Button("选择文件夹"))
        {
            // 获取 Project 窗口中选择的对象
            var selection = Selection.GetFiltered(typeof(DefaultAsset), SelectionMode.Assets);
            if (selection.Length > 0)
            {
                // 获取选中对象的路径
                selectedFolderPath = AssetDatabase.GetAssetPath(selection[0]);
                
                // 验证是否是文件夹
                if (!AssetDatabase.IsValidFolder(selectedFolderPath))
                {
                    EditorUtility.DisplayDialog("错误", "请选择文件夹而不是文件!", "确定");
                    selectedFolderPath = null;
                }
            }
            else
            {
                EditorUtility.DisplayDialog("提示", "请在Project窗口中选择一个文件夹", "确定");
            }
        }
        
        // 2. 纹理设置区域
        EditorGUILayout.Space();
        EditorGUILayout.LabelField("纹理设置", EditorStyles.boldLabel);
        
        // 纹理类型下拉菜单
        textureType = (TextureImporterType)EditorGUILayout.EnumPopup("纹理类型:", textureType);
        
        // 平台特定设置
        EditorGUILayout.Space();
        EditorGUILayout.LabelField("平台设置", EditorStyles.helpBox);
        
        // Android平台设置
        androidFormat = (TextureImporterFormat)EditorGUILayout.EnumPopup("Android格式:", androidFormat);
        
        // iOS平台设置
        iosFormat = (TextureImporterFormat)EditorGUILayout.EnumPopup("iOS格式:", iosFormat);
        
        // 其他设置
        EditorGUILayout.Space();
        EditorGUILayout.LabelField("其他设置", EditorStyles.helpBox);
        
        generateMipMaps = EditorGUILayout.Toggle("生成MipMap:", generateMipMaps);
        isReadable = EditorGUILayout.Toggle("可读写:", isReadable);
        
        // 3. 执行按钮区域
        EditorGUILayout.Space();
        if (GUILayout.Button("执行批量处理", GUILayout.Height(30)))
        {
            if (string.IsNullOrEmpty(selectedFolderPath))
            {
                EditorUtility.DisplayDialog("错误", "请先选择文件夹!", "确定");
                return;
            }
            
            // 确认对话框
            if (EditorUtility.DisplayDialog("确认", 
                $"确定要处理文件夹 {selectedFolderPath} 下的所有图片吗?", "确定", "取消"))
            {
                // 开始批量处理
                BatchProcessTextures();
            }
        }
        
        // 显示处理进度
        if (progress > 0)
        {
            EditorGUILayout.Space();
            EditorGUI.ProgressBar(EditorGUILayout.GetControlRect(), progress, "处理进度");
        }
        
        EditorGUILayout.EndScrollView();
    }
    
    // 批量处理纹理的核心方法
    private void BatchProcessTextures()
    {
        try
        {
            // 1. 收集所有图片文件
            List<string> imagePaths = CollectImageFiles(selectedFolderPath);
            if (imagePaths.Count == 0)
            {
                EditorUtility.DisplayDialog("提示", "选择的文件夹中没有图片文件", "确定");
                return;
            }
            
            // 2. 开始资源编辑事务
            AssetDatabase.StartAssetEditing();
            
            // 3. 处理每个图片文件
            for (int i = 0; i < imagePaths.Count; i++)
            {
                string path = imagePaths[i];
                
                // 更新进度
                progress = (float)i / imagePaths.Count;
                EditorUtility.DisplayProgressBar("处理中", $"正在处理: {Path.GetFileName(path)}", progress);
                
                // 获取纹理导入器
                TextureImporter importer = AssetImporter.GetAtPath(path) as TextureImporter;
                if (importer == null) continue;
                
                // 修改纹理设置
                ModifyTextureSettings(importer);
                
                // 保存修改
                importer.SaveAndReimport();
            }
        }
        catch (System.Exception e)
        {
            Debug.LogError($"批量处理失败: {e.Message}");
        }
        finally
        {
            // 确保总是结束资源编辑和清理进度条
            AssetDatabase.StopAssetEditing();
            EditorUtility.ClearProgressBar();
            progress = 0;
            
            // 刷新资源数据库
            AssetDatabase.Refresh();
            
            EditorUtility.DisplayDialog("完成", "批量处理完成!", "确定");
        }
    }
    
    // 收集指定文件夹下的所有图片文件
    private List<string> CollectImageFiles(string folderPath)
    {
        List<string> imagePaths = new List<string>();
        
        // 获取文件夹绝对路径
        string fullPath = Path.Combine(Application.dataPath, folderPath.Replace("Assets/", ""));
        
        // 搜索所有图片文件
        string[] extensions = { "*.png", "*.jpg", "*.jpeg", "*.tga", "*.psd" };
        foreach (string ext in extensions)
        {
            foreach (string file in Directory.GetFiles(fullPath, ext, SearchOption.AllDirectories))
            {
                // 转换为Unity相对路径
                string relativePath = "Assets" + file.Replace(Application.dataPath, "").Replace('\\', '/');
                imagePaths.Add(relativePath);
            }
        }
        
        return imagePaths;
    }
    
    // 修改纹理导入设置
    private void ModifyTextureSettings(TextureImporter importer)
    {
        // 基本设置
        importer.textureType = textureType;
        importer.mipmapEnabled = generateMipMaps;
        importer.isReadable = isReadable;
        
        // Android平台设置
        TextureImporterPlatformSettings androidSettings = importer.GetPlatformTextureSettings("Android");
        androidSettings.overridden = true;
        androidSettings.format = androidFormat;
        importer.SetPlatformTextureSettings(androidSettings);
        
        // iOS平台设置
        TextureImporterPlatformSettings iosSettings = importer.GetPlatformTextureSettings("iPhone");
        iosSettings.overridden = true;
        iosSettings.format = iosFormat;
        importer.SetPlatformTextureSettings(iosSettings);
        
        // 如果设置为Sprite类型，添加一些默认设置
        if (textureType == TextureImporterType.Sprite)
        {
            importer.spritePixelsPerUnit = 100;
            importer.spritePivot = Vector2.one * 0.5f;
            importer.spriteImportMode = SpriteImportMode.Single;
        }
    }
