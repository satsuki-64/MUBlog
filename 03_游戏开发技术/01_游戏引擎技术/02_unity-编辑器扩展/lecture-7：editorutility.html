Lecture 7：EditorUtility

基础概念

什么是 EditorUtility

概念：EditorUtility

  • 概念：
    • EditorUtility 是 Unity 编辑器中的一个静态工具类，专门用于编辑器脚本开发​（继承自 Editor 或 EditorWindow 的脚本）；
    • 它提供了一系列辅助功能来简化自定义编辑器功能的实现，包括对话框、进度条、文件操作、对象状态管理等常见编辑器操作；
    • 使用 EditorUtility 需要引入 UnityEditor 命名空间，并且其相关代码只能出现在编辑器脚本中​。编辑器脚本为在 Editor 文件夹下的脚本，或使用 UNITY_EDITOR 宏定义包裹的代码；
  • 分类：EditorUtility 与 EditorGUIUtility
    • EditorUtility 更偏向通用功能工具，而 EditorGUIUtility 更偏向编辑器 UI 开发辅助；
  • 示例：
    • 此代码中的 EditorUtility 提供 InstanceIDToObject，根据上文传入的实例 ID，得到当前实例 ID 所对应的 Object 对象；
    • 再通过 AssetDatabase.GetAssetPath 得到此对象对应的资产的路径；
    • string path = AssetDatabase.GetAssetPath(EditorUtility.InstanceIDToObject(instanceID));

什么是 EditorGUIUtility

概念：EditorGUIUtility

  • 概念：
    • EditorGUIUtility 是 Unity 编辑器扩展开发中的一个实用工具类，它为 EditorGUI 和 EditorGUILayout 提供了丰富的辅助功能；
  • 分类：常用方法

EditorUtility 基础操作

对话框操作

概念：对话框操作

  • 概念：确认对话框
    • DisplayDialog 用于显示一个模态对话框，包含标题、提示信息和按钮；
    • 返回值表示当前是否点击对话框的按钮，false 表示点击取消、不按按钮；
  • 代码：
if(EditorUtility.DisplayDialog("标题","提示信息","按钮"))
{
	// 用户点击“按钮”执行的操作
	Debug.Log("点击按钮");
}
else
{
	// 将对话框关闭，进入此部分
	Debug.Log("点击取消");
}

概念：复杂对话框

  • 概念：
    • DisplayDialogComplex 可以显示三个按钮的复杂对话框，返回值为整数表示用户点击了哪个按钮；
  • 代码：
// 显示三按钮对话框
int result = EditorUtility.DisplayDialogComplex("复杂对话框", "请选择一个操作", "选项1", "取消", "选项2");

switch (result)
{
    case 0: // 第一个按钮
        Debug.Log("选项1被按下了");
        break;
    case 1: // 第二个按钮（取消）
        Debug.Log("取消被按下了");
        break;
    case 2: // 第三个按钮
        Debug.Log("选项2被按下了");
        break;
}

进度条操作

概念：进度条

  • 概念：
    • DisplayProgressBar 用于在编辑器顶部显示一个进度条，常用于长时间操作的进度反馈；
    • 其中参数一表示进度条窗口的标题，参数二表示进度条显示的文本信息，参数三表示当前进度条的进度；
  • 代码：
// 显示进度条
EditorUtility.DisplayProgressBar("处理中", "正在处理数据...", 0.5f);

概念：带取消按钮的进度条

  • 概念：
    • DisplayCancelableProgressBar 显示一个带取消按钮的进度条，允许用户中断长时间操作；
    • 返回值​：如果用户点击了取消按钮返回 true，否则返回 false；
  • 代码：
// 显示可取消进度条
if (EditorUtility.DisplayCancelableProgressBar("处理中", "正在处理数据...", 0.5f))
{
    // 用户点击了取消按钮
    Debug.Log("操作被用户取消");
}

代码：完整使用示例

try
{
    for (int i = 0; i < 100; i++)
    {
        // 检查用户是否取消了操作
        if (EditorUtility.DisplayCancelableProgressBar("处理中", $"正在处理项目 {i}", i / 100f))
        {
            break; // 用户取消，中断操作
        }
        
        // 执行实际操作
        System.Threading.Thread.Sleep(50);
    }
}
finally
{
    // 确保无论发生什么情况都关闭进度条
    EditorUtility.ClearProgressBar();
}

文件与文件夹操作

概念：打开文件面板

  • 概念：
    • OpenFilePanel 用于显示一个系统文件选择对话框，允许用户选择已存在的文件；
    • 参数一表示面板框标题，参数二表示初始时打开的目录，参数三表示要选择文件类型；
  • 代码：
string path = EditorUtility.OpenFilePanel("选择文件",Application.dataPath,"png");
if(!string.IsNullOrEmpty(path))
{
	Debug.Log($"选中的文件为：{path}");
}

概念：打开带过滤的文件面板

  • 概念：
    • OpenFilePanelWithFilters 提供更灵活的文件过滤选项，支持多个过滤条件；
    • 使用一个 string 数组来存储过滤条件；
  • 代码：
// 使用过滤条件打开文件面板
string[] filters = new string[] { "图像文件", "png,jpg,jpeg", "文本文件", "txt", "所有文件", "*" };
string path = EditorUtility.OpenFilePanelWithFilters("选择文件", Application.dataPath, filters);

概念：保存文件的面板

  • 概念：
    • SaveFilePanel 显示一个文件保存对话框，允许用户选择保存文件的位置和名称；
    • SaveFilePanelInProject 类似于 SaveFilePanel，但限制在当前 Unity 项目 Assets 目录内操作，返回相对路径；
  • 代码：
// 打开文件保存面板
string path = EditorUtility.SaveFilePanel("保存文件", Application.dataPath, "default", "txt");

if (!string.IsNullOrEmpty(path))
{
    // 将数据保存到文件
    System.IO.File.WriteAllText(path, "文件内容");
}

// 在项目内保存文件
string path = EditorUtility.SaveFilePanelInProject("保存资源", "newAsset", "asset", "请输入描述");

if (!string.IsNullOrEmpty(path))
{
    // 创建资源
    AssetDatabase.CreateAsset(new Material(Shader.Find("Standard")), path);
}

概念：选择与保存文件夹面板

  • 概念：
    • OpenFolderPanel 显示一个选择文件夹的对话框，允许用户选择已存在的文件夹；
    • SaveFolderPanel 显示一个选择或创建文件夹的对话框，允许用户选择现有文件夹或输入新文件夹名称；
  • 代码：
// 打开文件夹选择面板
string folderPath = EditorUtility.OpenFolderPanel("选择文件夹", Application.dataPath, "");

if (!string.IsNullOrEmpty(folderPath))
{
    Debug.Log($"选中的文件夹: {folderPath}");
}

// 打开文件夹保存面板
string folderPath = EditorUtility.SaveFolderPanel("选择保存位置", Application.dataPath, "");

if (!string.IsNullOrEmpty(folderPath))
{
    Debug.Log($"选定的保存位置: {folderPath}");
}

对象状态管理

概念：对象状态管理

  • 概念：设置对象启用状态
// 设置对象启用状态
GameObject obj = Selection.activeGameObject;
if (obj != null)
{
	// SetObjectEnabled 用于设置 GameObject 的启用状态（激活/禁用）
    EditorUtility.SetObjectEnabled(obj, true);
}
  • 概念：获取对象启用状态
// 获取对象启用状态
GameObject obj = Selection.activeGameObject;
if (obj != null)
{
	// GetObjectEnabled 用于获取 GameObject 当前的启用状态
    bool isEnabled = EditorUtility.GetObjectEnabled(obj);
    Debug.Log($"对象启用状态: {isEnabled}");
}
  • 概念：设置脏标志 - SetDirty
// 标记对象为已修改
GameObject obj = Selection.activeGameObject;
if (obj != null)
{
    Undo.RecordObject(obj, "修改对象");
    obj.name = "新名称";
	// SetDirty 用于标记对象为已修改（脏），告诉 Unity 该对象需要保存
    EditorUtility.SetDirty(obj);
}
  • 概念：清除脏标志
// 清除对象修改标志
GameObject obj = Selection.activeGameObject;
if (obj != null)
{
    EditorUtility.ClearDirty(obj);
}

资源依赖获取

代码

GameObject obj = Selection.activeGameObject;
if (obj != null)
{
	// CollectDependencies 用于查找指定对象所依赖的所有资源，返回一个包含所有依赖项的数组
    Object[] dependencies = EditorUtility.CollectDependencies(new Object[] { obj });
    
    Debug.Log($"找到 {dependencies.Length} 个依赖资源");
    foreach (Object dep in dependencies)
    {
        Debug.Log($"依赖: {dep.name} ({dep.GetType().Name})");
    }
    
    // 可以选择所有依赖资源
    Selection.objects = dependencies;
}
