编辑器扩展常用特性

概念：编辑器扩展常用特性

  • 概念：[CanEditMultipleObjects] 特性
    • [CanEditMultipleObjects] 是 Unity 编辑器扩展中的一个重要特性，它允许自定义编辑器（Custom Editor）同时支持对多个选中的对象进行编辑；
    • 添加此特性后，你就可以在自定义的 OnInspectorGUI 方法中，通过一套 UI 控件同时修改所有选中对象的公共属性值，极大提升批量处理对象的效率；
  • 概念：[ContextMenu("Name")] 特性
    • 用于向脚本组件添加自定义的上下文菜单项。当你在 Inspector 窗口中右键点击附加了该脚本的组件时，会出现一个带有指定名称的菜单选项，点击后可触发相应的脚本方法；
    • Unity 编辑器会扫描所有 MonoBehaviour 派生类中的 [ContextMenu] 特性，并在生成组件上下文菜单时将这些特性标记的方法添加到菜单列表中。当用户选择该菜单项时，编辑器通过反射机制调用对应的方法；
    • 方法可以无参数，或是使用 MenuCommand 参数（提供上下文信息）；
  • 概念：[InitializeOnLoad] 特性
    • 此特性用于标记一个类，指示 Unity 在编辑器启动时或脚本重新编译后自动执行该类的静态构造函数，这使得开发者可以在不手动操作的情况下执行一些初始化工作；
    • 当 Unity 编辑器启动或检测到脚本变化时，它会进行域重新加载​（Domain Reload）。在这个过程中，编辑器会查找所有带有 [InitializeOnLoad] 特性的类，并自动调用它们的静态构造函数；
  • 概念：[ExecuteInEditMode] 特性
    • [ExecuteInEditMode] 是 UnityEngine 命名空间下的特性，允许脚本在编辑器非运行模式下也会执行相应的生命周期方法；
    • Unity 编辑器会特别处理带有 [ExecuteInEditMode] 特性的脚本，即使不在播放模式下，也会调用它们的生命周期方法（如 Update、OnGUI 等）；
    • 不过，这些方法的调用频率与播放模式下不同 - 它们只在特定事件发生时被调用；
  • 概念：[InitializeOnLoadMethod] 特性
    • [InitializeOnLoadMethod] 是 UnityEditor 命名空间下的特性，用于标记一个静态方法，使其在编辑器启动时或脚本重新编译后自动执行；
    • 与 [InitializeOnLoad] 不同，它直接标记方法而不是类，使用更加灵活；
注意：关于编辑器扩展
    1. 当使用 SerializedObject 和 SerializedProperty 来操作多个选中对象，但某个属性值不同时，SerializedProperty 的 hasMultipleDifferentValues 属性会返回 true；
    1. [ContextMenu("Reset Value")] 标记的方法必须是非静态方法；
    1. 如果当前类当中有名为 value 的字段，[ContextMenu("Reset Value")] 标记的方法可以直接绑定到此字段；

自定义独立 Inspector

PropertyAttribute 类

概念：PropertyAttribute 类

  • 概念：
    • 将自定义的类继承自 PropertyAttribute，表明自己自定义的类是一个属性特性，用于为字段添加元数据；
    • 其中包含一些用于指定存储参数的公共字段，比如 min 和 max 是公共字段，用于存储属性使用时传入的参数（如 [MyTestAttribute(0,100)] 中的 0 和 100），这些参数定义了整数字段的允许范围；
  • 作用：
    • 继承自 PropertyAttribute 的类仅作为数据容器，定义了“需要什么”（这里是一个取值范围），而不涉及“如何实现”；
  • 代码：
using UnityEngine;
using System.Collections;

public class MyTestAttribute : PropertyAttribute 
{
    public int max;
    public int min;

    public MyTestAttribute(int a, int b)
    {
        min = a;
        max = b;
    }
}

PropertyDrawer 类

概念：PropertyDrawer 类

  • 概念：​[CustomPropertyDrawer(typeof(MyTestAttribute))]​ 特性
    • 此特性告诉 Unity，这个继承自 PropertyDrawer 的类（下方代码中的 MyTestDrawer 类）专门用于绘制带有 MyTestAttribute 特性的字段；
    • 可以在其中的 OnGUI 方法中定义属性在 Inspector 中的实际绘制逻辑，比如：
      • position: 定义了绘制控件在 Inspector 中的位置和大小（一个矩形区域）
      • property: 表示正在绘制的序列化属性，用于获取和设置其值；
      • label: 表示该属性的标签（在 Inspector 中显示的名称）
  • 概念：(MyTestAttribute)base.attribute
    • (MyTestAttribute)base.attribute 用于获取应用到当前字段上的 MyTestAttribute 实例，从而拿到配置的 min 和 max 值；
  • 概念：OnGUI 参数
    • Rect position：定义了绘制区域的位置（X, Y）和尺寸（宽度 Width, 高度 Height），所有的 EditorGUI 绘制方法通常都需要一个 Rect 参数来指定位置；
    • SerializedProperty property：对你正在绘制的那个字段的封装引用，通过它可以获取当前值：property.intValue, property.floatValue, property.stringValue 等；
    • GUIContent label：通常用于获取字段在 Inspector 中显示的名称（label.text）
  • 代码：
using UnityEditor;
using System.Collections.Generic;
using UnityEngine;

[CustomPropertyDrawer(typeof(MyTestAttribute))]
public class MyTestDrawer : PropertyDrawer
{
	public override void OnGUI(Rect position,SerializedProperty property,GUIContent label)
	{
		MyTestAttribute attribute = (MyTestAttribute)base.attibute;
		
		property.intValue = Mathf.Min
		(
			Mathf.Max
			(
				EditorGUI.IntField(position, label.text, property.intValue),
				attribute.min
			), 
			
			attribute.max
		);
	}
}

// ReadOnlyDrawer.cs
// 此脚本必须放在名为"Editor"的文件夹中
using UnityEditor; // 需要UnityEditor命名空间
using UnityEngine;

/// <summary>
/// 用于绘制标记了[ReadOnly]特性的属性的自定义PropertyDrawer。
/// CustomPropertyDrawer属性将其与ReadOnlyAttribute关联起来。
/// </summary>
[CustomPropertyDrawer(typeof(ReadOnlyAttribute))]
public class ReadOnlyDrawer : PropertyDrawer
{
    /// <summary>
    /// 重写OnGUI方法以定义属性的绘制方式。
    /// </summary>
    /// <param name="position">在Inspector上绘制属性的屏幕区域</param>
    /// <param name="property">被绘制的SerializedProperty对象，包含其值和元数据</param>
    /// <param name="label">属性的显示标签</param>
    public override void OnGUI(Rect position, SerializedProperty property, GUIContent label)
    {
        // 保存当前的GUI启用状态
        bool previousGUIState = GUI.enabled;

        // 禁用GUI，使接下来的控件变为灰色且不可交互
        GUI.enabled = false;

        // 使用默认的属性字段绘制，但现在是禁用的
        // 这个方法的参数是：位置、属性、标签
        EditorGUI.PropertyField(position, property, label);

        // 将GUI的启用状态恢复为之前的状态
        GUI.enabled = previousGUIState;
    }
}
  • 代码：使用方法
using UnityEngine;

public class ExampleScript : MonoBehaviour
{
    // 这个值可以在脚本中修改，但在Inspector中不可编辑
    [ReadOnly]
    public int nonEditableInt = 10;

    // 显示一个不可编辑的字符串
    [ReadOnly]
    public string displayOnlyString = "This is read-only";

    // 甚至可以显示一个不可编辑的引用
    [ReadOnly]
    public GameObject someGameObjectReference;

    // 其他可正常编辑的字段
    public float normalEditableFloat = 5.0f;
}

  • ReadOnlyAttribute 特性类不能放在 Editor 文件夹中，因为它需要在运行时也可被访问；
  • 因此继承自 PropertyAttribute 的类不能和继承自 PropertyDrawer 的类放在一起；

using UnityEditor;
using UnityEngine;

[CustomEditor(typeof(UnityEditor.DefaultAsset))]
public class CustomDefaultAssetInspector : Editor
{
	public override void OnInspectorGUI()
	{
		// 获取当前CustomEditor得到的Unity资源实例，实例的引用为target变量
		string assetPath = AssetDatabase.GetAssetPath(target);
		
		// 判断是否是文件夹
		if(AssetDatabase.IsValidFolder(assetPath))
		{
			Debug.Log("当前选择的是文件夹");
		}
		else if(assetPath.EndWith(".unity"))
		{
			Debug.Log("当前选择的是场景文件");
		}
		else
		{
			Debug.Log("默认绘制");
			base.OnInspectorGUI();
		}
	}
}

自定义不可识别文件的打开

概念：自定义不可识别文件的打开

  • 概念：[OnOpenAssetAttribute] 特性
    • Unity 中的 [OnOpenAssetAttribute] 允许你在 Unity 编辑器中自定义资源的打开行为。当你双击 Project 窗口中的某个资源时，标记了此特性的静态方法就会被调用；
  • 方法：
      1. 标记了 [OnOpenAssetAttribute] 的方法，表示处理资源打开事件的回调；
      • 标记 [OnOpenAssetAttribute] 的方法需要满足以下要求：
      • （1）静态方法；
      • （2）返回值类型为 bool；
      • （3）函数参数为两个 int，分别代表 instanceID 和 Line；
      1. [OnOpenAssetAttribute] 特性的索引参数，用于控制多个 OnOpenAsset 回调的执行顺序。索引从 0 开始；
      • 当在 Project 窗口中双击一个资源时，Unity 会按照顺序索引，从小到大依次尝试所有标记了此特性的方法，直到返回值为真；
  • 概念：System.Diagnostics.Process 与 ProcessStartInfo 类
    • System.Diagnostics.Process 类是 .NET 中用于与操作系统进程交互的核心类，它允许你启动、停止、管理和监控本地及远程进程；
    • 其中，最基本的方式是使用 Process.Start 方法，其可以直接传递一个可执行文件路径，或者通过 ProcessStartInfo 对象进行更详细的配置；
    • ProcessStartInfo 类允许你设置启动进程时的各种参数，如命令行参数 (Arguments)、工作目录 (WorkingDirectory)、是否重定向标准输入输出 (RedirectStandardInput, RedirectStandardOutput, RedirectStandardError)、窗口样式 (WindowStyle) 等
  • 代码：
using UnityEngine;
using UnityEditor;
using UnityEditor.Callbacks; // 引入包含OnOpenAssetAttribute等回调特性的命名空间

public class MyAssetHandler
{
    // 当在Project窗口中双击一个资源时，Unity会按照顺序索引从小到大依次尝试所有标记了此特性的方法
    // 方法签名必须匹配：静态方法，返回bool，接受int类型的instanceID和int类型的line参数
    [OnOpenAssetAttribute(1)]
    public static bool step1(int instanceID, int line)
    {
	    // 返回false表示此方法未处理该资源的打开操作
	    // Unity将继续尝试执行下一个顺序索引的OnOpenAsset方法或默认行为
        return false; 
    }

    // 另一个OnOpenAssetAttribute特性，顺序索引为2，它将在索引为1的方法之后被调用。
    [OnOpenAssetAttribute(2)]
    public static bool step2(int instanceID, int line)
    {
        // 通过资源的实例ID获取其在Unity项目中的相对路径
        // 返回值例如："Assets/MyFolder/MyFile.xx"
        string path = AssetDatabase.GetAssetPath(EditorUtility.InstanceIDToObject(instanceID));
        
        // 将Unity项目的相对路径转换为系统上的绝对路径。
        // Application.dataPath 是项目Assets文件夹的绝对路径
        // 例如："C:/MyProject/Assets"
        // path.Replace("Assets/", "") 移除路径开头的"Assets/"部分
        // 之所以要移除，是因为Application.dataPath中以及包含了Asset/的路径
        string name = Application.dataPath + "/" + path.Replace("Assets/", "");

        // 检查文件扩展名是否为".xx"
        if (name.EndsWith(".xx"))
        {
            // 创建一个新的进程对象
            System.Diagnostics.Process process = new System.Diagnostics.Process();
            // 创建进程启动信息对象，用于配置如何启动外部程序
            System.Diagnostics.ProcessStartInfo startInfo = new System.Diagnostics.ProcessStartInfo();
            
            // 设置进程窗口样式为隐藏（此处可能希望是正常窗口，考虑改为Normal）
            startInfo.WindowStyle = System.Diagnostics.ProcessWindowStyle.Hidden;
            // 设置要启动的外部程序的完整路径（这里是Sublime Text 3的exe文件路径）
            startInfo.FileName = "D:/Program Files/Sublime Text 3/sublime_text.exe";
            // 设置传递给外部程序的命令行参数，这里将文件绝对路径传递给它
            startInfo.Arguments = name;
            // 将配置好的启动信息赋给进程对象
            process.StartInfo = startInfo;
            // 启动进程，用Sublime Text打开指定的.xx文件
            process.Start();
            // 返回true告知Unity此资源的打开操作已被处理，Unity将不再执行默认的打开行为。
            return true;
        }
        return false; // we did not handle the open
    }
}
