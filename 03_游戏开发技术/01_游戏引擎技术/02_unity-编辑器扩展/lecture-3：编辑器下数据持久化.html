Lecture 3：编辑器下数据持久化

EditorPrefs

基础概念

概念：EditorPrefs

  • 概念：
    • EditorPrefs 是 Unity 提供的用于在编辑器环境下持久化存储键值对数据的工具类；
    • 这是一种可以在项目之间共享的数据保存方式，适用于跨 Unity 编辑器共享数据而不受项目约束
    • 其与 PlayerPrefs 的区别在于：
        1. PlayerPrefs：用于运行时（游戏内）数据存储，支持多平台；
        1. EditorPrefs：仅限编辑器工具开发，存储路径和机制与平台无关；
    • EditorPrefs 支持 string、int、float、bool 等数据类型，不同操作系统下路径不同，比如 Windows 注册表、macOS 的 plist 文件；
  • 概念：不同版本存储的 EditorPrefs
    • 保存的值可能会受到 Unity 大版本的影响，比如在 Unity 4.x 中保存的值只能在 Unity 4.x 中使用，Unity 5.x 也仅在 Unity 5.x 中可用；
  • 特点：
      1. 它不需要考虑运行时的效率问题，所以没有采用 PlayerPrefs 的优化方式，而是直接保存。EditorPrefs 中比 PlayerPrefs 多了一个 Bool 类型的键值对；
      1. 所有通过 EditorPrefs 保存的值均以明文形式保存，所以切勿保存重要信息；
      1. 在 Unity 编辑器环境中，谨慎调用 EditorPrefs.DeleteAll()。如果这样做下次打开 Unity 会发现以前保存的打开项目，全部消失了；

存储方法

概念：存储方法

    1. public static void SetString(string key, string value);
    • key：唯一标识数据的字符串（区分大小写）
    • value：要存储的字符串内容（不支持 null，若传入会转为空字符串）
    1. public static string GetString(string key, string defaultValue = "");
    • key：要检索的键名
    • defaultValue：键不存在时的返回值（默认空字符串）
  • 代码：存储数据
// 存储整型数据
EditorPrefs.SetInt("EditorWindowWidth", 800);

// 存储浮点型数据
EditorPrefs.SetFloat("EditorSnapValue", 0.5f);

// 存储字符串数据
EditorPrefs.SetString("LastOpenedScene", "Assets/Scenes/Main.unity");

// 存储布尔值数据(EditorPrefs特有，PlayerPrefs没有)
EditorPrefs.SetBool("ShowWelcomeScreen", false);
  • 代码：读取数据
// 读取数据时需要提供键名和可选的默认值(当键不存在时返回)

// 读取整型数据(带默认值)
int windowWidth = EditorPrefs.GetInt("EditorWindowWidth", 1024);

// 读取浮点型数据(带默认值)
float snapValue = EditorPrefs.GetFloat("EditorSnapValue", 1.0f);

// 读取字符串数据(带默认值)
string lastScene = EditorPrefs.GetString("LastOpenedScene", "");

// 读取布尔值数据(带默认值)
bool showWelcome = EditorPrefs.GetBool("ShowWelcomeScreen", true);
  • 代码：检查数据存在
// 检查键是否存在
if(EditorPrefs.HasKey("EditorWindowWidth")) {
    // 键存在时的处理
}

代码示例

代码

using UnityEditor;
using UnityEngine;

public class MyCustomWindow : EditorWindow
{
    private static MyCustomWindow window;
    private string someText = "";
	
	// 创建一个窗口并展示
    [MenuItem("Window/My Custom Window")]
    private static void ShowWindow()
    {
	    // GetWindow<MyCustomWindow> 中的泛型参数 <MyCustomWindow>​ 明确指定了你要获取或创建的窗口类型
	    // 这个方法会在当前所有打开的编辑器窗口中寻找第一个类型为 MyCustomWindow 的窗口实例
        window = GetWindow<MyCustomWindow>("Custom Window");
        window.Show();
    }
    
    // 开启窗口时，先读取之前保存的窗口名称与位置信息
    private void OnEnable()
    {
        // 读取保存的数据
        // 先检查在EditorPrefs中是否有MyCustomWindow_SomeText，如果有的话则获取
        if(EditorPrefs.HasKey("MyCustomWindow_SomeText"))
        {
            someText = EditorPrefs.GetString("MyCustomWindow_SomeText");
        }
        
        // 读取窗口位置
        if(EditorPrefs.HasKey("MyCustomWindow_PosX"))
        {
            position = new Rect(
                EditorPrefs.GetFloat("MyCustomWindow_PosX"),
                EditorPrefs.GetFloat("MyCustomWindow_PosY"),
                EditorPrefs.GetFloat("MyCustomWindow_Width", 400), // 后背的 “400” 表示默认值
                EditorPrefs.GetFloat("MyCustomWindow_Height", 300)
            );
        }
    }
    
    // 绘制GUI时，使用在OnEnable中从EditorPrefs中读取的数据
    private void OnGUI()
    {
        someText = EditorGUILayout.TextField("Some Text", someText);
        
        if(GUILayout.Button("Save"))
        {
            EditorPrefs.SetString("MyCustomWindow_SomeText", someText);
        }
    }
    
    private void OnDestroy()
    {
        // 保存窗口位置和大小
        EditorPrefs.SetFloat("MyCustomWindow_PosX", position.x);
        EditorPrefs.SetFloat("MyCustomWindow_PosY", position.y);
        EditorPrefs.SetFloat("MyCustomWindow_Width", position.width);
        EditorPrefs.SetFloat("MyCustomWindow_Height", position.height);
    }
}

EditorUserSetting

概念：EditorUserSetting

  • 概念：
    • EditorUserSettings 是 Unity 编辑器提供的一种数据存储机制，专门用于在项目内部保存和共享加密数据，特别适合存储敏感信息如密码、访问令牌等；
    • 与 EditorPrefs 不同，EditorUserSettings 以二进制形式存储数据，提供了基本的安全保障；
    • 保存的数据会持久存储在项目文件中，即使关闭并重新打开 Unity 编辑器也会保留；
  • 概念：存储位置
      1. Unity 2021 及更新版本​：
      • 存储在项目目录的 UserSettings 文件夹下的 EditorUserSettings.asset 文件中，以文本格式；
      1. 旧版本 Unity：
      • 存储在项目 Library 目录下的 EditorUserSettings.asset 文件中（二进制格式）
  • 总结：
  • 代码：使用方法
using UnityEditor;

// 注意：和EditorPrefs 直接存储加载不同，其方法中间的标识为 Config
public class EditorUserSettingsExample
{
    [InitializeOnLoadMethod]
    static void SaveSettings()
    {
        // 存储字符串值
        EditorUserSettings.SetConfigValue("UserEmail", "user@example.com");
        
        // 存储密码或令牌
        EditorUserSettings.SetConfigValue("AccessToken", "encrypted_token_here");
    }
}

// 读取数据
string email = EditorUserSettings.GetConfigValue("UserEmail");
string token = EditorUserSettings.GetConfigValue("AccessToken");

SerializedObject 类

基础概念

概念：SerializedObject 类

  • 概念：
    • SerializedObject 是 UnityEditor 命名空间下的类，用于以通用方式编辑 Unity 对象的序列化字段；
    • 它与 SerializedProperty 配合使用，可以自动处理撤销操作、Prefab 覆盖以及在 Inspector 中的正确样式显示；
  • 概念：SerializedObject 实现原理
      1. 使用 SerializedObject 类得到一个目标 Unity 对象的数据流，允许同时编辑对象共享的序列化数据；
      1. Unity 将对象序列化为二进制或 YAML 文本数据，SerializedObject 负责这些序列化操作；
      1. SerializedObject 在内部进行缓存，需要手动调用 Update 方法同步最新数据；
      1. 自动注册撤销操作，而直接修改 UnityEngine.Object 需要手动实现撤销；
  • 概念：创建 SerializedObject 对象
    • 注意在多目标编辑时，只能编辑这些对象共有的序列化字段；
  • 概念：void Update();
    • 更新序列化对象的状态，确保它反映目标对象的当前状态；
    • 当保持对 SerializedObject 的引用超过一帧时，必须手动调用此方法；
    • 如图所示是在访问 SerializedObject 之前先对其进行更新，以使其保持最新状态；
  • 概念：bool ApplyModifiedProperties();
    • 将更改应用到内部缓存，使其保持最新；
    • 通过 ApplyModifiedProperties 应用更改，这样将此视为一组；
    • 除非有特定的条件可以应用更改，否则将 Update 放在方法的第一行上，并将 ApplyModifiedProperties 放在方法的最后一行；
  • 补充：Unity 中可序列化的对象
    • UnityEngine.Object 的派生类（我们经常使用的 MonoBehaviour，ScriptableObject，Editor，EditorWindow 等）中字段是否可以序列化要看一下几点：
        1. 必须是公共变量或具有 Seri​​alizeField 属性的字段；
        1. 必须是 Unity 支持可序列化的类型，比如 sbyte，short，int，long，byte，ushort，uint，ulong，float，double，bool，char，string，UnityEngine.Object，具有 Seri​​alizable 属性和结构的类等；
    • 不能序列化的常见情况：
        1. 变量不能带 static，const，readonly 修饰符；
        1. 不能是抽象类；
    • 补充：
      • 我们经常看到一些书中说把需要在监视器中展示的字段设成公共变量，这只是为了方便非程序员更容易理解，而使其成为公共变量只是序列化的条件之一，有时候给私有变量添加 SerializeField 属性是更加合理的解决方案；

Object 和 SerializedObject 的关系

分析：UnityEngine.Object 和 SerializedObject 的关系

  • 总结：
    • 在 Unity 编辑器中，所有的游戏对象（UnityEngine.Object）都会被转换为 SerializedObject 并进行处理；
    • 当我们在 Inspector 中编辑组件的值时，不是在编辑 Component 组件的实例、或者其中某个字段的实例，而是在编辑其序列化对象，即 SerializedObject 的实例；
  • 分析：
    • 将 UnityEngine.Object 保存为资源时，会将其另存为二进制或 YAML 文本数据，而 SerializedObject 负责这些序列化;
    • 要将 UnityEngine.Object 保存为资源，必须将其先转为 SerializedObject，然后转换后的 SerializedObject 会创建两个文件：一个资源文件和一个 .meta 文件；
    • 资源文件是真实对象的序列化版本，而 .meta 文件保存导入的相关设置等；
    • 而什么是导入的相关设置？假设当前序列化了一个 Texture2D 的对象，当转换为 SerializedObject 时，Texture2D 对象也带有很多导入相关的设置。将 Texture2D 保存为资源时，是无法将这些设置写入纹理文件的，因此只能将其写入 .meta 文件，也就是元数据；
    • 同样，导入资源时，会从资源和 .meta 文件（如果没有 .meta 文件，默认情况下会自动生成）生成 SerializedObject，并将其转换为 UnityEngine.Object；
  • 概念：为什么要使用 SerializedObject
    • 在 Unity 编辑器中或者在编辑器扩展中，所有对象的操作应该尽可能用 SerializedObject，这是因为 SerializedObject 不仅能处理序列化的数据，而且还能处理 Undo 和 Selection 的操作；
      1. Undo：
      • 当使用 SerializedObject 编辑值时，撤消过程可以自动注册。如果直接编辑 UnityEngine.Object 实例，则必须自己实现撤消过程；
      1. Selection：
      • 当我们在项目窗口中选择资源时，它会立即对其进行反序列化以获取 UnityEngine.Object 的实例，并在 Inspector 中显示它的值。这样处理的主要用处是可以允许我们在选择多个对象的时候同时进行编辑；

FindProperty 方法

概念：FindProperty 方法

  • 概念：FindProperty 方法
    • 使用 FindProperty 方法获得序列化对象的属性之后，通过 vector3Value、intValue、floatValue 等来对其属性值进行设置；
  • 特点：
  • 代码：用 SerializedObject 重置选中游戏对象位置 
using UnityEditor;
using UnityEngine;
using System.Linq;

static class GameObjectPositionResetter
{
    [MenuItem("Edit/Reset Selected Objects Position")]
    static void ResetPosition()
    {
        // 1. 获取UnityEngine.Object对象实例
        // 获取所有选中对象的Transform组件
        var transforms = Selection.gameObjects.Select(go => go.transform).ToArray();
        
        // 2. 创建 SerializedObject 实例
        var serializedObject = new SerializedObject(transforms);
        
        // 3. 调用 SerializedObject 的成员被方法，获取要更改的属性
        // 查找并修改localPosition属性
        // 注意：可以Shift+右键点击Inspector中的属性名查看其路径
        var positionProperty = serializedObject.FindProperty("m_LocalPosition");
        
        // 4. 更改属性的值
        positionProperty.vector3Value = Vector3.zero;
        
        // 5. 应用修改
        serializedObject.ApplyModifiedProperties();
    }
}
  • 代码：在一个 SerializedObject 中处理多个 UnityEngine.Objects
Rigidbody[] rigidbodies = /* 获取组件 */;

var serializedObject = new SerializedObject(rigidbodies);

serializedObject.FindProperty("m_UseGravity").boolValue = true;

迭代器处理序列化对象

概念：迭代器处理序列化对象

  • 概念：GetIterator() 迭代器
    • GetIterator() 是 SerializedObject 类的方法，返回一个 SerializedProperty 对象，该对象指向序列化对象的第一个属性；
    • 这个初始的 SerializedProperty 实际上位于"根"位置（depth 为 -1），并不直接对应任何可见属性。相比直接使用 FindProperty 查找特定属性，GetIterator 更适合需要遍历未知属性结构的场景；
  • 概念：NextVisible(bool enterChildren) 方法
    • NextVisible(bool enterChildren) 是 SerializedProperty 的方法，将迭代器移动到下一个可见属性；
    • 参数 enterChildren 为 true 时，会进入复合类型（如结构体、数组等）的子属性；
    • 返回 false 时表示没有更多属性可供遍历，循环终止；
    • 对于数组元素，NextVisible(true) 会逐个访问数组元素及其子属性；
  • 注意：
      1. NextVisible 方法与 Next() 方法不同，NextVisible() 会跳过 Unity 内部隐藏的属性；
      1. enterChildren 参数​仅在需要时设置为 true，避免不必要的深度遍历；
  • 总结：
  • 代码：
SerializedProperty prop = serializedObject.GetIterator();
while (prop.NextVisible(true))
{
	// 将读取得到的对象逐个以字段的形式显示出来
    EditorGUILayout.PropertyField(prop);
}

PrefabUtility 类

概念：PrefabUtility 类

  • 概念：
    • PrefabUtility 是 Unity 编辑器 API 中处理预制体 (Prefab) 的核心类，它为开发者提供了创建、修改、实例化和管理预制体的强大功能集；
    • 预制体文件并不直接包含资源内容，而是通过 GUID (全局唯一标识符) ​和本地 ID 建立与外部资源 (如纹理、材质、网格等) 的引用关系。这种设计使得预制体能够灵活适应资源位置的变化，同时保持轻量级；
  • 概念：PrefabUtility 的实现
    • PrefabUtility 类作为 Unity 编辑器 API 的一部分，​桥接了编辑器用户界面与预制体的底层数据管理系统。它直接操作预制体资源文件 (.prefab) 和场景中的预制体实例，维护着复杂的引用关系网络；
    • 每个预制体资源都有唯一的 GUID，存储在 .meta 文件中，用于唯一标识资源而不依赖路径。除了会有 GUID 外，还会有运行时生成的实例 ID，其作为运行时生成的临时标识符，用于高效管理预制体实例；
    • 当生成一个预制体时，Unity 会：
        1. 加载预制体资源并解析其结构；
        1. 为每个对象和组件生成运行时实例的实例 ID；
        1. 应用所有属性值；
        1. 建立与原始预制体的连接关系；
    • 而 PrefabUtility 维护着一个变更追踪系统，当原始预制体被修改时，它会：
        1. 检测所有受影响的实例；
        1. 识别哪些修改可以安全传播 (未被实例覆盖的属性)；
        1. 自动更新符合条件的实例；
  • 方法：
      1. 预制体的创建于保存 - 非连接版本
      • 将场景中的 GameObject 保存为预制体资源，如果路径已存在则覆盖；
      1. 预制体的创建于保存 - 连接版本
      • 保存预制体并保持与场景实例的连接，支持撤销操作；
      1. 预制体加载与编辑：
      • GameObject prefabContents = PrefabUtility.LoadPrefabContents("Assets/Prefabs/MyPrefab.prefab");
      • 将预制体加载到内存中进行编辑，返回可修改的 GameObject；
      • 注意必须与 UnloadPrefabContents 配对使用以避免内存泄漏；
      1. 预制体卸载：
      • PrefabUtility.UnloadPrefabContents(prefabContents);
      • 释放由 LoadPrefabContents 加载的预制体内存；
      1. 预制体的实例化：
      • GameObject instance = PrefabUtility.InstantiatePrefab(prefabAsset) as GameObject;
      • 实例化预制体资源，其相比于 Object.Instantiate，可以正确的处理预制体连接关系，- 支持后续的预制体更新传播；
      1. 预制体的修改：
      • 将实例上的修改应用回原始预制体；
