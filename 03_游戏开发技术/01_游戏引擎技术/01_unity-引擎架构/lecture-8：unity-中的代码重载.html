代码重载流程

AppDomain 基础概念

概念：AppDomain 基础概念

  • 概念：
    • AppDomain 是 CLR（公共语言运行时）中提供的一种隔离机制，用于在同一个进程中运行多个 .NET 应用程序，每个 AppDomain 都有自己的虚拟地址空间、线程和对象，实现了进程内应用程序的隔离；
    • 具体而言，每个 AppDomain 维护自己的加载器堆，用于存储加载的程序集、类型系统等元数据。跨不同的 App Domain 对象访问必须通过明确的跨域通信机制（如代理或序列化），同时每个 AppDomain 可以配置不同的代码访问安全策略；
  • 概念：进程隔离机制
    • AppDomain 被设计为一种轻量级的"逻辑进程"，与操作系统进程相比具有以下特点：
    • （1）创建和销毁 AppDomain 的开销远小于操作系统进程；
    • （2）每个 AppDomain 有独立的内存空间和资源，一个 AppDomain 中的异常不会影响其他 AppDomain；
    • （3）多个 AppDomain 共享同一个 CLR 实例和操作系统进程资源；
面试题：解释 AppDomain 在 Unity 代码重载中的作用，并说明为什么 Unity 需要使用域重载机制；
  • AppDomain（应用程序域）是 .NET CLR 提供的隔离机制，允许在同一个进程内运行多个应用程序，每个域拥有独立的虚拟地址空间、加载的程序集和静态变量状态；在 Unity 编辑器中，脚本域用于隔离编辑时和运行时的代码执行环境；
  • Unity 使用域重载的主要目的是确保脚本状态重置：当开发者修改代码并重新编译时，Unity 会销毁当前 AppDomain 并创建一个新域，重新加载所有程序集，从而清除所有静态字段和注册事件等残留状态，避免旧代码影响新执行环境；这保证了进入播放模式时，项目状态与真实构建版本一致；
  • 域重载的触发场景包括：脚本修改后编译、进入播放模式时启用域重载选项；其过程包括卸载旧域、保留序列化字段值、加载新程序集并重建对象层次；

播放模式进入流程

概念：播放模式进入流程

  • 概念：
    • 在编辑器中进入播放模式时，Unity 会执行两个重要的操作，以确保您的项目在编辑器中启动时与构建后一样：
    • （1）重置脚本状态。这也被称为“域重载”；
    • （2）重载当前场景；
    • 在开发过程中，快速迭代时间很重要，因为重置场景和脚本状态所需的时间可能会成为障碍，所以 Unity 提供了配置进入播放模式时发生情况的功能；
    • 可以选择禁用域重载和场景重载操作，这些选项由 可配置的进入播放模式 功能提供；

Play Mode 加载配置

Domain Reload 配置

概念：Domain Reload 配置

  • 概念：为什么需要 Domain Reload 配置
    • 域重新加载会重置您的脚本状态，并且默认启用。它提供了一个全新的脚本状态，并在每次进入播放模式时重置所有静态字段和已注册的处理程序；
    • 这意味着每次在 Unity 编辑器中进入播放模式时，项目都以非常类似于第一次在构建中启动时的方式开始播放；
    • 域重新加载需要时间，并且此时间会随着项目中脚本的数量和复杂性而增加。当进入播放模式需要很长时间时，快速迭代项目变得更加困难，因此 Unity 提供关闭域重新加载选项；
  • 概念：禁用域重新加载
      1. 点击 Edit > Project Setting 打开项目设置窗口；
      1. 选择 Editor；
      1. 在 When Entering Play Mode 中选择要配置的模式；

分析：禁用域重新加载时的静态字段重置

  • 分析：
    • 禁用域重新加载后，代码中静态字段的值不会自动重置为其原始值，因此需要添加明确执行此操作的代码；
    • 为了确保即使在禁用域重新加载时计数器也能重置，必须使用此特性：
      • [RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.SubsystemRegistration)]
    • 使用此特性，可以显式重置目标静态字段的值；
  • 示例：静态字段
using UnityEngine;

public class StaticCounterExampleFixed : MonoBehaviour
{
    static int counter = 0;

	// 显式重置目标静态字段的值
    [RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.SubsystemRegistration)]
    static void Init()
    {
		Debug.Log("Counter reset.");
		counter = 0;   
    }

    void Update()
    {
		if (Input.GetButtonDown("Jump"))
		{
			counter++;
			Debug.Log("Counter: " + counter);
		}
    }
}
  • 示例：静态事件
using UnityEngine;
public class StaticEventExampleFixed : MonoBehaviour
{
    [RuntimeInitializeOnLoadMethod]
    static void RunOnStart()
    {
		Debug.Log("Unregistering quit function");
		Application.quitting -= Quit;
    }

    void Start()
    {
		Debug.Log("Registering quit function");
		Application.quitting += Quit;
    }

    static void Quit()
    {
        Debug.Log("Quitting the Player");
    }
}

场景重新加载配置

概念：场景重新加载配置

  • 概念：场景重新加载
    • 默认情况下，场景重新加载已启用，这意味着当进入播放模式时，Unity 会销毁所有现有的场景游戏对象，并从磁盘重新加载场景；
    • Unity 执行此操作所需的时间会随着场景复杂度的增加而增加，这意味着随着项目变得越来越复杂，必须在按下“播放”按钮后等待更长的时间才能使场景在编辑器中完全加载；
  • 概念：禁用场景重新加载
    • 当禁用场景重新加载时，此过程将花费更少的时间。这能够更快速地迭代项目的开发，Unity 不会从磁盘重新加载场景，而是只重置场景中已修改的内容。这避免了卸载和重新加载场景的时间和性能影响，Unity 仍然会调用相同的初始化函数，例如 OnEnable、OnDisable 和 OnDestroy，就好像它是新加载的一样；
  • 分析：跳过场景重新加载的影响
    • 禁用场景重新加载对项目的副作用应该很小。但是，由于场景重新加载与域重新加载紧密相关，因此存在一些重要的差异；
    • （1）在播放模式下返回编辑模式时，未序列化的 ScriptableObject 和 MonoBehaviour 字段会保留分配给它们的值，这是因为 Unity 不会重新创建现有对象或调用构造函数；
    • （2）Unity 在域重新加载期间将数组/列表类型的空私有和内部字段转换为空数组/列表对象，并且它们在运行时（非编辑器）脚本保持非空；
    • （3）使用 ExecuteInEditMode 或 ExecuteAlways 的脚本不会接收 OnDestroy 或 Awake 调用。Unity 不会调用 Awake，并且仅在 EditorApplication.isPlaying 已经是真值的情况下，才会在播放模式更改时调用 OnEnable，其中 Awake/OnEnable 方法会检查 EditorApplication.isPlaying 属性；

相关概念

当前脚本域

概念：当前脚本域 - Current Scripting Domain

  • 概念：脚本域
    • Unity 基于 .NET 的 AppDomain 概念实现的隔离环境，用于加载和执行程序集；
    • 当前脚本域，指的是脚本重载前正在运行的 AppDomain，包含所有已加载的程序集、静态变量和运行时对象引用。在域重载时，该域会被完整卸载以清除旧状态；
  • 概念：Unity 中的域
    • Unity 启动时创建主 AppDomain，加载 Assembly-CSharp 和 Assembly-CSharp-Editor；
    • 编辑器代码（Assembly-CSharp-Editor）仅在 Unity 编辑器的 AppDomain 中运行，不会污染玩家设备的运行时环境；
  • 分析：卸载域
    • Unity 会销毁当前域中的所有托管对象（如 MonoBehaviour 实例），但保留序列化字段（public 或 [SerializeField]）的值，以便重载后恢复；
    • 静态变量和未序列化的动态数据会被重置，需通过 [DidReloadScripts] 手动恢复；
  • 分析：域重载
    • 新创建的脚本域会重新加载所有程序集（包括新编译的程序集），并重建场景中的对象层次结构；
    • 序列化字段的值从旧域复制到新域，确保对象状态连续性；

分析：完整的修改脚本流程

    1. 开发者修改 PlayerMovement.cs 并保存；
    1. Unity 检测变更，重新编译 PlayerMovement.cs 及其依赖，生成新的 Assembly-CSharp.dll；
    1. 当前脚本域被销毁，但保留 PlayerMovement 组件的 public float speed 字段值；
    1. 新域加载新程序集，并重建场景中的 Player 对象，恢复 speed 值；
    1. [DidReloadScripts] 标记的方法被调用，重新初始化静态资源；

DidReloadScripts

概念：DidReloadScripts

  • 概念：
    • 在 Unity 中，[DidReloadScripts] 是一个编辑器专用的回调属性（Attribute），用于在脚本重新编译后自动触发指定的方法；
    • 它常用于编辑器扩展开发，优化工作流程并处理脚本重载后的数据恢复或初始化；
  • 概念：自动触发时机
    • 当 Unity 完成：
        1. 脚本的重新编译（如修改脚本后保存、导入新脚本等）；
        1. 重载脚本域（Domain Reload）
    • 完成上述两步后，标记了 [DidReloadScripts] 的方法会被自动调用；
    • 适用于需要在脚本重新加载后执行初始化或状态恢复的场景；
  • 分析：与类似回调的区别
      1. InitializeOnLoad：在编辑器启动或脚本编译完成后触发，但仅限静态构造函数；
      1. InitializeOnLoadMethod：功能类似 [DidReloadScripts]，但更通用（不限于脚本重载场景）
      1. DidReloadScripts：专为脚本重载后的逻辑设计，触发时机更精确；
  • 分析：实现原理
      1. 检测脚本变更并触发编译；
      1. 生成程序集后卸载当前脚本域（Domain Unload）
      1. 重新加载新程序集（Domain Reload）
      1. 调用标记了 [DidReloadScripts] 的方法；
    • 在域重载过程中，Unity 会序列化所有 MonoBehaviour 和 ScriptableObject 的公共字段及标记了 [SerializeField] 的私有字段，[DidReloadScripts] 方法通常用于处理非序列化的动态数据（如静态变量、运行时生成的资源）

分析：动态获得运行时类型

分析：动态获得运行时类型

  • 代码：
// 1. 获得当前AppDomain下的所有程序集
Assembly[] assemblies = AppDomain.CurrentDomain.GetAssemblies();

// 2. 获得当前项目下的Assembly-CSharp程序集
Assembly defaultAssembly = assemblies.First(assembly => assembly.GetName().Name == "Assembly-CSharp");

// 3. 利用反射，得到以“selectionObject.name”为名称的类型
var type = defaultAssembly.GetType(generateClassName);
  • 分析：Assembly[] assemblies = AppDomain.CurrentDomain.GetAssemblies();
    • 获取当前应用程序域（AppDomain）中已加载的所有程序集；
    • 在 Unity 编辑器中，这包括：
        1. 核心程序集，如 UnityEngine.dll、UnityEngine.CoreModule.dll；
        1. 项目程序集，如 Assembly-CSharp 和 Assembly-CSharp-Editor；
        1. 第三方插件程序集，如 Newtonsoft.Json.dll；
    • AppDomain.CurrentDomain：获取当前脚本运行的应用程序域，GetAssemblies() 返回域中所有已加载程序集的数组，用于后续筛选操作；
  • 分析：Assembly defaultAssembly = assemblies.First(assembly => assembly.GetName().Name == "Assembly-CSharp");
    • 从所有程序集中筛选出名为 Assembly-CSharp 的程序集（即项目默认的运行时代码程序集）；
    • First()：使用 LINQ 方法返回第一个匹配的程序集，若未找到会抛出异常；
    • assembly.GetName().Name：获取程序集的名称（如 "Assembly-CSharp"）
  • 分析：var type = defaultAssembly.GetType(generateClassName);
    • 通过反射从 Assembly-CSharp 程序集中获取指定类名的 Type 对象，GetType(string name) 在程序集中查找完全匹配的类；

完整代码

using System.IO;
using System.Linq;
using UnityEngine;
using UnityEditor;

namespace EditorExtension
{
    public class CreateComponentCode : MonoBehaviour
    {
        // 如果不将优先级设置为最高，则当前“EditorExtention-CreateCode”不会在右键的时候显示出来
        [MenuItem("Component/@EditorExtention-CreateCode",false,0)]
        static void CreateCode()
        {
            var selectionObject = Selection.objects.First();
            
            if (selectionObject == null)
            {
                Debug.LogWarning("Selection object is null");
            }
 
            string FolderPath = Application.dataPath + "/Scripts";
            
            if (Directory.Exists(FolderPath) == false) 
            {
                Directory.CreateDirectory(FolderPath);
            }
            
            string scriptsFile = FolderPath + $"/{selectionObject.name}.cs";
            
            // 创建一个文件之后，如果不刷新Unity Editor的话，不会在Project中显示文件
            StreamWriter fileStream = File.CreateText(scriptsFile);
            fileStream.Write(@"
using UnityEngine;
public class {0} : MonoBehaviour
{{    
}}
",selectionObject.name);
            fileStream.Close();
            
            // 对Asset进行刷新
            AssetDatabase.Refresh();
        }
    }   
}
