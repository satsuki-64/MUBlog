Lecture 3：Command Buffer

CommandBuffer 基础

CommandBuffer 基础概念

概念：CommandBuffer 基础概念

  • 概念：
    • CommandBuffer 是 Unity 中用于预录制渲染命令的容器，它允许开发者将多个 GPU 指令批量组织，然后在特定渲染阶段执行。这个清单是交给 GPU 去执行的，包含了一系列具体的图形指令，例如“设置渲染目标”、“清除屏幕”、“绘制这个网格”、“将这张纹理复制到另一张”等等；
    • 与传统的即时模式渲染不同，你首先在 CPU 端创建并填充这个命令列表，然后选择在渲染流程的某个特定时机点（例如，在渲染不透明物体之后，在绘制天空盒之前），将这个完整的列表提交给 GPU；
    • 这意味着你不再完全受限于 Unity 内置的渲染顺序，可以自由地插入自定义的绘制调用、后处理效果、管理复杂的渲染目标，比如下图所示，绿色点的部分都是可以插入 Command Buffer 地方；
    • 从而实现诸如自定义阴影、高级贴花、复杂模糊特效等超出标准管线功能的效果；
  • 图示：绿色部分为可以插入 CommandBuffer 的位置
    • 分为前向渲染和延迟渲染两种；
  • 特点：
      1. 延迟执行：命令被录制后不会立即执行，而是在需要时提交给 GPU；
      1. 精确时序控制：可以绑定到渲染管线的具体位置；
      1. 多线程支持：支持在多线程环境中构建命令，提高 CPU 利用率；
  • 分析：
    • 与传统在 MonoBehaviour 的 Update 或 OnRenderObject 中即时发布渲染命令的方式不同，CommandBuffer 的核心思想是 “延迟执行”​ 和 “批量提交”；
    • 性能角度看，它将多个离散的渲染调用打包成一个批次提交，有效减少了 CPU 与 GPU 之间的通信开销和状态切换，对于需要大量重复绘制操作的场景（如绘制大量辅助线或特效）性能提升尤为显著；
  • 概念：CommandBuffer 的实例化与释放
    • 实例化：CommandBuffer cmd = new CommandBuffer();
    • 一个好的习惯是给它命名以便调试：cmd.name = "MyCustomEffect";
    • 使用 new CommandBuffer() 创建的资源，在使用完毕后，必须调用 cmd.Release() 来释放其占用的原生资源；
总结：
  • CommandBuffer 提供精确时序控制，Compute Shader 提供并行计算能力，GraphicsBuffer 提供高效数据通道；

最简 CommandBuffer 代码示例

代码：最简 CommandBuffer 代码示例

  • 概念：Compute Shader
// 定义内核函数名称
#pragma kernel RedTint

// 声明可读写的输出纹理
RWTexture2D<float4> Result;

// 定义线程组布局：8x8x1 的线程
[numthreads(8,8,1)]
void RedTint (uint3 id : SV_DispatchThreadID)
{
    // 每个线程将对应的像素设置为红色
    Result[id.xy] = float4(1.0, 0.0, 0.0, 1.0);
}
  • 代码：
using UnityEngine;
using UnityEngine.Rendering;

public class ComputeRedTint : MonoBehaviour
{
    public ComputeShader computeShader; // 在Inspector中分配RedTint.compute
    public RenderTexture outputTexture; // 用于显示结果的纹理
    
    private CommandBuffer commandBuffer;
    private int kernelIndex;
    
    void Start()
    {
        // 创建CommandBuffer并命名以便调试
        commandBuffer = new CommandBuffer();
        commandBuffer.name = "ComputeRedTint";
        
        // 获取Compute Shader中的内核函数索引
        kernelIndex = computeShader.FindKernel("RedTint");
        
        // 创建输出纹理（512x512），启用随机写入
        outputTexture = new RenderTexture(512, 512, 0, RenderTextureFormat.ARGB32);
        outputTexture.enableRandomWrite = true;
        outputTexture.Create();
        
        // 将纹理设置为材质的显示纹理
        GetComponent<Renderer>().material.mainTexture = outputTexture;
        
        // 执行计算
        ExecuteComputeShader();
    }
    
    void ExecuteComputeShader()
    {
        // 1. 将输出纹理设置到Compute Shader
        commandBuffer.SetComputeTextureParam(computeShader, kernelIndex, "Result", outputTexture);
        
        // 2. 计算线程组数量（确保覆盖整个纹理）
        uint threadGroupSizeX, threadGroupSizeY, threadGroupSizeZ;
        computeShader.GetKernelThreadGroupSizes(kernelIndex, out threadGroupSizeX, out threadGroupSizeY, out threadGroupSizeZ);
        
        int threadGroupsX = Mathf.CeilToInt(outputTexture.width / (float)threadGroupSizeX);
        int threadGroupsY = Mathf.CeilToInt(outputTexture.height / (float)threadGroupSizeY);
        
        // 3. 调度Compute Shader执行
        commandBuffer.DispatchCompute(computeShader, kernelIndex, threadGroupsX, threadGroupsY, 1);
        
        // 4. 立即执行CommandBuffer中的所有命令
        Graphics.ExecuteCommandBuffer(commandBuffer);
        
        Debug.Log($"调度的线程组: {threadGroupsX}x{threadGroupsY}, 每个线程组: {threadGroupSizeX}x{threadGroupSizeY} 线程");
    }
    
    void OnDestroy()
    {
        // 清理资源
        commandBuffer?.Dispose();
        if (outputTexture != null)
        {
            outputTexture.Release();
            Destroy(outputTexture);
        }
    }
}

CommandBuffer 设置变量

SetBuffer - 用于 XXXBuffer 对象

概念：CommandBuffer 设置变量相关方法 - SetBuffer

  • 概念：
    • SetBuffer 方法用于将一个特定的缓冲区对象，如 ComputeBuffer 或 GraphicsBuffer，与着色器中的指定属性进行绑定；
    • 其核心功能是建立 CPU 端数据容器与 GPU 端着色器程序之间的数据通道，使得大量结构化数据（如顶点数组、变换矩阵数组、粒子属性等）能够被着色器高效访问；
  • 参数：
    • 参数一：nameID
      • 整数类型，表示着色器中缓冲区属性的唯一标识符；
      • 通常通过 Shader.PropertyToID(string propertyName) 方法预先获取，使用 ID 比直接传递属性名字符串在性能上更优；
    • 参数二：value
      • 要绑定的缓冲区对象，类型为 ComputeBuffer 或 GraphicsBuffer；
      • 该缓冲区应已在之前被创建并填充了所需的数据；
  • 示例：
internal GraphicsBuffer m_GpuChunks;
internal GraphicsBuffer m_GpuView;
internal GraphicsBuffer m_GpuSortKeys;

internal static class Props  
{
	public static readonly int SplatViewData = Shader.PropertyToID("_SplatViewData");
	public static readonly int SplatChunks = Shader.PropertyToID("_SplatChunks");
	public static readonly int OrderBuffer = Shader.PropertyToID("_OrderBuffer");
}

void InitSortBuffers(int count)

    m_GpuSortKeys?.Dispose();    
    m_GpuSortKeys = new GraphicsBuffer(GraphicsBuffer.Target.Structured, count, 4) { name = "GaussianSplatSortIndices" };  
  
    // init keys buffer to splat indices  
    m_CSSplatUtilities.SetBuffer
    (
	    (int)KernelIndices.SetIndices, 
	    Props.SplatSortKeys, 
	    m_GpuSortKeys
	);   
}

public Material SortAndRenderSplats(Camera cam, CommandBuffer cmb)  
{
	mpb.SetBuffer(GaussianSplatRenderer.Props.SplatChunks, gs.m_GpuChunks);  
	mpb.SetBuffer(GaussianSplatRenderer.Props.SplatViewData, gs.m_GpuView);  
	mpb.SetBuffer(GaussianSplatRenderer.Props.OrderBuffer, gs.m_GpuSortKeys);
}

SetFloat / SetInteger - 用于所有着色器

概念：SetFloat / SetInteger

  • 概念：
    • SetFloat 和 SetInteger 方法用于为所有着色器设置一个全局的浮点数或整数值；
    • 它们的主要作用是在渲染过程中动态地向着色器传递标量参数，从而控制渲染效果，例如调整颜色强度、切换混合模式、控制纹理平铺次数等；
    • 在底层实现上，SetGlobalInt 是 SetGlobalFloat 的别名，两者在处理上基本一致。但对于着色器中明确声明为 Integer 类型的属性，建议使用 SetGlobalInteger 方法以确保类型的精确匹配；
  • 参数：
    • 参数一：nameID
      • 整数类型，表示着色器中缓冲区属性的唯一标识符；
      • 通常通过 Shader.PropertyToID(string propertyName) 方法预先获取，使用 ID 比直接传递属性名字符串在性能上更优；
    • 参数二：value
      • 要传递的浮点或整数值；
  • 代码：
CommandBuffer cmd = new CommandBuffer();
cmd.name = "Set Shader Parameters";

// 设置一个控制波纹扭曲强度的浮点数
int distortionStrengthID = Shader.PropertyToID("_DistortionStrength");
cmd.SetGlobalFloat(distortionStrengthID, 0.5f);

// 设置一个整数用于在着色器中切换不同后处理模式（例如1=黑白，2=复古，3=冷色调）
int effectModeID = Shader.PropertyToID("_EffectMode");
cmd.SetGlobalInteger(effectModeID, 2);

// 执行一个全屏后处理Blit操作，上述参数将在材质中被使用
cmd.Blit(sourceTexture, destTexture, postProcessMaterial);

SetComputeXXXParam - 用于全局计算着色器

概念：SetComputeXXXParam

  • 概念：
    • 此方法的目标是设置整个 Compute Shader 的全局变量，参数类型包括基本数据类型，比如 int, float, Matrix4x4, Vector4 等；
    • 其全局共享，无需指定内核索引，通常用于传递常量、配置参数，不需要指定计算着色器中的内核函数索引；
  • 定义：
    • public void SetComputeIntParam([NotNull] ComputeShader computeShader, int nameID, int val)
  • 示例：
internal void SortPoints(CommandBuffer cmd, Camera cam, Matrix4x4 matrix)  
{
	// 指定位置数据的编码格式，对应 Shader.PropertyToID("_SplatFormat"); 格式为 VectorFormat
	cmd.SetComputeIntParam(m_CSSplatUtilities, Props.SplatFormat, (int)m_Asset.posFormat);
	
	// 指定 本地 -> 世界 -> 相机 空间变换的矩阵
	cmd.SetComputeMatrixParam(m_CSSplatUtilities, Props.MatrixMV, worldToCamMatrix * matrix);
	
	// 指定 splat 的数量
	cmd.SetComputeIntParam(m_CSSplatUtilities, Props.SplatCount, m_SplatCount);
	
	// 指定分块数量
	cmd.SetComputeIntParam(m_CSSplatUtilities, Props.SplatChunkCount, m_GpuChunksValid ? m_GpuChunks.count : 0);
}

SetComputeMatrixParam - 用于全局计算着色器

概念：SetComputeMatrixParam

  • 概念：CommandBuffer.SetComputeMatrixParam
    • CommandBuffer.SetComputeMatrixParam 函数的主要作用，是将一个 4x4 的矩阵值从 C# 脚本端设置到 GPU 的计算着色器当中指定的全局参数上；
    • 可以通过两种方式来指定目标参数：
        1. 使用参数字符串名：直接传入着色器中定义的变量名；
        1. 使用属性名称 ID：通过 Shader.PropertyToID 获取的整型 ID，这种方式性能通常更好，尤其是在频繁调用时；
    • 并且在一个计算着色器资源内部，常量缓冲区（Constant Buffer）是所有 Kernel 之间共享的，当你使用此函数设置了一个矩阵参数后，该着色器中的所有内核都能访问到更新后的这个矩阵值；
  • 方法：
      1. 实例化一个 CommandBuffer；
      1. 使用 SetComputeMatrixParam 方法将计算好的矩阵（如变换矩阵、视图投影矩阵等）添加到命令缓冲区中；
      1. 通过 CommandBuffer.DispatchCompute 方法指定执行的计算着色器内核和线程组数量；
      1. 最终将填充好的命令缓冲区添加到 ScriptableRenderContext 中并提交执行；
  • 代码：
// ... 在C#脚本中 ...
// 1. 创建命令缓冲区
CommandBuffer cmd = new CommandBuffer();
cmd.name = "Matrix Setup and Dispatch";

// 2. 定义计算着色器和矩阵参数
public ComputeShader computeShader;
private Matrix4x4 worldViewProjectionMatrix;
private int kernelHandle;
private int matrixPropertyID;

void Start() 
{
    kernelHandle = computeShader.FindKernel("CSMain");
    // 使用 PropertyID 提高效率
    matrixPropertyID = Shader.PropertyToID("_WorldViewProjMatrix"); 
}

void Update() 
{
    // 更新矩阵（例如，每帧根据相机和物体变换计算）
    worldViewProjectionMatrix = ... ; 

    cmd.Clear();
    
    // 将矩阵参数添加到命令缓冲区
    // 参数一：指定 Shader
    // 参数二：指定要更改的全局属性 ID
    // 参数三：指定要更改的值
    cmd.SetComputeMatrixParam(computeShader, matrixPropertyID, worldViewProjectionMatrix);
    
    // 根据数据大小调度计算着色器
    cmd.DispatchCompute(computeShader, kernelHandle, threadGroupsX, threadGroupsY, threadGroupsZ);
    
    // 将命令缓冲区加入渲染上下文（在SRP中）
    // Graphics.ExecuteCommandBuffer(cmd); // 立即执行方式示例
}

SetComputeBufferParam - 用于内核函数

概念：SetComputeBufferParam

  • 概念：
    • SetComputeBufferParam 的核心作用让你能够在 CPU 端准备数据，通过 CommandBuffer 将这个数据缓冲区精准地传递给 GPU 端的某个计算着色器的特定内核函数；
    • 计算着色器中的每个内核函数（Kernel）都可以有自己独立的资源绑定槽位（Resource Binding Slots），这意味着，不同的内核可能会期望不同结构或用途的缓冲区；
    • 通过指定内核索引，Unity 能够确保缓冲区被正确地连接到该内核所期望的特定输入/输出槽位上；
  • 参数：
    • 参数一：computeShader
      • 指定要设置参数的计算着色器资源；
    • 参数二：内核
      • 指定计算着色器中某个特定内核（Kernel）的索引，可通过 ComputeShader.FindKernel 获取；
    • 参数三：nameID
      • 缓冲区变量在着色器中的唯一整数 ID；
      • 推荐使用 Shader.PropertyToID("BufferName") 获取，性能优于字符串；
    • 参数四：buffer
      • 要传递给计算着色器的实际缓冲区对象，其中包含了计算所需的数据；
  • 分析：和 SetBuffer 的区别
    • SetBuffer 设置的是一个全局着色器属性。一旦设置，所有支持该全局属性的着色器都能访问到它；
    • SetComputeXXXParam 用于全局计算着色器，一旦设置，指定的计算着色器当中的所有内核函数都可以访问它；
    • SetComputeBufferParam 作用范围是特定计算着色器的特定内核，是局部的、精确的配置；
  • 代码：C# 
// ... 在C#脚本中 ...
// 1. 准备计算着色器和数据
public ComputeShader computeShader;
private ComputeBuffer particleDataBuffer;
private int kernelHandle;
private int bufferPropertyID;

void Start() 
{
    // 查找内核
    kernelHandle = computeShader.FindKernel("CSMain");
    
    // 获取缓冲区属性的整数ID（优化性能）
    bufferPropertyID = Shader.PropertyToID("_ParticleData");
    
    // 创建并填充缓冲区（假设每个粒子有4个float的数据）
    particleDataBuffer = new ComputeBuffer(particleCount, sizeof(float) * 4);
    
    // ... 为 particleDataBuffer 填充数据 ...
}

void Update() 
{
    CommandBuffer cmd = new CommandBuffer();
    cmd.name = "Dispatch Particle Simulation";

    // 2. 关键步骤：将缓冲区与计算着色器内核中的特定属性绑定
    cmd.SetComputeBufferParam(computeShader, kernelHandle, bufferPropertyID, particleDataBuffer);

    // 3. 分派计算任务
    cmd.DispatchCompute(computeShader, kernelHandle, threadGroupsX, threadGroupsY, threadGroupsZ);

    // 4. 执行命令缓冲区（例如，立即执行）
    Graphics.ExecuteCommandBuffer(cmd);
    // 或添加到相机的渲染事件中：Camera.main.AddCommandBuffer(...);
}

调用 Command Buffer 执行命令

CommandBuffer 的执行

两种方法

概念：CommandBuffer 执行

  • 概念：
      1. 关联到相机事件
      • 使用 Camera.AddCommandBuffer(CameraEvent event, CommandBuffer buffer) 将 CommandBuffer 绑定到相机渲染流程的特定节点，Unity 会在渲染该相机的过程中，在指定节点自动执行这些命令；
      1. 立即执行
      • 使用 Graphics.ExecuteCommandBuffer(cmd) 立即在当前帧提交并执行命令；
  • 示例：使用方法一，将 Command Buffer 关联相机
public CommandBuffer InitialClearCmdBuffer(Camera cam)
{
	// 1. 新建 Command Buffer 并指定名称
	m_CommandBuffer ??= new CommandBuffer {name = "RenderGaussianSplats"};
	
	if (GraphicsSettings.currentRenderPipeline == null && cam != null && !m_CameraCommandBuffersDone.Contains(cam))
	{
		// 2. 使用 AddCommandBuffer 方法，将当前 CommandBuffer 指定到相机上
		// 参数一用于指定当前的渲染命令的时机，这里指定是在 当前前向渲染透明物体之前
		cam.AddCommandBuffer(CameraEvent.BeforeForwardAlpha, m_CommandBuffer);
		m_CameraCommandBuffersDone.Add(cam);
	}

	// get render target for all splats
	m_CommandBuffer.Clear();
	return m_CommandBuffer;
}
  • 示例二：使用方法二，立刻执行
using CommandBuffer cmb = new CommandBuffer();

SetAssetDataOnCS(cmb, KernelIndices.UpdateEditData);

cmb.SetComputeBufferParam(m_CSSplatUtilities, (int)KernelIndices.UpdateEditData, Props.DstBuffer, m_GpuEditCountsBounds);

cmb.SetComputeIntParam(m_CSSplatUtilities, Props.BufferSize, m_GpuEditSelected.count);

m_CSSplatUtilities.GetKernelThreadGroupSizes((int)KernelIndices.UpdateEditData, out uint gsX, out _, out _);

cmb.DispatchCompute(m_CSSplatUtilities, (int)KernelIndices.UpdateEditData, (int)((m_GpuEditSelected.count+gsX-1)/gsX), 1, 1);

Graphics.ExecuteCommandBuffer(cmb);

AddCommandBuffer 方法

概念：AddCommandBuffer 方法

  • 概念：
    • AddCommandBuffer 方法允许你将自定义的渲染命令列表（Command Buffer）精确地插入到官方的渲染流程中，从而实现强大的自定义渲染效果和优化；
    • AddCommandBuffer 可以用于 Camera 或者 Light 组件，因此其方法主要分为针对相机的自定义操作、以及针对灯光的自定义操作；
  • 概念：Camera.AddCommandBuffer
    • 目的：
      • 特定的 Camera 组件，在相机渲染流程的特定阶段插入自定义操作，如后处理、遮挡渲染等；
    • 参数：
      • 参数一：CameraEvent
      • 参数二：Command Buffer
  • 概念：CameraEvent
    • CameraEvent 枚举定义了相机渲染管线中的各个关键时间点，它决定了你的 Command Buffer 在何时被执行；
      1. BeforeForwardOpaque：
      • 在渲染不透明物体之前执行；
      • 这是最常用的插入点之一，适合用于绘制深度图、遮挡物，或者为后续渲染准备自定义的 G-Buffer 等操作；
      1. AfterForwardOpaque：
      • 在渲染不透明物体之后、天空盒之前执行；
      • 非常适合用于处理屏幕空间效果（SSR, SSAO）、或者在不透明物体之上绘制一些自定义的遮罩；
      1. BeforeSkybox：在天空盒渲染之前执行；
      1. AfterSkybox：在天空盒渲染之后、透明物体之前执行；
      1. BeforeImageEffects：在 Unity 的后处理效果（Image Effects）应用之前执行；
      • 如果你自定义的后处理效果需要基于当前完整的场景画面（包含透明物体），并希望在 Unity 官方后处理之前应用，就应选择此点；
      1. BeforeForwardAlpha：在渲染透明物体之前执行
  • 代码：
// 初始化并返回一个用于高斯溅落渲染的命令缓冲区
public CommandBuffer InitialClearCmdBuffer(Camera cam)
{
    // 如果 m_CommandBuffer为null 则创建新实例，否则使用现有实例
    m_CommandBuffer ??= new CommandBuffer { name = "RenderGaussianSplats" };
    
    // 检查是否为内置渲染管线 (Built-in Render Pipeline) 且相机有效且未被处理过
    // GraphicsSettings.currentRenderPipeline == null 表示当前使用内置渲染管线
    // cam != null 确保传入的相机引用有效
    // !m_CameraCommandBuffersDone.Contains(cam) 防止对同一相机重复添加命令缓冲区
    if (GraphicsSettings.currentRenderPipeline == null && cam != null && !m_CameraCommandBuffersDone.Contains(cam))
    {
        // 将命令缓冲区添加到相机的渲染流程中
        // CameraEvent.BeforeForwardAlpha：在渲染透明物体之前执行，确保高斯溅落在不透明物体之后、标准透明物体之前渲染
        cam.AddCommandBuffer(CameraEvent.BeforeForwardAlpha, m_CommandBuffer);
        
        // 将相机标记为"已处理"，避免重复添加命令缓冲区
        // 这对于多相机场景尤其重要，确保每个相机只被设置一次
        m_CameraCommandBuffersDone.Add(cam);
    }

    // 清空命令缓冲区中的所有先前命令
    // 这确保每次调用此方法时都从一个干净的状态开始
    // 对于需要每帧更新的动态渲染内容非常重要
    m_CommandBuffer.Clear();
    
    // 返回准备好的命令缓冲区供外部填充具体的渲染命令
    return m_CommandBuffer;
}

ExecuteCommandBuffer 方法

概念：ExecuteCommandBuffer 方法

  • 概念：

void DispatchUtilsAndExecute(CommandBuffer cmb, KernelIndices kernel, int count)
{
	// 先获得当前 Compute Shader 当中某个 函数的 线程模型参数 gsX
	m_CSSplatUtilities.GetKernelThreadGroupSizes((int)kernel, out uint gsX, out _, out _);
	
	// 为 CommandBuffer 添加 SplatUtilities 这个计算Shader当中的计算任务
	cmb.DispatchCompute(m_CSSplatUtilities, (int)kernel, (int)((count + gsX - 1)/gsX), 1, 1);

	Graphics.ExecuteCommandBuffer(cmb);
}

DrawProcedural

概念：DrawProcedural

  • 概念：
    • DrawProcedural 与传统渲染流程需要先在内存中准备好顶点数据再上传至 GPU 不同，它利用 Shader Model 4.5+ 硬件特性，允许着色器通过系统内置的值，如 SV_VertexID 和 SV_InstanceID，直接从自定义的 GPU 缓冲区（如 ComputeBuffer）中读取数据，从而动态生成顶点信息；
    • 这种方法特别适用于渲染由大量简单、规则或由计算着色器动态生成的几何体，例如粒子系统、海量草叶、体素、点云以及全屏后处理效果；
  • 参数：
    • 参数一：缓冲区
      • 可以为 Graphic Buffer 或其他的缓冲区数据；
      • 是一个可选参数，它用于指定一个可选的索引缓冲区；
      • 当提供时，渲染将使用该索引缓冲区来引用顶点。这允许重用顶点数据，从而节省内存；
      • 在程序化绘制中，如果几何体结构复杂，例如，多个三角形共享顶点，使用索引缓冲区会非常高效。如果不需要，可以传入 null；
    • 参数二：变换矩阵 (matrix)
      • 此矩阵会将着色器中生成的顶点位置进行变换；
      • 比如传入：matrix = gs.transform.localToWorldMatrix;
    • 参数三：材质
      • 所使用的着色器必须能够处理程序化绘制，这意味着顶点着色器不应依赖传统的顶点输入（如 POSITION, NORMAL 语义），而应主要依靠 SV_VertexID（顶点 ID）和 SV_InstanceID（实例 ID）来定位需要读取的数据；
    • 参数四：shaderPass
      • 指定使用材质着色器中的哪一个通道；
      • Unity 的着色器可以包含多个通道，以应对不同的渲染状态，通常，默认主通道的索引是 0。如果传入 -1，则会按顺序激活并执行着色器中的所有通道；
    • 参数五：topology
      • 定义如何将顶点组装成基本图元。常见的类型有：
      • 比如使用 MeshTopology topology = MeshTopology.Triangles; 来定义其拼成三角形；
    • 参数六：索引数量
      • 要渲染的索引数量；
      • 如果提供了 indexBuffer，这个值应等于索引缓冲区中的索引数。如果没有提供 indexBuffer，这个值就直接等于要渲染的顶点数量。例如，用三角形拓扑绘制一个四边形（由两个三角形组成），需要 6 个索引（或顶点）
    • 参数七：实例数量 - indexCount
      • 指定要绘制的几何体实例的数量；
      • 实例化允许在一次绘制调用中渲染多个相同的几何体，极大提升性能；
      • 在着色器中，可以通过 SV_InstanceID 来区分每个实例，从而为每个实例设置不同的属性（如位置、颜色、大小等）
    • 参数八：材质属性块 - MaterialPropertyBlock
      • 一个可选参数；
      • 允许您在渲染时传递额外的材质属性，而无需创建新的材质实例。这对于优化至关重要，可以使用它来为每次绘制调用设置不同的参数（如颜色、纹理、浮点数等），即使它们使用同一个材质球，避免了材质实例化带来的开销；
  • 代码：
public Material SortAndRenderSplats(Camera cam, CommandBuffer cmb)  
{
	Material displayMat = gs.m_RenderMode switch  
	{  
	    GaussianSplatRenderer.RenderMode.DebugPoints => gs.m_MatDebugPoints,  
	    GaussianSplatRenderer.RenderMode.DebugPointIndices => gs.m_MatDebugPoints,  
	    GaussianSplatRenderer.RenderMode.DebugBoxes => gs.m_MatDebugBoxes,  
	    GaussianSplatRenderer.RenderMode.DebugChunkBounds => gs.m_MatDebugBoxes,  
	    _ => gs.m_MatSplats  
	};  
	if (displayMat == null)  
	    continue;
    
	int indexCount = 6;
	int instanceCount = gs.splatCount;
	MeshTopology topology = MeshTopology.Triangles;
	if (gs.m_RenderMode is GaussianSplatRenderer.RenderMode.DebugBoxes or GaussianSplatRenderer.RenderMode.DebugChunkBounds)
		indexCount = 36;
	if (gs.m_RenderMode == GaussianSplatRenderer.RenderMode.DebugChunkBounds)
		instanceCount = gs.m_GpuChunksValid ? gs.m_GpuChunks.count : 0;
	
	cmb.DrawProcedural
	(
		gs.m_GpuIndexBuffer, 
		matrix, 
		displayMat, 
		0, 
		topology, 
		indexCount, 
		instanceCount, 
		mpb
	);
}

分发方法 - DispatchCompute

概念：DispatchCompute 方法

  • 概念：参数介绍
    • （1）computeShader：指定包含要执行核函数的 Compute Shader 资源；
    • （2）kernelIndex：指定 Compute Shader 中具体的核函数（Kernel）索引；
    • （3）threadGroupX、Y、Z：各个维度上的线程组数量；
  • 示例：
uint gsX;

// 计算维度
m_CSSplatUtilities.GetKernelThreadGroupSizes((int)KernelIndices.CalcViewData, out uint gsX, out _, out _);

// 分发任务
// 参数一：指定计算 Shader 为 m_CSSplatUtilities
// 参数二：指定使用的函数
// 参数三：维度
cmb.DispatchCompute(m_CSSplatUtilities, (int)KernelIndices.CalcViewData, (m_GpuView.count + (int)gsX - 1)/(int)gsX, 1, 1);
