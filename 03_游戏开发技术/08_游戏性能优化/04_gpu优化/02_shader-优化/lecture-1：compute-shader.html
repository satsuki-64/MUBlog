Lecture 1：Compute Shader

Compute Shader

Compute Shader 基础概念

概念：Compute Shader 基础概念

  • 概念：
    • Compute Shader 是运行在图形处理器（GPU）上的一种特殊程序。与专门用于处理顶点或像素的传统着色器不同，Compute Shader 不直接参与图形渲染管线，而是专注于进行通用计算；
  • 概念：Compute Shader 的线程模型
    • 理解 Compute Shader 的关键在于掌握其线程模型；
    • 线程组：threads X、Y、Z
      • 一组线程的集合；
      • 每个线程组包含固定数量的线程，这些线程在组内可以协作，例如访问共享内存。线程组在 GPU 上被调度执行；
      • 比如在 Compute Shader 代码中，使用 [numthreads(X, Y, Z)] 属性来声明每个线程组中包含的线程数量及其三维布局。例如，[numthreads(8,8,1)] 表示一个线程组包含 8x8x1=64 个线程；
    • 调度：group X、Y、Z
      • 从 C# 脚本端，通过 ComputeShader.Dispatch 函数调用来启动计算，该调用指定了需要在 X、Y、Z 三个维度上分别启动多少个线程组；
      • C# 脚本中通过 ComputeShader.Dispatch(kernelIndex, groupsX, groupsY, groupsZ) 方法调用，它定义了总共要执行多少个线程组。例如，Dispatch(4, 3, 2) 表示调度 4x3x2=24 个线程；
    • 总线程数：
      • 总的线程数量是这三个层级的乘积：总线程数 = (groupsX * groupsY * groupsZ) * (X * Y * Z)
      • 每个线程在执行时，可以通过一系列系统值获取自己的身份标识，这对于索引数据至关重要；
  • 概念：Compute Shader 的数据交互
    • Compute Shader 需要与 CPU 或其他着色器阶段交换数据，主要通过缓冲区和可读写纹理实现；
      1. ComputeBuffer / GraphicsBuffer
      • 这是 Compute Shader 中最常用的数据载体，用于存储结构化数据（如自定义结构体的数组）
      • 在 HLSL 中，对应的类型是 StructuredBuffer<T>（只读）和 RWStructuredBuffer<T>（可读写）
      1. RWTexture2D
      • 当计算任务与图像处理相关时，可以使用 RWTexture2D<float4> 类型。在 C# 端，这对应着一个设置了 enableRandomWrite = true 的 RenderTexture；
      • 这使得 Compute Shader 能够直接写入纹理的任意像素，结果可以用于材质显示或后续渲染；

Compute Shader 使用

内核函数及其参数

概念：内核函数及其参数

  • 概念：
    • 每个内核函数的参数使用特定的系统值语义来声明，它们为每个线程提供其在整个计算任务中的坐标或标识；
  • 分类：
      1. SV_DispatchThreadID - 全局线程 ID
      • 这是最常用的标识符，它表示当前线程在整个调度（Dispatch）​ 中的全局唯一坐标；
      • 其计算公式为：
        • SV_DispatchThreadID = SV_GroupID * numthreads + SV_GroupThreadID；
      • 举例：
        • 代码当中利用 id.x 作为一维数组的索引，线程组在 Y 和 Z 维度上大小都是 1，所以 id.x 就是从 0 到 (threadGroupsX * GROUP_SIZE - 1) 的全局线性索引；
      1. SV_GroupID - 线程组 ID
      • 表示当前线程所在的线程组在整个调度中的三维坐标；
      • 其取值范围从 (0,0,0) 到 (threadGroupsX-1, threadGroupsY-1, threadGroupsZ-1)；
      1. SV_GroupThreadID - 组内线程 ID
      • 表示当前线程在其所属的线程组内部的三维坐标。其取值范围从 (0,0,0) 到 (numthreads.x-1, numthreads.y-1, numthreads.z-1)；
      • 当线程组内的线程需要协作（例如访问共享内存）时非常有用；

示例：使用 SV_DispatchThreadID 进行全局数据处理 - 处理一张 256x256 的纹理

  • Shader 端：
#pragma kernel CSMain
RWTexture2D<float4> Result; // 可读写的纹理

// 每个线程组8x8个线程
[numthreads(8,8,1)]
void CSMain (uint3 id : SV_DispatchThreadID)
{
    // 检查索引是否在纹理范围内，防止越界
    if(id.x < 256 && id.y < 256)
    {
        // 直接使用全局ID id.xy 来操作像素
        Result[id.xy] = float4((float)id.x / 255.0, (float)id.y / 255.0, 0.0, 1.0);
    }
}
  • 分析：
    • id.x 的范围不是 8，而是 线程组数量 * 每个线程组的线程数；
    • 比如在上述例子中，C# 脚本里 Dispatch 方法的参数是 (32, 32, 1)，所以 id.x 的范围是 0 到 32 * 8 - 1 = 255；
    • 因此，X 方向总线程 = 32（数） 8（每队线程数） = 256，Y 方向总线程 = 32（小队数） 8（每队线程数） = 256，总共会有 256 * 256 = 65,536 个线程同时执行你的 CSMain 函数；
    • 因为 id 参数被标记为 SV_DispatchThreadID，它代表的是全局索引。如果你在函数内部想获取线程在组内的局部索引，应该使用 uint3 groupThreadId : SV_GroupThreadID 参数。那时，groupThreadId.x 的范围才会是 0 到 7； + C# 端
public ComputeShader shader;
void ProcessTexture()
{
    int kernelHandle = shader.FindKernel("CSMain");
    RenderTexture tex = new RenderTexture(256, 256, 0);
    tex.enableRandomWrite = true; // 允许Compute Shader写入
    tex.Create();

    shader.SetTexture(kernelHandle, "Result", tex);
    // 调度足够的线程组：256/8 = 32, 32x32=1024个线程组
    shader.Dispatch(kernelHandle, 256 / 8, 256 / 8, 1);
}

Dispatch 方法

概念：Compute Shader 使用方法

  • 概念：Dispatch
    • 定义：
      • ComputeShader.Dispatch(int kernelIndex, int threadGroupsX, int threadGroupsY, int threadGroupsZ)；
    • 作用：
      • 通过此方法来启动计算任务；
    • 解释：
        1. kernelIndex：
        • 通过 ComputeShader.FindKernel ("内核函数名") 获取；
        • 其指定要执行 Compute Shader 中的哪个内核函数，代码中多个 #pragma kernel 声明的每个内核（如 CSSetIndices, CSCalcDistances）都有其对应的索引；
        1. threadGroups X, Y, Z：
        • 这三个参数定义了在三维空间中，你要启动多少个线程组。它们决定了计算任务的总体规模；
  • 定义：
    • 线程总数

Compute Shader 使用示例

最简 Compute Shader 代码示例

代码：最简 Compute Shader 代码示例

  • 代码：一个 .compute 文件 
// 1. 声明内核函数名为 Inverter
// 所有要被 C# 端调用的函数都需要使用 #pragma kernel 来声明
#pragma kernel Inverter 

// 2. 定义数据
Texture2D<float4> _ReadTexture; // 输入纹理（只读）
RWTexture2D<float4> _WriteTexture; // 输出纹理（可读写）

// 3. 定义线程模型
[numthreads(16,16,1)]
// 4. 实现函数体
void Inverter (uint3 id : SV_DispatchThreadID) 
{
    _WriteTexture[id.xy] = 1 - _ReadTexture[id.xy]; // 每个线程处理一个像素
}
  • 代码：C# 端
public class ComputeInvertTest : MonoBehaviour 
{
    // 1. 获取计算 Shader
    public ComputeShader _computeShader;
    
    public Texture2D _sourceTexture;
    private RenderTexture _resultTexture;
    private int _kernelIndex;

    void Start() 
    {
        _resultTexture = new RenderTexture(_sourceTexture.width, _sourceTexture.height, 0);
        _resultTexture.enableRandomWrite = true; // 关键：启用随机写入
        _resultTexture.Create();

		// 2. 获取内核函数名
        _kernelIndex = _computeShader.FindKernel("Inverter");
        _computeShader.SetTexture(_kernelIndex, "_ReadTexture", _sourceTexture);
        _computeShader.SetTexture(_kernelIndex, "_WriteTexture", _resultTexture);

        // 3. 计算线程组数量
        uint threadGroupSizeX, threadGroupSizeY, threadGroupSizeZ;
        _computeShader.GetKernelThreadGroupSizes(_kernelIndex, out threadGroupSizeX, out threadGroupSizeY, out threadGroupSizeZ);
        int groupX = Mathf.CeilToInt((float)_sourceTexture.width / threadGroupSizeX);
        int groupY = Mathf.CeilToInt((float)_sourceTexture.height / threadGroupSizeY);
        
        // 4. 分发任务
        _computeShader.Dispatch(_kernelIndex, groupX, groupY, 1);
        
        // 5. 使用计算得到的结果
        // 此时 _resultTexture 已存储处理后的图像
    }
}
  • 分析：如何计算线程组的数量
    • 比如上面的代码是使用 Compute Shader 计算纹理上各个像素的值，因此可以先定义线程组当中的 XYZ，比如 [numthreads(16,16,1)] 代表一个线程组当中有 256 个线程；
    • 然后，再使用纹理尺寸除以线程组尺寸，分别得到线程组在 XYZ 上的数值，比如：
      • （1）定义 uint 类型的线程组变量：uint threadGroupSizeX, threadGroupSizeY, threadGroupSizeZ;
      • （2）使用 GetKernelThreadGroupSizes 方法：
        • _computeShader.GetKernelThreadGroupSizes(_kernelIndex, out threadGroupSizeX, out threadGroupSizeY, out threadGroupSizeZ);
        • 使用 GetKernelThreadGroupSizes 方法得到 [numthreads(16,16,1)] 中定义的数值；
      • （3）计算：
        • groupX = Mathf.CeilToInt((float)_sourceTexture.width / threadGroupSizeX);
        • 这段代码的含义是使用纹理的宽度除以线程组内的 X，进而得到线程数量；
    • 得到线程数量后，可以开始此任务：_computeShader.Dispatch(_kernelIndex, groupX, groupY, 1);
