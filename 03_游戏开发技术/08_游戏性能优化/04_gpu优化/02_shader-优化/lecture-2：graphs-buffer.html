Lecture 2：Graphs Buffer

Graphs Buffer

Graphs Buffer 基础概念

引入：CPU 与 GPU 的通信瓶颈

  • 传统的渲染流程中，每帧都需要将网格、材质属性等数据从 CPU 内存上传至 GPU 显存，这个过程如果频繁进行，会成为主要的性能瓶颈；
  • GraphicsBuffer 的核心思想是预加载：将那些不常变化的数据（如静态模型的顶点、实例化数据、裁剪信息）提前一次性或增量式地上传到 GPU 显存中并驻留，在后续的渲染帧中直接复用，避免了重复传输；

概念：Graphs Buffer

  • 总结：
    • Graphs Buffer 是一个提供对 GPU 显存中结构化数据块进行直接访问的对象，其目标是最小化 CPU 与 GPU 之间的数据传输开销，实现数据驱动的批量渲染与计算；
  • 概念：
    • Graphs Buffer 是 Unity 新一代的图形缓冲区，专为现代渲染管线设计；
    • Graphs Buffer 是图形系统中用于存储和管理图形数据（如顶点、纹理、渲染指令等）的内存区域。它与 ​Buffer Queue ​ 机制紧密相关，协同工作以实现图形数据的高效生产和消费；
  • 特点：
      1. 支持多种缓冲区类型，比如如顶点、索引、结构化缓冲区等，并且可通过 Compute Shader 进行并行处理；
      1. 极高的数据吞吐效率，特别适用于处理静态或半静态的大规模数据；
  • 区别：Compute Buffer vs Graphs Buffer
  • 示例：
// Graphs Buffer创建示例
GraphicsBuffer buffer = new GraphicsBuffer(
    GraphicsBuffer.Target.Vertex, 
    GraphicsBuffer.Usage.WriteOnly,
    vertexCount, 
    vertexStride
);

      1. Structured Buffer（结构化缓冲区）
      • 含义：用于存储自定义结构体数组，是 GPU 计算中最常用的缓冲区类型；
      • 内存分布特点：
        • 元素按结构体定义顺序连续排列，每个结构体内部字段有特定的对齐要求，通常用于实例化渲染的实例数据；
      • 示例输入：
      1. Vertex Buffer（顶点缓冲区）
      • 含义：专门用于存储网格的顶点数据，如位置、法线、UV 坐标等；
      • 内存分布特点：
        • 顶点属性可以交错排列或分离排列；
        • 每个顶点包含多个属性（位置、法线、UV 等），属性间有特定的对齐和填充要求；
      • 图示：
      1. Index Buffer（索引缓冲区）
      • 含义：存储顶点索引，定义三角形或线的连接关系；
      • 内存分布特点：
        • 纯粹的整数数组（16 位或 32 位），每个索引指向顶点缓冲区中的顶点位置；
        • 索引值连续排列，无填充
      • 类型选择：
        • 16 位索引：最多 65535 个顶点，节省内存；
        • 32 位索引：支持更多顶点，内存占用更大；
      1. Raw Buffer（原始缓冲区）
      • 含义：提供字节级访问的最灵活缓冲区类型；
      • 内存分布特点：
        • 纯粹的字节流，无特定结构；
        • 完全由应用程序控制数据解释；
        • 可用于自定义编码或特殊格式数据；

Graphs Buffer 与 Compute Shader

  • 概念：Compute Shader 代码
#pragma kernel CSMain

// 可读写结构化缓冲区
RWStructuredBuffer<InstanceData> _InstanceDataBuffer;

[numthreads(64, 1, 1)]
void CSMain (uint3 id : SV_DispatchThreadID) {
    uint index = id.x;
    
    // 更新实例数据
    InstanceData data = _InstanceDataBuffer[index];
    data.position += float3(0, sin(_Time.y + index * 0.1), 0);
    data.color = lerp(float4(1,0,0,1), float4(0,0,1,1), sin(_Time.y + index * 0.01));
    
    _InstanceDataBuffer[index] = data;
}
  • 概念：C# 代码 - 使用 SetBuffer 直接上传数据到 Compute Shader 
public class ComputeParticleSystem : MonoBehaviour 
{
    public ComputeShader computeShader;
    public Material renderMaterial;
    public Mesh particleMesh;
    
    private GraphicsBuffer _particleBuffer;
    private int _kernelIndex;
    private int _particleCount = 10000;
    
    void Start() {
        // 创建缓冲区
        _particleBuffer = new GraphicsBuffer(
            GraphicsBuffer.Target.Structured,
            _particleCount,
            Marshal.SizeOf<InstanceData>()
        );
        
        // 初始化数据
        InstanceData[] initialData = new InstanceData[_particleCount];
        for (int i = 0; i < _particleCount; i++) {
            initialData[i] = new InstanceData {
                position = Random.insideUnitSphere * 10,
                rotation = Quaternion.identity,
                scale = Vector3.one * Random.Range(0.1f, 0.5f),
                color = Random.ColorHSV()
            };
        }
        _particleBuffer.SetData(initialData);
        
        // 获取核函数
        _kernelIndex = computeShader.FindKernel("CSMain");
    }
    
    void Update() {
        // 设置Compute Shader参数
        computeShader.SetBuffer(_kernelIndex, "_InstanceDataBuffer", _particleBuffer);
        computeShader.SetFloat("_Time", Time.time);
        
        // 调度计算
        computeShader.Dispatch(_kernelIndex, Mathf.CeilToInt(_particleCount / 64f), 1, 1);
        
        // 渲染实例
        renderMaterial.SetBuffer("_InstanceDataBuffer", _particleBuffer);
        Graphics.DrawMeshInstancedIndirect(
            particleMesh,
            0,
            renderMaterial,
            new Bounds(Vector3.zero, Vector3.one * 20),
            _particleBuffer
        );
    }
    
    void OnDestroy() {
        _particleBuffer?.Release();
    }
}

MaterialPropertyBlock 与 Graphs Buffer

概念：MaterialPropertyBlock 与 Graphs Buffer

  • 概念：
    • MaterialPropertyBlock 允许在不破坏 GPU 实例化合批的前提下，为使用同一材质的多个物体设置不同的属性值；
    • 传统上，如果直接修改 Material 的属性，即使只改变一个数值，Unity 也会为这个新参数组合创建新的材质实例，从而导致合批中断；
    • 而 MaterialPropertyBlock 则巧妙地将这些“差异化”的参数打包，在绘制调用（Draw Call）时一并传入，从而保持合批；
  • 实现：
    • （1）创建一个 MaterialPropertyBlock 实例；
    • （2）使用 SetBuffer 等方法将你的 GraphicsBuffer 设置到该实例中；
    • （3）通过 Renderer.SetPropertyBlock方法将该属性块应用到特定的渲染器上；

Compute Buffer

Compute Buffer 基础概念

概念：Compute Buffer 基础概念

  • 概念：什么是 Compute Buffer
    • ComputeBuffer 是 Unity 提供的一个类，是 Unity 中专门用于通用 GPU 计算的数据缓冲区，主要与 Compute Shader 配合使用，用于在 CPU 和 GPU 之间创建和操作一个数据缓冲区，这个缓冲区可以存储结构化的数据，例如自定义结构体或者数组，并且可以被 Compute Shader 和常规的 Shader 进行访问；
    • ComputeBuffer 的主要目的是允许 GPU 直接访问和处理大量数据，从而实现高效的并行计算或复杂的渲染效果；
  • 概念： 创建 Compute Buffer
    • 定义：
    • 解释：
      • 参数一：count
        • 指定缓冲区中元素的数量，要存储 1000 个粒子的数据，count 就设为 1000。它决定了缓冲区的“长度”
      • 参数二：stride
        • 指定缓冲区中每个元素的大小（字节数）
        • 务必确保 C# 端的结构体与 Compute Shader 中定义的结构体内存布局完全一致。在 Shader 中，float3 通常也会占用 16 字节（由于精度对齐），为避免跨平台问题，有时在 C# 中直接使用 float4/Vector4 是更稳妥的做法；
      • 参数三：type - 可选
        • 指定缓冲区的特殊类型和用途，默认为 ComputeBufferType.Structured；
        • 高级用法时需指定，如用于追加消费的 Append/Counter，或用于间接绘制的 IndirectArguments，或者常量；

  • 使用：
    • （1）在 C# 中创建和设置 Compute Shader：
      • dataBuffer = new ComputeBuffer(dataCount, sizeof(float) * 4);
    • （2）将数据设置到 ComputeBuffer 当中：
        1. 创建数据：
        1. 设置数据：
        • dataBuffer.SetData(initialData);
    • （3）找到 Compute Shader 中的函数：
        1. 得到 Handle：int kernelHandle = omputeShader.FindKernel("CSMain");
        1. 根据 Handle 数值，设置缓冲区：
        • 其中 DataBuffer 表示要设置的属性的名字，kernelHandle 是索引，dataBuffer 是目标数据；
        • computeShader.SetBuffer(kernelHandle, "DataBuffer", dataBuffer);
    • （4）调度任务执行：
        1. 或者线程组信息：
        • int threadGroups = Mathf.CeilToInt(1024 / 64.0f);
        1. 调度：
        • computeShader.Dispatch(kernelHandle, threadGroups, 1, 1);
  • 代码：C# 部分
using UnityEngine;

public class ComputeBufferExample : MonoBehaviour
{
    public ComputeShader computeShader; // 拖拽你的Compute Shader到这里

    private ComputeBuffer dataBuffer;
    private int kernelHandle;

    void Start()
    {
        // 1. 定义数据
        int dataCount = 1024;
        Vector4[] initialData = new Vector4[dataCount];
        for (int i = 0; i < dataCount; i++)
        {
            initialData[i] = new Vector4(i, i * 2, i * 3, 1.0f);
        }

        // 2. 创建Compute Buffer
        // 参数1: 元素数量
        // 参数2: 每个元素的字节大小（stride）
        dataBuffer = new ComputeBuffer(dataCount, sizeof(float) * 4);

        // 3. 将数据设置到Buffer中
        dataBuffer.SetData(initialData);

        // 4. 找到Compute Shader中的内核函数
        kernelHandle = computeShader.FindKernel("CSMain");

        // 5. 将Compute Buffer设置到Compute Shader
        computeShader.SetBuffer(kernelHandle, "DataBuffer", dataBuffer);
    }

    void Update()
    {
        // 6. 每帧调度Compute Shader执行
        // 假设我们的Compute Shader使用了 [numthreads(64,1,1)]
        // 那么需要调度的线程组数量 = 数据总数 / 每个线程组的线程数
        int threadGroups = Mathf.CeilToInt(1024 / 64.0f);
        computeShader.Dispatch(kernelHandle, threadGroups, 1, 1);

        // 7. (可选) 将结果读回CPU
        // Vector4[] results = new Vector4[1024];
        // dataBuffer.GetData(results);
        // Debug.Log(results[0]);
    }

    void OnDestroy()
    {
        // 8. 非常重要！必须手动释放Compute Buffer，否则会造成内存泄漏
        if (dataBuffer != null)
            dataBuffer.Release();
    }
}

// Compute Shader 示例
#pragma kernel CSMain

// 定义一个与C#端匹配的结构体
struct MyData
{
    float4 value;
};

// 声明一个可读写的结构化缓冲区
RWStructuredBuffer<MyData> DataBuffer;

// 可能需要的其他参数
float DeltaTime;

[numthreads(64,1,1)]
void CSMain (uint3 id : SV_DispatchThreadID)
{
    // 1. 获取当前线程的全局索引
    uint index = id.x;

    // 2. 边界检查！非常重要，防止索引越界
    if (index >= DataBuffer.Length)
        return;

    // 3. 读取数据
    MyData data = DataBuffer[index];

    // 4. 进行计算（例如，一个简单的正弦波动画）
    data.value.y += sin(data.value.x + DeltaTime) * 0.1;

    // 5. 将修改后的数据写回缓冲区
    DataBuffer[index] = data;
}

[mindmap]

  • Compute Buffer 使用
    • 第一步：创建 Compute Buffer；
    • 第二步：往 Compute Buffer 填充数据，并上传至 GPU
    • 第三步：将 Buffer 数据传递给 Shader

注意：使用 Compute Buffer 的注意事项
    1. ComputeBuffer 是非托管资源，必须手动调用 Release() 方法进行释放，通常在 OnDestroy() 或 OnDisable() 中进行，以避免内存泄漏；
    1. 尽量减少 CPU 和 GPU 之间的数据传递次数，因为这类操作（如频繁调用 SetData）可能带来开销；

Compute Buffer 和 Graphs Buffer 的区别

分析：Compute Buffer 和 Graphs Buffer 的区别

  • 作用：
    • （1）Compute Buffer​ 的目标是通用计算，它是为 Compute Shader 量身定制的数据容器，其首要任务是高效地处理与图形渲染管线无关的任意结构化数据，在这里数据可以被并行地读取、修改、写入，进行各种复杂的计算，例如物理模拟、粒子系统更新、图像处理、人工智能算法等。它的关注点在于数据本身的并行计算，而不关心这些数据最终如何被渲染到屏幕上；
    • （2）Graphics Buffer ​ 的目标则是高效渲染，它集成于传统的图形渲染管线，其作用就是以最高的效率向顶点着色器或曲面细分着色器等图形管线阶段提供几何数据，比如顶点和索引，数据流动模式高度特化。在 Shader 当中使用 GraphsBuffer 时，通常在顶点着色器对缓冲区中的顶点进行只读、顺序式的访问；
  • 技术差异：
    • （1）Graphs Buffer 可以直接使用 DrawProcedural、DrawMeshInstancedIndirect 等现代渲染命令中，作为顶点/索引数据的来源；
    • （2）Compute Buffer 独立于渲染管线之外，它通过 Compute Shader 进行计算，计算结果如果需要用于渲染，则需要通过 Material 的 SetBuffer 方法传递给顶点片元 Shader ，或者先写入一个 Graphics Buffer；
