模型资源优化分析

前置优化分析 - 模型导入前的分析

概念：具体分析步骤

  • 第一步：检查 DCC 软件导出步骤
    • （1）优化原始导入模型文件，删除不需要的数据；
    • （2）注意单位的统一；
    • （3）导出的网格必须是多边形拓扑，不能是贝塞尔曲线、样条曲线、细分曲面等，这些在 Unity 当中都不支持；
    • （4）在导出之前确保所有 Deformers 都烘培到网格模型上，比如骨骼的形变已经烘培到蒙皮的权重上等；
    • （5）不建议模型使用的纹理随模型导出；
    • （6）如果需要导入 Blend shape normals 时，必须指定光滑组 Smooth groups；
    • （7）建议导出时不携带如摄像机、灯光、材质等场景信息；
  • 第二步：分析原始模型的性能影响点
    • （1）最小化面数，不需要微三角形面，三角面尽量分布均匀；
      • 在 Unity 编辑器中，在多角度对导入的模型进行观察，进来不要有微三角面；
      • 微三角面指的是，一个三角面只包含个位数个像素。这个时候要使用 LOD 或者让美术对原始模型进行调整；
    • （2）合理的拓扑结构与平滑组，尽可能是闭包；
      • 尽可能是闭包，而不是没有焊接的面去组合，这可能回导致 Unity 后期烘培的错误；
    • （3）尽量少的材质个数；
    • （4）尽可能少的蒙皮网格；
    • （5）尽可能少的骨骼数量；
    • （6）FK 与 IK 节点分离，导出时删除 IK 骨骼节点；
注意：
    1. 尽可能使用 FBX 格式作为模型的导入链；
    1. 将一些不需要的、DCC 软件中设置的选项关闭
    1. 如果当前模型不具有动画，把模型的 ImportSetting 中的 Animation 中的 Animation Type 设置为 None，并且需要勾选 Import Animation；

Mesh 与 Geometry 的设置

概念：Mesh 网格的设置

  • 概念：Mesh Compression
    • Mesh 压缩，如果开启之后会减少 Mesh 的体积，但是可能会出错；
  • 概念：Read/Write
    • 如果启用，Unity 会在内存和 GPU 中同时保留网格数据的副本，允许脚本在运行时通过代码动态的修改网格；
    • 这会使一份网格数据在内存中存在两份，增加内存开销。​除非需要在运行时修改网格（例如程序化生成地形、破碎效果），或者是应用蒙皮动画，否则绝大多数情况应始终保持禁用；
  • 概念：Optimize Mesh
    • 用于优化网格的顶点索引顺序，以提高运行时 GPU 的渲染效率；
    • 它通过重新排列三角形顶点的顺序，来提高 GPU 的顶点缓存命中率。对于来自不同 3D 软件、顶点顺序可能未优化的网格，建议设置为“Everything”，让 Unity 为其计算最佳顺序；
  • 概念：Keep Quads（保留四边形）​​
    • 如果网格中包含四边面，启用此选项可以保留它们；
    • 这主要对 Unity 的 Tessellation（曲面细分）等后期处理效果有益。如果模型完全由三角面构成或不需要细分，此选项无影响；

Deformers 与蒙皮权重

概念：Deformers 与蒙皮权重

  • 概念：Deformers（形变器）​
    • 指能驱动网格顶点产生动态变化的元素，如骨骼（Armature）​、形态键（Blend Shapes）​​ 等，将动态计算过程转换为静态数据，确保 Unity 无法计算的复杂驱动关系得以保留；
  • 概念：烘培
    • 比如在 Blender 中，一个角色的动画可能由骨骼动画、IK 约束、形态键甚至复杂的驱动程序共同控制。这些是过程性的，意味着 Blender 在每一帧都在实时计算这些关系并得出顶点最终位置。然而，游戏引擎（如 Unity）的动画系统通常无法理解或高效运行这些 Blender 特有的计算逻辑；
    • 因此，​烘焙的作用就是充当一个“翻译官”，它让 Blender 在导出前，​预先执行完所有复杂的计算，将最终结果（即每一帧每一个顶点的准确位置）直接“烙印”在模型数据上（例如，转换为纯粹的关键帧动画和固定的蒙皮权重数据）；
    • 这样，Unity 在运行时只需要读取这些简单的、优化后的数据，就能准确还原出动画效果，而不必承担复杂的实时计算开销；
  • 概念：蒙皮权重
    • 定义网格上每个顶点受哪些骨骼影响，以及影响程度的数值，这是骨骼形变信息的最终载体。烘焙后，骨骼对网格的控制完全由权重数据决定；
    • 这背后的原理依赖于线性混合蒙皮（Linear Blending Skinning, LBS）​​ 算法。该算法认为，网格上任意一个顶点的最终位置，可以由影响它的多个骨骼的变换矩阵及其权重加权平均计算得出：
  • 实现：在 Blender 中的操作要点
      1. 检查骨骼与权重​：
      • 确保 Deform 骨骼​（即实际影响网格的骨骼）权重绘制正确；
      • 在 Blender 的权重绘制模式下，检查权重过渡是否平滑，没有不合理的拉扯或变形；
      1. 烘培动画：
      • 如果你的模型带有动画，在导出 FBX 时，务必在导出设置中勾选“烘焙动画（Bake Animation）”选项​；
      • 这会强制 Blender 将所有约束、驱动器等产生的动画效果转换为纯粹的关键帧动画，这是 Unity 能够完美读取的格式；
      1. 应用变换：
      • 在导出前，选中你的模型和骨骼骨架，使用 Ctrl + A 并选择“应用全部变换”。这能将物体的缩放、旋转值归零，避免导入 Unity 后出现意想不到的缩放或朝向问；
