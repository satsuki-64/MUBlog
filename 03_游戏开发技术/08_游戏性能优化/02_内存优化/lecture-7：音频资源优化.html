Lecture 7：音频资源优化

压缩格式及其特点

不同压缩格式及其特点

概念：不同压缩格式及其特点

  • 概念：MP3
    • MP3 的全称是 MPEG-1 Audio Layer III 或 MPEG-2 Audio Layer III。它是一种诞生于上世纪 90 年代初的有损音频压缩格式；
    • 它的核心原理是利用心理声学模型，去除人耳不敏感或不易察觉的音频信息，从而达到高压缩比。例如，它会舍弃在强音信号遮蔽下难以听见的弱音信号，并对高频部分进行较大比例的压缩甚至忽略；
    • MP3 支持可变的比特率，允许用户根据需要在文件大小和音质之间进行权衡，常见的比特率范围从 128kbps 到 320kbps；
  • 概念：Vorbis - 通常以 .ogg 格式存在
    • Vorbis 是一种开放且免专利费的有损音频压缩格式，通常封装在 Ogg 容器中，因此常被称为 Ogg Vorbis；
    • 在技术上，Vorbis 同样采用基于心理声学模型的有损压缩，但使用了与 MP3 不同的算法。普遍认为，在相同的比特率下，Vorbis 的音质通常优于 MP3，尤其在较低比特率时表现更为出色；
    • 它原生支持可变比特率和多声道音频，格式非常灵活，能够不断改进编码算法而不会影响旧版解码器的兼容性；
  • 概念：WAV
    • WAV 是微软公司为 Windows 操作系统开发的一种标准音频文件格式，它是一种容器格式，通常用于存储未经压缩的 PCM 音频数据，从而保证极高的保真度；
    • WAV 文件的最大特点就是能够无损地保存原始声音的波形，因此音质极佳。也正因为其通常存储的是未压缩的数据，导致文件体积非常庞大，不适合长时间录制或网络传输；
    • 它支持多种音频采样频率、量化位数和声道数，并也支持一些压缩算法，如 ADPCM，但其最经典和常见的形态仍是未压缩的 PCM 格式；
  • 概念：PCM
    • PCM 是脉冲编码调制的缩写，它是一种将模拟声音信号转换为数字信号的最基本、最直接的编码方式，可以认为它是数字音频的原始形态；
    • PCM 编码不进行任何压缩，它通过固定的采样频率和量化位数来记录声音。例如，CD 音质的 PCM 音频采样率为 44.1kHz，量化位数为 16 位；
    • 由于是未压缩的原始数据，PCM 格式的音频具有最高的保真度，被视为其他所有有损或无压缩格式的参考基准；
  • 概念：ADPCM
    • ADPCM 是一种针对 16 位 PCM 样本的差分信号进行压缩的技术；
    • 它的核心思想是记录相邻音频样本之间的差值，并对其进行自适应量化，而不是直接记录每个样本的完整幅度值。这种方法的压缩比相对固定，通常能达到 4:1 左右；
    • ADPCM是一种有损压缩，但它的编解码算法相对简单，对计算能力的要求很低，解码速度极快；

总结

总结：音频格式的选择
  • 根据平台选择合理的音频设置，原始音频资源尽量采用未压缩 WAV 格式；
  • 移动平台大多数声音尽量采用 Vorbis 压缩设置，IOS 平台或不打算循环的声音可以选择 MP3 格式；
  • 对于简短、常用的音效，可以采用解码速度快的 ADPCM 格式 (PCM 为未压缩格式)；

采样率与加载类型

音频源文件的采样率

概念：音频采样及其原理

  • 概念：
    • 主要基于奈奎斯特-香农采样定理，该定理指出：要无失真地记录和还原一个模拟信号，​采样频率必须高于原始信号中最高频率成分的 2 倍；
    • 如果采样率不足，就会发生一种称为“混叠”的失真，即高频信号被错误地记录成低频噪声；
  • 概念：人耳的听觉范围
    • 人耳的听觉范围通常在 ​ 20 Hz 到 20 kHz ​ 之间，因此，为了完整覆盖人耳能听到的所有频率，采样率至少需要达到 40 kHz。这就是为什么 ​ 44.1 kHz ​ 成为 CD 音频标准的原因，它略高于 40 kHz，为抗混叠滤波器留出了余量，从而能够无损地重现人耳听觉范围内的全部声音频率；
  • 总结：
    • 在专业音乐制作、影视录音和混音阶段，通常会选择 96 kHz 或 192 kHz 的高采样率，为后期处理（如大幅度的音高修正、慢放）提供了更大的空间；
    • 一般的音频产品，可以使用 44.1Hz；
    • 对于移动平台，可以设置为 22050HZ，越低的采样频率生成的导入文件越小；

加载类型

概念：加载类型 - Load Type

  • 概念：Decompress On Load - 用内存换 CPU
    • 这种方式在音频资源加载完成后，会立即将其完全解压缩，变成未压缩的 PCM 数据存储在内存中；
    • 播放时，音频引擎直接读取内存中的原始数据，无需实时解压，因此 CPU 占用极低，播放响应最快；
    • 这种加载模式的代价是内存占用很高，解压后的数据体积通常是压缩状态的数倍（Vorbis 格式可能达 10 倍，ADPCM 格式约 3.5 倍）；
    • 它最适合播放频繁、要求零延迟的短音效，如 UI 反馈、武器射击声。务必注意文件大小，过大的文件会迅速耗尽内存；
  • 概念：Compressed In Memory
    • 这种方式将音频文件以压缩格式（如 Vorbis、MP3、ADPCM）加载到内存中，在播放时才进行解压，解压开销因格式而异，Vorbis/MP3 较高，ADPCM 较低；
    • 它适用于长度中等或较长的音效，以及对内存敏感但可接受少量 CPU 开销的音频；
  • 概念：Streaming
    • Streaming 模式不将整个音频文件加载到内存，而是开辟一个小的缓冲区（约 200KB），在播放时动态地从磁盘流式读取并解码数据。这使其内存占用最小，但会产生持续的磁盘 I/O 和实时解码开销，​CPU 占用最高；
    • 一个关键限制是，​同一音频流无法同时播放多个实例​（如重叠的环境声），否则会创建多个缓冲区，增加 CPU 和内存负担；
    • 它专为长时间播放且通常只需单实例的音频设计，如背景音乐或环境音效；
总结：加载类型
  • Decompress On Load：如果单个音频资源大小小于 200 KB，可以开启此选项；
  • Compressed In Memory：对于复杂音效，大小大于 200 kb，长度超过 5 秒的音效采用 CompressedInMemory 模式；
  • Streaming：背景音乐或者较大的音效文件；

补充

  • 静音：不要只是将音量大小设置为 0，而是应该摧毁 Audio Source 组件；
  • 音道：移动平台对音乐音效统一采用单通道设置 (Force to Mono)；
