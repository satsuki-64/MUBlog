CG 语法基础知识

语句编写位置

概念：CG 语句编写位置

  • 对于 顶点\片元着色器 来说 CG 语句需要写在 Pass 渲染通道语句块中；
  • 我们需要在 Pass 语句块中加入指令：
    • CGPROGRAM
    • //...
    • ENDCG
  • 在这两个指令之间，就是我们书写 CG 代码的地方；

编译指令

概念：重要的编译指令 - 指定着色器函数

  • 概念：
    • 在真正书写 CG 代码之前我们需要先使用 #pragma 声明编译指令；
    • 下面这两个编译指令的作用，是将 顶点\片元着色器 实现定位到两个函数中，之后我们只需要在这两个函数中书写 shader 逻辑即可；
  • 语法：定义实现顶点/片元着色器代码的函数名称
    • #pragma vertex XXXName：其中的名称为顶点着色器的函数名；
    • #pragma fragment XXXName：其中的名称为片元着色器的函数名；
  • 举例：
SubShader
{
	Tags {"RenderType"="Opaque"}
	LOD 100
	
	Pass
	{
		CGPROGRAM
		#pragma vertex vert
		#pragma fragment frag
		
		#include "UnityCG.cginc"
		
		struct appdata
		{
			// ...
		}
		
		struct v2f
		{
			// ...
		}
		
		v2f vert(appdata v)
		{
			v2f o;
			return o;
		}
		
		fixed frag()
		{
			fixed4 col;
			return col;
		}
		ENDCG
	}
}

变体

什么是变体

概念：什么是变体

  • 概念：
    • Shader 的变体，就是其编译得到的各个 Shader 的集合，每一个 Shader 变体对应一个材质的实例；
  • 特点：
    • 可将多个功能与效果集成到一个 Shader 内，便于使用与管理；
    • 比如：可以将很多一个功能都写到一个 Shader 当中，然后通过为 Shader 的不同变体、只是要其中的某几个功能，使得当前的 Shader 适用不同环境；
    • 同时，如果变体数量过多，会导致加载时间过长与内存占用增加
    • 在游戏一开始的时候，就会将 Shader 加载进来，因此如果单个 Shader 很大的话、可能在一开始加载时需要很久；
  • 概念：Shader 变体的类型
      1. 无论如何都会被编译的变体；
      1. 通过材质的使用情况来决定是否编译的变体；
  • 图示：

概念：#pragma multi_compile [选项1] [选项2]

  • 概念：
    • 生成多个 Shader 变体以支持不同功能组合（如雾效、阴影）；
    • Unity 会为所有可能的组合编译独立变体；
  • 示例：
#pragma multi_compile _ USE_FOG USE_SHADOWS

概念：#pragma shader_feature [选项]

  • 概念：
    • 类似 multi_compile，但仅编译实际使用的变体，减少冗余代码，适用于材质面板可开关的功能；
  • 示例：
#pragma shader_feature _USE_SPECULAR

变体的声明与使用

概念：变体的声明与使用

  • 概念：变体的声明
    • #pragma multi_compile _ NAME：变体名必须为全大写，可通过添加 _ 来定义一个默认空的变体；
  • 图示：当前 Shader 的变体数量
  • 概念：空变体
    • 概念：
      • 空变体 (Empty Variant)是 Unity Shader 变体系统中的一种特殊设计，通过在 #pragma multi_compile 或 #pragma shader_feature 指令中使用下划线 _ 来表示；
      • 空变体通过复用"无关键字"状态作为默认关闭状态，避免了为每个功能开关都定义两个独立关键字（如同时定义 _DISSOLVEENABLED_OFF 和 _DISSOLVEENABLED_ON），从而显著节省关键字资源；
    • 示例：
      • #pragma multi_compile _ _DISSOLVEENABLED_ON 中的下划线 _ 表示默认变体​（无关键字定义）
      • 在材质 Inspector 面板中，Toggle 类型的属性，如 [Toggle]_DISSOLVEENABLED("Disslove Enabled", int) = 0 会自动映射到空变体机制：
        • 关闭状态（值为 0）：使用空变体 _
        • 开启状态（值为1）：使用带关键字的变体 _DISSOLVEENABLED_ON；
      • 因此 _ 表示空变体，对应功能关闭状态；_DISSOLVEENABLED_ON 表示功能开启状态；
      • 这种设计源于 Unity 对 Shader 关键字数量的严格限制（全局关键字上限 256 个），空变体机制允许开发者在不消耗额外关键字的情况下实现功能开关；
  • 补充：变体名与 Material Property 的 Toggle 属性联动
    • 当 Shader 的 Properties 中定义了 [Toggle]_DissolveEnabled("DissolveEnabled",int) = 0 时，Unity 会自动生成一个名为 _DISSOLVEENABLED_ON 的全局关键字（Keyword）；
    • 若要在 Shader 代码中通过预处理器（如 #ifdef）响应此 Toggle 开关，必须使用完全相同的变量名​（区分大小写）并在末尾追加 _ON；
    • Toggle 属性值为 1 时，Unity 会启用 _DISSOLVEENABLED_ON 关键字；为 0 时则禁用。multi_compile 定义的变体会根据此关键字状态动态切换；
    • Unity 会根据 Material 当前启用的关键字选择对应的变体；
      • 若未启用 _DISSOLVEENABLED_ON，则使用默认变体（_）；
      • 若启用，则切换到定义了 _DISSOLVEENABLED_ON 的变体；
  • 代码：
Shader "taecg/Character Unlit"
{
    Properties
    {
	    [Header(Base)]
		[NoScaleOffset]_MainTex("MainTex",2D) = "white"{}
		_Color("Color",Color) = {0,0,0,0}

		[Space(10)]
		[Header(DissolveTex)]
		[Toggle]_DissolveEnabled("DissolveEnabled",int) = 0
		_Clip("Clip",Range(0,1)) = 0
		[NoScaleOffset]_DissolveTex("_DissolveTex",2D) = "white"{}
		_RampTex("RampTex(RGB)",2D) = "white"{}
    }
    
    SubShader
    {        
        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            
            // 定义 Shader 变体，这里使用了空格，因此默认使用的是空变体
            // 这里有一个规则，就是其变体如果想要被属性中的Toggle控制，这里必须名字一致，并再在最后加上一个ON
            #pragma multi_compile _ _DISSOLVEENABLED_ON
            
            #include "UnityCG.cginc"
            
            sampler2D _MainTex;
            sampler2D _DissolveTex; float _DissolveTex_ST;
            sampler _RampTex;
            
            fixed4 _Color;
            fixed4 _Clip;

            struct appdata
            {
				float4 vertex : POSITION;
				float4 uv : TEXCOORD0;
            };

            struct v2f
            {
				float4 pos : SV_POSITION;
				float4 uv : TEXCOORD0;
            };
            
            v2f vert(appdata v)
            {
				v2f o;
				o.pos = UnityObjectToClipPos(v.vertex);
				o.uv.xy = v.uv.xy;
				o.uv.zw = v.uv * _DissolveTex_ST.xy + _DissolveTex_ST.zw;
				return o;
            }
            
            fixed4 frag (v2f i) : SV_Target
            {
	            fixed4 c;
	            fixed4 tex = tex2D(_Tex,i.uv.xy);
	            c = tex;
	            c += _Color;
	            
	            // 用If来判断是否使用变体
	            #if _DISSOLVEENABLED_ON
			        fixed4 dis = tex2D(_DissolveTex,i.uv.zw)
		            clip(dis.r-_Clip);
			        fixed dissolveValue = saturate(dissolveTex.r - _Clip)/(_Clip + 0.1 - _Clip); 
			        fixed4 rampTex = tex1D(_RampTex,dissolveValue);
			        C += rampTex;
		        #endif
		        
				return tex;
            }
            ENDCG
        }
    }
}

其他常见编译指令

概念：其他常见编译指令

  • 概念：#pragma multi_compile_fog
    • 自动生成雾效的变体，包含线性/指数/指数平方三种衰减方式，配合 UNITY_FOG_COORDS 等宏实现雾效混合；
    • 可以在 Unity 中的 RenderSettings.fogMode 中动态选择雾效计算方式；
  • 概念：几何着色器 - #pragma geometry [函数名]
    • 启用几何着色器，用于动态生成或修改图元（如将点扩展为三角形）
  • 概念：曲面细分着色器 - #pragma hull/ #pragma domain
    • 启用曲面细分着色器，用于动态增加模型细节（如地形 LOD）
  • 概念：限定特定渲染平台的编译 - #pragma exclude_renderers/ #pragma only_renderers
    • 限制 Shader 在特定渲染平台（如 D 3 D 11、GLES）的编译，优化包体大小；
    • 比如：#pragma only_renderers d3d11 // 仅DirectX 11平台

数据类型

基础数据类型

概念：基础数据类型

  • 概念：基础数据类型
  • 注意：
    • 其中的 fixed 类型在浮点类型中用的最多，其默认不需要加符号，另外两种需要添加符号；
  • 概念：纹理采样器类型
    • sampler：通用的纹理采样器，可以用于处理各种不同维度和类型的纹理；
    • sampler 1D：用于一维纹理，通常用于对一维纹理进行采样，例如从左到右的渐变色；
    • sampler 2D：用于二维纹理，最常见的纹理类型之一。它用于处理二维图像纹理，例如贴图；
    • sampler 3D：用于三维纹理，通常用于体积纹理，例如体积渲染；
    • samplerCUBE：用于立方体纹理，通常用于处理环境映射等需要立方体贴图的情况；
    • samplerRECT：用于处理矩形纹理，通常用于一些非标准的纹理映射需求；
    • 他们都是用于处理纹理 (Texture) 数据的数据类型，他们的主要区别是纹理的维度和类型；

概念：基础复合数据类型

向量类型

概念：向量类型

  • 概念：
    • 向量类型属于 CG 语言的内置数据类型，内置的向量类型是基于基础数据类型声明的；
    • 向量的最大维度不超过 4 维，数据类型可以是任意数值类型；
  • 语法：
    • 二维：数据类型2 = 数据类型2(n1,n2)
    • 三维：数据类型3 = 数据类型3(n1,n2,n3)
    • 四维：数据类型4 = 数据类型4(n1,n2,n3,n4)
  • 举例：
fixed2 f2 = fixed2(3.4,6.1);
fixed3 f2 = fixed3(120.23,213.2,124.56);
int4 i4 = int4(1,2,3,4);

运算符

概念：加法/乘法运算符

  • 分析：贴图和颜色的加法
    • 贴图和颜色的加法会使颜色值向白色方向靠近，整体效果变得更亮；
    • 当两个颜色相加时，每个通道 (R,G,B) 的值会直接相加，可能导致值超过 1.0，因此直接被截断；
    • 此效果可以用于多光源贡献叠加计算 (环境光+漫反射+镜面反射)，或者 HDR 高光增强；
  • 分析：贴图和颜色的乘法
    • 乘法操作会产生颜色相互抵消的效果，整体变暗；
    • 当贴图和黑色相乘时，最后得到的结果为黑色，如果贴图和白色相乘时，最后的结果为原本的贴图的颜色（即不变）；
    • 常用于颜色调制、阴影计算或颜色过滤；

概念：运算符与短路现象

  • 概念：C# 语言中的短路现象
    • 短路 (Short-circuiting)是 C# 中逻辑运算符 && (逻辑与)和 || (逻辑或)的一个重要特性，它能够显著提高代码的效率和安全性；
    • 短路是指在逻辑表达式的求值过程中，如果根据已经计算的部分已经能够确定整个表达式的结果，那么将不再计算剩余的部分，这种特性使得程序可以避免不必要的计算，从而提高性能并防止潜在的错误；
  • 分析：C# 中的 && 的短路特性
    • 对于逻辑与运算符 &&，如果第一个操作数的结果为 false，那么整个表达式的结果必然为 false，因为无论第二个操作数的结果是什么，都无法改变整个表达式的结果为 false；
  • 举例：
    • 在下面代码中，由于 a 是 false，所以 b 的值并不重要，整个表达式 a && b 的结果肯定是 false；
    • 因此，SomeMethodThatMayThrowException() 方法不会被调用，从而避免了可能发生的异常；
bool a = false;
bool b = SomeMethodThatMayThrowException();
if (a && b) 
{
    // 这段代码永远不会执行，因为a是false，所以b不会被评估
}
  • 分析：C# 中的 || 的短路特性
    • 对于逻辑或运算符 ||，如果第一个操作数的结果为 true，那么整个表达式的结果必然为 true，因为无论第二个操作数的结果是什么，都无法改变整个表达式的结果为 false；
  • 举例：
    • 由于a是true，所以b的值并不重要，整个表达式a || b的结果肯定是true。因此，SomeMethodThatMayThrowException()方法不会被调用
bool a = true;
bool b = SomeMethodThatMayThrowException();
if (a || b) 
{
    // 这段代码会执行，因为a是true，所以b不会被评估
}
  • 分析：短路现象的技术原理
    • 逻辑运算符总是先对左侧的表达式求值，再对右边的表达式求值，这个顺序是保证的，因此对于 &&，如果左边的表达式结果为 false，就不再对右边的表达式求值；
  • 结论：
      1. 总是使用短路版本​：
      • 在大多数情况下，应该优先使用 && 和 || 而不是非短路版本的 & 和 |，因为前者更高效且更安全；
      1. 利用短路进行空值检查​：
      • 短路特性常用于安全地检查对象是否为 null 后再访问其成员；
  • 区别：短路和非短路运算符
    • C# 中除了短路逻辑运算符 && 和 || 外，还有非短路的 & 和 | 运算符；
      1. && 和 || 会短路，而 & 和 | 会对所有操作数进行求值；
      1. && 和 || 只能用于布尔操作数，而 & 和 | 可以用于整数类型的位运算；
      1. 在布尔上下文中，& 和 | 会计算所有操作数，即使结果已经确定；
  • 补充：
    • CG 语句当中的运算符，不存在短路现象

函数

概念：CG 语法当中的函数 - 无返回值

  • 语法：
void name(in 参数类型 参数名, out 参数类型 参数名)
{
	// 函数体
}
  • 参数：
    • void：无返回值；
    • name：函数名；
    • in：表示是输入参数，表示由函数外部传递给函数内部，内部不会修改该参数，只会使用该参数进行计算，允许有多个；
    • out：表示是输出参数，表示由函数内部传递给函数的调用者，在函数内部必须对该参数值进行初始化或修改，允许有多个；
  • 举例：
void test(int fixed inF, out fixed outF)
{
	outF = inF + 10;
}

v2f vert(appdata v)
{
	fixed f = 10;
	fixed f2;
	test(f,f2);
	// 之后：f2的值为20
}
  • 注意：
    • in 和 out 都可以省略，省略后就没有了 in 和 out 相关的限制；
    • 虽然可以省略，但是建议大家在编写 shader 时不要省略 in 和 out，因为他们可以明确参数的传递方式，提高代码的可读性和可维护，可以让我们更容易的理解函数是如何与参数交互的，减少潜在的误解可能；

概念：CG 语法当中的函数 - 有返回值

  • 语法：
type name(in 参数类型 参数名)
{
	// 函数体
	return 返回值;
}
  • 注意：
    • 虽然可以在有返回值的函数中使用 out 参数，但是这并不是常见做法；
    • 除非是一些自定义逻辑函数，对于 顶点/片元着色器函数 只会使用单返回值的方式进行处理；
  • 举例：
float test2(in float inF)
{
	return inF * 2;
}

顶点与片元着色器语法

顶点与片元着色器基本结构

代码

Shader "Unlit/Test"
{
	Properies
	{
	
	}
	SubShader
	{
		Pass
		{
			CGPROGRAM
			#prama vertex myVert
			#prama fragment myFrag
			
			// 这是一个顶点着色器的回调函数
			// 顶点着色器返回的结果是一个四维的坐标
			// 这里用到了“语义”的语法，表示当前的变量 v 是模型顶点的坐标，如果没有这些语义，那么渲染器就完全不知道渲染器输入输出的是什么，因此 Shader 就完全没有作用
			// POSITION：把模型的顶点坐标填充到输入的参数 v 当中，其会将模型顶底的坐标传递到这个 v 变量上
			// SV_POSITION：用于表示顶点着色器输出的内容，是裁剪空间中的顶点坐标
			float4 myVert(float4 v : POSITION) : SV_POSITION
			{
				// 老版本：
				// 用系统自带的内置函数 mul，而 MVP 矩阵是模型内置的矩阵，将其得到在裁剪空间的坐标
				// return mul(UNITY_MATRIX_MVP,v)
				
				// 新版本：
				// 和上面的老版本的矩阵乘法的效果一样，新版本只是将其封装了起来
				return UnityObjectToClipPos(v);
			}
			
			// 片元着色器回调函数
			// SV_Target：告诉渲染器，把用户输出颜色存储到一个渲染目标中，这里将输出到默认的帧缓存中
			fixed4 myFrag() : SV_Target
			{
				// 返回红色
				return fixed4(1,0,0,1);
			}
			
			ENDCG
		}
	}
}

语义

概念：语义

  • 概念：
    • CG 语言中提供了语义这种特殊关键字，用于修饰函数中的传入参数和返回值；
    • 它的主要作用是让 Shader 知道从哪里读取数据，并把数据输出到哪里，让我们在 Shader 开发当中可以获取到想要的数据，并且可以把数据传递出去；
    • 即：语义是赋给 Shader 输入和输出的字符串标识，用于表达参数的含义和数据流向；
  • 注意：
    • Unity 中只支持 CG 当中的部分语义；
  • 语法：语义通常附加在变量声明之后，使用冒号分隔 
struct a2v {
    float4 vertex : POSITION;    // 使用POSITION语义声明顶点位置
    float3 normal : NORMAL;      // 使用NORMAL语义声明法线
    float4 texcoord : TEXCOORD0; // 使用TEXCOORD0语义声明纹理坐标
};

概念：常用语义 - 应用阶段 -> 顶点着色器

  • 概念：
    • CPU 准备模型数据（顶点位置、法线、UV 等），这些数据通过语义标记后传递给 GPU；
    • 应用阶段传递模型数据给顶点着色器时 Unity 支持的语义；
    • 一般在顶点着色器回调函数的传入参数中应用；
  • 分类：
    • POSITION：模型空间中的顶点位置，通常是 float4 类型；
    • NORMAL：顶点法线，通常是 float3 类型；
    • TANGENT：顶点切线，通常是 float4 类型；
    • TEXCOORDn：比如 TEXCOORD0, TEXCOORD1、....
      • 该顶点的纹理坐标，通常是 float2 或者 float4 类型；
      • TEXCOORD0、TEXCOORD1 等语义的本质是声明一个插值寄存器，用于在顶点着色器和片元着色器之间传递数据，​不限制数据类型的具体维度；
      • 例如，TEXCOORD2 可以存储 float3、float4 甚至 float2，只要总数据量不超过寄存器容量（通常为4个浮点数）；
      • TEXCOORD0 表示第一组纹理坐标，依次类推；
      • 纹理坐标：也称 UV 坐标，表示该顶点对应纹理图像上的位置；
    • COLOR：顶点颜色，通常是 fixed2 或者 fixed4 类型；

概念：常用语义 - 顶点着色器 -> 片元着色器

  • 概念：
    • 顶点着色器处理带 POSITION 语义的顶点数据，进行坐标空间转换（模型空间→世界空间→视图空间→裁剪空间），输出带 SV_POSITION 语义的裁剪空间坐标；
    • 从顶点着色器传递数据给片元着色器时 Unity 支持的语义；
    • 一般在顶点着色器回调函数的返回值中应用；
  • 分类：
    • SV_POSITION：
      • 裁剪空间中的顶点坐标 (必备)；
    • COLOR0：
      • 有时候可能会在顶点着色器阶段处理一些颜色相关的数据，因此可以用此 COLOR0，通常用于输出第一组顶点颜色，不是必须的；
    • COLOR1：
      • 通常用于输出第二组顶点颜色 (不是必须的)；
    • TEXCOORD0~7：
      • 通常输出纹理坐标，不是必须的；

概念：常用语义 - 片元着色器的输出

  • 概念：
    • 片元着色器输出时 Unity 支持的常用语义；
    • 一般在片元着色器回调函数的返回值中应用；
  • 分类：
    • SV_Target：输出值会存储到渲染目标中；
    • SV_Depth：数据类型为 float，表示自定义深度值（可选）；

顶点与片元着色器传递更多参数

概念：顶点着色器获取更多参数

  • 概念：
    • 当我们在顶点着色器当中想要获取更多模型相关信息时，可以使用结构体对数据进行封装，通过对结构体中成员变量加语义的方式，来定义想要获取的信息；
  • 代码：
Shader "Unlit/Test"
{
	Properies
	{
	
	}
	SubShader
	{
		Pass
		{
			CGPROGRAM
			#prama vertex vert
			#prama fragment frag
			
			// a：应用阶段 v：顶点阶段 a2v：从应用阶段拿数据输入到顶点阶段
			struct a2v
			{
				// 顶点坐标
				float4 vertex : POSITION; 
				
				// 顶点法线
				float3 normal : NORMAL;
				
				// 纹理坐标(uv坐标)
				float2 uv : TEXTCOOD0;
			};
			
			float4 vert(a2v data) : SV_POSITION
			{
				// 获得数据要通过点的方式获得
				return UnityObjectToClipPos(data.vertex);
			}
			
			fixed4 frag() : SV_Target
			{
				// 返回红色
				return fixed4(1,0,0,1);
			}
			
			ENDCG
		}
	}
}

概念：片元着色器获取更多参数

  • 概念：
    • 和顶点着色器一样，需要更多参数时、需要封装结构体；
    • 但需要注意的是，片元着色器中获取的数据基本上都是由顶点着色器传递过来的，所以我们封装的结构体还需要作为顶点着色器的返回值类型；
  • 代码：
Shader "Unlit/Test"
{
	Properies
	{
	
	}
	SubShader
	{
		Pass
		{
			CGPROGRAM
			#prama vertex vert
			#prama fragment frag
			
			// a：应用阶段 v：顶点阶段 a2v：从应用阶段拿数据输入到顶点阶段
			struct a2v
			{
				// 顶点坐标
				float4 vertex : POSITION; 
				
				// 顶点法线
				float3 normal : NORMAL;
				
				// 纹理坐标(uv坐标)
				float2 uv : TEXCOORD0;
			};
			
			// 从顶点着色器传递给片元着色器的 结构体数据同样这里面的成员也需要用语义去进行修饰
			struct v2f
			{
				// 裁剪空间下的坐标
				float4 position : SV_POSITION;
				
				// 顶点法线
				float3 normal : NORMAL;
				
				// 纹理坐标
				float2 uv : TEXCOORD0;
			};
			
			// 更改返回值类型，在函数当中去存储需要传递过去的值
			v2f vert(a2v data)
			{
				// 需要传递给片元着色器的数据
				v2f v2fdata;
				v2fdata.vertex = UnityObjectToClipPos(data.vertex); 
				
				// 这两个数据直接转存
				v2fdata.normal = data.normal;
				v2fdata.uv = data.uv;
				
				return v2fdata;
			}
			
			// 在调用顶点着色器时，其返回值会传递到这个v2f的参数data当中
			fixed4 frag(v2f data) : SV_Target
			{
				// 返回红色
				return fixed4(1,0,0,1);
			}
			
			ENDCG
		}
	}
}
  • 总结：
    • 如果顶点/片元着色器想要传递更多参数我们需要通过结构体进行封装，用语义修饰结构体成员变量来达到目的
    • 注意只有顶点/片元着色器的回调函数相关参数和返回值才需要通过语义修饰，一般的自定义函数是不需要语义的，因为我们自己调用自己的自定义函数是可以明确知道每个参数的作用的；

struct appdata_base {

};

ShaderLab 属性类型和 CG 变量类型的匹配关系

分析：ShaderLab 属性类型和 CG 变量类型的匹配关系

  • 概念：两种类型
      1. ShaderLab 属性：ShaderLab 是 Unity 的材质属性声明语言，用于在 Inspector 面板中定义可调节的材质参数，面向美术和设计师，提供直观的 UI 控件，与 Unity 的材质系统（Material）绑定，可以在运行时动态修改；
      1. CG/HLSL：CG/HLSL 是 GPU 着色器编程语言，用于编写顶点/片元着色器的计算逻辑，直接运行在 GPU 上，需要精确控制数据类型（如精度 float / half / fixed）；
  • 概念：CG 中变量类型的对应 ShaderLab 的属性类型
    • 分析：
      • Unity 在编译 Shader 时，会自动将 ShaderLab 属性转换为 CG/HLSL 变量，以便在着色器代码中使用；
      • 例如：ShaderLab 的 Color 对应 CG 的 float4，2D 对应 sampler2D；
    • 总结：
  • 概念：如何在 CG 语句块中使用 ShaderLab 属性
    • 概念：
      • 直接在 Pass 中声明对应 ShaderLab 属性的同名变量即可；
      • 只要 Pass 中声明的变量和 ShaderLab 属性同名，则下面就可以直接使用；
    • 示例：
Shader "Unlit/Test"
{
    Properties
    {
        _MyInt("MyInt",Int) = 1
        _MyFloat("MyFloat",Float) = 1
        _MyRange("MyRange",Range(0,5)) = 1

        _MyColor("MyColor",Color) = (0,0,0,0)
        _MyVector("MyVector",Vector) = (1,2,0,0)

        _My2D("My2D",2D) = ""{}
        _My2DArray("My2DArray",2DArray) = ""{}
        _MyCube("MyCube",Cube) = ""{}
    }
    SubShader
    {
        Tags { "RenderType"="Transparent" }
        LOD 100

        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #include "UnityCG.cginc"
            
			float _MyInt;
			float _MyFloat;
			fixed _MyRange;
            
            fixed4 _MyColor;
            float4 _MyVector;
            
            sampler2D My2D;
            samplerCUBE _MyCube;
            
            struct a2v
            {
                float4 vertex : POSITION;
                float3 normal : NORMAL;
                float2 uv : TEXCOORD0;
            };

            struct v2f
            {
                float4 position : SV_POSITION;
                float3 normal : NORMAL;
                float2 uv : TEXCOORD0;
            };
            
            v2f vert(a2v data)
            {
                v2f v2fdata;
                v2fdata.position = UnityObjectToClipPos(data.vertex);
                v2fdata.normal = data.normal;
                v2fdata.uv = data.uv;
                return v2fdata;
            }

            fixed4 frag(v2f data) : SV_Target
            {
                return _MyColor; 
            }

            ENDCG
        }
    }
}

CG 内置函数与文件

CG 内置函数

概念：CG 内置函数

  • 概念：CG 内置函数是什么
    • Unity 中的 CG (或 HLSL)内置函数是 GPU 着色器编程中预先定义好的函数库，它们为开发者提供了高效的数学运算、几何计算、纹理操作等功能，可以直接在 Shader 中使用而无需自行实现；
    • 这些函数通常与 GPU 指令直接对应，执行效率非常高；
  • 概念：数学函数相关
    • 三角函数：
      • 补充：sincos 函数特性
        • 唯一同时输出两个结果的函数，通过引用参数返回；
        • float s, c; sincos (x, s, c);
        • 其中 s=sin (x), c=cos (x)；
    • 向量|矩阵：
      • 补充：
        • 乘仅支持三维向量，其他函数（如点乘）支持任意相同维度向量；
    • 数值相关：
  • 概念：纹理函数
    • 二维纹理：
      • 这些纹理采样函数返回值为 fixed4 类型的颜色值；
        1. tex2D(sampler2D tex, float2 s)：二维纹理查询；
        1. tex2D(sampler2D tex, float2 s, float2 dsdx, float2 dsdy)：使用导数值查询纹理采样（自动计算 mipmap 层级）；
        1. tex2D(sampler2D tex, float3 sz)：二维纹理查询+深度值比较（需硬件支持）；
        1. tex2D(sampler2D tex, float3 sz, float2 dsdx, float2 dsdy)：带导数的深度比较纹理采样；
    • 立方体纹理：
      • texCUBE(samplerCUBE tex,float3 s)：查询立方体纹理；
      • texCUBE(samplerCUBE tex,float3 s,float3 dsdx, float3 dsdy)：查询立方体纹理；
    • 其他纹理：
      • tex1D(sampler1D tex, float s)：基础一维纹理查询；
      • tex1D(sampler1D tex, float s, float dsdx, float dsdy)：使用导数值（dsdx/dsdy）查询一维纹理；
      • tex1D(sampler1D tex, float2 sz)：一维纹理查询+深度值比较（sz.z 为比较值）；
    • 总结：方法规律
      • texXXX(samplerXXX,floatX,[dsdx,dsdy],[depth])；
  • 举例：从纹理中提取颜色
fixed4 frag(v2f data) : SV_Target
{
	// 从一个二维纹理当中，根据UV坐标、取出颜色
	fixed COLOR = tex2D(_My2D,data.uv)
	return color;
}

CG 内置文件

概念：CG 内置文件

  • 概念：
    • 我们可以在 Unity 的安装目录中找到 CG 内置文件，在 Editor->Data->CGIncludes 中可以找到；
    • 后缀为 cginc 的文件，其为 CG 语言内置文件后缀；
    • 后缀为 glslinc 的文件，其为 GLSL 语言的内置文件；
    • 他们是预定义的 Shader 文件，里面包含了一些已经写好的 Shader 相关逻辑；
    • 其作用和 CG 内置函数一样，可以提升我们的 shader 开发效率，可以直接使用其中的方法等内容来进行逻辑开发；
  • 概念：cginc 文件
    • 专为 NVIDIA 的 C for Graphics (CG)​ 语言设计，是一种面向 GPU 编程的高级着色器语言，语法类似 C 语言但针对图形渲染优化；
    • 包含预定义的结构体、辅助函数、宏​（如坐标变换、光照计算等）以及全局变量​（如 UNITY_MATRIX_MVP 用于模型-视图-投影矩阵变换）；
    • 比如 Unity Shader 开发通过 #include "UnityCG.cginc" 引入，简化顶点/片元着色器编写；
  • 概念：glslinc 文件
    • 用于 OpenGL Shading Language (GLSL)​，是 OpenGL/WebGL 的标准着色器语言， OpenGL/WebGL开发通过 #include "common.glslinc" 引入共享代码；

概念：常用内置文件

  • 概念：UnityCG.cginc
    • 包含最常用的帮助函数、宏和结构体等；
  • 概念：Lighting.cginc
    • 包含各种内置光照模型；
    • 如果编写的是 surface shader (标准表面着色器)，会自动包含进来；
  • 概念：UnityShaderVariables.cginc
    • 编译 Unityshader 时，会自动包含进来。包含许多内置的全局变量；
  • 概念：HLSLSupport.cginc
    • 编译 Unityshader 时会自动包含进来。声明了很多用于跨平台编译的宏和定义；
    • HLSLSupport.cginc 自动处理不同平台的宏定义差异（如 DirectX/OpenGL）；

概念：使用 CG 内置文件

  • 概念：
    • 在 CG 语句块中进行引用，通过编译指令：#include "内置文件.cginc" 的形式来进行引用；
    • 接下来就可以在 CG 语音中使用其中内容；
  • 举例：
    • #include "UnityCG.cginc"

概念：CG 内置文件常用内容

  • 概念：常用方法 - UnityCG.cginc
      1. float3 WorldSpaceViewDir(float4 v)：
      • 输入一个模型空间中的顶点位置，返回世界空间中从该点到摄像机的观察方向；
      1. float3 ObjSpaceViewDir(float4 v)：
      • 输入一个模型空间中的顶点位置，返回模型空间中从该点到摄像机的观察方向；
      1. float3 WorldSpaceLightDir(float4 v)：
      • 仅用于向前渲染中，输入一个模型空间中的顶点位置，返回世界空间中从该点到光源的光服方向；
      1. float3 ObjSpaceLightDir(float4 v)：
      • 仅用于向前渲染中，输入一个模型空间中的顶点位置，返回模型空间中从该点到光源的光方向；
      1. float3 UnityObjectToWorldNormal(float3 norm)：
      • 把法线方向从模型空间转换到世界空间中；
      1. float 3 UnityobjectToWorldDir (in float 3 dir)：
      • 把方向矢量从模型空间转换到世界空间中；
      1. float 3 UnityWorldToobjectDir (float 3 dir)：
      • 把方向矢量从世界空间转换到模型空间中；
  • 概念：常用结构体 - UnityCG.cginc
      1. appdata_base：用于顶点着色器输入，顶点位置、顶点法线、第一组纹理坐标；
      1. appdata_tan：用于顶点着色器输入，顶点位置、顶点法线、顶点切线、第一组纹理坐标；
      1. appdata_full：用于顶点着色器输入，顶点位置、顶点法线、顶点切线、四组 (或更多)纹理坐标；
      1. appdata_img：用于顶点着色器输入顶点位置、第一组纹理坐标；
      1. v2f_img：用于顶点着色器输出，裁剪空间中的位置，纹理坐标；
  • 概念：常用矩阵
  • 补充：
    • Object2World：当前的模型矩阵，用于将顶点/方向矢量从模型空间变换到世界空间；
