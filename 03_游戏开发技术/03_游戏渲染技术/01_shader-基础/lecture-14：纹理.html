Lecture 14：纹理

纹理基础概念

纹理压缩格式

概念：纹理压缩格式

  • 概念：Texture Compression - 纹理压缩
    • 设置发布后的纹理压缩格式。默认使用 Use PlayerSetting 中设置的压缩格式，纹理将遵循在 ​ Edit -> Project Settings -> Player -> Other Settings ​ 中为对应平台（如 Android）设置的默认纹理压缩格式；
    • ETC：
      • Ericsson Texture Compression，ETC1 ​ 是初始版本，一个主要限制是不支持透明通道（Alpha 通道）​；
      • 对于不透明的纹理（如漫反射贴图、法线贴图），它是非常安全的选择。主要用于需要最大限度兼容老旧 Android 设备的不透明纹理，对于带透明度的纹理，需要将颜色和 Alpha 通道拆分成两张 ETC1 纹理，这会增加工作量和不便；
    • ETC2：
      • ETC1 的升级版，是 ​ OpenGL ES 3.0 ​ 的标准功能，支持透明通道（Alpha 通道）​，并且压缩质量比 ETC1 更好，​ Android 开发的首选格式，可以统一设置为 ETC2，并为不支持 ETC2 的老设备在 ​​“ETC2 fallback”​​ 选项中设置一个后备格式，这样就能在保证新设备高质量的同时兼容老设备；
    • ASTC：
      • 提供从高压缩率（低质量）到低压缩率（高质量）的多种块大小选项（如 4x4, 6x6, 8x8 等），开发者可以根据纹理内容在质量和大小之间进行精细权衡；
      • 支持透明通道和各种颜色格式，质量通常优于 ETC2，需要设备硬件支持，并非所有 Android 设备都支持，但近年来主流中高端芯片均已支持；
    • DXT：
      • S3 Texture Compression，也称为 S3TC，是 PC 和游戏主机（如 Xbox, PlayStation）上的传统标准格式，​主要用于 PC 和主机平台的游戏开发。在移动平台（Android/iOS）上应避免使用；
    • PVRTC：
      • PowerVR Texture Compression，是苹果设备（iOS, tvOS）芯片（历代 A 系列芯片）的专属原生格式，在 iPhone 和 iPad 上具有最佳的兼容性和性能；
  • 概念：ETC2
    • 分类：
      • 32-bit：32 位
        • 以全分辨率存储 RGBA 纹理，使用 32 位存储每个像素，质量最高内存占用最多；
      • 16-bit：16 位 -
        • 以全分辨率存储 RGBA 纹理，使用 16 位存储每个像素，质量较低，纹理可能会丢失有价值的颜色信息，内存占用相对较少；
      • 32-bit, half resolution：32 位
        • 半分辨率以半分辨率存储 RGBA 纹理，使用 32 位存储每个像素，由于是半分辨率，纹理会变得模糊，内存占用较小；
    • 概念：
      • ETC2 算法的基础操作单元是一个 4x4 的像素块，核心思想是为每个 4x4 块计算出一个或两组“基色”，然后块内每个像素的颜色通过选择基色或从基色插值得到的颜色来表示；
      • 编码器会尝试多种分区模式（如不分区、水平分区、垂直分区等），为整个块或每个分区生成两组基色（RGB444 或 RGB555 格式）。每个像素存储一个几位（例如 2 位或 3 位）的“调制索引”，通过查找预定义的调制表，从一个或两组基色中计算出最终颜色；
      • 同时，为了更精确地表示颜色，ETC2 引入了差分模式。在这种模式下，存储的两组基色不是独立的，而是存在关联。例如，第二组基色可以通过对第一组基色的 RGB 分量分别加上一个差分值得到。这种方式可以在不显著增加数据量的前提下，提供更丰富的颜色变化范围
  • 总结：
    • 对于安卓而言，统一可以使用 ETC2，然后再在 ETC2 Fallback 中设置一个后备格式：
总结：关于纹理压缩
    1. 如果使用的不是 ETC2 压缩，大多数压缩格式，使用 32-bit 备用取决于 GPU 支持什么；
    1. 平台默认纹理压缩格式：
    • Android：RGB Compressed ETC [RGB 24 bit]、RGBA Compressed ETC [RGBA 32 bit]；
    • IOS：RGB Compressed PVRTC 4 bit、RGBA Compressed PVRTC 4 bit；
    1. 平台不支持指定纹理压缩格式会发生什么？
    • 对纹理资源进行重新解压为 RGBA 32，耗时；
    1. 对之前解压的不支持的部分和新解压的部分都存储；
    • 内存损失；
    1. 纹理压缩对纹理尺寸有什么要求？
    • 理想情况：都是 2 的幂 (NPOT)
    • 非理想情况：
      • 如果平台支持：纹理大小所需内存会增加，采样可能更慢；
      • 平台不支持：对纹理进行缩放和填充达到下一个 2 的幂大小，占用更多内存，加载速度变慢；

图形流水线中的纹理

概念：图形流水线中的纹理

  • 概念：应用程序数据加载阶段
    • 第一步：纹理加载
      • 将硬盘中的纹理资源加载至系统内存，再传输至 GPU 显存，供后续阶段使用；
      • 通过图形 API（如 OpenGL 的 glTexImage2D）将图像数据（如 PNG、JPG）转换为纹理对象；
    • 第二步：纹理传输
      • 为模型顶点关联纹理坐标（UV 坐标），定义纹理如何映射到表面；
      • 在顶点数据中附加 vec2 类型的 UV 坐标（范围 [0,1]），例如四边形顶点需定义四个角的 UV 值；
  • 概念：顶点着色器
    • 情况一：传递或变换纹理坐标，​不直接采样纹理​（因顶点着色器无纹理访问权限）
    • 情况二：动态生成 UV（如滚动效果： uv.x += time）
  • 概念：片元着色器阶段
    • 对纹理进行采样；

纹理分类

概念：纹理分类

  • 概念：颜色纹理
    • 颜色纹理是最基础的纹理类型，通过将二维图像（如 PNG/JPG）映射到物体表面，模拟材质的外观细节；
    • 其原理是通过 UV 坐标将纹理像素（texel）与模型表面点对应，在片元着色器中采样纹理颜色并与光照计算结合。作用包括增强视觉真实感（如木纹、金属锈迹）、减少几何复杂度（通过贴图替代建模细节）以及实现风格化渲染；
  • 概念：几何纹理（Geometric Texture）​
    • 几何纹理通过修改模型顶点位置直接改变几何形状，如雕刻细节或植被分布；
    • 其原理是顶点着色器中根据纹理值偏移顶点坐标，作用在于动态生成复杂表面（如地形起伏）或实现 GPU 驱动的细节层次（LOD），但会显著增加计算开销；
  • 概念：凹凸纹理（Bump Mapping）​​
    • 凹凸纹理通过扰动表面法线模拟微观凹凸，无需改变几何体；
    • 原理是使用高度图（Height Map）计算法线偏移量，在光照计算中伪造阴影与高光；
    • 作用为低成本增加表面粗糙度（如橘子皮），但轮廓仍保持平滑；
  • 改变：视差纹理
    • 视差纹理是凹凸纹理的进阶，根据视角和高度图动态偏移 UV 坐标，产生视差效果；
    • 原理是通过步进追踪（Ray Marching）模拟深度遮挡，作用在于增强立体感（如砖墙缝隙），比凹凸纹理更真实，但计算量更大；
  • 概念：法线纹理
    • 法线纹理存储切线空间下的法线向量，直接替换光照计算中的法线。原理是将 RGB 通道对应法线 XYZ 分量，通过切线空间转换到世界空间；
    • 作用是在低模上实现高模细节（如游戏角色肌肉线条），性能开销低且效果显著；

纹理的多级渐远 Mipmap

基础概念

概念：什么是纹理的多级渐远 Mipmap

  • 概念：
    • Mipmap 可理解为是纹理的 LOD，开启了 Mipmap 之后，大概会增加 30% 的显存；
    • Mipmap 的核心思想是预先生成一系列逐渐缩小的纹理，每一级的分辨率都是上一级的一半。例如一张 128×128 的原始纹理，会生成 64×64、32×32、16×16、8×8、4×4、2×2，直到 1×1 像素的一系列纹理；
  • 概念：Mipmap 的生成和过滤技术
    • Mipmap 的生成包含两个关键步骤：
      1. 滤波处理：
      • 均值滤波：取像素周围 3×3 区域的颜色平均值；
      • 高斯滤波：基于高斯分布计算权重，中心像素权重最高，边缘逐渐降低；
      1. 采样处理：
      • 二分下采样：隔行隔列取样；
      • 计算统计值：取 4 个相邻像素的均值；
  • 示例：
  • 特点：
      1. 可以解决当高分辨率纹理应用于远处物体时，会出现摩尔纹、锯齿和抖动的现象；
      1. 避免远处物体仍采样高分辨率纹理造成的带宽浪费和计算资源浪费；
      1. 保证物体在不同距离观察时的纹理表现自然过渡；

Unity 中使用 Mipmap

概念：Unity 中使用 Mipmap

  • 方法：在 Unity 中开启 Mipmap
  • 概念：tex2Dlod 函数
    • 语法：
      • fixed4 tex2Dlod(sampler2D s, float4 coord)
    • 解释：
      • s：要采样的 2D 纹理采样器；
      • coord：一个四维向量，xy 分量表示 UV 坐标，z 分量通常为 0，w 分量指定要使用的 Mipmap 层级 (LOD)；
      • 与常规的 tex2D 不同，tex2Dlod 允许开发者直接指定 Mipmap 层级；
      • 并且 tex2Dlod 是少数可以在顶点着色器阶段使用的纹理采样函数之一，这使得它特别适合用于程序化网格变形(如地形高度图采样)；
  • 代码：控制贴图的 Mipmap 效果
Shader "Unlit/Course_10_Mipmap"
{
    Properties
    {
        _MainTex ("Texture", 2D) = "white" {}
        [IntRange]_Mipmap("Mipmap",Range(0,12)) = 0
    }
    SubShader
    {
        Tags { "RenderType"="Opaque" }
        LOD 100

        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #include "UnityCG.cginc"

            struct appdata
            {
                float4 vertex : POSITION;
                float2 uv : TEXCOORD0;
            };

            struct v2f
            {
                float2 uv : TEXCOORD0;
                float4 vertex : SV_POSITION;
            };

            sampler2D _MainTex;
            float4 _MainTex_ST;
            half _Mipmap;

            v2f vert (appdata v)
            {
                v2f o;
                o.vertex = UnityObjectToClipPos(v.vertex);
                o.uv = TRANSFORM_TEX(v.uv, _MainTex);
                return o;
            }

            fixed4 frag (v2f i) : SV_Target
            {
                // tex2Dlod的第二个参数是一个四维向量
                float4 uvMipmap = float4(i.uv,0,_Mipmap);
                fixed4 c = tex2Dlod(_MainTex, uvMipmap);
                
                return c;
            }
            ENDCG
        }
    }
}

纹理串流系统

概念：纹理串流系统

  • 概念：
    • Unity 的纹理串流系统 (Texture Streaming System)，官方称为 Mipmap Streaming System，是 Unity 自 2018.3 版本引入的一项高级纹理内存管理技术，旨在优化纹理内存使用并提高加载性能；
    • 纹理串流系统是一种动态纹理管理技术，它根据摄像机与物体的距离，​智能加载所需精度的 Mipmap 层级，而非传统方式下将所有层级的 Mipmap 全部加载到显存中；
    • 这种机制可以显著减少纹理内存占用，特别是在处理高分辨率纹理时效果更为明显；
  • 作用：
      1. 初始加载时间缩短，因只需加载较低分辨率的 Mipmap；
      1. 运行时根据摄像机距离动态调整纹理精度；
      1. 减少 GPU 内存带宽使用，提高渲染效率；
  • 分析：
    • 在传统 Mipmap 机制中，启用 Mipmap 的纹理会预生成并加载所有层级的 Mipmap 到显存中，占用内存约为原始纹理的 4/3 倍（例如 2MB 纹理开启 Mipmap 后约占用 2.7MB）；
    • 而纹理串流系统则只加载当前需要的 Mipmap 层级，显著降低了内存占用；
  • 实现：
      1. Mipmap 层级计算​：
      • CPU 根据渲染物体与摄像机的距离，计算出当前需要加载的 Mipmap 层级（称为"x"级）
      • Mipmap 层级计算公式基于物体在屏幕空间中的覆盖面积与纹理分辨率的关系；
      1. 选择性加载​：
      • 系统仅加载 x 级及更高层级的 Mipmap（即 x, x+1, ..., n 级），而丢弃 0 到 x-1 级的 Mipmap；
      • 例如当计算出的层级 x=2 时，系统会加载 2 级及以上的 Mipmap，而 0 级和 1 级不会被加载到显存中；

纹理过滤

分析：为什么要纹理过滤

  • 引入：
    • 事实上没有一个纹理上的纹素是与屏幕上的像素是一一对应的；
    • 有可能当前像素上所对应的纹理，在纹理当中占多个像素，此时该选择哪一个作为采样值？
  • 概念：过滤方式
      1. Point (no flter) 无过滤
      • 不进行过滤，只采样离采样点最近的那一个纹素；
      1. Biinear 双线性过滤
      • 在采样点周围采样 4 个纹素，并进行加权平均得出最后的值 ；
      • 离采样点越近的纹素权重越高；
      1. Trilinear 三线性过滤
      • 在双线性过滤的基础上再加上对 Mipmap 的远近级别过滤；
      • 采样 8 个纹素, 然后进行插值；

纹理的环绕模式

概念：什么是纹理的环绕模式

  • 概念：纹理的环绕模式
    • 在 Unity 中，纹理的 Wrap Mode（包裹模式）决定了当纹理坐标超出 [0,1] 范围时纹理如何表现；
    • 这一设置对 3D 模型的纹理贴图重复或 2D 游戏中精灵 UV 超出边界时的视觉效果有重要影响；
  • 概念：Repeat 模式
    • 默认的纹理环绕模式，当纹理坐标超过 1 时，整数部分被舍弃，只使用小数部分进行采样，效果是纹理会不断平铺重复，创建无缝的外观；
  • 概念：Clamp 模式
    • 当纹理坐标超出 [0,1] 范围时，颜色会被"夹"到最边缘的像素颜色，会导致纹理在边缘出现一条明显的颜色线；
    • 当需要清晰边缘的 2D 精灵或 UI 元素，不希望纹理重复的特殊艺术风格时；
  • 代码：
Shader "Unlit/Course_10_Mipmap"
{
    Properties
    {
        _MainTex ("Texture", 2D) = "white" {}
        [KeywordEnum(Repeat,Clamp)]_WrapMode("WrapMode",int) = 0
        [IntRange]_Mipmap("Mipmap",Range(0,12)) = 0
    }
    SubShader
    {
        Tags { "RenderType"="Opaque" }
        LOD 100

        Pass
        {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #pragma shader_feature _WRAPMODE_REPEAT _WRAPMODE_CLAMP
            #include "UnityCG.cginc"

            struct appdata
            {
                float4 vertex : POSITION;
                float2 uv : TEXCOORD0;
            };

            struct v2f
            {
                float2 uv : TEXCOORD0;
                float4 vertex : SV_POSITION;
            };

            sampler2D _MainTex;
            float4 _MainTex_ST;
            half _Mipmap;

            v2f vert (appdata v)
            {
                v2f o;
                o.vertex = UnityObjectToClipPos(v.vertex);
                o.uv = TRANSFORM_TEX(v.uv, _MainTex);
                return o;
            }

            fixed4 frag (v2f i) : SV_Target
            {
                // WarpMode
                #if _WRAPMODE_REPEAT
                    i.uv = frac(i.uv);
                #elif _WRAPMODE_CLAMP
                    i.uv = clamp(i.uv,0,1);
                #endif
                
                float4 uvMipmap = float4(i.uv,0,_Mipmap);
                fixed4 c = tex2Dlod(_MainTex, uvMipmap);
                return c;
            }
            ENDCG
        }
    }
}

Cubemap 的采样方式

概念：Cubemap 的采样方式

  • 概念：
    • Cubemap 的核心采样原理是基于三维方向向量而非传统的 UV 坐标；
    • 在一个中心位于原点的单位立方体，任何从中心出发的方向向量都会与立方体的某一个面相交，这个交点就是采样点；
  • 分析：采样步骤
      1. 确定主方向轴​：找出方向向量中绝对值最大的分量（x、y 或 z），这决定了采样哪个立方体面；
      • 该分量的符号决定是正轴面还是负轴面，例如最大分量为正 X，则采样 +X 面；为负 X，则采样 -X 面；
      1. ​归一化处理​：将方向向量除以其最大分量的绝对值，将坐标映射到 [-1,1] 范围；
      • 例如，向量 (-0.8, 0.5, 0.4)，最大绝对值为 0.8（X 轴，负方向），计算后得到 (-1, 0.625, 0.5)；
      1. ​UV 坐标转换​：将三维坐标转换为二维 UV 坐标（范围 [0,1]），具体转换公式取决于所选的面；
      • 例如，对于 -X 面，常用 u = (z / |max| + 1) / 2 和 v = (y / |max| + 1) / 2；

  • 实现：在贴图的 Inspector 更改为以下属性即可；

法线纹理

法线纹理基础概念

概念：法线纹理基础概念

  • 作用：
    • 法线贴图（Normal Map）是一种通过修改表面法线方向来模拟凹凸细节的纹理技术，用于给几何物体添加细节；
  • 概念：法线贴图的数据形式
    • 法线贴图存储的是切线空间（Tangent Space）下的法线向量（RGB 对应 XYZ 分量），即：存储的是 XYZ 三个轴向上的 XYZ 的偏移数据，Z 轴表示的就是几何表面的高度信息；
    • 当存储一张贴图后，会将 XYZ 变成 RGB，而 XYZ 的取值范围为 [-1,1]，因此所以需要 xyz*0.5+0.5 来变换到 (0~1) 的区间；
    • 比如，通常情况下 xyz=(0,0,1) 转换成 RGB=(0.5,0.5,1)；

切线空间

概念：切线空间

  • 概念：
    • 切线空间 (Tangent Space) 是一个由顶点为原点组合而成的独立的坐标空间，在这个坐标空间中，X 轴代表顶点切线，Y 轴表示顶点副切线，Z 轴代表顶点法线；
    • 切线空间是 3D 图形学中用于描述模型表面局部几何特性的坐标系，由三个相互垂直的向量构成；
        1. 切线（Tangent, T）​​：与模型表面的 UV 坐标 U 轴方向对齐，表示纹理的水平展开方向；
        1. ​副切线（Bitangent, B）​​：与 UV 坐标 V 轴方向对齐，由切线与法线叉乘得到（即 B=T×N）；
        1. 法线（Normal, N）​​：垂直于模型表面，指向外部，是切线空间的 Z 轴；
    • 每个顶点拥有独立的切线空间，原点位于顶点位置，坐标系随表面曲率变化。切线和副切线的方向由模型的 UV 展开方向决定，确保纹理映射与几何结构一致；

    • 如果是在世界空间当中应用法线纹理中的法线信息进行计算，将会导致部分角度的凹凸效果错误；

法线纹理代码实现

概念：法线纹理代码实现

  • 概念：UnpackNormal
    • UnpackNormal 是 Unity Shader 中用于从法线贴图（Normal Map）采样并还原法线向量的核心函数，其作用是将存储在纹理中的压缩法线数据（通常为 RGB 颜色值）转换为切线空间下的三维法线向量（范围 [-1, 1]）；
    • 具体而言，法线贴图的像素值（[0,1]）需映射到法线向量的 [-1,1] 范围。例如，默认法线 (0,0,1) 对应贴图中的 (0.5,0.5,1)（蓝紫色）
    • 并且此方法自动处理不同平台的法线贴图压缩格式（如 DXT5nm、RGBA8），确保跨平台兼容性；
  • 实现：切线空间下法线纹理的使用
      1. 在 appdata 中定义 NORMAL 和 TANGENT 语义；
      • NORMAL 语义：模型空间法线向量；
      • TANGENT 语义：模型空间切线向量（含副切线方向符号 w 分量）；
      1. 在 v2f 中声明三个变量用于组成切线空间下的旋转矩阵；
      1. 在顶点着色器中执行以下代码：
      • 转换切线到世界空间：
        • half3 worldTangent = UnityObjectToWorldDir(v.tangent.xyz);
      • 计算副切线的方向符号：
        • fixed tangentSign = v.tangent.w * unity_WorldTransformParams.w;
      • 计算世界空间的副切线：
        • half3 worldBinormal = cross(o.worldNormal, worldTangent) * tangentSign;
      • 构建 TBN 矩阵的列向量；
        • o.tSpace0 = float3(worldTangent.x, worldBinormal.x, o.worldNormal.x);
        • o.tSpace1 = float3(worldTangent.y, worldBinormal.y, o.worldNormal.y);
        • o.tSpace2 = float3(worldTangent.z, worldBinormal.z, o.worldNormal.z);

            #pragma vertex vert
            #pragma fragment frag
            #pragma shader_feature _WRAPMODE_REPEAT _WRAPMODE_CLAMP

            {
                float4 vertex : POSITION;
                float2 uv : TEXCOORD0;

                float2 uv : TEXCOORD0;
                float4 vertex : SV_POSITION;
                float3 localPos : TEXCOORD1;
                float3 worldPos : TEXCOORD2;
                half3 worldNormal : TEXCOORD3;
                
                // 三个float3类型的变量构成切线转置矩阵
                float3 tSpace0 : TEXCOORD4;
                float3 tSpace1 : TEXCOORD5;
                float3 tSpace2 : TEXCOORD6;
            };

            sampler2D _MainTex;
            float4 _MainTex_ST;
            half _Mipmap;
            samplerCUBE _CubeMap;
            sampler2D _NormalTex;

            v2f vert (appdata v)
            {
                v2f o;
                o.vertex = UnityObjectToClipPos(v.vertex);
                o.uv = TRANSFORM_TEX(v.uv, _MainTex);
                o.worldPos = mul(unity_ObjectToWorld,v.vertex);
                o.localPos = v.vertex.xyz;
                o.worldNormal = UnityObjectToWorldNormal(v.normal);

                // 切线相关计算，主要是切线的转置矩阵的计算
                // UnityObjectToWorldDir 方法可以将本地空间下的切线，转化为世界空间下切线
                half3 worldTangent = UnityObjectToWorldDir(v.tangent);
                
                // 计算副切线
                // 通过切线和法线的叉积得到，其中 worldBinormal 表示方向。因为副切线在使用叉积计算时，其会有前后两个方向，因此需要用符号来指明具体哪个方向
                // 由 v.tangent.w 决定切线的方向
                fixed tangentSign = v.tangent.w * unity_WorldTransformParams.w;
                
                // 由叉积计算得到副切线
                half3 worldBinormal = cross(o.worldNormal,worldTangent) * tangentSign;

                // 填充转置矩阵
                o.tSpace0 = float3(worldTangent.x,worldBinormal.x,o.worldNormal.x);
                o.tSpace1 = float3(worldTangent.y,worldBinormal.y,o.worldNormal.y);
                o.tSpace2 = float3(worldTangent.z,worldBinormal.z,o.worldNormal.z);
                
                return o;

            {
                #if _WRAPMODE_REPEAT
                    i.uv = frac(i.uv);
                #elif _WRAPMODE_CLAMP
                    i.uv = clamp(i.uv,0,1);
                #endif
                
                float4 uvMipmap = float4(i.uv,0,_Mipmap);
                fixed4 c = tex2Dlod(_MainTex, uvMipmap);

                // 法线纹理
                fixed3 normalTex = UnpackNormal(tex2D(_NormalTex,i.uv));
                // Lambert模型是max(0,dot(N,L))
                fixed3 N_Normal = normalize(normalTex); // 当前法线贴图上定义的法线
                fixed3 L = _WorldSpaceLightPos0.xyz;

                // 通过矩阵相乘（利用点积进行计算）得到世界空间下的法线（法线纹理）
                half3 worldNormal = half3(dot(i.tSpace0,normalTex),dot(i.tSpace1,normalTex),dot(i.tSpace2,normalTex));

                // 第一个参数是立方体纹理，第二个参数是一个三维向量

                cubemap = texCUBE(_CubeMap,R);

反射环境

什么是反射环境

概念：什么是反射环境

  • 概念：
    • 在 Unity 中，​反射环境（Reflection Environment）​​ 是指通过模拟物体表面对周围环境的反射现象，增强场景真实感的技术体系。其核心是通过捕捉和存储环境光信息（如天空盒、静态物体或动态物体），并将其应用于材质表面，实现金属、玻璃等材质的镜面或模糊反射效果；
  • 实现：
    • 反射环境基于环境映射（Environment Mapping）​技术，主要依赖立方体贴图（Cubemap）​存储场景的六个方向视图（上下左右前后）；
    • 当光线照射到物体表面时，根据视角方向和法线计算反射向量，并通过该向量对 Cubemap 采样，获取反射颜色；

使用 Shader 对反射探针进行采样

概念：使用 Shader 对反射探针进行采样

  • 方法：
// 1. 计算当前像素点到摄像机的视线方向（世界空间）
half3 worldView = normalize(UnityWorldSpaceViewDir(i.worldPos));

// 2. 根据视线方向和表面法线计算反射向量（光路可逆）
half3 R = reflect(-worldView, N);  // N为预处理后的世界空间法线

// 3. 使用反射向量采样反射探针的立方体贴图
// unity_SpecCube0：当前激活的反射探针数据（含HDR信息）
half4 cubemap = UNITY_SAMPLE_TEXCUBE(unity_SpecCube0, R);

// 4. 解码HDR格式的立方体贴图，得到真实颜色值
// unity_SpecCube0_HDR：存储贴图的HDR解码参数
half3 skyColor = DecodeHDR(cubemap, unity_SpecCube0_HDR);
