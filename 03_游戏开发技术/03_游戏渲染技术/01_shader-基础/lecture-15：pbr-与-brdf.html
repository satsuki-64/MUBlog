PBR 基础概念

什么是 PBR

概念：什么是 PBR

  • 概念：
    • PBR (Physically Based Rendering)，中文译为基于物理的渲染；
    • PBR 是一种渲染方式，是使用基于物理原理和微平面理论的光照模型，以及使用从现实中测量的表面参数来准确表示真实世界材质的渲染理念；
    • PBR 是一系列技术的集合，包含 GI、PBS 等；
  • 引入：
    • Lecture 5：Physically Based Materials - 基于物理的材质
  • 概念：PBS - 基于物理的着色
    • PBS (Physically Based Shadering)，中文译为基于物理的着色；
    • PBS 是为了对光和材质之间进行更加真实的建模，实质上讲解基于物理的渲染本质就是 PBS；
    • Unity 中的 PBS 分为 Standard 和 Standard (Specular Setup)；

什么是 PBS

概念：什么是 PBS

  • 概念：
    • PBS 的核心理论包含五个部分；
      1. 物质的光学特性；
      1. 微平面理论；
      1. 能量守恒；
      1. 菲尼尔反射；
      1. 线性空间光照；
  • 概念：物质的光学特性
    • 现实世界中的物质根据导电性可分为三大类：绝缘体、半导体、导体；
    • 渲染领域中大多分为两大类：导体 (金属)、绝缘体 (非金属)；
    • 金属：
      • 金属的特点为：金属具有很高的反射率，其反射率大于 0.5。同时，金属会吸收出任何的折射光，因此金属不会出现任何的次表面散射和透明效果，并且具有丰富的高光颜色；
      • 并且，金属的所有可见颜色都来自反射；
    • 非金属：
      • 非金属具有很低的反射率，其反射率小于 0.06，并且非金属会产生高光反射和漫反射线性；
      • 非金属的高光反射为单色或者灰色；
  • 概念：微表面理论
    • 图示：
    • 分析：
      • 现实世界的表面大多都不是光学光滑的，这种微观几何上的变化会导致每个表面点对光有不同的反射和折射；
      • 所以基于物理渲染的 PBS 技术都是基于微平面理论的，它假想任何平面都是由微平面组成的，根据这些微平面粗糙程度的不同，采用粗糙度贴图或者高光度贴图来进行表示；
    • 图示：凹凸不平的表面
    • 概念：微平面理论
      • 一个平面越是粗糙，这个平面上的微平面排列就越是混乱；
      • 这样，光射到物体表面后，会产生角度混乱的反射，反射的角度也十分大；
      • 如果表面比较光滑，则高光反射的方向就也基本是一致的，则进入颜色的高光也会更加清晰；
  • 概念：能量守恒
    • 出射光线的能量永远不能超过入射光线的能量 (自发光体除外)；
    • 随着粗糙度的上升，镜面反射区域的面积会增加，基于能量守恒，故镜面反射区域的亮度则会降低；
    • 比如：假设当前由入射光，并产生漫反射和镜面反射
    • 则此时的 镜面反射+漫反射<=入射光；
  • 概念：菲尼尔反射
    • 菲涅尔效应是一种表示看到光线的反射率与视角相关的现象，也就是光源入射方向与平面法线方向夹角的对应关系。夹角越大，反射越大，亮度也就越大，反之夹角越小，反射就越小，亮度也就越小；
    • 而这效应中的反射率，就被称为菲涅尔反射；
  • 概念：线性空间光照
    • 为了保证光照渲染的正确性，所以最好是在线性空间中进行操作与计算，这样才能尽最大的还原现实世界中光与物质的交互；
    • 因此：Unity 的 Shader 当中的光照计算要放到线性空间当中；

BRDF

概念：BRDF - Bidirectional Reflectance Distribution Function

  • 概念：
    • BRDF 用于描述在某一个点，对于不同从不同的立体角入射光线之后，它所产生的反射的光线的能量；
    • 即：BRDF 用于描述物体表面如何在某个入射方向上，将光线反射到另一个方向；
    • BRDF 定义如何去分配光线反射到哪些方向、什么范围的方向，用于量化入射光方向到出射光方向的反射行为；
    • 比如：如果是镜面反射，则在镜面反射的方向上、聚集了入射的所有能量，反射到某一个反射的角度上；
    • 比如：如果是漫反射，则当前入射的能量将会被均等的分配到不同的范围方向上；
  • 目的：
    • BRDF 描述光线和物体是如何作用的，进而定义物体的不同材质；
  • 公式：
    • Radiance 可以通俗理解为“一个微小表面区域朝某个特定方向发出的光的‘亮度’”，Irradiance 衡量的是单位时间、单位面积上接收到的总光能流量，可以通俗理解为 “一个微小表面区域所接收到的光照‘强度’”；
    • BRDF 表示任何一个出射方向上的 Radiance 的微分，除以在入射点上、一个 Irradiance 的微分，即使用 Radiance 除以 Irradiance；
    • 其中 代表的就是 Radiance，其中 代表的就是 Irradiance；

迪士尼原则的 BRDF

引入：为什么需要迪士尼原则的 BRDF

  • 现在的微表面模型确实很强，但是并不适合表示真实的材质；
  • 比如：测量出来的物体表面的 BRDF，其找不到合适的微表面模型的参数来表示它；
    • 比如：一个刷了绿色油漆的木头，其不吸收光，但是因为光线为了进入油漆，就会形成一个高光，并且还会有一些光反射出来，因此会产生一些 Diffuse 的效果；
    • 而这个效果微表面模型不好表现，微表面模型只能表示一层的效果；
  • 并且：微表面模型不好用。对于艺术家而言，要选择各种各样的材质而言，由于是 PBR，因此其参数都是一些物理量，而这对于艺术家而言不友好；
    • 比如：对于金属材质而言，其金属度是一个复数 n-ik，而这个东西对于艺术家而言进行调整而言无法调整；
    • 而为艺术家提供合适的参数进行更快、更好的调整是一个很有意义的事情；
  • 因此还需要一类“艺术家友好”的材质，可以用艺术家们使用起来方便，而并不要求在物理上正确；

概念：迪士尼设计原则

    1. 看起来直观，易于使用；
    1. 不希望这个框架变得很复杂；
    1. 对于艺术家而言，设计出来的参数应当是以可视化的方式进行调整；
    1. 为了营造特殊效果，可以将属性设置为大于 1 或者小于 0；
    1. 不能因为组合效果而导致程序出现 Bug，而是得到不错的结果；

概念：迪士尼原则下的 BRDF 效果

  • 概念：
    • Subsurface：次表面散射 -> 类似于球被按扁的效果；
    • Specular：控制有多少镜面反射的效果；
    • Specular Tint：镜面反射反映出来的部分的颜色，是更偏白还是更偏其底下物体的颜色；
    • Roughness：粗糙程度；
    • Anisotropic：各向异性程度；
    • Sheen：在表面上有一层体积的效果，在 Grazing Angle 上有类似于雾化的效果；
    • Sheen Tint：在 Sheen 效果的位置上的颜色；
    • ClearCoat：镀膜；
    • ClearcoatGloss：光滑程度；
  • 注意：
    • 参数空间越大，越容易产生冗余 -> 对于不同参数的组合，可能造成和不调整之前的效果 -> 在各个不同的 Local 的范围内已经能得到类似的效果；
  • 特点：
    • 优点：
      • 艺术家友好，可视化、能看懂；
      • 能够以一套模型、实现很丰富的效果；
      • 开源实现；
      • 巨大的参数空间 -> 表现能力强；
    • 缺点：
      • 不是基于物理的实现；
      • 巨大的参数空间 -> 很多的冗余现象；

    • 迪士尼原则中的 BRDF 模型是拟合的能量守恒的模型；

Unity 中的 Gamma 与 Linear 空间

概念：Unity 中的 Gamma 与 Linear 空间

  • 概念：Unity 中的 Gamma 空间
    • 这张原始贴图，在 Unity 中 Gamma 空间时，无论是 SRGB 或者非 SRGB 存储，都会被认为是 SRGB 格式；
    • 因此，这张图片存储时会稍微亮一些，是一个凸函数曲线；
  • 概念：Unity 中的 Linear 空间 - sRGB 图片
    • sRGB 图片初始时是经过了 Gamma 矫正的，因此是一个凸函数。因此在进入 Shader 之前，需要先移除 Gamma 矫正；
    • 然后再在 Shader 处理之后，再加上 Gamma 矫正，使得其在显示器显示的时候、符合 Linear，最后被人眼看到的就是凹函数；
    • 在 Linear 空间下时，输入到 Shader 着色器当中的颜色是经过了 Gamma 矫正的，然后再在 Shader 处理完之后、进行一次 Gamma 矫正，使得显示器输出为线性的；
    • 因此黄框的部分就是 Unity 做的事情；
  • 概念：Unity 中的 Linear 空间 - 非 sRGB 图片
  • 图示：在 Unity 中的设置
面试题：在 Unity 当中，使用 sRGB 的图片在 Gamma 和 Linear 模式下，其图片的色彩是否是凸函数；
  • Gamma：
    • 在 Unity 的 Gamma 颜色空间​下，sRGB 图片存储的颜色值本身是经过 Gamma 编码（Encoding Gamma）的，其变换曲线是凹函数（Concave Function）；
    • 即 sRGB 图片的存储曲线是 ，这是一个指数小于 1 的幂函数，其图像是凹的。然后在 Unity 的 Gamma 工作流中，引擎不会对标记为 sRGB 的纹理进行任何转换处理，Shader 中采样得到的颜色值直接就是文件中存储的、经过 x0.45 编码的非线性值；
  • Linear：
    • 在 Unity 的 Linear 颜色空间​（线性工作流）下，当一张图片被标记为 ​ sRGB ​ 时，图片文件中存储的原始数据其编码曲线是凹函数，但 Unity 在采样时会对其进行转换，使其在线性空间内参与计算；
    • 当你在 Unity 中使用 ​ Linear Color Space ​（线性颜色空间），并将纹理的导入设置中的 ​ sRGB (Color Texture)​​ 选项勾选时，Unity 会指示图形 API（如 OpenGL 或 DirectX）在采样这张纹理时，​自动进行一个“解码”操作；
    • 这个解码操作近似于应用 ​​ ​​ 的变换。这是一个指数大于 1 的幂函数，其图像是凸的；

                // 是否开启球谐

                #endif

                float3 tSpace0 : TEXCOORD4;
                float3 tSpace1 : TEXCOORD5;
                float3 tSpace2 : TEXCOORD6;
              
                // 雾
                UNITY_FOG_COORDS(7)

            {
                float4 vertex : POSITION;
                float4 tangent : TANGENT;
                float3 normal : NORMAL;
                float4 texcoord : TEXCOORD0;
                float4 texcoord1 : TEXCOORD1;
                float4 texcoord2 : TEXCOORD2;

            v2f vert (appdata v)
            {
                v2f o;

                // _MainTex的UV

                fixed tangentSign = v.tangent.w * unity_WorldTransformParams.w; 
                // 世界空间下的副切线
                fixed3 worldBinormal = cross(worldNormal, worldTangent) * tangentSign;

                o.tSpace0 = float3(worldTangent.x, worldBinormal.x, worldNormal.x);
                o.tSpace1 = float3(worldTangent.y, worldBinormal.y, worldNormal.y);
                o.tSpace2 = float3(worldTangent.z, worldBinormal.z, worldNormal.z);

                o.worldNormal = worldNormal;

// 定义函数：计算基于粗糙度的环境镜面反射
// 参数：
//   UNITY_ARGS_TEXCUBE(tex) - 立方体贴图纹理和采样器
//   half4 hdr - HDR 编码参数，用于解码HDR颜色
//   Unity_GlossyEnvironmentData glossIn - 包含粗糙度和反射方向的数据结构
half3 Unity_GlossyEnvironment (UNITY_ARGS_TEXCUBE(tex), half4 hdr, Unity_GlossyEnvironmentData glossIn)
{
    // 从输入数据中获取感知粗糙度(perceptual roughness)
    // 感知粗糙度是艺术家友好的0-1线性值，不同于物理粗糙度
    half perceptualRoughness = glossIn.roughness /* perceptualRoughness */ ;

    // 以下是粗糙度重映射的两种方法，通过条件编译选择
    // 方法1：精确但计算复杂的物理正确方法（当前被禁用）
    // TODO: 警告：Morten的重映射可能仅适用于离线卷积，运行时卷积的影响待验证
    #if 0
        // 将感知粗糙度转换为物理粗糙度
        float m = PerceptualRoughnessToRoughness(perceptualRoughness);
        
        // 定义极小的epsilon值，避免除以零
        const float fEps = 1.192092896e-07F;
        
        // 将粗糙度重新映射为镜面反射功率(specular power)
        // 参考论文中的方程21：https://dl.dropboxusercontent.com/u/55891920/papers/mm_brdf.pdf
        float n = (2.0/max(fEps, m*m))-2.0;
        
        // 从n_dot_h公式重映射到n_dot_r公式
        // 参考：https://s3.amazonaws.com/docs.knaldtech.com/knald/1.0.0/lys_power_drops.html
        n /= 4;
        
        // 将结果重映射回感知粗糙度(平方根形式)
        perceptualRoughness = pow(2/(n+2), 0.25);
    #else
        // 方法2：近似但高效的粗糙度重映射公式
        // 这是一个与上面复杂计算近似的结果，但计算量小很多
        // 公式：r = r * (1.7 - 0.7 * r)
        // 由于粗糙度与反射探针的mip变化不星现线性正比，所以需要一个公式来改变
        perceptualRoughness = perceptualRoughness*(1.7 - 0.7*perceptualRoughness);
    #endif

    // 将感知粗糙度转换为mipmap级别
    // 粗糙度越大，使用的mipmap级别越高，反射越模糊
    half mip = perceptualRoughnessToMipmapLevel(perceptualRoughness);
    
    // 获取反射方向向量
    half3 R = glossIn.reflUVW;
    
    // 对tex进行贴图，只不过是根据LOD的级别来进行的采样
    // UNITY_SAMPLE_TEXCUBE_LOD宏处理不同平台的纹理采样差异，R代表采样坐标，mip代表采样的级别
    half4 rgbm = UNITY_SAMPLE_TEXCUBE_LOD(tex, R, mip);

    // 解码HDR颜色并返回结果
    // 这会将编码的HDR值转换为线性空间中的RGB颜色
    return DecodeHDR(rgbm, hdr);
}

struct SurfaceOutputStandard  
{  
    fixed3 Albedo;      // base (diffuse or specular) color  
    float3 Normal;      // tangent space normal, if written  
    half3 Emission;  
    half Metallic;      // 0=non-metal, 1=metal  
    // Smoothness is the user facing name, it should be perceptual smoothness but user should not have to deal with it.    // Everywhere in the code you meet smoothness it is perceptual smoothness    half Smoothness;    // 0=rough, 1=smooth  
    half Occlusion;     // occlusion (default 1)  
    fixed Alpha;        // alpha for transparencies  
};

// 基于物理渲染PBR的标准光照计算函数
// 输入参数：
//   s - 物体表面数据信息
//   viewDir - 当前像素点到摄像机的观察方向（世界空间）
//   gi - 全局光照数据的，包括GI漫反射和GI镜面反射
inline half4 LightingStandard(SurfaceOutputStandard s, float3 viewDir, UnityGI gi)
{
    // 归一化法线向量
    s.Normal = normalize(s.Normal);

	// 这部分函数（DiffuseAndSpecularFromMetallic）做了三件事情：
	// 1. 计算镜面反射颜色；2. 计算Albedo反射率；3. 计算菲尼尔反射率；
    // 从金属度分离漫反射和镜面反射颜色
    half oneMinusReflectivity;  // 1 - 菲涅尔反射率
    half3 specColor;            // 镜面反射颜色
    
    s.Albedo = DiffuseAndSpecularFromMetallic(
        s.Albedo,              // 基础颜色（可能含金属影响）
        s.Metallic,            // 金属度（0非金属，1金属）
        /*out*/ specColor,     // 输出的镜面反射颜色
        /*out*/ oneMinusReflectivity // 输出的漫反射权重
    );

    // 预乘Alpha处理，用于适配基于PBR的透明度混合模式处理，如果使用的是不透明材质，则以下计算不会发挥作用
    // 注：Unity标准着色器使用 _SrcBlend = One, _DstBlend = OneMinusSrcAlpha
    // 仅漫反射部分受透明度影响，镜面反射保持全强度
    half outputAlpha;          // 输出：最终透明度
    s.Albedo = PreMultiplyAlpha
    (
        s.Albedo,              // 原始颜色
        s.Alpha,               // 原始透明度
        oneMinusReflectivity,  // 漫反射权重
        /*out*/ outputAlpha    // 输出的预乘后透明度
    );

    // 这里就是具体的调用BRDF计算的函数
    // 调用Unity内置BRDF函数计算光照（Cook-Torrance模型）
    half4 c = UNITY_BRDF_PBS(
        s.Albedo,              // 预乘后的漫反射颜色，即物体表面的基础颜色
        specColor,             // 镜面反射颜色
        oneMinusReflectivity,  // 漫反射反射率，其中（oneMinusReflectivity）漫反射率 = 1- 镜面反射率
        s.Smoothness,          // 光滑度（控制高光锐度）
        s.Normal,              // 物体表面的空间法线
        viewDir,               // 观察方向
        gi.light,              // 直接光源数据（方向+颜色）
        gi.indirect            // 间接光数据（环境光+反射探针）
    );

    // 写入预乘后的透明度
    c.a = outputAlpha;
    
    return c;
}

// 从金属度属性，计算得到漫反射Diffuse和高光Specular
inline half3 DiffuseAndSpecularFromMetallic (half3 albedo, half metallic, out half3 specColor, out half oneMinusReflectivity)  
{
	// 镜面高光颜色 = 
	// 当 metallic=0 时（非金属或者绝缘体时），当前高光等于unity_ColorSpaceDielectricSpec.rgb，其表示非金属的默认反射率(4%左右)
	// unity_ColorSpaceDielectricSpec 表示绝缘体的通用反射颜色，为迪士尼经过大量测量得到，值为 0.04 左右
	// 当 metallic=1 时（金属时），当前直接返回albedo，即物体自身的颜色
    specColor = lerp (unity_ColorSpaceDielectricSpec.rgb, albedo, metallic);  
    
    // Reflectivity 表示反射率，而 oneMinusReflectivity 表示 1-反射率，即表示漫反射的反射率
    // 计算 1-反射率
    oneMinusReflectivity = OneMinusReflectivityFromMetallic(metallic);  
    
    // 最后返回的是：物体的基础颜色 * 漫反射反射率，最后得到物体的 Albedo
    return albedo * oneMinusReflectivity;  
}

inline half OneMinusReflectivityFromMetallic(half metallic)  
{  
    // 计算 1-反射率，其公式为：
    // 1-reflectivity = 1-lerp(dielectricSpec, 1, metallic) = lerp(1-dielectricSpec, 0, metallic)    
    // store (1-dielectricSpec) in unity_ColorSpaceDielectricSpec.a, then    //   1-reflectivity = lerp(alpha, 0, metallic) = alpha + metallic*(0 - alpha) =    //                  = alpha - metallic * alpha    half oneMinusDielectricSpec = unity_ColorSpaceDielectricSpec.a;  
    return oneMinusDielectricSpec - metallic * oneMinusDielectricSpec;  
}

// 主物理基于渲染（PBR）的BRDF实现
// 源于迪士尼（Disney）的工作，并以Torrance-Sparrow微表面模型为基础
//
// 核心公式：
//   BRDF = kD / pi + kS * (D * V * F) / 4
//   出射光强度 I = BRDF * (N·L) * 光源颜色
//
// 使用的模型：
// * NDF（法线分布函数）（取决于 UNITY_BRDF_GGX 定义）：
//  a) 归一化的BlinnPhong
//  b) GGX（更精确的物理模型）[1,5](@ref)
// * 可见性项（V）使用Smith模型[5](@ref)
// * 菲涅尔项（F）使用Schlick近似[1,2](@ref)
half4 BRDF1_Unity_PBS (half3 diffColor, half3 specColor, half oneMinusReflectivity, half smoothness,
    float3 normal, float3 viewDir,
    UnityLight light, UnityIndirect gi)
{
    // 将直观的“光滑度”转换为感知的“粗糙度”，便于计算
    // 感性粗糙度 = 1 - smoothness
    float perceptualRoughness = SmoothnessToPerceptualRoughness (smoothness);
    
    // 半角向量（H），位于光线方向(L)和视线方向(V)中间
    // light.dir：灯光的方向
    // viewDir：视角的方向
    // 灯光方向和视角方向相加，可以得到它们的半角向量
    // 这里的向量归一化使用的是 Unity_SafeNormalize，此方法保证分母不为0，而使用的一种安全的归一化
    float3 halfDir = Unity_SafeNormalize (float3(light.dir) + viewDir);

    // 对于可见像素，N·V（法线与视线点积）不应为负，但透视投影和法线贴图可能导致此情况。
    // 理想情况应修改法线使其面向摄像机以避免伪影，但这会增加少量ALU开销。
    // 替代方案是直接取N·V的绝对值（精确度较低但可行）。
    // 以下宏定义允许控制此行为。如果平台ALU资源紧张，可设置为0。
    // 此校正对于使用SmithJoint可见性函数的GGX尤其重要，因为粗糙表面的高光边缘伪影更明显。
    // 编辑：默认禁用此代码，因其与SpeedTree中使用的双面光照不兼容。
    #define UNITY_HANDLE_CORRECTLY_NEGATIVE_NDOTV 0

    #if UNITY_HANDLE_CORRECTLY_NEGATIVE_NDOTV
        // 根据法线与视线点积的值来计算法线应向视线方向偏移的量
        half shiftAmount = dot(normal, viewDir);
        normal = shiftAmount < 0.0f ? normal + viewDir * (-shiftAmount + 1e-5f) : normal;
        // 此处理论上应重新归一化法线，但为了节省ALU，我们省略此步（因为偏移量很小）。
        // normal = normalize(normal);

        float nv = saturate(dot(normal, viewDir)); // TODO: 此处的saturate可能非必要
    #else
	    // 使用绝对值来限制伪影
	    // 设备的性能比较差时，直接使用绝对值来确保N·V（法线与视线点积）不应为负
        half nv = abs(dot(normal, viewDir));    
    #endif

	// 各种方向的点击运算，都是后背会用到的点积向量
    // 计算法线与光线方向的点积(N·L)，并限制在[0,1]
    float nl = saturate(dot(normal, light.dir));
    // 计算法线与半角向量的点积(N·H)，并限制在[0,1]
    float nh = saturate(dot(normal, halfDir));

    // 计算光线方向与视线方向的点积(L·V)，并限制在[0,1]
    half lv = saturate(dot(light.dir, viewDir));
    // 计算光线方向与半角向量的点积(L·H)，并限制在[0,1]
    half lh = saturate(dot(light.dir, halfDir));

    // -------------------------- 漫反射项 --------------------------
    // 迪士尼原则的漫反射计算方法
    // 注：DisneyDiffuse 函数已包含复杂的散射计算，最后乘上N·L来模拟光照衰减
	// 注意：理论上，应在漫反射项除以Pi，而不是在镜面反射项乘Pi。但这样做会有两个问题：
    // 1. 会使着色器看起来比传统 Legacy 着色器暗很多；
	// 2. 在引擎端，当“非重要”光源被注入环境球谐（SH）时，它们也必须除以Pi
    half diffuseTerm = DisneyDiffuse(nv, nl, lh, perceptualRoughness) * nl;

    // -------------------------- 镜面反射项 --------------------------

	// 声明一个学术上的粗糙度，比感性粗糙度会更加精确一点
    // 将感知粗糙度转换为实际用于计算的粗糙度
    float roughness = PerceptualRoughnessToRoughness(perceptualRoughness);

	// 使用 GGX 作为发现分布，并实现光照模型
    #if UNITY_BRDF_GGX 
        // 粗糙度为0意味着完全没有镜面反射。使用max(roughness, 0.002)来匹配高清渲染管线(HDRenderLoop)的粗糙度重映射。
        roughness = max(roughness, 0.002);
        // 使用Smith联合GGX可见性项计算V
        float V = SmithJointGGXVisibilityTerm (nl, nv, roughness);
        // 使用GGX法线分布函数计算D
        float D = GGXTerm (nh, roughness);
    #else // Legacy 分支：使用传统模型
        half V = SmithBeckmannVisibilityTerm (nl, nv, roughness);
        half D = NDFBlinnPhongNormalizedTerm (nh, PerceptualRoughnessToSpecPower(perceptualRoughness));
    #endif

    // 根据Torrance-Sparrow模型，镜面反射项 = V * D * PI
    // 其表示DFV中的V项。最后乘以PI的原因，是因为上面漫反射部分本应该乘以的PI没有乘，因此在下面这部分需要补回来
    float specularTerm = V * D * UNITY_PI;

	    // 如果处于Gamma颜色空间，对镜面反射项取平方根以调整其亮度，使其在此空间下看起来更正确
	#   ifdef UNITY_COLORSPACE_GAMMA
	        specularTerm = sqrt(max(1e-4h, specularTerm));
	#   endif
	
	// 在某些情况下（特别是在Metal平台上），specularTerm * nl 可能为NaN
	// 使用max()确保其为一个合理的值，使得高光不会小于0
	specularTerm = max(0, specularTerm * nl);
	
	// 这个宏是材质上的是否启用高光的开关
	// 如果定义了关闭镜面高光的编译宏，则将镜面反射项强制设为0
	#if defined(_SPECULARHIGHLIGHTS_OFF)
	    specularTerm = 0.0;
	#endif
	
	    // -------------------------- 表面反射衰减系数 --------------------------
	    // surfaceReduction = Int D(NdotH) * NdotH * Id(NdotL>0) dH = 1/(roughness^2+1)[3](@ref)
	    // 该系数用于间接镜面反射计算，模拟粗糙表面对环境反射的衰减
	    half surfaceReduction;
	#   ifdef UNITY_COLORSPACE_GAMMA // Gamma空间下的近似公式
	        // 使用 1-0.28*x^3 来近似模拟 (1/(x^4+1))^(1/2.2) 在定义域[0;1]上的值，以节省计
	        surfaceReduction = 1.0 - 0.28 * roughness * perceptualRoughness;
	#   else // 线性空间下的精确公式
	        // 结果衰减范围在 [0.5, 1] 之间[3](@ref)
	        surfaceReduction = 1.0 / (roughness * roughness + 1.0);
	#   endif

    // 为了提供真正的Lambert漫反射光照，我们需要能够完全关闭镜面反射。
    // 如果specColor为0（例如纯黑色），则将镜面反射项乘以0
    specularTerm *= any(specColor) ? 1.0 : 0.0;

    // 计算边缘光泽项（grazingTerm），用于在视线与表面几乎平行时增强镜面反射（菲涅尔效应）[3](@ref)
    half grazingTerm = saturate(smoothness + (1 - oneMinusReflectivity));

    // -------------------------- 最终颜色合成 --------------------------

    // 1. 漫反射部分：表面漫反射颜色 * (全局间接漫反射光 + 直接光颜色 * 直接光的漫反射项)
    half3 diffuse = diffColor * (gi.diffuse + light.color * diffuseTerm);
    
    // 2. 直接镜面反射部分：镜面反射项 * 直接光颜色 * 菲涅尔项（使用半角向量LH）
    half3 specular = specularTerm * light.color * FresnelTerm (specColor, lh) 
    
    // 3. 间接镜面反射部分：表面衰减 * 全局间接镜面反射光 * 基于视角和边缘光泽项的菲涅尔插值
    half3 ibl = surfaceReduction * gi.specular * FresnelLerp (specColor, grazingTerm, nv); 

	// 最终颜色由三部分组成：漫反射部分 + 直接镜面反射部分 + 间接镜面反射部分
    half3 color = diffuse + specular + ibl;
    
    // 最后的BRDF最终返回的值，是一个表示颜色的四维向量
    // 其中返回的颜色的 alpha=1
    return half4(color, 1);
}

// Note: Disney diffuse must be multiply by diffuseAlbedo / PI. This is done outside of this function.  
half DisneyDiffuse(half NdotV, half NdotL, half LdotH, half perceptualRoughness)  
{  
    half fd90 = 0.5 + 2 * LdotH * LdotH * perceptualRoughness;  
    // Two schlick fresnel term  
    half lightScatter   = (1 + (fd90 - 1) * Pow5(1 - NdotL));  
    half viewScatter    = (1 + (fd90 - 1) * Pow5(1 - NdotV));  
  
    return lightScatter * viewScatter;  
}

D 项：法线分布函数

概念：BRDF 的高光反射项

  • 概念：D 项 - 法线分布函数
    • 分析：
      • 视线方向和光方向所形成半角向量，加入当半角项链和微平面法线相重叠时，此时表示当前光的射线可以经由当前表面上的点反射到眼睛当中，因此视线可以看到光的颜色；
      • 即：仅当 m=h 的表面点的朝向才会将光线 | 反射到视线 V，其它表面点对 BRDF 没有贡献；
      • 因此可以通过一个概念分布，来描述当前平面上法线的分布情况；
    • 图示：
      • 当前光的反射不能到达眼睛；
    • 公式：GGX 公式
    • 代码：Unity 中的 GGX 实现
inline float GGXTerm (float NdotH, float roughness)  
{  
	// a2 表示 a^2
    float a2 = roughness * roughness;  
    // NdotH 表示 n*h
    float d = (NdotH * a2 - NdotH) * NdotH + 1.0f; // 2 mad  
    // UNITY_INV_PI表示 1/Π  
    // 通过加1e-7f，表示一个非常小的值，用于保证分母不为零
    return UNITY_INV_PI * a2 / (d * d + 1e-7f);   
}
V 项：遮蔽阴影项

概念：V 项 - 遮蔽阴影项

  • 概念：
    • 在微表面理论中，​几何函数 G(l, v, h) ​​ 描述了微表面中同时未被入射方向 l 和观察方向 v 遮挡的微表面的比例；
    • 为了方便计算，许多现代实现（包括 Unity）常将几何函数 G 与 BRDF 分母中的校正因子 4 * (n·l) * (n·v) 合并为一个可见性项（Visibility Term）​，即 ​ V = G / (4 * (n·l) * (n·v))​​，即 V 项；
  • 代码：

// 此函数计算几何项G的可见性部分V，其中V = G / (4 * NdotL * NdotV)

{
// 原论文提供了两种实现，通过宏`#if 0`来切换选择
#if 0

    // lambda_v = (-1 + sqrt(a2 * (1 - NdotL2) / NdotL2 + 1)) * 0.5f;
    // lambda_l = (-1 + sqrt(a2 * (1 - NdotV2) / NdotV2 + 1)) * 0.5f;
    // 几何项 G = 1 / (1 + lambda_v + lambda_l);

    half lambdaV    = NdotL * sqrt((-NdotV * a2 + NdotV) * NdotV + a2);

// Unity中实际使用的版本
#else
    // ## 业界广泛采用的近似公式 (默认使用) ##

    // 当粗糙度a为0（完全光滑）时，LambdaV退化为NdotL * NdotV, LambdaL退化为NdotV * NdotL
    // 当粗糙度a为1（完全粗糙）时，LambdaV和LambdaL均退化为NdotL * NdotV

    float lambdaL = NdotV * (NdotL * (1 - a) + a);

// 针对不同平台微调极小值，防止除法除以零

#else

Unity 的 BRDF 中的间接镜面反射

概念：Unity 的 BRDF 中的间接镜面反射

  • 概念：
    • Unity 的 BRDF 中的间接镜面反射是使用 IBL 实现的，而 IBL 称之为 Image Based Lighting，即基于图像的光照；
  • 代码：
    • surfaceReduction * gi.specular * FresnelLerpFast (specColor, grazingTerm, nv)
  • 解释：
    • 其中的 gi.specular 表示 GI 计算得到的全局的高光镜面反射；
    • 但由于模型整体都可能具有高光镜面反射，因此需要进行高光效果的控制；
    • 其中：surfaceReduction 用于控制高光的衰减；
    • 其中：FresnelLerpFast 控制镜面反射在不同角度下的过渡；
    • 通过 surfaceReduction 和 FresnelLerpFast 一起控制，可以得到不同视角下的正确的 gi.specular 全局镜面光照效果；

代码：surfaceReduction 的计算

	// 1-0.28*x^3 as approximation for (1/(x^4+1))^(1/2.2) on the domain [0;1]  
	// 在 Grmma 空间需要使用感性的粗糙度 perceptualRoughness
	surfaceReduction = 1.0-0.28*roughness*perceptualRoughness;

	// 并且由于分母加了1.0，因此其值在0.5~1之间
	// 如果粗糙度为0时当前的 surfaceReduction 结果就为 1，因此当前全局光照的镜面高光就没有衰减
	surfaceReduction = 1.0 / (roughness*roughness + 1.0);           // fade \in [0.5;1]  
#   endif
