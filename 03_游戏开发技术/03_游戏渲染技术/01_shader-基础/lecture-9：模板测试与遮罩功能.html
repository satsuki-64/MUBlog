Lecture 9：模板测试与遮罩功能

模板测试基础概念

模板缓冲区

概念：什么是模板缓冲区

  • 概念：模板缓冲区
    • 模板缓冲区（Stencil Buffer）是 Unity 渲染管线中用于精细化控制像素可见性的关键技术，其核心作用是通过预设规则，动态的筛选或标记像素，实现高级视觉效果；
  • 作用：
    • 模板缓冲区是一个与屏幕像素逐一对应的 8 位整型缓存区（取值范围 0-255），每个像素存储一个独立的模板值 Stencil Value，它不直接参与颜色计算，而是作为“遮罩层”决定后续渲染中哪些像素允许被修改；
    • 模板测试（Stencil Test）发生在片段着色器之后、深度测试之前，若测试失败，则直接丢弃当前像素，避免不必要的深度计算和颜色写入；

什么是模板测试

概念：模板测试

  • 概念：
    • 模板测试是图形渲染管线中的一个关键阶段，位于片段着色器处理之后、深度测试之前；
    • 它的核心功能是通过比较预设的参考值与模板缓存中的值，决定像素是否被渲染到屏幕上，从而实现精细的像素级控制；
    • 模板测试基于缓存值执行逻辑判断，比如“仅渲染模板值= 1 的像素”；
  • 流程：
      1. 初始化：清空模板缓存（如设为 0）；
      1. 标记阶段：渲染遮罩物体，修改模板值（如 Pass Replace 将镜面区域设为 1）；
      1. 应用阶段：渲染目标物体，通过测试（如 Comp Equal）筛选像素；

Stencil 语法

概念：Stencil 语法

  • 语法：
Stencil 
{
    Ref referenceValue      // 参考值（0-255）
    ReadMask readMask       // 读取掩码（默认255）
    WriteMask writeMask     // 写入掩码（默认255）
    Comp comparisonFunction // 比较函数（如 Greater、Equal）
    Pass stencilOperation   // 测试通过时的操作（如 Replace、Keep）
    Fail stencilOperation   // 测试失败时的操作
    ZFail stencilOperation  // 模板通过但深度测试失败时的操作
}
  • 参数：
      1. Ref：定义用于比较的参考值（referenceValue），范围 0-255；
      • 比如：Ref 2 表示参考值为 2，将与模板缓冲区中的值比较；
      1. ReadMask/WriteMask：
      • ReadMask ​：按位掩码，用于过滤参考值和模板值的比较位，例如：ReadMask 127（二进制 01111111）会忽略最高位，默认 255（全部读取）；
      • WriteMask：控制写入模板缓冲区时的掩码操作，默认 255（全部写入）；
      1. Comp：定义参考值与模板值的比较规则
      • Greater（>）、GEqual（>=）、Less（<）、LEqual（<=）
      • Equal（=）、NotEqual（!=）
      • Always（总是通过）、Never（总是失败）
      • 只有通过模板测试，才能进行接下来像素的渲染；
      1. Pass/Fall/ZFail：
      • Pass：模板测试和深度测试均通过时，对模板值的操作；
      • Fail：模板测试失败时的操作（如 Keep 保留原值）；
      • ZFail：模板测试通过但深度测试失败时的操作；
  • 公式：
    • 左半部分：
      • Shader 中自定义的 Ref 值和 ReadMask 中的值进行与运算；
    • 右半部分：
      • 模版缓冲区中的值和 ReadMask 中的值进行与运算；
    • 中间部分：
      • 将左半部分和右半部分经过 & 运算之后的值，使用 Comp 定义的比较方法（大于等于或其他）来进行比较，如果符合比较方法，则表示通过测试；

代码实现

代码：Shader 中实现模板测试

Shader "taegc/UIGray"
{
	Properties
	{
		[PerRendererData]_MainTex("Tex",2D) = "white"{}
		_Color("Color",Color) = {1,1,1,1}
		_Ref("Stencil Ref",int) = 0
		
		[Enum(UnityEngine.Renderer.CompareFuction)]
		_StencilComp("Stencil Comp",int) = 0
		
		[Enum(UnityEngine.Renderer.StencilOp)]
		_StencilOp("Stencil Op",int) = 0
	}

	SubShader
	{
		// 注意：如果是用于UI的Shader，一般为"Queue" = "Transparent"
		Tags{"Queue" = "Transparent"}
		
		// 为了实现类似png的部分区域透明效果，要使用OneMinusSrcAlpha
		Blend ScrAlpha OneMinusSrcAlpha
		
		Stencil 
		{
			// 参考值（0-255），这里用_Ref属性的值去和模板缓冲区中的值进行比较
		    // 这里假设设置为1
		    Ref [_Ref]  
		    
		    // 十进制的 3 = 二进制的 00000011，因此如果当Ref=1、5时效果都是对的
		    // 因为 01 & 11 = 1, 101 & 11 = 1，这两个都和缓冲区中的值一致（假设当前缓冲区的值为1）
		    // 而如果Ref=3，则 11 & 11 = 11 = 3(十进制)，而当前缓冲区值为1，不相等 
		    ReadMask 3       
		    // 写入掩码（默认255）
		    
		    // 比较函数
		    Comp _StencilComp 
		    
		    // Keep：如果通过，则不做任何改变
		    Pass _StencilOp   // 测试通过时的操作（如 Replace、Keep）
		}
		
		Pass
		{
			CGPROGRAM
			#pragma vertex vert
			#pragma fragment frag
			#include "UnityCG.cginc"
			
			struct appdata
			{
				float4 vertex : POSITION;
				fixed2 uv : TEXCOORD0;
				
				// 如果需要获得当前的顶点颜色，需要使用"COLOR"语义
				fixed4 color : COLOR;
			};
			
			struct v2f
			{
				float4 pos : SV_POSITION;
				
				// v2f当中的语义只有SV_POSITION是强制的，其他都是自己设计的含义
				fixed2 uv : TEXCOORD0;
				fixed4 color : COLOR;
			};
			
			sampler2D _MainTex;
			
			v2f vert(appdata v)
			{
				v2f o;
				o.pos = UnityObjectToClipPos(v.vertex);
				o.uv = v.uv;
				o.color = v.color;
				return o;
			}
			
			fixed4 frag(v2f i)
			{
				fixed4 c;
				fixed4 mainTex = tex2D(_MainTex,i.uv);
				c = mainTex;
				
				// 将其他的颜色混合到主纹理上
				c *= i.color;
				return c;
			}
			
			ENDCG
		}
	}
}

UGUI 与模板测试

分析：UGUI 与模板测试

  • 概念：
    • 模板缓冲区的使用会增加 GPU 负担，尤其在多层嵌套时，每个 Mask 会增加 2 个额外 DrawCall ，这两个 DrawCall 一个用于写入模板值，一个用于恢复；
    • 具体而言，加入在 Unity UGUI 中使用 Mask 组件时，其分为两个 DrawCall：
      1. 写入模板值：绘制遮罩形状
      • （1）开启模板写入（Stencil Write）
      • （2）将特定参考值（如 Ref 1）写入 Mask 区域所有像素的模板缓冲区；
      • （3）通常关闭颜色写入​（不输出颜色，只修改模板缓冲区）
      1. 恢复模板状态：重置或清除模板值
      • （1）绘制一个与 Mask 相同形状的几何体；
      • （2）将之前写入的模板值恢复​（例如，执行 DecrementSaturate 操作或直接写入 0）
      • （3）同样不输出颜色，只操作模板缓冲区；
  • 分析：
    • 模板缓冲区是一个“全局”资源。如果一个 Mask 写入了一个模板值（比如 1），但不在使用完毕后将其恢复为初始值（通常是 0），那么所有后续的、也使用模板测试的 UI 元素（例如另一个 Mask）都会受到这个残留值的影响，导致渲染错误；
    • 因此，​第二个 DrawCall 对于维护模板缓冲区的正确状态、避免后续渲染污染至关重要。这也是多层 Mask 嵌套时性能开销会急剧放大的原因，因为每个 Mask 都需要完成这一套“写入-恢复”的操作；

通道 ColorMask

概念：通道 ColorMask

  • 概念：
    • ColorMask 是 Unity ShaderLab 中的一个渲染状态指令，用于精确控制 GPU 对颜色缓冲区各通道（RGBA）的写入权限；
    • 其核心功能是通过掩码机制，选择性地启用或禁用颜色通道的写入操作；
    • 默认值为 ColorMask RGBA，即表示写入所有通道，ColorMask 0 为不输出任何颜色，因此用于仅更新深度或模板缓冲区；
    • 如果是 ColorMask R，则表示只通过红色的信息；
    • 如果是 ColorMask RG，则表示只通过红、绿色的信息；

UI 遮罩

基础概念

概念：UI 遮罩

  • 引入：
    • Unity 提供了两种主要的 UI 遮罩技术：
        1. 基于模板缓冲区的 Mask；
        1. 基于裁剪矩形的 RectMask2D；
    • 这两种技术在实现原理、性能表现和使用场景上各有特点，Mask 是 Unity UGUI 系统中传统的遮罩组件，它基于 GPU 的模板缓冲区 (Stencil Buffer) 实现遮罩效果；
    • 但是模板缓冲区的使用会增加 GPU 负担，尤其在多层嵌套时，每个 Mask 会增加 2 个额外 DrawCall ，这两个 DrawCall 一个用于写入模板值，一个用于恢复；
    • 因此，还有 RectMask2D 的技术，其是 Unity 提供的另一种遮罩实现，它专门针对矩形遮罩场景进行了优化，性能通常优于 Mask；
  • 概念：
    • RectMask2D 的实现不依赖模板缓冲区，而是基于裁剪矩形 (Clipping Rectangle) 和顶点裁剪技术；
      1. 裁剪矩形计算：
      • 根据 RectTransform 计算出一个裁剪矩形区域；
      1. 顶点裁剪：
      • 子元素的顶点数据会被裁剪到该矩形内，超出部分不参与绘制；
      1. Shader 处理：
      • 通过 Shader 中的 UNITY_UI_CLIP_RECT 机制，丢弃矩形外的像素；
      • 当父级物体当中有 ReckMask2D 组件时，UNITY_UI_CLIP_RECT 被激活，需要先手动定义变体：
        • #pragma multi_compile _ UNITY_UI_CLIP_RECT
  • 概念：UNITY_UI_CLIP_RECT
    • UNITY_UI_CLIP_RECT 是 RectMask2D 在 Shader 中实现的关键技术，其工作原理如下：
      1. RectMask 2D 将裁剪矩形作为 _ClipRect 参数传递给 Shader；
      1. 在片元着色器中，判断当前像素是否在 _ClipRect 定义的矩形内；
      1. 透明度处理：不在矩形内的像素会被设置为完全透明 (alpha=0)；
      1. 像素丢弃：Shader 会丢弃 alpha 小于 0.001 的元素；
  • 分析：UnityGet2DClipping 的实现原理
// 接收裁剪矩形参数
float4 _ClipRect;

// 判断像素是否在裁剪矩形内
inline float UnityGet2DClipping(in float2 position, in float4 clipRect)
{
	// 如果当前的clipRect.xy的值大于position.xy，则返回1，否则返回0
	// 如果当前的position.xy的值大于clipRect.zw，则返回1，否则返回0
	// 因此此float2 的inside的实际含义，表示的是当前片元所处的位置、其是否位于当前RectMask2D的位置里面
	// 如果当前片元所处的屏幕坐标位置在clipRect外面，则返回的值就是0，然后可以利用Alpha值和UnityGet2DClipping返回的值来相乘，实现Mask的效果
    float2 inside = step(clipRect.xy, position.xy) * step(position.xy, clipRect.zw);
    return inside.x * inside.y;
}

// 在片元着色器中应用裁剪
fixed4 frag (v2f i) : SV_Target
{
    // 计算裁剪
    float alpha = UnityGet2DClipping(i.worldPosition.xy, _ClipRect);
    
    // 丢弃透明像素
    clip (color.a - 0.001);
    
    return color;
}

Shader "taegc/UIGray"
{
	Properties
	{
		[PerRendererData]_MainTex("Tex",2D) = "white"{}

		_Ref("Stencil Ref",int) = 0

		_StencilComp("Stencil Comp",int) = 0
		
		[Enum(UnityEngine.Renderer.StencilOp)]
		_StencilOp("Stencil Op",int) = 0

		_GrayEnabled("GrayEnabled",int) = 0
	}

	SubShader
	{
		Tags{"Queue" = "Transparent"}
		Blend ScrAlpha OneMinusSrcAlpha
		ColorMask 0
		
		Stencil 
		{
		    Ref [_Ref]
		    ReadMask 3       
		    Comp _StencilComp
		    Pass _StencilOp
		}
		
		Pass
		{
			CGPROGRAM
			#pragma vertex vert
			#pragma fragment frag
			#include "UnityCG.cginc"
			
			// 注意：当前使用到的UnityGet2DClipping方法定义在"UnityUI.cginc"当中
			// 因此不要忘记引用此脚本
			#include "UnityUI.cginc"
			#pragma multi_compile _ UNITY_UI_CLIP_RECT
			#pragma multi_compile _ _GRAYENABLED_ON
			
			struct appdata
			{
				float4 vertex : POSITION;
				fixed2 uv : TEXCOORD0;
				fixed4 color : COLOR;
			};
			
			struct v2f
			{
				float4 pos : SV_POSITION;
				fixed2 uv : TEXCOORD0;
				fixed4 color : COLOR;
				float4 vertex : TEXCOORD1;
			};
			
			sampler2D _MainTex;
			float4 _ClipRect;
			
			v2f vert(appdata v)
			{
				v2f o;
				o.pos = UnityObjectToClipPos(v.vertex);
				o.uv = v.uv;
				o.color = v.color;
				o.vertex = v.vertex;
				return o;
			}
			
			fixed4 frag(v2f i)
			{
				fixed4 c;
				fixed4 mainTex = tex2D(_MainTex,i.uv);
				c = mainTex;
				c *= i.color;
				
				#if UNITY_UI_CLIP_RECT
					// 方法一：使用 if 语句
					// if(_ClipRect.x<i.vertex.x && i.veretx.x<_ClipRect.z && _ClipRect.y>i.vertex.y && i.veretx.y<_ClipRect.w ) return 1;
					// else return 0;
					
					// 方法二：使用Step
					// 如果 a>b 则返回 1，否则则返回 0
					// 可以使用step函数来优化掉if语句
					// fixed2 rect = step(_ClipRect.xw,i,vertex.xy) * step(i.vertex.xy,_ClipRect.zw)
					// c.a *= rect.x * rect.y;

				#endif

					// c.rgb = c.r * 0.22 + c.g * 0.707 + c.b * 0.071;
					
					// 内置的去色方法
					c.rgb = Luminance(c.rgb);
				#endif
				
				return c;
			}
			
			ENDCG
		}
	}
