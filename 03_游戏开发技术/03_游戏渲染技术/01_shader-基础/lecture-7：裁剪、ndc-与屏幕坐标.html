Lecture 7：裁剪、NDC 与屏幕坐标

各类坐标

裁剪空间坐标

概念：裁剪空间坐标

  • 概念：
    • 裁剪空间是顶点经过 MVP 和投影矩阵变换后所在的空间，这个空间的关键在于其裁剪范围：任何一个顶点，只有当其 x, y, z 分量都满足 -w ≤ value ≤ w 时，才会被保留并进行后续处理，否则将被裁剪掉；
    • 这个以 w 为动态边界的立方体区域称为视锥体；
  • 概念：透视除法
    • 将裁剪坐标的 x, y, z 三个分量同时除以 w 分量称之为透视除法；
    • 其结果就是将坐标转换到了归一化设备坐标空间
总结：
  • 裁剪空间坐标是 MVP 变换后、再经过投影变换的坐标，它是一个带有特殊 w 分量的齐次坐标；
  • 而 NDC 坐标则是对裁剪空间坐标进行“透视除法”后得到的标准化坐标，是进行屏幕映射前的最后一步；
总结：完整坐标变换
  • 模型坐标 → (模型矩阵) → 世界坐标 → (视图矩阵) → 观察坐标 → (投影矩阵) → 裁剪坐标 → (透视除法) → NDC坐标 → (视口变换) → 屏幕坐标

NDC 坐标

概念：NDC 坐标 - 归一化设备坐标系

  • 概念：
    • NDC 空间是一个与设备无关的标准化立方体。无论原始视锥体形状和大小如何，经过透视除法后，所有可见顶点的坐标都会被归一化到这个立方体内（例如，OpenGL 中 XYZ 范围都是[-1, 1])，这为后续映射到具体的屏幕像素坐标奠定了基础；
    • NDC 是三维图形渲染管线中的标准化中间坐标系，范围固定为：
  • 特点：
      1. 消除屏幕分辨率差异，统一不同设备的坐标空间；
      1. x 轴：左边界 -1，右边界 1（水平方向）
      1. y 轴：下边界 -1，上边界 1（垂直方向，​与屏幕 Y 轴相反）
      1. z 轴：近裁剪面 -1，远裁剪面 1（深度方向）

屏幕空间坐标

概念：屏幕坐标

  • 概念：
    • 屏幕坐标系统主要分为基于像素的坐标和基于归一化的坐标两种形式；
  • 概念：像素坐标系
    • 像素坐标系统 (Pixel Coordinate System) 是数字图像处理和计算机视觉中最基础的坐标表示方法，它以实际像素为单位描述图像上每个点的位置；
    • 原点位置​通常位于图像的左上角（虽然某些系统可能定义在左下角或其他位置），坐标值直接对应显示设备上的物理像素点，便于直接测量和定位图像中的具体元素；
    • 相同 NDC 坐标在不同分辨率屏幕上映射的像素坐标不同，坐标值为整数，表示具体像素位置；
  • 概念：归一化坐标
    • 归一化坐标系统 (Normalized Coordinate System) 是一种与具体分辨率无关的比例表示方式，坐标值被标准化到特定范围内（通常是 [0,1] 或 [-1,1]）；
    • 在 OpenGL 中，归一化坐标系原点在中心，xy 轴范围都是 [-1,1]。而对于矩形屏幕（宽高比非 1:1），宽度可能会归一化到 [-aspectRatio, aspectRatio] 的范围；
  • 概念：归一化平面
    • 在计算机视觉和 3D 图形中，还存在"归一化平面" (Normalized Plane)的概念，这是一个虚拟的二维平面，用于将 3D 空间中的点投影到相机成像平面前的一个中间阶段；
    • 3D 点 P(X,Y,Z) 被投射到归一化平面上的 P'(X/Z, Y/Z, 1)，这个平面位于相机前方，Z 坐标为 1 的位置；
  • 关系：
      1. 归一化坐标和像素坐标的关系：
      • 屏幕归一化坐标当前像素总像素
      1. 坐标转换关系：
      • 模型坐标模型矩阵世界坐标视图矩阵相机坐标投影矩阵裁剪坐标透视除法坐标视口变换实际像素位置
      1. 透视除法：
      • 裁剪坐标的数据格式：齐次坐标 (x_clip, y_clip, z_clip, w_clip)；
      • 将齐次坐标进行透视除法，得到：
  • 总结：
    • NDC 是设备无关的标准化空间，而像素坐标是设备相关的最终输出。屏幕归一化坐标和归一化平面则是转换过程中的中间表示；

Unity 中屏幕坐标

概念：Unity 中屏幕坐标

  • 概念：Unity 当中获取当前像素坐标 - _ScreenParams 参数
    • _ScreenParams 是 Unity Shader 中一个非常重要的内置变量，它提供了当前渲染目标的尺寸信息，在屏幕空间效果、全屏特效和分辨率相关计算中扮演着关键角色；
    • _ScreenParams 是一个 float4 类型的 Unity 内置变量，包含以下四个分量：
        1. x 分量​：表示当前渲染目标（通常是屏幕或 RenderTexture）的宽度，以像素为单位；
        1. y 分量​：表示当前渲染目标的高度，以像素为单位；
        1. z 分量​：值为 1.0 + 1.0/width，常用于纹理采样时的边界处理；
        1. w 分量：值为 1.0 + 1.0/height，同样用于纹理采样的边界处理；
    • 将像素坐标转换为 [0,1] 范围的归一化坐标是 _ScreenParams 最常见的用途之一；
      • float2 normalizedUV = pixelCoord / _ScreenParams.xy;
  • 概念：VPOS - 表示当前片元的像素所在位置
    • VPOS 是 Unity Shader 中一个特殊的语义，用于在片段着色器中获取当前像素的屏幕空间坐标；
  • 图示：直接在片元着色器返回 screenUV = screenPos. xy/_ScreenParams. xy 的画面效果
    • 左下角为屏幕坐标 UV 为 0,0 因此为黑色，右上角因此为 1,1；
  • 代码：
Shader "taecg/HeatDistort"
{
	SubShader
	{
		Pass
		{
			CGPROGRAM
			#pragma vertex vert
			#pragma fragment frag
			#include "UnityCG.cginc"
			
			struct appdata
			{
				float4 vertex : POSITION;
				float2 uv : TEXCOORD0;
			};
			
			struct v2f
			{
				float4 vertex : SV_POSITION;
			};
			
			v2f vert(
				float4 vertex : POSITION,
				out float4 pos : SV_POSITION
			)
			{
				v2f o;
				pos = UnityObjectToClipPos(vertex);
				return o;
			}
			
			fixed4 frag(v2f i,UNITY_VPOS_TYPE screenPos : VPOS) : SV_Target
			{
				fixed2 screenUV = screenPos.xy / _ScreenParams.xy;
				return fixed4(screenUV,0,1);
			}
			
			ENDCG
		}
	}
}

坐标变换

UnityObjectToClipPos

概念：UnityObjectToClipPos

  • 概念：裁剪空间
    • 裁剪坐标系是通过投影矩阵​（正交或透视）将顶点从观察空间（相机空间）转换而来的 4D 齐次空间；
    • 其范围由视锥体决定，超出 [-w, w] 范围的图元会被裁剪；
      • 透视投影​：W 值随距离变化，实现近大远小效果；
      • 正交投影​：W 恒为 1，无透视变形；
    • 裁剪空间使用齐次坐标，包含 (x, y, z, w) 分量，需通过透视除法（x/w, y/w, z/w）转换为 NDC（归一化设备坐标）；
    • 正交投影时，齐次裁剪空间坐标等于 NDC 坐标；
  • 概念：
    • 将模型空间（Object Space）的顶点坐标转换到裁剪空间（Clip Space）​；
  • 概念：UnityObjectToClipPos 方法的输入输出
    • 输入：
      • 模型空间的顶点坐标；
      • 语义：float4 vertex : POSITION；
    • 输出：
      • 裁剪空间坐标（float4 pos : SV_POSITION），其范围由投影矩阵决定；
      • 正交投影：坐标范围为 [-1, 1]（NDC 空间）；
      • 透视投影：坐标范围为 [-w, w]，其中 w 为齐次分量，即当前 View 空间的深度；
  • 作用：
    • 完成 模型空间→世界空间→视图空间→裁剪空间 的矩阵变换（MVP 变换），是顶点着色器的核心步骤，用于后续光栅化和片元处理；
  • 总结：从 UnityObjectToClipPos 到屏幕坐标归一化 0~1
    • o.pos = UnityObjectToClip(v.vertex);
      • 此时 o.pos 的数值，为当前屏幕的像素坐标系下的位置的值；
    • fixed2 uv = o.pos.xy / _ScreenParams.xy;
      • 此时 uv 的数值，代表的是像素坐标系归一化后的数值，即 0~1；
  • 总结：世界坐标系和裁剪坐标系
    • 顶点需依次通过 Model → View → Projection 矩阵（MVP）从模型空间转换到裁剪空间；

正交投影下裁剪空间坐标

分析：正交投影下裁剪空间坐标

  • 分析：
    • 在正交投影中，视锥体是一个长方体（非透视变形），由六个裁剪平面定义：左（left）、右（right）、下（bottom）、上（top）、近（near）、远（far）；
  • 参数：
    • Size = 50：表示视锥体高度的一半（即半高），因此总高度为 100；
    • Near = 0.3：近裁剪平面到相机的距离（正值）
    • Far = 120：远裁剪平面到相机的距离（正值）
    • 宽高比（Aspect）：投影矩阵依赖 Aspect。为简化计算，假设 Aspect = 1（正方形视图），但推导会保留 Aspect 变量。实际中 Aspect 由屏幕分辨率决定（如 16:9）
  • 推导：正交投影下裁剪空间坐标
    • 相机位于原点，Unity 使用右手坐标系：x 轴向右，y 轴向上，z 轴指向相机后方；
      • 近裁剪平面位于 z=−Near=−0.3，远裁剪平面位于 z=−Far=−120；
      • 视锥体：top=Size=50, bottom=−Size=−50, right=Size×Aspect, left=−Size×Aspect；

补充：平行光写入 ShadowMap 的坐标值
  • 平行光使用正交投影（Orthographic Projection），其核心优势是不会改变深度的线性关系。然后写入 ShadowMap 时，使用的是经过归一化后的 NDC 坐标中的 Z 值；
  • 裁剪空间坐标进行透视除法（除以 W 分量），变换到归一化设备坐标空间。对于正交投影，W 分量始终为 1，所以这一步主要是将 XYZ 分量的范围映射到 [-1, 1]；
  • 接着，为了适配纹理的 [0, 1] 采样范围，需要将 NDC 空间的 Z 值（范围 [-1, 1]）通过公式：
    • Z' = 0.5 * Z_ndc + 0.5 归一化到 [0, 1] 之间
  • 这个 Z' 就是最终准备存入 Shadow Map 的值；

ComputeScreenPos

概念：ComputeScreenPos

  • 概念：
    • 基于裁剪空间坐标计算屏幕空间坐标，输出的结果是一个齐次坐标（float4 screenPos）；
  • 概念：输入输出
    • 输入：
      • 裁剪空间坐标（通常来自 UnityObjectToClipPos 的输出）；
    • 输出：
        1. xy 分量：范围 [0, w]（w 为齐次分量）；
        1. z 分量：与输入裁剪空间的 z 值相同；
        1. w 分量：存储输入的裁剪空间 w 值（即视空间深度）；
  • 作用：
    • 为片元着色器提供屏幕 UV 坐标的基础数据；
    • 需在片元着色器中除以 w 分量（透视除法），将 xy 映射到 [0, 1] 的屏幕 UV 范围；
  • 总结：
  • 流程：
  • 示例：
v2f vert(appdata v) {
    v2f o;
    o.clipPos = UnityObjectToClipPos(v.vertex);
    
    // 预处理屏幕坐标
    o.screenPos = ComputeScreenPos(o.clipPos); 
    return o;
}

fixed4 frag(v2f i) : SV_Target {
    float2 screenUV = i.screenPos.xy / i.screenPos.w; // 透视除法
    float depth = tex2D(_CameraDepthTexture, screenUV).r; // 采样深度图
}

屏幕扭曲效果

概念：屏幕扭曲效果具体实现

  • 分类：
      1. 特效中常见的扭曲效果 -> 热扭曲；
      1. 水体等特殊表现的地方；
  • 概念：实现思路
      1. 扭曲材质赋予面片：
      • 希望当前的扭曲效果只会在屏幕上部分透过去的区域进行；
      1. 抓取当前一帧的图片内容：
      • 通过对渲染的图片进行扭曲，来实现扭曲效果；

      1. 抓取屏幕坐标：
      • 当前的屏幕渲染出的内容的坐标，比如：左下角是 0，0，右上角是 1，1；
      1. 利用屏幕坐标对抓取的图片采样；
      1. 最后，再采样张扰动贴图做扭曲；

GrabPass 基础概念

概念：GrabPass 屏幕抓取

  • 概念：
    • GrabPass 是 Unity ShaderLab 中的一个特殊通道 (Pass) 类型，它能够在对象被渲染前捕获屏幕内容并将其存储到一张纹理中；
    • 这张"抓取纹理"可以在同一着色器的后续通道中使用，为实现各种基于屏幕图像的视觉效果提供了基础；
    • 从技术实现角度看，GrabPass 的工作原理是：
        1. 抓取时机：在对象即将被渲染前执行；
        1. 抓取范围：仅包含当前帧缓冲区中位于该对象后面的内容；
        1. 纹理创建：自动生成一张与屏幕分辨率相同的临时纹理；
        1. 数据：将 GPU 帧缓冲区内容复制到纹理中，供后续 Pass 使用；
  • 特点：
    • 抓取的纹理分辨率与显示设备一致，在高分辨率设备上会产生较大数据量；
    • 通常需要在透明队列 (Transparent Queue)中使用，以确保正确捕获被遮挡的内容；
    • 每帧都会重新捕获，适合动态变化的场景；
  • 图示：将抓取到的画面作为片元着色器的输出
  • 代码：
Shader "taecg/HeatDistort"
{
	SubShader
	{
		// 通常需要在透明队列 (`Transparent Queue`)中使用，以确保正确捕获被遮挡的内容；
		Tags{"Queue" = "Transparent"}
	
		// 如果这个Pass里面什么都不写，Unity会默认自动抓取一张图片
		GrabPass
		{
			
		}
	
		Pass
		{
			CGPROGRAM
			#pragma vertex vert
			#pragma fragment frag
			#include "UnityCG.cginc"
			
			struct appdata
			{
				float4 vertex : POSITION;
				float2 uv : TEXCOORD0;
			};
			
			struct v2f
			{
				float4 vertex : SV_POSITION;
			};
			
			// 这个名称是Unity当中规定的
			sampler2D _GrabTexture;
			
			v2f vert(
				float4 vertex : POSITION,
				out float4 pos : SV_POSITION
			)
			{
				v2f o;
				pos = UnityObjectToClipPos(vertex);
				return o;
			}
			
			fixed4 frag(v2f i,UNITY_VPOS_TYPE screenPos : VPOS) : SV_Target
			{
				fixed2 screenUV = screenPos.xy/_ScreenParams.xy;
				
				fixed4 grabTex = tex2D(_GrabTexture,screenUV);
				
				return fixed4(screenUV,0,1);
			}
			
			ENDCG
		}
	}
}

			};
			
			struct v2f
			{
				float4 uv : TEXCOORD0;
				float4 pos : SV_POSITION;
			};
			
			// 使用GrabPass中定义的名称
			sampler2D _GrabTex;
			sampler2D _DistortTex;float4 _DistortTex_ST;
			fixed _Distort;
			float _SpeedX,_SpeedY;
			
			v2f vert(appdata v)
			{
				v2f o;
				o.pos = UnityObjectToClipPos(v.vertex);
				o.uv = TRANSFORM_TEX(v.uv,_DistortTex) + float2(_SpeedX,_SpeedY)*_Time.y;
				return o;
			}
			
			fixed4 frag(v2f i,UNITY_VPOS_TYPE screenPos : VPOS) : SV_Target
			{
				// 得到当前片元，在经过归一化之后的屏幕空间坐标
				fixed2 screenUV = screenPos.xy/_ScreenParams.xy;
				
				// 取出扭曲贴图上的值
				fixed4 distortTex = tex2D(_DistortTex,i.uv);
				
				// 在“屏幕自身的UV”和“扭曲贴图的值作为UV”间插值
				float2 uv = lerp(screenUV,distortTex,_Distort);
				fixed4 grabTex = tex2D(_GrabTex,uv);
				
				return grabTex;
			}
			
			ENDCG
		}
	}
