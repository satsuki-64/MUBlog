Lecture 12：全局光照

什么是全局光照

基础概念

概念：全局光照

  • 概念：
    • 全局照明 (Global llumination)，简称 GI，指既考虑场景中直接来自光源的光照 (Directlight) 又考虑经过场景中其他物体反射后的光照 (lndirect light) 的一种渲染技术。使用全局照明能够有效地增强场景的真实感；
    • 即：全局照明 = 直接光照 (Direct Lignt) + 间接光照 (lndirect Light)
  • 概念：Unity 当中的 GI 方案
    • 方案一：预计算实时 GI
      • 预计算实时 GI 是在实时显示之前，把需要的数据进行烘焙好。烘焙生成光照数据资产，用于运行时实时生成和更新低分辨率光照贴图，然后在运行时再进行读取，以便可以实时的与光源进行互动；
      • 因此预计算实时 GI 允许在运行时改变光源（如颜色、强度、位置），间接光照会实时更新，运行时有一定的 CPU 开销用于计算间接光照的更新；
    • 方案二：烘培 GI
      • 烘焙 GI 是把光源对场景中静态物体的光照效果提前烘焙到一张张的光照纹理中，然后再把光照纹理采样到物体表面以产生全局光照效果的方式；
      • 烘培 GI 生成静态的光照贴图纹理​（Lightmap），作为资源存储在项目，​不允许运行时改变光源，光照效果完全静态。在 Unity 当中，烘培光照分为几种模式：Baked Indirect 实时计算和 Subtractive/Shadowmask 部分或全部烘焙；

两类 GI 技术

预计算实时 GI

概念：预计算实时 GI

  • 概念：
    • 预计算实时 GI 是一种混合解决方案，它结合了预计算和实时更新的优势，能够在保持交互式帧速率的同时允许实时反射照明；
    • 在编辑器中对场景中静态几何体周围的光线反弹进行计算，并将数据存储为"照明数据资产" Lighting Data Asset；
    • 在运行的时候，利用预计算数据在运行时以交互方式生成和更新一组低分辨率光照贴图，当光源属性 (如颜色、强度)或位置发生变化时，系统会基于预计算数据实时更新间接光照效果；
  • 特点：
    • Unity 早期版本使用 Enlighten 作为采用预计算实时 GI 作为其后端技术，2020LTS 是最后一个支持版本；
    • 不生成传统光照贴图纹理，而是存储为可在运行时生成光照贴图的信息，适合光照变化较慢的场景 (如昼夜渐变)，不适合快速变化的光源；

烘培 GI

概念：烘培 GI

  • 概念：
    • 烘焙 GI 是一种完全预计算的解决方案，它将光照信息提前烘焙到纹理中，运行时直接采样使用，适合静态场景的高质量光照表现；
    • 此方案完全预计算，在编辑阶段计算静态物体的直接光照、间接光照和阴影信息，生成光照贴图 (Lightmaps) 作为项目资源，运行时不可更改；
    • 只对标记为 Static 的物体有效，动态物体需依赖光照探针；
  • 分类：三种烘培系统

Unity 当中的 GI 技术

概念：Unity 当中的 GI 技术

  • 概念：Baked Indirect
    • 只烘焙间接光，直接光与阴影需要实时计算；
    • 运行时改变灯光时，直接光和阴影会变化，间接光不变，其内存开销小，但是性能开销大；
  • 概念：Subtractive
    • 静态物体的直接光和间接光都烘焙，仅动态物体接收实时的直接光；
    • 性能消耗最小，手游低端机常用，动态物体只能在静态物体上投射主方向光的阴影；
  • 概念：Shadowmask
    • 作用：
      • 同 Baked Indirect，但静态物体的阴影也会被烘焙到 Shadowmask 中；
      • 内存中多存储一张 Shadowmask 贴图，比 Baked Indirect 减少了静态物体阴影计算；
    • 概念：
      • Shadowmask 是一种结合了烘焙阴影与实时阴影的混合技术，它通过阴影遮罩贴图来存储静态物体的阴影信息，同时保留动态物体的实时阴影计算能力；
        1. 静态物体处理：
        • 将静态物体的直接光照产生的阴影烘焙到专门的 Shadowmask 贴图中（阴影遮罩），这张贴图记录了哪些区域应该被阴影覆盖；
        1. 动态物体处理：
        • 动态物体仍然使用实时阴影贴图 (ShadowMap) 技术，在运行时计算阴影；
      • 因此可知，当前 Shadowmask 的烘培结果会存储到两张光照贴图上，一张存储直接光的效果、一张存储阴影的遮罩效果；
      • 在渲染时，系统会根据 Shadowmask 贴图和实时阴影贴图的信息，智能地混合静态和动态阴影，确保视觉一致性；
    • 分析：
      • 因为当前 Shadowmask 存储的是一个阴影的遮罩图，因此当光照的角度变化后，当前阴影的位置不会发生变化，仅会动态改变阴影的强弱；
  • 补充：Shadowmask 的 Distance Shadowmask
    • Shadowmask 的一个改进版本是 Distance Shadowmask，它在近距离使用实时阴影，远距离使用烘焙阴影，进一步提升了视觉效果；

与烘培相关的内置宏以及变量

概念：与烘培相关的内置宏

  • 概念：#pragma mutil_compile_fwdbase novertexlight nodynlightmap nodirlightmap
    • 定义在 LightMode=ForwardBase 的 Pass 中，在此 Pass 中仅只持一个平行灯以及其它逐像素或 SH 时；
    • 这个指命的作用是一次性生成 Unity 在 ForwardBase 中需要的各种内置宏；
  • 分类：

概念：与烘培相关的变量

  • 概念：unity_LightmapST
    • unity_LightmapST 是一个关键的内置变量，用于处理光照贴图（Lightmap）的 UV 坐标变换，与全局光照（GI）系统紧密配合；
    • 其为 float4 类型，包含缩放（xy）和偏移（zw）参数，使用方法如下：
      • lightmapUV = input.uv1 * unity_LightmapST.xy + unity_LightmapST.zw
    • 此代码的功能为：将模型第二套 UV（TEXCOORD1）转换为光照贴图的正确采样坐标，其中 input.uv1 是模型原始的 UV1 坐标，存储静态光照贴图数据；
  • 概念：与 unity_Lightmap 的协作
    • unity_Lightmap 是存储烘焙光照数据的纹理，unity_LightmapST 确保其正确映射到模型表面；
  • 概念：unity_DynamicLightmapST
    • unity_DynamicLightmapST 是一个关键的内置变量，用于处理动态光照贴图的 UV 坐标变换；
    • 它与静态光照贴图的 unity_LightmapST 类似，但专门针对实时更新的动态全局光照 Realtime GI 的数据；
      • dynamicUV = input.uv2 * unity_DynamicLightmapST.xy + unity_DynamicLightmapST.zw
    • 静态光照贴图 UV（unity_LightmapST）处理预烘焙的静态场景光照，而 unity_DynamicLightmapST 用于运行时更新的动态间接光照；

lightmapUV

概念：lightmapUV

  • 概念：
    • Lightmap UV 用于将烘焙好的光照贴图（Lightmap）精确映射到静态物体的表面，替代实时计算间接光和阴影，大幅降低运行时性能开销；
    • 在混合光照如 Baked Indirect 或 Shadowmask 中，Lightmap UV 区分静态与动态物体的光照来源：静态物体通过光照贴图采样，动态物体通过光探针插值；
  • 原理：
    • Unity 通过内置变量 unity_LightmapST（缩放和偏移）将模型原始 UV1 转换为光照贴图空间坐标；
    • float2 lightmapUV = v.uv1 * unity_LightmapST.xy + unity_LightmapST.zw;
  • 图示：烘培得到的光照贴图
    • 此光照贴图中的 Tilling 或者 Offset 的值，存储在 unity_LightmapST 或者是 unity_DynamicLightmapST 当中；
    • 经过了 float2 lightmapUV = v.uv1 * unity_LightmapST.xy + unity_LightmapST.zw; 的采样后，可以获取到当前模型上的纹理，对应在光照贴图上采样的坐标；

// 这段代码主要负责汇总和处理各种间接光照来源，计算物体表面接收到的间接漫反射光照：indirect.diffuse
// 其工作流程是：首先处理阴影混合（如 Shadowmask）来优化实时阴影与烘焙阴影的过渡
// 然后按优先级，采样并叠加不同来源的光照信息，这包括烘焙光照贴图 (Baked Lightmaps)​、实时光照贴图 (Realtime Lightmaps)​​ 以及光照探针通过球谐函数提供的照明 (Light Probes/SH)​​
// 最后，将综合后的间接光结果与环境光遮蔽 (Occlusion) 相乘，得到最终应用于表面的间接漫反射光照输出
UnityGI UnityGI_Base(UnityGIInput data, half occlusion, half3 normalWorld) 
{
    UnityGI o_gi;
    
    // 1. 阴影混合（Shadowmask关键）
    #if defined(HANDLE_SHADOWS_BLENDING_IN_GI)
        half bakedAtten = UnitySampleBakedOcclusion(data.lightmapUV.xy, data.worldPos);
        data.atten = UnityMixRealtimeAndBakedShadows(data.atten, bakedAtten, ...);
    #endif
    
    // 2. 静态烘焙 GI 采样
    #if defined(LIGHTMAP_ON)
        half4 bakedColorTex = UNITY_SAMPLE_TEX2D(unity_Lightmap, data.lightmapUV.xy);
        half3 bakedColor = DecodeLightmap(bakedColorTex);
        // 方向光贴图增强凹凸细节
        #ifdef DIRLIGHTMAP_COMBINED
            o_gi.indirect.diffuse += DecodeDirectionalLightmap(...);
        #endif
    #endif
    
    // 3. 实时GI采样（动态物体）
    #ifdef DYNAMICLIGHTMAP_ON
        half4 realtimeColorTex = UNITY_SAMPLE_TEX2D(unity_DynamicLightmap, data.lightmapUV.zw);
        o_gi.indirect.diffuse += DecodeRealtimeLightmap(realtimeColorTex);
    #endif
    
    return o_gi;
}

// 计算直接光照的效果
inline fixed4 LightingLambert(SurfaceOutput s,UnityGI gi)
{
	// 这里的变量 c 代表的是直接光计算得到的颜色
	fixed4 c;
	c = UnityLambertLight(s,gi.light);
	
	// 如果有间接光，则将间接光的效果加到直接光上
	#ifdef UNITY_LIGHT_FUNCTION_APPLY_INDIRECT
		// 材质的反射率 × 反射光的颜色 × 漫反射强度
		// 直接光照与间接光照的混合 
		c.rgb += s.Albedo * gi.indirect * diffuse; 
	#endif
	
	return c;
}
  • 分析：UnityLambertLight 函数
inline fixed4 UnityLambertLight (SurfaceOutput s, UnityLight light)
{
    // 计算法线与光照方向的点积，确保不小于0
    fixed diff = max(0, dot(s.Normal, light.dir));

    // 计算最终颜色
    fixed4 c;
    c.rgb = s.Albedo * light.color * diff;  
    c.a = s.Alpha;                          // 保留原始透明度
    
    return c;
}

间接光照实现

分析：间接光照实现

  • 总结：
    • LightingLambert_GI > UnityGlobalIllumination > UnityGI_Base + UnityGI_IndirectSpecular
  • 分析：
    • 在 LightingLambert_GI 函数当中，调用了 UnityGlobalIllumination 函数，然后再在 UnityGlobalIllumination 函数当中，调用了 UnityGI_Base 函数中；
    • 因此 UnityGI_Base 函数就是 Unity 中实现间接光照的函数，它首先初始化 GI 数据结构，然后处理实时阴影与烘焙阴影的混合，接着按特定优先级（球谐函数 SH > 烘焙光照贴图 Baked Lightmap > 动态光照贴图 Realtime Lightmap）采集和叠加各种间接光源的漫反射颜色，最终将结果乘以环境遮挡系数后输出，从而综合模拟光线在场景中的多次反弹效果；
  • 代码：
// 输入参数：
// data: 包含光照输入数据（直接光、世界坐标、视角方向等）
// occlusion: 环境遮挡系数（0-1，用于减弱间接光强度）
// normalWorld: 世界空间下的表面法线
inline UnityGI UnityGI_Base(UnityGIInput data, half occlusion, half3 normalWorld) 
{
    // 1. 初始化GI输出结构体，用于存储最终计算得到的GI信息
    UnityGI o_gi;
    ResetUnityGI(o_gi); // 重置所有字段为默认值，将颜色、方向、漫反射、高光都设为默认值

    // 分支一：阴影混合处理（实时阴影与烘焙阴影的过渡）
    // 计算在哦Distance Shadowmask中、实时阴影和混合阴影的过渡效果
    #if defined(HANDLE_SHADOWS_BLENDING_IN_GI)
        half bakedAtten = UnitySampleBakedOcclusion(data.lightmapUV.xy, data.worldPos);
        float zDist = dot(_WorldSpaceCameraPos - data.worldPos, UNITY_MATRIX_V[2].xyz);
        float fadeDist = UnityComputeShadowFadeDistance(data.worldPos, zDist);
        
        // 更改衰减值，此值和ShadowMask有关
        data.atten = UnityMixRealtimeAndBakedShadows(data.atten, bakedAtten, UnityComputeShadowFade(fadeDist));
    #endif

    // 2. 传递直接光数据，比如平行光的颜色和方向
    // 将主平行灯的信息传入GI当中
    o_gi.light = data.light;
    // 应用阴影衰减，将衰减应用到灯光的颜色当中
    o_gi.light.color *= data.atten; 

    // 分支二：球谐光照（SH）计算 
    // 当未使用光照贴图时，用球谐函数模拟环境光，主要用于光照探针
    #if UNITY_SHOULD_SAMPLE_SH
        o_gi.indirect.diffuse = ShadeSHPerPixel(normalWorld, data.ambient, data.worldPos);
    #endif

    // 分支三：采用Baked GI时
    #if defined(LIGHTMAP_ON)
        // 采样静态光照贴图，其中unity_Lightmap贴图表示的就是静态烘培得到的贴图 
        half4 bakedColorTex = UNITY_SAMPLE_TEX2D(unity_Lightmap, data.lightmapUV.xy);
	    
	    // 由于存储的烘培得到的贴图是RGBM格式的，因此需要对采样得到的贴图值进行解码
        half3 bakedColor = DecodeLightmap(bakedColorTex);

        // 方向性光照贴图，用于增强法线细节
        // 当Lightmapping Setting中的Directional Mode为Directional模式是，进入以下分支
        #ifdef DIRLIGHTMAP_COMBINED
            fixed4 bakedDirTex = UNITY_SAMPLE_TEX2D_SAMPLER(unity_LightmapInd, unity_Lightmap, data.lightmapUV.xy);
            o_gi.indirect.diffuse += DecodeDirectionalLightmap(bakedColor, bakedDirTex, normalWorld);
            
            // 阴影混合处理（动态阴影与烘焙阴影）
            #if defined(LIGHTMAP_SHADOW_MIXING) && !defined(SHADOWS_SHADOWMASK) && defined(SHADOWS_SCREEN)
                ResetUnityLight(o_gi.light);
                o_gi.indirect.diffuse = SubtractMainLightWithRealtimeAttenuationFromLightmap(
                    o_gi.indirect.diffuse, data.atten, bakedColorTex, normalWorld);
            #endif
        #else
            // 非方向性光照模式时，贴图直接叠加颜色
            o_gi.indirect.diffuse += bakedColor;
        #endif
    #endif

    // 分支四：采用Runtime GI时
    #ifdef DYNAMICLIGHTMAP_ON
	    // 类似于Baked GI模式，也是采样贴图、然后解码
        fixed4 realtimeColorTex = UNITY_SAMPLE_TEX2D(unity_DynamicLightmap, data.lightmapUV.zw);
        half3 realtimeColor = DecodeRealtimeLightmap(realtimeColorTex);
        
        // 方向性动态光照贴图
        #ifdef DIRLIGHTMAP_COMBINED
            half4 realtimeDirTex = UNITY_SAMPLE_TEX2D_SAMPLER(unity_DynamicDirectionality, unity_DynamicLightmap, data.lightmapUV.zw);
            o_gi.indirect.diffuse += DecodeDirectionalLightmap(realtimeColor, realtimeDirTex, normalWorld);
        #else
            o_gi.indirect.diffuse += realtimeColor;
        #endif
    #endif

    // 应用环境遮挡系数
    o_gi.indirect.diffuse *= occlusion;
    return o_gi;
}

补充：环境光遮挡系数

  • 作用：
    • 环境遮挡系数（Occlusion）在 PBR 渲染管线中，核心作用是模拟环境光（间接光）在物体表面凹陷、缝隙或紧邻区域因光线被遮挡而变暗的现象，从而显著增强场景的深度感和真实感；
  • 特点：
      1. 环境遮挡系数通常只影响间接光照部分：
      • 直接光照（如平行光、点光源）的阴影效果由专门的阴影技术处理，环境遮挡不直接影响它；
      1. 基于环境光遮蔽衰减，符合能量规则：
      • PBR 模型中，光线击中表面后分为反射（镜面光）和折射（漫反射）两部分，且能量守恒要求反射光与折射光之和不能超过入射光能。环境遮挡系数在最终阶段对间接光的这两部分进行统一的衰减，正是对这一物理规律的遵循；
  • 实现：SSAO
    • SSAO 是环境光遮蔽的实时渲染实现的版本，其核心在于，其​完全利用当前屏幕缓冲区（主要是深度缓冲 - Depth Buffer 和法线缓冲 - Normal Buffer）中的信息，在屏幕空间（即二维像素空间）中来估算三维世界中某个着色点（Shading Point）周围环境光被遮挡的程度；

GI 与光照探针

什么是光照探针

概念：光照探针

  • 概念：
    • Unity 中的光照探针（Light Probes）​是一种用于为动态（非 Static ）物体提供间接光照和全局光照数据的技术。它通过捕捉场景中特定位置的光照信息，并在运行时通过插值计算将这些信息应用到动态物体上，从而解决动态物体无法直接受益于烘焙光照的问题；
    • 光照探针通过在场景中预置采样点（探针组）记录周围环境的光照数据（如间接光、反射光），动态物体移动时会根据最近的探针数据插值计算其表面光照，使其与静态环境的光照效果保持一致；
    • 在每个探针当中，使用球谐函数（Spherical Harmonics）​编码光照信息，仅需 27 个浮点数，内存占用极低；
  • 分析：

分析：光照探针在 GI 中的实现

  • 概念：UNITY_SHOULD_SAMPLE_SH 宏定义
    • UNITY_SHOULD_SAMPLE_SH 是 Unity Shader 中用于控制是否采样球谐光照（Spherical Harmonics, SH）​的关键宏，其定义如下：
    • 相关条件解释如下：
        1. LIGHTPROBE_SH：当前 Shader 支持光照探针，需在 Shader 中启用 #pragma multi_compile LIGHTPROBE_SH 时启用；
        1. 排除 UNITY_PASS_FORWARDADD（前向附加通道，如额外点光源）
        1. 排除 UNITY_PASS_SHADOWCASTER（阴影投射通道）
        1. 排除 UNITY_PASS_META（光照贴图烘焙通道）
  • 分析：何时使用 SH
    • 如果物体同时有静态和动态光照贴图（LIGHTMAP_ON），Unity 会优先使用光照贴图数据，跳过 SH 计算以节省性能；
    • 无光照贴图时​：必须依赖 SH（光照探针）提供间接光照；

// 前向渲染路径时，球谐、逐顶点、环境光的计算都是在这里

	// UNITY_SAMPLE_FULL_SH_PER_PIXEL 表示是否逐片段的进行采样

				unity_4LightPosX0, unity_4LightPosY0, unity_4LightPosZ0,
				unity_LightColor[0].rgb, unity_LightColor[1].rgb,

				unity_4LightAtten0, o.worldPosition, o.worldNormal
			);
		#endif

		// 应用光照探针的球谐光照数据
		o.sh = ShadeSHPerVertex(worldNormal, o.sh);
	#endif
#endif // !LIGHTMAP_ON

计算顶点球谐光照

分析：计算顶点球谐光照

  • 概念：使用球谐函数近似光照
    • 球谐函数通过正交基（勒让德多项式）将复杂的环境光照投影为少量系数，比如 Unity 使用 3 阶共 9 个系数，实现高效存储和计算；
    • 光照探针在烘焙时预计算球谐系数，运行时通过法线方向插值重建光照，适合动态物体融入静态环境的间接光照；
  • 分析：ShadeSHPerVertex 实现
    • 将 Shader 编写在
      • FUZHUANG 的编写；
      • 用于全像素的计算，
    • 主要用于区分当前使用那一条分支来处理球谐光照；
      1. 全像素级计算（UNITY_SAMPLE_FULL_SH_PER_PIXEL）​​
      • 高精度需求时，球谐光照完全在片元着色器计算，顶点着色器仅传递法线；
      1. 完全完全逐顶点计算（低端设备）​​
      • 低端 GPU（如 Shader Model 2.0）无法支持复杂片元计算，直接调用 ShadeSH9 函数、在顶点着色器完成全部球谐光照计算；
      1. 分工计算：
      • （1）顶点着色器​：处理 L2 阶（二次项）球谐光照，因其计算量较大但插值误差影响小；
      • （2）片元着色器​：处理 L0/L1 阶（常数/线性项），保证高频细节（如伽马校正）的精度；
  • 代码：ShadeSHPerVertex
// 函数：ShadeSHPerVertex
// 作用：计算顶点级别的球谐光照（光照探针数据），并叠加到输入的 ambient 值上
// 参数：
//   - normal：归一化的世界空间法线（用于计算光照方向）
//   - ambient：初始环境光值（通常包含逐顶点光照或其他预计算光照）
// 返回值：叠加球谐光照后的环境光值（RGB）
half3 ShadeSHPerVertex (half3 normal, half3 ambient) 
{
    // 条件编译分支 1：全像素级球谐光照（跳过顶点计算）
    #if UNITY_SAMPLE_FULL_SH_PER_PIXEL
        // 完全在片元着色器中计算球谐光照，此处无需处理
        // 直接返回输入的 ambient 值
        
    // 条件编译分支 2：低端设备或简单渲染管线（完全逐顶点计算）
    #elif (SHADER_TARGET < 30) || UNITY_STANDARD_SIMPLE
        // 调用 ShadeSH9 计算完整的 3 阶球谐光照（9 个系数）
        // max(0, ...) 确保光照非负
        ambient += max(half3(0,0,0), ShadeSH9(half4(normal, 1.0)));
        
    // 条件编译分支 3：中高端设备（顶点与像素分工计算）
    #else
        // 将输入的 ambient 从伽马空间转换到线性空间（如需）
        #ifdef UNITY_COLORSPACE_GAMMA
            ambient = GammaToLinearSpace(ambient);
        #endif
        
        // 仅计算 L2 阶（二次项）球谐光照，叠加到 ambient
        // 注：L0（常数项）和 L1（线性项）已在片元着色器中处理
        ambient += SHEvalLinearL2(half4(normal, 1.0));
    #endif
    return ambient;
}
  • 分析：ShadeSHPerPixel 函数实现
    • 该函数的核心是通过球谐函数（SH）分阶计算间接光照；
    • Unity 将 SH 光照分为三阶（L0/L1/L2），其中 L0（常数项）和 L1（线性项）通常包含低频光照信息，而 L2（二次项）贡献高频细节。根据目标平台性能，函数动态选择计算方式：
      • 高配设备（UNITY_SAMPLE_FULL_SH_PER_PIXEL）在片元着色器完成全阶计算；
      • 中端设备由顶点着色器处理 L2 阶，片元着色器补充 L0/L1 阶；
      • 低端设备（如 S hader Model 2.0）则完全依赖顶点着色器；
    • 当启用 UNITY_LIGHT_PROBE_PROXY_VOLUME 时，函数通过 unity_ProbeVolumeParams.x 检测 LPPV 是否激活。若激活，则调用 SHEvalLinearL0L1_SampleProbeVolume 从代理体积中采样光照数据，否则回退到传统光照探针的 SH 计算；
    • LPPV 通过空间划分和流式加载优化了大场景的动态光照插值，尤其适合开放世界或大型动态物体（如载具），避免了传统探针组的接缝问题；
    • 函数始终以线性空间处理光照计算，仅在最终输出前根据 UNITY_COLORSPACE_GAMMA 决定是否转换为伽马空间。这种设计确保了光照叠加（如 ambient += ambient_contrib）的物理正确性。此外，通过 max(half3(0, 0, 0), ...) 强制非负值，防止因法线或探针数据错误导致的异常暗区。这种鲁棒性处理使得函数能稳定适配不同渲染路径（如前向渲染或延迟渲染）的光照需求；
  • 代码：
// 函数：ShadeSHPerPixel
// 作用：在片元着色器中计算球谐光照（SH）的贡献，并叠加到输入的 ambient 值上，支持光照探针代理体积（LPPV）和传统光照探针
// 参数：
//   - normal：归一化的世界空间法线（用于计算光照方向）
//   - ambient：初始环境光值（通常包含逐顶点计算的L2阶SH光照或其他预计算光照）
//   - worldPos：当前片元的世界坐标（用于LPPV采样）
// 返回值：叠加球谐光照后的环境光值（RGB）
half3 ShadeSHPerPixel(half3 normal, half3 ambient, float3 worldPos) 
{
    half3 ambient_contrib = 0.0; // 初始化球谐光照贡献值

    // 条件编译分支1：全像素级球谐光照计算（高精度模式）
    #if UNITY_SAMPLE_FULL_SH_PER_PIXEL
        // 判断是否使用光照探针代理体积（LPPV）
        #if UNITY_LIGHT_PROBE_PROXY_VOLUME
            if (unity_ProbeVolumeParams.x == 1.0) // 检查LPPV是否激活
                ambient_contrib = SHEvalLinearL0L1_SampleProbeVolume(half4(normal, 1.0), worldPos); // 从LPPV采样L0/L1阶SH
            else
                ambient_contrib = SHEvalLinearL0L1(half4(normal, 1.0)); // 传统光照探针的L0/L1阶计算
        #else
            ambient_contrib = SHEvalLinearL0L1(half4(normal, 1.0)); // 默认L0/L1阶计算
        #endif

        ambient_contrib += SHEvalLinearL2(half4(normal, 1.0)); // 叠加L2阶SH光照
        ambient += max(half3(0, 0, 0), ambient_contrib); // 叠加并确保非负

        // 伽马空间转换（如需）
        #ifdef UNITY_COLORSPACE_GAMMA
            ambient = LinearToGammaSpace(ambient);
        #endif

    // 条件编译分支2：低端设备或简单渲染管线（完全逐顶点计算，此处无需处理）
    #elif (SHADER_TARGET < 30) || UNITY_STANDARD_SIMPLE
        // 伽马转换已在顶点着色器完成（见ShadeSHPerVertex）

    // 条件编译分支3：中高端设备（L2阶逐顶点计算，L0/L1阶逐像素计算）
    #else
        // 处理L0/L1阶SH光照（类似分支1的逻辑）
        #if UNITY_LIGHT_PROBE_PROXY_VOLUME
            if (unity_ProbeVolumeParams.x == 1.0)
                ambient_contrib = SHEvalLinearL0L1_SampleProbeVolume(half4(normal, 1.0), worldPos);
            else
                ambient_contrib = SHEvalLinearL0L1(half4(normal, 1.0));
        #else
            ambient_contrib = SHEvalLinearL0L1(half4(normal, 1.0));
        #endif

        ambient = max(half3(0, 0, 0), ambient + ambient_contrib); // 叠加L0/L1阶贡献（L2阶已在顶点着色器处理）
        #ifdef UNITY_COLORSPACE_GAMMA
            ambient = LinearToGammaSpace(ambient); // 伽马转换
        #endif
    #endif

    return ambient; // 返回最终环境光
}

使用 Meta Pass 计算自发光物体的间接光

概念：使用 Meta Pass 计算自发光物体的间接光

  • 概念：
    • Meta Pass 是 Unity 光照烘焙系统的专用 Shader Pass，用于在烘焙光照贴图时提取物体的反照率（Albedo）​和自发光（Emission）​数据，参与全局间接光照计算；
  • 代码：
Shader "Custom/MetaPassExample" {
    Properties {
        _Color ("Albedo (GI)", Color) = (1,1,1,1)  // 仅影响烘焙的间接光颜色
        _EmissionColor ("Emission (GI)", Color) = (0,0,0,0)
        _MainTex ("Albedo Tex (GI)", 2D) = "white" {}
    }

    SubShader {
        // 常规渲染Pass（此处省略）
        // ...

        // Meta Pass（光照烘焙专用）
        Pass {
            Name "META"
            Tags { "LightMode" = "Meta" }  // 标记为Meta Pass
            Cull Off  // 禁用裁剪确保所有面参与烘焙

            CGPROGRAM
            #pragma vertex vert_meta
            #pragma fragment frag_meta
            #pragma shader_feature _EMISSION  // 支持自发光开关
            #pragma shader_feature EDITOR_VISUALIZATION  // 兼容材质验证器

            #include "UnityCG.cginc"
            #include "UnityMetaPass.cginc"

            sampler2D _MainTex;
            float4 _MainTex_ST;
            fixed4 _Color;
            fixed4 _EmissionColor;

            // 顶点输入结构
            struct appdata_meta {
                float4 vertex : POSITION;
                float2 uv : TEXCOORD0;
                float2 uv1 : TEXCOORD1;  // 光照贴图UV1
                float2 uv2 : TEXCOORD2;  // 动态光照贴图UV2
            };

            // 顶点着色器
            v2f_meta vert_meta(appdata_meta v) {
                v2f_meta o;
                // 转换顶点位置到光照贴图空间
                o.pos = UnityMetaVertexPosition(v.vertex, v.uv1, v.uv2, unity_LightmapST, unity_DynamicLightmapST);
                o.uv = TRANSFORM_TEX(v.uv, _MainTex);  // 处理纹理UV
                #ifdef EDITOR_VISUALIZATION
                    // 编辑器可视化数据（可选）
                    o.vizUV = 0;
                    o.lightCoord = 0;
                #endif
                return o;
            }

            // 片元着色器
            fixed4 frag_meta(v2f_meta i) : SV_Target {
                UnityMetaInput metaIN;
                UNITY_INITIALIZE_OUTPUT(UnityMetaInput, metaIN);

                // 从纹理和颜色属性获取反照率
                fixed4 tex = tex2D(_MainTex, i.uv);
                // 计算自发光的效果
                metaIN.Albedo = tex.rgb * _Color.rgb;

                // 设置自发光（需启用_EMISSION）
                #ifdef _EMISSION
                    metaIN.Emission = _EmissionColor.rgb * tex.a;  // 使用纹理Alpha控制自发光强度
                #else
                    metaIN.Emission = 0;
                #endif

                // 返回烘焙数据（自动处理反照率/自发光切换）
                // 这是Unity内部计算Emissions的间接光反射的函数
                return UnityMetaFragment(metaIN);
            }
            ENDCG
        }
    }
    FallBack "Diffuse"
}
