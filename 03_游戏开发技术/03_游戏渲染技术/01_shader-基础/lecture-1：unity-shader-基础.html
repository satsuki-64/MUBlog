Lecture 1：Unity Shader 基础

Unity Shader 基础概念

基础概念

概念：Unity shader 和 shader 的区别

  • 概念：
    • shader 是一个更通用的概念，用于描述图形渲染程序中的着色器程序，传统 Shader（着色器）是运行在 GPU 上的小程序，用于控制图形渲染管线中的各个阶段，决定如何将 3D 模型渲染到 2D 屏幕上；
    • 而 Unity shader 是特指在 Unity 中使用的着色器，你可以认为 Unity shader 是对 shader 的一种封装，通过 ShaderLab 语言编写，Unity 会自动编译为不同平台所需的底层 Shader 代码，并且在一个文件中可以包含顶点着色器、片元着色器以及渲染设置；
    • 它是对底层图形渲染技术的封装，它提供了一种叫做 shaderLab 的语言来让我们更加轻松的编写和管理着色器；
  • 概念：Unity 中的材质和 Shader
    • 材质是 Unity 中用于定义物体表面视觉属性的资源，它结合了 Shader 与参数设置；
    • 如果我们想要在 unity 中体现出一个 shader 的渲染效果，必须配合使用材质和 Shader (Unity shader)才能达到目标，材质必须指定一个 Unity Shader 作为其渲染逻辑；
  • 总结：
    • 输入的贴图或者颜色等，加上对应的 Shader，以及对 Shader 的特定的参数设置，将这些内容 (Shader 及输入参数)打包存储在一起，得到的就是一个 Material (材质)；

概念：四个基础概念

  • 概念：GLSL
    • 基于 OpenGL 的着色器语言，主要用于 OpenGL/WebGL 平台；
    • 跨平台性较好，但依赖显卡驱动编译，不同硬件效果可能不一致，Unity 支持 GLSL，但官方推荐使用 Cg/HLSL，因其跨平台适配更优；
  • 概念：HLSL
    • 微软为 Direct3D（DirectX） 设计的着色器语言，主要应用于 Windows/Xbox 平台；
    • 其语法与 Cg 高度相似，两者可视为同一种语言（因 NVIDIA 与微软合作开发）；
    • Unity 底层会将 Cg/HLSL 代码编译为平台特定的语言（如 GLSL 或 HLSL）；
  • 概念：CG - C for Graphics
    • 由 NVIDIA 与微软联合开发的跨平台着色器语言，旨在兼容 OpenGL 和 Direct3D；
    • 语法与 HLSL 几乎一致，代码可互相转换，真正的跨平台性，能编译为 GLSL 或 HLSL 等目标语言，Unity 早期主要使用 Cg，后逐渐转向 HLSL，但两者语法兼容，代码通常以 CGPROGRAM 块编写；
  • 概念：ShaderLab
    • Unity 封装的专属着色器框架，用于简化跨平台 Shader 开发，其并非传统着色器语言，而是基于 HLSL/Cg 的抽象层，提供声明式语法（如 Properties、SubShader 等）；
    • ShaderLab 支持嵌入 Cg/HLSL 或 GLSL 代码块（CGPROGRAM 或 GLSLPROGRAM），自动处理平台适配，开发者无需直接编写 GLSL/HLSL；
  • 关系：
      1. HLSL 与 Cg 语法几乎相同，GLSL 独立；ShaderLab 通过封装 Cg/HLSL 实现统一编写；
      1. 开发者用 ShaderLab 编写框架，在 CGPROGRAM 块中写 Cg/HLSL 代码， Unity 编译时，将 Cg/HLSL 转换为目标平台语言（如 GLSL for OpenGL，HLSL for Direct3D）；

使用/创建材质

概念：使用/创建材质

  • 总结：创建 Shader 并关联材质
      1. 创建 Shader： 编写或选择现有的 Unity Shader，定义渲染逻辑和可调参数；
      1. 创建材质：在 Project 视图中新建 Material 资源；
      1. 关联 Shader：将 Shader 赋给材质，此时材质会显示 Shader 中定义的所有属性；
      1. 配置参数：在材质 Inspector 中调整各种属性（颜色、纹理、数值等）；
      1. 应用渲染：将材质赋给场景中的 Mesh Renderer 或 Particle System；
  • 总结：创建材质
      1. 创建好材质后，可以选中它后，在 Inspector 窗口中 shader 选项中选择对应的着色器进行关联使用
      1. Inspector 窗口下方的内容就是选中的 shader 提供的可编辑变化的相关变量，他们会直接影响渲染结果
      1. 关联好 shader 后，材质需要赋值给 GameObject 对象上依附的 Mesh Renderer 等相关渲染器组件上进行使用；
  • 总结：创建 Shader
      1. 选择创建不同的 Shader
      • (1) standard surface shader：标准曲面着色器
        • 包含标准光照模型的表面着色器模板；
      • (2) Unlit Shader
        • 不包含光照的基本顶点/片元着色器；
      • (3) Image Effect Shader
        • 用于实现屏幕后处理效果的基本模板；
      • (4) Compute Shader
        • 利用 GPU 并行计算一些和常规渲染流水线无关的内容；
      • (5) Ray Tracing Shader
        • 用于实现光线追踪效果的着色器；
  • 概念：Shader 重要选项
      1. Surface Shader (表面着色器)：
      • Show generated code (显示生成的代码)；
      • 如果是表面着色器，这里可以点击对应的按钮，在这里可以查看对应的代码；
      1. Fixed function
      • 是否使用固定管线进行渲染 (非常者的版本使用的方式)；
      1. Preprocess only
      • 是否开启预处理阶段执行特定代码标记；
      1. Compiled code (Compile and show code)
      • 编译并显示代码；
      • 可以查看在不同平台时使用不同的图形接口程序时编译出来的代码，可以帮助我们优化着色器性能；
      1. Propertoes
      • 可以编辑的属性；
  • 总结：

ShaderLab 基础

ShaderLab 基础概念

概念：ShaderLab 基础概念

  • 概念：
    • 无论我们编写哪种类型的 shader，或是选择哪种语言去编写 shader，在 Unity 中总会通过 ShaderLab 语言对其进行包装和组织；
  • 概念：ShaderLab 基本结构
    • ShaderLab 主要由以下四个部分构成：
      1. Shader 名称；
      1. Shader 属性；
      1. n 个子着色器；
      1. 备用 Shader；
  • 概念：ShaderLab 语法：
shader "着色器名称"
{
	// 着色器属性
	// 其中的属性可以在材质面板上看到
	Properties
	{
	
	}

	// 子着色器
	SubShader
	{
		// 顶点-片段着色器 或 表面着色器 或 固定函数着色器
	}
	
	SubShader
	{
		// 更为精简的版本
		// 用于适配旧的设备或平台
	}
	// ... 可以有n个SubShader块

	// 如果前面的SubShader全部都不能使用，则使用这个备用的Shader
	Fallback "备用的Shader"
}

Shader 的名称

概念：Shader 的名称

  • 概念：
    • Shader 的名称在 Shader 文件当中，修改 Shader 名称只需要直接在文件中修改；
    • Shader 的名字决定了在村质面板的选择路径；
  • 图示：使用 Shader
    • 根据 Shader 文件中的名词的层级，来选择 Shader；
  • 注意：
      1. Shader 名称不能为中文；
      1. 建议 Shader 名称和其文件名一致；

Shader 的属性

概念：Shader 的属性

  • 概念：Shader 属性的作用
    • 在 Shader 编写时我们经常会用到不同类型的变量或贴图等资源，为了增加 shader 的可调节性，有些变量不会直接在 shader 程序中写死，而是作为开放的属性显示在我们的材质面板上，供我们使用时调节，而这些开放的属性就是通过属性来定义的；
  • 特点：Shader 属性的特点
      1. 可以在材质面板被编辑；
      1. 可以在后续当作输入变量提供给所有子着色器使用；
  • 图示：
  • 概念：Shader 属性语法
    • Shader 属性是存在于 shader 语句块中的 Properties 属性语句块，我们只需要在 properties 语句块中按照语法规则声明属性即可；
    • Unity shader 的属性主要分成三大类：数值、颜色和向量、纹理贴图；
    • 其基本语法如下：
_Name("Display Name" , type) = defaultValue[{options}]

// 1. _Name：属性名称，规则是需要在前面加一个下划线，方便在之后获取
// 2. Display Name：材质面板上显示的名字
// 3. type：属性的类型，不可以使用中文
// 4. defaultValue：将shader指定给材质的时候初始化的默认值
  • 概念：Shader 的基础数值类型
Shader "UI/ShaderTest"
{
	Properties
	{
		_MainTex("Texture",2D) = "white" {}
		
		// 1. 整型
		// _Name("Display Name", Int) = number
		_MyInt("MyInt",Int) = 1
		
		// 2. 浮点数
		_MyFloat("MyFloat",Float) = 0.5
		
		// 3. 范围类型
		_MyRange("MyRange",Range(1,5)) = 2
	}
	// ...
}
  • 注意：
    • Unity Shader 中的数值类型属性基本都是浮点型 (Float)数据，虽然提供了整数 (Int)，但是编译时最终都会转换为浮点型，因此我们更多的使用的还是 Float 类型；
  • 概念：Shader 的颜色/向量类型
    • 颜色是由 RGBA 四个分量代表，注意颜色值中的 RGBA 的取值范围是 0~1 (映射 0~255)
    • 向量是由 xyzw 四个分量代表，其值大小没有限制；
    • 他们都可以由一个四个数组成的类型表示；
Shader "UI/ShaderTest"
{
	Properties
	{
		// 1. 颜色
		// _Name("Display Name",Color) = (number1,number2,number3,number4)
		_MyColor("MyColor",Color) = (0.5,0.5,0.5,1)
		
		// 2. 向量
		// _Name("Display Name",Vector) = (number1,number2,number3,number4)
		_MyVector("MyVector",Vector) = (0.5,0.5,0.5,1)
	}
	// ...
}
  • 概念：纹理贴图类型属性
      1. 2D 纹理：最常用的纹理，漫反射贴图、法线贴图都属于 2D 纹理；
      • _Name("Display Name",2D) = "defaulttexture"{}
      1. 2D Array 纹理：纹理数组，允许在纹理中存储多层图像数据，每层看做一个 2D 图像，一般使用脚本创建，较少使用，了解即可；
      1. Cube map texture 纹理：立方体纹理，由前后左右上下 6 张有联系的 2D 贴图拼成的立方体，比如天空盒和反射探针；
      1. 3D 纹理：一般使用脚本创建，极少使用，了解即可；
  • 注意：关于 defaulttexture 默认值取值
    • 不写：默认贴图为空；
    • white：默认白色贴图 (RGBA: 1 1 1 1)
    • black：默认黑色贴图 (RGBA: 0,0,0,1)
    • gray：默认灰色贴图 (RGBA: 0.5,0.5,0.5,1)
    • bump：默认凸贴图 (RGBA: 0.5,0.5,1,1)
      • 一般用于法线贴图默认贴图
    • red：默认红色贴图 (RGBA: 1,0,0,1)
    • 手动设置贴图默认值：
      • 在 Shader 的界面设置默认贴图；
  • 注意：
    • 默认贴图的名称为 _MainTex("MainTex",2D) = "white"{}
// 贴图纹理类型
_My2D("My2D", 2D) = "White"{}
_MuCube("MyCube",Cube) = ""{}
  • 注意：
    • 属性的最后不需要跟上分号；

Shader 的子着色器

SubShader 基本构成

概念：SubShader 基本构成

  • 概念：
    • 每一个 shader 中都会包含至少个 subshader，当 Unity 想要显示一个物体的时候，就会在 shader 文件中去检测这些 subshader 语句块，然后选择第一个能够在当前显卡运行的 subshader 进行执行；
    • 因此在一个 shader 当中实现一些高级效果时，为了避免在在某些设备上无法执行，可能会存在多个 subshader 语句块，用于适配这些低端设备；
    • subshader 当中包含最终的渲染相关代码，决定了最终的染效果；
  • 概念：SubShader 基本构成
      1. 渲染标签：
      • 通过标签来确定什么时候、以及什么方式来对物体进行渲染；
      1. 渲染状态：
      • 通过状态来确定渲染时的提出方式、深度测试方式、混合方式等；
      1. 渲染通道：
      • 具体实现着色器代码的地方，每个 subshader 语句块中至少有一个渲染通道，可以有多个；
  • 概念：渲染通道与渲染
    • 在 Subshader 中，每定义一个渲染通道 Pass，就会让物体执行一次渲染，n 个 Pass，就会有 n 次渲染；
    • 在实现一些复杂渲染效果时，需要使用多个 Pass 进行组合实现，但是我们要尽量减少它的数量，更多的 Pass 会增加性能消耗；
  • 举例：
shader "着色器名称"
{
	// 着色器属性
	// 其中的属性可以在材质面板上看到
	Properties
	{
	
	}

	// 子着色器
	SubShader
	{
		Tags{"标签名1" = "标签值1" "标签名2" = "标签值2"}
			
		// 2. 渲染状态
		
		// 3. 渲染通道
		Pass
		{
			// 第一个渲染通道
		}
		
		Pass
		{
			// 第二个渲染通道
		}
	}
	
	SubShader
	{
		// 更为精简的版本
		// 用于适配旧的设备或平台
	}
	// ... 可以有n个SubShader块

	// 如果前面的SubShader全部都不能使用，则使用这个备用的Shader
	Fallback "备用的Shader"
}

SubShader 标签

基础概念

概念：SubShader 标签

  • 概念：
    • SubShader 标签是 Unity Shader 编程中控制渲染行为的关键组成部分，它定义了着色器与 Unity 渲染引擎之间的通信规则，即：通过标签来确定什么时候以及如何对物体进行渲染；
    • 比如标签有以下几大作用：
        1. 控制渲染顺序​：通过"Queue"标签指定物体在渲染管线中的绘制顺序，确保透明物体在不透明物体之后渲染，避免深度冲突；
        1. 着色器分类与替换​："RenderType"标签允许开发者对着色器进行分类（如不透明/透明），便于运行时通过 Camera.RenderWithShader 或 Camera.SetReplacementShader 实现着色器替换功能；
        1. 批处理控制​："DisableBatching"标签可以禁用 Unity 的动态批处理，解决某些特殊着色器在批处理后出现错误的问题；
        1. 阴影与投影控制​：通过"ForceNoShadowCasting"和"IgnoreProjector"标签控制物体是否投射阴影或受 Projector 组件影响；
        1. 精灵图集兼容性​："CanUseSpriteAtlas"标签指示着色器是否兼容精灵（Spite）图集，对于 2D 游戏开发尤为重要；
        1. 材质预览控制​："PreviewType"标签决定材质在 Inspector 面板中的预览显示方式（如球形、平面等）；
  • 概念：基本语法结构
    • SubShader 标签采用严格的键值对(Key/Value Pair)语法格式：
      • Tags{ "标签名1" = "标签值1" "标签名2" = "标签值2" "标签名3"="标签值2" ......
    • 基本结构如下：
// 写法一：标签分开书写
Tags {
    "Queue" = "Transparent"
    "RenderType" = "Transparent"
}

// 写法二：标签合并书写
Tags { "Queue" = "Transparent" "RenderType" = "Transparent" }

渲染队列

概念：渲染队列

  • 概念：
    • Queue 标签是最重要的 SubShader 标签之一，它决定了物体的渲染顺序；
    • Unity 先从 Render Queue 的值最小的物体开始渲染；
    • 其中：Render Queue = 2500 是一个分水岭；
        1. 渲染队列数目 小于 2500 的对象，都被认为是不透明的物体，其采用数值上从前往后渲染；
        1. 渲染队列数目 高于 2500 的物体，被被认为是半透明物体，其采用数值上从后往前渲染；
    • 可通过在值后加数字的方式来改变队列："Queue"="Geometry+1”；
  • 概念：常用渲染队列值
    • Background​​：值 1000，最先渲染，用于天空盒等背景元素；
    • Geometry：值 2000，其同时是默认队列，大部分不透明物体使用此队列；
    • AlphaTest​：值 2450，需要进行 Alpha 测试的物体（如带镂空的植被）
      • 有透明通道的，需要进行 Alpha 测试的几何体会使用该列；
      • 当所有 Geometry 队列实体绘制完后再绘制 AlphaTest 队列，效率更高；
    • Transparent​：值 3000，透明物体队列，渲染顺序从远到近
      • 该队列中几何体按照由远到近的顺序进行绘制，半透明物体的渲染队列，所有进行透明混合的几何体都应该使用该队列；
      • 比如：玻璃材质，粒子特效；
    • Overlay​：值 4000，最后渲染，用于 GUI、镜头光晕等叠加效果；
      • 用是放在最后渲染的队列，用于叠加渲染的效果，比如镜头光晕等；
    • 自定义队列值：基于 Unity 预先定义好的这些渲染队列标签来进行加减运算，进而来定义自己的渲染队列；
      • 注意：加/减符号不需要空格
      • 自定义队列只能基于预先定义好的各类型进行计算，不能在 shader 中直接赋值数字；
      • 如果实在想要直接赋值数字，可以在材质面板中进行设置；
  • 图示：Unity 的 Frame
      1. 先渲染不透明对象：OpaqueGeometry；
      1. 再渲染透明物体：TransparentGeometry；
  • 举例：
Tags
{
	"RenderType" = "Opaque"
	"Quene" = "Background"
}
  • 举例：自定义队列值
Tags { "Queue" = "Geometry+10" } // 在Geometry之后10个单位渲染，所对应的队列为2010

渲染类型

概念：渲染类型 - RenderType

  • 概念：
    • RenderType 标签用于着色器分类，是实现着色器替换的关键；
    • 对着色器进行分类，之后可以用于着色器替换功能，摄像机上有对应的 API，可以指定这个渲染类型来替换成别的着色器；
  • 语法：
    • Tags { "RenderType" = "TypeName" }
    • Opaque​：不透明物体，如大部分场景物体；
    • Transparent​：透明物体，如玻璃、粒子效果）
    • TransparentCutout：用于透明测试的 Shader，比如植物叶子；
    • Background​：天空盒等背景元素；
    • Overlay​：叠加效果，如 GUI、Halo （光环）、Flare（光晕）；
    • 自定义类型：可以定义任意字符串作为自定义类型；

批处理

概念：批处理

  • 概念：什么是批处理
    • 批处理是指将多个物体的渲染操作合并为一个批次 (Batch)，从而减少 Draw Call 的数量。Draw Call 是 CPU 向 GPU 发起的一次渲染指令，每次调用都会触发 GPU 绘制一个物体；
    • Unity 提供两种主要的批处理方式：静态批处理 (Static Batching)和动态批处理 (Dynamic Batching)，它们的工作原理和适用场景各不相同；
  • 概念：静态批处理
    • 静态批处理在运行开始阶段 (场景加载时或构建时)，将标记为 Static 的物体合并到一个新的网格结构中；
        1. 将静态物体从模型空间变换到世界空间下；
        1. 为它们构建一个更大的顶点和索引缓冲区；
        1. 对于使用同一材质的物体，Unity 只需调用一个 Draw Call 绘制全部物体；
    • 静态批处理的方式只在初始化时合并一次，运行时不再处理，并且合并后的网格数据已转换到世界空间；
  • 概念：动态批处理
    • 动态批处理是 Unity 在每帧运行时自动执行的：
        1. 检查符合条件的小型动态物体；
        1. 将这些物体的顶点数据在 CPU 上转化为世界空间；
        1. 合并顶点数据后，一次性发送给 GPU；
        1. 使用同一材质进行渲染；
    • 其技术特点是每帧都会重新合并网格，因此物体可以移动，但需要在 CPU 上进行顶点变换和合并操作；
  • 概念：
    • 当使用批处理时，模型会被变换到世界空间中，模型空间会被丢弃；
    • 这可能会导致某些使用模型空间的顶点数据的 shader 最终无法实现想要的结果；
    • 因此，可以通过开启禁用批处理来解决该问题；
  • 语法：
    • 总是禁用批处理：Tags{ "DisableBatching"= "True" }
    • 不禁用批处理：Tags{ "DisableBatching"= "False" }
    • LOD 效果激活时才会禁用批处理，主要用于地形系统上的树：Tags{ "DisableBatching"= "LODFading" }

SubShader 渲染状态

概念：SubShader 渲染状态

  • 概念：
    • 通过状态来确定渲染时的剔除方式、深度测试方式、混合方式等等内容；
  • 语法：
    • 渲染状态 状态类型
    • 渲染状态是通过 渲染状态关键词+空格+状态类型 决定的，如果存在多个渲染状态，可以通过空行隔开；

剔除方式

概念：剔除方式

  • 概念：
    • 设置多边形的剔除方式，有背面剔除、正面剔除、不剔除；
    • 所谓的剔除，就是不渲染，背面剔除就是背面不渲染，正面剔除就是正面不渲染，不剔除就是都渲染；
  • 概念：三种类别
    • Cull Back：背面剔除 -> 进入模型的内部之后，内部看不到模型；
    • Cull Front：正面剔除 -> 正面不渲染，进入模型之后、看到背面渲染；
    • Cull Off：不剔除 -> 正面和背面都进行渲染，因此进入模型之后也能看到贴图；
    • 不设置的话，默认为背面剔除；

深度缓冲和深度测试

概念：深度缓冲和深度测试

  • 概念：深度缓冲
    • 概念：
      • 是否写入深度缓冲；
      • 深度缓冲 (Depth Buffer) 是一个与屏幕像素对应的缓冲区，用于存储每个像素的深度值 (距离相机的距离)；
      • 在渲染场景之前，深度缓冲被初始化为最大深度值，表示所有像素都在相机之外，最后留在深度缓冲中的信息会被渲染；
    • 语法：
      • ZWrite On：写入深度缓冲；
      • ZWrite Off：不写入深度缓冲；
    • 注意：
        1. 不设置的话，默认为写入；
        1. 在做透明等特殊效果时，会设置为不写入，保证半透明物体下面的物体可以被渲染；
  • 概念：深度测试
    • 概念：
      • 设置深度测试的对比方式；
      • 深度测试的主要目的是确保在渲染时，像素按照正确的深度 (距离相机的距离) 顺序进行绘制，从而创建正确的遮挡关系和透视效果；
      • 在渲染场景之前，深度缓冲被初始化为最大深度值，表示所有像素都在相机之外；
      • 在渲染过程中，对于每个像素，深度测试会将当前像素的深度值与深度缓冲中对应位置的值进行比较，一般情况下：
        1. 如果当前像素的深度值小于深度缓冲中的值，说明当前像素在其他物体之前，它会被绘制，并更新深度缓冲；
        1. 如果当前像素的深度值大于等于深度缓冲中的值，说明当前像素在其他物体之后，它会被丢弃，不会被绘制, 并保持深度缓冲不变；

  • 注意：
    • 我们只有在实现一些特殊效果时才会修改深度测试方式，比如透明物体渲染会修改为 Less，描边效果会修改为 Greater 等；

混合方式

概念：混合方式

  • 概念：混合方式
    • 混合方式是在深度测试之后才会进行的，在经过了混合之后，将会将最终结果输出到颜色缓冲区当中；
    • 可以设置渲染图像的混合方式，实现多种颜色叠加混合，比如透明、半透明效果和遮挡的物体进行颜色混合；
    • 需要多种颜色叠加渲染时，就需要设置混合方式，具体情况具体处理；
  • 公式：
    • 最终颜色 = 源颜色 * SrcFactor + 目标颜色 * DstFactor
  • 概念：混合流程
    • 概念：
        1. 判断是否开启混合；
        1. 开启混合后，得到片元的颜色值 + 得到目标颜色，即已经存在在颜色缓冲区中的颜色；
        1. 进行混合操作；
        1. 结束，更新颜色缓冲区中的颜色；
      • 因此，混合就是将当前片元原本的颜色、将其和颜色缓冲区中的颜色进行混合，混合之后再存储颜色缓冲区当中；
    • 图示：
  • 语法：
      1. 正常透明混合：Blend SrcAlpha OneMinusSrcAlpha
      • 公式：FinalColor = SrcColor * Src.a + DstColor * (1 - Src.a)
      • 作用：标准透明度混合，模拟物理透明效果；
      1. 线性减淡：Blend One One
      • 公式：FinalColor = SrcColor * 1 + DstColor * 1
      • 作用：简单将源颜色和目标颜色相加，常用于发光体、粒子效果；
      1. 过滤颜色：Blend OneMinusDstColor One
      • 公式：FinalColor = SrcColor * (1 - DstColor) + DstColor * 1
      • 作用：类似 Photoshop 中的"滤色"模式，源颜色乘以 (1 - 目标颜色)，进而保留较亮部分；
      1. 正片叠底：Blend DstColor Zero
      • 公式：FinalColor = SrcColor * DstColor + DstColor * 0
      • 作用：源颜色乘以目标颜色，而目标颜色乘以 0、表示不参与，常用于阴影、颜色叠加；
      1. X 光：Blend DstColor SrcColor
      • 公式：FinalColor = SrcColor * DstColor + DstColor * SrcColor
      • 作用：产生颜色交换的特殊效果；
      1. 透明度混合：Blend One OneMinusSrcAlpha
      • 公式：FinalColor = SrcColor * 1 + DstColor * (1 - Src.a)
      • 作用：- 源颜色已经预乘了 Alpha 值，目标颜色乘以(1 - 源 Alpha 值)，常用于 UI 元素和粒子系统；
  • 注意：
    • 默认情况下不会进行混合；

其他渲染状态

概念：其他渲染状态

  • 概念：LOD
    • 控制 LOD 级别，在不同距离下使用不同的渲染方式处理；
    • 比如：LOD 100；
  • 概念：ColorMask
    • 设置颜色通道的写入蒙版，默认蒙版为 RGBA；
  • 注意：
    • 以上这些状态不仅可以在 subshader 语句块中声明，Pass 渲染通道语句块中也可以声明这些渲染状态；
    • 如果在 subshader 语句块中使用，会影响之后的所有渲染通道 Pass ；
    • 如果在 Pass 语句块中使用，则只会影响当前 Pass 渲染通道，不会影响其他的 Pass；

SubShader Pass 渲染通道

概念：SubShader Pass 渲染通道

  • 概念：
    • Pass 渲染通道是具体实现着色器代码的地方，每个 SubShader 语句块中至少有一个渲染通道，可以有多个；
  • 语法：
Pass
{
	1. Name 名称
	2. 渲染标签
	3. 渲染状态
	4. 其他着色器代码
}

概念：Pass 名称

  • 概念：Pass 名称
    • 对 Pass 命名的主要目的，是可以利用 UsePass 命令在其他 Shader 当中复用该 Pass 的代码；
    • 只需要在其他 Shader 当中使用：
      • UsePass "Shader路径/Pass名"
    • 既可以复用 Pass 代码；
  • 注意：
    • Unity 内部会把 Pass 名称转换为大写字母，因此在使用 UsePass 命令时必须使用大写形式的名字；
  • 举例：
UsePass "UI/MyShader/MyLesson"

Pass
{
	Name "MyLesson"
	// ...
}

概念：Pass 渲染标签

  • 概念：
    • Pass 中的标签和 SubShader 中使用的标签不同；
  • 总结：Tags{ LightMode = 标签值 }
    • LightMode 标签可以指定该 Pass 应该在哪个阶段执行，比如可以将着色器代码分配给适当的渲染阶段，以实现所需的效果；
  • 总结：Tags{ RequireOptions = 标签值 }
    • 用于指定当满足某些条件时才渲染该 Pass；
    • Tags{ "RequireOptions"="SoftVegetation"}
      • 仅当 Quality 窗口中开启了 SoftVegetation 时才渲染此通道；
  • 总结：Tags{ PassFlags = 标签值 }
    • Tags{ "PassFlags"= "onlyDirectional"}
    • 在 ForwardBase 向前渲染的通道类型中使用时，此标志的作用是仅允许主方向光和环境光/光照探针数据传递到着色器，这意味着非重要光源的数据将不会传递到顶点光源或球谐函数着色器变量；

概念：Pass 渲染状态

  • 概念：
    • 在 SubShader 语句块中学习的渲染状态同样适用于 Pass；
    • 比如：
      • 剔除方式决定了模型正面背面是否能够被渲染；
      • 深度缓冲和深度测试决定了深度关系的确定以及透明效果的正确表达等；
      • 混合方式决定了透明半透明颜色的正确表现，以及一些特殊颜色效果的表现；
    • 这些渲染状态都可以在单个 Pass 中进行设置；
  • 注意：
    • 如果在 subshader 语句块中使用会影响之后的所有渲染通道 Pass；
    • 如果在 Pass 语句块中使用只会影响当前 Pass 渲染通道，不会影响其他的 Pass；

概念：GrabPass 命令

  • 概念：
    • 可以利用 GrabPass 命令，把即将绘制对象时的屏幕内容抓取到纹理中，在后续通道中即可使用此纹理，从而执行基于图像的高级效果；
    • 经常用于进行后处理的效果；
  • 举例：
// 将绘制当前Pass渲染通道之前，将当前渲染通道的结果抓取到 _BackgroundTexture 中
GradPass
{
	"_BackgroundTexture"
}

Pass
{
	// ...
}
  • 注意：
    • 该命令一般写在某个 Pass 前，在之后的 Pass 代码中可以利用 BackgroundTexture 变量进行处理；

Shader 的备用着色器

基础概念

概念：Shader 的备用着色器

  • 概念：备用 Shader 的作用
    • ShaderLab 当中，允许有多个 SubShader 子着色器当执行渲染时，会从上到下使用第一个能够正常执行的 SubShader 子着色器来渲染对象；
    • 那有没有可能所有 SubShader 子着色器都不能够在显卡上执行呢? 答案是肯定的，有可能出现在某一个设备上，我们自定义的所有 Subshader 都无法正常执行 (设备显卡无法支持一些 API 等情况)；
    • 那么这时备用 shader 就可以起到很大作用了，它至少可以让对象能够正常渲染出来；
  • 语法：
shader "着色器名称"
{
	// 着色器属性
	// 其中的属性可以在材质面板上看到
	Properties
	{
	
	}

	// 子着色器
	SubShader
	{
		// 顶点-片段着色器 或 表面着色器 或 固定函数着色器
	}
	
	SubShader
	{
		// 更为精简的版本
		// 用于适配旧的设备或平台
	}
	// ... 可以有n个SubShader块

	// 如果前面的SubShader全部都不能使用，则使用这个备用的Shader
	Fallback "备用的Shader"
}

备用 Shader 语法

概念：备用 Shader 语法

  • 概念：
    • Fallback "Shader名"
    • 或者：Fallback off
    • 在 Fallback 关键词后面空格，并通过一个字符串来告诉 Unity “最低级的 Unity shader”是谁；
    • 也可以直接关闭 Fallback 功能，但是如果关闭那就意味着“放弃治疗”；
  • 举例：
Fallback "VertexLit"

Shader 形式

什么是 Shader 形式

概念：Shader 形式

  • 概念：
    • 通过学习我们知道在 Unity shader 当中我们可以通过 ShaderLab 语法去设置很多内容比如属性、渲染状态、渲染标签等等，但是其最主要的作用是需要指定各种着色器所需的代码；
    • 而这些着色器代码即可以放在 subshader 子着色器语句块中，也可以放在其中的 Pass 渲染通道语句块中；
    • 不同的 shader 形式放置着色器代码的位置也有所不同；
  • 分类：
      1. 表面着色器；
      1. 顶点/片元着色器；
      1. 固定函数着色器；

表面着色器

概念：表面着色器

  • 概念：
    • 表面着色器 (surface shader)是 Unity 自己创造的一种着色器代码类型；
    • 它的本质是对顶点/片元着色器的一层封装，它需要的代码量很少，很多工作都帮助我们去完成了；
    • 其缺点是渲染的消耗较大，可控性较低；
    • 它的优点在于，它帮助我们处理了很多光照细节，我们可以直接使用，而无需自己计算实现光照细节；
  • 使用：
    • 选择创建 Standard Surface Shader；
  • 特点：
    • 着色器相关代码被放在 SubShader 语句块中 (并非 Pass)的 CGPROGRAM 和 ENDCG 之间；
  • 总结：
  • 示例：基础的 PBR 材质
// 声明Shader路径及名称，在材质面板中显示为"Custom/PBRExample"
Shader "Custom/PBRExample" {
    // 属性定义在材质面板中可调节的参数
    Properties {
        // 主纹理（漫反射贴图），默认白色纹理
        // "Albedo (RGB)"为Inspector面板显示的提示文本
        _MainTex ("Albedo (RGB)", 2D) = "white" {}
        
        // 金属度参数，范围0（非金属）到1（金属），默认0
        _Metallic ("Metallic", Range(0,1)) = 0.0
        
        // 光滑度参数，范围0（粗糙）到1（光滑），默认0.5
        _Glossiness ("Smoothness", Range(0,1)) = 0.5
    }

    // 【子着色器块】核心逻辑实现区域
    SubShader {
        // 渲染标签：声明该Shader渲染不透明物体
        Tags { "RenderType"="Opaque" }
        
        // 细节级别（Level of Detail），当相机距离超过200单位时可能降级
        LOD 200

        // 开始CG编程语言块
        CGPROGRAM
            // 编译指令：声明表面着色器函数为surf，使用Standard光照模型
            // fullforwardshadows：表示支持所有光源阴影
            #pragma surface surf Standard fullforwardshadows
            
            // 编译指令：指定 Shader 编译目标为Shader Model 3.0
            #pragma target 3.0

            // 声明在CG中使用的变量（需与Properties中定义的名称一致）
            sampler2D _MainTex;  // 主纹理采样器
            half _Metallic;      // 金属度（half为中等精度浮点数）
            half _Glossiness;    // 光滑度

            // 【输入结构体】定义从顶点着色器传递给表面函数的参数
            struct Input {
                float2 uv_MainTex; // 主纹理的UV坐标（必须以uv_开头）
            };

            // 【表面着色函数】核心计算逻辑
            // Input IN：输入数据
            // inout SurfaceOutputStandard o：可修改的表面输出参数
            void surf (Input IN, inout SurfaceOutputStandard o) {
                // 采样主纹理颜色（根据UV坐标获取像素RGBA值）
                fixed4 c = tex2D(_MainTex, IN.uv_MainTex);
                
                // 设置表面漫反射颜色（Albedo）
                o.Albedo = c.rgb;
                
                // 设置金属度（0为非金属，1为金属）
                o.Metallic = _Metallic;
                
                // 设置光滑度（影响高光反射的锐利程度）
                o.Smoothness = _Glossiness;
                
                // 设置透明度（此处未使用透明效果，仅保留原始纹理的Alpha通道）
                o.Alpha = c.a;
            }
        // 结束CG编程语言块
        ENDCG
    }
    // 备选着色器（当主SubShader不支持时回退使用Diffuse着色器）
    FallBack "Diffuse"
}
  • 示例：动态颜色波动效果
// 声明Shader路径及名称，在材质面板中显示为"Custom/WavingColor"
Shader "Custom/WavingColor" {
    // 【属性块】定义在材质面板中可调节的参数
    Properties {
        // 基础颜色属性，默认白色（RGBA格式）
        _MainColor ("Base Color", Color) = (1,1,1,1)
        // 波动速度参数，范围0到5，默认1（控制颜色变化快慢）
        _WaveSpeed ("Wave Speed", Range(0,5)) = 1
    }

    // 【子着色器块】核心渲染逻辑
    SubShader {
        // 渲染标签：声明该Shader渲染不透明物体
        Tags { "RenderType"="Opaque" }

        // 开始CG编程语言块
        CGPROGRAM
            // 编译指令：声明表面着色器函数为surf，使用Lambert光照模型（简化漫反射）
            #pragma surface surf Lambert

            // 声明CG中使用的变量（需与Properties中定义的名称一致）
            fixed4 _MainColor;  // 基础颜色（fixed4为低精度RGBA）
            float _WaveSpeed;    // 波动速度

            // 【输入结构体】定义从顶点着色器传递给表面函数的参数
            struct Input {
                float3 worldPos; // 当前像素的世界坐标（用于空间计算）
            };

            // 【表面着色函数】核心动态效果逻辑
            // Input IN：输入数据（包含世界坐标）
            // inout SurfaceOutput o：可修改的表面输出参数
            void surf (Input IN, inout SurfaceOutput o) {
                // 基于时间的正弦波计算（_Time.y为自场景加载后的秒数）
                // 公式：sin(时间 * 速度) → 生成[-1,1]的周期性波动值
                float wave = sin(_Time.y * _WaveSpeed);
                
                // 设置表面漫反射颜色（Albedo）
                // 将基础颜色与波动值结合：1 + wave*0.2 → 颜色在80%~120%范围内波动
                o.Albedo = _MainColor.rgb * (1 + wave * 0.2);
                
                // 设置透明度（此处固定为1，表示完全不透明）
                o.Alpha = 1;
            }
        // 结束CG编程语言块
        ENDCG
    }
    // 备选着色器（当主SubShader不支持时回退使用内置Diffuse着色器）
    FallBack "Diffuse"
}

顶点/片元着色器

概念：顶点/片元着色器

  • 概念：
    • 我们可以在创建 shader 时，选择创建 Unlit Shader 来快速创建顶点/片元着色器模板通过观察；
    • 顶点/片元着色器的着色器代码是编写在 Pass 语句块中，我们需要自己定义每个 Pass 需要使用的 Shader 代码；
    • 虽然比起表面着色器来说我们需要编写的代码较多，但其可控性更强，可以控制更多的渲染细节；
    • 好处是灵活性更高，决定对性能影响的高低；
  • 特点：
      1. 需要在 Pass 渲染通道中编写着色器逻辑
      1. 可以使用 CG 或 HLSL 两种 shader 语言去编写 shader 逻辑；
      1. 代码量较多，灵活性较强，性能消耗更可控，可以实现更多渲染细节
      1. 适用于光照处理较少，自定义渲染效果较多时 (移动平台首选)；

固定函数着色器

概念：固定函数着色器

  • 概念：
    • 表面着色器和顶点/片元着色器这两种 Unity shader 形式都使用了可编程管线；
    • 而对于一些老设备 (DX 7.0、OpenGL 1.5 或 OpenGL ES 1.1)，它们不支持可编程管线；
    • 着色器这时就需要使用固定函数着色器来进行渲染，这些着色器只能实现一些非常简单的效果；
    • 即使我们现在在 Unity 中使用固定函数着色器来编写 shader，在内部也会被编译为顶点/片元着色器，因此真正意义的固定函数着色器已经不存在了；
  • 特点：
      1. 需要在 Pass 渲染通道中编写着色器逻辑
      1. 需要使用 shaderLab 语法中的渲染设置命令来编写，而非 CG 和 HLSL 着色器语言；
  • 示例：
// 定义Shader路径及名称，在材质面板中显示为"Custom/FixedFunctionExample"
Shader "Custom/FixedFunctionExample" {
    // 【属性块】定义在材质面板中可调节的参数
    Properties {
        // 主颜色属性，默认粉色（RGBA格式）
        _MainColor ("Main Color", Color) = (1, 0.5, 0.5, 1)
        // 环境光颜色，默认灰色（模拟间接光照）
        _Ambient ("Ambient Color", Color) = (0.3, 0.3, 0.3, 1)
        // 高光颜色，默认白色
        _Specular ("Specular Color", Color) = (1, 1, 1, 1)
        // 高光强度，范围0到8，默认4
        _Shininess ("Shininess", Range(0, 8)) = 4
    }

    // 【子着色器块】固定管线的核心逻辑
    SubShader {
        // 【Pass块】定义单个渲染通道
        Pass {
            // 启用固定函数材质定义
            Material {
                // 漫反射颜色（直接受光源影响）
                Diffuse [_MainColor]
                // 环境光颜色（不受光源方向影响）
                Ambient [_Ambient]
                // 高光反射颜色
                Specular [_Specular]
                // 高光反射强度（值越大高光越集中）
                Shininess [_Shininess]
            }
            // 启用光照计算（固定管线必需）
            Lighting On
            // 启用独立高光计算（增强镜面反射效果）
            SeparateSpecular On
        }
    }
    // 备选着色器（当主SubShader不支持时回退使用Diffuse）
    FallBack "Diffuse"
}
