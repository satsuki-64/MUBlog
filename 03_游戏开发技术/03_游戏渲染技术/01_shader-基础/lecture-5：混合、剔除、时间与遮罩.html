混合模式 - Blend

混合的作用

概念：混合的作用

  • 概念：
    • 混合的作用就是用于实现各种半透明的效果；
    • 颜色混合是指将源颜色 (当前 Shader 计算出的颜色，比如当前模型上某个点对应的片元的颜色) 与目标颜色 (已经存在于颜色缓存中的颜色，通常是当前背景中的颜色) 按照给定的公式进行混合，生成最终的新颜色；
  • 概念：混合操作
    • 概念：
      • 当前的片段着色器计算出来的颜色，称之为源颜色，然后再经过了 Blend 之后的颜色、称之为目标颜色，在写入帧缓冲区时会经过 Blend 操作；
      • 可以使用 SrcFactor BlendOP DstFactor 的公式、来实现源颜色的操作，得到看起来好像半透明的、实际上只是正常颜色的半透明颜色；
    • 图示：
  • 分类：Unity 中的混合模式
    • Unity 当中的默认混合模式为 One Zero，即源颜色的因子为 1，用源颜色的值覆盖帧缓冲区的颜色；
    • 默认情况下，Unity 中的混合是关闭的 (Blend Off)，这意味着不会进行任何混合操作，新绘制的像素会直接覆盖原有的像素；

混合因子与混合操作

概念：混合因子与混合操作

  • 概念：混合基础公式
    • 公式：
    • 参数：
      • SourceColor：当前 Shader 计算出的颜色（源颜色）
      • SrcFactor：源颜色的混合因子；
      • DestColor：帧缓冲区中已存在的颜色（目标颜色）
      • DstFactor：目标颜色的混合因子；
      • [BlendOp]：混合运算符（默认为 Add）
  • 概念：混合运算符
    • 混合运算符决定了如何组合源颜色和目标颜色；
      1. Add：
      • FinalColor = (Src * SrcFactor) + (Dst * DstFactor)
      • 最常见的混合方式，用于透明和加法混合；
      1. Max：
      • FinalColor = max(Src, Dst)
      • 取两个颜色的最大值；
      1. Sub：
      • FinalColor = (Src * SrcFactor) - (Dst * DstFactor)
      • 源颜色中减去目标颜色；
  • 概念：混合因子 - Blend Factors
      1. One：
      • 使用完整的 1.0 作为乘数，比如操作为 Add 时，FinalColor = Src + Dst；
      • 颜色叠加变亮，适合火焰、发光体等，简单相加会使颜色值可能超过 1.0 (被钳制)，产生高亮效果；
      1. Zero：
      • 使用 0.0 作为乘数，完全忽略源或目标颜色，用于屏蔽某些颜色分量；
      1. ScrColor：
      • 使用源颜色的 RGB 值作为乘数，源颜色会影响混合结果的比例，用于自定义的乘法混合效果；
      1. ScrAlpha：
      • 使用源颜色的 Alpha 值作为乘数，透明度决定混合强度，实现标准透明混合：Blend SrcAlpha OneMinusSrcAlpha；
  • 代码：设置自定义混合因子的混合效果
Shader "Unlit/Test"
{
	Properties
	{
		// 括号里面设置的就是当前显示的枚举内容，并且显示的是Unity中自己带的枚举
		[Enum(UnityEngine.Rendering.BlendMode)]_SrcBlend("Src Blend",int) = 0
		[Enum(UnityEngine.Rendering.BlendMode)]_DestBlend("Dest Blend",int) = 0
		_MainTex("Texture",2D) = "white"{}
	}
	
	SubShader
	{
		Tags{"Queue" = "Transparent"}
		
		// 用于设置自定义混合因子的混合效果
		Blend [_SrcBlend] [_DestBlend]
		
		// ... 
	}
}

总结：常见混合操作

总结：常用混合操作

  • 默认模式：Blend One Zero
    • 作用：
      • 默认不透明效果；
    • 图示：
  • 标准透明混合：Blend SrcAlpha OneMinusSrcAlpha：
    • 公式：
    • 作用：
      • 根据源颜色的透明度因子（SrcAlpha）进行线性插值混合；
      • 最终颜色 = 源颜色 × 源透明度 + 目标颜色 × (1 - 源透明度)；
      • 实现自然透明效果，源颜色根据其透明度与目标颜色混合，源透明度越高，源颜色占比越大；
    • 应用：
      • 玻璃、水面、UI 元素等半透明物体；
    • 图示：
  • 加法混合：Blend One One：
    • 公式：
    • 作用：
      • 将源颜色与目标颜色直接相加，产生增亮的效果；
      • 最终颜色 = 源颜色 × 1 + 目标颜色 × 1
    • 应用：
      • 火焰、魔法特效、发光粒子等需要表现光晕叠加的效果；
    • 图示：
  • 乘法混合：Blend DstColor Zero：
    • 公式：
    • 作用：
      • 源颜色与目标颜色相乘；
      • 最终颜色 = 源颜色 × 目标颜色 + 目标颜色 × 0
      • 结果颜色比原始颜色更暗，适用于阴影叠加或纹理混合（如磨损效果）
    • 应用：
      • 阴影投射、滤镜效果、颜色叠加；
    • 图示：
  • 预乘混合：`Blend One OneMinusSrcAlpha：
    • 公式：
    • 作用：
      • 处理已预乘 Alpha 的纹理，避免颜色渗色；
    • 应用：
      • 美术工具导出的带透明通道资源；
  • 自定义混合：Blend SrcAlpha OneMinusSrcAlpha, One Zero // RGB与Alpha通道独立混合
    • 自定义混合；
  • 变暗混合：BlendOp Min + Blend One One
    • 公式：
    • 作用：
      • 保留两个颜色中较暗的部分；
    • 应用：
      • 雾气效果、暗角处理；
  • 变亮混合：
    • 公式：
    • 作用：
      • 保留两个颜色中较亮的部分；
    • 应用：
      • 高光叠加、光晕效果；

面剔除 - Cull

概念：面剔除和面的可见性

  • 概念：
    • 指令：
      • Cull Off | Back | Front
    • 解释：
      • Off：无论正面、背面都进行渲染；
      • Back：只渲染正面，背面剔除；
      • Front：只渲染背面，正面剔除；
    • 注意：
      • Unity 默认物体的模式为 Cull Back；
  • 分析：如何区分正反面
    • 根据法线的方向；
    • 法线正方向的面为正面，法线负方向的面为背面；
  • 图示：
    • 假设一个立方体，无论什么角度、最多只能看到三个面，那剩下这些面的渲染方式就是由面剔除来指定；
  • 代码：
Shader "Unlit/Test"
{
	Properties
	{
		[Enum(UnityEngine.Rendering.BlendMode)]_SrcBlend("Src Blend",int) = 0
		[Enum(UnityEngine.Rendering.BlendMode)]_DestBlend("Dest Blend",int) = 0
		
		// Unity提供的枚举指令
		[Enum(UnityEngint.Rendering.CullMode)]_Cull("Cull",int) = 0
		_MainTex("Texture",2D) = "white"{}
	}
	
	SubShader
	{
		Tags{"Queue" = "Transparent"}
		
		// 用于设置自定义混合因子的混合效果
		Blend [_SrcBlend] [_DestBlend]
		
		// 只渲染正面
		Cull [_Cull]
	}
}

Intensity 光强度

代码

Shader "Unlit/Test"
{
	Properties
	{
		[Enum(UnityEngine.Rendering.BlendMode)]_SrcBlend("Src Blend",int) = 0
		[Enum(UnityEngine.Rendering.BlendMode)]_DestBlend("Dest Blend",int) = 0
		
		// Unity提供的枚举指令
		[Enum(UnityEngint.Rendering.CullMode)]_Cull("Cull",int) = 0
		_MainTex("Texture",2D) = "white"{}
		_Color("Color",Color) = {1,1,1,1}
		
		// 默认值如果是0，则没有颜色，因此默认值一般都是1
		_Intensity("Intensity",Range(-4,4)) = 1
	}
	
	SubShader
	{
		Tags{"Queue" = "Transparent"}
		Blend [_SrcBlend] [_DestBlend]
		Cull [_Cull]
		
		Pass
		{
			CGPROGRAM
			#pragma vertex vert
			#pragma fragment frag
			#include "UnityCG.cginc"
			
			struct appdata
			{
				float4 vertex : POSITION;
				float2 uv : TEXCOORD0; 
			};
			
			struct v2f
			{
				float2 uv : TEXCOORD0;
				float4 vertex : SV_POSITION;
			};
			
			sampler2D _MainTex;
			float4 _MainTex_ST;
			float4 _Color;
			half _Intensity;
			
			v2f vert(appdata v)
			{
				v2f o;
				o.vertex = UnityObjectToClipPos(v.vertex);
				o.uv = TRANSFORM_TEX(v.uv,_MainTex);
			}
			
			fixed4 frag(v2f i) : SV_POSITION
			{
				fixed4 c = tex2D(_MainTex,i.uv);
				
				// 直接使用加法时，当前色彩很容易过曝，一次一般是使用加法
				// c*=_Color + _Intensity;
				c *= _Color * _Intensity;
			}
			
			ENDPROGRAM
		}
	}
}

Shader 中的时间

概念：Shader 中的时间

  • 概念：_Time.xyzw
    • _Time 是 Unity 提供的一个内置变量，用于在 Shader 中获取时间信息；
    • _Time 是一个 float4 类型的向量，包含四个分量 (x,y,z,w)，每个分量都代表不同的时间计算方式，方便开发者根据不同的需求选择合适的时间变量；
  • 分类：
    • 概念：_Time.x (t/20)
      • 基础时间 (t) 除以 20；
      • 适用于非常缓慢的动画效果；
    • 概念：_Time.y (t)
      • 直接等于基础时间 (t);
      • 适用于大多数需要时间参数的效果；
    • 概念：_Time.z(t*2)
      • 基础时间 (t) 乘以 2，适用于需要快速变化的效果；
    • 概念：_Time.w(t*3)
      • 基础时间 (t) 乘以 3，最快的时间变化；
  • 分析：实现贴图的流动效果
    • 贴图的流动效果的实现，主要依赖于改变当前贴图 UV 的 Offset，然后使用不同的 Time 实现快或慢的流动效果；
  • 代码：
Shader "Unlit/Test"
{
	Properties
	{
		[Enum(UnityEngine.Rendering.BlendMode)]_SrcBlend("Src Blend",int) = 0
		[Enum(UnityEngine.Rendering.BlendMode)]_DestBlend("Dest Blend",int) = 0
		[Enum(UnityEngint.Rendering.CullMode)]_Cull("Cull",int) = 0
		_MainTex("Texture",2D) = "white"{}
		_Color("Color",Color) = {1,1,1,1}
		_Intensity("Intensity",Range(-4,4)) = 1
		_MainUVSpeedX("MainUVSpeed X",float) = 0
		_MainUVSpeedY("MainUVSpeed Y",float) = 0
	}
	
	SubShader
	{
		Tags{"Queue" = "Transparent"}
		Blend [_SrcBlend] [_DestBlend]
		Cull [_Cull]
		
		Pass
		{
			CGPROGRAM
			#pragma vertex vert
			#pragma fragment frag
			#include "UnityCG.cginc"
			
			struct appdata
			{
				float4 vertex : POSITION;
				float2 uv : TEXCOORD0; 
			};
			
			struct v2f
			{
				float2 uv : TEXCOORD0;
				float4 vertex : SV_POSITION;
			};
			
			sampler2D _MainTex;
			float4 _MainTex_ST;
			float4 _Color;
			half _Intensity;
			float4 _MainUVSpeedX,_MainUVSpeedY;
			
			v2f vert(appdata v)
			{
				v2f o;
				o.vertex = UnityObjectToClipPos(v.vertex);
				// 可以将其拆解出来：o.uv = v.uv*_MainTex_ST.xy + _MainTex_ST.zw; 
				// xy控制Tiling，zw控制Offset
				o.uv = TRANSFORM_TEX(v.uv,_MainTex) + float2(_MainUVSpeedX,_MainUVSpeedY) * _Time.y;
				return o;
			}
			
			fixed4 frag(v2f i) : SV_POSITION
			{
				fixed4 c = tex2D(_MainTex,i.uv);
				
				// 直接使用加法时，当前色彩很容易过曝，一次一般是使用加法
				// c*=_Color + _Intensity;
				c *= _Color * _Intensity;
				return c;
			}
			ENDPROGRAM
		}
	}
}

遮罩效果

基础概念

概念：遮罩效果

  • 概念：
    • 遮罩效果是一种通过特定规则控制图像部分区域显示或隐藏的技术；
    • 在 Unity 中，遮罩本质上是一个过滤系统，它决定哪些像素应该被渲染以及如何渲染；
    • 遮罩可以基于多种条件，如透明度、颜色值、深度信息或自定义的计算逻辑；
    • 具体而言，在图形渲染中，这转化为对片元 (Fragment)的筛选和处理过程；
  • 概念：Unity 中常见遮罩
    • Alpha 遮罩​：基于纹理的 Alpha 通道控制透明度，这是最基础的遮罩形式；
    • 裁剪遮罩 (Alpha Clipping)​：使用阈值硬性裁剪像素，适合创建锐利的边缘效果；
    • 范围遮罩​：根据数值范围 (如亮度、深度)决定像素可见性；
    • 反向遮罩：反转常规遮罩的逻辑，隐藏原本显示的区域；
    • 动态遮罩​：通过脚本实时控制遮罩参数，创建随时间变化的效果；

代码实现

代码

Shader "Unlit/Test"
{
	Properties
	{
		[Header(RenderingMode)]
		[Enum(UnityEngine.Rendering.BlendMode)]_SrcBlend("Src Blend",int) = 0
		[Enum(UnityEngine.Rendering.BlendMode)]_DestBlend("Dest Blend",int) = 0
		[Enum(UnityEngint.Rendering.CullMode)]_Cull("Cull",int) = 0
		
		[Header(Base)]
		_MainTex("Texture",2D) = "white"{}
		_Color("Color",Color) = {1,1,1,1}
		_Intensity("Intensity",Range(-4,4)) = 1
		_MainUVSpeedX("MainUVSpeed X",float) = 0
		_MainUVSpeedY("MainUVSpeed Y",float) = 0

		[Header(Mask)]
		_MaskTex("MaskTex",2D) = "white"{}
		_MaskUVSpeedX("MaskUVSpeed X",float) = 0
		_MaskUVSpeedY("MaskUVSpeed Y",float) = 0
	}
	
	SubShader
	{
		Tags{"Queue" = "Transparent"}
		Blend [_SrcBlend] [_DestBlend]
		Cull [_Cull]
		
		Pass
		{
			CGPROGRAM
			#pragma vertex vert
			#pragma fragment frag
			#include "UnityCG.cginc"
			
			struct appdata
			{
				float4 vertex : POSITION;
				float2 uv : TEXCOORD0; 
			};
			
			struct v2f
			{
				float4 uv : TEXCOORD0;
				float4 vertex : SV_POSITION;
			};
			
			sampler2D _MainTex; float4 _MainTex_ST;
			sampler2D _MaskTex; float4 _MaskTex_ST;
			float4 _MainTex_ST;
			float4 _Color;
			half _Intensity;
			float _MainUVSpeedX,_MainUVSpeedY;
			float _MaskUVSpeedX,_MaskUVSpeedY;
			
			v2f vert(appdata v)
			{
				v2f o;
				o.vertex = UnityObjectToClipPos(v.vertex);
				o.uv.xy = TRANSFORM_TEX(v.uv,_MainTex)+float2(_MainUVSpeedX,_MainUVSpeedY)*_Time.y;
				
				// 把模型原始的 UV 放到 zw 里面，给 Mask 贴图使用
				o.uv.zw = TRANSFORM_TEX(v.uv,_MaskTex)+float2(_MaskUVSpeedX,_MaskUVSpeedY)*_Time.y;
			}
			
			fixed4 frag(v2f i) : SV_POSITION
			{
				fixed4 c = tex2D(_MainTex,i.uv.xy);
				fixed4 maskTex = tex2D(_MaskTex,i.uv.zw);
				c *= _Color * _Intensity;
				
				// 将输出的颜色片元乘以maskTex，因为maskTex中黑色的区域其值为0，因此乘以0之后即为透明
				c *= maskTex;
				return c;
			}
			
			ENDPROGRAM
		}
	}
}
