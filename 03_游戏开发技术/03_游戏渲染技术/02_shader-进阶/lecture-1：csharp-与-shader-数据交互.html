Lecture 1：CSharp 与 Shader 数据交互

C# 与 Shader 数据交互

方式一：属性直接控制

概念：属性直接控制

  • 概念：属性直接控制
    • 这是最直接和常见的交互方式，通过 C# 脚本直接修改材质属性，从而影响 Shader 的渲染效果；
    • 具体来讲，在 Shader 的 Properties 块中定义的属性，可以通过 C# 脚本进行访问和修改；
  • 代码：Shader 中的属性
Properties
{
    _MainTex ("Texture", 2D) = "white" {}
    _Color ("Color", Color) = (1,1,1,1)
    _FloatValue ("Float Value", Float) = 1.0
    _VectorValue ("Vector Value", Vector) = (0,0,0,0)
}
  • 代码：C# 中对属性进行控制
using UnityEngine;

public class MaterialPropertyController : MonoBehaviour
{
	// 声明要被编辑的材质对象
    public Material targetMaterial;
    
    // 在Inspector中可以被运行时动态修改的属性
    public Color targetColor = Color.red;
    public float floatValue = 1.0f;
    public Vector4 vectorValue = Vector4.zero;

    void Update()
    {
        // 设置颜色属性
        targetMaterial.SetColor("_Color", targetColor);
        
        // 设置浮点数属性
        targetMaterial.SetFloat("_FloatValue", floatValue);
        
        // 设置向量属性
        targetMaterial.SetVector("_VectorValue", vectorValue);
    }
}
  • 补充：
    • 除了直接传入 Material 类型的对象外，经常会使用 Renderer 类来获取对象上的材质，进而进行材质属性的修改，其用法和上述代码基本一致；
总结：属性直接控制
  • 这种方法适用于需要对单个材质实例进行独立控制的场景，Material targetMaterial 代表了一个具体的材质实例，而 C# 中的脚本则对此实例进行修改；
注意：属性直接控制的缺点
  • 缺点一：材质实例化​
    • 每次通过 renderer.material 获取材质属性并进行修改时，Unity 会为该渲染对象创建一个新的材质实例（Material Instance）​，这意味着即使多个物体原本共享同一个材质以进行合批（Batching）优化，修改后每个物体都会拥有自己独立的材质实例，导致 Draw Call 数量增加，从而增加 CPU 的开销，在对象数量多或修改频繁时可能引起性能下降；
  • 缺点二：合批破坏​
    • 这一点实际上和缺点一是类似的问题。无论是动态合批（Dynamic Batching）、静态合批（Static Batching）以及 GPU 实例化（GPU Instancing），这几种合批都要求渲染对象使用完全相同的材质；
    • 通过属性直接控制的方式修改材质属性，会使材质实例变得独一无二，从而破坏合批条件，导致渲染效率降低；
  • 缺点三：内存占用增加
    • 每个新创建的材质实例都会占用额外的内存；
    • 如果场景中有大量对象都需要独立的材质属性，这些材质实例累积起来会消耗可观的内存；

方式二：材质属性块

概念：材质属性块 - MaterialPropertyBlock

  • 概念：
    • MaterialPropertyBlock 用于在运行时动态修改渲染器的材质属性，而无需创建新的材质实例；
    • MaterialPropertyBlock 的本质是一个轻量级的属性覆盖容器。它允许你为使用同一材质资产的不同渲染器设置不同的属性值​（如颜色、纹理、浮点数等），而不会破坏材质的共享状态；
    • 由于所有对象仍然引用同一个底层材质资产，Unity 的动态批处理和 GPU 实例化等优化机制更有可能生效，从而提升渲染性能；
  • 原理：
    • Unity 的渲染系统在绘制每个对象时，会检查其渲染器是否附加了 MaterialPropertyBlock；
    • 如果有，则使用 MaterialPropertyBlock 中存储的值，来临时覆盖底层共享材质中的对应属性值，这个覆盖仅对该次渲染调用有效，不会修改原始的材质资产；
    • 在底层，通过 MaterialPropertyBlock 设置的属性值最终会被传递到 GPU 的常量缓冲区​ (Constant Buffer) 或统一缓冲区​ (Uniform Buffer) 中，Shader 在运行时从这些缓冲区读取最新的值，这就实现了对共享材质属性的个性化覆盖；

·using UnityEngine;

namespace Test
{
    public class MaterialPropertyTest : MonoBehaviour
    {
        public Color targetColor = Color.red;
        public float smoothness = 0.5f;
        public Texture customTexture;

        // 1， 声明当前物体的Renderer对象
        private Renderer render;
        
        // 2. 声明当前要修改的MaterialPropertyBlock
        private MaterialPropertyBlock propertyBlock;

        private void Start()
        {
            // 3. 分别获得渲染组件以及创建MaterialPropertyBlock实例
            render = GetComponent<Renderer>();
            propertyBlock = new MaterialPropertyBlock();
            
            // 4. 获取渲染器当前的属性块，以保留之前设置过的属性
            // 此代码将 render 对象上的 MaterialPropertyBlock 赋值到 propertyBlock 变量上
            render.GetPropertyBlock(propertyBlock);
            
            // 5. 设置属性
            if (propertyBlock!=null)
            {
                propertyBlock.SetColor("_Color", targetColor);
                propertyBlock.SetFloat("_Smoothness", smoothness);
                
                if (customTexture!=null)
                {
                    propertyBlock.SetTexture("_MainTex", customTexture);
                }
            }

            // 6. 将属性块应用至渲染器
            render.SetPropertyBlock(propertyBlock);
        }

    }
}

总结：关于材质属性块
    1. 对大量需要修改属性的对象，尽量在循环中复用同一个 MaterialPropertyBlock 实例，避免在循环内部创建新实例；
    1. 虽然 MaterialPropertyBlock 本身有助于合批，但需注意过多不同的属性组合仍可能阻止合批。例如，1000 个对象共享材质但通过 MaterialPropertyBlock 设置了 1000 种不同的颜色，Unity 可能无法将它们全部合批。理想情况是让大多数对象共享相同的属性配置；
    1. MaterialPropertyBlock 中设置的属性是临时的，仅存在于运行时。​不会被保存到场景或材质资产中；

方式三：全局属性控制

概念：全局属性控制

  • 概念：
    • 全局属性（Global Properties）是在 Shader 中定义，但值由 C# 脚本统一设置，并对所有使用该属性的 Shader 立即生效的属性；
    • 与每个材质独有的属性不同，全局属性更像一个全局变量，修改一处，处处更新；
  • 方法：Shader.SetGlobalXXX
    • 可以使用 Shader.SetGlobalXXX 系列方法来设置不同类型的全局属性；
    • 除了使用方法来设置，还可以直接在 Shader 文件中声明全局变量，其无需在 Properties 块中声明，直接在 CG/HLSL 代码中声明即可；

C# 与 Shader 变体控制

Shader 本地关键字

概念：Shader 本地关键字

  • 区别：本地关键字和全局关键字的区别
  • 概念：
    • 在 C# 脚本中，通过 Material 类的方法来操控特定材质球上的本地关键字状态；
  • 使用：
    • （1）启用与禁用：
      • 使用 Material.EnableKeyword 和 Material.DisableKeyword；
      • 更高效的做法是使用 Material.SetKeyword(LocalKeyword, bool) 方法，它根据传入的布尔值自动切换状态；
    • （2）访问状态：
      • 通过 Material.IsKeywordEnabled 方法查询某个关键字在当前材质上是否已启用；
    • （3）关键字缓存：
      • 为了获得最佳性能，特别是在需要频繁设置关键字时，建议预先创建并缓存 LocalKeyword 结构体，而不是在每次调用时传递关键字名称字符串；
      • LocalKeyword 结构体与特定的 Shader 实例相关联，不能在不同的 Shader 之间混用，即使它们有关键字同名；
  • 代码：
using UnityEngine;
using UnityEngine.Rendering;

public class MaterialKeywordExample : MonoBehaviour 
{
    public Material material;
    private LocalKeyword exampleFeatureKeyword; // 缓存LocalKeyword

    void Start() 
    {
        // 获取材质使用的Shader，并创建对应的LocalKeyword
        Shader shader = material.shader;
        exampleFeatureKeyword = new LocalKeyword(shader, "EXAMPLE_FEATURE_ON");
    }

    public void ToggleEffect() 
    {
        // 使用缓存的LocalKeyword进行高效操作
        bool currentState = material.IsKeywordEnabled(exampleFeatureKeyword);
        material.SetKeyword(exampleFeatureKeyword, !currentState);
    }
    
    private void Update()
    {
	    if(Input.GetKeyDown(Space))
	    {
		    ToggleEffect();
	    }
    }
}

Shader 关键字是否可以被使用

分析：Shader 关键字是否可以被使用

  • 分析：
    • 一个本地关键字能否被全局关键字重写，取决于它在 Shader 源文件中的声明方式；
      1. 使用 shader_feature_local 或 multi_compile_local 声明的关键字具有局部作用域，不能被全局关键字重写；
      1. 使用 shader_feature 或 multi_compile 声明的关键字具有全局作用域，可以被同名的全局关键字重写；
  • 注意：
    • 如果一个材质实例已经通过 C# 代码（如 material.EnableKeyword）启用了一个全局作用域的本地关键字，那么这个关键字在该特定材质上的状态将固定，不再受同名全局关键字状态变化的影响；
    • 这类似于修改了预制体实例的某个属性后，该属性不再受原始预制体控制；
  • 总结：
      1. 对于仅限单个 Shader 使用的效果，优先使用 shader_feature_local 和 multi_compile_local 来声明关键字。这有助于节省宝贵的全局关键字名额（上限 256 个），并减少意外的全局干扰；
      1. 缓存 LocalKeyword 结构体以避免重复的查找和字符串比较，从而提升性能；
      1. 警惕运行时切换变体的开销：启用或禁用使用着色器变体的关键字可能导致 Unity 切换到不同的着色器变体。如果这个变体是第一次被使用，可能会引起卡顿，因为图形驱动需要时间准备；

C# 控制 Shader 变体开关

概念：C# 控制 Shader 变体开关

  • 概念：
    • Shader 变体是通过 ​预处理指令​定义的，它们会在编译时生成多个不同版本的 Shader 代码，EnableKeyword 的作用就是在运行时激活或禁用与特定关键字关联的 Shader 变体，从而改变渲染行为；
    • 比如：#pragma multi_compile _EMISSION_ON _EMISSION_OFF
    • 此代码定义了两个变体，可以对其进行控制：
    • material.EnableKeyword("_EMISSION_ON"); // 激活发射效果
    • material.DisableKeyword("_EMISSION_OFF"); // 禁用无发射效果的变体
  • 代码：
IEnumerator WaitDead(float time)
{
	float _time = 0;
	while(true)
	{
		_time += Time.deltaTime;
		yield return new WaitForEndOfFFrame();

			yield return;
		}

			mat.EnableKeyword("_DISSOLVEENABLED_ON");

		}
	}
