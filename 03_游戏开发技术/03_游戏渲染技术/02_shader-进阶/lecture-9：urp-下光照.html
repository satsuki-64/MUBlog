URP 光照参数

概念：URP 光照参数

  • 概念：主光源
    • Per Pixel ​：
      • 这是主光源的渲染方式；
      • Per Pixel 意味着光照计算会在屏幕上的每个像素进行，能产生最精确、质量最高的光照和阴影效果；
      • 这是最常用的设置。另一种选项是“Per Vertex”（逐顶点），计算在模型的顶点上进行，效果较差但性能开销低，现在已较少使用；
    • Cast Shadows（投射阴影）​​：
      • 勾选后，场景中的主方向光将能够投射动态阴影；
    • Shadow Resolution（阴影分辨率）​​：
      • 这个值决定了主光源阴影地图的大小（2048x2048），分辨率越高，阴影的边缘越清晰、细节越多，但消耗的显卡内存和计算资源也越多；
  • 概念：光照探针
    • Light Probe Groups（光照探针组）​​： 这个设置决定了如何为动态物体提供烘焙的间接光照，选择 ​Light Probe Groups ​ 允许你在场景中放置“光照探针”，动态物体会在运行时采样这些探针的位置的光照信息，从而融入烘焙好的光照环境，实现逼真的间接光照效果。如果选择“Off”，动态物体将无法接收到烘焙的间接光；
  • 概念：附加光源
    • 这部分控制除了主光源之外的其他实时光源（如点光源、聚光灯）；
    • Shadow Atlas Resolution（阴影图集分辨率）​​：
      • URP 会将所有附加光源的阴影打包到一张大的纹理图集上，这个分辨率就是这张大图集的大小；
      • 其中 2048 是一个较高的设置，适合中高端设备；
    • Shadow Resolution Tiers（阴影分辨率层级）​​：
      • 这里允许你为不同等级的图形质量设置不同的阴影分辨率；
  • 概念：Cookie 图集
    • Cookie 可以理解为光源的“遮罩贴图”，用于产生复杂的光影形状；
    • Cookie Atlas Resolution（Cookie 图集分辨率）​​： ​2048，所有附加光源使用的 Cookie 纹理会被打包到一张图集上，这个值设置了图集的大小。分辨率越高，Cookie 的细节越清晰；

// 顶点着色器主函数
// 输入：Attributes input 结构体，包含模型空间下的顶点位置、法线、纹理坐标等属性
// 输出：Varyings output 结构体，包含准备传递给片段着色器进行插值的数据
Varyings LitPassVertexSimple(Attributes input)
{
    // 初始化输出结构体，所有成员默认设为0
    Varyings output = (Varyings)0;

    // Instance ID：设置当前渲染实例的ID
    UNITY_SETUP_INSTANCE_ID(input);
    // 将实例ID从输入传递到输出，以便片段着色器使用
    UNITY_TRANSFER_INSTANCE_ID(input, output);
    // 初始化立体渲染（VR）相关数据
    UNITY_INITIALIZE_VERTEX_OUTPUT_STEREO(output);

    // 将顶点从模型空间(Object Space)变换到齐次裁剪空间(Homogeneous Clip Space)
    // 这个函数同时计算了世界空间、观察空间等多个空间下的位置
    // 所有光照计算的基础，因为光照都在世界空间进行
    VertexPositionInputs vertexInput = GetVertexPositionInputs(input.positionOS.xyz);
    
    // 法线和切线变换，将法线和切线从模型空间变换到世界空间
    // 其切线和副切线为 real3 类型
    VertexNormalInputs normalInput = GetVertexNormalInputs(input.normalOS, input.tangentOS);

    // 雾效因子计算：根据顶点在裁剪空间的深度(Z值)计算雾效因子
    // 如果雾效在片段着色器计算(_FOG_FRAGMENT)，则顶点着色器中设为 0
    #if defined(_FOG_FRAGMENT)
        half fogFactor = 0;
    #else
        half fogFactor = ComputeFogFactor(vertexInput.positionCS.z);
    #endif

    // 纹理采样
    output.uv = TRANSFORM_TEX(input.texcoord, _BaseMap);
    // 世界空间位置
    output.positionWS.xyz = vertexInput.positionWS;
    // 裁剪空间位置：最终顶点位置，用于光栅化
    output.positionCS = vertexInput.positionCS;

    // 如果启用了法线贴图，需要输出完整的切线空间基向量
    #ifdef _NORMALMAP
        // 获取世界空间视图方向（从顶点指向相机）
        // GetWorldSpaceViewDir 返回的是 _WorldSpaceCameraPos 参数，没有做其他处理
        half3 viewDirWS = GetWorldSpaceViewDir(vertexInput.positionWS);
        // 打包法线WS和视图方向X分量（为了节省插值寄存器）
        output.normalWS = half4(normalInput.normalWS, viewDirWS.x);
        // 打包切线WS和视图方向Y分量
        output.tangentWS = half4(normalInput.tangentWS, viewDirWS.y);
        // 打包副切线WS和视图方向Z分量
        output.bitangentWS = half4(normalInput.bitangentWS, viewDirWS.z);
    #else
        // 没有法线贴图时，只需归一化世界空间法线
        output.normalWS = NormalizeNormalPerVertex(normalInput.normalWS);
    #endif

    // 处理静态光照贴图UV，对应光照系统的烘焙光照部分
    OUTPUT_LIGHTMAP_UV(input.staticLightmapUV, unity_LightmapST, output.staticLightmapUV);
    
    // 如果启用了动态全局光照，处理动态光照贴图UV
    #ifdef DYNAMICLIGHTMAP_ON
        output.dynamicLightmapUV = input.dynamicLightmapUV.xy * unity_DynamicLightmapST.xy + unity_DynamicLightmapST.zw;
    #endif

    // 计算球谐函数光照数据，用于动态物体的间接光照
    // 对应 "Light Probe System" 设置为 "Light Probe Groups"，这使得动态物体也能接收到烘焙的间接光照信息
    OUTPUT_SH4(vertexInput.positionWS, output.normalWS.xyz, GetWorldSpaceNormalizeViewDir(vertexInput.positionWS), output.vertexSH, output.probeOcclusion);

    // 如果启用了逐顶点附加光照（Additional Lights的Per Pixel设为"逐顶点"的模式）
    #ifdef _ADDITIONAL_LIGHTS_VERTEX
        // 计算顶点光照：简单的光照模型，性能较好但质量较低
        half3 vertexLight = VertexLighting(vertexInput.positionWS, normalInput.normalWS);
        // 将雾效因子和顶点光照数据打包到一个四维向量中（优化插值寄存器使用）
        output.fogFactorAndVertexLight = half4(fogFactor, vertexLight);
    #else
        // 否则只输出雾效因子（对应你图片中Additional Lights设置为"Per Pixel"逐像素光照）
        output.fogFactor = fogFactor;
    #endif

    // 如果需要在顶点着色器中计算阴影坐标（用于某些阴影技术如阴影级联）
    // 对应你图片中 "Main Light" -> "Cast Shadows" 和 "Shadow Resolution" 设置
    // 也对应 "Additional Lights" -> "Cast Shadows" 和阴影质量设置
    #if defined(REQUIRES_VERTEX_SHADOW_COORD_INTERPOLATOR)
        output.shadowCoord = GetShadowCoord(vertexInput);
    #endif

    return output;
}

ForwarLit 的片段着色器实现 - 光照部分

分析：ForwarLit 的片段着色器实现

// 片段着色器主函数
// 输入：Varyings input - 由顶点着色器输出并经过插值后的数据
// 输出：outColor - 主要的颜色输出到渲染目标（SV_Target0）
// 可选输出：outRenderingLayers - 渲染层级信息，用于灯光层级（Light Layers）功能（对应图片中 _LIGHT_LAYERS 设置）
void LitPassFragmentSimple(
    Varyings input
    , out half4 outColor : SV_Target0
#ifdef _WRITE_RENDERING_LAYERS
    , out float4 outRenderingLayers : SV_Target1
#endif
)
{
    // -------------------------------------
    // Instance ID 和 Stereo 设置
    // 设置实例ID和立体渲染眼睛索引，确保多实例渲染和VR渲染的正确性
    UNITY_SETUP_INSTANCE_ID(input);
    UNITY_SETUP_STEREO_EYE_INDEX_POST_VERTEX(input);

    // -------------------------------------
    // 表面数据初始化
    // 核心步骤：从纹理（如_BaseMap, _SpecGlossMap）和材质属性中采样并初始化表面外观数据
    // 这包括反照率（Albedo/Alpha）、法线、光滑度、镜面反射、自发光等
    SurfaceData surfaceData;
    InitializeSimpleLitSurfaceData(input.uv, surfaceData);

    // -------------------------------------
    // LOD 淡入淡出处理
    // 在LOD（细节层次）切换时进行平滑过渡，避免突兀的模型切换
    #ifdef LOD_FADE_CROSSFADE
        LODFadeCrossFade(input.positionCS);
    #endif

    // -------------------------------------
    // 光照输入数据初始化，其中包括了：世界空间位置、法线、视图方向、阴影坐标、光照贴图/探针数据等
    // 此函数处理了您图片中几乎所有光照设置所需的数据
    InputData inputData;
    InitializeInputData(input, surfaceData.normalTS, inputData);
    
    // 设置调试纹理数据，用于在材质的调试模式下显示（如显示UV、法线等）
    SETUP_DEBUG_TEXTURE_DATA(inputData, UNDO_TRANSFORM_TEX(input.uv, _BaseMap));

    // -------------------------------------
    // 贴花（Decal）处理
    // 如果启用了DBuffer延迟贴花，将贴花效果应用到表面数据上
    // 这发生在光照计算之前，因此贴花会影响最终的光照结果
    #if defined(_DBUFFER)
        ApplyDecalToSurfaceData(input.positionCS, surfaceData, inputData);
    #endif

    // -------------------------------------
    // 烘焙全局光照数据初始化
    // 根据顶点着色器传递的数据（光照贴图UV、球谐函数SH），初始化烘焙的间接光照
    // 这直接对应您图片中 "Light Probe System" 设置为 "Light Probe Groups"
    InitializeBakedGIData(input, inputData);

    // -------------------------------------
    // 核心光照计算
    // 使用经典的 Blinn-Phong 光照模型计算最终的片段颜色
    // 1. 主光源（Main Light）：计算方向光，并应用您图片中设置的阴影（Cast Shadows, Shadow Resolution 2048）
    // 2. 附加光源（Additional Lights）：根据您图片中的设置（Per Pixel, Per Object Limit 4, Cast Shadows, Shadow Atlas Resolution 2048）计算每个像素受多个光源的影响
    // 3. 间接光：结合上面初始化的烘焙GI数据
    // 4. 光照模型：使用简化的 Blinn-Phong 模型计算高光反射和漫反射
    half4 color = UniversalFragmentBlinnPhong(inputData, surfaceData);

    // -------------------------------------
    // 雾效混合
    // 将计算出的颜色与场景雾效进行混合，距离越远受雾的影响越大
    color.rgb = MixFog(color.rgb, inputData.fogCoord);

    // -------------------------------------
    // Alpha 通道处理
    // 根据材质的表面类型（透明/不透明）进行最终的Alpha通道调整
    color.a = OutputAlpha(color.a, IsSurfaceTypeTransparent(_Surface));

    // 输出最终颜色到渲染目标
    outColor = color;

    // -------------------------------------
    // 渲染层级输出（可选）
    // 如果启用了渲染层级写入（对应图片中 _LIGHT_LAYERS 设置），则输出当前片段的渲染层级
    // 这允许灯光通过 Rendering Layers 属性选择性地影响特定物体，是高级灯光过滤技术
    #ifdef _WRITE_RENDERING_LAYERS
        uint renderingLayers = GetMeshRenderingLayer();
        outRenderingLayers = float4(EncodeMeshRenderingLayer(renderingLayers), 0, 0, 0);
    #endif
}

// 初始化简单光照表面数据的函数
// 此函数负责从各种纹理和材质属性中采样，并填充 SurfaceData 结构体
// 该结构体描述了物体表面的视觉属性（颜色、光滑度等），是光照计算的输入数据
// 输入：uv - 纹理坐标，用于采样各种纹理贴图
// 输出：outSurfaceData - 填充好的表面数据，将用于后续的光照计算
inline void InitializeSimpleLitSurfaceData(float2 uv, out SurfaceData outSurfaceData)
{
    // 初始化表面数据结构体，将所有成员设置为0或默认值
    outSurfaceData = (SurfaceData)0;

    // 这是表面最主要的基础颜色信息，从基础贴图(_BaseMap)采样颜色和 Alpha 值
    // 其效果等效于 SAMPLE_TEXTURE2D(_BaseMap, sampler_BaseMap, uv)
    half4 albedoAlpha = SampleAlbedoAlpha(uv, TEXTURE2D_ARGS(_BaseMap, sampler_BaseMap));
    // 计算最终的Alpha值：纹理Alpha × 材质颜色Alpha
    outSurfaceData.alpha = albedoAlpha.a * _BaseColor.a;
    
    // Alpha Test（镂空测试）：如果Alpha低于_Cutoff阈值，则丢弃该片段
    // 对应材质中的"Alpha Clipping"功能，常用于树叶、栅栏等效果
    outSurfaceData.alpha = AlphaDiscard(outSurfaceData.alpha, _Cutoff);

    // 计算基础反照率颜色：纹理 RGB × 材质颜色 RGB
    outSurfaceData.albedo = albedoAlpha.rgb * _BaseColor.rgb;
    
    // Alpha 调制：根据透明度对反照率进行调制，用于实现正确的透明效果
    // 对于透明表面，这会根据 Alpha 混合模式调整基础颜色
    outSurfaceData.albedo = AlphaModulate(outSurfaceData.albedo, outSurfaceData.alpha);

    // 采样高光光泽度贴图，获取高光颜色和光滑度数据
    // 如果未使用高光贴图，则返回_SpecColor和默认光滑度
    half4 specularSmoothness = SampleSpecularSmoothness(uv, outSurfaceData.alpha, _SpecColor, TEXTURE2D_ARGS(_SpecGlossMap, sampler_SpecGlossMap));
    
    // 金属度设为0，因为这是Simple Lit着色器，不使用基于物理的金属度工作流
    // Simple Lit使用传统的高光颜色工作流，而非金属/粗糙度工作流
    outSurfaceData.metallic = 0.0; // unused in simple lit
    
    // 设置高光反射颜色：从贴图或_SpecColor获取
    outSurfaceData.specular = specularSmoothness.rgb;
    // 光滑度用于控制高光反射的集中程度（镜面反射的清晰度）
    outSurfaceData.smoothness = specularSmoothness.a;

    // -------------------------------------
    // 法线贴图处理
    // 从法线贴图(_BumpMap)采样切线空间法线数据
    // 如果未使用法线贴图，则返回默认的平面法线(0,0,1)
    // 法线贴图用于增加表面细节，使平面看起来有凹凸感
    outSurfaceData.normalTS = SampleNormal(uv, TEXTURE2D_ARGS(_BumpMap, sampler_BumpMap));

    // 环境光遮蔽和自发光
    // 环境光遮蔽设为1.0（无遮蔽），Simple Lit通常不使用复杂的AO贴图
    outSurfaceData.occlusion = 1.0;
    
    // 采样自发光贴图，并与自发光颜色(_EmissionColor)相乘
    // 自发光部分不受光照影响，始终以全亮度显示，用于发光材质
    outSurfaceData.emission = SampleEmission(uv, _EmissionColor.rgb, TEXTURE2D_ARGS(_EmissionMap, sampler_EmissionMap));
}

InputData 初始化

概念：InputData 的初始化

  • 概念：
    • 此函数的核心作用是将顶点着色器传递的分散数据，整合为光照计算所需的统一格式；
    • 它处理了世界空间法线（考虑法线贴图）、视图方向、阴影坐标（根据阴影设置选择计算方式）、雾效因子和顶点光照数据；
    • 特别重要的是，它根据图片中的"Per Pixel"设置区分了逐顶点与逐像素光照路径，并为调试模式准备光照贴图、光照探针等数据，是连接几何数据与复杂光照计算的数据处理步骤；
  • 代码：InputData 的初始化
// 初始化光照输入数据函数
// 此函数将顶点着色器输出的数据组装成光照计算所需的统一格式
void InitializeInputData(Varyings input, half3 normalTS, out InputData inputData)
{
    // 初始化输出结构体
    inputData = (InputData)0;

    // 世界空间位置：光照计算的基础坐标
    inputData.positionWS = input.positionWS;
    
	// 调试模式下需要裁剪空间位置
	#if defined(DEBUG_DISPLAY)
	    inputData.positionCS = input.positionCS;
	#endif

    // 法线处理分支：根据是否使用法线贴图选择不同路径
    #ifdef _NORMALMAP
        // 从打包的数据中解包出世界空间视图方向（存储在normalWS.w, tangentWS.w, bitangentWS.w中）
        half3 viewDirWS = half3(input.normalWS.w, input.tangentWS.w, input.bitangentWS.w);
        // 构建切线空间到世界空间的变换矩阵（用于将法线贴图从切线空间变换到世界空间）
        inputData.tangentToWorld = half3x3(input.tangentWS.xyz, input.bitangentWS.xyz, input.normalWS.xyz);
        // 将切线空间法线变换到世界空间（应用法线贴图效果）
        inputData.normalWS = TransformTangentToWorld(normalTS, inputData.tangentToWorld);
    #else
        // 没有法线贴图时，直接使用插值后的世界空间视图方向
        half3 viewDirWS = GetWorldSpaceNormalizeViewDir(inputData.positionWS);
        // 使用插值后的世界空间法线
        inputData.normalWS = input.normalWS;
    #endif

    // 归一化处理：确保法线和视图方向为单位向量（插值可能破坏归一化）
    inputData.normalWS = NormalizeNormalPerPixel(inputData.normalWS);
    viewDirWS = SafeNormalize(viewDirWS);

    // 设置最终的世界空间视图方向
    inputData.viewDirectionWS = viewDirWS;

    // 阴影坐标计算（对应图片中Main Light和Additional Lights的Cast Shadows设置）
    #if defined(REQUIRES_VERTEX_SHADOW_COORD_INTERPOLATOR)
        // 情况1：使用在顶点着色器中预计算好的阴影坐标
        inputData.shadowCoord = input.shadowCoord;
    #elif defined(MAIN_LIGHT_CALCULATE_SHADOWS)
        // 情况2：在片段着色器中将世界坐标变换到阴影贴图空间
        // 这里会使用图片中设置的Shadow Resolution 2048等高精度阴影贴图
        inputData.shadowCoord = TransformWorldToShadowCoord(inputData.positionWS);
    #else
        // 情况3：无阴影（对应图片中Cast Shadows未勾选的情况）
        inputData.shadowCoord = float4(0, 0, 0, 0);
    #endif

    // 雾效和顶点光照数据处理
    #ifdef _ADDITIONAL_LIGHTS_VERTEX
        // 如果使用逐顶点附加光照（对应图片中Additional Lights的Per Pixel设为逐顶点模式）
        inputData.fogCoord = InitializeInputDataFog(float4(inputData.positionWS, 1.0), input.fogFactorAndVertexLight.x);
        // 提取逐顶点计算的光照数据（性能较好但质量较低）
        inputData.vertexLighting = input.fogFactorAndVertexLight.yzw;
    #else
        // 使用逐像素附加光照（对应图片中Per Pixel设置）时，顶点光照为0
        inputData.fogCoord = InitializeInputDataFog(float4(inputData.positionWS, 1.0), input.fogFactor);
        inputData.vertexLighting = half3(0, 0, 0);
    #endif

    // 标准化屏幕空间UV，用于屏幕空间效果如SSAO、屏幕空间阴影等
    inputData.normalizedScreenSpaceUV = GetNormalizedScreenSpaceUV(input.positionCS);

    // 调试显示相关的数据准备
    #if defined(DEBUG_DISPLAY)
    #if defined(DYNAMICLIGHTMAP_ON)
        inputData.dynamicLightmapUV = input.dynamicLightmapUV.xy; // 动态光照贴图UV
    #endif
    #if defined(LIGHTMAP_ON)
        inputData.staticLightmapUV = input.staticLightmapUV; // 静态光照贴图UV
    #else
        inputData.vertexSH = input.vertexSH; // 球谐函数数据（对应图片中Light Probe Groups设置）
    #endif
    #if defined(USE_APV_PROBE_OCCLUSION)

主光计算：基于 BlinnPhong 光照模型

UniversalFragmentBlinnPhong 函数

分析：UniversalFragmentBlinnPhong 函数

  • 分析：UniversalFragmentBlinnPhong 函数的特点
      1. 代码将阴影（ShadowMask）、环境光遮蔽（AO）、全局光照（GI）等数据预处理与核心光照计算分离。这种设计允许在不同渲染路径（如前向/延迟）中复用相同的数据准备逻辑，而只需调整光照计算部分；
      1. 通过 #if defined(_ADDITIONAL_LIGHTS) 等宏动态包含或排除代码块，避免编译未使用的功能（如无附加光源时直接跳过循环），减少着色器变体大小和 GPU 指令数；
      1. 与 Phong 模型（依赖反射向量 R 和视角向量 V 的点积）不同，Blinn-Phong 通过半角向量 H = normalize(L + V) 计算高光，仅需一次点积 dot(N, H)，避免昂贵的 reflect 计算；
      1. URP 在保留 PBR 管线的如阴影、AO、GI 的同时，使用 Blinn-Phong 作为高光模型。这种设计既支持物理可信的漫反射和环境光，又通过传统模型实现风格化高光；
  • 分析：多光源的处理
    • URP 使用单 Pass 处理所有光源，通过 GetAdditionalLight 循环叠加光照颜色。相比传统多 Pass 渲染，减少了 Draw Call 和 Overdraw；

/// URP 中基于 Blinn-Phong 光照模型的通用片段着色器函数。
/// 结合了物理光照（PBR）与传统 Blinn-Phong 高光，支持多光源、阴影、光照层和调试显示。
/// 注：Blinn-Phong 模型通过半角向量计算高光，性能优于 Phong 模型，高光过渡更柔和
/// </summary>
/// <param name="inputData">包含世界空间法线、位置、UV 等输入数据</param>
/// <param name="surfaceData">包含表面材质属性（漫反射颜色、高光、光滑度等）</param>

	    half4 debugColor;

	    {
	        return debugColor; // 直接返回调试颜色（如法线、UV 可视化）
	    }

    uint meshRenderingLayers = GetMeshRenderingLayer();

    half4 shadowMask = CalculateShadowMask(inputData);

    if (IsMatchingLightLayer(mainLight.layerMask, meshRenderingLayers))
    #endif
    {
        lightingData.mainLightColor += CalculateBlinnPhong(mainLight, inputData, surfaceData);
    }

	    uint pixelLightCount = GetAdditionalLightsCount();

	    #if USE_FORWARD_PLUS

	    #ifdef _LIGHT_LAYERS
	        if (IsMatchingLightLayer(light.layerMask, meshRenderingLayers))
	    #endif
	        {
	            lightingData.additionalLightsColor += CalculateBlinnPhong(light, inputData, surfaceData);
	        }
	    }
	    #endif

	    #ifdef _LIGHT_LAYERS

	    #endif

	        }

    #endif

    #if defined(_ADDITIONAL_LIGHTS_VERTEX)

    #endif

    return CalculateFinalColor(lightingData, surfaceData.alpha);

/// <summary>
/// 【核心算法】从烘焙光照图（Lightmap）数据中减去主直接光的贡献。
/// 这是实现 Subtractive 混合光照模式的关键步骤，用于解决“双倍光照”问题。
/// </summary>
/// <param name="mainLight">主光源数据（方向、颜色、阴影衰减等）</param>
/// <param name="normalWS">世界空间法线向量</param>
/// <param name="bakedGI">原始的烘焙GI颜色值</param>
/// <returns>经过减法处理后的新烘焙GI颜色值</returns>
half3 SubtractDirectMainLightFromLightmap(Light mainLight, half3 normalWS, half3 bakedGI)
{
    // 核心目标：在一个已经包含了烘焙主太阳光光照和阴影的表面上，让实时阴影也能正确工作。
    // 算法三步走：
    // 1. 通过减去被实时阴影遮挡区域的光照贡献，计算出阴影区域可能的光照值。
    //    a) 保留其他烘焙光和光线反弹。
    //    b) 消除背对光源的几何体上的阴影。
    // 2. 与用户定义的阴影颜色（_SubtractiveShadowColor）进行钳位，防止过暗。
    // 3. 选择原始光照贴图值和计算值中较暗的一个作为最终结果。

    // --- 步骤 1: 估算光照贡献并执行减法 ---
    // 获取主光源的阴影强度，这可能小于1.0（用于软阴影过渡）。
    half shadowStrength = GetMainLightShadowStrength();
    // 计算兰伯特（Lambertian）余弦项，表示表面接收到的光照量。
    // 法线与光源方向点积，并限制在0~1。背对光源的面接收度为0。
    half contributionTerm = saturate(dot(mainLight.direction, normalWS));
    // 计算理论上的主光直接照射到该点的光照颜色和亮度。
    half3 lambert = mainLight.color * contributionTerm;

    // 关键理解：`mainLight.shadowAttenuation` 是实时阴影衰减（1.0表示无阴影，0.0表示全黑）。
    // 因此 `(1.0 - mainLight.shadowAttenuation)` 就是“阴影遮罩”，表示阴影覆盖的区域。
    // 这计算出的是“被实时阴影遮挡掉的那部分主光光照能量”。
    half3 estimatedLightContributionMaskedByInverseOfShadow = lambert * (1.0 - mainLight.shadowAttenuation);

    // 从原始的烘焙GI中，减去阴影区域不应有的那部分直接光能量。
    // 这样就得到了一个理论上的、已考虑实时阴影的GI值。
    half3 subtractedLightmap = bakedGI - estimatedLightContributionMaskedByInverseOfShadow;

    // --- 步骤 2: 应用用户控制的阴影颜色和强度 ---
    // 使用用户定义的 _SubtractiveShadowColor 设置一个阴影颜色的下限，防止阴影区域死黑。
    half3 realtimeShadow = max(subtractedLightmap, _SubtractiveShadowColor.xyz);
    // 根据阴影强度，在原始烘焙GI和计算出的实时阴影GI之间进行插值。
    // 例如，如果shadowStrength为0.5，则结果是半影过渡效果。
    realtimeShadow = lerp(bakedGI, realtimeShadow, shadowStrength);

    // --- 步骤 3: 选择最暗的结果，保证视觉正确性 ---
    // 最终，取原始烘焙GI和处理后的实时阴影GI中较暗的一个。
    // 这确保了混合结果不会比原始光照更亮，符合物理直觉。
    return min(bakedGI, realtimeShadow);
}

CalculateBlinnPhong 函数

分析：CalculateBlinnPhong 函数

  • 分析：CalculateBlinnPhong 函数特点
      1. 合并了距离衰减与阴影衰减​（实时阴影贴图采样），确保光照强度随距离减小并受阴影遮挡影响；
      1. 平滑度转换：通过 exp2(10 * smoothness + 1) 将艺术家友好的 [0,1] 平滑度参数转换为指数形式的高光衰减系数，扩大高光范围的艺术控制力；
  • 代码：
/// <summary>
/// 计算单个光源的 Blinn-Phong 光照贡献（漫反射 + 高光反射）
/// 注意：此函数仅计算单个光源，需在循环中调用以累积多光源贡献
/// </summary>
/// <param name="light">光源数据结构（颜色、方向、衰减等）</param>
/// <param name="inputData">输入数据（法线、视角方向等）</param>
/// <param name="surfaceData">表面材质数据（漫反射颜色、高光强度等）</param>
/// <returns>该光源对当前片元的最终光照颜色（half3）</returns>
half3 CalculateBlinnPhong(Light light, InputData inputData, SurfaceData surfaceData)
{
    // 计算衰减后的光源颜色
    // 光照衰减后颜色 = 光照颜色 * (距离衰减 * 阴影衰减)
    // distanceAttenuation: 基于平方反比定律的距离衰减（平行光=1.0，点光源=1/r²）
    // shadowAttenuation: 阴影遮挡衰减（1.0=无阴影，0.0=全阴影）
    half3 attenuatedLightColor = light.color * (light.distanceAttenuation * light.shadowAttenuation);

    // 计算漫反射颜色（Lambert模型）
    // 公式：diffuse = lightColor * max(0, dot(N, L)) 
    // 注：Lambert模型仅依赖法线与光源方向，与视角无关
    half3 lightDiffuseColor = LightingLambert(attenuatedLightColor, light.direction, inputData.normalWS);

    // 初始化高光颜色（默认无高光）
    half3 lightSpecularColor = half3(0,0,0);
    
    // 仅在启用了 高光贴图或高光颜色 时执行，其中 _SPECGLOSSMAP 表示高光贴图
    #if defined(_SPECGLOSSMAP) || defined(_SPECULAR_COLOR)

	    // 将平滑度（0~1）转换为高光指数（Shininess）
	    // 公式：smoothness = 2^(10 * inputSmoothness + 1)，扩大高光衰减范围
	    half smoothness = exp2(10 * surfaceData.smoothness + 1);
	
	    // 计算 Blinn-Phong 模型的高光反射颜色
	    // 核心公式：specular = lightColor * pow(max(0, dot(N, H)), smoothness) 
	    // 其中 H 为半角向量：normalize(L + V)，替代 Phong 模型中的反射向量 R，计算更高效
	    lightSpecularColor += LightingSpecular
	    (
		    attenuatedLightColor, 
		    light.direction, 
		    inputData.normalWS, 
		    inputData.viewDirectionWS, 
		    half4(surfaceData.specular, 1), 
		    smoothness
		);
    
    #endif

    // 根据 Alpha 混合模式合成最终颜色，其中 lightDiffuseColor 来自 _BaseMap 贴图
    #if _ALPHAPREMULTIPLY_ON
	    // Alpha预乘模式：漫反射颜色已预乘Alpha，需单独叠加高光（用于透明材质）
	    return lightDiffuseColor * surfaceData.albedo * surfaceData.alpha + lightSpecularColor;
    #else
	    // 标准模式：漫反射与表面颜色相乘后直接叠加高光
	    return lightDiffuseColor * surfaceData.albedo + lightSpecularColor;
    #endif
}

分析：计算高光反射 - LightingSpecular

  • 代码：
/// <summary>
/// 基于 Blinn-Phong 模型计算高光反射颜色
/// 核心思想：通过法线（Normal）与半角向量（Halfway Vector）的夹角来计算高光强度，比传统的 Phong 模型更高效。
/// </summary>
/// <param name="lightColor">光源颜色（RGB），决定高光的基础色调和亮度</param>
/// <param name="lightDir">从着色点指向光源的方向向量（单位化）</param>
/// <param name="normal">表面法线向量（单位化）</param>
/// <param name="viewDir">从着色点指向相机（视角）的方向向量（单位化）</param>
/// <param name="specular">材质的高光颜色（RGB）和强度（A），通常来自材质属性或贴图</param>
/// <param name="smoothness">光滑度参数，控制高光斑点的集中程度（值越大，高光越锐利）</param>
/// <returns>计算得到的高光反射颜色（RGB）</returns>
half3 LightingSpecular(half3 lightColor, half3 lightDir, half3 normal, half3 viewDir, half4 specular, half smoothness)
{
    // 计算半角向量：光源方向与视角方向的中间向量
    // Blinn-Phong 模型的关键优化，用半角向量 H 替代 Phong 模型中的反射向量 R，减少计算量
    // SafeNormalize 确保零向量的安全性，避免归一化出错
    float3 halfVec = SafeNormalize(float3(lightDir) + float3(viewDir));
    
    // 计算法线（N）与半角向量（H）的夹角的余弦值
    // saturate 将点积结果限制在 [0, 1] 范围内，因为负值意味着光源在表面背面，不产生高光
    half NdotH = half(saturate(dot(normal, halfVec)));
    
    // 使用幂函数（pow）计算高光衰减项，这是高光的核心数学表达
    // smoothness 参数在此通常被缩放（如 exp2(10 * smoothness + 1)），作为幂指数，控制高光衰减速度
    // 指数越大，高光越集中，形成更小更亮的耀斑；指数越小，高光越分散
    // 注释指出使用 float 精度防止 half 精度在低端GPU上可能出现的带状瑕疵（banding）
    half modifier = pow(float(NdotH), float(smoothness)); 
    
    // 将高光衰减项与材质的高光颜色（specular.rgb）相乘，得到初步的高光反射值
    // 为避免某些移动平台编译器错误，此处显式使用 float3 类型
    float3 specularReflection = specular.rgb * modifier;

    // 将初步高光反射与光源颜色结合，返回最终的高光颜色
    // 光源颜色决定了高光的整体色调和亮度（例如，白光产生白色高光，红光产生红色高光）
    return lightColor * specularReflection;
}

补充：SafeNormalize

  • 分析：
    • 先对其使用 max 保证点击不为 0，因为取倒数的时候、分母为 0 很危险，因此对其使用一个 max 保证安全；
  • 代码：
// Normalize that account for vectors with zero length  
float3 SafeNormalize(float3 inVec)  
{  
    float dp3 = max(FLT_MIN, dot(inVec, inVec));  
    return inVec * rsqrt(dp3);  
}

前向渲染路径实现

分析：前向渲染路径实现

  • 概念：
    • 前向渲染是 Unity 内置渲染管线和 URP 早期版本的默认渲染路径，其主要分为两个部分；
      1. Base Pass​：
      • 首先执行一次 LightMode 为"ForwardBase"的 Pass。这个 Pass 会计算最重要的一个逐像素平行光​（通常是主方向光）、所有逐顶点光照和球谐光照（SH）​，以及环境光、自发光和光照贴图；
      1. Additional Passes ​：
      • 然后，对于每一个额外的需要逐像素处理的光源，都会再执行一次 LightMode 为"ForwardAdd"的 Pass；
      • 为了防止覆盖 Base Pass 的结果，这个 Pass 通常需要设置混合模式为 Blend One One（叠加混合） ，默认情况下，这些附加光源不产生实时阴影，除非在 Shader 中使用 #pragma multi_compile_fwdadd_fullshadows 编译指令来显式开启；
  • 代码：

#endif

		// 以便将多个Pass的光照结果叠加到帧缓冲区。

前向+渲染路径实现

分析：前向+渲染路径实现

  • 概念：
    • 传统前向渲染在处理多个光源时，需要为每个受影响的物体绘制多次（每个逐像素光源一次），导致 Draw Call 数量激增（），成为性能主要瓶颈；
    • 前向+路径通过将光照计算延迟到屏幕空间，并将光源分配工作交给高度并行的计算着色器，使得绘制调用次数与物体数量 () 基本无关，从而高效处理大量动态光源；
    • 所有可见光源的数据（如位置、颜色、衰减参数）被预先收集到结构化缓冲区中。在着色器中，通过光源索引、直接从这些缓冲区中读取数据，这是在单 Pass 内实现多光源计算的基础；
  • 代码：

计算逐像素光效果

概念：计算逐像素光效果 - GetAdditionalPerObjectLight

  • 概念：
    • 根据 GetAdditionalLight 函数返回的索引值，计算对应的光的逐像素光照效果；
    • 其中 USE_STRUCTURED_BUFFER_FOR_LIGHT_DATA 可以理解为 PC 平台下执行的，else 当中的可以理解为非 PC 比如移动平台；
  • 代码：
// 根据 per-object 光源索引获取特定附加光源的完整数据
// 参数 perObjectLightIndex: 光源在数据缓冲区中的索引
// 参数 positionWS: 当前像素的世界空间位置，用于计算光照方向和各种衰减
Light GetAdditionalPerObjectLight(int perObjectLightIndex, float3 positionWS)
{
    // 根据不同平台和配置使用不同的光源数据存储方式
    #if USE_STRUCTURED_BUFFER_FOR_LIGHT_DATA
        // 使用结构化缓冲区（Structured Buffer）存储光源数据（通常用于现代API如D3D11+、Vulkan等）
        float4 lightPositionWS = _AdditionalLightsBuffer[perObjectLightIndex].position;        // 光源位置（.w为0表示平行光，1表示点光源/聚光灯）
        half3 color = _AdditionalLightsBuffer[perObjectLightIndex].color.rgb;                  // 光源颜色和强度
        half4 distanceAndSpotAttenuation = _AdditionalLightsBuffer[perObjectLightIndex].attenuation; // 距离和聚光灯衰减参数
        half4 spotDirection = _AdditionalLightsBuffer[perObjectLightIndex].spotDirection;      // 聚光灯方向（世界空间）
        uint lightLayerMask = _AdditionalLightsBuffer[perObjectLightIndex].layerMask;          // 光源层级掩码，用于光源层级过滤
    #else
        // 使用传统分离的常量数组存储光源数据（兼容旧平台或GLES）
        float4 lightPositionWS = _AdditionalLightsPosition[perObjectLightIndex];               // 光源位置
        half3 color = _AdditionalLightsColor[perObjectLightIndex].rgb;                         // 光源颜色
        half4 distanceAndSpotAttenuation = _AdditionalLightsAttenuation[perObjectLightIndex];  // 衰减参数
        half4 spotDirection = _AdditionalLightsSpotDir[perObjectLightIndex];                   // 聚光灯方向
        uint lightLayerMask = asuint(_AdditionalLightsLayerMasks[perObjectLightIndex]);        // 层级掩码（转换为uint类型）
    #endif

    // 计算从表面点到光源的方向向量（统一处理平行光和点光源/聚光灯的巧妙设计）
    // 平行光：lightPositionWS.w = 0.0，此时lightVector = lightPositionWS.xyz - 0 = 光源方向
    // 点光源/聚光灯：lightPositionWS.w = 1.0，此时lightVector = 光源位置 - 表面位置（标准向量计算）
    float3 lightVector = lightPositionWS.xyz - positionWS * lightPositionWS.w;
    
    // 计算到光源距离的平方（避免开方操作优化性能），使用HALF_MIN防止除零
    float distanceSqr = max(dot(lightVector, lightVector), HALF_MIN);

    // 标准化光照方向：先计算平方根的倒数（rsqrt）再进行乘法，比先开方再除法更高效
    half3 lightDirection = half3(lightVector * rsqrt(distanceSqr));
    
    // 计算完整的光源衰减：结合距离衰减和角度衰减（聚光灯）
    // DistanceAttenuation: 计算基于距离的衰减（使用distanceSqr和衰减参数xy）
    // AngleAttenuation: 计算聚光灯的角度衰减（使用聚光灯方向、光照方向和衰减参数zw）
    // 某些平台需要完整的float精度以保证计算准确性
    float attenuation = DistanceAttenuation(distanceSqr, distanceAndSpotAttenuation.xy) * 
                       AngleAttenuation(spotDirection.xyz, lightDirection, distanceAndSpotAttenuation.zw);

    // 构建并返回Light结构体
    Light light;
    light.direction = lightDirection;                   // 标准化后的光照方向
    light.distanceAttenuation = attenuation;            // 计算出的综合衰减值
    light.shadowAttenuation = 1.0;                      // 阴影衰减初始值（可在后续的GetAdditionalLight重载中被阴影数据覆盖）
    light.color = color;                                // 光源颜色（包含强度）
    light.layerMask = lightLayerMask;                   // 光源层级掩码，用于光源与物体的层级匹配

    return light;
}

计算灯光衰减 - 距离衰减

概念：计算灯光衰减 - 距离衰减

  • 代码：灯光衰减
float attenuation = DistanceAttenuation(distanceSqr, distanceAndSpotAttenuation.xy) 
					* AngleAttenuation(spotDirection.xyz, lightDirection, distanceAndSpotAttenuation.zw);
  • 方法：DistanceAttenuation 的作用
    • 这个函数的核心目标是在保持物理正确性的同时，提供艺术友好的控制。纯粹的物理衰减（1/r²）在距离很近时强度会趋于无穷大导致过曝，而在距离很远时又不会完全归零。URP 通过平滑范围衰减来解决这两个问题；
    • 点光、聚光灯、平行灯都具有距离衰减；
  • 概念：DistanceAttenuation 中灯光衰减的实现
    • （1）计算距离代表灯光距离平方的 distanceSqr
      • 其中的 distanceSqr 的计算如下：
        • float distanceSqr = max(dot(lightVector, lightVector), HALF_MIN);
      • 其效果为灯光方向向量的点积，效果是离灯光越近、颜色越黑，离灯光越远，颜色越白；
      • 注意这个衰减目前是一个固定的衰减，和 Light 上的 Range 属性无关，只和距离灯光距离有关；
    • （2）获取灯光的衰减参数
      • 其中另一个参数，表示预计算的衰减参数，其计算来自于：
        • half4 distanceAndSpotAttenuation = _AdditionalLightsAttenuation[perObjectLightIndex];
      • 使用灯光的系数，读取出其衰减参数；
    • （3）加上平滑过渡项：
      • smoothFactor 表示一个在光源范围内为 1，在范围边界平滑过渡到 0 的因子，通过
        • factor = distanceSqr * distanceAttenuation.x
      • 其计算相对距离，再使用 saturate(1.0 - factor²)² 创建平滑的过渡曲线；
    • （4）计算最终衰减系数
      • factor 它本身不是一个衰减值，而是一个中间量，用于在下一步生成平滑的衰减曲线，紧接着的代码会利用这个 factor 来计算平滑衰减系数：
        • half smoothFactor = saturate (half (1.0) - factor * factor);
        • smoothFactor = smoothFactor * smoothFactor;
      • 其中的 factor * factor 相当于对“相对距离比率”进行平方。这会使衰减曲线在靠近光源时变化更平缓，而在边界处变化更迅速，形成更自然的过渡；
        • 然后 1.0 - factor * factor 表示计算距离饱和值的“反向值”，离光源越远（factor 越大），这个值越小；
      • 最后 saturate(...)：将结果限制在 [0, 1] 范围内，确保数值稳定性；
      • smoothFactor * smoothFactor：对结果再次平方，使衰减曲线的边缘更加平滑，避免光照在边界处产生生硬的截断效果；

/// <summary>
/// 计算点光源或聚光灯基于距离的衰减值
/// 结合了物理正确的平方反比衰减和游戏用的平滑范围衰减
/// </summary>
/// <param name="distanceSqr">从表面点到光源距离的平方，计算时光线向量的点积: dot(lightVector, lightVector)</param>
/// <param name="distanceAttenuation">预计算的衰减参数，用于平滑过渡到光源范围极限</param>
/// <returns>光照衰减乘数，范围[0, 1]</returns>
float DistanceAttenuation(float distanceSqr, half2 distanceAttenuation)
{
    // 我们为附加方向光和点光源使用共享的距离衰减计算
    // 对于方向光，衰减值将为1（因为方向光无距离衰减）
    
    // 1. 基础物理衰减：平方反比定律 (Intensity ∝ 1/r²)
    // 根据物理光学，光强与距离平方成反比
    // rcp(distanceSqr) 等价于 1.0 / distanceSqr，但计算效率更高
    float lightAtten = rcp(distanceSqr);
    
    // 将 half2 精度参数转换为 float2 以保证计算精度
    float2 distanceAttenuationFloat = float2(distanceAttenuation);

    // 2. 平滑过渡因子计算 (用于在光源范围边界平滑衰减至 0)
    // 使用Unity光照贴图器相同的平滑算法，保证高质量视觉效果
    // factor 计算当前距离相对于衰减范围的比率
    half factor = half(distanceSqr * distanceAttenuationFloat.x);
    
    // 平滑因子计算：当factor接近1时(到达光源范围)，smoothFactor平滑过渡到0
    // saturate将值限制在[0,1]范围内，然后通过两次平方创造平滑的S形曲线
    half smoothFactor = saturate(half(1.0) - factor * factor);
    smoothFactor = smoothFactor * smoothFactor;

    // 3. 结合物理衰减与艺术控制的范围衰减
    // 在光源范围内：返回物理正确的平方反比衰减
    // 在范围边界处：平滑过渡到0，避免光照突然消失的突兀感
    return lightAtten * smoothFactor;
}

计算灯光衰减 - 角度衰减

概念：计算灯光衰减 - 角度衰减

  • 概念：
    • 角度衰减是聚光灯独有的光照特性，用于模拟光线被限制在特定圆锥范围内的效果。它与距离衰减共同构成聚光灯的完整衰减；
    • URP 中的聚光灯使用内外双锥角机制来控制光照范围，这比单一角度的光锥产生更柔和、更真实的边缘过渡；
      • 内锥角 (innerSpotAngle)：光锥内部完全照亮区域的角度，在此区域内，光照强度保持最大值（无角度衰减）
      • 外锥角 (outerSpotAngle)：光锥的最大边界角度，超出此角度的区域完全不受该聚光灯影响；
  • 图示：内锥角和外锥角

  • 分析：half SdotL = dot(spotDirection, lightDirection)
    • 点积运算 dot(spotDirection, lightDirection) 在数学上等于两个向量夹角的余弦值（Cosθ）。在聚光灯的计算中，θ 角就是表面点相对于聚光灯的光锥中心轴的偏移角度；
      1. 当 SdotL = 1（即 Cosθ = 1, θ = 0°）：表面点恰好位于光锥的中心轴上，受到最强烈的光照；
      1. 当 SdotL = cos(outerAngle)：表面点恰好位于聚光灯定义的外锥角边界上。外锥角是聚光灯光照所能达到的最大角度；
      1. 当 SdotL = 0（即 Cosθ = 0, ``θ = 90°）：表面点位于光锥的 90 度边界上。在简单的衰减模型中，光照衰减为 0；
  • 代码：half atten = saturate(SdotL * spotAttenuation.x + spotAttenuation.y)
    • 将前面计算得到的点积结果（SdotL），通过一个线性变换，映射为聚光灯的角度衰减系数。这个变换使得衰减系数在聚光灯的内锥角范围内为 1（完全照亮），在外锥角处衰减为 0，并在内外锥角之间平滑过渡；
    • 其中：
        1. spotAttenuation.x：预计算的缩放因子，值为 1.0 / (cosInnerAngle - cosOuterAngle)。它将点积值缩放到合适的范围，以便进行内外锥角之间的插值；
        1. spotAttenuation.y：预计算的偏移因子，值为 -cosOuterAngle * spotAttenuation.x。它的主要作用是对变换后的点积值进行平移，确保当 SdotL 等于外锥角余弦值时，整个表达式的结果为 0；
    • 因此，其作用就是利用预计算好的参数，将方向一致性的点积结果（SdotL），快速转换成一个在聚光灯光锥范围内平滑变化的强度系数；
  • 代码：
// 函数：计算聚光灯的角度衰减
// 参数：
//   - spotDirection: 聚光灯照射方向的相反方向（单位向量），通常由引擎预计算[1](@ref)
//   - lightDirection: 从表面点指向聚光灯光源的方向（单位向量）
//     - spotAttenuation: 预计算的衰减系数，其中：
//        .x = 1.0 / (cosInnerAngle - cosOuterAngle) [即 invAngleRange]
//        .y = -cosOuterAngle / (cosInnerAngle - cosOuterAngle) [即 -cosOuterAngle * invAngleRange]
half AngleAttenuation(half3 spotDirection, half3 lightDirection, half2 spotAttenuation)
{
    // 角度衰减的线性衰减公式基础形式为：
    // attenuation = (SdotL - cosOuterAngle) / (cosInnerAngle - cosOuterAngle)
    // 此公式可重写为以下形式，更利于GPU并行计算（合并为一个乘加指令MAD）：
    // attenuation = SdotL * invAngleRange + (-cosOuterAngle * invAngleRange)
    // 即 = SdotL * spotAttenuation.x + spotAttenuation.y

    // 计算聚光灯反向方向与光线方向的点积（S·L）
    // 此值衡量表面点相对于聚光灯光锥中心的方向一致性：
    //   SdotL = 1  -> 表面点完全位于光轴中心线上（最亮）
    //   SdotL = 0  -> 表面点位于光锥的90度边界
    //   SdotL = cosOuterAngle -> 表面点位于外锥角边界
    half SdotL = dot(spotDirection, lightDirection);

    // 应用重写后的线性衰减公式，并使用saturate将结果限制在[0, 1]范围内
    // 当SdotL >= cosInnerAngle时，atten为1（处于内锥角完全照亮区）
    // 当SdotL介于cosInnerAngle和cosOuterAngle之间，atten线性衰减（在内外锥角间平滑过渡）
    // 当SdotL <= cosOuterAngle时，atten为0（处于外锥角之外无光照）
    half atten = saturate(SdotL * spotAttenuation.x + spotAttenuation.y);

    // 对线性衰减结果进行平方处理
    // 目的：增强角度衰减的效果，使光锥边缘的过渡更加平滑自然，避免生硬的截断感
    return atten * atten;
}
