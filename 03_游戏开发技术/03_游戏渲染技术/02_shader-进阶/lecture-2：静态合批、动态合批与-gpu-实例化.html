Lecture 2：静态合批、动态合批与 GPU 实例化

什么是批次

概念：什么是批次 - Batch

  • 概念：Batch
    • 把需要渲染的数据加载到显存并设置好渲染状态，然后 CPU 调用 GPU 渲染的过程为一个 Batch；
    • 因此一个 Batch 其分为两个部分，第一个部分是将数据提交到显存，然后设置数据的渲染状态。然后第二部分，CPU 向 GPU 发送一个 DrawCall，进而操纵 GPU 进行计算。Batch 数量越少，通常代表优化得越好；
    • Unity 引擎在准备渲染一帧时，会检查所有需要绘制的物体。它会将使用相同材质的物体归为一组，形成一个 Batch。理想情况下，一个 Batch 中的内容会被一次性提交给 GPU，只产生一次 Draw Call；
  • 概念：Setpass Call
    • SetPass Call 的本质是 CPU 命令 GPU 切换渲染状态，为接下来的绘制（Draw Call）做准备。因此每次 GPU 切换一个 Pass 前，都会产生一次 SetpassCall；
    • 每次状态切换都可能打断 GPU 当前的工作流水线。GPU 需要等待状态切换完成并同步数据，这会导致 GPU 暂时空闲，降低并行处理效率；
    • 并且切换状态常伴随着新的资源绑定，其中最耗时的操作之一就是着色器 Uniform /常量数据的传输，在现代渲染管线（如 URP 的 SRP Batcher）中，通过使用常量缓冲区（CBuffer/UBO）持久化保存材质属性，可以大幅减少这部分开销；
    • 一个 Shader 可以包含多个 Pass，而​每执行一个 Pass，通常就会产生一个 SetPassCall 和一个 Draw Call；​
  • 目标：
    • 最核心的目标，就是将 Batch 降低，进而可以减少 Setpass Call 和 DrawCall；
  • 分类：降低 Batch 的方法
      1. 动态合批：Dynamic Batching；
      1. 静态合批：Static Batching；
      1. GPU 实例化：GPU Instance；
分析：批次和 DrawCall 的关系
  • 理想情况下，一个 Batch 对应一次 Draw Call。这是因为：
    • 当一组物体共享相同的渲染状态​（如相同的材质、Shader、纹理等）时，CPU 可以将它们组织到同一个 Batch 中；
    • 处理这个 Batch 时，CPU 只需设置一次渲染状态，然后通过一次 Draw Call ​ 指令，就能将这批物体的所有几何数据（如顶点、索引）发送给 GPU 进行绘制；
  • 特殊情况下，一个 Batch 也可以对应多次 DrawCall
    • 如果一个 Shader 包含多个 Pass（例如，一个用于主体渲染，一个用于阴影投射），那么每个 Pass 通常都需要单独的 SetPass Call 和 Draw Call；
    • 因此，渲染一个使用多 Pass Shader 的物体，即使它在一个 Batch 内，也可能产生多次 Draw Call；

两类合批技术

动态合批

概念：动态合批

  • 概念：
    • 动态合批是 Unity 引擎中的一种优化技术，用于在运行时自动将多个小网格合并成更大的网格，以减少绘制调用，从而提高渲染性能；
    • 多个对象共享相同的材质时，Unity 会在 CPU 端将这些对象的顶点数据合并成一个批次，然后一次性发送到 GPU 进行渲染；
  • 概念：动态合批的规则
      1. 材质相同是合批的前提，但除了渲染阴影对象外，即使相同材质，但不同材质实例，则也不能合并；
      1. 动态合批支持不同网格的合批，即可以有很多不同的对象、使用同一个批次；
      1. 单个网格最多 300 个顶点，900 个顶点属性
      • 注意：顶点属性的上限可能未来版本会调整；
      • 如果 Shader 中用到了网格的 Position、normal 和 UV 的话，则最多是 300 个顶点；
      • 如果 Shader 中用到了网格的 Position、Normal、uv0、uv1 和 tangent 的话，则最多是 180 个顶点；
      1. 具有光照贴图的游戏对象如果有附加渲染器参数时，如果需要动态合批这些对象，他们必须指向相同的光照贴图位置 UV；
      • 因此大部分的光照贴图对象都不能动态合批；
      1. 有多 Shader Pass 的游戏对象无法做动态合批；
      1. 受多个光照影响的游戏对象，满足动态合批条件合批后，只会受一个光源的影响；
      1. 延迟渲染下不支持动态合批；
  • 图示：模型的 Mesh 上的详细信息
    • 当前的网格符合合批的要求，因为其顶点数符合要求，小于 300；
注意：Unity 中的动态合批
    1. Unity 中的动态合批的规则是默认的，不能更改规则，需要遵守 Unity 中的设定；
    1. 即使在 Unity 的 appdata 或者是 v2f 中定义了多个属性，如果其中有些属性没有在着色器中被使用，则其不会被计入到批次的顶点数当中；
    1. 对于镜像的情况（Scale 为负值）时，其会打断合批，比如图示内容；
    1. Lightmap 对象无法合批；

静态合批

概念：静态合批

  • 概念：
    • 静态合批​（Static Batching）是 Unity 的一种优化技术，通过在构建时或运行时将多个静态物体（即不移动的物体）的网格合并为一个大网格，从而减少 Draw Calls；
    • 它适用于共享相同材质的静态对象，如建筑、地形或装饰物；
    • 在预处理阶段（如场景加载时），Unity 收集所有标记为静态的物体，按材质分组，并将它们的网格数据转换到世界空间后，合并成一个或多个大网格。运行时，这些合并的网格作为一个整体渲染，显著降低 CPU 和 GPU 的通信开销；
    • 静态合批主要在构建时（Build Time）或场景加载时自动进行，但也可以在运行时通过代码手动触发，例如使用 StaticBatchingUtility.Combine 方法；
  • 方法：启用静态合批
    • 在 ​Project Settings > Player > Other Settings​ 中，确保勾选了 Static Batching​ 选项，通常默认开启；
    • 在场景中选中需要合批的物体，在 Inspector 面板中勾选 Static 或​ Batching Static​ 选项。勾选后，该物体及其子物体都会参与合批，但不能再运行时移动、旋转或缩放；
  • 概念：Unity 中静态合批的规则
      1. 要静态合批的物体，必须标记 Batching Static；
      1. 合批的物体需要使用同一个材质球（具体来讲，同一个材质实例），并且材质对应的 Shader 必须兼容合批
      • 例如，不能使用访问对象独立矩阵，如 _Object2World 的 Shader，推荐使用 Unity 内置宏如 UnityObjectToClipPos；
      • 不过也可以使用同一个大贴图、但是使用其中不同的小贴图，这可以使用 Unity 当中的图集功能，图集的原理就是将很多个贴图放到一张大贴图当中，给其他物体共享，但是为不同的物体制定不同的 UV；
      1. 物体必须有 ​ Mesh Filter ​ 和 ​Mesh Renderer ​组件，且网格已启用 Read/Write ​功能，顶点数大于 0；
      1. 符合顶点数限制，单个合批组的顶点总数不能超过 64,000（），如果超过，Unity 会自动拆分成多个批次；
      1. 使用多 Pass Shader、GrabPass​ 或 ​MaterialPropertyBlock ​ 会破坏合批；
  • 图示：在 Unity 中静态合批后，得到的大网格
    • 合成得到的同一个 Mesh
  • 缺点：
    • 使用静态合批之后，会将原本使用同一个 Mesh 的多个物体合成为一个大的 Mesh，因此原本只需要存储一个 Mesh 的包体，现在变成了存储完整大物体的包体，因此增加了体积；
  • 代码：手动进行静态合批

{
    public GameObject rootObject; // 指定根物体
    
    void Start()
    {
        // 合批根物体下的所有静态物体
        StaticBatchingUtility.Combine(rootObject);
    }
}

GPU 实例化

GPU 实例化基础概念

概念：GPU 实例化

  • 概念：
    • GPU Instancing 是一种通过减少绘制调用（Draw Calls）来提升渲染性能的技术；
    • 如果场景中有很多同样模型的物体，可以使用 GPU 实例化，使得批次数量降到很低。当超过 500 个物体时，才会使用第二个批次；
    • 即：它允许使用单一绘制调用，来渲染多个相同网格的不同实例，每个实例可以有独立的变换矩阵（位置、旋转、缩放）和材质参数，特别适合创建植被、石头等场景中大量重复的网格对象；
    • GPU 实例化将多个对象的顶点数据整合为单一连续数组，利用 GPU 内存预取机制提升带宽利用率。实例化数据缓冲区存储每个实例的变换矩阵、材质参数等差异化信息；
    • 在顶点着色器阶段，会自动注入实例序号（如 gl_InstanceID），通过条件编译指令流控制实现材质切换，并可在计算着色器中进行实例化参数预处；
  • 特点：
    • 优点：
      • 单次绘制调用即可渲染海量实例，并且支持动态修改实例属性（如位置、颜色、缩放），每个实例可以独立调整；
    • 缺点：
      • 实例必须使用相同的网格和相同的材质实例​，但可通过 MaterialPropertyBlock 微调参数如颜色；
      • 由于常量缓冲区大小限制（通常每个缓冲区不得大于 64KB），一个批次最多渲染 1023 个实例（使用 DrawMeshInstanced 时），超出的需要分批处理；
      • 需要硬件 API 支持 + Shader 支持 + 脚本支持；
      • Shader 支持指的是着色器必须支持实例化，即需要添加 #pragma multi_compile_instancing；
  • 概念：GPU 实例化的限制
      1. 不支持使用不同材质的对象；
      1. 使用多 Pass 着色器时只有第一个 Pass 可以实例化；
      1. 前向渲染时受多个光源影响的物体，只有 Base Pass 可以实例化；
      1. 使用光照贴图的物体或受不同光照探针/反射探针影响的物体可能无法实例化；
      1. 一个批次最多渲染 1023 个实例；
      1. 移动平台上需要 OpenGL ES 3.0 以上版本的支持，并且 GPU 实例化与 SRP Batcher 不兼容；
      1. 渲染顶点数较少的网格时，效率可能会较差；
  • 示例：C# 脚本
public class GPUInstancing : MonoBehaviour
{
	public GameObject Prefab;
	
	public int Count = 100;
	
	public float Range = 30;
	
	private GameObject tree = null;
	private Color randomColor = null;
	
	private void Start()
	{
		randomColor = new Color(0,0,0,1);
		
		for(int i = 0;i<Count;i++)
		{
			Vector2 pos = Random.insideUnitCircle * Range;
			tree = Instantiate(Prefab,new Vector3(pos.x,0,pos.y),Quaternion.identity);
			randomColor = new Color(Random.value,Random.value,Random.value);
			
			// 必须通过材质属性快来修改每个材质的属性
			MaterialPropertBlock prop = new MaterialPropertBlock();
			prop.SetColor("_Color",randomColor);
			tree.GetComponentInChildern<MeshRenderer>().SetPropertBlock(prop);
		}
	}
}

	            // 7. 通过UNITY_ACCESS_INSTANCED_PROP调用常量寄存器中的属性值

GPU 实例化的使用

概念：如何使用 GPU 实例化

  • 概念：#pragma multi_compile_instancing
    • #pragma multi_compile_instancing 是 Unity 着色器中的一个重要编译指令，用于启用 ​ GPU Instancing​ 功能；
    • 它让 Unity 为支持实例化的着色器生成特定变体（Variant），从而能在单个绘制调用（Draw Call）中批量渲染多个使用相同网格和材质（但某些属性可能不同）的对象，极大提升渲染效率；
  • 总结：GPU 实例化步骤

      1. ​#pragma multi_compile_instancing
      • 添加此指令后会使材质面板上曝露 Instancing 开关，同时会生成相应的 Instancing 变体
      1. UNITY_VERTEX_INPUT_INSTANCE_ID
      • 在顶点着色器的输入（appdata）和输出（v2f，可选项）中添加；
      • UNITY_VERTEX_INPUT_INSTANCE_ID 的效果如下：
      • 使用了 UNITY_VERTEX_INPUT_INSTANCE_ID 宏的实际效果，等同于以下代码：
      • Unity 使用 INSTANCE_ID 来标识不同的 GPU 实例；
      1. UNITY_INSTANCING_BUFFER_START(arrayName) / UNITY_INSTANCING_BUFFER_END(arrayName)​​
      • 将每个你需要实例化的属性都封装在这个常量寄存器中。其中的参数，为声明的常量寄存器的名称；
      • 可以将需要放入常量寄存器的属性，都写在 START 和 END 之间；
      1. UNITY_DEFINE_INSTANCED_PROP(type, name)​
      • 在上面的 START 和 END 间把需要的每条属性加进来

      • 需放在顶点着色器/片断着色器（可选）中最开始的地方，这样才能访问到全局的 unity_InstanceID
      1. UNITY_TRANSFER_INSTANCE_ID(v, o)​
      • 当需要将实例化 ID 传到片断着色器时，在顶点着色器中添加；
      1. UNITY_ACCESS_INSTANCED_PROP (arrayName, propName)​
      • 在片段着色器当中访问具体的实例化变量；

using UnityEngine;

public class SimpleInstancingExample : MonoBehaviour
{
	// 多个模式使用的同一个网格
    public Mesh meshToDraw; 
    public Material materialToUse; // 在Inspector中分配一个材质（需启用GPU Instancing）
    public int numberOfInstances = 100; // 想要创建的实例数量

    private Matrix4x4[] instanceMatrices; // 存储所有实例变换矩阵的数组

    void Start()
    {
        // 确保材质启用GPU Instancing
        materialToUse.enableInstancing = true;

        // 初始化矩阵数组
        instanceMatrices = new Matrix4x4[numberOfInstances];

        // 为每个实例生成一个随机的位置和缩放
        for (int i = 0; i < numberOfInstances; i++)
        {
            Vector3 randomPosition = new Vector3(Random.Range(-10f, 10f), 0f, Random.Range(-10f, 10f));
            Vector3 randomScale = Vector3.one * Random.Range(0.5f, 2.0f);
            Quaternion rotation = Quaternion.identity; // 本例中使用默认无旋转

            // 创建变换矩阵 (TRS: Translation, Rotation, Scale)
            instanceMatrices[i] = Matrix4x4.TRS(randomPosition, rotation, randomScale);
        }
    }

    void Update()
    {
        // 每帧调用DrawMeshInstanced来绘制所有实例
        // 注意：实例数量不能超过1023，否则需要分批次绘制
        if (instanceMatrices != null && instanceMatrices.Length > 0)
        {
            Graphics.DrawMeshInstanced(meshToDraw, 0, materialToUse, instanceMatrices);
        }
    }
}

using UnityEngine;
using System.Collections;

public class AdvancedGPUInstancingExample : MonoBehaviour
{
    [Header("实例设置")]
    public Mesh instanceMesh; // 要实例化的网格
    public Material instanceMaterial; // 基础材质（必须启用GPU Instancing）
    public int instanceCount = 1000; // 实例数量

    [Header("动画设置")]
    public float minAnimationSpeed = 0.5f;
    public float maxAnimationSpeed = 2.0f;
    public float oscillationAmplitude = 2.0f; // 振荡幅度

	[Header("实例颜色与动画属性")]
    private Matrix4x4[] instanceMatrices; // 存储实例的变换矩阵
    private MaterialPropertyBlock propertyBlock; // 材质属性块
    private Vector4[] instanceColors; // 每个实例的颜色
    private float[] animationTimeOffsets; // 每个实例的动画时间偏移
    private float[] animationSpeeds; // 每个实例的动画速度

    void Start()
    {
        // 检查平台支持
        if (!SystemInfo.supportsInstancing)
        {
            Debug.LogError("当前平台不支持GPU Instancing!");
            return;
        }

        // 确保材质启用实例化
        instanceMaterial.enableInstancing = true;

        // 初始化数组
        instanceMatrices = new Matrix4x4[instanceCount];
        instanceColors = new Vector4[instanceCount];
        animationTimeOffsets = new float[instanceCount];
        animationSpeeds = new float[instanceCount];
        
        propertyBlock = new MaterialPropertyBlock();

        // 初始化实例数据
        for (int i = 0; i < instanceCount; i++)
        {
            // 随机位置（分布在球形范围内）
            Vector3 position = Random.insideUnitSphere * 10f;
            Quaternion rotation = Random.rotation;
            Vector3 scale = Vector3.one * Random.Range(0.5f, 2.0f);

            instanceMatrices[i] = Matrix4x4.TRS(position, rotation, scale);

            // 随机颜色
            instanceColors[i] = new Vector4(
                Random.Range(0.2f, 1.0f), // R
                Random.Range(0.2f, 1.0f), // G
                Random.Range(0.2f, 1.0f), // B
                1.0f // A
            );

            // 随机动画参数
            animationTimeOffsets[i] = Random.Range(0f, 100f);
            animationSpeeds[i] = Random.Range(minAnimationSpeed, maxAnimationSpeed);
        }

        // 将数组数据设置到MaterialPropertyBlock
        propertyBlock.SetVectorArray("_InstanceColor", instanceColors);
        propertyBlock.SetFloatArray("_TimeOffset", animationTimeOffsets);
        propertyBlock.SetFloatArray("_AnimationSpeed", animationSpeeds);
    }

    void Update()
    {
        if (instanceMatrices == null || instanceMatrices.Length == 0)
            return;

        // 更新实例位置（示例：简单的上下振荡动画）
        for (int i = 0; i < instanceCount; i++)
        {
	        // 从变换矩阵中提取第i个实例的原始位置向量
	        // 因为第4列（索引3）存储的是位置信息（x, y, z坐标和齐次坐标w）
            Vector3 originalPos = instanceMatrices[i].GetColumn(3);
            
            // 计算当前帧该实例在Y轴上的偏移量，产生正弦波状的上下运动
            float yOffset = Mathf.Sin((Time.time * animationSpeeds[i]) + animationTimeOffsets[i]) * oscillationAmplitude;
            Vector3 newPos = new Vector3(originalPos.x, originalPos.y + yOffset, originalPos.z);
            
            // 获取原有的旋转和缩放，只更新位置
            Quaternion rotation = instanceMatrices[i].rotation;
            Vector3 scale = instanceMatrices[i].lossyScale;
            instanceMatrices[i] = Matrix4x4.TRS(newPos, rotation, scale);
        }

        // 分批绘制实例（每批最多1023个）
        int batches = Mathf.CeilToInt(instanceCount / 1023f);
        for (int i = 0; i < batches; i++)
        {
            int startIndex = i * 1023;
            int count = Mathf.Min(1023, instanceCount - startIndex);
            
            Matrix4x4[] batchMatrices = new Matrix4x4[count];
            System.Array.Copy(instanceMatrices, startIndex, batchMatrices, 0, count);

            // 绘制实例
            Graphics.DrawMeshInstanced(
                instanceMesh, 
                0, 
                instanceMaterial, 
                batchMatrices, 
                count, 
                propertyBlock,
                UnityEngine.Rendering.ShadowCastingMode.On,
                true
            );
        }
    }

    // 动态修改特定实例的颜色（示例方法）
    public void ChangeInstanceColor(int instanceIndex, Color newColor)
    {
        if (instanceIndex >= 0 && instanceIndex < instanceCount)
        {
            instanceColors[instanceIndex] = newColor;
            propertyBlock.SetVectorArray("_InstanceColor", instanceColors);
        }
    }

    // 动态修改所有实例的动画速度（示例方法）
    public void ChangeAllAnimationSpeeds(float newMinSpeed, float newMaxSpeed)
    {
        for (int i = 0; i < instanceCount; i++)
        {
            animationSpeeds[i] = Random.Range(newMinSpeed, newMaxSpeed);
        }
        propertyBlock.SetFloatArray("_AnimationSpeed", animationSpeeds);
    }
}

Shader "Custom/AdvancedInstancedShader"
{
    Properties
    {
        _MainTex ("Albedo (RGB)", 2D) = "white" {}
        _Glossiness ("Smoothness", Range(0,1)) = 0.5
        _Metallic ("Metallic", Range(0,1)) = 0.0
    }
    SubShader
    {
        Tags { "RenderType"="Opaque" }
        LOD 200

        CGPROGRAM
        // 物理基于标准光照模型，并在所有灯光类型上启用阴影
        #pragma surface surf Standard fullforwardshadows
        // 使用着色器模型3.0目标，以获得更好看的光照
        #pragma target 3.0
        // 启用GPU实例化支持
        #pragma multi_compile_instancing

        sampler2D _MainTex;

        struct Input
        {
            float2 uv_MainTex;
        };

        half _Glossiness;
        half _Metallic;

        // 实例化属性缓冲区
        UNITY_INSTANCING_BUFFER_START(Props)
            // 每个实例的颜色
            UNITY_DEFINE_INSTANCED_PROP(float4, _InstanceColor)
            // 每个实例的时间偏移
            UNITY_DEFINE_INSTANCED_PROP(float, _TimeOffset)
            // 每个实例的动画速度
            UNITY_DEFINE_INSTANCED_PROP(float, _AnimationSpeed)
        UNITY_INSTANCING_BUFFER_END(Props)

        void surf (Input IN, inout SurfaceOutputStandard o)
        {
            // 从纹理获取基础颜色
            fixed4 baseColor = tex2D (_MainTex, IN.uv_MainTex);
            
            // 获取实例特定的颜色
            fixed4 instanceColor = UNITY_ACCESS_INSTANCED_PROP(Props, _InstanceColor);
            
            // 结合基础颜色和实例颜色
            fixed4 finalColor = baseColor * instanceColor;
            
            // 可选：基于时间动态调整颜色（示例）
            float timeOffset = UNITY_ACCESS_INSTANCED_PROP(Props, _TimeOffset);
            float animationSpeed = UNITY_ACCESS_INSTANCED_PROP(Props, _AnimationSpeed);
            float timeFactor = sin(_Time.y * animationSpeed + timeOffset) * 0.5 + 0.5;
            
            // 应用动态颜色效果
            finalColor.rgb = lerp(finalColor.rgb, finalColor.rgb * 1.2, timeFactor);

            o.Albedo = finalColor.rgb;
            o.Metallic = _Metallic;
            o.Smoothness = _Glossiness;
            o.Alpha = finalColor.a;
        }
        
        ENDCG
    }
    
    FallBack "Diffuse"
}

总结：四类技术的优缺点
  • 总结：
    • 物体完全静态、不动：静态合批；
    • 物体动，且数量多、模型相同：GPU 实例化；
    • 物体动，但数量少、模型简单：动态合批；
    • 物体动，且模型不同、Shader 相同：SRP Batcher；
