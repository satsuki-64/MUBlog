Lecture 4：URP 与 SRP Batcher

什么是 URP 渲染管线

URP 基础概念

概念：什么是 URP 渲染管线

  • 概念：
    • Unity 的 ​ URP（Universal Render Pipeline，通用渲染管线）是一款轻量级、高性能且高度可定制的渲染管线​​，它旨在为广泛的平台（包括移动设备、PC 和主机）提供高质量的图形渲染，并在性能与视觉效果之间取得良好平衡；
    • URP 的核心在于其可编程性和模块化，它允许开发者通过 C# 脚本和可配置的资源来控制渲染流程的几乎所有方面，取代了传统内置渲染管线中许多固定的、不可修改的部分；
    • URP 和 HDRP 同时并存，其前身是 LWRP，它们两者都是使用 SRP 可编程渲染管线编写出来的；
  • 组成：
    • （1）URP Asset (渲染管线资源)​​：
      • 这是一个配置文件​（一种 ScriptableObject），用于定义 URP 渲染管线的全局设置，例如渲染比例、HDR 支持、光照质量、阴影质量、后处理效果等；
      • 你可以在项目中创建多个 URP Asset，并为不同平台或质量等级使用不同的配置；
    • （2）Renderer Features (渲染器特性)​​：
      • 这是 URP ​扩展性的重要体现，可以创建自定义的 Renderer Feature，并向渲染管线中添加特定的 Scriptable Render Pass，这允许你在渲染过程的特定阶段（例如在不透明物体渲染之后、天空盒渲染之前）插入自定义的渲染逻辑，从而实现诸如全屏特效、模糊、轮廓线等特殊效果；
    • （3）Scriptable Render Passes (可编程渲染通道)​​：
      • 这是 URP ​渲染工作的基本单元，URP 的渲染过程由一系列按顺序执行的 Render Pass 组成，例如深度预处理、不透明物体渲染、天空盒渲染、透明物体渲染、后处理等；
  • 方法：创建 URP 配置文件
    • 在 Project 中创建 URP 配置文件，其中会引用到 Forward Renderer 前向渲染；
    • 将创建的 URP 配置进行设置；

URP 语法基础

概念：URP 语法基础

  • 概念：
    • URP Shader 仍然采用 Unity 经典的 ShaderLab 语法框架，但内部编程语言从 CG 过渡到了 HLSL，基本结构保持 Shader{ SubShader { Pass{} }, FallBack } 的形式；

URP 下缓冲区

概念：URP 下缓冲区

  • 概念：_CameraDepthTexture：
    • 每个像素的深度值（非线性，通常为 0-1），可以用于软粒子、雾效、景深、水面交互、边缘检测，启动需要在 URP Asset 或 Camera 组件中勾选 ​ Depth Texture；
  • 概念：_CameraOpaqueTexture
    • 存储不透明物体渲染完成后的颜色信息（RGB），启动需要在 URP Asset 或 Camera 组件中勾选 ​ Opaque Texture；
  • 概念：_CameraNormalsTexture
    • 存储屏幕空间法线信息（通常编码为 XY），通过自定义 Renderer Feature 生成，或启用内置 SSAO 效果间接获取；
  • 代码：
Shader "Example/URPScreenBuffers" {
    Properties 
    {
        // 可在此定义属性，如控制显示模式的变量
    }
    SubShader 
    {
        Tags 
        { 
            "RenderType"="Opaque" 
            "RenderPipeline"="UniversalPipeline"
        }

        Pass 
        {
            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag

            // 包含URP核心库和屏幕纹理库
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Core.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/DeclareDepthTexture.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/DeclareOpaqueTexture.hlsl"
            
            // 假设已启用或生成法线纹理，手动声明
            TEXTURE2D_X_FLOAT(_CameraNormalsTexture);
            SAMPLER(sampler_CameraNormalsTexture);

            struct Attributes 
            {
                float4 positionOS : POSITION;
            };

            struct Varyings 
            {
                float4 positionCS : SV_POSITION;
                float4 screenPos : TEXCOORD0; // 用于计算屏幕UV
            };

            Varyings vert(Attributes IN) 
            {
                Varyings OUT;
                VertexPositionInputs positionInputs = GetVertexPositionInputs(IN.positionOS.xyz);
                
                OUT.positionCS = positionInputs.positionCS;
                OUT.screenPos = ComputeScreenPos(OUT.positionCS); // 正确计算屏幕坐标
                
                return OUT;
            }

            half4 frag(Varyings IN) : SV_Target 
            {
                // 1. 计算标准化屏幕UV
                float2 screenUV = IN.screenPos.xy / IN.screenPos.w;

                // 2. 采样屏幕缓冲区
                // 采样深度并转换为线性眼空间深度
                real rawDepth = SampleSceneDepth(screenUV);
                real linearDepth = LinearEyeDepth(rawDepth, _ZBufferParams);
                // 采样不透明颜色
                float3 sceneColor = SampleSceneColor(screenUV);
                // 采样法线（需解码）
                float4 packedNormal = SAMPLE_TEXTURE2D_X(_CameraNormalsTexture, sampler_CameraNormalsTexture, screenUV);
                float3 worldNormal = normalize(unpackNormal(packedNormal)); // 假设有解码函数

                // 3. 从深度和UV重构世界位置（关键步骤）
                float3 worldPos = ComputeWorldSpacePosition(screenUV, rawDepth, UNITY_MATRIX_I_VP);

                // 4. 应用示例：根据深度可视化或进行后续计算
                // 例如，将线性深度作为颜色输出
                half depthVisual = saturate(linearDepth / 10.0); // 按需缩放
                return half4(depthVisual.rrr, 1.0);

                // 或者返回场景颜色
                // return half4(sceneColor, 1.0);
            }
            ENDHLSL
        }
    }
}

补充：GetVertexPositionInputs 方法的实现

  • 代码：
  • 解析：
    • 将一个顶点从对象空间 (Object Space)​​ 变换到渲染管线所需的其他所有关键空间，并将这些变换后的坐标打包到一个结构体中，方便后续使用；
  • 代码：
// 方法定义：输入一个对象空间的顶点坐标，返回一个包含所有变换后坐标的结构体
VertexPositionInputs GetVertexPositionInputs(float3 positionOS)
{
    // 声明一个 VertexPositionInputs 类型的结构体变量来存储结果
    VertexPositionInputs input;

    // 1. 对象空间 -> 世界空间 (World Space)
    // 这是最基础的变换，将顶点从模型自身的坐标系转换到整个游戏世界的全局坐标系。
    // 这是后续所有其他变换的基础。
    input.positionWS = TransformObjectToWorld(positionOS);

    // 2. 世界空间 -> 视图空间 (View Space / Eye Space)
    // 将顶点从世界坐标转换到相机视角下的坐标。
    // 在这个空间中，相机位于原点，视线方向通常是 -Z 轴。
    // 常用于一些需要基于视角的计算。
    input.positionVS = TransformWorldToView(input.positionWS);

    // 3. 世界空间 -> 齐次裁剪空间 (Homogeneous Clip Space)
    // 这是最关键的一步变换。它将顶点转换到裁剪空间，其坐标用于判断顶点是否在摄像机的可视范围内（裁剪）。
    // 这个坐标也是最终传递给光栅化阶段的坐标。
    // .w 分量非常重要，它通常用于透视校正（透视除法）。
    input.positionCS = TransformWorldToHClip(input.positionWS);

    // 4. 计算标准设备坐标 (Normalized Device Coordinates - NDC)
    // 通过齐次裁剪空间坐标来计算 NDC。
    // NDC 是一个标准化后的坐标空间，其 x, y 坐标范围通常是 [-1, 1]（或 [0, 1]），
    // z 坐标范围是 [0, 1]（或 [-1, 1]），具体取决于图形 API。
    float4 ndc = input.positionCS * 0.5f;

    // 4.1 处理 NDC 的 xy 分量
    // 将裁剪空间坐标的 xy 分量从 [-w, w] 映射到 [0, w] (*0.5f 已完成),
    // 然后通过加上 .zw 分量（即 [w, w]）来完成透视除法后的偏移，最终得到 [0, 1] 范围的 NDC
    // _ProjectionParams.x 是一个Unity内置变量，用于处理不同图形API（如DirectX和OpenGL）在UV朝向和NDC z范围上的差异
    // 这里乘以 _ProjectionParams.x（通常为 1 或 -1）主要是为了翻转Y坐标，以适配不同的渲染API坐标系
    input.positionNDC.xy = float2(ndc.x, ndc.y * _ProjectionParams.x) + ndc.zw;

    // 4.2 处理 NDC 的 zw 分量
    // 直接将裁剪空间坐标的 zw 分量存储到 NDC 的 zw 分量。
    // .z 分量在透视除法后将成为深度值，.w 分量是用于插值的透视校正因子。
    input.positionNDC.zw = input.positionCS.zw;

    // 返回包含了所有空间坐标的结构体
    return input;
}

概念：URP 框架相关知识点

  • 概念：RenderPipeline 标签
    • 明确指定 SubShader 兼容的渲染管线，确保该 SubShader ​仅在被 URP 激活时生效。若未声明或值不匹配，URP 会忽略该 SubShader，导致材质显示为粉色；
    • 比如："RenderPipeline" = "UniversalRenderPipeline" 用于声明该 SubShader 专为 URP 设计；
    • 这个渲染管线标签, 对应的管线 C# 代码 UniversalRenderPipeline.cs 中的 Shader.globalRenderPipeline = UniversalPipeline, LightweightPipeline，只有带有 UniversalPipeline 或 LightweightPipeline 的 Tags 的 SubShader 才会生效；

最简 URP Shader 框架

Shader "MyShader/URP/SimpleUnlit" // Shader在材质下拉菜单中的路径和名称
{
    Properties // 在材质面板中显示的属性
    {
        // 声明一个颜色属性，默认白色
        _BaseColor ("Base Color", Color) = (1, 1, 1, 1)
    }
    SubShader
    {
        Tags
        {
            // 告知Unity此Shader用于URP管线
            "RenderPipeline"="UniversalPipeline"
            // 渲染类型为不透明[1,4](@ref)
            "RenderType"="Opaque"
        }

        Pass // 一个渲染通道
        {
            HLSLPROGRAM // URP使用HLSL语言
            // 定义顶点着色器函数名
            #pragma vertex vert
            // 定义片元着色器函数名
            #pragma fragment frag

            // 包含URP的核心库[1,3]
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Core.hlsl"

            // 定义一个常量缓冲区，用于包裹材质属性，有助于性能优化（如SRP Batcher）
            CBUFFER_START(UnityPerMaterial)
                half4 _BaseColor;
            CBUFFER_END

            // 顶点着色器的输入结构
            struct Attributes
            {
                float4 positionOS : POSITION; // 模型空间下的顶点位置
            };

            // 顶点着色器的输出/片元着色器的输入结构（插值后）
            struct Varyings
            {
                float4 positionCS : SV_POSITION; // 裁剪空间下的顶点位置
            };

            // 顶点着色器
            Varyings vert(Attributes IN)
            {
                Varyings OUT;
                // 将顶点位置从模型空间转换到裁剪空间
                OUT.positionCS = TransformObjectToHClip(IN.positionOS.xyz);
                return OUT;
            }

            // 片元着色器
            half4 frag(Varyings IN) : SV_Target
            {
                // 返回在Properties中定义的_BaseColor
                return _BaseColor;
            }
            ENDHLSL
        }
    }
    // 如果SubShader不适用时的备选Shader
    FallBack "Hidden/Universal Render Pipeline/FallbackError"
}

常量缓冲区 CBUFFER

概念：常量缓冲区 CBUFFER

  • 概念：
    • 常量缓冲区是 GPU 的显存中的一块区域，GPU 读取这部分的数据的速度很快，用于存储着色器程序执行过程中需要的不变数据；
    • 这里的不变指的是在一次绘制调用 (Draw Call) 或计算着色器分派 (Dispatch) 过程中，这些值保持固定；
    • 定义在 CBUFFER_START 和 CBUFFER_END 之间声明，其内部的所有数据都放到一个常量缓冲区；
  • 代码：
// UnityPerMaterial：表示每个材质球定义一次常量缓冲区
CBUFFER_START(UnityPerMaterial)
	half4 _Color;
CBUFFER_END

SRP Batcher

SRP Batcher 基础概念

概念：什么是 SRP Batcher

  • 总结：
    • SRP Batcher ​ 的核心目标，是为了显著降低由 SetPass Call 带来的 CPU 开销，将材质属性持久化在 GPU 的常量缓冲区中，绘制时仅需绑定数据偏移指针；
  • 作用：
    • SRP Batcher 是 Unity 的​可编程渲染管线（SRP）​​中的一项 ​ CPU 渲染优化技术，它的核心目标是减少 CPU 准备和提交渲染命令的开销，从而提升渲染效率，尤其在场景中存在大量使用相同着色器但不同材质的对象时；
    • 传统的无 SRP Batcher 的问题，在于每次绘制使用新材质时，CPU 都需重新设置并上传所有材质属性到 GPU。材质越多，CPU 设置开销越大；
    • 它会为每个材质维护一个在 GPU 显存中的常量缓冲区（CBuffer），材质属性（如 _Color，_MainTex_ST 等）会提前存入并持久化。当渲染大量使用相同着色器变体但材质属性值不同的物体时，CPU 无需再为每个物体重新上传所有材质数据，而只需快速告知 GPU “去使用缓冲区中第 N 个材质的数据块”，从而极大地减少了每次绘制的设置工作。
    • 其​不直接减少 Draw Call 的数量，而是极大降低每个 Draw Call 之间 CPU 的准备工作和状态切换开销；
  • 概念：与常量缓冲区的关系
    • SRP Batcher 依赖于 SRP 中常量缓冲区的实现机制；

合批技术汇总

概念：合批技术汇总

  • （1）静态合批、动态合批与 GPU 实例化都是适用于内置渲染管线与 URP 渲染管线的，而 SRP Batcher 只支持 URP 渲染管线；
  • （2）静态合批通常使用在静态场景烘培的时候，其要求可以是不同的网格，但是必须是同一个材质球；
  • （3）动态合批的限制，包括顶点属性的数量在 900 以下，可以是不同的网格，但是必须是同一个材质球；
  • （4）GPU 实例化，要求可以使用不同的材质球，并且材质球上面的参数不一样，但是需要是同一个网格；
  • （5）SRP 可以拥有不同的网格或者材质球，但是必须要是相同的 Shader；

SRP 合批的要求

概念：SRP 合批的要求

  • 概念：合批的具体要求
    • （1）必须得是普通网格，蒙皮网格不可以，粒子对象也不能合批；
    • （2）已经是网格时，其网格使用的 Shader 必须支持 SRP Batcher（即网格必须得是静态网格，其必须支持合批）；
    • （3）使用 MaterialPropertyBlocks 的材质对象不能合批
  • 概念：Shader 中支持 SRP Batcher
    • （1）定义 CBUFFER 常量缓冲区；
    • （2）在 CBUFFER_START 中定义 UnityPerMaterial；
    • （3）将变量放到 UnityPerDraw 当中（这一步由 Unity 自动自己完成，这一步是把 Unity 内部的一些变量放到 CBUFFER 当中）；

SRP Batcher 的实现

概念：SRP Batcher 的实现

  • 概念：传统渲染管线下的调用
  • 概念：URP 渲染管线下的 CBUFFER
    • Unity 内的对象和 Material 材质被分开处理。对于 Unity 内的对象，CPU 为其单独进设置，将其放入到 GPU 当中的 Per Object large Buffer 当中；
    • 对于 Material 材质，只有当材质内容变化时才会更新，CPU 进行材质内容设置，对于每一个材质球设置一个 CBUFFER。哪一个发生了变化，哪一个就进行更新；
    • 最后，在 Shader 当中统一对 Per Object large Buffer 和 CBUFFER 进行调用；
