DepthOnlyPass 的作用

概念：DepthOnlyPass 的作用

  • 概念：
    • DepthOnlyPass ​是 URP 中带有 LightMode="DepthOnly"​ 的着色器通道，用于在不透明渲染之前（或需要时）把场景中物体的深度写入深度缓冲（通常是 _CameraDepthTexture），从而得到一张可被后续屏幕空间效果使用的深度图；
    • 它常由渲染器在需要深度预通道时自动调度执行。URP 也支持在条件允许时直接从相机深度缓冲拷贝一份深度图，以避免额外的绘制；但当拷贝条件不满足或需要提前写入深度时，就会执行 DepthOnlyPass。该通道对诸如屏幕空间环境光遮蔽（SSAO）、景深、屏幕空间反射、以及从屏幕空间重建世界坐标等效果至关重要；
  • 分析：
    • 此 Shader 的 Copy Depth Pass 只会在不开启 MSAA 才会开启，如果开启 MSAA ，如下图所示：
    • 则此时 Copy Depth Pass 会消失，进而出现另一个 Depth Prepass：
  • 总结：
    • 如果当前的项目没有开启抗锯齿 MSAA，则 Unity 会自己添加 Copy Depth 得到深度；
    • 如果开启了 MSAA，则需要在 Shader 中加入 DepthOnly 的 Pass，不然不渲染深度；

// 高光光滑度采样函数，支持两种高光，分别是独立的高光贴图和颜色通道作为高光
half4 SampleSpecularSmoothness(float2 uv, half alpha, half4 specColor, TEXTURE2D_PARAM(specMap, sampler_specMap))
{
   half4 specularSmoothness = half4(0, 0, 0, 1);
   
   // 根据材质类型选择采样方式
   #ifdef _SPECGLOSSMAP
       // 使用独立高光贴图
       specularSmoothness = SAMPLE_TEXTURE2D(specMap, sampler_specMap, uv) * specColor;
   #elif defined(_SPECULAR_COLOR)
       // 使用颜色通道作为高光
       specularSmoothness = specColor;
   #endif
   
   // 从基础Alpha通道获取光滑度（需_SMOOTHNESS_TEXTURE_ALBEDO_CHANNEL_A宏支持）
   #ifdef _GLOSSINESS_FROM_BASE_ALPHA
       specularSmoothness.a = alpha;
   #endif
   
   return specularSmoothness;
}

inline void InitializeSimpleLitSurfaceData(float2 uv, out SurfaceData outSurfaceData)
{
   // 初始化表面数据结构体
   outSurfaceData = (SurfaceData)0;
   
   // 采样基础纹理（Albedo + Alpha）
   half4 albedoAlpha = SampleAlbedoAlpha(uv, TEXTURE2D_ARGS(_BaseMap, sampler_BaseMap));
   
   // 处理Alpha通道（裁剪测试）
   outSurfaceData.alpha = albedoAlpha.a * _BaseColor.a;
   outSurfaceData.alpha = AlphaDiscard(outSurfaceData.alpha, _Cutoff); // Alpha裁剪
   
   // 计算最终Albedo值（基础色 * 材质色）
   outSurfaceData.albedo = albedoAlpha.rgb * _BaseColor.rgb;
   outSurfaceData.albedo = AlphaModulate(outSurfaceData.albedo, outSurfaceData.alpha); // Alpha调制
   
   // 采样高光/光滑度信息
   half4 specularSmoothness = SampleSpecularSmoothness(
       uv, 
       outSurfaceData.alpha, 
       _SpecColor, 
       TEXTURE2D_ARGS(_SpecGlossMap, sampler_SpecGlossMap)
   );
   
   // 填充表面数据结构体
   outSurfaceData.metallic = 0.0;        // URP简单光照模型不使用金属度
   outSurfaceData.specular = specularSmoothness.rgb; // 高光强度
   outSurfaceData.smoothness = specularSmoothness.a; // 光滑度
   
   // 采样法线贴图（默认BumpMap）
   outSurfaceData.normalTS = SampleNormal(uv, TEXTURE2D_ARGS(_BumpMap, sampler_BumpMap));
   
   // 环境遮蔽（默认1.0，无遮蔽）
   outSurfaceData.occlusion = 1.0;
   
   // 自发光计算
   outSurfaceData.emission = SampleEmission(
       uv, 
       _EmissionColor.rgb, 
       TEXTURE2D_ARGS(_EmissionMap, sampler_EmissionMap)
   );
}

{

   UNITY_SETUP_INSTANCE_ID(input);           // 初始化实例ID
   UNITY_TRANSFER_INSTANCE_ID(input, output);// 传递实例ID
   UNITY_INITIALIZE_VERTEX_OUTPUT_STEREO(output); // 初始化立体渲染

   #if defined(_ALPHATEST_ON)
       // 转换纹理坐标（考虑主贴图平铺参数）
       output.uv = TRANSFORM_TEX(input.texcoord, _BaseMap);
   #endif
   // 将模型空间坐标转换为裁剪空间坐标
   output.positionCS = TransformObjectToHClip(input.position.xyz);
   return output;
}

half DepthOnlyFragment(Varyings input) : SV_TARGET
{

   UNITY_SETUP_STEREO_EYE_INDEX_POST_VERTEX(input); // 设置立体渲染眼索引

   // 如果开启了 Alpha 测试，则部分 Alpha 小于阈值的区域将会被剔除
   #if defined(_ALPHATEST_ON)
       // 采样基础贴图的 Alpha 通道

           input.uv, 
           TEXTURE2D_ARGS(_BaseMap, sampler_BaseMap)

   #endif

   return input.positionCS.z;

    • 同时，使用 clip() 函数进行裁剪，丢弃 Alpha 值小于 _Cutoff 的片元，避免透明区域参与后续光照计算；
  • 概念：Alpha 方法
// URP Alpha计算核心函数
// 功能：根据材质属性计算最终Alpha值，并执行Alpha裁剪
// 参数：
//   albedoAlpha: 基础纹理Alpha值（来自_AlbedoAlpha采样）
//   color: 材质色（包含漫反射颜色和高光信息）
//   cutoff: Alpha裁剪阈值（_Cutoff参数）
half Alpha(half albedoAlpha, half4 color, half cutoff)
{
   half alpha;
   
   // 材质Alpha来源判断（条件编译）
   #if !defined(_SMOOTHNESS_TEXTURE_ALBEDO_CHANNEL_A) && !defined(_GLOSSINESS_FROM_BASE_ALPHA)
       // 常规情况：Albedo Alpha与材质色Alpha相乘
       // 适用场景：独立Alpha通道（如半透明材质）
       alpha = albedoAlpha * color.a;
   #else
       // 优化情况：直接使用材质色Alpha通道
       // 适用场景：高光/光滑度使用颜色通道时（节省纹理采样）
       alpha = color.a;
   #endif

   // 执行Alpha裁剪（核心逻辑）
   alpha = AlphaDiscard(alpha, cutoff);

   return alpha;
}

// AlphaDiscard函数实现（补充说明）
// 作用：根据阈值进行Alpha测试并丢弃无效片元
half AlphaDiscard(half alpha, half cutoff)
{
   // 剔除低于阈值的片元（深度测试前执行）
   clip(alpha - cutoff); // 等效于 if(alpha < cutoff) discard;
   
   // 返回处理后的Alpha值（用于后续混合或深度写入）
   return alpha;
}

ShaderCaster Pass 的作用

概念：ShaderCaster Pass 的作用

  • 概念：ShaderCaster Pass 的作用
      1. 在光源处添加一个相机，同时打开深度测试与写入，并生成 ShadowMap（深度图）
      1. 正常绘制场景，把当前片断的转换到光源空间下，计算得到的深度与采样 ShadowMap 中的深度值进行比较，如果大于则表示该片断处于阴影中，反之则不在阴影中；

ShaderCaster 相关设置

概念：ShaderCaster 相关设置

  • 图示：
  • 解释：ShaderCaster 的作用
    • 开启了名称为 ShaderCaster 的 Pass 之后，其就可以被绘制到 ShadowMapping 当中；
    • 如果关闭了此 ShaderCaster 的 Pass ，则当前物体可以接收阴影，但是不能自己产生投影；
    • 因为当前物体（立方体）的深度图没有被计算到 Shadowmapping 当中，因此其遮住的部分不能正常产生阴影。但是其可以接受阴影，因为其他物体可以正常写入深度值；
  • 分析：ZWrite On
    • 这是开启深度缓冲写入的关键命令，它指示 GPU 将通过深度测试的片元的深度值写入深度缓冲区（在此处即是 ShadowMap）
  • 分析：ZTest LEqual
    • 设置深度测试函数为“小于等于”，确保更靠近光源的物体才能写入深度；
  • 分析：ColorMask 0
    • 禁用颜色输出，因为生成 ShadowMap 时我们只关心深度信息，这有助于提升性能；

顶点着色器 - ShadowPassVertex

分析：顶点着色器 - ShadowPassVertex

  • 概念：ShadowCaster 的顶点着色器的作用
      1. 将顶点从对象空间变换到世界空间；
      1. 在世界坐标当中，应用阴影偏移（Shadow Bias），包括深度偏移和法线偏移；
      1. 将顶点变换到光源的裁剪空间，输出位置 output.positionCS，这个变换后的坐标的 Z 分量，本质上就代表了该顶点相对于光源的深度值；
  • 代码：顶点着色器
Varyings ShadowPassVertex(Attributes input)
{
   Varyings output;
   
   // 处理GPU Instancing实例ID
   // 为每个实例分配唯一标识符，支持批量渲染
   UNITY_SETUP_INSTANCE_ID(input);  
   UNITY_TRANSFER_INSTANCE_ID(input, output); // 传递实例ID到输出结构体

   // 处理透明度测试相关的UV坐标（如果启用_AlPHATEST_ON）
   #if defined(_ALPHATEST_ON)
   // 将UV坐标根据_BaseMap的Tiling/Offset参数进行变换
   output.uv = TRANSFORM_TEX(input.texcoord, _BaseMap);
   #endif

   // 核心阴影坐标计算
   // 调用URP内置函数完成以下操作：
   // 1. 对象空间→世界空间坐标转换
   // 2. 世界空间→光源空间坐标转换
   // 3. 应用阴影偏移（Depth Bias/Normal Bias）
   // 4. 裁剪空间变换
   // 其底层就是调用 UNITY_MATRIX_VP 矩阵，将当前点进行转换
   output.positionCS = GetShadowPositionHClip(input);

   return output;
}

// 阴影投射顶点着色器核心函数
// 作用：将顶点坐标转换为光源空间裁剪坐标，并应用阴影偏移
// 输入：Attributes结构体包含顶点位置、法线等基础属性
// 输出：经过阴影矩阵变换后的齐次裁剪空间坐标
float4 GetShadowPositionHClip(Attributes input)
{
   // 将顶点位置从对象空间转换到世界空间
   float3 positionWS = TransformObjectToWorld(input.positionOS.xyz);
   
   // 将法线从对象空间转换到世界空间
   float3 normalWS = TransformObjectToWorldNormal(input.normalOS);

   // 根据光源类型计算世界空间光照方向
   #if _CASTING_PUNCTUAL_LIGHT_SHADOW
   // 点光源/聚光灯：计算从光源位置到顶点的方向向量
   float3 lightDirectionWS = normalize(_LightPosition - positionWS);
   #else
   // 方向光：直接使用预定义的光照方向
   float3 lightDirectionWS = _LightDirection;
   #endif

   // 应用阴影偏移并转换到齐次裁剪空间
   // 重点：使用 ApplyShadowBias 方法，包含深度偏移和法线偏移计算
   float4 positionCS = TransformWorldToHClip(
       ApplyShadowBias(positionWS, normalWS, lightDirectionWS)
   );

   // 处理深度值裁剪（防止Z-fighting）
   positionCS = ApplyShadowClamping(positionCS);

   return positionCS;
}

深度偏移

自遮挡问题原理

分析：自遮挡问题原理

  • 图示：
  • 分析：
    • 当使用 Shadow Mapping 生成 Shadow Mapp 图时，其原理是从光源方向、使用一个虚拟摄像机，逐像素的发射射线，然后射线穿过到的地方、记录其深度值；
    • 因此，当通过虚拟摄像机的若干计算深度的射线打向地板后，其与地面上的物体的对应是一个离散值，因为是每一个像素记录一个深度值；
    • 因此，当像素密度不高的时候，这个时候、较少的像素记录了场景中的地面上物体对应的深度信息，而当 Camera 看向这个地方的时候，可能会产生此点的深度值被前面记录的深度更小的平面遮挡的问题；
    • 当前 Camera 看向的这个地方未被场景中物体遮挡，但是因为此点的深度值被前面记录的深度更小的平面遮挡，因此此点会被认为被遮挡，简称为自遮挡；
    • 当光线几乎垂直于平面时，这个问题最小；
    • 当光线和平面几乎平行时，这个问题最大；
  • 解决：引入 Shadow Bias
    • 引入一段 Shadow Bias 的距离，降低自遮挡的问题；

    • 在 GetShadowPositionHClip 当中的 ApplyShadowBias 代码负责实现深度偏移；
  • 代码：ApplyShadowBias 深度偏移实现
// 阴影偏移计算函数
// 作用：根据光源方向和法线方向，对顶点世界坐标进行阴影偏移处理
// 参数：
//   positionWS: 顶点世界空间坐标
//   normalWS: 顶点世界空间法线
//   lightDirection: 光源方向向量（已归一化）
// 返回值：经过阴影偏移调整后的世界坐标
float3 ApplyShadowBias1(float3 positionWS, float3 normalWS, float3 lightDirection)
{
   // 计算光源方向与法线的反向点积（范围0~1）
   // 该值反映法线与光源方向的夹角（0=平行，1=垂直）
   float invNdotL = 1.0 - saturate(dot(lightDirection, normalWS));
   
   // 根据夹角计算法线偏移缩放因子
   // 当法线与光源平行时（invNdotL=1），偏移量最大
   // 当法线与光源垂直时（invNdotL=0），无偏移
   float scale = invNdotL * _ShadowBias.y;

   // 深度偏移（Depth Bias）
   // 沿光源方向后移顶点，防止自阴影（Shadow Acne）
   // _ShadowBias.xxx 表示XYZ三个方向的偏移量
   positionWS = lightDirection * _ShadowBias.xxx + positionWS;

   // 法线偏移（Normal Bias）
   // 沿法线反方向后移顶点，防止阴影剥离（Peter Panning）
   // 负号表示向光源内部偏移，scale控制偏移强度
   positionWS = normalWS * (-scale) + positionWS;

   return positionWS;
}
  • 分析：Unity 当中实现 ShadowBias 的原理；
    • 因为假设当前 Shadow Mapping 上的值和物体在灯光空间下的深度值一样时、可能会产生不正确的效果，因此 ShadowBias 控制的就是当前 Shadow Mapping 的大小；
    • 通过引入一个 ShadowBias，将假设原本计算 Shadow Mapping 为 10 的地方，加上一个 ShadowBias=0.2，则此时记录下来的 Shadow Mapping 上此像素点的值即为 10.2，则此时物体的深度值就小于 Shadow Mapping；

// 计算主灯光的阴影 - 负责混合主光源的实时阴影与烘焙阴影的核心函数
half MainLightShadow(float4 shadowCoord, float3 positionWS, half4 shadowMask, half4 occlusionProbeChannels)
{
	// 获取主光源的阴影参数（如强度、软硬阴影类型等）
	// x: shadowStrength 阴影强度, y: >= 1.0 if soft shadows, 0.0 otherwise 如果是软阴影则为 1，否则为 0
	// z: main light fade scale, w: main light fade bias
	half4 shadowParams = half4(_MainLightShadowParams);
	
	// 采样实时阴影贴图：将像素的阴影坐标与 ShadowMap 中的深度值进行比较
	// 返回的 realtimeShadow 值（0到1）表示该点被阴影遮挡的程度，0表示完全遮挡
	half realtimeShadow = MainLightRealtimeShadow1(shadowCoord, shadowParams, GetMainLightShadowSamplingData1());

	// 处理烘焙阴影：根据灯光烘焙模式（如Shadowmask或Subtractive）获取烘焙阴影值
	#ifdef CALCULATE_BAKED_SHADOWS
		// 从阴影遮罩纹理或光照探针中提取烘焙阴影信息
		half bakedShadow = BakedShadow(shadowMask, occlusionProbeChannels, shadowParams);
	#else
		// 如果未启用烘焙阴影计算，则默认视为无阴影遮挡（值为1.0）
		half bakedShadow = half(1.0);
	#endif

	// 计算阴影淡出因子：根据像素与世界空间中原点的距离，使阴影在指定范围内平滑淡出
	#ifdef MAIN_LIGHT_CALCULATE_SHADOWS
		half shadowFade = GetMainLightShadowFade(positionWS);
	#else
		// 如果未启用主光源阴影计算，则淡出因子为1.0（无淡出效果）
		half shadowFade = half(1.0);
	#endif

	// 混合实时阴影和烘焙阴影：根据阴影淡出因子和当前烘焙模式策略进行混合
	return MixRealtimeAndBakedShadows(realtimeShadow, bakedShadow, shadowFade);
}

// 返回值：half类型，0表示完全在阴影中，1表示完全不受阴影遮挡
// 参数shadowCoord：像素在阴影空间中的坐标，用于采样阴影贴图

阴影贴图采样 - SampleShadowmap

分析：阴影贴图采样 - SampleShadowmap

  • 代码：SampleShadowmap
// 阴影贴图采样函数 - URP阴影系统的核心采样例程
// 参数说明：
//   ShadowMap, sampler_ShadowMap: 阴影纹理及其专用的深度比较采样器
//   shadowCoord: 阴影坐标（通常包含透视除法前的xyzw分量）
//   samplingData: 阴影采样数据（如软阴影质量、偏移量、纹理尺寸等）
//   shadowParams: 阴影参数（x=阴影强度, y=软阴影质量级别, z=阴影偏移, w=阴影强度等）
//   isPerspectiveProjection: 是否为透视投影（透视投影需要透视除法，正交投影不需要）
real SampleShadowmap1(TEXTURE2D_SHADOW_PARAM(ShadowMap, sampler_ShadowMap), float4 shadowCoord, 
                     ShadowSamplingData samplingData, half4 shadowParams, bool isPerspectiveProjection = true)
{
    // ========== 透视投影处理 ==========
    // 根据投影类型决定是否进行透视除法：将裁剪空间坐标归一化到NDC空间[-1,1]或[0,1]
    // 编译器会在编译时优化掉未使用的分支（如果isPerspectiveProjection在编译时已知）
    if (isPerspectiveProjection)
        shadowCoord.xyz /= shadowCoord.w; // 透视除法：将齐次坐标转换为三维坐标

    real attenuation;      // 最终阴影衰减值（0=完全阴影, 1=无阴影）
    real shadowStrength = shadowParams.x; // 阴影强度参数（控制阴影黑暗程度）

    // ========== 多质量级别软阴影采样 ==========
    // 根据不同的软阴影质量设置，采用不同的采样策略（性能与质量的权衡）
    // 这些质量级别主要为需要严格静态分支的平台设计
    
    #if defined(_SHADOWS_SOFT_LOW)
        // 低质量软阴影：使用简单的滤波方案（如2x2 PCF），性能最佳
        attenuation = SampleShadowmapFilteredLowQuality(TEXTURE2D_SHADOW_ARGS(ShadowMap, sampler_ShadowMap), shadowCoord, samplingData);
    
    #elif defined(_SHADOWS_SOFT_MEDIUM)
        // 中等质量软阴影：平衡性能与效果（如4x4 PCF或优化滤波）
        attenuation = SampleShadowmapFilteredMediumQuality(TEXTURE2D_SHADOW_ARGS(ShadowMap, sampler_ShadowMap), shadowCoord, samplingData);
    
    #elif defined(_SHADOWS_SOFT_HIGH)
        // 高质量软阴影：使用更复杂的滤波算法（如5x5 PCF或方差阴影），效果最好但性能开销最大
        attenuation = SampleShadowmapFilteredHighQuality(TEXTURE2D_SHADOW_ARGS(ShadowMap, sampler_ShadowMap), shadowCoord, samplingData);
    
    #elif defined(_SHADOWS_SOFT)
        // 通用软阴影处理：根据shadowParams.y参数动态选择软阴影质量
        if (shadowParams.y > SOFT_SHADOW_QUALITY_OFF) // 检查是否启用软阴影
        {
            // 使用可配置的滤波采样（可能根据硬件能力自适应）
            attenuation = SampleShadowmapFiltered(TEXTURE2D_SHADOW_ARGS(ShadowMap, sampler_ShadowMap), shadowCoord, samplingData);
        }
        else
        {
            // 硬阴影：单次采样，无滤波（性能最好，阴影边缘锐利）
            attenuation = real(SAMPLE_TEXTURE2D_SHADOW(ShadowMap, sampler_ShadowMap, shadowCoord.xyz));
        }
    
    #else
        // 无软阴影：直接采样阴影贴图，使用硬件深度比较
        attenuation = real(SAMPLE_TEXTURE2D_SHADOW(ShadowMap, sampler_ShadowMap, shadowCoord.xyz));
    
    #endif

    // ========== 阴影强度应用 ==========
    // 将原始衰减值根据阴影强度进行插值：控制阴影的明暗程度
    // 例如：shadowStrength=0.5时，将衰减从[0,1]映射到[0.5,1]，使阴影变淡
    attenuation = LerpWhiteTo(attenuation, shadowStrength);

    // ========== 阴影边界检查 ==========
    // 检查当前阴影坐标是否超出光源的视锥体范围
    // 超出范围的像素应返回1.0（无阴影），因为这些位置不受该光源的阴影影响
    // TODO: 此处可考虑使用分支指令来优化性能（在某些平台上）
    return BEYOND_SHADOW_FAR(shadowCoord) ? 1.0 : attenuation;
}
  • 分析：
    • 可以看到，其中的以 Filtered 结尾的方法，全部都是软阴影的采样方法。这部分代码的整体含义，就是根据当前的设置、选择是软阴影还是硬阴影来处理，然后再应用上阴影强度，最后返回；

// 函数：SampleShadowmapFiltered
// 作用：根据设定的软阴影质量，选择对应质量的滤波方法进行阴影采样。
// 参数：
//   TEXTURE2D_SHADOW_PARAM(ShadowMap, sampler_ShadowMap) - 阴影贴图及其采样器
//   float4 shadowCoord - 阴影坐标（用于在阴影贴图中定位和比较深度）
//   ShadowSamplingData samplingData - 阴影采样数据，包含滤波质量和贴图尺寸等信息
real SampleShadowmapFiltered(TEXTURE2D_SHADOW_PARAM(ShadowMap, sampler_ShadowMap), float4 shadowCoord, ShadowSamplingData samplingData)
{
    real attenuation = real(1.0); // 初始化阴影衰减值为1.0（表示完全不受阴影遮挡）

    // 根据设定的软阴影质量，选择不同精度的滤波函数
    if (samplingData.softShadowQuality == SOFT_SHADOW_QUALITY_LOW)
    {
        // 低质量滤波，通常使用较少的采样点，性能开销最小
        attenuation = SampleShadowmapFilteredLowQuality(TEXTURE2D_SHADOW_ARGS(ShadowMap, sampler_ShadowMap), shadowCoord, samplingData);
    }
    else if(samplingData.softShadowQuality == SOFT_SHADOW_QUALITY_MEDIUM)
    {
        // 中等质量滤波，在效果和性能间取得平衡
        attenuation = SampleShadowmapFilteredMediumQuality(TEXTURE2D_SHADOW_ARGS(ShadowMap, sampler_ShadowMap), shadowCoord, samplingData);
    }
    else // SOFT_SHADOW_QUALITY_HIGH
    {
        // 高质量滤波，提供最平滑的阴影边缘，但性能开销最大
        attenuation = SampleShadowmapFilteredHighQuality(TEXTURE2D_SHADOW_ARGS(ShadowMap, sampler_ShadowMap), shadowCoord, samplingData);
    }

    return attenuation; // 返回计算得到的阴影衰减值
}

// 函数：SampleShadowmapFilteredHighQuality
// 作用：通过7x7的Tent滤波器（帐篷滤波器）进行16个采样点的高质量软阴影计算。
// 原理：在阴影坐标周围一片区域内进行多次采样，并根据滤波核函数为每个采样点分配权重，最后加权平均。

                + fetchesWeights[6] * SAMPLE_TEXTURE2D_SHADOW(ShadowMap, sampler_ShadowMap, float3(fetchesUV[6].xy, shadowCoord.z))
                + fetchesWeights[7] * SAMPLE_TEXTURE2D_SHADOW(ShadowMap, sampler_ShadowMap, float3(fetchesUV[7].xy, shadowCoord.z))
                + fetchesWeights[8] * SAMPLE_TEXTURE2D_SHADOW(ShadowMap, sampler_ShadowMap, float3(fetchesUV[8].xy, shadowCoord.z))

                + fetchesWeights[15] * SAMPLE_TEXTURE2D_SHADOW(ShadowMap, sampler_ShadowMap, float3(fetchesUV[15].xy, shadowCoord.z));
}
  • 分析：
    • 核心原理是比较当前片元在光源空间中的深度值 (shadowCoord.z) 与阴影贴图（ShadowMap）中存储的最近深度值，如果当前片元深度更大（即更远），说明它在阴影中。硬阴影的边缘会出现锯齿，软阴影通过在阴影边缘周围进行多次采样并混合来模拟面光源产生的柔和阴影过渡。高质量滤波（如这里的 7x7 Tent 滤波）采样次数多，权重分布更合理，效果也更平滑，但性能代价更高；
    • 其中 shadowCoord 它的 xy 分量在阴影贴图中查找，而 z 分量是当前片元在光源空间下的深度，用于与贴图中的深度值进行比较；

  • 代码：LerpWhiteTo
// 函数定义：从"白色"（代表无阴影，值为1.0）线性插值到目标值b
// 参数b: 目标值，通常代表阴影的强度或颜色
// 参数t: 插值因子，通常来自阴影贴图采样结果或阴影强度系数
real LerpWhiteTo(real b, real t)  
{  
    // 计算（1 - t），即偏向"白色"（无阴影）部分的权重
    real oneMinusT = 1.0 - t;  
    // 返回混合结果： (1 - t) * 1 + b * t
    return oneMinusT + b * t;  
}
  • 分析：
    • 这个函数本质上是标准线性插值（Lerp）公式 lerp(a, b, t) = a * (1 - t) + b * t 的一个特化和优化形式，在这里起始值 a 被固定为 1.0；
    • 在图形学，特别是阴影计算中，1.0 通常表示“白色”或“无遮挡”（完全光照），而参数 b 则代表阴影的强度（例如，b 为 0 表示全黑，即完全阴影；b 为 0.5 表示半影区域）
