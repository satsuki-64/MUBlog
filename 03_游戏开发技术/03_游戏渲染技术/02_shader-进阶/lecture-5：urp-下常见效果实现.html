纹理与采样器的分离定义

基础概念

概念：基础概念

  • 概念：
    • URP 将纹理和采样器分离定义，主要基于以下几点考量：
    • （1）分离后，​多个不同的纹理可以共享同一个采样器状态​（如相同的过滤模式和寻址模式）。这在需要统一采样规则的场景（如大量使用相同材质属性的物体）中非常有用，避免了重复定义采样器的开销；
    • （2）渲染时，切换采样器状态可能带来性能开销。通过共享采样器，GPU 在渲染过程中可以减少采样器状态的切换次数，从而提升渲染效率；
    • （3）现代图形 API（如 Vulkan、DirectX 12）更倾向于显式且精细地控制资源。纹理与采样器分离的定义方式更符合这些 API 的设计哲学，有助于降低代码的耦合度，并提供更直接的控制；
  • 分析：URP 和内置渲染管线在纹理采样定义上的主要区别
  • 代码：示例
// 在Properties块中声明，便于在材质面板上赋值
_MainTex("Main Tex", 2D) = "white" {}

CBUFFER_START(UnityPerMaterial)
	fixed4 _Color;
CBUFFER_END

// 在HLSL代码块中
// 1. 定义纹理
TEXTURE2D(_MainTex);        // 声明纹理
float4 _MainTex_ST;         // 声明纹理的缩放(Scale)和平移(Offset)变量，用于TRANSFORM_TEX

// 2. 定义采样器
// 2.1 方式一
// 声明一个使用纹理自身导入设置的采样器
// 注意：这里的名称不能写错，必须是 sampler_MainTex，不然采样器会工作失效
SAMPLER(sampler_MainTex);   

// 2.2 方式二
// 直接自定义一个采样器状态（例如，点过滤且重复寻址）
// 重点是后背的“Point_Repeat”
#define smp sampler_Point_Repeat
SAMPLER(smp);
// 还可以定义更复杂的，比如sampler_clampU_mirrorV



// 3. 在片元着色器中进行采样
// 使用默认采样器
half4 color = SAMPLE_TEXTURE2D(_MainTex, sampler_MainTex, uv);
color = color * _Color;

// 使用自定义采样器
half4 color2 = SAMPLE_TEXTURE2D(_MainTex, SAMPLER_POINT_REPEAT, uv);

什么是采样器

概念：什么是采样器

  • 概念：
    • ​采样器定义了如何从纹理中获取颜色信息的一套规则，而纹理本身只是一张图片数据；
    • 采样器是一个独立对象，在 URP 中，纹理和采样器是分开定义的。​多个不同的纹理可以共享同一个采样器，这有助于优化性能；
  • 功能：
    • 采样器主要定义以下两个功能：
      1. ​过滤模式 (Filter Mode)​​：
      • 决定如何混合纹理像素（纹素）以生成平滑（线性）或锐利（点）的外观；
      1. 寻址模式 (Wrap Mode)​​：
      • 定义当 UV 坐标超出 [0, 1] 范围时如何采样，如重复、钳制等；
  • 概念：SAMPLER(samplerName)
    • 对于两种模式：
      • filter：point/linear/triLinear；
      • wrap：clamp/repeat/mirror
  • 图示：不同的过滤模式
    • Blinear
    • no filter

URP 下常用参数

概念：Camera 相关参数

  • 概念：相机位置 - _WorldSpaceCameraPos
    • float3 类型的向量，提供了相机在世界空间中的坐标位置；
    • 这是最常用的相机变量之一，常用于计算视角方向、高光反射以及基于距离的效果；
  • 概念：计算相机前方方向
    • 方法：
      • -1 * mul(UNITY_MATRIX_M, transpose(mul(UNITY_MATRIX_I_M, UNITY_MATRIX_I_V)) [2].xyz)
    • 解释：
      • 用于计算相机正前方（Forward）向量在世界空间中的方向，其原理是从视图空间的 Z 轴方向（即相机前方）通过一系列矩阵逆变换和转置，最终转换到世界空间；
  • 概念：相机投影类型判断
    • 变量：
      • unity_OrthoParams.w
    • 解释：
      • 其值为 1.0 时，表示当前相机是正交投影；
      • 值为 0.0 时，表示是透视投影；
      • 在 Shader 中需要根据投影类型进行不同计算时（如深度处理）会用到此变量；
  • 概念：近裁剪面距离
    • 变量：
      • _ProjectionParams.y
    • 解释：
      • 一个 float 值，表示相机近裁剪面到相机的距离。任何比这个距离更近的物体都将被裁剪，不会显示；

带贴图映射与雾效的模板

Shader "Unlit/SimplesUnlit"
{
    Properties
    {
        _BaseColor("BaseColor",Color) = (1,1,1,1)
        _BaseMap("BaseMap",2D) = "white"{}
    }
    
    SubShader
    {
        Tags
        {
            "RenderType"="Opaque"
            "RenderPipeline" = "UniversalPipeline"
            "IgnoreProjector" = "True"
        }
        LOD 100

        Pass
        {
            Name "SimplesUnlit"
            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #pragma multi_compile_fog
            #include "Packages/com.unity.render-pipelines.core/ShaderLibrary/Color.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Core.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Lighting.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/ShaderGraphFunctions.hlsl"

            struct Attributes
            {
                float4 positionOS : POSITION;
                float2 uv : TEXCOORD0;
            };

            struct Varyings
            {
                float4 positionCS : SV_POSITION;
                float2 uv : TEXCOORD0;
                float fogCoord : TEXCOORD1;
            };

			// 变量定义在 CBuffer 当中
            CBUFFER_START(UnityPerMaterial)
            half4 _BaseColor;
            CBUFFER_END
            TEXTURE2D(_BaseMap);SAMPLER(sampler_BaseMap);float4 _BaseMap_ST;

            Varyings vert (Attributes v)
            {
                Varyings o;
                o.positionCS = TransformObjectToHClip(v.positionOS);
                o.uv = TRANSFORM_TEX(v.uv,_BaseMap);
                o.fogCoord = ComputeFogFactor(o.positionCS.z);

                return o;
            }

            half4 frag (Varyings i) : SV_Target
            {
                half4 color;
                half4 baseMap = SAMPLE_TEXTURE2D(_BaseMap,sampler_BaseMap,i.uv);
                color = baseMap * _BaseColor;
                color.rgb = MixFog(color.rgb,i.fogCoord);
                
                return color;
            }
            
            ENDHLSL
        }
    }
}

URP 下实现菲尼尔效果

Shader "Unlit/Examples/Ghost"
{
    Properties
    {
        _Color("Color",Color) = (1,0,0,1)
        _Fresnel("Pow(X) Offset(Y) Top(Z)",Vector) = (1,1,1,1)
    }
    
    SubShader
    {
        Tags
        {
            // 1. 将渲染类型改为透明，队列设置为透明队列
            "RenderType"="Transparent"
            "Queue"="Transparent" // 确保在透明阶段渲染，不增加此项会导致透明物体的渲染被不透明物体遮挡
            "RenderPipeline" = "UniversalPipeline"
        }
        LOD 100
        
        // 2. 推荐使用更适合半透明效果的混合模式
        // 标准的Alpha混合
        // Blend SrcAlpha OneMinusSrcAlpha
        // 如果你确实需要明亮的叠加效果，可以保留 Blend One One，但需注意其特性
        Blend One One
        // Blend One Zero
        
        // 3. 关闭深度写入，防止半透明部分出现深度排序问题[7](@ref)
        ZWrite Off

        Pass
        {
            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag

            // URP渲染管线下通常都要引用的hlsl文件
            #include "Packages/com.unity.render-pipelines.core/ShaderLibrary/Color.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Core.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Lighting.hlsl"

            struct Attributes
            {
                float4 vertexOS : POSITION;
                
                // 模型在本地空间下的法线坐标
                half3 normalOS : NORMAL;
            };

            struct Varyings
            {
                // 模型在世界空间下的法线坐标
                half3 normalWS : TEXCOORD0;

                // 世界空间
                float3 positionWS : TEXCOORD1;

                // 模型空间
                float4 vertexOS : TEXCOORD2;

                // 裁剪空间
                float4 vertexCS : SV_POSITION;
            };

            CBUFFER_START(UnityPerMaterial)
                float4 _Color;
                float4 _Fresnel;
            CBUFFER_END

            Varyings vert (Attributes v)
            {
                Varyings o;

                // 从模型空间变换到世界空间的法线
                o.normalWS = TransformObjectToWorldNormal(v.normalOS);
                
                // 从模型空间变换到裁剪空间
                o.vertexCS = TransformObjectToHClip(v.vertexOS);

                // 从模型空间变换到世界空间
                o.positionWS = TransformObjectToWorld(v.vertexOS);

                o.vertexOS = v.vertexOS;
                return o;
            }

            // 注意：这里的返回值一定要是half4类型！不要写成half3类型
            half4 frag (Varyings i) : SV_Target
            {
                half3 N = i.normalWS;
                half3 V = normalize(_WorldSpaceCameraPos - i.positionWS);
                float f = (1 - max(0,dot(N,V)));
                half fresnel = pow(f,_Fresnel.x);
                float4 color = half4(fresnel * _Color.rgb, fresnel * _Color.a);  

                // 创建出从上到下的黑白遮罩
                half mask = max(0,i.vertexOS.y + i.vertexOS.z + _Fresnel.y);
                float4 ColorNew = lerp(color, _Color, mask*_Fresnel.z);
                
                return ColorNew * mask;
            }
            
            ENDHLSL
        }
    }
}

概念：URP 下实现顶点偏移

  • 概念：实现基础的偏移效果
    • 代码：
      • v.vertexOS.x += sin(_Time.y);
    • 分析：
      • 在顶点着色器当中，直接更改当前模型的本地坐标，实现顶点偏移；
      • 注意此代码应当在其他顶点变换之前：
  • 概念：根据 Y 轴位置决定偏移大小
    • 代码：
      • v.vertexOS.x += sin(_Time.y + v.vertexOS.y);
    • 分析：
      • 当前模型的位置越高，其偏移范围越大；
  • 概念：分别对 X 轴和 Z 轴进行偏移
    • 代码：
      • v.vertexOS.x += sin((_Time.y + v.vertexOS.y)*_OffsetRange.x) * _OffsetRange.y;
      • v.vertexOS.z += sin((_Time.y + v.vertexOS.y)*_OffsetRange.z) * _OffsetRange.w;
    • 分析：
      • 分别对 X 轴和 Z 轴进行偏移，同时使用 _OffsetRange 变量来控制偏移的速度以及强度；
  • 代码：

            {

                half mask = max(0,i.vertexOS.y + i.vertexOS.z + _Fresnel.y);
                float4 ColorNew = lerp(color, _Color, mask*_Fresnel.z);

                // 4. 混合雾效颜色和原颜色
                ColorNew.rgb = MixFog(ColorNew,i.fogCoord);
                
                return ColorNew * mask;
            }
            
            ENDHLSL
        }
    }
}

雾效的实现：混合雾效颜色

分析
[list2node]

  • 第一步：MixFog
    • 传进啦 ide 第一个颜色是物体本身的颜色，第二个参数是雾效因子；
    • 传入的第二个参数，表示 Unity 当中的雾的颜色；
  • 第二步：MixFogColor
    • 如果定义了三种雾类型其中的任意一个，则往下执行：
    • 如果开启了雾，则开始计算雾效强度 ComputeFogIntensity；
  • 第三步：ComputeFogIntensity
    • 首先定义雾的强度为 0，然后根据不同的模式，分别按照雾效因子，计算雾效强度；
    • 可以看到，如果是线性雾，则当前不对雾效因子进行处理，直接返回雾效因子；
    • 如果是指数雾，则需要要将雾效因子进行指数处理。并且使用 saturate 来确保结果在 [0,1] 的范围内；
  • 第四步：混合颜色
    • fragColor = lerp(fogColor, fragColor, fogIntensity);
    • 第一个值，是雾的颜色，第二个值是物体本身的颜色，因此 fogIntensity 越大、雾效越弱；
    • 当物体离相机越来越远时，显示雾的颜色，越来越近时显示物体自己的颜色；
