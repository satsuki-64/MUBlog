Lecture 11：URP 下法线与法线纹理

法线基础

面法线和顶点法线基础概念

概念：面法线和顶点法线

  • 概念：
    • 面法线是垂直于多边形平面的向量，用于描述多边形所面对的方向。在 3D 图形中，每个三角形面都有一个面法线，可以通过三个顶点的位置计算得出；
    • 顶点法线是与顶点相关联的法线向量，用于确定光线照射到表面的角度。在光照计算中，Direct3D 或 OpenGL 需要知道每个顶点的表面方向（法线）
    • 在 Unity、UE 等软件当中，主要使用顶点法线，面法线主要在 DCC 软件当中使用；
  • 概念：从面法线到顶点法线
    • 顶点法线可以通过计算共享该顶点的所有面法线的平均值来获得。具体算法如下：
    • （1）遍历每个顶点，找到共享该顶点的所有三角形；
    • （2）计算每个三角形的面法线：对于三角形 ABC，计算向量 AB 和 AC，然后通过叉乘运算得到面法线；
    • （3）累加面法线：将所有共享该顶点的面法线相加；
    • （4）归一化处理：将累加结果除以面数，然后进行归一化处理；

Unity 中的顶点索引 SV_VertexID

概念：顶点索引 SV_VertexID

  • 概念：什么是顶点索引
    • SV_VertexID 是 Unity Shader 中的一个系统值语义，表示正在处理的顶点的索引；
    • 它是一个无符号整数 (uint)类型的变量，从 0 开始依次递增，用于标识当前正在顶点着色器中处理的顶点是第几个顶点；
    • SV_VertexID 在顶点动画中非常有用，可以对指定顶点进行采样。通过顶点 ID，开发者可以实现基于顶点索引的特殊效果，如对特定顶点进行位移、旋转或缩放，实现波浪、扭曲等动态效果，或者创建基于顶点索引的颜色变化；

切线与副切线

切线与副切线基础概念

概念：切线与副切线

  • 概念：法线和切线
    • 通过引入纹理坐标（UV）​ 来确定一条在特定规则下唯一的切线；
    • 在 3D 图形学中，我们之所以要确定唯一的切线，是为了构建一个位于模型表面的局部坐标系，也就是切线空间（Tangent Space）。这个空间对于正确使用法线贴图等技术至关重要；
    • 而切线通常与纹理坐标的 U 轴（即水平方向）对齐，作为坐标系的 X 轴；

VertexNormalInputs GetVertexNormalInputs(float3 normalOS)
{
    VertexNormalInputs tbn;

    // 初始化切线为世界空间的X轴正方向 (1, 0, 0)
    tbn.tangentWS = real3(1.0, 0.0, 0.0);
    // 初始化副切线为世界空间的Y轴正方向 (0, 1, 0)
    tbn.bitangentWS = real3(0.0, 1.0, 0.0);
    // 将物体空间法线变换到世界空间，并归一化（在TransformObjectToWorldNormal内部通常已处理）
    tbn.normalWS = TransformObjectToWorldNormal(normalOS);

    return tbn;
}

// 当模型提供切线数据时，使用此函数构建正确的切线空间
VertexNormalInputs GetVertexNormalInputs(float3 normalOS, float4 tangentOS)
{
    VertexNormalInputs tbn;

    // 注意：此实现遵循 MikkTSpace 标准。
    // 提示：为了符合该标准，在顶点着色器中仅进行变换，将归一化操作留到片元着色器中进行
    
    // 计算副切线方向的符号
    // tangentOS.w 的值存储的是副切线的方向，-1或者1
    // 其是一个符号分量（通常为+1或-1），用于指示切线空间的手性（handedness）。
    // GetOddNegativeScale() 是一个Unity内部函数，通常用于处理某些变换（如非均匀缩放）可能导致的镜像反射。
    // 最终的符号(sign)用于确保副切线方向正确，以处理纹理镜像等情况[2](@ref)。
    real sign = real(tangentOS.w) * GetOddNegativeScale();

    // 将物体空间法线变换到世界空间
    tbn.normalWS = TransformObjectToWorldNormal(normalOS);
    // 将物体空间切线变换到世界空间
    tbn.tangentWS = real3(TransformObjectToWorldDir(tangentOS.xyz));
    
    // 计算世界空间的副切线。
    // 通过法线和切线的叉积得到副切线的方向。
    // 然后乘以之前计算出的符号(sign)，以确保整个切线空间（TBN矩阵）的方向正确。
    // 这个符号对于处理UV镜像至关重要，它能修正因纹理镜像导致的副切线方向翻转
    tbn.bitangentWS = real3(cross(tbn.normalWS, float3(tbn.tangentWS))) * sign;

    return tbn;
}

real GetOddNegativeScale()
{
    // FIXME: We should be able to just return unity_WorldTransformParams.w, but it is not
    // properly set at the moment, when doing ray-tracing; once this has been fixed in cpp,
    // we can revert back to the former implementation.
    return unity_WorldTransformParams.w >= 0.0 ? 1.0 : -1.0;
}

/// <summary>
/// 将法线向量从模型空间（Object Space）变换到世界空间（World Space）。
/// 此函数会正确处理非均匀缩放，确保变换后的法线仍与模型表面垂直。
/// </summary>
/// <param name="normalOS">模型空间中的法线向量（通常是单位向量）。</param>
/// <param name="doNormalize">是否对结果进行归一化。建议保持为true，以确保变换后的法线仍是单位向量。</param>
/// <returns>世界空间中的法线向量。</returns>
float3 TransformObjectToWorldNormal(float3 normalOS, bool doNormalize = true)
{
    // 条件编译：检查是否启用了“假设统一缩放”优化宏（UNITY_ASSUME_UNIFORM_SCALING）
    // 若启用，表示可以安全地假设模型只经历了统一缩放（各轴向缩放比例相同）
    #ifdef UNITY_ASSUME_UNIFORM_SCALING
        // 在统一缩放情况下，法线变换可直接使用方向变换函数，因为缩放不会改变法线的垂直性
        return TransformObjectToWorldDir(normalOS, doNormalize);
    #else
        // 对于非均匀缩放，必须使用模型变换矩阵的"逆转置矩阵"来正确变换法线。
        // 原因：非均匀缩放会扭曲模型表面，若直接用原变换矩阵变换法线，结果将不再垂直于表面。
        // 逆转置矩阵（即世界到模型空间矩阵的3x3部分）能修正这种扭曲。
        // 数学原理：normalWS = normalOS * (M^{-1})^T，其中M是模型变换矩阵。
        // 这里通过右乘世界到模型空间的矩阵（GetWorldToObjectMatrix的3x3部分）来实现。
        float3 normalWS = mul(normalOS, (float3x3)GetWorldToObjectMatrix());

        // 由于变换过程可能改变向量长度，通常需要重新归一化以保持其为单位向量。
        // 这在后续的光照计算（如点积）中至关重要。
        if (doNormalize)
            return SafeNormalize(normalWS); // 使用安全的归一化函数，避免零向量导致除零错误

        // 如果明确不需要归一化（例如出于性能考虑，且能保证输入是单位向量且变换不影响长度），则直接返回
        return normalWS;
    #endif
}

/// <summary>
/// 将方向向量（如光线方向、切线方向）从模型空间变换到世界空间。
/// 注意：此函数适用于方向向量，不适用于法线（法线在非均匀缩放下需特殊处理）。
/// </summary>
/// <param name="dirOS">模型空间中的方向向量（例如光线方向或切线向量）。</param>
/// <param name="doNormalize">是否对结果进行归一化。建议为true，以确保方向向量为单位长度。</param>
/// <returns>世界空间中的方向向量。</returns>
float3 TransformObjectToWorldDir(float3 dirOS, bool doNormalize = true)
{
    // 条件编译：处理光线追踪（Ray Tracing）着色器阶段与非光线追踪阶段的矩阵获取差异
    #ifndef SHADER_STAGE_RAY_TRACING
        // 非光线追踪阶段：使用标准的模型到世界矩阵（GetObjectToWorldMatrix）的3x3部分进行变换。
        // 方向向量的w分量为0，因此只需3x3矩阵（忽略平移影响）。
        float3 dirWS = mul((float3x3)GetObjectToWorldMatrix(), dirOS);
    #else
        // 光线追踪阶段：使用特定于光线追踪的模型变换矩阵（ObjectToWorld3x4）。
        float3 dirWS = mul((float3x3)ObjectToWorld3x4(), dirOS);
    #endif

    // 变换后归一化，以修正缩放可能导致的向量长度变化
    if (doNormalize)
        return SafeNormalize(dirWS);

    return dirWS;
}

/// <summary>
/// 安全的向量归一化函数。避免在输入向量长度为零或极小时出现除零错误或浮点数精度问题。
/// 通过计算平方根的倒数（rsqrt）进行优化，比直接使用sqrt和除法效率更高。
/// </summary>
/// <param name="inVec">需要归一化的输入向量。</param>
/// <returns>归一化后的单位向量。如果输入向量长度接近零，则返回零向量或极小值向量。</returns>
float3 SafeNormalize(float3 inVec)
{
    // 计算输入向量的点积（即长度的平方）
    float dp3 = dot(inVec, inVec);

    // 使用max函数确保分母不为零：如果dp3小于FLT_MIN（一个极小的正数），则取FLT_MIN。
    // 这避免了当输入向量长度接近零时，rsqrt(0)导致的无限大或NaN问题。
    dp3 = max(FLT_MIN, dp3);

    // 计算归一化结果：向量乘以长度的平方根的倒数（rsqrt）。
    // rsqrt(dp3) 等价于 1.0 / sqrt(dp3)，但计算效率更高。
    return inVec * rsqrt(dp3);
}

法线格式

概念：法线格式

  • 概念：
    • 法线贴图是一种通过修改表面法线方向来模拟凹凸细节的纹理技术，无需增加模型几何复杂度。在切线空间中，法线向量 (x,y,z) 的取值范围为 [-1,1]，通过 RGB=(Normal+1)/2 转换为 [0,1] 的颜色值存储到纹理中；
    • 由于法线贴图存储的是向量信息而非颜色，传统的纹理压缩方法会导致质量损失。因此需要专门的压缩格式来优化存储和渲染性能；
  • 概念：DXT5nm 格式
    • 格式：
    • 概念：
      • DXT5nm 格式将法线向量的 X 分量存储在 Alpha 通道，Y 分量存储在 Green 通道，Z 分量通过计算得出；
      • 这种存储方式利用了法线是单位向量的特性：Z = sqrt(1 - X² - Y²)；
      • DXT5 格式中，A 通道获得 8 位精度，G 通道获得 6 位精度，R 和 B 通道分别获得 5 位精度。相比未压缩的 8 位 RGB 纹理，获得3:1的压缩率；
    • 实现：Unity 当中的解码函数
  • 概念：BC5 格式
    • 格式：
    • 概念：
      • BC5（也称为 3Dc）是专门为法线贴图设计的压缩格式，将法线的 X 和 Y 分量分别存储在 RG 通道中，Z 分量同样通过计算得出；
      • BC5 格式专门针对双通道数据优化，每个通道获得独立的高质量压缩，相比 DXT5nm，BC5 提供更好的压缩质量，特别是在高动态范围的法线贴图上；

总结：使用切线空间下的法线
    1. 定义顶点属性：在着色器的顶点输入中，定义包含法线和切线信息的属性，通常使用 NORMAL 和 TANGENT 语义。
    1. 声明中间变量：在顶点着色器和片段着色器之间传递的变量中，声明三个变量（normalWS, tangentWS, bitangentWS），用于在后续步骤中构建从切线空间到世界空间的旋转矩阵。
    1. 在顶点着色器中构建切线空间：
    • 将物体空间法线转换为世界空间法线；
    • 将物体空间切线转换为世界空间切线；
    • 通过法线和切线的叉积并乘以切线的 w 分量符号，计算出世界空间的副切线，从而共同构成完整的 TBN 矩阵。
    1. 在片段着色中转换法线：
    • 从法线贴图中采样并解压得到切线空间下的法线向量；
    • 使用在顶点着色器中计算好的 TBN 矩阵，将法线从切线空间变换到世界空间，以便进行光照等计算。

Shader "Unlit/ParallaxMap"
{
    Properties
    {
        _BaseColor("BaseColor",Color) = (1,1,1,1)
        [MainTexture]_BaseMap("BaseMap",2D) = "white"{}
        [Normal]_NormalMap("NormalMap",2D) = "bump"{}
    }
    
    SubShader
    {
        Tags
        {
            "RenderType"="Opaque"
            "RenderPipeline" = "UniversalPipeline"
            "IgnoreProjector" = "True"
        }
        LOD 100

        Pass
        {
            Name "SimplesUnlit"
            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #pragma multi_compile_fog
            #include "Packages/com.unity.render-pipelines.core/ShaderLibrary/Color.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Core.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Lighting.hlsl"
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/ShaderGraphFunctions.hlsl"

            struct Attributes
            {
                float4 positionOS : POSITION;
                float2 uv : TEXCOORD0;
                half3 normalOS : NORMAL;
                half4 tangentOS : TANGENT;
            };

            struct Varyings
            {
                float4 positionCS : SV_POSITION;
                float2 uv : TEXCOORD0;
                float fogCoord : TEXCOORD1;
                half4 normalWS : TEXCOORD2;
                half4 tangentWS : TEXCOORD3;
                half4 bitangentWS : TEXCOORD4;
            };
            
            CBUFFER_START(UnityPerMaterial)
	            half4 _BaseColor;
            CBUFFER_END
            TEXTURE2D(_BaseMap);SAMPLER(sampler_BaseMap);float4 _BaseMap_ST;
            TEXTURE2D(_NormalMap);SAMPLER(sampler_NormalMap);

            Varyings vert (Attributes v)
            {
                Varyings o;
                o.positionCS = TransformObjectToHClip(v.positionOS);
                o.uv = TRANSFORM_TEX(v.uv,_BaseMap);
                o.fogCoord = ComputeFogFactor(o.positionCS.z);
                
                o.normalWS.xyz = TransformObjectToWorldNormal(v.normalOS);
                o.tangentWS.xyz = TransformObjectToWorldDir(v.tangentOS.xyz);
                half sign = v.tangentOS.w * GetOddNegativeScale();
                o.bitangentWS.xyz = cross(o.normalWS,o.tangentWS) * sign;
                
                return o;
            }

            half4 frag (Varyings i) : SV_Target
            {
                half4 color;
                half4 baseMap = SAMPLE_TEXTURE2D(_BaseMap,sampler_BaseMap,i.uv);
                color = baseMap * _BaseColor;
                color.rgb = MixFog(color.rgb,i.fogCoord);

                half3 normalMap = UnpackNormal(SAMPLE_TEXTURE2D(_NormalMap,sampler_NormalMap,i.uv));
                half3 normalWS = mul(normalMap,half3x3(i.tangentWS.xyz,i.bitangentWS.xyz,i.normalWS.xyz));
                
                return color;
            }
            
            ENDHLSL
        }
    }
}
