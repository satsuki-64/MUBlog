Lecture 12：默认实参与调试帮助

默认实参

概念：默认实参

  • 概念：
    • 某些函数有这样一种形参，在函数的很多次调用中它们都被赋予一个相同的值，此时我们把这个反复出现的值称为函数的默认实参 (default argument)；
    • 调用含有默认实参的函数时，可以包含该实参，也可以省略该实参；
    • 当设计含有默认实参的函数时，其中一项任务是合理设置形参的顺序，尽量让不怎么使用默认值的形参出现在前面，而让那些经常使用默认值的形参出现在后面；
  • 注意：
    • 一旦某个形参被赋予了默认值，它后面的所有形参都必须有默认值；
  • 示例：默认实参的使用：
    • typedef string::size_type sz;
    • string s1(sz ht=24,sz wid = 80,char back=' ');
  • 概念：默认实参初始化
    • 局部变量不能作为默认实参；
    • 除此之外，只要表达式的类型能转换成形参所需的类型，该表达式就能作为默认实参；
  • 示例：默认实参初始化
class Function

{

public:

    typedef string::size_type sz;

    sz ht();

    sz wd = 64;

    char t = 'w';

    string szeTest(sz = ht(),sz = wd,char = t);

};

调试帮助

assert 预处理宏

概念：assert 预处理宏

  • 概念：
    • 预处理宏其实是一个预处理变量，它的行为有点类似于内联函数；
    • 首先对 expr 求值，如果表达式为假 (即 0)，assert 输出信息并终止程序的执行；
    • 如果表达式为真 (即非 0)，assert 什么也不做；
  • 注意：
      1. assert 宏定义在 cassert 头文件中；
      1. 应该使用 assert 而不是 std::assert，也不需要为 assert 提供 using 声明；
  • 分析：assert 宏常用于检查“不能发生”的条件
    • assert(word.size() > threshold)

NDEBUG 预处理变量

概念：NDEBUG 预处理变量与 assert

  • 概念：
    • assert 的行为依赖于一个名为 NDEBUG 的预处理变量的状态；
    • 如果定义了 NDEBUG，则 assert 什么也不做，默认状态下没有定义 NDEBUG，此时 assert 将执行运行时检查；
  • 使用：
    • 我们可以使用一个 #define 定义 NDEBUG ，从而关闭调试状态；

概念：NDEBUG 预处理变量与代码

  • 概念：
    • 除了用于 assert 外，也可以使用 NDEBUG 编写自己的条件调试代码；
    • 如果 NDEBUG 未定义，将执行 #ifndef 和 #endif 之间的代码；
    • 如果定义了 NDEBUG，这些代码将被忽略掉；
  • 举例：
void print()
{
#ifndef NDEBUG

	cout << __func__ << ":测试" << endl;

#endif
}
  • 分类：常用变量
    • __FILE__：当前文件名的字符串字面值；
    • __LINE__：当前行号；
    • __TIME__：编译时间；
    • __DATA__：日期
    • __func__：当前函数名；
  • 举例：
cout<<"Error:"<<__FILE__<<":in function"<<__func__;
