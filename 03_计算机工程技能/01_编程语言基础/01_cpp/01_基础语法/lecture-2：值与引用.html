Lecture 2：值与引用

引用传递和值传递

概念：基础概念

  • 概念：形式参数
    • 在函数声明 / 定义时，写在函数名后的括号内的参数，是用来接收调用时传入数据的 “占位符”；
    • 形参是函数内部的局部变量，仅在函数被调用时分配内存，函数执行结束后立即释放，作用域仅限函数内部；
  • 概念：实际参数
    • 在函数调用时，写在函数名后的括号内的具体值 / 变量 / 表达式，是传递给形参的 “实际数据”；
    • 实参是调用函数的上下文中已存在的变量 / 常量 / 表达式，其值（或地址）会传递给形参，实参的内存不受函数调用的影响（除非传引用 / 指针修改其值）；
  • 示例：
#include <iostream>
using namespace std;

// 形参是引用（&num）：相当于实参的“别名”
void Increase(int& num) { 
    num++; // 修改形参（实际是修改实参本身）
}

int main() 
{
    int val = 5;
    
    // 实参是val，传递给引用形参num
    Increase(val); 
    cout << val << endl; // 输出6（实参被修改）
    return 0;
}

引用传递和值传递区别

概念：引用传递和值传递区别

  • 概念：值传递
    • 值传递的是一个值的副本，形参是实参的拷贝，改变形参的值并不会影响外部实参的值。如果是一个函数中传入值，对形参的操作不会影响实参的值；
  • 概念：引用传递
    • 概念：
      • 被调函数的形参是「原变量的别名」（编译器层面的绑定）；虽然栈上也会存实参地址，但编译器强制让这个形参和原变量 “绑定死”—— 语法上你无法修改这个地址，操作引用就等价于直接操作原变量；
      • 对形参的操作其实就是对实参的操作，在引用传递过程中，被调函数的形式参数虽然也作为局部变量在栈中开辟了内存空间，但是这时指针指向的位置是由函数放进来的实参变量的地址。被调函数对形参的任何操作都被处理成间接寻址，即通过栈中存放的地址访问主调函数中的实参变量；
      • 正因为如此，被调函数对形参做的任何操作都影响了主调函数中的实参变量。
      • 即：引用传递传递的是引用对象的内存地址，函数对形参的操作会影响实参的值，实参的值会随着形参的值得改变而改变；
    • 总结：
      • 引用是一个别名，它在逻辑上不是独立的，它的存在具有依附性，所以引用必须在一开始就被初始化，而且其引用的对象在其整个生命周期中是不能被改变的（自始至终只能依附于同一个变量）；
  • 概念：指针传递
    • 概念：
      • 指针传递参数本质上是值传递的方式，它所传递的是一个地址值；
      • 指针的本质：指针变量本质上就是一个整型数值，其数值的含义是内存地址；
      • 值传递过程中，被调函数的形式参数作为被调函数的局部变量处理，即在栈中开辟了内存空间以存放由主调函数放进来的实参的值，从而成为了实参的一个副本。值传递的特点是被调函数对形式参数的任何操作都是作为局部变量进行，不会影响主调函数的实参变量的值；
    • 总结：
      • 指针从本质上讲就是存放变量地址的一个变量，在逻辑上是独立的，它可以被改变，包括其所指向的地址的改变和其指向的地址中所存放的数据的改变；
  • 补充：指针和变量的自增自减有什么不同？
    • 变量的自增自减是改变变量的值，指针的自增自减是改变指针的指向地址；

概念：指针和引用的区别

  • 概念：
    • 指针和引用经常用于函数的参数传递，然而，指针传递参数和引用传递参数是有本质上的不同的；
    • 指针传递参数本质上是值传递的方式，它所传递的是一个地址值。值传递过程中，被调函数的形式参数作为被调函数的局部变量处理，即在栈中开辟了内存空间以存放由主调函数放进来的实参的值，从而成为了实参的一个副本。值传递的特点是被调函数对形式参数的任何操作都是作为局部变量进行，不会影响主调函数的实参变量的值；
    • 而在引用传递过程中，被调函数的形式参数虽然也作为局部变量在栈中开辟了内存空间，但是这时存放的是由主调函数放进来的实参变量的地址。被调函数对形参的任何操作都被处理成间接寻址，即通过栈中存放的地址访问主调函数中的实参变量。正因为如此，被调函数对形参做的任何操作都影响了主调函数中的实参变量；
    • 引用传递和指针传递是不同的，虽然它们都是在被调函数栈空间上的一个局部变量，但是任何对于引用参数的处理都会通过一个间接寻址的方式操作到主调函数中的相关变量。而对于指针传递的参数，如果改变被调函数中的指针地址，它将影响不到主调函数的相关变量。如果想通过指针参数传递来改变主调函数中的相关变量，那就得使用指向指针的指针，或者指针引用；
  • 概念：从编译的角度
    • 程序在编译时分别将指针和引用添加到符号表上，符号表上记录的是变量名及变量所对应地址。指针变量在符号表上对应的地址值为指针变量的地址值，而引用在符号表上对应的地址值为引用对象的地址值。符号表生成后就不会再改，因此指针可以改变其指向的对象（指针变量中的值可以改），而引用对象则不能修改；
  • 总结：
    • 相同点：
      • 指针和引用都是地址，指针指向一块内存，它的内容是所指内存的地址；而引用则是某块内存的别名；
    • 不同点：
      • （1）引用无需解引用，指针需要解引用；
      • （2）引用在定义时被初始化一次，之后不可变，指针可变；
      • （3）引用不能为空，指针可以为空；
      • （4）程序为指针变量分配内存区域，而引用不需要分配内存区域，所以指针自增操作是指针变量的自增，引用自增操作是变量值的自增；
      • （5）引用没有 const，指针有 const，const 的指针不可变；
      • （6）“sizeof 引用”得到的是所指向的变量 (对象)的大小，而“sizeof 指针”得到的是指针本身的大小；
      • （7）引用是类型安全的，而指针不是，引用比指针多了类型检查；
场景 选择 原因
修改实参且确保非空 引用 简洁、安全（无空指针风险）
参数可选（允许为空） 指针 可传nullptr表示无参数
动态改变指向 指针 引用无法重绑定
兼容 C 代码 指针 C 语言没有引用
传递大对象（避免拷贝） const 引用 const int& 安全且无拷贝

}

{
    int number = 10;
    std::cout << "Fun1():" << std::endl;
    Fun1(number);

    std::cout << "Fun2():" << std::endl;
    Fun2(number);

  • 分析：

struct 示例

概念：struct 示例

  • 代码：
#include <iostream>
using namespace std;

// 定义值类型结构体（包含大数组，拷贝成本极高）
struct List {
    int data[1000];
    int len;
    // 构造函数：初始化len为0
    List() : len(0) {}
};

// 1. 指针传递：形参是List*（存储原结构体的地址副本）
void ModifyListByPtr(List* listPtr) {
    // 必须用->操作符访问成员（指针解引用+成员访问）
    if (listPtr != nullptr) { // 必须判空，否则崩溃
        listPtr->len = 10;    // 修改原结构体的len
        listPtr->data[0] = 100; // 修改原结构体的数组
        // 指针可重指向其他List对象（仅修改形参指针，不影响主调函数）
        List temp;
        listPtr = &temp; 
        listPtr->len = 20; // 仅修改temp，和原结构体无关
    }
}

// 2. 引用传递：形参是List&（原结构体的别名）
void ModifyListByRef(List& listRef) {
    // 直接用.操作符访问成员（引用是别名，等价于操作原结构体）
    listRef.len = 20;       // 修改原结构体的len
    listRef.data[0] = 200;  // 修改原结构体的数组
    // 错误：引用无法重绑定到其他对象（编译器强制约束）
    // List temp;
    // listRef = temp; // 这不是重绑定，是把temp的值拷贝给原结构体！
}

int main() {
    List myList; // 定义原结构体（值类型，存储在栈上）
    cout << "修改前：len=" << myList.len << ", data[0]=" << myList.data[0] << endl;

    // 指针传递调用：必须传地址（&myList）
    ModifyListByPtr(&myList);
    cout << "指针传递后：len=" << myList.len << ", data[0]=" << myList.data[0] << endl; // len=10, data[0]=100

    // 引用传递调用：直接传变量名（myList）
    ModifyListByRef(myList);
    cout << "引用传递后：len=" << myList.len << ", data[0]=" << myList.data[0] << endl; // len=20, data[0]=200

    return 0;
}
  • 总结：
对比维度 指针传递（List*） 引用传递（List&）
成员访问方式 必须用->（如listPtr->len） 直接用.（如listRef.len），更符合值类型的操作习惯
空值处理 允许传nullptr（需手动判空，否则->操作崩溃） 必须初始化（只能传合法的 List 对象），无空引用，编译器强制非空
重绑定能力 形参指针可重指向其他 List 对象（如listPtr = &temp），仅修改副本，不影响原结构体 一旦绑定原结构体，无法重指向其他对象；若写listRef = temp，是值拷贝（把 temp 的值覆盖原结构体），而非重绑定
拷贝开销 仅传递地址（4/8 字节），无结构体拷贝 仅传递地址（4/8 字节），无结构体拷贝
语法安全性 易出现空指针、野指针崩溃（如传 nullptr 未判空） 更安全：编译器保证引用指向有效对象，无空指针风险
函数调用写法 必须传地址（ModifyListByPtr(&myList)） 直接传变量名（ModifyListByRef(myList)），更简洁
const 适配 const List*：需(*listPtr).len或listPtr->len（只读） const List&：直接listRef.len（只读），语法更简洁（推荐用于只读场景）

右值引用

右值引用基础概念

概念：右值基础概念

  • 概念：为什么需要右值引用
    • 右值引用主要用于解决不必要的 Copy，当程序在装载对象的时候经常会制造出临时对象；
    • 而这是如果时 Copy 行为，因为要将这个对象完整复制一份，因此更加消耗性能。右值引用就是用于解决不必要的拷贝；
    • 即当右手边是右值时，左手边可以直接引用右边的值，而无需重新分配内存；
  • 概念：左值与右值
    • 左值与左值引用：
      • 左值就是变量，可以放在等于号的左右两边；
      • int a = 5;
      • int &ref_a = a; // 左值引用指向左值，编译通过
      • int &ref_a = 5; // 左值引用指向了右值，会编译失败
      • 引用是变量的别名，由于右值没有地址，没法被修改，所以左值引用无法指向右值
      • 但是，const 左值引用是可以指向右值：
        • const int &ref_a = 5; // 编译通过
      • const 左值引用不会修改指向值，因此可以指向右值，这也是为什么要使用 const & 作为函数参数的原因之一，如 std::vector 的 push_back：
        • void push_back (const value_type& val);
      • 如果没有 const，vec.push_back(5) 这样的代码就无法编译通过了；
    • 右值与右值引用：
      • 右值就是内容本身，只可以放在等于号的右边；
      • 比如：临时对象 string() = "World"; 此代码创建了一个 String 类型的临时对象；
  • 举例：
    • 在下面的例子中，foo 返回的内容为右值，无法直接对其取地址；
    • 在 int x = foo() 的时候，直接将“5”这个 int 类型的对象进行了复制，然后赋值到了 x 变量上；
#include <iostream>
using namespace std;

int main() 
{
    int&& s1 = 123; // 核心代码
    cout << "s1的值: " << s1 << endl;       // 输出：123
    cout << "s1的地址: " << &s1 << endl;    // 输出：s1有内存地址（证明是对象）
    s1 = 456;                               // 可以修改（右值引用是可修改的）
    cout << "修改后s1的值: " << s1 << endl; // 输出：456
    return 0;
}
  • 分析：
    • 123 是纯右值（没有内存地址），编译器会先创建一个临时的 int 对象（比如命名为 temp），值为 123，这个临时对象是 “将亡值（xvalue）”（属于右值的一种）
    • 右值引用 s1 绑定到这个临时对象 temp 上。根据 C++ 标准规则：绑定到右值引用的临时对象，其生命周期会被延长至与引用变量（s1）的生命周期一致，直到 s 1 出作用域销毁；
    • 此时 s1 不再是 “纯右值”，而是一个有名字的右值引用（本质是左值！因为它有内存地址、可被取址），可以被修改（如 s1=456）
  • 补充：
    • 对于 int 这类内置类型（char、float、bool 等），右值引用没有 “性能收益”；
    • 因为 int 的大小只有 4 字节，“移动” 和 “拷贝” 都是直接复制值，没有资源（如堆内存、文件句柄）可转移。移动语义的价值体现在带资源的自定义类型（如 std::string、std::vector）上；

右值引用语法

概念：右值引用的使用

  • 概念：
    • 右值引用的意义，就是右值引用可以使我们对右值取地址；
    • 右值引用专门为右值而生，可以指向右值，不能指向左值；
  • 语法：
    • &&
  • 注意：
    • 执行指针的浅拷贝，上一根指针不可再被使用；
  • 代码：
#include <iostream>
#include <vector>
#include <string>

using namespace std;

int main() 
{
    vector<string> vec;
    vec.reserve(3); // 预分配空间，避免扩容

    // 1. 拷贝插入：创建临时string，拷贝到vec
    string s1 = "copy";
    vec.push_back(s1);
    cout << "s1（拷贝后）: " << s1 << endl; // 输出：copy

    // 2. 移动插入：将临时string（右值）移动到vec
    string s2 = "move";
    // 原本的 s2 是左值，但是通过 std::move 将其变成右值，并且push_back这些STL标准库的容器都提供了移动语义的接口，因此直接将 move(s2) 得到的右值引用的所有权移动到 vec 身上
    // 因此 s2 就失去了拥有权
    vec.push_back(move(s2));
    cout << "s2（移动后）: " << s2 << endl; // 输出：空

    // 3. emplace_back 直接构造（本质也是利用移动/构造优化）
    vec.emplace_back("emplace"); // 直接在vec的内存中构造string，无拷贝/移动

    cout << "vec的元素: ";
    for (const auto& s : vec) cout << s << " "; // 输出：copy move emplace
    cout << endl;

    return 0;
}

分析：右值引用的应用

  • 概念：
    • STL 当中的 insert() 函数；
  • 分析：
    • 当使用 insert(&x) 时，其本质上是进行的赋值；
    • 当使用 insert(&&x) 时，其会有两种可能性：
      • 可能性一：传入右值
        • 此时将会调用执行传进来的对象“x”对应的 ctor 函数（右值引用的版本），通过构造函数获得对象浅拷贝的结果；
        • 本质上就是直接把对象的指针直接偷过来，之后不可再使用其；
      • 可能性二：传入左值
        • 传入的值为左值时，执行第一个重载

示例：右值引用与 struct

概念：右值引用与 struct

  • 示例：
#include <iostream>
using namespace std;

struct List 
{
    int data[1000];
    int len;
    List() : len(0) {
        cout << "默认构造：分配栈上数组" << endl;
    }
    
    // 默认拷贝构造（编译器自动生成）：逐字节拷贝data[1000] + len
    List(const List& other) 
    {
        // 手动实现默认拷贝逻辑，便于观察
        for (int i = 0; i < 1000; i++) 
        {
            data[i] = other.data[i];
        }
        
        len = other.len;
        cout << "拷贝构造：拷贝1000个int（开销大）" << endl;
    }
};

int main() 
{
    // 1. 创建临时List对象（右值）
    List tempList;
    tempList.data[0] = 100;
    tempList.len = 1;

    // 2. 右值引用绑定“即将销毁的右值”（用std::move将左值转为右值）
    List&& rvalueRef = std::move(tempList); 
    cout << "右值引用访问：" << rvalueRef.data[0] << endl; // 输出100

    // 3. 用右值引用初始化新对象：依然触发拷贝构造（无移动语义）
    List newList = std::move(rvalueRef); 
    return 0;
}
  • 分析：
    • data[1000] 是栈上数组，栈内存的生命周期由作用域决定（随函数 / 对象销毁自动释放），无法转移所有权。即使绑定了右值引用，创建新对象时依然要完整拷贝 1000 个 int，移动语义在此场景下 “无效”；
    • 若未自定义移动构造函数，编译器会优先调用拷贝构造函数处理右值（因为拷贝构造的参数 const T& 可以绑定右值），导致即使是临时对象，也会触发高开销的拷贝；
  • 示例：改造 List
struct List 
{
    int* data; // 堆上动态数组
    int len;
    int capacity; // 数组容量

    // 1. 默认构造：分配堆上数组
    List(int cap = 1000) : len(0), capacity(cap) {
        data = new int[capacity]; // 堆上分配1000个int
        cout << "默认构造：分配堆上数组（地址：" << (void*)data << "）" << endl;
    }

    // 2. 拷贝构造：深拷贝（完整复制堆上数组，开销大）
    List(const List& other) 
    {
        len = other.len;
        capacity = other.capacity;
        data = new int[capacity]; // 重新分配堆内存
        for (int i = 0; i < len; i++) {
            data[i] = other.data[i]; // 逐元素拷贝
        }
        cout << "拷贝构造：深拷贝堆数组（地址：" << (void*)data << "）" << endl;
    }

    // 3. 析构函数：释放堆内存（避免泄漏）
    ~List() {
        if (data != nullptr) {
            delete[] data;
            cout << "析构：释放堆数组（地址：" << (void*)data << "）" << endl;
            data = nullptr;
        }
    }

    // 4. 移动构造函数（核心：右值引用参数 + 转移资源）
    List(List&& other) noexcept // noexcept：告诉编译器无异常，容器会优先调用 
    { 
        // 转移资源所有权（仅拷贝指针，无数组拷贝）
        data = other.data;
        len = other.len;
        capacity = other.capacity;

        // 置空源对象的指针（避免析构时重复释放）
        other.data = nullptr;
        other.len = 0;
        other.capacity = 0;

        cout << "移动构造：转移堆数组（地址：" << (void*)data << "）" << endl;
    }

    // 5. 移动赋值运算符（处理a = std::move(b)场景）
    List& operator=(List&& other) noexcept {
        if (this == &other) 
        { // 防止自赋值
            return *this;
        }

        // 释放当前对象的堆资源
        if (data != nullptr) 
        {
            delete[] data;
        }

        // 转移源对象的资源
        data = other.data;
        len = other.len;
        capacity = other.capacity;

        // 置空源对象
        other.data = nullptr;
        other.len = 0;
        other.capacity = 0;

        cout << "移动赋值：转移堆数组（地址：" << (void*)data << "）" << endl;
        return *this;
    }

    // 禁用拷贝赋值（可选，突出移动语义）
    List& operator=(const List& other) = delete;
};

int main() {
    // 1. 创建临时List对象（右值）
    List tempList;
    tempList.data[0] = 100;
    tempList.len = 1;

    // 2. 用std::move将左值转为右值，触发移动构造
    List newList = std::move(tempList); 

    // 3. 验证：tempList的资源已被转移（指针为空）
    cout << "tempList.data是否为空：" << (tempList.data == nullptr ? "是" : "否") << endl;
    cout << "newList.data[0]：" << newList.data[0] << endl;

    return 0;
}

移动语义

左值和右值

概念：左值和右值

  • 总结：
值类别 定义 例子 能否被移动
左值 能取地址、生命周期长的对象（有名字） int a = 10; 中的a；List list; 中的list 可通过std::move转为右值后移动
右值 不能取地址、生命周期极短的对象（无名字） 临时对象List()；函数返回的临时对象createBigList()；字面量10 天然可被移动

移动语义基本概念

概念：移动语义

  • 引入：
    • C++11 之前，对象的 “复制” 只有拷贝语义：当我们用一个对象初始化另一个对象（如 List a = b;），或给对象赋值（如 a = b;）时，编译器会调用拷贝构造函数 / 拷贝赋值运算符，对对象的所有资源（尤其是堆上资源，如动态数组、文件句柄、内存缓冲区）做 “深拷贝”—— 即重新分配一块内存，再把原对象的内容逐字节复制过去；
    • 临时对象的资源 “拷贝完就扔”，完全没必要：移动语义的核心目标就是解决这种 “无意义拷贝” 的性能问题；
  • 概念：
    • 将一个对象（通常是生命周期极短的右值）的「资源所有权」直接转移给另一个对象，而非复制资源本身；
    • 当传入的值为左值，但是你明确的知道此左值再未来不会被再进行操作，此时就可以在传入参数时用 std::move() 将其指明为右值进行传入，从而进行浅拷贝；
    • std::move（函数，<utility>）是将左值转为右值引用；std::move（算法，<algorithm>）是将一个范围的元素移动到另一个范围，避免拷贝元素；
  • 补充：
    • 很多人误以为 std::move 会 “移动资源”，这是典型误区；
    • std::move 的唯一作用是：将传入的参数（无论左值 / 右值）强制转换成对应的右值引用类型，它本身不执行任何内存操作、不转移任何资源
    •   // 模板版本（简化）
template <typename T>
typename remove_reference<T>::type&& move(T&& t) noexcept {
    // 核心：静态转换为右值引用
    return static_cast<typename remove_reference<T>::type&&>(t);
}
    • 它的本质是一个类型转换函数，没有任何逻辑（如内存拷贝、资源转移），只是改变了参数的 “值类别”（左值 → 右值引用）
  • 语法：
    • std::move()
  • 举例：
int a = 5; // a是个左值
int &ref_a_left = a; // 左值引用指向左值
int &&ref_a_right = std::move(a); // 通过std::move将左值转化为右值，可以被右值引用指向
 
cout << a; // 打印结果：5

代码示例

概念：移动语义在 STL 中的使用

  • 概念：
    • 移动语义（基于右值引用 T&&）的核心是资源转移而非拷贝，STL（标准模板库）中的容器、算法、工具函数都广泛实现了移动语义，目的是避免不必要的深拷贝，大幅提升性能；
    • 所有 STL 容器（如 vector、string、map 等）都默认实现了移动构造函数和移动赋值运算符；
  • 示例：STL 容器的移动构造与移动赋值
#include <iostream>
#include <vector>
#include <string>

using namespace std;

void printVector(const vector<string>& v, const string& name) {
    cout << name << " size: " << v.size() << ", data: ";
    for (const auto& s : v) cout << s << " ";
    cout << endl;
}

int main() {
    // 1. 构造一个包含大量数据的vector
    vector<string> vec1 = {"apple", "banana", "cherry", "date"};
    printVector(vec1, "vec1（移动前）");

    // 2. 移动构造 vec2：将vec1的资源的所有权转移给vec2
    vector<string> vec2(std::move(vec1));
    printVector(vec2, "vec2（移动构造后）");
    printVector(vec1, "vec1（被移动后）"); // 移动后vec1处于"有效但未指定"状态（通常为空）

    // 3. 移动赋值vec3
    vector<string> vec3;
    vec3 = move(vec2);
    printVector(vec3, "vec3（移动赋值后）");
    printVector(vec2, "vec2（移动后）");

    return 0;
}

// 结果：
vec1（移动前） size: 4, data: apple banana cherry date 
vec2（移动构造后） size: 4, data: apple banana cherry date 
vec1（移动后） size: 0, data: 
vec3（移动赋值后） size: 4, data: apple banana cherry date 
vec2（移动后） size: 0, data:
  • 分析：
    • vector<string> vec2(move(vec1))：调用 vector 的移动构造函数，仅转移 vec1 的内存指针（而非拷贝字符串数据），时间复杂度
    • 移动后原对象（vec1）的状态：C++ 标准规定其处于 “有效但未指定” 状态（通常为空），不能再依赖其原有数据，但可以安全地调用析构函数或重新赋值；
  • 示例：函数返回 STL 容器时的移动语义
#include <iostream>
#include <vector>

using namespace std;

// 函数返回一个大vector
vector<int> createBigVector() {
    vector<int> vec(1000000, 1); // 100万个元素
    cout << "函数内vec的地址: " << &vec[0] << endl;
    return move(vec); // 显式移动（也可省略，编译器会自动优化）
}

int main() 
{
    vector<int> result = createBigVector();
    cout << "main中result的地址: " << &result[0] << endl;
    cout << "result大小: " << result.size() << endl;
    return 0;
}

// 结果：
函数内vec的地址: 0x55f8a7a2a010
main中result的地址: 0x55f8a7a2a010
result大小: 1000000
  • 分析：
    • 返回的 vec 是局部对象（右值），编译器会自动调用移动构造函数，将 vec 的内存缓冲区直接转移给 result，而非拷贝 100 万个 int；
  • 示例：移动元素
#include <iostream>
#include <vector>
#include <string>
#include <algorithm> // std::move算法的头文件

using namespace std;

int main() {
    vector<string> src = {"hello", "world", "move", "semantics"};
    vector<string> dest(src.size()); // 预分配空间

    // 使用std::move算法，将src的元素移动到dest
    move(src.begin(), src.end(), dest.begin());

    cout << "dest的元素: ";
    for (const auto& s : dest) cout << s << " "; // 输出：hello world move semantics
    cout << endl;

    cout << "src的元素（移动后）: ";
    for (const auto& s : src) cout << "[" << s << "] "; 
    // 输出：[] [] [] []（string移动后变为空）
    cout << endl;

    return 0;
}

Forwarding

概念：转交

  • 概念：不完美转交
    • 当一个函数在参数中接受右值时，会将此右值当成变量成为另一个函数的参数；
    • 传入到此函数当中的另一个函数时，其作为参数传入进来的值在使用时就又变为了左值。此时就是不完美转交；
  • 概念：完美转交
    • std::forward<T1>(t1)
  • 举例：
template<typename T1,typename T2>
void functionA(T1&& t1,T2&& t2)
{
	functionB(std::forward<T1>(t1),std::forward<T2>(t2));
}
