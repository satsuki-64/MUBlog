PBR 与 PBR 材质

概念：PBR 与 PBR 材质

  • 概念：
    • 为了进行真实的渲染，因此需要满足一定的光线传播的物理规律，因此需要 PBR 的材质来提供这些信息；
    • PBR 因此不只是限制在渲染这一部分，也可以使用在其他的领域。在实时渲染当中，材质的丰富程度要落后于离线渲染，并且准确度和质量也要差于离线渲染；

    • 在实时渲染当中的 PBR 基本都不是完全基于物理的，因为实时渲染当中做了大量的简化，因此不能完全说是基于物理的；

G：阴影-遮蔽项

概念：阴影-遮蔽项

  • 概念：
    • 阴影-遮蔽项（Shadowing-Masking Term）​，也称为几何衰减项（Geometric Attenuation Term）​，用 G 表示；
    • 用于描述微表面之间由于不规则几何结构而相互产生阴影和遮蔽，从而导致光能损失的比例；
    • （1）遮蔽：当入射光 i ​ 照射到表面时，一个微表面的突起可能会挡住光线，使其无法到达旁边的“山谷”，这部分被挡住的光线就发生了“遮蔽”；
    • （2）阴影​​：当出射光 o ​ 从表面反射向眼睛时，一个微表面的突起可能会在旁边“山谷”的反射光路径上投下阴影，使得这部分反射光无法进入眼睛，这就发生了“阴影”；
    • G 项就是一个介于 [0.0, 1.0] 之间的系数，它计算的是未被遮蔽的入射光和未被阴影遮挡的出射光占总光量的百分比。G = 1.0 表示完全没有能量损失；G = 0.0 表示光线被完全阻挡；
  • 作用：
    • 微表面不是绝对光滑的，光线在微观层面上必然会产生遮挡。如果没有 G 项，BRDF 模型就无法真实地模拟这种能量损失，会导致渲染结果过亮，尤其是在掠射角（Grazing Angle）​​ 观察时，表面会显得不自然地明亮甚至发光。G 项确保了能量守恒；
    • 实际的渲染效果上看，在物体边缘等掠射角角度，G 项的值会变小，使得表面看起来更暗，这与现实世界的观察结果一致；
  • 原理：
    • G 项的原理是基于对微表面几何结构的统计模型。它不关心每一个微表面的具体形状，而是从宏观概率上计算光线被遮挡的可能性；
    • 常见的 G 项模型（如 Smith 模型、GGX 配套的 Smith-GGX 函数）会分别计算入射方向的遮蔽 G₁(i) 和出射方向的阴影 G₁(o)，最终的 G(i, o, h) 是它们的乘积：
      • G(i, o, h) = G₁(i) * G₁(o)
    • 其计算通常依赖于两个核心参数：
      • 粗糙度（Roughness）​​：表面越粗糙，微表面的起伏越大、越复杂，相互遮挡的可能性就越高，G 值就越小；
      • 角度（Angle）​​：光线与法线的夹角越大（越接近掠射角），微表面投射的阴影就越长，遮挡就越严重，G 值也越小；

D：微表面的法线分布

概念：微表面的法线分布

  • 概念：两类表面
    • 根据微表面的不同状态，会有不同的法线分布；
    • 比如：Glossy 物体的表面更类似于一个平面，因此它们的法线角度更加的类似，并且接近和宏观的表面垂直，因此其反射的范围更集中；
    • 比如：Diffuse 物体表面凹凸不同，因此法线混乱，进而反射角度四面八方，表面粗糙、因此反射的范围更大；
    • 在 Diffuse 类似于是将 Glossy 的表面上下拉伸；
    • 即：法线分布越集中，其效果越 Glossy，而法线分布越分散，其效果越 Diffuse；

GGX 模型

概念：GGX 模型

  • 概念：
    • GGX 的法线函数具有 Long Tail 的特点；
    • 比如如图所示的函数，GGX 函数具有长尾的性质，其不像 Beckmann 函数、很快就会衰减为 0，而 GGX 在函数值变得很大之后、其依然具有一定的值；
    • 即 GGX 边缘会非常缓慢的减少值，比如对于 Beckmann 的高光效果而言，其只在反射的部分角度具有高光，其他角度很快救会衰减为黑色区域。而 GGX 则不会；
  • 原理：
    • n ​是宏观表面法线；
    • m ​是微表面法线，通常用半程向量 h 代替，因为只有 m = h 的微表面才能将光从 l 反射到 v；
    • ​α ​ 是粗糙度参数（Roughness），通常传入的是 α = roughness²，这样做是为了让艺术家在 0-1 范围内调整粗糙度时，视觉变化更线性、更直观；

代码：GGX 的实现

  • 代码：GGX 核心代码
// 此代码直接对应了GGX的数学公式
float DistributionGGX(float NdotH, float roughness)
{
    float a = roughness * roughness; // α²
    float a2 = a * a;                // α⁴
    float NdotH2 = NdotH * NdotH;    // (n·h)²

    float denom = (NdotH2 * (a2 - 1.0) + 1.0);
    denom = UNITY_PI * denom * denom; // π * denom²

    return a2 / max(denom, 0.001); // 避免除以0
}

Shadings-Masking Term

基础概念

概念：Shadings-Masking Term

  • 目的：
    • Shadings-Masking Term 用于解决微表面之间产生的自遮挡问题；
    • Shadings-Masking Term 可以在当出现 Grazing Angle 时，避免边缘变得非常非常的亮（球体时），进而符合真实的物理效果；
  • 分析：什么是 Shadings-Masking Term
    • 分析：
        1. 从 light 出发，会出现被遮挡到的区域，其称之为 Shadings；
        1. 从眼睛出发，看不到一些微表面，这种现象称之为 Masking；
      • 由于这些自遮挡的问题，会导致最终计算出来的光照效果肯定没那么暗，因此需要一项来表示这种变暗的效果，比如说在 Grazing Angle 时、让其变暗很多；
      • 因此 Shadings-Masking Term 应当有的效果是：当光线和法线角度接近时，让 Shadings-Masking Term 接近于 1，即不做额外处理。而在靠近 Grazing Angle 时，Shadings-Masking Term 应当剧烈减少，一直直到减少为 0；

LTC 方法基础概念

概念：LTC - Linearly Transformed Cosines

  • 引入：
    • 线性变换的余弦 -> 用于做微表面模型的 Shading；
    • 其考虑的是 Shading，因此并不考虑物体的 Shadow；
    • 其需要考虑在多边形的光源、在一个微表面的模型下、无阴影的 Shading 的效果；
  • 概念：Lobe
    • 给定光照的方向后，其反射是一个类似于“花瓣”的二维的范围区域，其称之为 Lobe；
  • 概念：LTC
    • Heitz 等于 SIGGRAPH 2016 提出的 Linearly Transformed Cosines 方法，实现了对多边形光源光照的实时高质量渲染。Unity SRP 中便应用了这一方法实现面积光源的效果；
    • LTC（Linearly Transformed Cosines，线性变换余弦）​是一种用于高效计算多边形光源着色的数学方法，其核心思想是通过线性变换将复杂的BRDF​与光源的积分问题，转换为在变换后的余弦分布下的简化计算，从而避免昂贵的数值积分；

LTC 方法实现

概念：LTC 方法概念

  • 概念：
    • 多边形光源的着色需要求解光照辐射度积分​（即 BRDF 与光源的乘积积分），传统方法（如蒙特卡洛积分）计算成本高。LTC 提出用线性变换将 BRDF 的分布转换为标准的余弦分布​，从而将积分转化为解析解；
    • LTC 假设 BRDF 的分布可以通过一个 3×3 线性变换矩阵 M 作用到原始光线方向（ω）上，将其映射到变换后的方向（ω' = Mω）;
    • 多边形光源的几何形状（如顶点位置）通过矩阵 M 的逆（M⁻¹）变换到新的空间，此时积分被转换为对变换后多边形的余弦分布积分, 如图所示的效果；
    • 这样进行操作的目的，是因为在变换后的空间中，余弦分布对多边形的积分有解析解​（如通过 Stokes 定理或几何公式计算），无需数值积分；
  • 总结：
    • LTC 的思想，大概就是通过一个变换矩阵，把不规则的图形变换成规则的图形，便于进行积分求解；
  • 分析：
    • LTC 利用了线性变换的保形性，通过矩阵 M，BRDF 的“高光波瓣”被拉伸或压缩成一个接近球对称的分布（类似余弦分布），而多边形光源的形状也同步变换，使得积分域与分布对齐；
    • 而由于 余弦分布（Lambertian）的积分有闭式解，而 LTC 通过变换将复杂 BRDF“伪装”成余弦分布，从而复用其解析解；

迪士尼原则的 BRDF

引入

引入：为什么需要迪士尼原则的 BRDF

  • 现在的微表面模型确实很强，但是并不适合表示真实的材质；
  • 比如：测量出来的物体表面的 BRDF，其找不到合适的微表面模型的参数来表示它；
    • 比如：一个刷了绿色油漆的木头，其不吸收光，但是因为光线为了进入油漆，就会形成一个高光，并且还会有一些光反射出来，因此会产生一些 Diffuse 的效果；
    • 而这个效果微表面模型不好表现，微表面模型只能表示一层的效果；
  • 并且：微表面模型不好用。对于艺术家而言，要选择各种各样的材质而言，由于是 PBR，因此其参数都是一些物理量，而这对于艺术家而言不友好；
    • 比如：对于金属材质而言，其金属度是一个复数 n-ik，而这个东西对于艺术家而言进行调整而言无法调整；
    • 而为艺术家提供合适的参数进行更快、更好的调整是一个很有意义的事情；
  • 因此还需要一类“艺术家友好”的材质，可以用艺术家们使用起来方便，而并不要求在物理上正确；

迪士尼设计原则

概念：迪士尼设计原则

    1. 看起来直观，易于使用；
    1. 不希望这个框架变得很复杂；
    1. 对于艺术家而言，设计出来的参数应当是以可视化的方式进行调整；
    1. 为了营造特殊效果，可以将属性设置为大于 1 或者小于 0；
    1. 不能因为组合效果而导致程序出现 Bug，而是得到不错的结果；

具体效果

概念：迪士尼原则下的 BRDF 效果

  • 概念：
    • Subsurface：次表面散射 -> 类似于球被按扁的效果；
    • Specular：控制有多少镜面反射的效果；
    • Specular Tint：镜面反射反映出来的部分的颜色，是更偏白还是更偏其底下物体的颜色；
    • Roughness：粗糙程度；
    • Anisotropic：各向异性程度；
    • Sheen：在表面上有一层体积的效果，在 Grazing Angle 上有类似于雾化的效果；
    • Sheen Tint：在 Sheen 效果的位置上的颜色；
    • ClearCoat：镀膜；
    • ClearcoatGloss：光滑程度；
  • 注意：
    • 参数空间越大，越容易产生冗余 -> 对于不同参数的组合，可能造成和不调整之前的效果 -> 在各个不同的 Local 的范围内已经能得到类似的效果；
  • 特点：
    • 优点：
      • 艺术家友好，可视化、能看懂；
      • 能够以一套模型、实现很丰富的效果；
      • 开源实现；
      • 巨大的参数空间 -> 表现能力强；
    • 缺点：
      • 不是基于物理的实现；
      • 巨大的参数空间 -> 很多的冗余现象；
  • 补充：
    • 迪士尼原则中的 BRDF 模型是拟合的能量守恒的模型；
