OpenGL

基础概念

概念：OpenGL

  • 概念：
    • Is a set of APls that call the GPU pipeline from CPU
    • 从 CPU 发出的对 GPU 渲染管线进行操作的 API；
    • 其特点就是跨平台；
  • 实现：
      1. 第一步：确定要渲染的物体
      • 在 OpenGL 中，​VBO（Vertex Buffer Object，顶点缓冲对象）​是一种高效管理顶点数据，如位置、颜色、法线、纹理坐标等的机制；
      • 它通过将数据存储在显存（GPU 内存）中，显著提升了渲染性能；
      1. 第二步：放置画架
      • （1）视图变换；
        • 比如 MVP 变换；
        • 只需要在 OpenGL 中定义这些变换的属性即可，不需要自己写；
      • （2）创建 framebuffer；
      1. 第三步：确定画架
      • 指定了一个 framebuffer 后，在一个 framebuffer 上可以绘制很多图；
      • 渲染一次场景，可以得到很多的东西；
      1. 第四步：渲染
      • 指定顶点着色器和片段着色器；
      • 在顶点着色器中定义对每一个顶点的操作；
      • 在片段着色器中对每一个片段进行着色；
      1. 第五步：将三角形中的每个片段进行并行处理并进行深度测试
      • OpenGL calls user-specified fragment shader:Shading and lighting calculations
      • OpenGL handles z-buffer depth test unless overwritten

VBO

概念：VBO

  • 概念：
    • 数据存储位置：VBO 将原本由 CPU 维护的顶点数据（如数组）转移到 GPU 的显存中，避免了每次绘制时从 CPU 到 GPU 的数据传输开销；
    • ​缓冲类型：属于 OpenGL 的缓冲对象（Buffer Object）之一，绑定到 GL_ARRAY_BUFFER 目标后，专门用于存储顶点属性数据；
  • 作用：
      1. 静态数据（如模型顶点）只需一次性上传到 VBO，后续绘制时 GPU 直接读取显存数据，减少 CPU-GPU 通信延迟；
      1. 支持动态更新数据（如粒子系统），通过 glBufferSubData 可修改部分内容，无需重新分配整个缓冲；
      1. 将多个对象的顶点数据合并到单个 VBO 中，通过一次绘制调用（如 glDrawArrays）渲染大量几何体，降低 API 调用开销；

GLSL

概念：GLSL

  • 概念：
    • GLSL​（OpenGL Shading Language）是 OpenGL 的着色器编程语言，用于在 GPU 上执行可编程渲染管线的各个阶段（如顶点处理、片段处理等）；
    • 它类似于 C 语言，但专为图形计算优化，支持向量和矩阵运算，能够高效处理顶点、光照、纹理等图形数据；
  • 概念：着色器设置
      1. 创建 Shader；
      1. 编译着色器；
      1. 设置着色器到程序；
      1. 链接程序；
      1. 使用程序；
  • 概念：顶点着色器
    • 作用：
      • 处理每个顶点的位置、法线、纹理坐标等属性；
    • 示例：
#version 330 core
layout (location = 0) in vec3 aPos;  // 输入顶点位置
uniform mat4 model;                 // 模型变换矩阵
uniform mat4 view;                  // 观察矩阵
uniform mat4 projection;            // 投影矩阵

void main() 
{
    gl_Position = projection * view * model * vec4(aPos, 1.0);
}
  • 概念：片段着色器
    • 作用：计算每个像素（片段）的最终颜色
    • 示例：
#version 330 core
out vec4 FragColor;                // 输出颜色
uniform vec3 objectColor;          // 物体颜色（由CPU传入）

void main() 
{
    FragColor = vec4(objectColor, 1.0);
}

/**
 * 初始化并编译GLSL着色器
 * @param type 着色器类型（GL_VERTEX_SHADER / GL_FRAGMENT_SHADER）
 * @param filename 着色器源代码文件路径
 * @return 编译成功的着色器对象ID
 * @throws 编译失败时抛出异常
 */
GLuint initshaders(GLenum type, const char *filename) {
    // 1. 创建着色器对象
    // glCreateShader()创建一个指定类型的着色器对象（顶点/片段/几何着色器等）
    // 返回的shader是一个OpenGL对象标识符
    GLuint shader = glCreateShader(type);

    // 2. 读取着色器源代码文件
    // 使用辅助函数textFileRead()将文件内容读入字符串
    // 注意：textFileRead()需要自行实现（示例见下方补充代码）
    std::string str = textFileRead(filename);

    // 3. 准备传递给OpenGL的源代码格式
    // 分配足够存储文件内容+终止符的内存空间
    GLchar* cstr = new GLchar[str.size() + 1];
    
    // 保留原始指针的const副本（OpenGL要求const指针）
    // 注意：此处原代码的注释"weirdness"可能间接转换方式
    const GLchar* cstr2 = cstr;
    
    // 将C++ string内容复制到C风格字符串
    strcpy(cstr, str.c_str());

    // 4. 设置着色器源代码
    // 参数说明：
    // - shader: 目标着色器对象
    // - 1: 源代码字符串数量
    // - &cstr2: 源代码字符串数组指针
    // - NULL: 字符串长度数组（NULL表示字符串以null结尾）
    glShaderSource(shader, 1, &cstr2, NULL);
    
    // 释放临时内存（原代码遗漏了这部分，可能造成内存泄漏）
    delete[] cstr;

    // 5. 编译着色器
    glCompileShader(shader);

    // 6. 检查编译状态
    GLint compiled;
    glGetShaderiv(shader, GL_COMPILE_STATUS, &compiled);

    // 7. 处理编译错误
    if (!compiled) {
        // 调用错误处理函数（示例实现见下方补充代码）
        shadererrors(shader);
        // 抛出异常（原代码使用魔数3，建议改用标准异常类型）
        throw std::runtime_error("Shader compilation failed");
    }

    return shader;
}

std::string textFileRead(const char* filename) {
    std::ifstream file(filename);
    if (!file.is_open()) {
        throw std::runtime_error(std::string("Failed to open file: ") + filename);
    }
    return std::string(
        (std::istreambuf_iterator<char>(file)),
        std::istreambuf_iterator<char>()
    );
}

链接 Shader 代码

代码

/**
 * 初始化并链接OpenGL着色器程序
 * @param vertexshader 已编译的顶点着色器对象ID
 * @param fragmentshader 已编译的片段着色器对象ID
 * @return 链接成功的程序对象ID
 * @throws 链接失败时抛出异常（实际项目建议使用标准异常类型）
 */
GLuint initprogram(GLuint vertexshader, GLuint fragmentshader) {
    // 1. 创建空的OpenGL程序对象
    // glCreateProgram()创建一个新的程序对象，返回其唯一标识符
    // 程序对象用于管理多个着色器（如顶点+片段着色器）的链接
    GLuint program = glCreateProgram();
    
    // 2. 存储链接状态的变量
    // GLint是OpenGL定义的整型，用于接收链接状态结果（GL_TRUE/GL_FALSE）
    GLint linked;

    // 3. 附加顶点着色器到程序
    // glAttachShader()将编译好的着色器附加到程序对象
    // 参数说明：
    // - program: 目标程序对象
    // - vertexshader: 已编译的顶点着色器对象
    glAttachShader(program, vertexshader);

    // 4. 附加片段着色器到程序
    // 同上，附加片段着色器
    glAttachShader(program, fragmentshader);

    // 5. 链接程序对象
    // glLinkProgram()将所有附加的着色器链接成可执行程序
    // 链接过程会检查着色器之间的兼容性（如变量匹配）
    glLinkProgram(program);

    // 6. 获取链接状态
    // glGetProgramiv()查询程序对象的状态信息
    // 参数说明：
    // - program: 目标程序对象
    // - GL_LINK_STATUS: 查询链接状态
    // - &linked: 存储查询结果的指针
    glGetProgramiv(program, GL_LINK_STATUS, &linked);

    // 7. 处理链接结果
    if (linked) {
        // 7.1 链接成功时激活该程序
        // glUseProgram()将程序对象设置为当前渲染状态的一部分
        // 之后的所有渲染调用都会使用这个程序
        glUseProgram(program);
    } else {
        // 7.2 链接失败时的处理
        // 调用错误报告函数（需自行实现，示例见下方补充代码）
        programerrors(program);
        
        // 抛出异常（原代码使用魔数4，建议改用标准异常）
        throw std::runtime_error("Shader program linking failed");
    }

    // 8. 返回成功链接的程序对象
    return program;
}

/**
 * 输出程序链接错误信息（辅助函数）
 * @param program 链接失败的程序对象ID
 */
void programerrors(GLuint program) {
    GLint length;
    // 获取错误日志长度
    glGetProgramiv(program, GL_INFO_LOG_LENGTH, &length);
    
    // 分配存储日志的内存
    GLchar* log = new GLchar[length];
    // 获取实际错误信息
    glGetProgramInfoLog(program, length, NULL, log);
    
    // 输出到标准错误流
    std::cerr << "Program linking error:\n" << log << std::endl;
    
    // 释放内存
    delete[] log;
}

顶点着色器

概念：顶点着色器

  • 代码：
// 顶点着色器代码（Vertex Shader）
#version 300 es  // 声明使用OpenGL ES 3.0版本

// ========== 输入属性（从CPU传递的逐顶点数据） ==========
// attribute表示每个顶点独有的属性数据
attribute vec3 aVertexPosition;  // 顶点位置（x,y,z）
attribute vec3 aNormalPosition;  // 顶点法向量（用于光照计算）
attribute vec2 aTextureCoord;    // 纹理坐标（u,v）

// ========== 统一变量（全局常量） ==========
// uniform表示所有顶点共享的全局数据
uniform mat4 uModelViewMatrix;   // 模型视图矩阵（物体空间->相机空间）
uniform mat4 uProjectionMatrix;  // 投影矩阵（相机空间->裁剪空间）

// ========== 输出变量（传递给片段着色器） ==========
// varying（或GLSL 3.0+的out）表示需要插值后传递给片段着色器的数据
out highp vec2 vTextureCoord;    // 插值后的纹理坐标（highp表示高精度）
out highp vec3 vFragPos;         // 片段的世界空间位置
out highp vec3 vNormal;         // 插值后的法向量

// ========== 主函数 ==========
void main(void) {
    // 将顶点位置传递给片段着色器（用于后续光照计算）
    vFragPos = aVertexPosition;
    
    // 将法向量传递给片段着色器（注意：需要转换为世界空间）
    vNormal = aNormalPosition;
    
    // 计算最终顶点位置（MVP矩阵变换）：
    // 1. 将顶点从模型空间转换到相机空间（uModelViewMatrix）
    // 2. 再从相机空间转换到裁剪空间（uProjectionMatrix）
    // 3. 输出到内置变量gl_Position（必须赋值）
    gl_Position = uProjectionMatrix * uModelViewMatrix * vec4(aVertexPosition, 1.0);
    
    // 传递纹理坐标给片段着色器（将自动在三角形内插值）
    vTextureCoord = aTextureCoord;
}
  • 概念：attribute
    • 作用：定义每个顶点独有的输入数据（如位置、法线、UV 坐标等）
    • 来源：由 CPU 通过 glVertexAttribPointer() 上传到 GPU；
    • 特点：
        1. 每个顶点独立使用；
        1. 只能在顶点着色器中使用；
        1. GLSL 3.0+ 版本中改用 in 关键字；
  • 概念：uniform
    • 作用：定义全局常量，所有顶点/片段共享同一值（如变换矩阵、光照参数）
    • 来源：通过 glUniformXXX() 系列函数从 CPU 设置；
    • 特点：
        1. 适合存储不频繁变化的数据（如相机矩阵）；
        1. 可在所有着色器阶段访问；
  • 概念：varying
    • 作用：在顶点着色器和片段着色器之间传递插值数据；
    • 原理：顶点着色器输出值会在图元（如三角形）内自动插值，片段着色器接收插值后的结果；
    • 补充：
        1. GLSL 1.0-3.0：使用 varying 关键字；
        1. GLSL 3.0+：顶点着色器用 out，片段着色器用 in；

片段着色器

概念：片段着色器

  • 代码：
// 片段着色器代码（Fragment Shader）
#version 300 es  // 声明使用OpenGL ES 3.0版本
precision highp float; // 设置默认浮点数精度

// ========== Uniform输入（全局常量） ==========
uniform sampler2D uSampler;    // 2D纹理采样器（用于贴图颜色采样）
uniform vec3 uKd;             // 漫反射系数（Diffuse Color）
uniform vec3 uKs;             // 镜面反射系数（Specular Color）
uniform vec3 uLightPos;       // 光源位置（世界空间坐标）
uniform vec3 uCameraPos;      // 相机位置（世界空间坐标）
uniform float uLightIntensity; // 光照强度乘数
uniform int uTextureSample;   // 纹理使用标志（0=禁用，1=启用）

// ========== Varying输入（从顶点着色器插值传入） ==========
in highp vec2 vTextureCoord;  // 插值后的纹理坐标（UV）
in highp vec3 vFragPos;       // 片段的世界空间位置
in highp vec3 vNormal;        // 插值后的法向量（需归一化）

// ========== 输出变量 ==========
out vec4 FragColor;           // 最终输出的片段颜色（RGBA）

void main() {
    // 1. 初始化基础颜色
    vec3 baseColor;
    
    // 2. 判断是否使用纹理采样
    if (uTextureSample == 1) {
        // 2.1 从纹理采样颜色（texture2D是GLSL ES 3.0的采样函数）
        // pow(..., 2.2) 执行gamma校正（假设纹理是sRGB空间）
        baseColor = pow(texture(uSampler, vTextureCoord).rgb, vec3(2.2));
    } else {
        // 2.2 直接使用漫反射系数作为颜色
        baseColor = u;
    }

    // 3. 光照计算（此处简化为仅使用基础颜色）
    // 实际项目应在此处添加：
    // - 法线归一化：vec3 N = normalize(vNormal);
    // - 光线方向计算：vec3 L = normalize(uLightPos - vFragPos);
    // - 视线方向计算：vec3 V = normalize(uCameraPos - vFragPos);
    // - 漫反射计算：float diff = max(dot(N, L), 0.0);
    // - 镜面反射计算（Phong/Blinn-Phong模型）...

    // 4. 应用光照强度（简化版）
    vec3 finalColor = baseColor * uLightIntensity;

    // 5. 输出最终颜色
    FragColor = vec4(finalColor, 1.0); // Alpha固定为1.0（不透明）
}

概念：BRDF

  • 概念：
    • BRDF​（Bidirectional Reflectance Distribution Function，双向反射分布函数）是计算机图形学和光学中用于描述物体表面如何反射光线的核心数学模型；
    • 它定义了光线从某个入射方向（​）照射到物体表面后，如何向特定出射方向（ωo ​）反射的比率；
    • BRDF 是渲染方程的关键组成部分，直接影响材质的外观表现，如金属、塑料、布料等；
    • 物理正确的 BRDF 必须满足能量守恒和互易性；
  • 定义：
    • BRDF 描述了单位入射光能量（​）在某个方向上反射的比例；

补充：渲染与蒙特卡洛积分

  • 概念：
    • 核心思想就是通过随机采样，近似计算定积分的数值方法；
    • 蒙特卡洛积分通过随机采样将渲染方程的复杂积分转化为可计算的求和问题，是路径追踪、光线追踪等现代渲染技术的数学基础；
    • 其核心优势在于高维适应性和对复杂光路的表达能力，尽管需要大量采样来降噪，但通过重要性采样等优化技术可显著提升效率；
  • 定义：
  • 关系：
    • 渲染方程需要计算半球域（Ω+）上的积分，因此可以使用蒙特卡洛积分方法；
